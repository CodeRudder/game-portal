## 5. 重复代码检测

### 5.1 权限检查逻辑重复 (P1)

`requirePermission(alliance, playerId, action)` 在两个文件中重复实现：

| 位置 | 文件 | 行号 | 说明 |
|------|------|:----:|------|
| 实现1 | AllianceHelper.ts | L24-36 | 独立纯函数，switch-case 4种action |
| 实现2 | AllianceSystem.ts | L288-300 | 私有方法，委托 AllianceHelper.requirePermission |

**说明**: AllianceSystem L288 已正确委托 AllianceHelper，但方法签名和 try-catch 包装仍存在冗余。AllianceSystem.hasPermission 同样委托 AllianceHelper.hasPermission。这不是代码重复问题，而是委托模式——AllianceSystem 暴露面向外部的 API，内部委托 Helper 实现。✅ 合理设计。

### 5.2 Boss数据不持久 (P1)

`AllianceBossSystem.getCurrentBoss(alliance)` 每次调用 `createBoss(alliance.level, alliance.lastBossRefreshTime)` 重建Boss，但：
- 不恢复 damageRecords（伤害记录丢失）
- 不恢复 currentHp（HP总是满的）
- 仅通过 alliance.bossKilledToday 判断是否已击杀

**建议**: Boss数据应持久化到 AllianceData 中（如 alliance.currentBoss 字段），或在 AllianceData 中增加 bossDamageRecords/bossCurrentHp 字段。

### 5.3 index.ts 重复别名导出 (P2)

```typescript
// AllianceSystem 已导出 DEFAULT_CREATE_CONFIG
export { DEFAULT_CREATE_CONFIG } from './AllianceSystem';
// alliance-constants 再次以别名导出
export { DEFAULT_CREATE_CONFIG as CREATE_CONFIG } from './alliance-constants';
```

两处导出同一符号（AllianceSystem 从 alliance-constants 导入后重新导出），index.ts 中存在重复导出。
