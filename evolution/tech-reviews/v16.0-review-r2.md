**DDD 违规: 1处(P2级)**

HeritageSimulation.ts 第28行:
```typescript
import { calcRebirthMultiplier } from '../prestige/RebirthSystem';
```
传承域直接引用了转生域(prestige)的函数。虽然两个域在业务上紧密关联(传承依赖转生倍率)，但违反了严格的DDD域隔离原则。

**建议**: 将 `calcRebirthMultiplier` 提取到 `core/prestige/` 或 `core/heritage/` 作为共享纯函数，消除引擎层跨域依赖。

## 5. 架构模式评估

### 5.1 回调模式 vs 直接依赖 (✅ 优秀)

HeritageSystem 通过 `setCallbacks()` 注入8个外部数据回调，完全解耦了武将/装备/资源/建筑等数据层依赖。这是比直接import更好的DDD实践。

### 5.2 系统-模块委托模式 (✅ 优秀)

HeritageSystem(有状态ISubsystem) 委托 HeritageSimulation(无状态纯函数)，职责清晰：
- System: 状态管理 + 生命周期 + 事件发射 + API门面
- Simulation: 纯计算逻辑 + 可独立测试

### 5.3 唯一跨域引用 (⚠️ P2)

HeritageSimulation → prestige/RebirthSystem.calcRebirthMultiplier 是唯一的引擎层跨域引用。
