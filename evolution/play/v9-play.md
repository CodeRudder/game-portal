## 玩家流程

### P1: 关闭游戏 → 重新打开 → 离线奖励弹窗

```
1. 玩家在主城界面，资源产出正常（grain=100/s, gold=20/s）
2. 玩家关闭浏览器标签页（触发 OfflineSnapshotSystem.createSnapshot）
3. 等待 10 分钟后重新打开游戏
4. 系统检测离线时长 > 5分钟（OFFLINE_POPUP_THRESHOLD=300s）
5. 弹出离线收益弹窗：
   - 离线时长: "10分钟"
   - 效率系数: 100%（0~2h 档位）
   - 各资源收益: grain +60,000 / gold +12,000
   - 来源占比: 资源产出 85% / 贸易 15%
6. 翻倍选项: [观看广告×2] [使用道具×2]
```

**验证点**: `shouldShowOfflinePopup(600) === true`

### P2: 离线 25 小时 — 6档衰减 + 封顶检测

```
1. 玩家下线 25 小时后回归
2. 系统计算分段收益:
   - 0~2h:  100% × 7,200s = 7,200 有效秒
   - 2~8h:   80% × 21,600s = 17,280 有效秒
   - 8~24h:  60% × 57,600s = 34,560 有效秒
   - 24~25h: 40% × 3,600s = 1,440 有效秒
   - 综合效率: (7200+17280+34560+1440)/90000 ≈ 67.2%
3. 弹窗显示: "25小时" / 效率 67% / 封顶: 否
4. 领取收益后，资源入账
```

**验证点**: `calculateSnapshot(90000, rates).overallEfficiency ≈ 0.672`

### P3: 离线 80 小时 — 封顶 72h

```
1. 玩家下线 80 小时后回归
2. 系统封顶至 72h（MAX_OFFLINE_SECONDS = 259,200s）
3. isCapped = true
4. 超出 72h 的 8 小时不产出任何收益
5. 弹窗显示: "已封顶（超过72h不再产出）"
```

**验证点**: `calculateSnapshot(288000, rates).isCapped === true`

### P4: 收益预估面板 — 交叉验证

```
1. 玩家打开"离线预估"面板
2. OfflineEstimateSystem 生成时间线:
   - 1h: 100%效率, grain=360,000
   - 2h: 100%效率, grain=720,000
   - 4h: 90%效率, grain=1,296,000
   - 8h: 80%效率, grain=2,304,000
   - 24h: 67%效率, grain=5,806,080
   - 48h: 52%效率, grain=9,011,520
   - 72h: 44%效率, grain=11,404,800
3. 推荐下线时长: 24h（效率>50%的最长时间）
4. 玩家实际下线 8h，回归后收益与预估 8h 数据交叉验证
5. 偏差 < 1%（仅取整差异）
```

**验证点**: `estimate(rates).recommendedHours === 24`

### P5: 双倍领取 + 资源溢出

```
1. 玩家离线 12h，grain 产出 432,000
2. 当前 grain=95,000, 上限=100,000（空间仅 5,000）
3. 玩家选择"观看广告×2"
4. 翻倍后 grain=864,000
5. 溢出检测: cappedEarned.grain=5,000, overflow.grain=859,000
6. 弹窗提示: "粮仓已满，溢出 859,000 粮食，请升级粮仓"
7. 玩家领取 5,000 grain
```

**验证点**: `applyCapAndOverflow(earned, current, caps).overflow.grain > 0`
