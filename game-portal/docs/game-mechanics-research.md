# 游戏机制研究报告

> 研究主题：经典城市建设游戏「龙之崛起」核心机制 & 放置/增量游戏设计最佳实践

---

## 一、龙之崛起（Emperor: Rise of the Middle Kingdom）核心玩法机制

### 1.1 城市建设与行人系统

龙之崛起是 Sierra 城市建设系列的最后一部作品，背景横跨中国古代 3000 年历史（从夏朝到宋金时期）。游戏的核心机制是**行人（Walker）系统**——每类建筑物会产生特定种类的行人，当行人经过房屋时，该房屋便获得了此类服务，从而具备升级的条件。

行人分为两类：**随机行人**和**目的行人**。目的行人有固定目的地（如配送员从仓库到磨坊），而随机行人则沿着道路随机游走（如巡视员、僧侣、道士）。玩家需要通过道路规划和路障放置来精确控制行人的行走路线，确保所有房屋都能获得足够的服务覆盖。这一系统是整个城市建设玩法的核心挑战，相比前作（如《法老王》《凯撒大帝》），龙之崛起简化了行人的移动步数和建筑密度要求，使得规则更易上手。

### 1.2 资源管理与生产链

游戏拥有复杂而精巧的**资源生产链系统**。基础资源包括粮食（稻田、渔场、猎场）、原材料（木材、石头、黏土、铜矿/铁矿）和加工品（青铜器/铁器、瓷器、丝绸、麻布、武器）。生产链呈现清晰的层级关系：如"黏土→瓷器"、"桑叶→丝绸"、"铜矿/铁矿→武器"。

食物系统尤为丰富，包括稻米、小麦、粟、大豆、鱼、肉等多种食物类型。平民住房只需一种食物即可维持，而精英住房则需要多种食物才能升级到更高等级。贸易系统允许玩家与其他 AI 城市建立进出口关系，通过贸易站交换物资。玩家还可以通过征服或外交手段与其他城市互动，被征服的城市需要每年上缴贡品。

### 1.3 宗教与英雄系统

龙之崛起的宗教系统是其最具特色的机制之一。游戏包含**四大宗教信仰**：祖先崇拜、道教、佛教和儒教。每种宗教都有对应的祭祀建筑和英雄人物：

- **祖先崇拜**：女娲、神农、伏羲——最为关键，因为只有祖先类英雄会因不满而降下天灾（洪水、地震、干旱）
- **道教**：灶君、关帝、西王母等
- **佛教**：达摩、观音等
- **儒教**：孔子、孙子、孟子等

英雄满意度从"愤怒"到"兴奋"共八个等级。当满意度达到"高兴"以上时，英雄就可能**显身降临城市**，提供各种强大效果：如孔子提升税收、孙子增强步兵士气并降低军事建筑费用、灶君祝福食物建筑增加产量。祭祀需要消耗食物、丝绸等资源，玩家需要在资源投入和英雄回报之间做出策略抉择。

### 1.4 军事与外交系统

军事系统包括**城墙防御**（城墙、城门、箭楼）和**军队训练**（弓兵营、步兵营、车兵/骑兵营、投石车营）。训练军队需要武器铺提供武器装备。军队既可用于保卫城市抵御入侵，也可主动出征征服其他城市。

外交系统允许玩家与其他城市建立**联盟、贸易或征服**关系。征服的城市需要纳贡，但过度压榨可能导致叛乱。玩家还可以派遣间谍进行渗透和破坏。军事系统与城市建设紧密交织——需要平衡军事开支与民生发展。

### 1.5 朝代战役与关卡系统

游戏的战役模式按**中国历史朝代**推进，从夏朝（约公元前 2033 年）开始，历经商、西周、东周（春秋战国）、秦、汉、隋、唐、直到宋金时期（公元 1234 年）。每个朝代包含多个关卡，每个关卡有不同的胜利条件和历史背景。

关卡目标多样，可能包括：达到特定人口数、积累一定财富、建造特定纪念建筑（如长城、大运河、兵马俑）、完成军事征服等。随着朝代推进，可用的建筑和技术逐步解锁，体现了从青铜时代到铁器时代的技术演进。这一设计赋予了游戏深厚的历史叙事感和教育意义。

## 二、三国志系列的地图与城市发展系统

### 2.1 六角格地图与区域占领

以三国志14为代表的现代三国志系列采用了**六角格（Hex）地图系统**。整个中国地图被划分为数万个六角格，每个格子都有地形属性（平原、山地、森林、河流等），直接影响行军速度、战斗加成和开发收益。地图上分布着**城市、区域和关隘**三种地区类型。

**涂色占领机制**是三国志14的核心玩法之一：军队经过的区域会自动变为己方势力的颜色，占领的地格越多，获得的资源和收入越高。游戏还引入了**包围占领**战术——当一座敌方城市周边的所有格子都被己方控制时，可以发动包围占领，瞬间将范围内所有敌军断粮陷入混乱，这是极具策略性的玩法。

### 2.2 城市内政系统

三国志14的内政系统相比前作进行了大幅简化，分为**区域内政**和**城市内政**两大部分。区域内政涉及商业、农业和兵营三个方向的开发进度，分别影响每月的资金收入、兵粮产出和士兵增长。每个区域只能指派一名武将负责，武将的能力值直接影响开发效率。

城市内政则包括**内政方针**的设定，共有"现状"、"商业"、"农业"、"兵营"和"委任"五种方针。玩家需要根据战略需求分配有限的武将资源——是优先发展经济充实国库，还是加紧募兵准备征战，这种资源分配的取舍是游戏策略深度的核心。武将的统率、武力、智力、政治、魅力等属性都会在不同方面影响城市运营效率。

### 2.3 势力发展与战略地图

三国志系列一贯强调**势力之间的博弈**。玩家控制的势力需要在地图上不断扩张领土，通过外交手段（结盟、停战、联姻）和军事行动来增强实力。地图上的**府（资源点）**系统让区域控制更具策略意义——占领更多府可以显著提升钱粮兵的收入。

从三国志系列的历史演进来看，地图系统从早期的简单区域划分逐步发展为精细的六角格系统，城市内政从繁琐的指令式操作简化为方针式管理，体现了"易于上手、难于精通"的设计理念。这一设计哲学对策略游戏的系统设计具有重要参考价值。

## 三、现代放置/增量游戏的深度设计

### 3.1 核心增长模型

放置/增量游戏的核心是**指数级增长曲线**。玩家通过重复操作（点击或自动化）积累资源，资源增长从线性逐步过渡到指数级。以 Antimatter Dimensions 为代表，游戏通过"维度"系统实现层层嵌套的增长：第一维度产出基础资源，第二维度产出第一维度，以此类推，形成 O(log n) 层级的指数塔（Tetration）。

这种设计的精妙之处在于**玩家始终处于"即将突破"的心理状态**——每当增长放缓时，新的机制或升级恰好解锁，重新点燃增长势头。优秀的增量游戏会精心计算每个阶段的增长速率和成本曲线，确保玩家不会长时间陷入停滞，也不会过快地耗尽内容。

### 3.2 声望（Prestige）系统

声望系统是增量游戏的**标志性设计**。当玩家在当前回合中的增长明显放缓时，可以选择主动重置进度，换取永久性的加成货币（如 Realm Grinder 中的宝石、Antimatter Dimensions 中的无限点数）。这种"退一步进两步"的设计创造了游戏的长线循环。

深度增量游戏通常采用**多层声望**结构：第一层声望重置基础资源，第二层声望重置第一层声望资源，依此类推。Antimatter Dimensions 就有 Infinity → Eternity → Reality 等多层声望，每层都引入全新的机制和挑战。Realm Grinder 则通过阵营（Faction）系统实现声望的多样性——每次声望重置后可以选择不同阵营，获得完全不同的加成和玩法，极大地提高了重玩价值。

### 3.3 自动化与离线进度

自动化是放置游戏从"点击器"进化为"放置器"的关键。游戏通常在早期要求玩家主动操作，然后逐步解锁**自动购买器、自动升级器**等自动化功能，让游戏在玩家不操作时也能持续运转。这种从"主动参与"到"被动观察"再到"策略规划"的节奏变化，是放置游戏的核心吸引力。

**离线进度**系统确保玩家关闭游戏后资源仍在积累，回归时获得一次性大量奖励。这种设计利用了"损失厌恶"心理——玩家不想浪费离线期间本可以获得的资源，从而形成每日登录的习惯。优秀的放置游戏会平衡在线和离线收益，避免纯挂机成为最优策略。

## 四、放置游戏中的雇佣/英雄系统设计模式

### 4.1 英雄雇佣与升级模型

放置游戏中的英雄系统通常遵循**"雇佣→升级→进化"**的三阶段模型。以 Clicker Heroes 为典型代表，玩家用基础货币雇佣英雄，每个英雄有独立的等级和 DPS（每秒伤害）属性。英雄升级成本呈指数增长，而收益增长相对平缓，迫使玩家在"集中升级一个英雄"和"分散投资多个英雄"之间做出策略选择。

**里程碑升级**是英雄系统的关键设计：当英雄达到特定等级（如 25、50、100、200）时，会解锁质变性的能力提升（如 4 倍 DPS、全局加成等）。这种设计创造了明确的短期目标，驱动玩家持续投入资源。

### 4.2 英雄稀有度与收集系统

现代放置游戏广泛借鉴了**卡牌收集**的设计元素。英雄被划分为不同稀有度（普通、稀有、史诗、传说），稀有度越高基础属性越强、获取难度越大。这种设计满足了玩家的收集欲望，同时为游戏提供了商业化空间。

Realm Grinder 的**阵营系统**将英雄/建筑选择与阵营绑定，不同阵营提供完全不同的增长机制和策略路线。例如精灵阵营擅长快速声望循环，而矮人阵营则专注于长期积累。这种"互斥选择"设计让每次声望重置都具有策略意义。

### 4.3 协同与组合效应

高级英雄系统设计强调**英雄之间的协同效应**。当特定英雄组合同时上场时，会触发额外的加成效果。这种设计鼓励玩家探索不同的英雄搭配，而不是简单地使用 DPS 最高的英雄。

**进化/觉醒系统**允许达到满级的英雄通过特殊资源进行质变升级，重置等级但获得全新的能力和更高的成长倍率。这与声望系统形成了有趣的呼应——微观层面的英雄进化和宏观层面的游戏声望重置遵循相同的设计逻辑。

## 五、H5 Canvas 游戏的高清分辨率适配方案

### 5.1 DevicePixelRatio 适配原理

在高清屏（Retina 等）上，Canvas 默认按 CSS 像素渲染，导致画面模糊。适配的核心原理是：获取设备的 `devicePixelRatio`（Retina 屏通常为 2 或 3），将 Canvas 的实际像素尺寸放大为逻辑尺寸 × devicePixelRatio，然后通过 CSS 将 Canvas 缩放回逻辑尺寸。

具体步骤为：
1. 获取 `const ratio = window.devicePixelRatio || 1`
2. 设置 Canvas 实际分辨率：`canvas.width = logicalWidth * ratio; canvas.height = logicalHeight * ratio`
3. 通过 CSS 设定显示尺寸：`canvas.style.width = logicalWidth + 'px'; canvas.style.height = logicalHeight + 'px'`
4. 缩放绘图上下文：`ctx.scale(ratio, ratio)`

这样所有绘图代码仍按逻辑像素编写，但实际渲染在高分辨率画布上，最终通过 CSS 缩小显示时实现超采样效果，画面清晰锐利。

### 5.2 动态适配与性能权衡

实际项目中需要考虑**动态分辨率适配**：当窗口大小变化或设备方向改变时，需要重新计算 Canvas 尺寸。同时，高 devicePixelRatio 意味着更多的像素需要渲染，在移动设备上可能造成性能问题。

常见的优化策略包括：设置 devicePixelRatio 上限（如最大取 2）、在性能不足时动态降低渲染分辨率、对静态元素使用离屏 Canvas 缓存。对于游戏中的文字渲染，建议使用 Canvas API 的 `textBaseline` 和 `textAlign` 属性确保文字对齐精确，配合高 DPI 适配可以获得媲美原生应用的文字清晰度。

### 5.3 跨设备一致性方案

为确保游戏在不同设备上的一致体验，推荐采用**逻辑坐标系**设计：所有游戏逻辑和布局都基于一个固定的逻辑分辨率（如 1920×1080），通过统一的缩放矩阵映射到实际 Canvas 尺寸。这样无论设备的物理分辨率如何，游戏布局都保持一致。

对于等距（Isometric）视角的游戏（如龙之崛起风格），还需要额外处理菱形网格在不同分辨率下的对齐问题。建议使用浮点坐标进行逻辑计算，仅在最终渲染时取整到物理像素，避免累积误差导致的视觉抖动。

## 总结与设计启示

| 设计维度 | 龙之崛起 | 三国志 | 放置游戏 |
|---------|---------|--------|---------|
| **核心循环** | 建造→服务→升级→扩张 | 内政→征兵→征战→扩张 | 点击→升级→自动化→声望 |
| **策略深度** | 行人路线规划、资源链管理 | 武将分配、区域优先级 | 增长曲线优化、声望时机 |
| **进度系统** | 朝代关卡递进 | 剧本势力发展 | 多层声望重置 |
| **角色系统** | 四教英雄召唤 | 千名武将收集 | 英雄雇佣与进化 |
| **社交/外交** | 城市间贸易与征服 | 势力外交与联盟 | 通常为单人体验 |

**核心设计原则**：
1. **渐进式复杂度**——从简单操作开始，逐步引入新机制
2. **有意义的选择**——每个决策都有真实的策略权衡
3. **多层次的进度感**——短期目标（升级建筑）、中期目标（完成关卡）、长期目标（通关朝代）
4. **正反馈循环**——每次投入都能看到明确的回报
5. **资源平衡挑战**——永远无法同时满足所有需求，迫使玩家做出取舍
