**五级确认策略**:

| 级别 | 触发条件 | 确认方式 | 可否跳过 |
|:----:|---------|---------|:-------:|
| L0 免确认 | 铜钱 ≤ 200 | 直接购买，Toast提示 | — |
| L1 轻确认 | 铜钱 201~2,000 | 内联确认条（卡片下方展开） | ✅ 勾选「不再提示」 |
| L2 标准确认 | 铜钱 > 2,000 或兵力/粮草消耗 | 标准确认弹窗（上图） | ❌ |
| L3 严格确认 | 元宝消耗（付费货币） | 弹窗 + 3秒倒计时 | ❌ |
| L4 高危确认 | 元宝 ≥ 500 或限定商品 | 弹窗 + 滑动确认 | ❌ |

**L3 严格确认（元宝消费）**:

```
│  [确认购买（3s）]  ← 倒计时结束前灰色不可点击  │
```

**L4 高危滑动确认**:
```

│  │  🐴 ────────────────── → 确认          │  │
```

**智能提醒**:
- 购买后剩余货币 < 日常消耗均值 → ⚠️ 黄色提醒
- 余额不足 → 价格变红 + 鼓声提示音
- 快速连续点击 → 500ms内仅响应首次

## 动效规范

| 动效 | 时长 | 缓动 | 触发条件 |
|------|------|------|---------|
| 购买成功·商品飞出 | 400ms | `ease-out` 弧线 | 确认购买 |
| 购买成功·铜钱散落 | 300ms | 重力下落 | 商品飞出后 |
| 货币余额滚动 | 400ms | `ease-out` | 余额变化 |
| 货币不足·价格抖动 | 300ms | ±3px 30Hz | 余额不足 |
| 特惠倒计时跳动 | 300ms | 放大120%→恢复 | 最后60秒每秒 |
| 商店刷新·旧卡滑出 | 300ms | `ease-in` | 补货/刷新 |
| 商店刷新·水墨过渡 | 300ms | 水墨晕染 | 旧卡滑出后 |
| 商店刷新·新卡滑入 | 300ms | `ease-out` 交错50ms | 水墨过渡后 |
| 降价提醒·Toast | 3000ms | 滑入→停留→滑出 | 收藏商品降价 |
| 收藏切换 | 200ms | `ease-out` 缩放 | 点击收藏 |
| NPC交易·拱手 | 800ms | 序列动画 | 交易确认 |
