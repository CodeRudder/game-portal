> **v4.0 重写说明**：基于 v3.0 全面重构。重点补充：①活动代币完整获取途径与七阶兑换商店体系；②离线活动进度三档机制与自动完成引擎；③手机端 375×667 全场景完整方案（含手势交互、底部导航、推送通知）；④双端布局统一规范（PC 1280×800 / 手机 375×667）。

## 一、活动体系总览

### 1.1 活动在放置循环中的角色

```
放置核心循环：挂机产出 → 资源积累 → 消费升级 → 解锁新内容 → 挂机产出（加速）

活动系统在循环中的角色：
  ┌──────────────────────────────────────────────────────────────┐
  │  挂机产出 ←── 活动期间产出倍率加成（丰收祭/双倍周/节日加成） │
  │  资源积累 ←── 活动奖励提供大量资源（里程碑/排行/兑换商店）   │
  │  消费升级 ←── 活动代币兑换稀有道具/武将碎片/加速道具        │
  │  解锁内容 ←── 限定活动解锁专属武将/皮肤/建筑外观            │
  │  挂机加速 ←── 活动自动推进，离线持续累积活动积分与代币       │
  └──────────────────────────────────────────────────────────────┘
```

### 1.2 活动类型矩阵

| 活动类型 | 持续时间 | 频率 | 代币类型 | 离线效率 | 核心玩法 |
|---------|:-------:|:----:|---------|:-------:|---------|
| **赛季活动** | 14-28天 | 每月1次 | 赛季令牌 | 50%自动 | 里程碑+排行+商店 |
| **限时活动** | 3-7天 | 每周1-2次 | 活动铜钱 | 30%自动 | 限定任务+兑换 |
| **日常活动** | 24h重置 | 每天 | 日常代币 | 100%自动 | 每日任务+签到 |
| **节日活动** | 7-14天 | 节日期间 | 节庆灯笼 | 50%自动 | 特殊玩法+限定奖励 |
| **联盟活动** | 3-5天 | 每两周1次 | 联盟勋章 | 仅个人部分 | 协作目标+排行 |

### 1.3 活动代币经济总览

```
代币生命周期：获取 → 累积 → 消耗（兑换/加速） → 结算转化

  ┌─────────────────────────────────────────────────────────────┐
  │  获取层（7种途径）                                          │
  │  ├─ 战斗积分：每次战斗自动 +5~20 代币（按活动类型）         │
  │  ├─ 里程碑达成：一次性 +200~5000 代币                      │
  │  ├─ 活动任务：每任务 +100~500 代币                          │
  │  ├─ 排行结算：期末 +500~5000 代币                           │
  │  ├─ 离线累积：+2~10 代币/小时（按活动效率系数）             │
  │  ├─ 每日活跃：达标 +150 代币/天                             │
  │  └─ 元宝购买：1 代币/元宝（可选付费加速）                   │
  │                                                             │
  │  消耗层（3种途径）                                          │
  │  ├─ 兑换商店：购买道具/武将碎片/限定物品                    │
  │  ├─ 里程碑加速：跳过当前任务，消耗 = 当前需求 ×50% 代币     │
  │  └─ 重置挑战：重置挑战次数，消耗因活动而异                  │
  │                                                             │
  │  结算层                                                     │
  │  └─ 活动结束后未使用代币 → 按 10% 比例转化为普通铜钱        │
  └─────────────────────────────────────────────────────────────┘
```

## 二、活动入口与总览

### 2.1 四大入口设计

| 入口位置 | 触发方式 | 显示内容 | 优先级 |

### 5.2 七阶兑换商店体系

商店按稀有度分为七个阶层，每阶层解锁条件不同：

| 阶层 | 名称 | 解锁条件 | 商品稀有度 | 每日刷新 |
|:----:|------|---------|----------|:-------:|
| 1 | 青铜铺 | 活动开始即解锁 | 普通资源 | ✅ |
| 2 | 白银肆 | 累计消耗 500 代币 | 进阶资源 | ✅ |
| 3 | 黄金坊 | 累计消耗 2000 代币 | 稀有道具 | ❌ 固定 |
| 4 | 翡翠阁 | 里程碑达成 3 个 | 武将碎片 | ❌ 固定 |
| 5 | 赤金殿 | 排行榜前 50% | 高级装备 | ❌ 固定 |

## 八、离线活动进度系统（放置核心）

### 8.1 离线活动进度三档机制

| 活动类型 | 积分累积 | 任务推进 | 商店兑换 | 里程碑推进 | 每日上限 |
|---------|:-------:|:-------:|:------:|:---------:|:-------:|
| **赛季活动** | ✅ 50%效率 | ✅ 累积任务自动 | ❌ | ✅ 自动推进 | 12h/天 |
| **限时活动** | ✅ 30%效率 | ⚠️ 仅累积任务 | ❌ | ✅ 自动推进 | 8h/天 |
| **日常活动** | ✅ 100%效率 | ✅ 全部自动 | ❌ | ✅ 自动推进 | 24h |
| **节日活动** | ✅ 50%效率 | ✅ 累积任务自动 | ❌ | ✅ 自动推进 | 12h/天 |
| **联盟活动** | ⚠️ 仅个人 | ⚠️ 仅个人任务 | ❌ | ⚠️ 个人里程碑 | 8h/天 |

### 8.2 离线积分计算公式

```
离线活动积分 = 基础每秒积分 × 离线效率系数 × min(有效离线时长, 上限时长)

离线效率系数 = 基础效率 × (1 + 声望等级 × 0.03)
  - 赛季活动基础效率: 50%
  - 限时活动基础效率: 30%
  - 日常活动基础效率: 100%
  - 节日活动基础效率: 50%

有效离线时长 = 实际离线时长（扣除超过上限部分）

声望等级加成示例：
  - 声望 Lv.0: 效率系数 = 基础 × 1.00
  - 声望 Lv.5: 效率系数 = 基础 × 1.15
  - 声望 Lv.10: 效率系数 = 基础 × 1.30
  - 声望 Lv.20: 效率系数 = 基础 × 1.60
```

## 十一、动效规范

### 11.1 里程碑达成动效

```
触发条件：里程碑积分达标
动画序列：
  1. 青铜鼎震动（0.3s，左右摇摆 5°）
  2. 鼎口金光爆发（0.5s，粒子向上喷射 20 个）
  3. 鼎中升腾烟雾（1.0s，水墨烟雾扩散）
  4. 奖励卡片从鼎中飞出（0.5s，弹性缓动）
  5. 进度条满格闪烁（0.5s，金色脉冲）
总时长：2.8s
音效：铜钟长鸣 + 烟雾嘶嘶声
```

### 11.2 兑换成功动效

```
触发条件：确认兑换后
动画序列：
  1. 商品卡片缩小至 50%（0.2s）
  2. 飞向屏幕右下角背包方向（0.5s，弧线轨迹）
  3. 到达后铜钱散落粒子（0.3s，8 个铜钱粒子）
  4. 代币余额数字滚动减少（0.5s）
音效：铜钱碰撞声 + 竹简展开声
```

### 11.3 活动即将结束动效

```
触发条件：活动剩余时间 < 24 小时
  - 横幅边框红色脉冲（2s 周期，持续）
  - 倒计时数字变红并轻微抖动
  - 活动入口图标添加沙漏动画
触发条件：活动剩余时间 < 1 小时
  - 边框脉冲加速至 0.8s 周期
  - 倒计时数字放大 120% 并闪烁
```

### 11.4 排行榜变动动效

```
触发条件：自己排名上升
  - 排名数字绿色闪烁 + 向上箭头（+3）
  - 头像框短暂金色光晕（0.5s）
触发条件：被超越
  - 排名数字红色闪烁 + 向下箭头（-2）
```

### 11.5 活动横幅轮播动效

```
切换动画：水墨晕染过渡（0.6s，ease-out）
  - 当前横幅向左淡出（水墨化消散效果）
  - 新横幅从右侧淡入（水墨晕染展开）
手动滑动：跟随手指，松手后弹性回弹或切换
```

### 11.6 商店阶层解锁动效

```
触发条件：满足阶层解锁条件
动画序列：
  1. 锁定图标震动（0.3s）
  2. 铜锁碎裂（0.5s，碎片飞散粒子）
  3. 阶层卡片金光展开（0.8s，从中心向外扩展）
  4. 新商品依次弹出（每项 0.2s，间隔 0.1s）
音效：锁链断裂 + 卷轴展开
```

## 十二、交互细节与边缘场景

### 12.1 活动到期处理

| 场景 | 处理方式 | UI 反馈 |
|------|---------|--------|
| 活动进行中玩家在线 | 正常倒计时，到期后弹窗提示结算 | 「活动已结束，正在结算...」 |
| 活动到期时玩家在活动页 | 自动弹出结算面板 | 展示获得奖励 + 排名 + 未使用代币处理 |
| 未使用代币处理 | 按比例转化为普通货币（代币×10%→铜钱） | 结算面板明确显示转化明细 |
| 活动到期时玩家离线 | 下次上线时在回归面板展示 | 灰色「已结束」标签 + 结算摘要 |

### 12.2 网络异常处理

| 场景 | 处理方式 |
|------|---------|
| 兑换时断网 | 本地缓存兑换请求，重连后自动提交，期间显示「处理中」 |
| 排行榜加载失败 | 显示上次缓存数据 + 「数据可能不是最新」提示 |
| 活动数据同步失败 | 使用本地缓存活动数据，顶部黄条提示「活动数据同步中」 |
| 离线回归数据同步 | 先展示本地计算结果，后台同步服务器数据后刷新 |

### 12.3 多活动并行规则

```
同时进行的活动上限：5 个
  - 最多 1 个赛季活动
  - 最多 2 个限时活动
  - 1 个日常活动（常驻）
  - 最多 1 个节日/联盟活动

活动优先级排序（横幅轮播）：
  1. 即将结束的活动（<24h）
  2. 新开始的活动（<24h）
  3. 有未领取奖励的活动
  4. 按剩余时间升序
```
