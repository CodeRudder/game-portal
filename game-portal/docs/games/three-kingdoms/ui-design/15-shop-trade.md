# 商店与交易 UI 设计（水墨江山·铜纹霸业 v4.0）

> **v4.0 重写说明**：全面升级为水墨江山·铜纹霸业风格，补全 PC 端 1280×800 + 手机端 375×667 双端完整布局；新增分级购买确认弹窗与五重误操作防护；新增自动补货引擎、收藏商品系统、降价提醒推送；补全手机端完整购物与交易交互方案；所有界面均以 ASCII 线框呈现双端对照。

## 四、🔴 购买确认弹窗与误操作防护

### 4.1 五级确认策略

| 级别 | 触发条件 | 确认方式 | 可否跳过 |
|:----:|---------|---------|:-------:|
| L0 免确认 | 铜钱 ≤ 200 | 直接购买，Toast 提示「已购买 ×X」 | — |
| L1 轻确认 | 铜钱 201 ~ 2,000 | 内联确认条（卡片下方展开） | ✅ 勾选「不再提示」 |
| L2 标准确认 | 铜钱 > 2,000 或兵力/粮草消耗 | 标准确认弹窗 | ❌ |
| L3 严格确认 | 元宝消耗（付费货币） | 弹窗 + 3 秒倒计时 | ❌ |
| L4 高危确认 | 元宝 ≥ 500 或限定商品 | 弹窗 + 滑动确认 | ❌ |

### 4.7 五重误操作防护

| # | 防护场景 | 规则 | UI 反馈 |
|:-:|---------|------|---------|
| 1 | 快速连续点击 | 同一商品 500ms 内仅响应首次点击 | 后续点击无反应 |
| 2 | 购买中切换 Tab | 弹出「购买进行中，确认离开？」 | 半透明遮罩 + 提示弹窗 |
| 3 | 货币不足 | 价格数字抖动变红 + 鼓声 | Toast「铜钱不足」 |
| 4 | 商品已售罄 | 购买按钮灰化 + 文字变更 | Toast「已售罄」 |
| 5 | 网络延迟 | 按钮 loading 旋转 + 文字 | 「处理中…」禁止重复点击 |

**额外防护**：
- 购买确认弹窗打开时按返回键 → 先关闭弹窗而非关闭页面
- 购买请求超时 → 本地缓存请求，重连后自动提交
- 购买失败（服务端拒绝）→ 回滚本地状态 + Toast 显示失败原因
- 重复提交 → 服务端幂等处理 + 本地 500ms 节流

## 十、动效规范

### 10.1 购买成功动效

```
触发：购买确认后
序列：
  1. 商品卡片缩小至 70%（0.15s，ease-in）
  2. 飞向右下角背包方向（0.4s，弧线轨迹，ease-out）
  3. 铜钱散落粒子（0.3s，8 个铜钱粒子，重力下落）
  4. 货币余额数字滚动减少（0.4s，递减动画）
  5. 库存数字减少（0.2s，弹跳动画）
音效：铜钱碰撞声 + 竹简展开声
总时长：1.25s
```

### 10.2 货币不足动效

```
触发：购买时货币不足
序列：
  1. 价格数字抖动（0.3s，左右 ±3px，30Hz）
  2. 价格数字变红（0.2s）
  3. 卡片边框红色闪烁（0.5s，2 次）
音效：低沉鼓声
```

### 10.3 限时商品倒计时动效

```
最后 10 分钟：
  - 边框脉动加速至 0.8s 周期
  - 倒计时数字放大 110% 并红色闪烁
最后 60 秒：
  - 边框火焰效果增强
  - 倒计时每秒跳动（放大 120% → 恢复，0.3s）
  - 背景轻微震动
```

### 10.4 NPC 交易确认动效

```
触发：交易确认后
序列：
  1. NPC「拱手」动画（0.8s）
  2. 商品从 NPC 侧飞向玩家背包（0.5s）
  3. 玩家出售物品从背包飞向 NPC（0.5s）
  4. 铜钱差额结算粒子（0.3s）
音效：古琴拨弦 + 铜钱声
```

### 10.5 商店刷新动效

```
触发：补货 / 刷新
序列：
  1. 旧商品卡片集体向左滑出（0.3s）
  2. 水墨晕染过渡（0.3s）
  3. 新商品卡片从右滑入（0.3s，交错延迟 50ms/张）
  4. 新商品卡片右上角「NEW」金色角标（3 秒后淡出）
音效：卷轴展开声
```

### 10.6 降价提醒动效

```
触发：收藏商品降价
  - 商店图标红点脉动（1.5s 周期）
  - 收藏 Tab 降价商品金色边框发光
  - 首次降价：主界面顶部滑入 Toast（3 秒后上滑消失）
  - 手机端：系统推送通知
```

## 十二、商品刷新与限购规则

### 12.1 刷新规则

| 商品类型 | 刷新周期 | 刷新数量 | 玩家可控 |
|---------|:-------:|---------|:-------:|
| 常驻基础商品 | 不刷新 | 始终存在 | ❌ |
| 随机商品 | 8h | 3~5 件 | ✅ 手动刷新（💰500） |
| 折扣商品 | 8h（概率） | 0~2 件 | ❌ |
| 限时特惠 | 24h | 2~4 件 | ❌ |
| 黑市商品 | 随机出现 | 3~5 件 | ❌ |

### 12.2 限购规则

```
限购类型：
  - 每日限购：每日 0 点重置购买次数
  - 终身限购：活动期间 / 账号生命周期内限量
  - 好感度限购：NPC 好感度达到后才解锁购买资格

限购显示：
  - 限购商品右上角「限购 N 件」标签
  - 已购数量实时更新：「已购 2/3」
  - 达到上限后购买按钮灰化 + 「已达上限」
```

### 12.3 网络异常处理

| 场景 | 处理方式 | UI 反馈 |
|------|---------|---------|
| 购买请求超时 | 本地缓存请求，重连后自动提交 | 按钮 loading + 「处理中…」 |
| 购买失败（服务端拒绝） | 回滚本地状态，显示失败原因 | Toast「购买失败：XXX」 |
| 刷新失败 | 使用上次缓存数据 | 黄条「商店数据同步中…」 |
| 重复提交 | 服务端幂等处理，仅执行一次 | 本地防重复点击（500ms 节流） |
