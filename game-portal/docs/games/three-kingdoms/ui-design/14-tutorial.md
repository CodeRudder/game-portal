# 新手引导 UI 设计（v3.1 大幅迭代）

> **v3.1 迭代说明**：在 v3.0 基础上补充引导状态服务端存储方案（账号绑定、跨设备同步），解决 Round 3 P0-6。

---

## 一、引导触发与全局规则

### 1.1 触发条件
- 首次进入游戏自动触发核心引导（步骤1-6），**不可跳过**
- 扩展引导（步骤7-12）在对应功能解锁时自动触发 mini-tutorial
- 已完成引导可在 **设置 → 引导回顾** 中重玩全部或单步

### 1.2 引导状态管理
```
引导状态机：
  [未开始] ──首次进入──→ [核心引导中] ──步骤6完成──→ [自由探索过渡] ──过渡完成──→ [自由游戏]
                              │                                                │
                              └──加速跳过──→ [自由探索过渡]                       └──条件触发──→ [Mini-tutorial]
```

- 引导进度实时保存至本地存储（localStorage），断线重连后从断点继续
- 每步引导有唯一标识符（如 `tutorial_step_01`），完成后标记 `completed: true`
- 引导状态与账号绑定，换设备登录同步进度

#### 引导状态服务端存储方案（v3.1 新增 — 解决 P0-6）

> **问题背景**：Round 3 评测发现引导状态仅存 localStorage，跨设备无法同步，换设备需重做引导。

**存储架构**：

```
┌───────────────────────────────────────────────────────┐
│                    服务端引导状态存储                     │
│                                                       │
│  存储方式：账号维度 KV 存储（Redis / 数据库 JSON 字段）  │
│  Key: account:{uid}:tutorial_state                     │
│  Value: {                                             │
│    "phase": "free_play",          // 当前阶段          │
│    "completed_steps": [1,2,3,4,5,6],  // 已完成步骤    │
│    "completed_events": ["E1","E2","E3"], // 已播放剧情  │
│    "current_substep": null,       // 当前子步骤        │
│    "skipped_events": ["E4"],      // 已跳过的剧情      │
│    "replay_count_today": 2,       // 今日重播次数      │
│    "last_update": "2025-07-12T14:30:00Z"               │
│  }                                                    │
│                                                       │
│  同步策略：                                            │
│  ① 登录时：服务端 → 客户端（下发最新引导状态）          │
│ ② 步骤完成时：客户端 → 服务端（实时上报）              │
│  ③ 退出游戏时：客户端 → 服务端（全量同步）             │
│  ④ 切换设备时：以服务端为准（冲突取进度更远的一方）     │
│                                                       │
│  冲突解决规则：                                        │
│  · 本地 completed_steps ⊂ 服务端 → 使用服务端状态      │
│  · 本地 completed_steps ⊃ 服务端 → 上传本地状态        │
│  · 两者不可比较（部分重叠）→ 合并取并集（取最大进度）   │
└───────────────────────────────────────────────────────┘
```

**同步时机详细说明**：

| 时机 | 方向 | 触发条件 | 数据内容 |
|------|------|---------|---------|
| **登录同步** | 服务端 → 客户端 | 账号登录成功后 | 完整引导状态（phase + completed_steps + completed_events） |
| **步骤完成同步** | 客户端 → 服务端 | 每个子步骤完成时 | `{step_id, substep_id, completed: true, timestamp}` |
| **退出同步** | 客户端 → 服务端 | 游戏退出/切至后台 | 完整引导状态全量快照 |
| **跨设备冲突** | 双向 | 登录时检测到本地与服务端不一致 | 取 completed_steps 并集 + completed_events 并集 |

**离线场景处理**：
- 离线期间完成的引导步骤保存在本地 localStorage
- 上线后执行「退出同步」逻辑，将离线完成的步骤上报服务端
- 如果服务端记录已被其他设备更新（进度更远），取并集合并

**数据安全**：
- 引导状态与账号 UID 绑定，非设备绑定
- 引导状态数据量极小（< 1KB），不影响性能
- 重播次数每日重置（服务端维护重置逻辑）

## 二、6步核心引导详细设计

### 步骤1：主城概览 ——「初入乱世，立足根基」

**教学目标**：认识主界面布局，了解资源栏、导航Tab、主城场景三大区域

**引导流程**：

| 子步骤 | 操作 | 遮罩区域 | 高亮元素 | 提示文案 | 完成条件 |
|:------:|------|---------|---------|---------|---------|
| 1-1 | 自动展示 | 全屏遮罩 | 主城全景 | 卷轴气泡：「主公，乱世已至。这便是您的根基——主城。」 | 3秒后自动继续 |
| 1-2 | 引导点击 | 遮罩+顶部镂空 | 资源栏区域 | 水墨手指→资源栏：「此处显示您的铜钱、粮草、兵力等资源。」 | 点击资源栏任意位置 |
| 1-3 | 引导点击 | 遮罩+底部镂空 | 导航Tab栏 | 水墨手指→Tab栏：「天下、出征、武将、内政、社交——五大模块尽在掌握。」 | 点击任意Tab后自动切回 |
| 1-4 | 引导点击 | 遮罩+中央镂空 | 主城建筑区域 | 水墨手指→空地：「这片空地，正待主公开疆拓土。」 | 点击主城任意空地 |
| 1-5 | 自动完成 | 遮罩消失 | 全局 | 金色光圈反馈：「很好！主公已熟悉城池全貌。」 | 自动进入步骤2 |

**视觉规范**：
- 遮罩：全屏 `rgba(0,0,0,0.70)` 黑色遮罩
- 镂空：目标区域外扩8px，圆角 `border-radius: 8px`
- 镂空描边：古铜金 `#C8A45C` 流光动画（1.5s循环，渐变位移）
- 卷轴气泡：宽 `280px`（PC）/ `240px`（手机），竹简底色 `#F5E6C8`，古铜金边框
- 水墨手指：脉动缩放动画 `scale(0.9→1.1)`，周期800ms

**手机端适配**：
- 遮罩比例不变，镂空区域根据手机布局重新映射
- 卷轴气泡宽度 `min(240px, 85vw)`，字体 `14px`
- 水墨手指改为「触摸涟漪」效果（同心圆扩散动画）

## 三、8段剧情事件详细设计

### E1：桃园结义（步骤1之前 · 开场剧情）

**触发时机**：首次进入游戏，加载完成后自动播放

**时长**：45秒

**背景插画描述**：
- 场景：春日桃园，粉色桃花漫天飞舞，三人在桃树下跪拜
- 画风：水墨渲染+淡彩，桃花用粉色点缀，远山用淡墨晕染
- 构图：三人居中偏下，桃树从左上延伸至右上，形成拱形框架
- PC端：全屏1920×1080插画，左右留白题诗
- 手机端：竖版插画 750×1334，聚焦三人上半身

**角色对话序列**：

| 时间轴 | 角色 | 对话 | 立绘表情 |
|:------:|------|------|---------|
| 0-5s | 旁白 | 「东汉末年，天下大乱。群雄并起，民不聊生。」 | 无（仅文字+背景） |

| 12-18s | 关羽 | 「关某愿追随兄长，肝脑涂地，在所不辞！」 | 拱手坚定 |
| 18-24s | 张飞 | 「俺也一样！谁敢欺负百姓，俺一丈八蛇矛伺候！」 | 挥拳豪迈 |
| 24-32s | 旁白 | 「三人于桃园焚香结义，立誓同心协力，救困扶危。」 | 三人跪拜画面 |
| 32-40s | 刘备 | 「今日起，你我兄弟同心，共创大业！」 | 举杯誓言 |
| 40-45s | 旁白 | 「主公，您的霸业之路，从此刻开始……」 | 画面渐暗→主城 |

**交互设计**：
- 点击任意位置推进下一段对话

## 四、新手引导跳过/加速机制

### 4.1 核心原则：不可跳过，但可加速

**设计理由**：
- 核心引导（步骤1-6）包含关键系统教学，跳过会导致后续体验断裂
- 但尊重玩家时间，提供加速通道减少等待感

### 4.2 加速机制详细设计

| 加速方式 | 触发条件 | 效果 | 冷却 |
|---------|---------|------|------|
| **对话加速** | 点击任意位置 | 跳过当前打字机效果，立即显示完整文字 | 无 |
| **剧情快进** | 点击「跳过▶▶」按钮 | 跳过当前剧情事件，直接进入下一步引导 | 每段剧情仅可跳过一次 |
| **动画加速** | 连续点击3次 | 当前动画速度×3（建造、战斗、研究） | 无 |
| **一键完成** | 引导步骤停留超过60秒 | 出现「快速完成」按钮，一键跳过当前子步骤 | 每步限用1次 |

### 4.3 跳过确认流程

```
点击「跳过▶▶」
  → 弹出确认卷轴：
    「跳过剧情将无法获得沉浸体验，确认跳过？」
    [再看看] [确认跳过]
  → 确认后：水墨晕染过渡（300ms）→ 下一步引导
```

### 4.4 不可跳过的内容

以下内容**绝对不可跳过**：
- 步骤1-1（主城概览总览）：首次必须看到完整界面
- 步骤2-3（确认建造）：必须完成一次建造操作
- 步骤4-4（首次战斗）：必须观看至少一场战斗
- 所有「引导点击」子步骤：必须实际点击目标元素

- 重播为**观看模式**：不消耗资源、不影响游戏进度
- 重播时使用简化的遮罩（50%透明），不锁定操作
- 剧情重播使用全屏演出模式，与首次观看一致
- 重播完成后弹出「回顾完毕」Toast提示
- 每次重播奖励：铜钱×100（每日限3次）
