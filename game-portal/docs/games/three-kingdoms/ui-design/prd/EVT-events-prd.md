## [EVT-1] 事件类型 {#evt-1}

### 功能描述

事件系统是放置核心循环中的随机变量层，通过不可预测的外部事件为玩家提供资源波动、剧情推进和决策乐趣。事件分为三大类：**随机事件**、**固定事件**、**连锁事件**。

> 🎨 → [UI: 事件布局](../ui-layout/EVT-events.md#evt-1)

### 事件类型矩阵

| 事件类型 | 触发方式 | 频率 | 与放置循环关系 | 离线处理 |
|---------|---------|:----:|--------------|---------|
| **随机遭遇** | 自动触发（定时器） | 1-2次/小时 | 资源获取/消耗 | 堆积待处理 |
| **剧情事件** | 条件触发（关卡/声望） | 随进度推进 | 解锁新内容 | 暂停等待 |
| **NPC事件** | NPC好感度/日程触发 | 0.5-1次/小时 | 社交资源获取 | 堆积待处理 |
| **天灾人祸** | 随机触发（概率性） | 0.3次/小时 | 资源消耗/防御考验 | 自动处理（按防御值） |
| **限时机遇** | 时间窗口触发 | 2-3次/天 | 高价值资源获取 | 错过即消失 |
| **里程碑事件** | 成就达成触发 | 随进度 | 正反馈强化 | 暂停等待 |

### 事件分类详解

#### 随机事件（Random Encounter）

- **定义**：由系统定时器驱动的不可预测事件，玩家需做出选择
- **子类型**：
  - 🟢 **奇遇**（正面）：商队来访、丰收之年、名士拜访、天降祥瑞
  - 🔴 **天灾**（负面）：洪水泛滥、瘟疫蔓延、蝗虫过境、地震
  - 🟠 **人祸**（负面）：流寇来袭、山贼劫道、叛军作乱
  - 🔵 **商队**（中性）：西域商人、行脚僧、神秘旅人
- **频率**：1-2次/小时，平均间隔 30-60 分钟
- **离线处理**：正面事件自动选择"保守"选项获 50% 奖励；负面事件按防御值自动减免

#### 固定事件（Fixed Event）

- **定义**：由特定条件触发的确定性事件，与游戏进度强关联
- **子类型**：
  - **剧情事件**：关卡推进到指定节点触发，推进主线故事
  - **NPC事件**：NPC好感度达到阈值或日程触发，解锁社交内容
  - **里程碑事件**：成就达成时触发，提供正反馈强化
- **特性**：不可跳过、不可重复（一次性），离线期间暂停等待玩家上线

#### 连锁事件（Chain Event）

- **定义**：由前序事件选择结果触发后续事件的链式事件序列
- **特性**：选择不同分支可触发不同后续事件，形成事件链
- **详见**：[EVT-4] 连锁事件

### 事件与放置核心循环关联

```
放置核心循环：挂机产出 → 资源积累 → 消费升级 → 解锁新内容 → 挂机产出（加速）

事件系统在循环中的角色：
  挂机产出 ←── 事件奖励加速产出（天降横财/商人来访）
  资源积累 ←── 事件提供额外资源（随机遭遇奖励）
  消费升级 ←── 事件触发升级需求（危机→需要强化）
  解锁内容 ←── 剧情事件解锁新地图/NPC/科技
  挂机加速 ←── 好事件提供临时产出倍率加成
```

### 事件类型标签颜色编码

| 标签 | 颜色 | 含义 |
|------|------|------|
| 🔴 天灾 | 红色 | 自然灾害，需紧急应对 |
| 🟠 人祸 | 橙色 | 人为祸患，需武力或智谋 |
| 🟢 奇遇 | 绿色 | 好运降临，获得奖励 |
| 🔵 商队 | 蓝色 | 交易机会，可选择交换 |

## [EVT-2] 触发条件 {#evt-2}

### 功能描述

定义各类事件的触发规则，包括时间触发、条件触发和概率触发三大机制，确保事件出现的节奏既不可预测又不会过于频繁。

> 🎨 → [UI: 事件布局](../ui-layout/EVT-events.md#evt-2)

### 时间触发规则

| 事件类型 | 触发间隔 | 首次触发 | 日上限 | 说明 |
|---------|---------|---------|:-----:|------|
| 随机遭遇 | 30-60 分钟（随机） | 进入游戏 10 分钟后 | 24 次 | 平均每小时 1-2 次 |
| NPC事件 | 60-120 分钟（随机） | 进入游戏 20 分钟后 | 12 次 | 平均每小时 0.5-1 次 |
| 天灾人祸 | 120-200 分钟（随机） | 进入游戏 30 分钟后 | 8 次 | 平均每 2-3 小时 1 次 |
| 限时机遇 | 固定时段触发 | 10:00 / 14:00 / 20:00 | 3 次 | 每次持续 30-60 分钟 |

### 条件触发规则

| 事件类型 | 触发条件 | 说明 |
|---------|---------|------|
| 剧情事件 | 关卡通关至指定节点 | 前置章节完成后自动触发下一章剧情 |
| NPC事件 | NPC好感度 ≥ 阈值 | 不同NPC有不同好感度触发线 |
| 里程碑事件 | 成就达成 | 如"首次占领城池""武将满级"等 |
| 连锁事件 | 前序事件选择特定分支 | 选择结果满足后续事件条件 |

### 概率触发公式

```
随机遭遇触发概率（每分钟检测一次）：
  P(触发) = 基础概率 × (1 + 玩家等级 × 0.02) × 时间衰减系数

  基础概率 = 0.03（3%）
  时间衰减系数 = max(0.5, 1 - 距上次事件分钟数 / 120)
  
  示例：
  - 等级 10，距上次事件 30 分钟：P = 0.03 × 1.20 × 0.75 = 2.7%
  - 等级 20，距上次事件 60 分钟：P = 0.03 × 1.40 × 0.50 = 2.1%

天灾人祸触发概率（每分钟检测一次）：
  P(触发) = 0.015 × (1 + 关卡难度 × 0.01)
  
  关卡难度随主线推进提升，后期天灾更频繁

限时机遇触发：
  固定时段触发，不使用概率公式
  触发后持续 30-60 分钟，玩家需在时间窗口内响应
```

### 通知优先级规则

```
同时仅显示1条急报横幅，按以下优先级排队：
  1. 限时机遇（倒计时<10分钟）— 红色边框 + 脉动
  2. 天灾人祸（需立即应对）— 橙色边框 + 震动
  3. 剧情事件（关键节点）— 金色边框
  4. 随机遭遇 — 铜色边框
  5. NPC事件 — 银色边框
  6. 里程碑事件 — 绿色边框

排队上限：最多缓存 5 条急报
超出策略：低优先级事件自动以"最小损失"方案处理
```

### 事件冷却机制

| 规则 | 说明 |
|------|------|
| 同类型冷却 | 同一子类型事件触发后 60 分钟内不再触发 |
| 同事件冷却 | 同一具体事件触发后 4 小时内不再重复 |
| 新手保护 | 前 30 分钟仅触发正面事件（奇遇/商队） |
| 连续负面保护 | 连续 2 次负面事件后，下一次必定为正面或中性 |

## [EVT-3] 事件选项 {#evt-3}

### 功能描述

每个事件提供 2-3 个选择分支，不同选择产生不同后果（奖励/惩罚），玩家的决策直接影响资源产出和游戏进程。

> 🎨 → [UI: 事件布局](../ui-layout/EVT-events.md#evt-3)

### 选项结构

每个事件包含以下数据结构：

```
事件 = {
  ID: string,
  标题: string,
  描述: string,
  类型: 随机 | 固定 | 连锁,
  子类型: 奇遇 | 天灾 | 人祸 | 商队 | 剧情 | NPC | 里程碑,
  选项列表: [
    {
      文本: string,
      后果: [
        { 资源类型, 数值变化, 概率: 100% | 概率值 },
        ...
      ],
      后续事件ID: string | null,  // 连锁事件用
      声望变化: int,
      好感度变化: { NPC_ID: int },
    }
  ],
  默认选项索引: int,  // 离线自动处理用
}
```

### 选项后果标注规则

- 🟢 绿色：正面效果（资源获得、属性提升）
- 🔴 红色：负面风险（资源损失、属性下降）
- 🟡 黄色：中性效果（有得有失）

#### 选项奖励缩放规则

```
资源奖励 = 基础值 × (1 + 主城等级 × 0.1)
资源损失 = 基础值 × max(0.3, 1 - 防御建筑总等级 / (防御建筑总等级 + 50))

示例：
  主城等级 10：奖励倍率 = 1 + 10 × 0.1 = 2.0x
  防御等级 30：损失减免 = 30 / (30 + 50) = 37.5%
```

### 离线自动处理规则

| 事件类型 | 离线处理方式 | 处理规则 |
|---------|:----------:|---------|
| 随机遭遇（正面） | 自动完成 | 自动选择"保守"选项，获得 50% 基础奖励 |
| 随机遭遇（负面） | 自动处理 | 按当前防御值自动计算损失（最低损失） |
| 随机遭遇（中性） | 堆积待处理 | 最多堆积 3 条，回归后以"急报堆"展示 |
| 剧情事件 | 暂停等待 | 不触发、不推进，等玩家在线时继续 |
| NPC事件 | 堆积待处理 | 最多堆积 2 条 |
| 天灾人祸 | 自动处理 | 按防御建筑等级计算减免，剩余损失自动扣除 |
| 限时机遇 | 错过消失 | 离线期间触发的限时机遇直接错过 |
| 里程碑事件 | 暂停等待 | 不触发，等玩家在线时触发 |

### 离线收益计算公式

```
离线随机遭遇（正面）自动收益 = 基础奖励 × 50% × min(离线时长/1小时, 8)
离线随机遭遇（负面）自动损失 = 基础损失 × max(0, 1 - 防御减免率)
防御减免率 = 防御建筑总等级 / (防御建筑总等级 + 50)
天灾自动损失上限 = 当前资源总量的 15%
```

### 离线事件堆积上限

| 参数 | 数值 |
|------|------|
| 最大堆积数 | 5 条（超出按优先级丢弃低优先级事件） |
| 堆积事件保留时长 | 72 小时（超时自动以"保守"方案处理） |
| 离线时长阈值 | < 30 分钟不触发任何离线事件处理 |

## [EVT-4] 连锁事件 {#evt-4}

### 功能描述

连锁事件是由前序事件选择结果触发后续事件的链式序列。玩家在第一环的选择决定后续事件的走向和最终奖励，增加策略深度和叙事沉浸感。

> 🎨 → [UI: 事件布局](../ui-layout/EVT-events.md#evt-4)

### 连锁事件触发规则

```
触发条件：
  1. 玩家在事件中选择特定分支选项
  2. 该选项标记了后续事件ID
  3. 后续事件的前置条件全部满足
  4. 触发延迟：选择后 10-30 分钟随机触发下一环

不触发条件：
  - 玩家选择无后续事件标记的选项 → 事件链中断
  - 后续事件前置条件不满足 → 事件链中断，无提示
  - 玩家离线 → 事件链暂停，上线后继续计时
```

### 事件链结构

```
事件链 = {
  链ID: string,
  链名称: string,
  总环节数: int,  // 通常 2-4 环
  事件序列: [
    {
      环节序号: int,
      事件ID: string,
      前置条件: { 关卡进度, 声望等级, ... },
      选项映射: {
        选项索引 → 后续环节序号 | null（链终止）
      }
    }
  ],
  最终奖励: { ... },  // 完成全部环节后额外奖励
}
```

### 连锁事件示例

#### 示例一：桃园结义链（3环）

| 环节 | 事件 | 选项A → 后续 | 选项B → 后续 |
|:----:|------|------------|------------|
| 1 | 路遇豪杰 | 结伴同行 → 环节2 | 各奔前程 → 链终止 |
| 2 | 桃园设宴 | 桃园结义 → 环节3 | 礼貌婉拒 → 链终止（获得少量铜钱） |
| 3 | 义薄云天 | 肝胆相照 → 链完成（最终奖励） | — |

**最终奖励**：获得武将「关羽·忠义」+ 声望+50 + 铜钱×3000

#### 示例二：黄巾余党链（4环）

| 环节 | 事件 | 选项A → 后续 | 选项B → 后续 | 选项C → 后续 |
|:----:|------|------------|------------|------------|
| 1 | 黄巾残党出没 | 出兵围剿 → 环节2 | 派人侦察 → 环节2（不同变体） | 置之不理 → 链终止 |
| 2 | 发现黄巾宝藏 | 全数充公 → 环节3 | 分给百姓 → 环节3（不同变体） | — |
| 3 | 黄巾首领现身 | 生擒招降 → 环节4 | 当场斩杀 → 链终止（获得装备） | — |
| 4 | 招降成功 | 委以重任 → 链完成 | 编入麾下 → 链完成（不同奖励） | — |

**最终奖励（委以重任）**：获得武将「周仓」+ 兵力+1000
**最终奖励（编入麾下）**：获得蓝色装备×1 + 兵力+500

#### 示例三：商路开辟链（3环）

| 环节 | 事件 | 选项A → 后续 | 选项B → 后续 |
|:----:|------|------------|------------|
| 1 | 商人求助 | 伸出援手 → 环节2 | 无暇顾及 → 链终止 |
| 2 | 商路受阻 | 武力清障 → 环节3 | 投资修路 → 环节3（不同变体） |
| 3 | 商路畅通 | 长期合作 → 链完成 | 一次性交易 → 链完成（不同奖励） |

**最终奖励（长期合作）**：解锁常驻商路，每日额外获得铜钱×500 + 粮草×300
**最终奖励（一次性交易）**：铜钱×5000 + 稀有材料×3

### 连锁事件数值规则

| 参数 | 规则 |
|------|------|
| 链长度 | 2-4 环（大部分为 3 环） |
| 环间延迟 | 10-30 分钟随机 |
| 链超时 | 单环等待超过 24 小时未响应 → 链自动终止 |
| 链并发 | 最多同时进行 1 条事件链 |
| 链中断补偿 | 中断时按已完成环节数给予 30% 阶段奖励 |
| 链完成奖励 | 完成全部环节后额外获得最终奖励 |
| 重复规则 | 同一事件链完成后 7 天冷却才可再次触发 |

### 连锁事件与剧情推进

```
剧情事件链 = 特殊连锁事件，与主线章节绑定

  第一章「黄巾之乱」剧情链：
    1-1 黄巾初现 → 1-2 颍川之战 → 1-3 广宗之战 → 1-4 张角决战

  剧情链特性：
    - 不受链并发限制（可与普通链并行）
    - 无环间延迟（连续触发）
    - 必须全部完成才能推进到下一章
    - 选择影响后续章节的NPC关系和可招募武将
```

### 跨系统联动路径

| 当前模块 | 目标模块 | 跳转路径 | 触发方式 |
|---------|---------|---------|---------|
