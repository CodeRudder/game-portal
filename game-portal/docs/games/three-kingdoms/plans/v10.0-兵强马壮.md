## 三、数据流设计

```
装备生成与获取
  │
  ├─ 关卡掉落
  │   ├─ 普通关卡: 白60%/绿30%/蓝8%/紫2%
  │   ├─ 精英关卡: 白30%/绿40%/蓝22%/紫7%/金1%
  │   └─ Boss关卡: 白10%/绿25%/蓝35%/紫22%/金8%
  │
  ├─ 装备炼制
  │   ├─ 基础炼制: 3件同品质 → 概率高一品质
  │   │   ├─ 3白→绿85%/蓝14%/紫1%
  │   │   ├─ 3绿→蓝80%/紫18%/金2%
  │   │   └─ 3蓝→紫75%/金22%
  │   ├─ 高级炼制: 5件 → 概率更优
  │   ├─ 定向炼制: +定向石 → 指定部位
  │   └─ 保底: 连续10次未紫→必紫 / 30次未金→必金
  │
装备属性计算
  │
  ├─ 主属性 = 基础值 × 品质倍率 × (1 + 强化等级 × 0.15~0.18)
  ├─ 副属性 = 随机抽取 × 品质倍率 × (1 + 强化等级 × 0.05~0.08)
  └─ 特殊词条 = 按品质概率附带(白0% / 蓝5% / 紫20% / 金100%)
  │
装备强化
  │
  ├─ 强化流程
  │   ├─ 选择装备 → 检查强化上限(品质决定)
  │   ├─ 消耗: 铜钱(按等级递增) + 强化石(按等级递增)
  │   ├─ 成功率判定: +1~+3必成 / +4~+5高概率 / +6起概率递减
  │   ├─ 成功: 等级+1 → 属性提升 → 动画反馈
  │   └─ 失败: 等级-1(不低于+5) → 失败动画
  │       └─ 使用保护符: 失败不降级(消耗保护符)
  │
  ├─ 自动强化
  │   ├─ 设置目标等级 → 系统自动连续强化
  │   ├─ 停止条件: 材料不足/铜钱不足/达到目标/手动停止
  │   └─ 可选: 自动使用保护符开关
  │
武将穿戴
  │
  ├─ 穿戴流程
  │   ├─ 从背包拖拽装备到武将槽位 → 部位匹配检查
  │   ├─ 穿戴新装备 → 自动卸下旧装备(返回背包)
  │   └─ 装备属性叠加到武将总战力
  │
  ├─ 一键穿戴推荐
  │   ├─ EquipmentRecommendSystem: 分析武将属性短板
  │   ├─ 按品质优先>属性增量优先>等级匹配优先排序
  │   ├─ 展示更换前后属性对比弹窗
  │   └─ 确认后批量穿戴
  │
  └─ 套装效果
      ├─ 检测同武将穿戴的同套装件数
      ├─ 2件: 激活2件套效果
      ├─ 4件: 激活4件套效果
      └─ 同一套装效果不叠加
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 炼制概率引发不满 | 连续炼制失败导致挫败感 | 保底机制(10次必紫/30次必金)+概率公示 |
| 强化降级过于严厉 | 玩家不敢强化高等级装备 | +5以下不降级+保护符可用+金色+12以上不降级 |
| 套装效果平衡性 | 某些套装过强导致同质化 | 7套套装各有侧重(攻击/防御/技能/机动)，持续监控数据 |
| 背包管理负担 | 装备过多时管理繁琐 | 批量分解+品质过滤+一键穿戴推荐+自动整理 |
| 装备数据量 | 大量装备实例占用存储 | 装备数据压缩存储，仅保存关键属性+种子 |
