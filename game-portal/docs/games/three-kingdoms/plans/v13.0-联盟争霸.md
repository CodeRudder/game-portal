## 三、数据流设计

```
联盟系统
  │
  ├─ 联盟管理
  │   ├─ 创建联盟: 消耗元宝×500 + 设定名称+宣言
  │   ├─ 搜索加入: 按名称搜索 / 系统推荐活跃联盟
  │   ├─ 权限体系: 盟主(全部) / 军师(审批+公告) / 成员(基础)
  │   └─ 成员上限: 联盟等级决定(初始20人,每级+5,上限50人)
  │
  ├─ 联盟等级
  │   ├─ 经验来源: 成员日常行为+联盟任务+Boss击杀
  │   ├─ 等级效果: 成员上限↑ / 联盟加成↑ / 商店解锁↑
  │   └─ 加成效果: 全成员资源产出+N% / 远征收益+N%
  │
  ├─ 联盟Boss
  │   ├─ 每日刷新Boss(难度随联盟等级递增)
  │   ├─ 全员参与讨伐(异步,每人每日3次挑战机会)
  │   ├─ 伤害统计 → 排行 → 按伤害比例分配奖励
  │   ├─ 击杀奖励: 全员获得公会币×30 + 天命×20
  │   └─ 未击杀: 按个人伤害发放参与奖
  │
  ├─ 联盟任务
  │   ├─ 每日3个联盟任务(全员共享进度)
  │   ├─ 个人贡献: 参与即获得贡献值
  │   └─ 完成奖励: 联盟经验 + 个人公会币
  │
  └─ 联盟商店
      ├─ 货币: 公会币(每日签到+Boss+任务获取)
      ├─ 商品: 招募令/装备箱/加速道具/武将碎片
      └─ 等级解锁: 联盟等级越高,商品越稀有

PvP赛季深化
  │
  ├─ 赛季周期: 28天(4周)
  ├─ 赛季主题: 每赛季独特视觉风格+专属称号
  ├─ 赛季专属: 头像框(按段位)+称号(王者专属)
  ├─ 结算流程:
  │   ├─ 赛季最后一天: 冻结排名
  │   ├─ 结算动画: 排名变化+段位变化+奖励展示
  │   └─ 奖励发放: 按最高段位→邮箱
  └─ 新赛季: 积分重置为当前段位最低值

活动系统
  │
  ├─ 活动管理
  │   ├─ 活动列表弹窗: 进行中/即将开始/已结束
  │   ├─ 并行上限: 最多5个活动同时进行
  │   │   ├─ 最多1个赛季活动
  │   │   ├─ 最多2个限时活动
  │   │   ├─ 1个日常活动(常驻)
  │   │   └─ 最多1个节日/联盟活动
  │   └─ 活动到期: 未使用代币→10%转铜钱
  │
  ├─ 活动任务
  │   ├─ 每日任务: 5个/天, 0点重置
  │   ├─ 挑战任务: 3个/期, 需手动完成
  │   ├─ 累积任务: 持续累积, 达成即完成
  │   └─ 奖励: 代币+积分+资源
  │
  ├─ 里程碑
  │   ├─ 积分累积 → 按顺序解锁节点
  │   ├─ 手动领取奖励(防止误操作)
  │   └─ 最终节点需手动确认(离线不自动完成)
  │
  ├─ 每日签到
  │   ├─ 7天循环: Day1铜钱→Day7求贤令(大奖)
  │   ├─ 连续加成: 3天+20%/7天+50%代币获取
  │   └─ 补签: 元宝×50/次, 每周最多2次
  │
  └─ 活动离线
      ├─ 赛季活动: 50%效率自动累积积分+代币
      ├─ 限时活动: 30%效率自动累积
      ├─ 日常活动: 100%效率自动累积
      └─ 节日活动: 50%效率自动累积
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 联盟数据同步 | 多人异步操作导致数据冲突 | 使用乐观锁+冲突自动合并+定期同步 |
| 联盟Boss平衡 | Boss过强/过弱 | 按联盟等级动态调整Boss难度 |
| 活动并行复杂度 | 5个活动同时运行增加理解和操作负担 | 活动列表清晰分类+横幅轮播优先级+代币独立 |
| 赛季结算公平性 | 赛季末冲分导致体验恶化 | 最后一天冻结排名+按最高段位结算 |
| 活动代币通胀 | 活动代币囤积导致价值稀释 | 活动到期未使用→10%转铜钱+代币独立不可跨活动 |
