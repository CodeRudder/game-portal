## 三、数据流设计

```
离线收益系统
  │
  ├─ 快照机制
  │   ├─ 玩家下线 → 立即记录快照
  │   │   ├─ 各资源净产出速率
  │   │   ├─ 建筑队列状态
  │   │   ├─ 科技队列状态
  │   │   ├─ 远征队列状态
  │   │   └─ 商队运输状态
  │   └─ 快照有效期72h → 超过封顶时长不再使用
  │
  ├─ 离线计算
  │   ├─ 玩家上线 → 计算离线时长
  │   ├─ 离线≤5分钟 → 静默添加收益，不弹窗
  │   ├─ 离线>5分钟 → 弹出离线收益弹窗
  │   │   ├─ 基础收益 = 净产出速率 × 离线秒数
  │   │   ├─ 效率系数 = 衰减系数 × (1 + Σ加成)
  │   │   │   ├─ 0~2h: 100% | 2~8h: 80% | 8~24h: 60%
  │   │   │   ├─ 24~48h: 40% | 48~72h: 25% | >72h: 15%
  │   │   │   └─ 加成: 科技+30% | VIP+20% | 声望+25% (上限+100%)
  │   │   ├─ 最终收益 = 基础收益 × 效率系数
  │   │   └─ 各系统修正: 建筑×1.2 | 远征×0.85
  │   └─ 资源溢出检测: 有上限资源截断+提示升级
  │
  ├─ 回归流程
  │   ├─ Step1: 离线收益弹窗
  │   │   ├─ 展示离线时长+效率系数+各资源收益
  │   │   ├─ 数字滚动动画(800ms)
  │   │   ├─ 翻倍选项: 广告×2(3次/天) 或 元宝×2(无限制)
  │   │   └─ 一键领取
  │   └─ Step2: 回归综合面板(可跳过)
  │       ├─ 建筑升级完成列表
  │       ├─ 科技研究完成列表
  │       ├─ 远征队归来摘要
  │       ├─ 待处理事件列表
  │       └─ 限时NPC提醒
  │
邮件系统
  │
  ├─ 邮件生成
  │   ├─ 系统邮件: 版本更新/维护通知 → 自动生成
  │   ├─ 奖励邮件: 活动奖励/排行奖励/补偿 → 自动生成+附件
  │   ├─ 社交邮件: 借将通知/援助通知/切磋结果 → 事件触发
  │   └─ 战斗邮件: 攻城战报/竞技场结算 → 战斗结束触发
  │
  ├─ 邮件管理
  │   ├─ 分类Tab: 全部/系统/战斗/社交/奖励
  │   ├─ 状态流转: 未读 → 已读未领 → 已读已领
  │   ├─ 过期处理: 超过保留时长 → 灰色+「已过期」戳记
  │   └─ 批量操作: 全选+一键领取+全部已读+删除已读
  │
  └─ 附件领取
      ├─ 单封: 点击邮件 → 展开详情 → 领取附件
      ├─ 批量: 勾选多封 → 一键领取全部附件
      └─ 领取后: 物品入背包/资源入账 → 状态变为已读已领
```

### 功能验收
- [ ] 离线≤5分钟静默添加收益，不弹窗
- [ ] 离线>5分钟弹出离线收益弹窗，展示完整收益信息
- [ ] 6档衰减系数正确应用（100%→80%→60%→40%→25%→15%）
- [ ] 科技/VIP/声望加成正确叠加，上限+100%
- [ ] 快照机制在下线时立即记录，72h有效期
- [ ] 封顶72h正确执行，超过部分不产出收益

- [ ] 资源溢出截断正确，提示升级对应建筑

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 离线计算数据量大 | 长时间离线后计算耗时 | 使用分段计算，预缓存中间结果 |
| 快照数据丢失 | 浏览器清理localStorage导致快照丢失 | 多重存储：localStorage + IndexedDB |
| 资源溢出玩家不满 | 离线收益因上限截断导致"白挂了" | 弹窗明确提示溢出+引导升级建筑 |
| 邮件系统性能 | 大量邮件导致列表卡顿 | 分页加载，每页20条，虚拟滚动 |
| 翻倍机制公平性 | 免费玩家vs付费玩家收益差距 | 广告翻倍免费可用(3次/天)，元宝翻倍为可选加速 |
