## 二、游戏引擎需求

### 新增子系统
| 子系统 | 职责 | 依赖 |
|--------|------|------|
| NPCPatrolSystem | NPC巡逻管理：路径定义/移动调度/碰撞检测 | NPCPathFollower.ts |
| NPCGiftSystem | NPC赠送管理：物品选择/偏好计算/好感度变化 | NPCNeedSystem.ts |

| QuestTrackerSystem | 任务追踪：实时检测任务进度变化，触发完成 | QuestSystem |

## 三、数据流设计

```
NPC巡逻与高级交互
  │
  ├─ NPC巡逻
  │   ├─ NPCPatrolSystem: 读取路径数据 → 驱动NPC移动
  │   ├─ NPCPathFollower.ts: 沿路径点移动，到达终点后折返
  │   └─ 地图渲染: NPC图标实时更新位置
  │
  ├─ NPC赠送
  │   ├─ 与NPC对话 → 选择「赠送」
  │   ├─ GiftSelectComponent: 显示背包物品，偏好物品标记⭐
  │   ├─ 选择物品 → NPCGiftSystem: 计算好感度（偏好×倍率）
  │   └─ 好感度提升 → 解锁检查 → Toast反馈
  │
  ├─ NPC切磋
  │   ├─ 与武将NPC对话 → 选择「切磋」
  │   ├─ NPCTrainingSystem: 进入简化战斗（无消耗）
  │   ├─ 胜利: 武将获得经验 + 随机道具
  │   └─ 失败: 无惩罚，提示提升方向
  │
  ├─ NPC离线行为
  │   ├─ 离线期间: NPC自动与玩家交互
  │   │   ├─ 商人NPC: 自动交易（出售低价值物品）
  │   │   ├─ 谋士NPC: 提供离线建议（科技推荐）
  │   │   └─ 武将NPC: 自动巡逻（发现资源点）
  │   └─ 回归 → NPC离线行为摘要面板
  │
事件系统深化
  │
  ├─ 连锁事件
  │   ├─ 完成事件A选择选项1 → 触发连锁事件B1
  │   ├─ 完成事件A选择选项2 → 触发连锁事件B2
  │   └─ 连锁链最多3层，避免无限递归
  │
  ├─ 历史剧情事件
  │   ├─ 满足条件触发（章节进度/特定时间）
  │   ├─ 全屏沉浸式展示：水墨背景+角色立绘+对话文本
  │   ├─ 玩家选择影响后续剧情走向
  │   └─ 完成后记录到事件日志
  │
任务系统
  │
  ├─ 任务面板
  │   ├─ Tab分类: 主线/支线/日常
  │   ├─ 任务列表: 名称+进度+奖励预览
  │   └─ 点击任务 → 展开详情 → 跳转按钮
  │
  ├─ 日常任务
  │   ├─ 每日0点刷新: 从20个任务池中随机抽取6个
  │   ├─ QuestTrackerSystem: 实时检测进度
  │   │   ├─ "升级建筑1次" → 监听建筑升级事件
  │   │   ├─ "通关关卡3次" → 监听战斗结算事件
  │   │   └─ "招募武将1次" → 监听招募事件
  │   ├─ 完成任务 → 领取奖励 → 活跃度+N
  │   └─ 活跃度达到阈值 → 解锁活跃度宝箱
  │
  ├─ 任务追踪
  │   ├─ 主界面左上角常驻面板（最多显示3个）
  │   ├─ 实时更新进度: "升级建筑 1/1 ✓"
  │   └─ 点击任务 → 跳转到对应功能界面
  │
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 任务系统涉及面广 | 任务目标覆盖所有子系统 | QuestTrackerSystem使用EventBus监听，各系统只需抛出事件 |
| NPC巡逻渲染性能 | 大量NPC同时移动导致卡顿 | 限制同屏NPC数量，视口外NPC仅逻辑更新不渲染 |
| 连锁事件数据复杂 | 事件链数据量大，配置易出错 | 使用JSON模板+验证器，单元测试覆盖所有链路 |
