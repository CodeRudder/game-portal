## 三、数据流设计

```
科技系统深化
  │
  ├─ 融合科技
  │   ├─ 玩家在两条路线各完成指定节点 → 解锁融合科技
  │   ├─ 融合科技节点显示在路线交叉处
  │   └─ 研究流程与普通科技相同
  │
  ├─ 科技联动
  │   ├─ TechLinkSystem: 注册所有联动关系
  │   │   ├─ 建筑: 经济科技→农田产出+20%
  │   │   ├─ 武将: 军事科技→骑兵攻击+15%
  │   │   └─ 资源: 文化科技→天命获取+10%
  │   └─ 科技完成 → 自动应用联动效果 → 相关系统数值更新
  │
  ├─ 离线研究
  │   ├─ 玩家离线 → 记录离线时间
  │   ├─ TechOfflineSystem: 研究进度 += 离线时间 × 效率衰减
  │   ├─ 回归 → 显示离线研究进度面板
  │   └─ 已完成的科技自动生效
  │
世界地图
  │
  ├─ 地图渲染
  │   ├─ MapRenderer: 读取地图数据 → 渲染地形底纹
  │   ├─ 三大区域着色区分
  │   ├─ 特殊地标标记（城池图标/关卡旗帜/资源点标记）
  │   └─ 己方领土高亮 + 产出气泡
  │
  ├─ 领土管理
  │   ├─ TerritorySystem: 管理所有领土状态
  │   ├─ 己方领土: 产出资源 → 汇入资源系统
  │   ├─ 领土等级: 消耗资源升级 → 产出增加
  │   └─ 产出气泡: 领土上方常驻显示产出值
  │
  └─ 攻城战
      ├─ 选择目标领土（必须相邻己方领土）
      ├─ SiegeSystem: 检查兵力/粮草 → 显示消耗预览
      ├─ 确认攻城 → 进入战斗（复用BattleEngine）
      ├─ 胜利 → 领土归属变更 → 开始产出资源
      └─ 失败 → 返回地图，推荐提升方向
```

## 四、验收标准

### 功能验收
- [ ] 融合科技在两条路线各完成指定节点后解锁
- [ ] 科技联动效果正确应用到建筑/武将/资源系统
- [ ] 离线期间研究继续，效率按衰减系数降低
- [ ] 回归面板显示离线研究进度和完成的科技
- [ ] 科技节点详情弹窗完整展示效果/前置/消耗/时间
- [ ] 世界地图正确渲染三大区域+6种地形+特殊地标
- [ ] 地图支持拖拽浏览和缩放
- [ ] 己方领土高亮显示，产出气泡常驻
- [ ] 领土产出正确汇入资源系统
- [ ] 领土可升级，升级后产出增加
- [ ] 攻城战需满足条件（相邻+兵力+粮草），胜利后领土归属变更
- [ ] 地图筛选功能按区域/地形/占领状态过滤

### 质量验收
- [ ] pnpm run build 成功
- [ ] 游戏逻辑层分支覆盖率 100%
- [ ] UI层分支覆盖率/通过率 100%
- [ ] 无控制台错误

### 可玩性验收
- [ ] 核心循环连通：研究科技→联动效果增强→攻城更容易→占领更多领土→更多资源→更多科技
- [ ] 世界地图给玩家全局视野，领土扩张有成就感
- [ ] 科技联动让玩家感受到"全面发展"的收益

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 世界地图渲染性能 | 大地图Canvas渲染卡顿 | 使用离屏Canvas预渲染地形，仅动态更新领土状态 |
| 科技联动效果复杂 | 效果叠加计算出错 | TechLinkSystem统一注册，各系统通过getTechBonus()查询 |
| 攻城战与关卡战斗差异 | 两套战斗逻辑维护成本高 | 复用BattleEngine，攻城战仅增加城防值计算 |
| 领土产出平衡性 | 领土产出过强/过弱 | 初期保守设定，根据测试数据调整 |
| 地图数据量大 | 三大区域+城池+地形=大量配置 | 使用MapGenerator程序化生成基础地形，手动配置关键地标 |
