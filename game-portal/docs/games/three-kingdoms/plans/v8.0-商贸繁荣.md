## 三、数据流设计

```
商店系统
  │
  ├─ 集市商店
  │   ├─ ShopSystem: 加载商品列表 → 渲染商品网格
  │   ├─ 点击商品 → 选择数量 → 购买确认弹窗
  │   ├─ CurrencySystem: 检查货币余额 → 按优先级扣除
  │   ├─ 购买成功 → 库存更新 → 物品入背包 → Toast反馈
  │   └─ 购买失败 → 价格变红+抖动 → 提示获取途径
  │
  ├─ 补货引擎
  │   ├─ 定时补货(每8h): 常驻补满 + 随机1~3件 + 20%折扣概率
  │   ├─ 离线补货(每8h离线): 最多累积2次 + 10%离线限定稀有
  │   └─ 手动刷新: 消耗500铜钱立即刷新随机商品
  │
贸易路线
  │
  ├─ 商路开通
  │   ├─ 检查前置条件(主城等级+前置商路) → 扣除开通费
  │   ├─ 开通后显示商路连线 → 解锁对应城市商品
  │   └─ 繁荣度初始30% → 完成贸易提升 → 产出倍率变化
  │
  ├─ 商队派遣
  │   ├─ 选择已开通商路 → 选择运输商品(按载重)
  │   ├─ 可选派遣护卫武将(降低遇盗概率50%)
  │   ├─ 商队出发 → 沿路线移动(动画) → 运输时间倒计时
  │   ├─ 到达 → 自动交易 → 货物存入仓库 → 铜钱入账
  │   └─ 离线期间: 继续运输 + 自动完成交易
  │
  ├─ 价格波动
  │   ├─ 每6h刷新: 实际价格 = 基础价格 × (1 ± 波动率)
  │   ├─ 连续涨跌不超过3次 → 防止极端价格
  │   └─ 科技「商业天赋」可查看下次行情趋势
  │
  └─ 贸易事件
      ├─ 每趟运输触发0~2个随机事件
      ├─ 有护卫: 山贼/暴雨自动处理(无损失)
      ├─ 无护卫: 玩家选择处理方式(缴纳/战斗/贿赂)
      └─ 离线默认保守选项(缴纳/等待)

货币体系
  │
  ├─ CurrencySystem: 统一管理8种货币余额
  ├─ 消耗优先级: 按商店类型区分
  │   ├─ 集市: 优先铜钱
  │   ├─ 军需处: 兵力+铜钱组合
  │   ├─ 限时特惠: 仅元宝
  │   └─ NPC交易: 以物易物+铜钱补差
  └─ 不足提示: 价格数字变红+抖动+获取途径引导
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 商店定价平衡性 | 定价过高/过低影响经济循环 | 参考PRD定价表，初期保守设定，根据数据调整 |
| 贸易利润占比失控 | 贸易收入过强导致建筑产出无意义 | 分阶段控制：早期10%→中期30%→后期40% |
| 价格波动极端化 | 连续涨跌导致暴利或暴亏 | 限制连续同方向不超过3次，设置波动上下限 |
| 货币体系复杂度 | 8种货币管理容易出错 | CurrencySystem统一管理，所有操作通过接口 |
| 商队离线计算 | 离线期间多次运输的数据量大 | 限制离线补货次数(最多2次/16h上限) |
