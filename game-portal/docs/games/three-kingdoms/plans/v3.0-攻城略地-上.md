## 二、游戏引擎需求

### 新增子系统
| 子系统 | 职责 | 依赖 |
|--------|------|------|
| BattleEngine | 回合制战斗模拟：速度排序→行动→伤害→状态→胜负判定 | HER(武将属性/技能) |
| BattleRenderer | 战斗场景Canvas渲染：武将站位/血条/怒气条/技能特效 | BattleEngine |
| CampaignProgressSystem | 关卡进度管理：解锁/通关/星级/章节切换 | BattleEngine |
| DamageCalculator | 伤害公式计算：攻击×倍率-防御×减免，含暴击/克制 | HER属性 |
| RewardDistributor | 战斗奖励计算：掉落表抽取+首通奖励+星级加成 | RES资源系统 |

### 改进现有子系统
| 子系统 | 改进内容 |
|--------|----------|
| CampaignSystem.ts | 连通UI，关卡进度→关卡地图渲染→点击关卡→战前布阵 |
| CampaignBattleSystem.ts | 确保战斗计算与BattleEngine接口一致，补充伤害公式 |
| ThreeKingdomsCampaign.ts | 补充6章关卡数据、敌方阵容配置、掉落表 |
| ThreeKingdomsCampaignManager.ts | 连通关卡解锁逻辑（前关通关→后关解锁） |
| GeneralTeamSystem(v2.0) | 战前布阵调用编队接口，支持临时调整阵容 |

### 基础设施
| 组件 | 说明 |
|------|------|
| BattleSceneComponent | 全屏战斗Canvas组件，管理战斗渲染循环 |
| StageMapComponent | 关卡地图组件（横向卷轴/纵向滚动），节点状态渲染 |
| BattleResultComponent | 战斗结算面板组件（胜利/失败分支） |
| FormationPanelComponent | 战前布阵面板组件（拖拽/一键布阵） |

## 三、数据流设计

```
玩家切换到「关卡」Tab
  │
  ├─ 关卡地图渲染
  │   ├─ CampaignProgressSystem: 读取进度 → 当前章节+关卡状态
  │   ├─ StageMapComponent: 渲染关卡节点（未解锁灰/可挑战亮/已通关绿/三星金）
  │   └─ 点击关卡节点 → 进入战前布阵
  │
  ├─ 战前布阵
  │   ├─ FormationPanelComponent: 显示当前编队 → 允许调整
  │   ├─ 一键布阵: 自动选择战力最高武将 → 填满6个位置
  │   ├─ 战力预估: 我方总战力 vs 敌方总战力 → 显示胜率
  │   └─ 点击「出征」→ 进入战斗
  │
  ├─ 战斗过程（自动战斗）
  │   ├─ BattleEngine: 初始化双方阵容 → 按速度排序
  │   ├─ 回合循环:
  │   │   ├─ 当前行动武将 → 选择目标 → 释放技能/普攻
  │   │   ├─ DamageCalculator: 计算伤害（含暴击/克制）
  │   │   ├─ 更新血条/怒气条 → BattleRenderer渲染
  │   │   └─ 检查胜负条件（一方全灭）
  │   └─ 战斗结束 → 进入结算
  │
  ├─ 战斗结算
  │   ├─ 胜利:
  │   │   ├─ RewardDistributor: 计算奖励（基础+首通+星级）
  │   │   ├─ 奖励飞出动画 → 资源/道具入账
  │   │   ├─ 更新关卡状态（通关/星级）
  │   │   └─ 解锁下一关
  │   └─ 失败:
  │       ├─ 显示失败原因（战力不足/阵容被克制）
  │       ├─ 推荐提升方向（升级武将/调整阵容）
  │       └─ 返回关卡地图
  │
  └─ 返回关卡地图 → 进度更新 → 解锁新关卡
```

## 四、验收标准

### 功能验收
- [ ] 关卡地图正确显示6章关卡，节点状态（未解锁/可挑战/已通关/三星）
- [ ] 点击关卡进入战前布阵，显示当前编队+敌方预览
- [ ] 一键布阵自动选择战力最高武将填满阵容
- [ ] 战力预估显示我方vs敌方对比
- [ ] 自动战斗按速度排序行动，伤害计算正确
- [ ] 技能在怒气满时自动释放
- [ ] 兵种克制关系生效（骑兵>步兵>弓兵>骑兵）
- [ ] 胜利结算显示奖励+星级评定，奖励正确入账
- [ ] 失败结算显示原因+推荐提升方向
- [ ] 通关后自动解锁下一关
- [ ] 首通奖励和星级奖励正确区分

### 质量验收
- [ ] pnpm run build 成功
- [ ] 游戏逻辑层分支覆盖率 100%
- [ ] UI层分支覆盖率/通过率 100%
- [ ] 无控制台错误

### 可玩性验收
- [ ] 核心循环连通：编队配置 → 挑战关卡 → 战斗结算 → 获得奖励 → 升级武将 → 挑战更难关卡
- [ ] 玩家能感受到进度：关卡逐个解锁、章节推进、星级提升
- [ ] 战斗有策略感：阵容搭配影响胜负，克制关系可利用

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 战斗引擎实现复杂 | 开发周期超预期 | 先实现自动战斗模式，半自动/手动延后到v4.0 |
| 战斗渲染性能问题 | 全屏Canvas+多武将+特效卡顿 | 使用离屏Canvas预渲染，限制同屏特效数量 |
| 关卡数据量不足 | 6章×多关=大量阵容配置 | 先完成前3章详细数据，后3章使用模板生成 |
| 战力预估不准确 | 玩家按推荐挑战仍失败 | 预估加入克制系数修正，显示推荐胜率而非绝对值 |
| 战斗节奏太慢 | 自动战斗观看体验差 | 默认2倍速，支持跳过（已通关关卡） |
