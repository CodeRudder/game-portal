## 🏗️ 测试架构

### 使用的真实对象（无 mock）
- **DataMigrator**: 版本迁移器，执行 v0→v1→...→v16 迁移链
- **repairWithBlueprint**: 蓝图修复器，递归补全缺失/错误字段
- **GameDataValidator**: 数据校验器，验证修复后数据合法性
- **ThreeKingdomsEngine**: 真实引擎实例，端到端反序列化验证
- **GameEventSimulator**: 测试辅助，提供 tick/建筑升级等操作

### 迁移链覆盖
```
v0 → v1 → v2 → v3 → v4 → v5 → v7 → v10 → v14 → v15 → v16
     ↑     ↑     ↑     ↑     ↑     ↑      ↑      ↑      ↑
   基础  武将  关卡  科技  装备  PvP  装备炼制 声望  离线  赛季
```

### 测试策略
1. **单元级**: 每步迁移函数独立验证
2. **集成级**: 迁移 + 蓝图修复 + 校验联动
3. **端到端级**: 迁移 → 修复 → 引擎反序列化 → tick → 保存

## 🐛 发现的边界条件

| 编号 | 发现 | 严重程度 | 状态 |
|------|------|----------|------|

## ✅ 结论

存档迁移全链路测试 62 个用例全部通过，覆盖：
- **版本迁移**: v0 到 v16 全版本链式迁移
- **蓝图修复**: 缺失字段补全、类型错误修正、NaN/Infinity/负数处理
- **数据完整性**: 迁移不丢失原始数据、额外字段向前兼容
- **优雅降级**: null/undefined/非对象/损坏数据安全处理
- **性能约束**: 迁移后数据大小不爆炸增长
- **端到端验证**: 迁移后引擎可正常初始化、tick、建筑操作、序列化保存
