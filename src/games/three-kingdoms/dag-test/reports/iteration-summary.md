## 2. Phase 2 改进详情

### 2.1 状态覆盖率提升方案

#### 根因分析
Phase 1结束时状态覆盖率仅80.5%，是5个维度中最低的。根因分析发现：
1. **STATE_PATTERNS匹配不足**：原有20个模式仅匹配76.7%的测试文件
2. **缺失状态节点**：缺少受伤/恢复/满级/被拒/放弃/暂停/扫荡等关键状态
3. **测试覆盖不完整**：254个测试文件不包含任何状态相关断言

#### 修复措施

| 措施 | 详情 | 效果 |
|------|------|------|
| 扩展STATE_PATTERNS | 从20个模式扩展至100+个模式，覆盖UI组件、游戏引擎、Mock等 | 匹配率76.7%→99.9% |
| 新增状态节点 | 9个新节点：building:max-level, hero:injured, hero:recovering等 | 节点86→95 |
| 新增状态转换 | 14条新边：受伤→恢复、申请被拒、科技暂停等 | 边83→97 |
| 新增测试文件 | state-dag-p2-paths.test.ts，69个测试用例 | 覆盖17个实体完整状态链 |
