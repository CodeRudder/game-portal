## 3. 任务计划

### Task 1 (P1): 行军精灵持续时间约束 (I11 Enhancement)

**目标**: 实现行军动画精灵的持续时间 clamp 约束, 符合 PRD 需求

**背景**: R15 已将行军精灵持续时间约束写入 PRD 和 flow 文档 (min 10s, max 60s). R16 进行了测试准备但代码未实现 (A2 P1 确认). R17 负责代码落地.

**实现要点**:
- 行军精灵创建时计算实际动画持续时间
- 若计算时长 < 10s, 强制使用 10s 最小值 (短距离行军)
- 若计算时长 > 60s, 强制使用 60s 最大值 (长距离行军)
- 在 `PixelWorldMap.tsx` march sprite 创建逻辑或 `SiegeBattleAnimationSystem.startSiegeAnimation()` 中应用约束
- 使用 `clamp(duration, MIN_MARCH_DURATION, MAX_MARCH_DURATION)` 模式

**影响范围**: `PixelWorldMap.tsx` march sprite 创建 / `SiegeBattleAnimationSystem.ts` 动画时长
**新增测试**: >= 4 个 (短距离 clamp / 长距离 clamp / 正常范围不 clamp / 边界值)
**验证**: 行军精灵动画时长在 [10s, 60s] 范围内

### Task 2 (P1): E1-3 行军 E2E 全链路测试

**目标**: 行军全链路端到端测试覆盖, 从发起行军到抵达目的地的完整流程

4. 多路并发行军: 各自独立完成
5. 行军取消: 中途取消, 资源回收

**影响范围**: 新增/扩展集成测试文件
**新增测试**: >= 5 个

### Task 3 (P0): I3 攻城锁定机制

**目标**: 实现攻城锁定机制, 防止并发攻城冲突

**核心逻辑**:

- 攻城发起时检查锁定状态, 已锁定则拒绝或排队
- 锁定状态持久化 (引擎层状态管理)
- 超时自动释放 (防止死锁)
- 攻城完成后释放锁定

**影响范围**: 攻城相关引擎模块 + WorldMapTab 攻城发起逻辑
**新增测试**: >= 3 个 (并发锁定测试 / 超时释放测试 / 锁定状态查询测试)
**验证**: 并发攻城被正确拒绝; 攻城完成后锁定释放; 超时自动释放
