## 对抗性评测重点

1. **链路完整性**: Phase 1→2→3→4是否无缝串联，无断裂点
2. **状态转换连续性**: SiegeTask状态机 preparing→marching→sieging→settling→returning→completed 是否每个转换都有代码路径
3. **事件驱动链路**: 确认每个阶段转换由正确的事件触发（非手动advanceStatus）
4. **数据一致性**: 攻城前后兵力/资源/领土数据是否正确变化
5. **并发安全**: 多个攻城任务并行时数据是否隔离
6. **策略差异化**: 不同策略的实际效果差异是否在E2E中体现
