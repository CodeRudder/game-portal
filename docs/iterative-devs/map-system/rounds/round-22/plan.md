# Round 22 计划

> **迭代**: map-system (攻城主流程集成测试)
> **轮次**: Round 22
> **来源**: FL-MAP-09 攻城主流程 — Phase 1 攻占准备
> **日期**: 2026-05-05

## 本轮焦点

> **Phase 1: 攻占准备 (P1~P5)** — 以攻城主流程为线索，验证从选择城池到创建攻占任务的完整链路

**用户线性链**: 选择城池/资源点 → 点击攻占 → 攻占弹框 → 条件校验 → 选择/创建编队 → 选择攻城策略 → 创建攻占任务 → 显示任务信息

### Stage P1: 选择攻城目标

| 序号 | 检查项 | 验收标准 |
|:----:|--------|---------|
| P1-1 | 城池点击交互 | 点击敌方/中立城池，高亮选中，弹出领土详情面板 |
| P1-2 | 己方城池分流 | 点击己方城池进入领土详情，不触发攻城 |
| P1-3 | 资源点分流 | 点击资源点进入领土征服流程 |
| P1-4 | 攻占按钮 | 领土详情面板包含"攻占"按钮 |

### Stage P2: 攻城确认弹窗

| 序号 | 检查项 | 验收标准 |
|:----:|--------|---------|
| P2-1 | 条件校验(领土相邻) | 有道路连接才允许攻城 |
| P2-2 | 条件校验(兵力门槛) | 兵力满足最低要求 |
| P2-3 | 条件校验(粮草充足) | 粮草 >= 基础消耗(5/秒) x 行军时间(秒) x (1 + 编队兵力/1000) |
| P2-4 | 条件校验(每日次数) | 未达上限(3次/天，00:00重置) |
| P2-5 | 条件校验(攻城冷却) | 同一目标4小时冷却已过 |
| P2-6 | 条件校验(并发上限) | 活跃任务 < 3 |
| P2-7 | 校验失败提示 | 红色提示未通过项 |
| P2-8 | 多条失败推荐 | 按优先级推荐解决方案 |

### Stage P3: 选择/创建编队

| 序号 | 检查项 | 验收标准 |
|:----:|--------|---------|
| P3-1 | 编队选择面板 | 嵌入确认弹窗，显示可选编队 |
| P3-2 | 将领选择 | 可选择可用将领 |
| P3-3 | 兵力分配 | 可分配兵力到编队 |
| P3-4 | 战力预览 | 实时显示编队战力 |
| P3-5 | 空状态处理 | 所有将领不可用时显示恢复倒计时 |

### Stage P4: 选择攻城策略

| 序号 | 检查项 | 验收标准 |
|:----:|--------|---------|
| P4-1 | 4种策略卡片 | 强攻/围困/夜袭/内应可选 |
| P4-2 | 策略效果预览 | 显示策略修正系数 |
| P4-3 | 夜袭令道具检查 | 无夜袭令时卡片置灰 |
| P4-4 | 内应信三态 | 可点击/暴露冷却/道具不足 |
| P4-5 | 策略选择确认 | 选择后更新确认弹窗 |

### Stage P5: 创建攻城任务

| 序号 | 检查项 | 验收标准 |
|:----:|--------|---------|
| P5-1 | 任务创建原子性 | 资源扣除+任务创建原子操作 |
| P5-2 | SiegeTask状态 | preparing → marching |
| P5-3 | 任务面板更新 | 新增任务卡片(编队摘要+状态+预计时间) |
| P5-4 | 攻城中标记 | 目标城池标记"攻城中" |
| P5-5 | 攻城锁定 | 攻城中其他编队不可重复攻 |
| P5-6 | 过渡动画 | 弹窗关闭+Toast+视口平移+精灵高亮 |

## 对抗性评测重点

- [ ] P2条件校验的完整性和顺序（6条校验是否全部实现）
- [ ] P3编队与P4策略的数据流是否真实串联（非mock）
- [ ] P5任务创建的原子性（资源扣费失败时是否回滚）
- [ ] Phase 1→Phase 2过渡是否无缝（P5→P6状态流转）
- [ ] 异常分支覆盖（ERR-S01~S05是否全部处理）
