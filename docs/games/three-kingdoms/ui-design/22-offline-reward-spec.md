# 离线收益统一规范（v1.0）

> **文档定位**：全局规范文档。所有模块的离线行为统一引用本文档，消除模块间定义冲突。
> **创建背景**：Round 2 评测发现 P0-1 问题——资源系统（24h封顶/5级衰减）与建筑系统（72h封底/4级衰减）两套衰减曲线完全不同，导致开发混乱。本文档统一定义全局离线效率系数和各系统离线行为规则。
> **引用关系**：08-resource-system.md、06-building-system.md、09-battle-system.md、10-expedition-system.md、10-event-system.md、08-npc-system.md 等模块的离线相关内容均以本文档为准。

## 一、设计原则

### 1.1 核心理念

> **放置游戏的灵魂**：玩家离线时游戏仍在"进行"，回归时获得满足感。离线收益的设计目标是让玩家"想回来"而非"必须回来"。

| 原则 | 说明 |
|------|------|
| **统一性** | 所有系统使用同一套离线效率系数公式，避免模块间冲突 |
| **公平性** | 离线收益始终低于在线收益（效率衰减），鼓励在线互动 |
| **可预测性** | 玩家可通过离线预估面板提前知道收益，降低焦虑感 |
| **防挂机** | 离线效率随时间递减，长时间离线收益边际递减 |
| **回归奖励** | 离线本身不应惩罚玩家，而是给予"欢迎回来"的正向体验 |

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| **离线时间** | 从玩家最后活跃时间到重新上线的时间差（秒） |
| **离线效率系数** | 离线收益占在线收益的比例，随离线时长递减 |
| **封顶时长** | 离线收益计算的最大时长，超过此时间不再产出 |
| **基础衰减** | 无任何加成时的离线效率系数 |
| **加成系数** | 科技/VIP/声望等提供的额外效率提升 |
| **快照值** | 玩家下线时各系统状态的记录值，作为离线计算的基准 |

## 二、全局离线效率系数 ★核心公式★

### 2.1 效率系数公式

```
离线效率系数 = 基础衰减系数 × (1 + Σ加成系数)

其中：
- 基础衰减系数：由离线时长决定的分段值（见2.2）
- 加成系数：科技 + VIP + 声望（叠加计算，上限+100%）
- 最终效率系数上限：100%（即使加成后也不超过在线效率）
```

### 2.2 基础衰减系数表

```
┌──────────────────┬──────────┬────────────────────────────┐
│  离线时长         │ 效率系数  │ 说明                       │
├──────────────────┼──────────┼────────────────────────────┤
│  0 ~ 2 小时      │  100%    │ 全额收益，短期离线无惩罚    │
│  2 ~ 8 小时      │   80%    │ 睡眠时段，轻度衰减          │
│  8 ~ 24 小时     │   60%    │ 工作日时段，中度衰减        │
│  24 ~ 48 小时    │   40%    │ 周末时段，明显衰减          │
│  48 ~ 72 小时    │   25%    │ 长期离线，大幅衰减          │
│  > 72 小时       │   15%    │ 封底值，防止无限积累        │
└──────────────────┴──────────┴────────────────────────────┘

封顶时长：72小时
超过72小时的离线时间不再产出任何收益
```

**衰减曲线可视化**：
```
效率%
100 ┤████████████
 80 ┤            ████████████████
 60 ┤                            ██████████████████████████
 40 ┤                                                        ████████████████
 25 ┤                                                                          ████████
 15 ┤                                                                                   ██████...
    └──────────────────────────────────────────────────────────────────────────────────→ 离线时长
    0h      2h        8h            24h                  48h              72h
```

### 2.3 加成系数

| 加成来源 | 加成值 | 累计上限 | 说明 |
|---------|:------:|:-------:|------|
| **科技加成** | +10%/+15%/+20% | +30% | 科技「离线收益I/II/III」 |
| **VIP加成** | +5%/+10%/+20% | +20% | VIP1/VIP3/VIP5 |
| **声望加成** | +3%/+5%/+10%/+25% | +25% | 声望等级1/3/5/10 |
| **总计上限** | — | **+100%** | 所有加成叠加后不超过+100% |

**加成计算示例**：
```
玩家A：科技II(+15%) + VIP3(+10%) + 声望Lv5(+10%)
加成系数 = 15% + 10% + 10% = 35%

离线8小时：效率 = 60% × (1 + 0.35) = 81%
离线24小时：效率 = 40% × (1 + 0.35) = 54%
```

### 2.4 各时段设计意图

| 时段 | 覆盖场景 | 设计意图 |
|------|---------|---------|
| 0-2h 100% | 午休、短暂离开 | 短期离线无惩罚，保证日常体验 |
| 2-8h 80% | 夜间睡眠 | 睡眠期间仍获较高收益，"睡一觉有收获" |
| 8-24h 60% | 工作日白天 | 鼓励午休或下班后上线一次 |
| 24-48h 40% | 周末/出差 | 仍有可观收益，不强制每日上线 |
| 48-72h 25% | 短期旅行/忙碌 | 收益大幅衰减，温和提醒回归 |
| >72h 15%封底 | 长期未登录 | 保底收益，回归时仍有"欢迎回来"的满足感 |

## 三、各系统离线行为统一规则

### 3.1 资源系统离线行为

| 属性 | 规则 |
|------|------|
| 计算公式 | 各资源净产出速率 × 离线秒数 × 离线效率系数 |
| 快照基准 | 下线时的净产出速率（产出-消耗） |
| 封顶时长 | 72小时（全局统一） |
| 溢出处理 | 有上限资源（粮草/兵力）溢出部分截断，不丢失 |
| 无上限资源 | 铜钱/天命无溢出风险，全额发放 |
| 特殊 | 天命产出需要天命阁建筑（转生后解锁） |

### 3.2 建筑系统离线行为

| 属性 | 规则 |
|------|------|
| 计算公式 | 建筑升级剩余时间 - (离线秒数 × 离线效率系数) |
| 效率系数 | 引用全局公式（见§2） |
| 封顶时长 | 72小时（全局统一） |
| 完成处理 | 离线期间完成的建筑升级正常结算 |
| 队列处理 | 建筑队列中的后续任务自动接续 |
| 特殊 | 建筑升级离线效率 = 资源效率 × 1.2（建筑升级有额外加成） |

### 3.3 科技系统离线行为

| 属性 | 规则 |
|------|------|
| 计算公式 | 科技研究剩余时间 - (离线秒数 × 离线效率系数) |
| 效率系数 | 引用全局公式（见§2） |
| 封顶时长 | 72小时（全局统一） |
| 完成处理 | 离线期间完成的科技研究正常结算 |
| 特殊 | 科技离线效率 = 资源效率 × 1.0（无额外加成） |

### 3.4 远征系统离线行为

| 属性 | 规则 |
|------|------|
| 计算公式 | 远征剩余时间 - (离线秒数 × 离线效率系数 × 0.85) |
| 效率系数 | 全局效率 × 0.85（战斗惩罚系数） |
| 封顶时长 | 72小时（全局统一） |
| 胜率计算 | 离线战斗胜率 = 在线胜率 × 0.85（无法使用技能/策略） |
| 战利品 | 按胜率比例发放，失败获得30%保底奖励 |
| 特殊 | 自动使用默认编队，不消耗扫荡令 |

### 3.5 事件系统离线行为

| 属性 | 规则 |
|------|------|
| 堆积上限 | 最多堆积5个未处理事件 |
| 保留时长 | 72小时（超时自动以"保守方案"处理） |
| 处理规则 | 离线期间事件不自动处理，回归后由玩家手动处理 |
| 紧急事件 | 仅在在线时触发，离线不生成紧急事件 |
| 负面效果 | 离线时间**不计入**负面效果持续时间（暂停计时） |
| 特殊 | 回归时如有堆积事件，在离线收益弹窗中提示 |

### 3.6 NPC系统离线行为

| 属性 | 规则 |
|------|------|
| NPC刷新 | 离线期间NPC正常刷新（按在线刷新周期） |
| 好感度衰减 | 离线期间好感度不衰减（暂停计时） |
| 切磋/拜访 | 离线期间不自动执行，回归后手动操作 |
| 特殊NPC | 限时NPC（如云游商人）离线期间正常计时，可能错过 |
| 特殊 | 回归时如有即将消失的限时NPC，军师建议优先提示 |

### 3.7 战斗系统离线行为

| 属性 | 规则 |
|------|------|
| 自动推图 | 开启"自动推图"后，离线期间自动推进已三星通关的关卡 |
| 效率系数 | 全局效率 × 0.80（战斗效率略低于在线） |
| 胜率 | 自动推图胜率 = 玩家战力 / 关卡推荐战力（上限100%） |
| 三星判定 | 离线推图不获得三星评价（仅一星通关） |
| 奖励 | 按一星标准发放奖励（约在线三星的60%） |
| 特殊 | 未三星通关的关卡不会离线推进 |

### 3.8 各系统离线行为汇总表

| 系统 | 效率系数 | 封顶时长 | 离线行为 | 回归提示 |
|------|:-------:|:-------:|---------|---------|
| 资源产出 | 全局系数 | 72h | 自动产出 | 弹窗展示收益 |
| 建筑升级 | 全局×1.2 | 72h | 自动推进 | "X座建筑升级完成" |
| 科技研究 | 全局×1.0 | 72h | 自动推进 | "X项科技研究完毕" |
| 远征战斗 | 全局×0.85 | 72h | 自动推进 | "远征队归来" |
| 事件系统 | — | 72h堆积 | 不自动处理 | "有X个待处理事件" |
| NPC系统 | — | — | 仅刷新 | "限时NPC即将离开" |
| 自动推图 | 全局×0.80 | 72h | 自动推进 | "已推进至第X关" |

## 四、离线收益展示统一规范

### 4.1 展示优先级

离线收益信息按以下优先级展示：

| 优先级 | 信息类型 | 展示位置 | 必须展示 |
|:------:|---------|---------|:-------:|
| P0 | 资源收益数值 | 离线收益弹窗主体 | ✅ |
| P0 | 离线时长 & 效率系数 | 弹窗标题区 | ✅ |
| P1 | 封顶提示（>72h） | 弹窗顶部警告条 | 条件触发 |
| P1 | 容量溢出风险 | 资源行内警告 | 条件触发 |
| P2 | 收益来源占比 | 产出明细区 | ✅ |
| P2 | 建筑完成通知 | 弹窗底部 | 条件触发 |
| P2 | 科技完成通知 | 弹窗底部 | 条件触发 |
| P3 | 领土变化摘要 | 弹窗领土区 | 条件触发 |
| P3 | 任务完成通知 | 弹窗底部 | 条件触发 |

### 4.2 数值格式规范

| 数据类型 | 格式 | 示例 |
|---------|------|------|
| 资源收益 | +{数值}，千位逗号 | +12,450 |
| 离线时长 | {X}时{X}分 | 8时32分 |
| 效率系数 | 百分比 | 60% |
| 容量百分比 | 百分比 + 进度条 | 82% ████░ |
| 收益来源 | 百分比 | 建筑68% · 领土25% |

### 4.3 动画规范

| 动画 | 时长 | 缓动 | 说明 |
|------|:----:|------|------|
| 弹窗弹入 | 300ms | ease-out-back | PC端缩放 / 手机端滑入 |
| 遮罩淡入 | 200ms | ease-out | — |
| 数字滚动 | 800ms | ease-out | 从0到最终值 |
| 资源飞入 | 500ms | ease-in | 粒子飞向资源栏 |
| 弹窗关闭 | 400ms | ease-in | PC端卷起 / 手机端下滑 |

## 五、离线回归欢迎流程

### 5.1 流程设计 ★统一规范★

> **核心改进**：Round 2 发现回归欢迎流程5步弹窗序列过长（R2-P1-1），且与离线收益弹窗数据展示重叠（R2-P1-10）。统一为2步流程。

```
[玩家上线]
     │
     ├── 离线 ≤ 5分钟 → [静默添加收益] → [正常游戏]
     │
     └── 离线 > 5分钟 → [Step 1: 离线收益弹窗]
                              │
                              │ （必须领取后关闭）
                              ↓
                        [Step 2: 回归综合面板]
                              │
                              │ （展示其他系统变化摘要）
                              ↓
                        [正常游戏]
```

### 5.2 Step 1：离线收益弹窗

- **内容**：资源收益 + 效率系数 + 封顶提示 + 溢出警告
- **必须操作**：点击"一键领取"关闭
- **不可跳过**：强化获得感仪式感
- **详细设计**：见08-resource-system.md §四

### 5.3 Step 2：回归综合面板

- **内容**：其他系统的离线变化摘要
- **可跳过**：点击"知道了"或点击遮罩关闭
- **展示信息**：

| 信息 | 展示条件 | 展示形式 |
|------|---------|---------|
| 建筑升级完成 | 有建筑在离线期间完成 | "🏗️ X座建筑升级完成" + 查看详情链接 |
| 科技研究完成 | 有科技在离线期间完成 | "🔬 X项科技研究完毕" + 查看详情链接 |
| 远征队归来 | 有远征队在离线期间返回 | "⚔️ 远征队归来，获得..." + 战利品摘要 |
| 自动推图进度 | 开启了自动推图 | "🗺️ 已推进至第X关" |
| 待处理事件 | 有堆积事件 | "📋 有X个待处理事件" + 去处理链接 |
| 限时NPC | 有即将消失的NPC | "👤 {NPC名}即将在X小时后离开" |
| 领土变化 | 有领土得失 | "🗺️ ✅占领X块 ❌失去X块" + 查看地图链接 |
| 版本更新 | 离线期间有更新 | "🎉 版本更新：{版本号}" + 更新内容链接 |

### 5.4 回归综合面板 UI

**PC端**：
```
+--[回归报告]─────────────────────────────────────+
|                                                  |
|  🏗️ 兵营升级至Lv5 · 铁匠铺升级至Lv3            |
|  🔬「兵法精通」研究完毕                          |
|  ⚔️ 远征队归来，获得 💰2,400 🌾800              |
|  🗺️ 已自动推进至第12关                           |
|  📋 有3个待处理事件                              |
|                                                  |
|              [ 知道了 ]                           |
+--------------------------------------------------+
```

- 尺寸：400×auto（最大400px高度）
- 样式：简化版卷轴面板，无遮罩（Step 1已关闭遮罩）
- 动画：淡入（200ms）

**手机端**：
```
+--[回归报告]─────────────+
|                          |
|  🏗️ 2座建筑升级完成     |
|  🔬 1项科技研究完毕     |
|  ⚔️ 远征队归来          |
|  📋 3个待处理事件       |
|                          |
|  [ 知道了 ]              |
+--------------------------+
```

- 形式：Bottom Sheet
- 每条信息一行，可点击展开详情
- 动画：从底部滑入（300ms）

### 5.5 回归流程时间预算

| 阶段 | 时长 | 说明 |
|------|:----:|------|
| Step 1 弹窗弹入 | 0.3s | — |
| Step 1 数字滚动 | 0.8s | 资源数值动画 |
| Step 1 玩家操作 | ~2s | 阅读收益 + 点击领取 |
| Step 1 领取动画 | 0.5s | 资源飞入 |
| Step 1 关闭动画 | 0.4s | 弹窗卷起 |
| Step 2 面板淡入 | 0.2s | — |
| Step 2 玩家操作 | ~1s | 快速浏览 + 关闭 |
| **总计** | **~5.2s** | 比v2.2的5步流程（~15s）大幅缩短 |

### 6.1 预估功能
