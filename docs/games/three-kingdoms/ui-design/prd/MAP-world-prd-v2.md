## [MAP-1] 地图渲染规则 {#map-1}

### 1.1 像素风格定义

**参考游戏**: 三国志(FC/SNES)、文明(像素版)、火焰纹章、高级战争

**渲染规格**:

| 参数 | 值 | 说明 |
|------|---|------|
| 色块尺寸 | 8×8px | 基础像素块，可缩放至4×4或16×16 |
| 地图总尺寸 | 320×240色块(2560×1920px) | 参考FC/SNES分辨率 |
| 渲染方式 | Canvas 2D / PixiJS Sprite | 逐色块绘制，非网格布局 |
| 色彩风格 | 16色调色板 | 复古像素风格，限制色彩数量 |
| 缩放范围 | 0.5x ~ 3.0x | 支持细节查看和全局预览 |

**设计原则**:
- 不使用网格布局，地图为有机自然的地形分布
- 城市之间通过道路直观连接，道路即为可达路径
- 地形过渡自然(渐变色块，非硬边界)
- 城市有独特像素图标，体现三国特色

### 1.2 调色板定义

```
// 地形色板(16色)
水 深: #1a3a5c  水 浅: #2d6b8a
山 深: #5a4a3a  山 浅: #8b7355
平 原: #7ec850  草 地: #a8d88a
森 林: #2e5a2e  树 冠: #3a7a3a
沙 漠: #d4b896  黄 土: #c4a876
城 墙: #8b6914  屋 顶: #c0392b
道 路: #b8a07a  土 路: #9a8a6a
雪 地: #e8e8f0  冰 面: #a8c8e8
```

### 1.3 地形类型(像素渲染)

| 地形 | 色块特征 | 渲染规则 | 战斗效果 |
|------|---------|---------|---------|
| 平原 | 绿色渐变(3种绿) | 随机散布浅色像素点(草地纹理) | — |
| 山地 | 棕色+灰色 | 三角形轮廓+阴影面(左下深色) | 防守+20% |
| 水域 | 蓝色渐变 | 波浪纹理(每帧偏移1px) | 不可通行 |
| 森林 | 深绿+浅绿 | 树冠圆形+树干竖线(随机密度) | 伏兵+15% |
| 城池 | 灰墙+红顶 | 城墙轮廓+城门+旗帜(阵营色) | 城防加成 |
| 关隘 | 灰墙+棕门 | 狭窄通道+两侧山壁 | 防御+40% |
| 道路 | 土黄色 | 3px宽路径+边缘阴影 | 行军加速 |

## [MAP-3] 行军动画系统 {#map-3}

### 3.1 行军精灵设计

**行军单位精灵**: 8×8像素小人队列，沿道路移动。

```
// 行军精灵帧(8×8)
帧1(左):  🏃💨    // 左行军
帧2(左):  🏃💨    // 左行军(腿交替)
帧3(右):  💨🏃    // 右行军
帧4(右):  💨🏃    // 右行军(腿交替)

// 队列显示: 最多显示5个小人，间距4px
// 兵力>1000: 5个小人
// 兵力500~1000: 3个小人
// 兵力<500: 1个小人
```

**行军动画参数**:

| 参数 | 值 | 说明 |
|------|---|------|
| 移动速度 | 2px/帧(60fps) | 基础速度 |
| 道路加成 | ×1.5 | 在道路上移动加速 |
| 山地减速 | ×0.5 | 穿越山地时减速 |
| 动画帧率 | 8fps | 精灵动画帧率 |
| 轨迹显示 | 半透明路径 | 行军路线可视化 |

### 3.2 行军路线预览

**点击城市后显示**:
1. 路径高亮(黄色虚线沿道路)
2. 预计到达时间(基于距离和地形)
3. 军队信息面板(兵力/将领/士气)
4. 路径上的地形标注(平原/山地/水域渡口)

**路线预览UI**:
```
┌─ 行军预览 ─────────────────────┐
│  目标: 许昌                      │
│  路线: 洛阳 → 虎牢关 → 许昌     │
│  距离: 480px(约48格)             │
│  预计时间: 2分30秒               │
│  地形: 平原(20%) + 关隘(80%)    │
│                                  │

│  战力: 4500                      │
│                                  │
│  [确认行军] [取消]               │
└──────────────────────────────────┘
```

### 3.3 行军状态机

```
IDLE → MARCHING → ARRIVED → (AUTO_SIEGE / IDLE)
         ↓
      INTERCEPTED → BATTLE → (VICTORY / DEFEAT)

      RETREATING → IDLE
```

## [MAP-4] 攻城动画系统 {#map-4}

### 4.1 攻城过程动画

**阶段1: 集结**(3秒)
- 军队精灵从四方汇集到城下
- 城墙上守军旗帜飘动
- 战鼓音效

**阶段2: 攻城**(5~15秒，取决于城防)
- 强攻: 撞车撞击城门动画 + 箭雨
- 围困: 军队围城 + 城内炊烟减少
- 夜袭: 夜色渐暗 + 火把 + 翻墙
- 内应: 城门打开 + 军队涌入

**阶段3: 结果**(2秒)
- 胜利: 城墙旗帜更换(己方色) + 欢呼动画
- 失败: 军队撤退 + 城墙守军欢呼

### 4.2 攻城特效精灵

```
// 撞车(16×8)
帧1: [==🚂]    // 撞车前进
帧2: [==🚂💥]  // 撞击城门
帧3: [==🚂]    // 撞车后退

// 箭雨(8×16, 多层)
帧1: ↗↗↗↗     // 箭矢飞出
帧2: ↓↓↓↓     // 箭矢落下
帧3: 💥💥💥💥  // 命中效果

// 火把(8×8)
帧1: 🔥        // 火焰高
帧2: 🔥        // 火焰低(交替)

// 城门破碎(16×16)
帧1: [████]    // 完整城门
帧2: [█▓▒░]    // 裂痕
帧3: [░  ░]    // 破碎
```

### 4.3 攻占成功/失败动画

**成功动画**(3秒):
1. 城墙旗帜降下(敌方色)
2. 己方旗帜升起(带飘动)
3. 城门打开
4. 军队入城
5. 城市名+归属变化文字弹出

**失败动画**(2秒):
1. 军队后退
2. 城墙守军欢呼
3. 损失兵力数字弹出
4. "攻城失败"文字

## [MAP-7] 编队系统 {#map-7}

### 7.1 编队定义

**出征编队(ExpeditionForce)**: 用于攻城/资源占领的战斗单位。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 编队ID |
| heroId | string | 将领ID（必须） |
| troops | number | 士兵数量（必须 > 0） |
| status | 'ready' \| 'marching' \| 'fighting' \| 'returning' | 编队状态 |
| targetId | string? | 目标领土ID |

### 7.2 编队约束

| 约束 | 错误码 | 说明 |
|------|--------|------|
| 无将领 | HERO_REQUIRED | 必须选择一个将领 |
| 无士兵 | TROOPS_REQUIRED | 必须分配士兵 |
| 将领重复 | HERO_BUSY | 将领已在其他编队中 |
| 将领受伤 | HERO_INJURED | 将领正在受伤恢复中 |
| 兵力不足 | INSUFFICIENT_TROOPS | 士兵数量不足 |
| 编队满 | MAX_FORCES_REACHED | 最多3个编队 |

### 7.3 编队UI

**位置**: 攻城确认弹窗中
**内容**:
- 可用将领列表（排除受伤和繁忙的）
- 兵力滑块（0~可调用上限）
- 编队确认按钮（校验约束后才能点击）
