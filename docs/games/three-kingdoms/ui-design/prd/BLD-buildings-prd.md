### 建筑依赖关系

```
                         ┌─────────┐
                         │  主城Lv1 │ ← 初始解锁
                         └────┬────┘
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
    农田/市集/矿场/伐木场 Lv1            兵营Lv2      ← 初始/主城Lv2 解锁
                                             │
         ┌───────────────┬───────────────────┤
         ▼               ▼                   ▼
      书院Lv3         工坊Lv3             医馆Lv4       ← 主城Lv3/4 解锁
         │               │
         └───────┬───────┘
                 ▼
         城墙Lv5 + 酒馆Lv5                            ← 主城Lv5 解锁
```

**核心规则**：每座建筑等级 ≤ 主城等级（主城 Lv10 时，其他建筑最高 Lv10）。

| 步骤 | 操作 | 目的 |
|:----:|------|------|
| 1 | 主城 → Lv5 | 解锁城墙、酒馆，全资源 +8% |

#### 中后期（第 7 天+）

| 策略 | 说明 |
|------|------|
| 主城优先 | 主城每级 +2% 全资源加成，ROI 最高 |
| 产出均衡 | 农田/市集/矿场/伐木场保持同步等级 |

| 贸易链 | 市集→市舶司形成贸易链，铜钱产出倍增 |
| 英雄招募 | 酒馆升级提升武将招募概率，中期重点 |
| 防御不急 | 城墙/医馆在 PvP 内容开放前可延后 |

### 资源死锁避免

**常见死锁场景**：

```
死锁：主城升级需要粮草 → 粮草不够需要升级农田 → 农田升级需要主城等级更高
```

**避免策略**：

| 策略 | 说明 |
|------|------|
| 主城等级预判 | 升主城前确保农田/市集等级已跟上 |
| 资源储备 | 升级前储备至少 1.5 倍所需资源 |
| 队列规划 | 利用队列系统提前安排升级顺序 |
| 离线收益 | 离线期间资源持续产出，可缓解短期缺口 |
| 科技辅助 | 点出「治国安邦」降低 10% 升级费用 |

| 原则 | 说明 |
|------|------|
| 选项不预显结果 | 选项只描述行为(如"正面迎战")，不显示具体损失/收益数值 |
| 结果执行后揭晓 | 根据武将属性、敌人规模、随机因素计算结果 |
| 胜负不同奖惩 | 胜利获得战利品+经验，失败损失货物+武将受伤 |
| 护卫影响结果 | 武力越高的护卫，战斗胜率越高 |
| 自动处理 | 武力≥80的护卫可自动处理事件，按最优策略选择 |

**等级效果**:

| 等级段 | 贸易折扣 | 繁荣度/秒 | 附加效果 |
|:------:|:-------:|:---------:|---------|
| Lv1~5 | -1%~-5% | 0.3~1.5 | 解锁基础商队(2支)，开通近程商路 |
| Lv6~10 | -6%~-10% | 1.8~3.0 | 解锁第3支商队，开通中程商路，NPC行商出现 |
| Lv11~15 | -11%~-14% | 3.5~5.5 | 商队收益+30%，开通远程商路，珍品商人出现 |
| Lv16~20 | -15%~-18% | 6.0~8.0 | 解锁跨服贸易，黑市/大师商人出现 |

- 城墙 → 守城Buff → 攻城防御

**贸易系统 ← 建筑**:
- 市舶司 → 商队派遣(定时收益)
- 市舶司 → 繁荣度(贸易事件触发)
- 市舶司 → 贸易折扣(减少购买价格)

**地图/攻城 ← 建筑**:
- 城墙 → 城防值 → 攻城防御计算

## [BLD-8] 资源链循环设计 {#bld-8}

### 资源链全景图

```
                    ┌──────────────────────────────────────┐
                    │            资 源 链 循 环             │
                    └──────────────────────────────────────┘

    ┌─────────┐    粮草     ┌─────────┐    兵力     ┌─────────┐
    │  农田   │ ──────────→ │  兵营   │ ──────────→ │  战斗   │
    └─────────┘             └─────────┘             └────┬────┘
                                                          │
    ┌─────────┐    铜钱                           战利品/资源回流
    │  市集   │ ──┬──────────────────────────────────────┘
    └─────────┘   │
                  │         ┌─────────┐    矿石+木材    ┌─────────┐
                  ├────────→ │  工坊   │ ←───────────── │ 矿场/伐木│
                  │         │(装备锻造)│                 └─────────┘
                  │         └────┬────┘
                  │              │ 装备
                  │              ▼
                  │         ┌─────────┐
                  │         │  英雄   │ ──→ 战力提升 → 产出加成回流
                  │         └─────────┘
                  │
                  ├────────→ 书院(科技) ──→ 全建筑加成回流
                  │
                  ├────────→ 酒馆(招募) ──→ 武将 → 建筑派驻加成
                  │
                  └────────→ 市舶司(贸易) ──→ 贸易折扣 → 省资源

    矿石+木材 ────────────→ 城墙(城防加固)
```

### 核心资源链路

| 链路 | 起点 | 中间 | 终点 | 循环回流 |
|------|------|------|------|---------|
| 粮草→兵力→战斗 | 农田 | 兵营 | 战斗 | 战利品→资源 |
| 矿石+木材+铜钱→装备→英雄 | 矿场/伐木场/市集 | 工坊 | 英雄 | 英雄加成→产出 |
| 铜钱→科技→加成 | 市集 | 书院 | 全建筑 | 加成→产出提升 |
| 铜钱+粮草→招募→英雄 | 市集/农田 | 酒馆 | 英雄系统 | 武将→建筑加成 |
| 铜钱+粮草→贸易→折扣 | 市集/农田 | 市舶司 | 商店 | 折扣→节省资源 |
| 矿石+木材→城防 | 矿场/伐木场 | 城墙 | 攻城防御 | 防御→安全 |

### 资源链平衡设计

**设计原则**:
1. **每条链路都有明确的"投入→产出→回流"路径**
2. **不存在僵尸建筑**(所有建筑都有消费端)
3. **资源链长度适中**(2~3步，避免过长导致玩家迷失)
4. **多条链路可并行**(玩家可同时推进多条链路)

**4种资源各有明确用途**:

| 资源 | 主要消费 | 设计意图 |
|------|---------|---------|
| 粮草 | 兵营/酒馆/建筑升级 | 军事扩张基础 |
| 铜钱 | 书院/酒馆/工坊/建筑升级 | 经济通用货币 |
| 矿石 | 工坊装备锻造/城墙/建筑升级 | 军事+防御原料 |
| 木材 | 工坊装备锻造/城墙/建筑升级 | 建设+防御原料 |

**兵营编队系统**:

| 功能 | 说明 |
|------|------|
| 编队管理 | 最多3支编队，每支可配置不同武将+兵力 |
| 训练模式 | 消耗粮草加速训练(兵营等级影响效率) |
| 编队出征 | 编队可直接用于地图征服/攻城 |
| 兵种搭配 | 不同兵种(步/骑/弓)有克制关系 |

**书院研究加速**:

| 功能 | 说明 |
|------|------|
| 研究队列 | 可排队多个科技研究 |
| 加速研究 | 消耗铜钱/元宝加速当前研究 |
| 研究预览 | 显示科技树及前置条件 |
| 研究推荐 | 根据当前状态推荐最优科技 |
