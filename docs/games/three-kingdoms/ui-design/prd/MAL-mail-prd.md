## [MAL-1] 邮件类型 {#mal-1}

### 功能描述

邮件系统是玩家接收系统通知、战斗战报、社交互动和奖励发放的核心通道。所有邮件按类型分类展示，支持筛选、批量操作和附件领取。

> 🎨 → [UI: 邮件分类Tab](../ui-layout/MAL-mail.md#mal-1)

### 邮件分类定义 {#mal-1-2}

| 分类 | 图标 | 主色调 | 说明 | 典型内容 |
|------|------|--------|------|---------|
| 📋 系统邮件 | 📜 卷轴 | `#C6A456` 古铜金 | 系统通知、维护公告、版本更新 | 服务器维护通知、活动开启通知 |
| ⚔️ 战斗邮件 | ⚔️ 刀剑 | `#B8372D` 朱砂红 | 战斗报告、攻城结果、竞技场结算 | 攻城胜利/失败报告、竞技场排名奖励 |
| 👥 社交邮件 | 🕊️ 信鸽 | `#4A8C6F` 松石绿 | 好友赠礼、联盟邮件、互动消息 | 好友赠送兵力、联盟战报、好友申请 |
| 🎁 奖励邮件 | 🎁 宝箱 | `#D4A843` 明金色 | 补偿奖励、活动奖励、成就奖励 | 维护补偿、活动排名奖励、签到奖励补发 |

### 邮件状态

| 状态 | 说明 |
|------|------|
| 未读 | 新到达邮件，标题加粗 + 发件人头像旁显示红点 |
| 已读未领 | 已查看但附件未领取，显示「待领取」标签 |
| 已读已领 | 附件已领取，整体透明度 60% + 显示「已领取」标签 |
| 已过期 | 超过保留时长，透明度 30% + 灰色叠加 + 「已过期」戳记 |

## [MAL-2] 邮件规则 {#mal-2}

> 🎨 → [UI: 邮件分类Tab](../ui-layout/MAL-mail.md#mal-1)

### 发送条件

| 触发场景 | 邮件类型 | 发送时机 | 是否有附件 |
|---------|---------|---------|:---------:|
| 服务器维护/更新 | 系统邮件 | 维护开始时自动发送 | ✅ 补偿附件 |
| 版本更新公告 | 系统邮件 | 更新推送时 | ❌ |
| 活动开启/结束 | 系统邮件 | 活动状态变更时 | ❌ |
| 攻城战结算 | 战斗邮件 | 战斗结束时 | ✅ 战利品 |
| 竞技场排名结算 | 战斗邮件 | 每日 21:00 结算 | ✅ 排名奖励 |
| 好友赠送兵力 | 社交邮件 | 赠送时即时发送 | ✅ 兵力 |
| 联盟战报 | 社交邮件 | 联盟战结束时 | ❌ |
| 好友申请 | 社交邮件 | 发出申请时 | ❌ |
| 七日签到奖励 | 奖励邮件 | 每日首次登录签到后 | ✅ 签到奖励 |
| 活动排名奖励 | 奖励邮件 | 活动结算时 | ✅ 排名奖励 |
| 成就达成奖励 | 奖励邮件 | 成就完成时 | ✅ 成就奖励 |
| 维护补偿 | 奖励邮件 | 维护结束后 | ✅ 补偿道具 |
| 离线收益 | 奖励邮件 | 玩家上线时自动生成 | ✅ 离线产出 |

### 过期时间规则

| 邮件类型 | 保留时长 | 过期后处理 |
|---------|---------|-----------|
| 系统邮件 | 30 天 | 自动删除（无附件不提醒） |
| 战斗邮件 | 7 天 | 自动删除 |
| 社交邮件 | 14 天 | 自动删除 |
| 奖励邮件 | 30 天 | 过期前 3 天每日提醒「有奖励即将过期」 |
| 离线收益邮件 | 7 天 | 附件未领取则铜钱部分补发 50% |

### 过期提醒机制

| 时间节点 | 提醒方式 |
|---------|---------|
| 过期前 3 天 | 邮件列表中该邮件显示 ⚠️ 黄色警告标签 |
| 过期前 1 天 | 登录时弹出 Toast：「您有 N 封邮件将于明天过期」 |
| 过期当天 | 邮件条目闪烁红色边框（每 5 秒闪烁一次） |
| 已过期 | 邮件变灰，附件不可领取，显示「已过期」戳记 |

### 容量上限

| 容量状态 | 数值范围 | 行为 |
|---------|---------|------|
| 正常 | 0 ~ 70 封（共 100 封） | 正常收发，容量条绿色 |
| 警告 | 71 ~ 90 封 | 新邮件正常接收，提示清理，容量条黄色 |
| 拥挤 | 91 ~ 99 封 | 新邮件正常接收，强烈提示清理，容量条红色脉动 |
| 满载 | 100 封 | 新邮件暂存队列（最多暂存 20 封），清理后补发，容量条红色闪烁 |

### 自动清理规则

- **清理时间**: 每日 05:00（服务器时间）
- **清理逻辑**:
  1. 扫描所有已过期邮件
  2. 有附件的奖励邮件：附件为铜钱/经验 → 补发 50% 到新邮件；附件为道具/装备 → 直接删除（不补发）；发送一封「过期补偿」系统邮件说明
  3. 无附件邮件 → 直接删除
  4. 已读已领邮件 → 超过保留期后直接删除（无补偿）
  5. 邮箱容量超 80% 时，优先清理最早的已读已领邮件

### 批量操作规则

| 操作 | 条件 | 说明 |
|------|------|------|
| 一键领取全部 | 存在未领取附件 | 领取所有邮件附件，不删除邮件 |
| 删除已读已领 | 存在已读且已领取的邮件 | 仅删除已读已领状态邮件 |
| 全选 | 手动勾选 | 可跨类型选择，未领附件邮件不可删除 |

## [MAL-3] 附件系统 {#mal-3}

> 🎨 → [UI: 邮件附件领取](../ui-layout/MAL-mail.md#mal-1)

### 附件类型

| 附件类型 | 图标 | 说明 | 堆叠规则 |
|---------|------|------|---------|
| 铜钱 | 💰 | 基础货币 | 直接累加到玩家铜钱余额 |
| 经验 | ⭐ | 武将经验 | 直接累加到当前出战武将 |
| 强化石 | 🧪 | 装备强化材料 | 堆叠到背包材料栏 |
| 装备 | 📦 | 装备物品 | 单独占一个背包格子 |
| 武将碎片 | 🧩 | 武将合成材料 | 堆叠到碎片仓库 |
| 招募令 | 🎫 | 武将招募凭证 | 堆叠到道具栏 |
| 元宝 | 💎 | 高级货币 | 直接累加到玩家元宝余额 |
| 兵力 | ❤️ | 挂机消耗资源 | 直接累加到兵力条（不超上限） |

### 领取规则

| 规则 | 说明 |
|------|------|
| 单封领取 | 打开邮件详情 → 点击「领取全部附件」 |
| 批量领取 | 邮件列表 → 点击「一键领取全部」，逐封领取 |
| 背包满处理 | 背包满时附件不可领取，邮件保持「已读未领」状态，提示「背包已满，请清理后领取」 |
| 货币类附件 | 铜钱、经验、元宝、兵力直接入账，不占背包格子 |
| 领取后状态 | 附件全部领取后邮件变为「已读已领」，不可重复领取 |

### 离线收益邮件特殊规则

#### 离线收益计算

| 离线时长 | 收益效率 | 邮件标题 |
|---------|---------|---------|
| 0 ~ 30 分钟 | 不生成邮件 | — |
| 30 分钟 ~ 2 小时 | 100% 效率 | 「离线收益 · 短暂小憩」 |
| 2 ~ 6 小时 | 100% 效率 | 「离线收益 · 充分休整」 |
| 6 ~ 12 小时 | 80% 效率 | 「离线收益 · 深度养精」 |
| 12 ~ 24 小时 | 60% 效率 | 「离线收益 · 久别重逢」 |
| 24 小时以上 | 50% 效率（上限 48 小时） | 「离线收益 · 主公归来」 |

#### 离线收益内容

| 收益类型 | 计算公式 | 说明 |
|---------|---------|------|
| 铜钱 | 基础产出/秒 × 离线秒数 × 效率系数 | 受城池建筑加成 |
| 经验 | 基础经验/秒 × 离线秒数 × 效率系数 | 受武将加成 |
| 装备掉落 | 挂机掉落概率 × 离线战斗次数 × 效率系数 | 最高蓝色品质 |
| 武将碎片 | 低概率掉落（受离线时长影响） | 长时间离线概率略高 |

### 签到规则

| 规则 | 说明 |
|------|------|
| 签到时间 | 每日首次登录自动弹出签到弹窗 |
| 连续签到 | 连续每日签到，中断则重新从 Day 1 开始 |
| 补签 | 不支持补签，断签后重新计算连续天数 |
| 奖励发放 | 签到后奖励以附件形式发放到邮件（同时可弹窗直接领取） |
