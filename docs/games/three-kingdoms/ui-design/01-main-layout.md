> **v3.0 迭代说明**：Round 1 评分 7.8/10（B+）。本次重点改进：(1) 补充离线收益系统化入口设计——回归核心闭环「回来→看到增长→知道做什么→行动」；(2) 完善「更多」下拉菜单完整功能入口定义；(3) 加强手机端资源栏可辨识性；(4) 移除PC端底部快捷键栏释放空间；(5) 重新规划手机端快捷图标条逻辑。

## 一、放置核心闭环设计

### 闭环理念

放置游戏的留存引擎是**回归惊喜感**。玩家每次打开游戏应经历：

```
  回来 → 看到增长 → 知道做什么 → 行动 → 离开（期待下次回归）
   │         │            │          │           │
   │         │            │          │           └─ 离线收益持续累积
   │         │            │          └─ 一键操作完成当日核心目标
   │         │            └─ 军师建议+红点系统精准引导
   │         └─ 离线收益弹窗 + 地图变化可视化 + 建筑升级完成
   └─ 加载→回归欢迎流程
```

### 回归欢迎流程（完整序列）

玩家打开游戏后的**弹窗优先级队列**：

| 优先级 | 弹窗类型 | 触发条件 | 可否跳过 | 说明 |
|:------:|---------|---------|:-------:|------|
| 1 | **离线收益弹窗** | 离线>5分钟 | ❌不可 | 全屏展示离线收益，必须领取 |
| 2 | **版本更新公告** | 有新版本 | ✅可跳 | 展示更新内容 |
| 3 | **限时活动提醒** | 有进行中活动 | ✅可跳 | 展示活动倒计时+奖励预览 |
| 4 | **回归欢迎面板** | 离线>1小时 | ✅可跳 | 综合变化摘要（见下文） |
| 5 | **军师建议更新** | 有新建议 | ❌不可 | 更新推荐内容 |

### 推荐优先级算法

```
优先级评分 = 基础权重 × 紧迫系数 × 收益系数

推荐类型及权重：
- 资源将溢出（>90%容量）    权重=100, 紧迫系数=容量百分比
- 武将可升级且材料充足       权重=80,  收益系数=战力提升百分比
- 新关卡解锁且战力足够       权重=70,  收益系数=经验/装备收益
- 科技研究完成可开启新项目   权重=60,  收益系数=科技效果
- 公会BOSS/活动限时          权重=90,  紧迫系数=剩余时间反比
- 建筑可升级且资源充足       权重=65,  收益系数=产出提升百分比
- 日常任务接近完成(>80%)     权重=55,  紧迫系数=完成度百分比
- 远征队列有空位             权重=45,  收益系数=预计奖励

取评分最高的3条显示 + 离线收益入口（固定）
```

|:------:|---------|---------|:---------:|
| 1 | 离线收益弹窗 | 全屏弹窗 | 1 |

### 通知排队规则

- **免打扰模式**：设置中可开启，关闭所有非关键通知（仅保留离线收益弹窗）
- **横幅自动消失**：5秒后自动淡出，点击可提前关闭
- **Toast堆叠**：最多3条，新消息从底部推入，旧消息上移

## 六、地图滚动交互规范

### 触发条件

- 鼠标进入地图边缘30px范围（PC端）
- 30px为可配置参数

### 滚动算法

```
edgeDistance = 30 - mouseDistanceFromEdge
speed = baseSpeed * (edgeDistance / 30)
scrollX += speed * directionX
scrollY += speed * directionY
```

### 边界Clamp

```
scrollX = clamp(scrollX, 0, mapWidth - viewWidth)
scrollY = clamp(scrollY, 0, mapHeight - viewHeight)
```

### 视觉反馈

- 鼠标进入边缘区域时，边缘显示淡金色光带提示滚动方向
- 滚动中地图平滑移动（lerp插值，系数0.15）
- **v3.0补充**：Tooltip显示时暂停边缘滚动（详见02-map-system.md）
