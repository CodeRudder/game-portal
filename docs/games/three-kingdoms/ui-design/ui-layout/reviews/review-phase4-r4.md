### 扣分项详细说明

#### 扣分项1: 交叉引用断链 (-0.4)

**核心问题**: R3评审报告声称"274条双向引用全部有效"，但R4全量Python脚本扫描发现:
- 实际引用总数: 244条（非274条）
- 有效引用: 198条
- 断链: 45条（41条UI→PRD + 4条PRD→UI）
- 有效率: 81.1%（非100%）

**断链根因分析**:
1. **子编号锚点未定义**（30条）: UI布局引用了PRD子章节锚点（如 `#her-4-2`），但PRD只在主章节定义了锚点（如 `#her-4`），子章节使用 `###` 标题但未添加 `{#锚点}`
2. **PRD章节缺失**（11条）: MAP-world-prd.md缺少MAP-4~MAP-7四个章节，NPC/PRS也缺少部分章节
3. **编号不对称**（4条）: PRD引用了UI中尚未定义的面板编号

**影响评估**: 
- 类型A（子编号锚点）: 内容存在但导航不精准，开发者可定位到主章节后手动查找，影响轻微
- 类型B（章节缺失）: 开发者无法从UI追溯到完整PRD，需参考其他模块推断，影响中等
- 类型C（反向断链）: PRD引用了不存在的UI面板，影响轻微

#### 扣分项2: MAP PRD章节缺失 (-0.1)

MAP-world.md已从87行扩展到216行（新增MAP-3攻城确认、MAP-4事件标记、MAP-5统计面板），但MAP-world-prd.md仍停留在198行/3个章节。UI布局中引用的 `#map-4`（领土系统）、`#map-5`（攻城战）、`#map-6`（地图事件）、`#map-7`（地图统计）在PRD中均不存在。

### 关键教训
