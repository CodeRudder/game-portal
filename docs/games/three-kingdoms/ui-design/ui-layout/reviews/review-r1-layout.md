7. **交互流程内嵌**: 布局图中直接标注了交互行为（如"点击→打开面板"）

### 4.2 问题项

#### P2-004: CBT 战斗画面布局图比例失真
- **位置**: CBT-combat.md [CBT-3-2]
- **问题**: 标注为 `1280×500px`，但 ASCII 图中我方/敌方各占 `640×500px`，合计仅1280px宽度。实际上还需要考虑间距和边距，实际可用宽度可能不足1280px
- **建议**: 补充边距/间距说明

#### P2-005: BLD 建筑网格实际尺寸与场景区不匹配
- **位置**: BLD-buildings.md [BLD-1]
- **问题**: 6列×5行网格，每格180×160px + 间距16px + 内边距24px
  - 宽度: 6×180 + 5×16 + 2×24 = 1080 + 80 + 48 = 1208px（场景区1280px，剩余72px）
  - 高度: 5×160 + 4×16 + 2×24 = 800 + 64 + 48 = 912px（场景区696px，超出216px）
  - **5行网格高度超出场景区高度**
- **影响**: 实际布局需要滚动或减少行数
- **建议**: 调整为4行或缩小卡片高度

#### P2-006: TEC 科技树画布尺寸标注模糊
- **位置**: TEC-tech.md [TEC-1]
- **问题**: 画布总尺寸标注为 `约 2400×1200px`，"约"字不够精确
- **建议**: 给出精确的画布尺寸，或说明动态计算规则
