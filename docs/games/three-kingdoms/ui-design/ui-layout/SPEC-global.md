## [SPEC-7] 弹窗层级规范 {#spec-7}

> 📖 → [PRD: 弹窗精简策略](../prd/SPEC-interaction.md#itr-7)

### 7.1 核心原则

```
规则1: 能用Toast解决的，不用弹窗
规则2: 能在面板内展开的，不弹窗
规则3: 只有不可逆操作才需要确认弹窗
规则4: 所有弹窗必须有 [×] 关闭按钮
```

### 7.2 功能层级结构

```
L0  游戏主窗口（资源栏A + 导航栏B + 场景区C）
│
├─ L1  主Tab（B区Tab切换，C区全屏场景）
│   ├─ 天下 → 世界地图
│   ├─ 出征 → 战役长卷
│   ├─ 武将 → 武将管理（含子Tab）
│   ├─ 科技 → 科技树
│   ├─ 建筑 → 城池建设
│   └─ 声望 → 声望系统
│
├─ L2  Tab内子Tab（C区内子导航）
│
├─ L3  详情面板/弹窗
│   ├─ D区右侧面板: 武将详情、建筑详情
│   ├─ E区居中弹窗: 关卡详情、科技详情、装备详情
│   └─ F区全屏: 战前布阵、战斗场景、NPC对话
│
├─ L4  操作弹窗（在L3内触发的子操作）
│   └─ 确认弹窗（仅重大决策）
│
└─ Toast  即时反馈（3秒消失，不遮挡）
```

### 7.3 确认弹窗使用场景（仅约11%的操作）

| 操作 | 确认原因 | 弹窗尺寸 |
|------|---------|---------|
| 转生 | 不可逆，重置大量进度 | 500×400px |
| 分解紫/金装备 | 高价值损失 | 400×300px |
| 出售蓝以上装备 | 价值较高 | 400×280px |
| 公会战宣战 | 影响全体成员 | 480×350px |
| 退出公会 | 不可逆 | 400×280px |
| 删除好友 | 不可逆 | 360×240px |
| 大量资源消耗(>1000) | 防止误操作 | 400×280px |
| 战斗撤退 | 放弃当前战斗 | 360×200px |
| 招募(消耗资源) | 资源消耗 | 400×300px |

### 7.4 直接执行+Toast的操作（约89%的操作，无需弹窗）

| 操作分类 | Toast内容 | Toast位置 |
|---------|-----------|----------|
| 建筑升级 | "农田升级至Lv.6，预计30秒" | 顶部居中 |
| 武将升级 | "关羽升级至Lv.26" | 顶部居中 |
| 装备穿戴 | "已装备青龙偃月刀到关羽" | 顶部居中 |
| 科技升级 | "铁器锻造升级至Lv.3" | 顶部居中 |
| 扫荡关卡 | "扫荡完成，获得XXX" | 顶部居中 |
| 领取奖励 | "获得金币×500" | 操作位置附近 |

### 7.5 弹窗尺寸规范

```
小弹窗(S):  320×200px  — 确认/提示/二次确认
中弹窗(M):  420×400px  — 升级/详情/编组
大弹窗(L):  600×500px  — 活动总览/招贤馆/设置
右侧面板:   380px(信息类) / 480px(详情类)
全屏覆盖:   1280×696(PC) / 375×507(手机)
Bottom Sheet: 高度40%~70% (手机端)
```

### 7.6 弹窗通用结构

```
┌──── 标题 ──────────────── ×关闭┐
│  标题文字(20px)     关闭按钮36×36│
├────────────────────────────────┤
│                                │
│        内容区域                 │
│        (可滚动)                 │
│                                │
├────────────────────────────────┤
│  [取消]          [确认操作]     │
│  底部操作栏 高度56px            │
└────────────────────────────────┘

打开动画: scale(0.95)→1 + fade 200ms
```
