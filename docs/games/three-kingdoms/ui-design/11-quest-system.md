# 任务系统 UI 设计（v2.0）

> **v2.0 迭代说明**：全面扩展任务分类体系、PC/手机双端布局、任务追踪面板、领取奖励动画、放置循环关联、跳转路径设计。评审反馈 —，首次完整版本。

---

## 一、入口与触发

| 入口位置 | 触发方式 | 视觉表现 |
|---------|---------|---------|
| 主界面右上角「任务」竹简图标 | 点击打开 | 未完成任务数红点角标，有新任务时竹简微晃 |
| 主界面顶部常驻追踪条 | 自动显示 | 追踪中任务的迷你进度条 + 名称 |
| "更多"下拉菜单 | 点击「任务」 | 与主入口一致 |
| 新任务推送 | 自动弹出 | 竹简从屏幕右侧滑入通知条，3秒后自动收回 |
| 任务完成 | 自动弹出 | 青铜印章「完成」戳记 + 奖励领取弹窗 |

## 二、任务分类详细设计

### 2.1 主线任务（Main Quest）

| 属性 | 规格 |
|------|------|
| 总数量 | 约 80~120 个（随版本章节逐步解锁） |
| 解锁规则 | 前置任务完成后自动解锁下一个 |
| 刷新规则 | 不刷新，永久存续直到完成 |
| 重置规则 | 不重置，账号永久进度 |
| 任务链结构 | 每章 8~12 个任务，完成后解锁下一章 |

**主线章节示例：**
```
第一章「黄巾之乱」：1-1 → 1-2 → ... → 1-10（通关后解锁第二章）
第二章「讨伐董卓」：2-1 → 2-2 → ... → 2-12
第三章「群雄割据」：3-1 → ...
...
```

**主线任务类型标签：**
- 🏯 攻城类：击败 X 个敌军 / 攻占 Y 座城池
- ⚔️ 养成类：将武将升级到 N 级 / 强化装备到 +M
- 📜 剧情类：阅读剧情对话 / 触发特定事件
- 🏗️ 建设类：建造/升级 N 座建筑

### 2.2 支线任务（Side Quest）

| 属性 | 规格 |
|------|------|
| 总数量 | 每章 3~5 个，全章节约 30~50 个 |
| 解锁规则 | 对应主线章节推进到指定节点后解锁 |
| 刷新规则 | 不刷新，一次性任务 |
| 重置规则 | 不重置 |
| 特点 | 奖励比主线更丰厚（含稀有道具），但难度更高 |

### 2.3 日常任务（Daily Quest）

| 属性 | 规格 |
|------|------|
| 总数量 | 每日 6 个（固定池 20 个中随机抽取） |
| 刷新规则 | 每日 05:00 重置，未完成进度清零 |
| 任务池 | 登录游戏、完成 1 次战斗、强化 1 件装备、炼制 1 次、招募 1 次、升级 1 座建筑、消耗 1000 铜钱、完成 3 次挂机、领取离线收益、升级 1 名武将 ... |
| 活跃度系统 | 每完成 1 个日常获得 20 活跃度，累计达 40/60/80/100 可领取额外宝箱 |

**日常任务活跃度宝箱：**

| 活跃度 | 奖励内容 |
|--------|---------|
| 40 | 铜钱 ×5000 + 强化石 ×2 |
| 60 | 元宝 ×50 + 招募令 ×1 |
| 80 | 元宝 ×100 + 紫色装备箱 ×1 |
| 100 | 元宝 ×200 + 金色装备碎片 ×3 |

### 2.4 周常任务（Weekly Quest）

| 属性 | 规格 |
|------|------|
| 总数量 | 每周 4 个（固定池 12 个中随机抽取） |
| 刷新规则 | 每周一 05:00 重置 |
| 任务池 | 累计击败 500 敌军、累计获得 10 件蓝色+装备、累计强化 20 次、累计炼制 10 次、累计招募 15 名武将、累计升级 30 座建筑 ... |
| 奖励等级 | 比日常高 3~5 倍，含周限稀有道具 |

### 2.5 成就任务（Achievement）

| 属性 | 规格 |
|------|------|
| 总数量 | 约 60~100 个（永久存续） |
| 解锁规则 | 部分初始可见，部分隐藏（达成条件后揭示） |
| 刷新规则 | 不刷新，永久追踪 |
| 分类标签 | 「战斗」「养成」「收集」「社交」「探索」五大类 |
| 奖励 | 成就点数 + 专属称号 + 稀有道具 |

**成就分级：**
- 🥉 铜级成就：基础目标（如"首次招募武将"）
- 🥈 银级成就：进阶目标（如"招募 10 名武将"）
- 🥇 金级成就：高级目标（如"集齐全阵营武将"）

## 六、任务领取和奖励展示动画

### 6.1 任务完成通知

**触发条件：** 任务进度达到目标值时自动触发

**动画序列（总时长 2.5s）：**

```
时间轴：
0ms      → 竹简条目边框闪烁古铜金（200ms × 3次）
600ms    → 青铜印章从上方落下盖在任务条目上（300ms ease-out bounce）
900ms    → 墨迹从印章中心向四周飞溅（8颗墨滴，抛物线轨迹）
1200ms   → 任务条目背景变为暗金色（500ms 渐变）
1500ms   → 奖励弹窗从屏幕底部弹入（400ms ease-out）
2000ms   → 奖励道具依次亮起（每个间隔 150ms）
2500ms   → 全部动画结束，等待玩家操作
```

- 全部飞完后，主界面资源数字跳动更新

## 七、任务与放置核心循环的关联

### 7.1 离线任务进度

| 任务类型 | 离线是否计算进度 | 说明 |
|---------|----------------|------|
| 主线·攻城类 | ✅ 是 | 离线挂机战斗自动推进，上限 2 小时进度 |
| 主线·养成类 | ❌ 否 | 需在线手动操作 |
| 日常·战斗类 | ✅ 是 | 离线挂机战斗计入日常战斗次数 |
| 日常·操作类 | ❌ 否 | 强化/炼制等需在线操作 |
| 周常 | 部分计算 | 累计击败数等自动计算，操作类不计 |
| 成就 | ✅ 全部计算 | 所有累计型成就离线持续追踪 |

### 7.2 自动完成机制

**自动完成条件：**
1. 玩家离线期间，挂机系统持续运行
2. 挂机产出的战斗胜利数、资源获取量等数据实时计入对应任务进度
3. 任务进度达到目标后标记为「待领取」状态
4. 玩家下次上线时，自动弹出完成通知

**离线任务进度上限：**
```
离线时间      任务进度上限
0~1小时      100% 正常计算
1~4小时      80% 效率（线性衰减）
4~8小时      50% 效率
8~12小时     30% 效率
12小时以上    20% 效率（最低值）
```

### 7.3 任务驱动的放置循环

```
                    ┌──────────────┐
                    │   主线任务    │
                    │ 引导解锁新玩法 │
                    └──────┬───────┘
                           │ 解锁
                           ▼
┌──────────┐      ┌──────────────┐      ┌──────────┐
│ 日常/周常 │ ───→ │  放置挂机战斗  │ ───→ │  资源产出  │
│ 提供每日目标│      │  自动推进进度   │      │ 用于养成   │
└──────────┘      └──────────────┘      └──────────┘
                           │                     │
                           ▼                     ▼
                    ┌──────────────┐      ┌──────────────┐
                    │  成就系统追踪  │      │  武将/装备强化  │
                    │  累计型目标    │      │  反哺主线进度   │
                    └──────────────┘      └──────────────┘
```

## 八、任务跳转路径设计

### 8.1 跳转映射表

| 任务关键词 | 跳转目标 | 跳转后高亮 |
|-----------|---------|-----------|
| 击败/战斗/攻占 | 战斗关卡选择界面 | 对应关卡按钮脉冲高亮 |
| 强化/升级装备 | 背包→装备详情→强化 | 强化按钮高亮 |
| 炼制/合成 | 背包→炼制界面 | 炼炉中心高亮 |
| 招募/获得武将 | 招募界面 | 招募按钮高亮 |
| 建造/升级建筑 | 城池建设界面 | 对应建筑高亮 |
| 登录/签到 | 自动完成，不跳转 | — |
| 消耗铜钱/元宝 | 商店界面 | 对应商品区域高亮 |
| 阅读剧情 | 剧情回放界面 | 对应章节高亮 |

### 8.2 跳转动画

1. 点击「前往」→ 任务面板自动收起（200ms ease-in）
2. 目标面板/界面从对应方向滑入（300ms ease-out）
3. 目标元素显示古铜金脉冲高亮边框（持续 3 秒后消失）
4. 高亮边框旁显示小箭头 + 任务名悬浮提示

### 8.3 跳转失败处理

| 情况 | 处理方式 |
|------|---------|
| 目标功能未解锁 | Toast 提示「需完成主线 X-X 解锁」，「前往」按钮置灰 |
| 目标功能需要更高等级 | Toast 提示「主公等级需达到 N 级」 |
| 跳转目标在子菜单深处 | 自动展开菜单层级，逐级高亮引导 |

|---------|------|---------|---------|
| 新任务竹简插入 | 400ms | `ease-out` | 新任务解锁 |
| 进度条流光扫过 | 600ms | `linear` | 进度更新 |
| 印章盖下 | 300ms | `ease-out bounce` | 任务完成 |
| 墨迹飞溅 | 500ms | `ease-out` | 印章落下后 |
| 奖励弹窗弹入 | 400ms | `ease-out` | 印章动画后 |
| 奖励数字滚动 | 800ms | `ease-out` | 弹窗展开后 |
| 道具飞向资源栏 | 600ms | `cubic-bezier(0.4,0,0.2,1)` | 一键领取 |
| 追踪条更新 | 200ms | `ease-out` | 进度变化 |
| Tab 切换 | 200ms | `ease-in-out` | 切换分类 |
| 面板滑入/滑出 | 300ms | `ease-out` / `ease-in` | 打开/关闭面板 |
| 高亮脉冲 | 3000ms | `ease-in-out` 循环 | 跳转后引导 |
| 未解锁任务闪烁 | 1500ms | `ease-in-out` 循环 | 前置任务刚完成时 |
