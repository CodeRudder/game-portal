# 交互设计规范（v3.0 大幅迭代）

> **v3.0 迭代说明**：全面扩展交互设计深度。核心改进：(1) 完整的5态交互状态设计（normal/hover/active/disabled/loading）；(2) 拖拽交互全流程规范；(3) 加载状态设计（骨架屏/进度条/加载动画）；(4) 错误状态设计（网络/操作/资源不足）；(5) 无障碍设计规范；(6) 动效性能预算。设计分辨率：PC 1280×800 / 手机 375×667。风格：水墨江山·铜纹霸业。

## 三、完整交互状态设计

### 3.1 按钮交互五态

#### Normal（默认态）

```
视觉表现：
  - 背景：古铜金渐变 linear-gradient(180deg, #D4AF5E, #C8A45C)
  - 边框：1px solid #A0824A
  - 文字：深褐色 #3D2B1F，毛笔字体
  - 阴影：0 2px 4px rgba(0,0,0,0.2)
  - 圆角：6px
  - 内边距：12px 24px
  - 光标：pointer
```

#### Hover（悬停态）— 仅桌面端

```
视觉表现：
  - 背景：亮金渐变 linear-gradient(180deg, #E0C070, #D4AF5E)
  - 边框：1px solid #C8A45C
  - 文字：深褐色 #3D2B1F
  - 阴影：0 4px 8px rgba(200,164,92,0.3)（阴影扩大+金色光晕）
  - 顶部高光条：1px 白色半透明条纹（模拟金属反光）
  - 过渡：150ms ease-out
  - 光标：pointer
  - Tooltip：悬停>500ms后显示简短描述
```

#### Active（按下态）

```
视觉表现：
  - 背景：暗金渐变 linear-gradient(180deg, #B8963E, #A0824A)
  - 边框：1px solid #8B6F3A
  - 文字：深褐色 #3D2B1F
  - 阴影：0 1px 2px rgba(0,0,0,0.3)（阴影收缩）
  - 缩放：transform: scale(0.96)
  - 过渡：80ms ease-in
  - 震动反馈（移动端）：10ms轻微震动
```

#### Disabled（禁用态）

```
视觉表现：
  - 背景：灰色 linear-gradient(180deg, #8A8A8A, #6E6E6E)
  - 边框：1px solid #5A5A5A
  - 文字：浅灰 #B0B0B0
  - 阴影：无
  - 透明度：0.6
  - 光标：not-allowed
  - Tooltip：悬停显示禁用原因（如「资源不足」或「等级不够」）
  
禁用原因提示样式：
  - 红色文字标注在按钮下方：「需要铜钱×500（当前×320）」
  - 不足部分用红色 #C0392B 标注
```

#### Loading（加载态）

```
视觉表现：
  - 背景：古铜金渐变（保持normal态颜色）
  - 文字：替换为加载指示器
  - 加载指示器：青铜鼎图标旋转（360°循环，1.2s/圈）
  - 透明度：0.8
  - 禁止重复点击（防抖500ms）
  - 光标：wait
  - 移动端无震动
```

### 3.2 五态过渡时序

```
Normal ──hover──→ Hover     (150ms ease-out)
Hover  ──leave──→ Normal    (150ms ease-out)
Hover  ──press──→ Active    (80ms ease-in)
Active ──release──→ Hover   (100ms ease-out)
Active ──cancel──→ Normal   (150ms ease-out)
Normal ──disable──→ Disabled (200ms ease-out, 透明度渐变)
Disabled ──enable──→ Normal  (200ms ease-out)
Normal ──loading──→ Loading  (300ms ease-out, 旋转淡入)
Loading ──complete──→ Normal (200ms ease-out, 旋转淡出)
```

### 3.3 卡片交互五态（武将卡/装备卡/建筑卡）

| 状态 | 视觉表现 | 触发方式 |
|------|---------|---------|
| Normal | 水墨底色卡片，古铜金细边框 | 默认 |
| Hover | 边框发光（金色光晕），微微上浮 translateY(-2px)，阴影加深 | 鼠标悬停 |
| Active | 按下缩放 scale(0.97)，边框内缩 | 鼠标按下/手指触摸 |
| Selected | 古铜金粗边框(2px)，左上角勾选标记，背景微亮 | 点击选中 |
| Disabled | 灰度滤镜 filter: grayscale(0.7)，透明度0.5 | 不可用状态 |

### 3.4 列表项交互五态

| 状态 | 视觉表现 | 触发方式 |
|------|---------|---------|
| Normal | 竹简底色 #F5E6C8，1px底部分隔线 | 默认 |
| Hover | 底色变亮 #FAF0D8，右侧出现操作箭头 | 鼠标悬停 |
| Active | 底色变深 #E8D5B0，按下反馈 | 点击 |

| Disabled | 文字变灰 #999，底色变淡 | 不可选 |

### 3.5 Tab交互五态

| 状态 | 视觉表现 | 触发方式 |
|------|---------|---------|
| Normal | 水墨底色，文字白色 | 默认 |
| Hover | 底色微亮，底部出现古铜金下划线(2px) | 鼠标悬停 |
| Active | 底部古铜金下划线(3px)，文字古铜金 | 点击 |
| Selected | 底部古铜金下划线(3px)持续显示，文字古铜金加粗 | 选中状态 |
| Disabled | 文字灰色，无下划线 | 不可用 |

## 四、拖拽交互详细规范

### 4.1 拖拽应用场景

| 场景 | 拖拽对象 | 目标区域 | 说明 |
|------|---------|---------|------|
| 武将编队 | 武将卡片 | 阵型槽位 | 从武将列表拖到战斗阵型 |
| 装备穿戴 | 装备图标 | 武将装备栏 | 从背包拖到武将装备槽 |
| 物品排序 | 物品卡片 | 列表位置 | 背包内物品排序 |
| 建筑放置 | 建筑图标 | 主城空地 | 从建筑面板拖到主城格子 |

### 4.2 拖拽全流程视觉反馈

#### 阶段1：拖拽开始（Drag Start）

```
触发条件：按住目标元素>150ms并移动>5px
视觉反馈：
  - 被拖拽元素：
    · 缩放 scale(1.08)
    · 旋转 rotate(-2deg)（微微倾斜，增加动感）
    · 阴影加深：0 8px 24px rgba(0,0,0,0.3)
    · 透明度：0.9
    · 古铜金光框出现（脉动动画）
  - 原位置：
    · 留下虚影（同形状，透明度0.2，虚线边框）
  - 有效目标区域：
    · 高亮提示（边框变亮+脉动）
  - 无效区域：
    · 无变化（放下将取消）
  - 移动端：
    · 触摸点上方30px显示拖拽元素（避免手指遮挡）
    · 轻微震动反馈（10ms）
```

#### 阶段2：拖拽进行中（Dragging）

```
视觉反馈：
  - 被拖拽元素跟随鼠标/手指移动（延迟<16ms，即1帧）
  - 经过有效目标区域时：
    · 目标区域边框变为古铜金 #C8A45C
    · 目标区域背景微亮 rgba(200,164,92,0.15)
    · 水墨涟漪效果（从中心向外扩散）
    · 光标变为 grabbing
  - 离开有效目标区域时：
    · 目标区域恢复原状
    · 光标变为 grabbing
  - 武将编队特殊反馈：
    · 经过已占用槽位时，原武将卡片向旁移开（200ms ease-out）
    · 空槽位显示虚线轮廓+「放置」文字
  - 移动端：
    · 拖拽元素跟随手指（偏移30px）
    · 经过有效区域时震动（10ms）
```

#### 阶段3：放下（Drop）

**成功放下**：

```
视觉反馈：
  - 被拖拽元素：
    · 缩放回 scale(1.0)（200ms spring）
    · 旋转回 rotate(0deg)（200ms spring）
    · 阴影恢复正常
    · 古铜金光框消失
  - 目标区域：
    · 闪光特效（白色光芒从中心扩散，300ms）
    · 吸附动画（元素从放下位置滑到目标中心，150ms ease-out）
  - 成功提示：
    · 短Toast「武将已上阵」/「装备已穿戴」（1.5s自动消失）
    · 移动端震动反馈（20ms中等）
  - 音效：
    · 清脆「咔嗒」声（竹简合拢音效）
```

**失败放下（无效区域）**：

```
视觉反馈：
  - 被拖拽元素弹回原位：
    · 弹性动画回弹（400ms spring(0.5, 0.7)）
    · 先超出原位10px再弹回（过冲效果）
  - 失败提示：
    · 红色Toast「无法放置到此处」（1.5s自动消失）
    · 移动端震动反馈（30ms中等）
  - 音效：
    · 低沉鼓声（错误提示音）
```

#### 阶段4：取消（Cancel）

```
触发条件：按Esc键 / 拖拽超过10秒无操作
视觉反馈：
  - 被拖拽元素淡出（200ms）→ 原位虚影淡入恢复（200ms）
  - 所有目标区域高亮取消
  - 无Toast提示（静默取消）
```

### 4.3 拖拽性能要求

| 指标 | 要求 |
|------|------|
| 拖拽响应延迟 | ≤16ms（60fps） |
| 元素跟随帧率 | ≥60fps |
| 目标区域检测频率 | 每帧检测 |
| 最大同时拖拽元素 | 1个 |
| 拖拽超时自动取消 | 10秒 |

## 五、加载状态设计规范

### 5.1 加载场景分类

| 场景 | 加载类型 | 预期时长 | 用户感知 |
|------|---------|---------|---------|
| 游戏启动 | 全屏加载 | 3-8s | 等待可接受 |
| 场景切换 | 全屏过渡 | 1-3s | 快速切换 |
| 面板打开 | 局部加载 | 0.3-1s | 几乎无感 |
| 数据请求 | 骨架屏 | 0.5-2s | 内容逐步呈现 |
| 战斗加载 | 全屏加载 | 2-5s | 期待感 |
| 资源下载 | 进度条 | 5-30s | 需要进度反馈 |

### 5.2 全屏加载动画

**游戏启动加载**：

```
视觉设计：
  - 背景：深墨色 #1A1A2E，水墨纹理渐隐渐现
  - 中央：青铜鼎图标（SVG，古铜金色）
    · 鼎身缓慢旋转（360°，8s/圈）
    · 鼎内冒出水墨烟雾（粒子效果，20个粒子）
  - 底部：水墨晕染进度条
    · 宽度：60%屏幕宽度，居中
    · 高度：4px
    · 填充色：古铜金渐变 #C8A45C → #D4AF5E
    · 填充动画：水墨晕染扩散效果（非线性填充）
  - 进度文字：加载进度百分比，古铜金色，位于进度条下方
  - 加载提示：随机显示游戏Tips
    · 「Tips: 武将之间有羁绊加成，合理搭配可大幅提升战力」
    · 每3秒切换一条，水墨淡入淡出

PC端（1280×800）：
  - 青铜鼎尺寸：120×120px
  - 进度条宽度：768px（60%）
  - Tips文字：16px

手机端（375×667）：
  - 青铜鼎尺寸：80×80px
  - 进度条宽度：300px（80%）
  - Tips文字：13px
```

**场景切换加载**：

```
视觉设计：
  - 过渡方式：水墨晕染遮罩
    · 从屏幕四角向中央扩散的墨色遮罩
    · 完全覆盖后切换场景内容
    · 新场景从中央向四周展开
  - 时长：800ms（遮罩收拢400ms + 展开400ms）
  - 中央显示：场景名称（如「主城」「赤壁」）
    · 毛笔字体，古铜金色
    · 水墨飞白效果
  - 无进度条（快速切换不需要）
```

### 5.3 骨架屏设计

**适用场景**：面板内容加载（武将列表、装备列表、邮件列表等）

```
骨架屏视觉规范：
  - 底色：竹简色 #F5E6C8
  - 骨架块颜色：#E0D0B0（比底色深一度）
  - 骨架块圆角：4px
  - 闪烁动画：从左到右的光带扫过（1.5s循环）
    · 光带颜色：rgba(255,255,255,0.3)
    · 移动方向：左→右
    · 宽度：40%骨架块宽度

骨架块形状映射：
  - 头像/图标：圆形骨架块（直径=实际图标直径）
  - 标题文字：矩形骨架块（宽度60-80%，高度16px）
  - 描述文字：矩形骨架块（宽度80-100%，高度12px，2-3行）
  - 按钮：圆角矩形骨架块（与实际按钮同尺寸）
  - 卡片：完整卡片轮廓骨架块

示例——武将列表骨架屏：
┌──────────────────────────────────┐
│  ┌──┐  ████████████              │  ← 头像(圆) + 名字(矩形)
│  └──┘  ████████                  │  ← 描述(矩形)
│  ─────────────────────────────── │  ← 分隔线
│  ┌──┐  ████████████              │
│  └──┘  ████████                  │
│  ─────────────────────────────── │
│  ┌──┐  ████████████              │
│  └──┘  ████████                  │
└──────────────────────────────────┘

最大骨架屏展示时间：3秒
超过3秒 → 显示加载失败提示（见错误状态章节）
```

### 5.4 局部加载指示器

**适用场景**：按钮点击后等待响应、小区域数据刷新

```
小型加载指示器：
  - 尺寸：20×20px（按钮内）/ 32×32px（面板内）
  - 样式：青铜鼎简化图标旋转
  - 旋转速度：1.2s/圈
  - 颜色：古铜金 #C8A45C
  - 透明度：0.8

按钮内加载：
  - 按钮文字替换为加载指示器
  - 指示器居中显示
  - 按钮宽度不变，高度不变
  - 禁止重复点击

面板内加载：
  - 指示器居中显示在内容区域
  - 下方显示「加载中...」文字（14px，灰色）
```

### 5.5 进度条设计

**确定性进度条**（已知总量的下载/安装）：

```
视觉规范：
  - 轨道：圆角矩形，底色 #2A2A3E，高度 6px（PC）/ 4px（手机）
  - 填充：古铜金渐变 linear-gradient(90deg, #C8A45C, #D4AF5E)
  - 填充顶部高光：1px rgba(255,255,255,0.3)
  - 圆角：3px
  - 动画：平滑填充（非线性，使用 ease-out 缓动）
  - 百分比文字：填充条右侧，古铜金色，12px
  - 接近完成（>90%）：填充条脉动发光
```

**不确定性进度条**（未知时长的等待）：

```
视觉规范：
  - 轨道：同上
  - 填充：古铜金色块（宽度30%），在轨道内左右往返移动
  - 往返动画：2s循环，ease-in-out
  - 无百分比文字
```

## 六、错误状态设计规范

### 6.1 错误分类与提示方式

| 错误类型 | 严重程度 | 提示方式 | 示例 |
|---------|---------|---------|------|
| 网络断开 | 🔴 严重 | 全屏提示面板 | 无法连接服务器 |
| 网络超时 | 🟡 中等 | Toast + 重试按钮 | 请求超时，请重试 |
| 操作失败 | 🟡 中等 | 内联错误提示 | 建造失败：位置已被占用 |
| 资源不足 | 🟢 轻微 | 按钮旁红色提示 | 铜钱不足（差200） |
| 权限不足 | 🟡 中等 | Toast | 等级不足，需Lv.10 |
| 数据异常 | 🔴 严重 | 弹窗 + 反馈入口 | 数据加载异常 |
| 服务器维护 | 🔴 严重 | 全屏公告 | 服务器维护中 |

### 6.2 网络错误设计

**网络断开全屏提示**：

```
视觉设计：
  - 背景：水墨山水画（灰度处理），营造「孤城」氛围
  - 中央面板：
    ┌──────────────────────────────┐
    │                              │
    │     🏔️（水墨山水剪影图标）     │
    │                              │
```

### 7.2 字体大小调节

**三级字体大小**：

| 级别 | 基准倍率 | 适用人群 |
|------|---------|---------|

- 字体放大后，面板宽度自适应扩展

## 八、动效性能预算

### 8.1 帧率要求

| 场景 | 目标帧率 | 最低帧率 | 说明 |
|------|:-------:|:-------:|------|
| 主城界面 | 60fps | 45fps | 建筑动画+资源飘字 |
| 天下地图 | 60fps | 30fps | 地图平移/缩放 |
| 武将面板 | 60fps | 60fps | 纯UI，无借口掉帧 |
| 战斗场景 | 30fps | 24fps | 特效密集，允许降帧 |
| 科技树 | 60fps | 45fps | 连线动画 |
| 引导/剧情 | 60fps | 60fps | 必须流畅 |
| 面板过渡 | 60fps | 60fps | 短暂动画，必须流畅 |

### 8.2 粒子数量上限

| 场景/特效 | 最大粒子数 | 粒子生命周期 | 说明 |
|----------|:---------:|:----------:|------|
| 资源飘字 | 10 | 1.2s | 同屏最多10个飘字 |
| 普攻特效 | ≤15 | 0.5s | 斩击弧线 |
| 武技特效 | ≤40 | 1.0s | 火攻水墨 |
| 终极技特效 | ≤80 | 1.5s | 天降陨石 |
| 胜利庆祝 | ≤50 | 3.0s | 金色粒子飘落 |
| 建筑升级 | ≤20 | 1.0s | 升级光效 |
| 引导流光 | ≤5 | 持续 | 镂空边缘流光 |
| 全局同屏上限 | **300** | — | 所有特效粒子总和 |

**粒子池管理**：
- 预分配粒子池（300个粒子对象），避免运行时GC
- 粒子复用：死亡粒子回收入池，新粒子从池中取
- 超过上限时：优先移除最旧的粒子（FIFO）
- 手机端上限降低50%（150个同屏上限）

### 8.3 动画时长标准

| 动画类别 | 时长范围 | 缓动函数 | 说明 |
|---------|:-------:|---------|------|
| 微交互（按钮按下/弹起） | 80-150ms | ease-out | 即时反馈 |
| 小过渡（Tooltip/下拉） | 150-250ms | ease-out | 轻盈感 |
| 中过渡（面板展开/关闭） | 250-400ms | ease-out | 自然展开 |
| 大过渡（页面切换/场景切换） | 400-800ms | ease-in-out | 沉浸感 |
| 特殊动画（战斗/剧情） | 800-2000ms | 自定义 | 戏剧性 |
| 持续动画（流光/脉动） | 800-2000ms/周期 | linear/ease-in-out | 循环播放 |

**缓动函数库**：

```javascript
const Easing = {
  linear:        t => t,
  easeIn:        t => t * t,
  easeOut:       t => t * (2 - t),
  easeInOut:     t => t < 0.5 ? 2*t*t : -1+(4-2*t)*t,
  spring: (stiffness, damping) => {
    // 弹性缓动，stiffness控制弹性，damping控制阻尼
    // 推荐值：spring(1, 0.8) 用于弹窗/卡片
  },
  waterInk: t => {
    // 水墨晕染专用缓动：快速展开→缓慢渗透
    // 前30%时间完成60%进度，后70%时间完成40%进度
    return t < 0.3 ? 2 * t : 0.6 + 0.4 * (t - 0.3) / 0.7;
  }
};
```

### 8.4 内存与GPU预算

| 指标 | PC端预算 | 手机端预算 | 说明 |
|------|---------|-----------|------|
| 动画帧内存 | ≤5MB | ≤2MB | 动画相关纹理/缓冲区 |
| Canvas层数 | ≤3层 | ≤2层 | 主场景+特效层+UI层 |
| GPU Draw Call | ≤50/帧 | ≤30/帧 | 合批渲染 |
| 动画线程 | requestAnimationFrame | requestAnimationFrame | 禁止setInterval |
| 长动画拆分 | >500ms必须可中断 | >500ms必须可中断 | 切换场景时立即停止 |

### 8.5 动画降级策略

```
性能检测（每5秒检测一次帧率）：
  fps ≥ 55  → 全品质动画
  fps 40-54 → 降级Level 1：关闭粒子效果，简化阴影
  fps 30-39 → 降级Level 2：关闭所有特效，仅保留基础过渡
  fps < 30  → 降级Level 3：关闭所有动画，纯静态UI

降级提示：
  首次降级时弹出一次性Toast：
  「检测到设备性能不足，已自动降低动画品质。
   可在设置→显示中手动调整。」
  
恢复策略：
  帧率连续10秒≥55fps → 自动恢复上一品质等级
```
