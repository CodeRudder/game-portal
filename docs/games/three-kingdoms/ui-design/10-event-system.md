> **迭代重点**：明确事件离线处理规则、强化事件与放置核心循环关联、补充手机端完整方案  
> **风格基调**：水墨江山·铜纹霸业

## 一、事件体系总览

### 1.1 事件分类与放置核心循环关联

```
放置核心循环：挂机产出 → 资源积累 → 消费升级 → 解锁新内容 → 挂机产出（加速）

事件系统在循环中的角色：
  ┌─────────────────────────────────────────────────┐
  │  挂机产出 ←── 事件奖励加速产出（天降横财/商人来访） │
  │  资源积累 ←── 事件提供额外资源（随机遭遇奖励）      │
  │  消费升级 ←── 事件触发升级需求（危机→需要强化）     │
  │  解锁内容 ←── 剧情事件解锁新地图/NPC/科技          │
  │  挂机加速 ←── 好事件提供临时产出倍率加成            │
  └─────────────────────────────────────────────────┘
```

### 1.2 事件类型矩阵

| 事件类型 | 触发方式 | 频率 | 与放置循环关系 | 离线处理 |
|---------|---------|:----:|--------------|---------|
| **随机遭遇** | 自动触发（定时器） | 1-2次/小时 | 资源获取/消耗 | 堆积待处理 |
| **剧情事件** | 条件触发（关卡/声望） | 随进度推进 | 解锁新内容 | 暂停等待 |
| **NPC事件** | NPC好感度/日程触发 | 0.5-1次/小时 | 社交资源获取 | 堆积待处理 |
| **天灾人祸** | 随机触发（概率性） | 0.3次/小时 | 资源消耗/防御考验 | 自动处理（按防御值） |
| **限时机遇** | 时间窗口触发 | 2-3次/天 | 高价值资源获取 | 错过即消失 |
| **里程碑事件** | 成就达成触发 | 随进度 | 正反馈强化 | 暂停等待 |

## 二、事件触发通知

### 2.1 急报横幅系统

- 主界面顶部中央降下「急报」横幅
- 配合战鼓音效（首次触发）/ 竹简展开音效（后续）
- 点击横幅展开事件面板
- 3秒未点击自动收起（存入事件日志）

#### 通知优先级规则

```
同时仅显示1条急报横幅，按以下优先级排队：
  1. 限时机遇（倒计时<10分钟）— 红色边框 + 脉动
  2. 天灾人祸（需立即应对）— 橙色边框 + 震动
  3. 剧情事件（关键节点）— 金色边框
  4. 随机遭遇 — 铜色边框
  5. NPC事件 — 银色边框
  6. 里程碑事件 — 绿色边框

排队上限：最多缓存5条急报
超出策略：低优先级事件自动以"最小损失"方案处理
```

### 2.2 事件与放置循环的数值关联

| 事件 | 放置循环影响 | 数值设计 |
|------|------------|---------|
| 商队来访 | 产出加速 | 下2小时产出×1.5 |
| 丰收之年 | 资源获取 | 粮草+500~2000（随主城等级缩放） |
| 流寇来袭 | 资源消耗 | 未处理损失粮草10%~20% |
| 名士拜访 | 解锁内容 | 解锁新NPC或新科技 |
| 天降祥瑞 | 挂机加速 | 全产出×2，持续1小时 |
| 瘟疫蔓延 | 产出降低 | 下2小时产出×0.5（可花费资源消除） |

## 三、🔴 事件离线处理规则（新增 P0-16）

### 3.1 离线处理策略总表

| 事件类型 | 离线处理方式 | 处理规则 | 回归展示 |
|---------|:----------:|---------|---------|
| **随机遭遇（正面）** | 自动完成 | 自动选择"保守"选项，获得50%基础奖励 | 离线摘要中列出 |
| **随机遭遇（负面）** | 自动处理 | 按当前防御值自动计算损失（最低损失） | 离线摘要中列出，损失项标红 |
| **随机遭遇（中性）** | 堆积待处理 | 最多堆积3条，回归后以"急报堆"展示 | 急报堆弹窗 |
| **剧情事件** | 暂停等待 | 不触发、不推进，等玩家在线时继续 | 无（不触发） |
| **NPC事件** | 堆积待处理 | 最多堆积2条 | 急报堆弹窗 |
| **天灾人祸** | 自动处理 | 按防御建筑等级计算减免，剩余损失自动扣除 | 离线摘要中标红展示 |
| **限时机遇** | 错过消失 | 离线期间触发的限时机遇直接错过 | 离线摘要中灰色标注"已错过" |
| **里程碑事件** | 暂停等待 | 不触发，等玩家在线时触发 | 上线后立即弹出 |

### 3.2 离线收益计算公式

```
离线随机遭遇（正面）自动收益 = 基础奖励 × 50% × min(离线时长/1小时, 8)
离线随机遭遇（负面）自动损失 = 基础损失 × max(0, 1 - 防御减免率)
防御减免率 = 防御建筑总等级 / (防御建筑总等级 + 50)
天灾自动损失上限 = 当前资源总量的 15%
```

### 3.3 离线事件堆积上限

- 最大堆积数：5条（超出按优先级丢弃低优先级事件）
- 堆积事件保留时长：72小时（超时自动以"保守"方案处理）
- 离线时长<30分钟：不触发任何离线事件处理

## 八、动效规范

| 动效名称 | 触发场景 | 动画描述 | 时长 | 缓动 |
|---------|---------|---------|:----:|:----:|
| 急报降下 | 事件触发 | 横幅从顶部滑入 + 屏幕微震 + 战鼓声 | 400ms | ease-out |
| 墨迹过渡 | 选择分支后 | 墨迹晕染覆盖当前画面 → 显示结果 | 500ms | ease-in-out |
| 竹简翻转 | 随机遭遇结果 | 竹简面板3D翻转显示结果面 | 300ms | ease-in-out |
| 倒计时脉动 | 限时事件<1小时 | 数字红色脉动 + 横幅边框闪烁 | 1500ms循环 | ease-in-out |
| 横幅碎裂 | 限时活动结束 | 横幅碎裂消散 + 墨迹飞溅 | 600ms | ease-in |
| 急报堆弹出 | 回归多事件 | 多张竹简依次弹出（间隔100ms） | 500ms | ease-out |
| 资源浮动 | 事件结算 | +200 / -100 数字上浮渐隐 | 800ms | ease-out |
| 日志展开 | 点击日志记录 | 记录行向下展开详情 | 200ms | ease-out |

### 性能降级策略

- 低端设备：关闭墨迹过渡（改为淡入淡出）、关闭横幅碎裂、关闭屏幕震动
- 中端设备：简化墨迹过渡（减少粒子）、保留基础动效
- 高端设备：全部动效开启
