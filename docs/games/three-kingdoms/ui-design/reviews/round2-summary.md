## 3. 跨模块共性问题（R2 新发现）

### 🔴 问题1：离线效率系数定义冲突（P0 级）
**影响范围**：资源系统 ↔ 建筑系统 ↔ 主界面（回归欢迎面板）  
**表现**：两套完全不同的衰减曲线（阶梯数/衰减速率/封顶时间均不同），将导致开发混乱和玩家体验不一致  
**建议**：创建独立的"离线收益系统设计"文档，统一定义全局离线效率系数

### 🟡 问题2：货币体系命名不统一（P1 级）
**影响范围**：战斗系统、武将系统、声望转生系统  
**表现**："天命""求贤令""招贤榜"等货币名称在不同文档中混用，未形成统一的货币体系表  
**建议**：创建"货币体系总览"文档，明确定义所有货币的名称、获取途径、用途、是否付费

### 🟡 问题3：回归欢迎体验碎片化（P1 级）
**影响范围**：主界面、资源系统、NPC系统、事件系统  
**表现**：主界面有5步回归弹窗序列、资源系统有离线收益弹窗、NPC系统有离线行为摘要、事件系统有急报堆——四个系统各自设计了独立的回归展示方案，缺少统一整合  
**建议**：定义全局"回归欢迎面板"，按模块Tab整合展示：总览/NPC动态/事件急报/资源收益

### 🟡 问题4：手机端并排对比布局可读性风险（P1 级）
**影响范围**：声望转生（当前 vs 转生后）、远征编队（可用武将 vs 队伍编排）  
**表现**：375px宽度下并排显示两列内容，每列仅约180px，文字频繁换行影响阅读  
**建议**：统一手机端"对比类"布局规范——优先使用Tab切换或上下堆叠，避免并排

### 🟢 问题5：离线效率与声望系统的联动设计优秀（正面发现）
**影响范围**：声望转生、远征系统、战斗系统  
**表现**：三个系统的离线机制均与声望等级挂钩（远征离线效率系数 = 0.5 + 声望等级×0.05），形成"声望越高→离线效率越高→资源获取越快→声望积累越快"的正向飞轮  
**评价**：Round 2 最出色的系统间联动设计

### 🟢 问题6：NPC系统与事件系统联动缺口（P2 级）
**影响范围**：NPC系统 ↔ 事件系统 ↔ 任务系统  
**表现**：NPC事件是否生成对应任务？事件日志与NPC"往昔"对话历史是否互通？两个文档各自设计了独立的摘要面板  
**建议**：建立NPC事件↔任务系统的关联，事件日志与NPC往昔共享底层数据
