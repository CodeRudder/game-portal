# v2.0 招贤纳士 — 经验教训

## 经验教训

### LL-v2-001: 门面导出应随功能同步更新
**现象**：v2.0新增了star-up.types等文件，但engine/index.ts门面未同步导出，导致UI层直接引用引擎内部。
**教训**：每次新增引擎文件时，必须同步更新门面导出。建立"新增即导出"规则(EVO-006)。

### LL-v2-002: 死代码是重复代码的副产品
**现象**：hero-config.ts中的QUALITY_PROBABILITIES是旧版遗留，新版已移到hero-recruit-config.ts。
**教训**：代码迁移后必须验证旧代码是否还被引用，无引用则立即删除。

### LL-v2-003: 测试脚本选择器需与实际DOM匹配
**现象**：测试脚本使用hero-card选择器，但实际DOM使用了不同的class名。
**教训**：测试脚本应先做DOM探测，确认实际class名后再编写断言。在game-actions中添加DOM探测工具函数。

### LL-v2-004: CSS文件500行限制可适当放宽
**现象**：HeroDetailModal.css 513行，主要是响应式媒体查询和动画。
**教训**：CSS文件的500行限制可适当放宽，或允许将媒体查询拆分为独立文件。
