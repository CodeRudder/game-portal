## ⚠️ 需要改进的

### 1. 子系统集成遗漏
- **问题**: CaravanSystem实现完整但未注册到主引擎
- **根因**: 开发时只关注TradeSystem，忽略了CaravanSystem的集成
- **改进**: 新增子系统时使用checklist：import → 字段 → 实例化 → 注册 → reset → getter → 类型声明

### 2. UI面板与引擎API不匹配
- **问题**: TradePanel调用`tradeSystem.sendCaravan()`但该方法不存在
- **根因**: UI开发时未参考引擎API文档
- **改进**: UI开发前先确认引擎API，或从engine-getters-types.ts自动生成API文档

### 3. UI面板过于简化
- **问题**: TradePanel最初只有189行，缺少商路地图、繁荣度、价格波动等核心展示
- **根因**: 初始版本只实现了最基本的数据展示
- **改进**: UI面板开发应对照PRD功能清单逐项实现

## 🔑 关键经验

1. **子系统注册checklist**: import → 字段 → new → register → reset → getter → types，缺一不可
2. **UI开发前确认API**: 避免调用不存在的引擎方法
3. **Mixin模式的getter管理**: engine-getters.ts + engine-getters-types.ts 需同步更新
4. **E2E测试先于手动验证**: 自动化测试能快速发现集成问题
