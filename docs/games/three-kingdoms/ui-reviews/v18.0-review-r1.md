| 11 | GuideOverlay 进度数据不存在 | `undefined` | 中 | UI 层未读取到引导进度数据 |

### 警告根因分析

**核心问题**: guide 模块未集成到引擎（P1 Bug）。`guide/` 目录下的子系统类已实现，但未通过 `engine-guide-deps.ts` 注入到 `ThreeKingdomsEngine`，导致：

1. `TutorialStorage` 未初始化 → localStorage 持久化失效
2. `FirstLaunchDetector` 未初始化 → 首次启动检测失效 → 引导不自动触发
3. 引擎无 guide getter → UI 层无法获取引导状态 → DOM 中无引导元素

**修复方案**: 创建 `engine-guide-deps.ts`，将 7 个 guide 子系统注册到引擎（已实施）。

## 结论

v18.0 新手引导功能的**引擎层代码已完整实现**（guide/ 目录 7 个子系统），但存在**集成遗漏**（P1 Bug），导致引导在桌面端无法自动触发。

移动端测试通过率较高（6/6 通过），说明 UI 组件本身质量良好。P1 修复后需重新执行 E2E 测试验证桌面端引导流程。
