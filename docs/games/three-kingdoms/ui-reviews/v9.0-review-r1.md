## 警告项说明

### B4: 快照机制 — localStorage 无存档数据

- **严重程度**: P3 轻微
- **原因**: 首次访问游戏页面，尚未开始游戏，localStorage 中不存在 `three-kingdoms-save` 键
- **影响**: 无法验证快照机制的完整性
- **建议**: 
  1. 在测试前置步骤中先执行一次游戏初始化（点击"开始游戏"），生成存档数据后再验证
  2. 或在测试脚本中构造 mock 存档数据写入 localStorage
- **验证方式**: 通过源码确认快照机制已实现（`save()` 方法在引擎中正常调用）

## 跳过项说明

### B2: 离线收益弹窗结构

- **原因**: 依赖 B4 的存档数据，无存档则无法模拟离线场景
- **建议**: 同 B4 的修复方案

### B3: 衰减系数配置

- **原因**: React Fiber 树遍历无法获取引擎实例（模块作用域隔离）
- **说明**: 这是预期行为，Vite + React 的模块系统天然隔离了内部状态
- **替代验证**: 通过源码确认 `DECAY_TIERS` 配置已定义
