## 1. 设计概述

### 1.1 设计目标

| 目标 | 指标 | 说明 |
|------|------|------|
| 收集驱动力 | 特定武将组合价值提升 | 搭档羁绊让低品质武将有独特价值 |
| 阵营策略性 | 编队需考虑阵营平衡 | 阵营羁绊鼓励同阵营收集 |
| 战力提升幅度 | 羁绊总加成 ≤ 基础战力的15~25% | 不压倒等级/品质/星级的核心地位 |
| 新手可感知 | 首次激活羁绊有明确反馈 | 引导玩家关注羁绊组合 |

### 1.2 设计原则

1. **独立乘区**：羁绊加成作为战力计算的独立乘区，不与品质/星级/装备乘区叠加计算
2. **渐进激活**：2人即可激活基础羁绊，降低门槛；4人/5人满羁绊为追求目标
3. **派驻减半**：派驻武将也算羁绊人数，但效果只有50%，鼓励上阵而非仅收集
4. **数值克制**：羁绊加成幅度控制在合理范围，不破坏现有经济和战力平衡

### 1.3 与现有战力体系的关系

现有战力公式（HER-1）：
```
单将战力 = (ATK×2.0 + INT×2.0 + CMD×1.5 + POL×1.0) × 等级系数 × 品质系数 × 星级系数 × 装备系数
```

羁绊系统新增 **羁绊系数** 作为第5个乘区：
```
单将战力 = (...原有公式...) × 羁绊系数
羁绊系数 = 1 + Σ(所有激活羁绊对该武将的加成百分比)
```

**示例**：关羽在"桃园结义+蜀国4人"激活时：
- 桃园结义：攻击+15% → 羁绊系数贡献 +0.15
- 蜀国4人：攻击+15%，防御+10%，生命+5% → 羁绊系数贡献 +0.30
- 羁绊系数 = 1 + 0.15 + 0.30 = 1.45
- 关羽战力 × 1.45（提升45%）

> **平衡说明**：满羁绊45%提升看似较高，但需要同时拥有并上阵4~5名特定武将，收集和编队成本极高。日常2~3人羁绊提升约15~25%，与升一星（15~25%）相当，不会压倒升星的核心地位。

## 2. 羁绊分类与效果

### 2.1 阵营羁绊（魏蜀吴群）

阵营羁绊基于编队中同阵营武将数量激活，所有同阵营武将共享加成。

| 阵营 | 2人激活 | 3人激活 | 4人激活 | 设计特色 |
|------|:------:|:------:|:------:|---------|
| **蜀国** | 攻击+5% | 攻击+10%，防御+5% | 攻击+15%，防御+10%，生命+5% | 攻防兼备，均衡型 |
| **魏国** | 防御+5% | 防御+10%，智力+5% | 防御+15%，智力+10%，被动触发率+10% | 铁壁防御，谋略型 |
| **吴国** | 速度+5% | 速度+10%，攻击+5% | 速度+15%，攻击+10%，暴击率+10% | 疾风迅雷，暴击型 |
| **群雄** | 智力+5% | 智力+10%，防御+5% | 智力+15%，防御+10%，技能范围+1 | 奇策百出，策略型 |

**阵营羁绊说明**：
- 2人激活门槛低，新手容易触发
- 4人激活为追求目标（编队6人中4人同阵营，剩余2人需考虑搭档羁绊或混搭）
- 各阵营特色与武将属性分布匹配（蜀多猛将→攻击、魏多谋士→智力/防御、吴多速攻→速度/暴击、群多奇人→智力/策略）

## 3. 羁绊等级系统

### 3.1 羁绊等级规则

羁绊等级由参与武将的最低星级决定：

| 羁绊等级 | 激活条件 | 效果倍率 | 说明 |
|:-------:|---------|:-------:|------|
| Lv1 | 所有参与武将 ≥ 1星 | ×1.0（基础效果） | 刚激活时的基础效果 |
| Lv2 | 所有参与武将 ≥ 3星 | ×1.5 | 效果提升50% |
| Lv3 | 所有参与武将 ≥ 5星 | ×2.0 | 效果翻倍，满级羁绊 |

**示例**：
- 桃园结义Lv1（刘关张均≥1星）：全攻击+15%
- 桃园结义Lv2（刘关张均≥3星）：全攻击+22.5%（15%×1.5）
- 桃园结义Lv3（刘关张均≥5星）：全攻击+30%（15%×2.0）

### 3.2 羁绊等级提升路径

```
激活羁绊(Lv1) → 升星参与武将 → 最低星级达标 → 羁绊自动升级
```

| 阶段 | 羁绊等级 | 武将星级要求 | 预估达成时间 | 碎片总消耗（每武将） |
|:----:|:-------:|:----------:|:----------:|:-----------------:|
| 新手期 | Lv1 | 1星（初始） | 招募即有 | 0 |
| 成长期 | Lv2 | 3星 | 2~3周 | 60碎片+15,000铜钱 |
| 后期 | Lv3 | 5星 | 1~2月 | 290碎片+85,000铜钱 |

### 3.3 羁绊等级可视化

| 等级 | 图标边框 | 特效 | 文字颜色 |
|:----:|---------|------|---------|
| Lv1 | 银色 | 无 | 白色 |
| Lv2 | 金色 | 微光 | 金色 |
| Lv3 | 赤金+流光 | 脉冲光环 | 橙色 |

## 4. 羁绊激活条件

### 4.1 激活规则

| 规则 | 说明 |
|------|------|
| 阵营羁绊 | 编队中同阵营武将数量达到阈值（2/3/4人） |
| 搭档羁绊 | 编队中同时包含羁绊所需的所有武将 |
| 派驻也算 | 派驻到建筑中的武将计入羁绊人数，但效果减半（50%） |
| 多羁绊叠加 | 同一武将可同时享受多个羁绊加成（阵营+搭档可叠加） |
| 实时计算 | 编队变更时立即重新计算羁绊激活状态 |

### 4.2 派驻减半规则

```
羁绊效果 = 基础效果 × 羁绊等级倍率 × 派驻系数
派驻系数：
  - 上阵武将：1.0（100%效果）
  - 派驻武将：0.5（50%效果）
  - 未上阵/未派驻：不计入羁绊人数
```

**示例**：蜀国4人羁绊（攻击+15%，防御+10%，生命+5%）
- 4人全部上阵：攻击+15%，防御+10%，生命+5%
- 3人上阵+1人派驻：攻击+15%×(3×1.0+1×0.5)/4 = +11.25%，防御+7.5%，生命+3.75%
- 2人上阵+2人派驻：攻击+15%×(2×1.0+2×0.5)/4 = +7.5%，防御+5%，生命+2.5%

> **设计意图**：派驻减半鼓励玩家将羁绊武将上阵而非仅派驻，同时不惩罚同时使用派驻功能的玩家。

### 4.3 羁绊冲突与优先级

| 场景 | 规则 | 说明 |
|------|------|------|
| 同类羁绊叠加 | 不叠加，取最高等级 | 如蜀国3人+蜀国4人同时满足，取4人效果 |
| 不同类羁绊叠加 | 可叠加 | 阵营羁绊+搭档羁绊效果相加 |
| 多个搭档羁绊 | 可叠加 | 如关羽同时享受"桃园结义"+"五虎上将" |
| 羁绊上限 | 羁绊系数上限2.0 | 防止极端组合导致战力膨胀 |

## 5. 羁绊战力集成

### 5.1 战力公式更新

```
单将战力 = 基础战力 × 羁绊系数

基础战力 = (ATK×2.0 + INT×2.0 + CMD×1.5 + POL×1.0)
           × 等级系数 × 品质系数 × 星级系数 × 装备系数

羁绊系数 = 1 + Σ(激活羁绊对该武将的属性加成%)
         = min(2.0, 1 + Σ加成)  // 上限2.0
```

### 5.2 羁绊对战力的影响分析

以关羽（EPIC品质，3星，Lv30）为例：

| 状态 | 战力计算 | 战力值 | 提升幅度 |
|------|---------|:------:|:-------:|
| 无羁绊 | 基础战力 × 1.0 | 2,850 | — |
| +蜀国3人 | 基础战力 × 1.15 | 3,278 | +15% |
| +蜀国4人 | 基础战力 × 1.30 | 3,705 | +30% |
| +蜀国4人+桃园结义Lv1 | 基础战力 × 1.45 | 4,133 | +45% |
| +蜀国4人+桃园结义Lv3 | 基础战力 × 1.60 | 4,560 | +60% |

> **平衡结论**：满羁绊（60%提升）需要同时拥有刘备+关羽+张飞+1名蜀国武将，且刘关张均5星。收集成本极高（3名EPIC武将升至5星需约870碎片+255,000铜钱），60%提升是对巨额投入的合理回报。日常2~3人羁绊约15~25%提升，与升一星效果相当。

### 5.3 羁绊与现有系统的数值兼容性

| 系统 | 现有提升幅度 | 羁绊提升幅度 | 是否兼容 |
|------|:----------:|:----------:|:-------:|
| 升1星（1→2星） | 全属性+15% | — | ✅ 基准 |
| 升2星（2→3星） | 全属性+17% | — | ✅ 基准 |
| 品质提升（RARE→EPIC） | 属性×1.15 | — | ✅ 基准 |
| 2人阵营羁绊 | — | 单属性+5% | ✅ 远低于升星 |
| 3人阵营羁绊 | — | 双属性+15% | ✅ 约等于升1星 |
| 满羁绊（4人+搭档Lv3） | — | 全属性+60% | ✅ 需极高收集成本 |

## 6. 羁绊数据结构

### 6.1 羁绊配置表（TypeScript接口）

```typescript
/** 羁绊类型 */
type BondType = 'faction' | 'partner';

/** 羁绊等级配置 */
interface BondLevelConfig {
  level: 1 | 2 | 3;
  minStar: number;        // 最低星级要求
  effectMultiplier: number; // 效果倍率
}

/** 属性加成 */
interface AttributeBonus {
  attack?: number;    // 攻击加成%
  defense?: number;   // 防御加成%
  intelligence?: number; // 智力加成%
  speed?: number;     // 速度加成%
  hp?: number;        // 生命加成%
  critRate?: number;  // 暴击率加成%
  critDamage?: number; // 暴击伤害加成%
  skillDamage?: number; // 技能伤害加成%
}

/** 羁绊配置 */
interface BondConfig {
  id: string;              // 羁绊ID，如 'bond_shu_4'
  name: string;            // 羁绊名称，如 '蜀国·众志成城'
  type: BondType;          // 羁绊类型
  description: string;     // 羁绊描述
  icon: string;            // 羁绊图标

  // 激活条件
  faction?: 'shu' | 'wei' | 'wu' | 'qun';  // 阵营羁绊：阵营
  factionCount?: 2 | 3 | 4;                  // 阵营羁绊：人数阈值
  heroIds?: string[];                         // 搭档羁绊：所需武将ID列表

  // 效果
  baseEffect: AttributeBonus;  // 基础效果（Lv1）
  levels: BondLevelConfig[];   // 等级配置

  // 元数据
  priority: number;  // 实现优先级（0=P0, 1=P1, 2=P2）
  batch: number;     // 实现批次（1/2/3）
}
```

### 6.3 羁绊计算逻辑（伪代码）

```typescript
/**
 * 计算编队中所有激活的羁绊
 * @param teamHeroIds 编队武将ID列表
 * @param garrisonHeroIds 派驻武将ID列表
 * @param heroStars 武将星级映射 { heroId: star }
 * @returns 每个武将的羁绊系数 { heroId: bondCoeff }
 */
function calculateBondEffects(
  teamHeroIds: string[],
  garrisonHeroIds: string[],
  heroStars: Record<string, number>
): Record<string, number> {
  const allActiveHeroIds = [...teamHeroIds, ...garrisonHeroIds];
  const bondCoeffs: Record<string, number> = {};

  // 初始化所有武将羁绊系数为1.0
  for (const heroId of allActiveHeroIds) {
    bondCoeffs[heroId] = 1.0;
  }

  // 1. 计算阵营羁绊
  for (const faction of ['shu', 'wei', 'wu', 'qun']) {
    const factionHeroes = allActiveHeroIds.filter(id => getHeroFaction(id) === faction);
    const maxCount = Math.min(factionHeroes.length, 4);

    if (maxCount >= 2) {
      const bondConfig = getFactionBond(faction, maxCount);
      const minStar = Math.min(...factionHeroes.map(id => heroStars[id] || 1));
      const levelConfig = getBondLevel(bondConfig, minStar);
      const effectValue = calculateTotalBonus(bondConfig.baseEffect) * levelConfig.effectMultiplier;

      for (const heroId of factionHeroes) {
        const garrisonMultiplier = garrisonHeroIds.includes(heroId) ? 0.5 : 1.0;
        bondCoeffs[heroId] += effectValue * garrisonMultiplier;
      }
    }
  }

  // 2. 计算搭档羁绊
  for (const bondConfig of PARTNER_BONDS) {
    const requiredIds = bondConfig.heroIds;
    const allPresent = requiredIds.every(id => allActiveHeroIds.includes(id));

    if (allPresent) {
      const minStar = Math.min(...requiredIds.map(id => heroStars[id] || 1));
      const levelConfig = getBondLevel(bondConfig, minStar);
      const effectValue = calculateTotalBonus(bondConfig.baseEffect) * levelConfig.effectMultiplier;

      for (const heroId of requiredIds) {
        if (allActiveHeroIds.includes(heroId)) {
          const garrisonMultiplier = garrisonHeroIds.includes(heroId) ? 0.5 : 1.0;
          bondCoeffs[heroId] += effectValue * garrisonMultiplier;
        }
      }
    }
  }

  // 3. 应用上限
  for (const heroId of Object.keys(bondCoeffs)) {
    bondCoeffs[heroId] = Math.min(2.0, bondCoeffs[heroId]);
  }

  return bondCoeffs;
}
```
