## 1. 武将↔派驻联动

> **实现状态**: 🔧 部分实现 — `BuildingSystem` 支持派驻槽位，`garrison-production.integration.test.ts` 验证了防御加成公式

### 1.1 派驻条件

| 条件 | 规则 | 引擎状态 |
|------|------|:-------:|
| 武将等级 | ≥ 建筑要求等级（市集Lv5需武将Lv10+） | ✅ 已实现 |
| 武将品质 | ≥ 建筑要求品质（部分高级建筑需FINE+） | 📝 设计中 |
| 派驻槽位 | 每建筑最多1名武将，全局无限制 | ✅ 已实现 |
| 派驻状态 | 已派驻武将不可重复派驻到其他建筑 | ✅ 已实现 |

### 1.2 派驻加成

| 建筑类型 | 加成公式 | 效果 | 引擎状态 |
|---------|---------|------|:-------:|
| 农田 | 武将POL × 0.01 × 建筑等级 | 粮草产出+% | ✅ 已实现 |
| 市集 | 武将POL × 0.015 × 建筑等级 | 铜钱产出+% | ✅ 已实现 |
| 兵营 | 武将CMD × 0.01 × 建筑等级 | 兵力产出+% | ✅ 已实现 |
| 书院 | 武将INT × 0.005 × 建筑等级 | 科技点产出+% | 📝 设计中 |
| 所有建筑 | 武将DEF × 0.003 | 防御加成（离线保护） | ✅ 已实现 |

> **引擎来源**: `garrison-production.integration.test.ts` — 防御加成 = 武将defense × 0.003

### 1.3 派驻收益

```
派驻产出加成 = 武将属性 × 属性系数 × 建筑等级
实际产出 = 基础产出 × (1 + 派驻加成)
```

**示例（诸葛亮 POL=120，市集Lv5）**：
- 加成 = 120 × 0.015 × 5 = 9.0 → 铜钱产出 +9.0/秒
- 市集Lv5基础 = 2.5/秒 → 实际 = 2.5 × (1 + 9.0) = 25.0/秒

### 1.4 派驻解除

| 操作 | 效果 | 冷却 |
|------|------|------|
| 解除派驻 | 产出回到基础值 | 无冷却 |
| 更换派驻 | 先解除再派驻 | 无冷却 |
| 武将升级 | 加成实时更新 | — |
| 武将升星 | 加成实时更新 | — |

## 2. 武将↔羁绊联动

> **实现状态**: 📝 设计中 — PRD已定义羁绊概念，引擎无羁绊系统实现

### 2.1 羁绊激活条件

| 羁绊类型 | 激活条件 | 效果范围 |
|---------|---------|---------|
| 阵营羁绊 | 同阵营≥2/3/4名武将 | 全阵营属性+% |
| 搭档羁绊 | 特定武将组合（如刘关张） | 特殊效果解锁 |
| 品质羁绊 | 同品质≥3/5名武将 | 全品质属性+% |
| 星级羁绊 | ≥3名武将达到指定星级 | 额外加成 |

### 2.2 阵营羁绊效果表

| 阵营 | 2人激活 | 3人激活 | 4人激活 |
|------|:------:|:------:|:------:|
| 蜀国 | ATK +5% | ATK +10%, HP +5% | ATK +15%, HP +10%, 技能CD -1 |
| 魏国 | DEF +5% | DEF +10%, INT +5% | DEF +15%, INT +10%, 被动触发率+10% |
| 吴国 | SPD +5% | SPD +10%, ATK +5% | SPD +15%, ATK +10%, 暴击率+10% |
| 群雄 | INT +5% | INT +10%, DEF +5% | INT +15%, DEF +10%, 技能范围+1 |

### 2.3 搭档羁绊示例

> **P1-3 更新**：搭档羁绊已从3组扩充到12组（蜀3/魏3/吴3/群3），大幅增强收集驱动力。

| 羁绊名 | 武将组合 | 效果 | 阵营 |
|--------|---------|------|:----:|
| 桃园结义 | 刘备+关羽+张飞（3人） | 攻击+15% | 蜀 |
| 五虎上将 | 关羽+张飞+赵云+马超+黄忠（任意3人） | 暴击率+10% | 蜀 |
| 卧龙凤雏 | 诸葛亮+庞统（2人） | 技能伤害+20% | 蜀 |
| 五子良将 | 张辽+徐晃+于禁+张郃+乐进（任意3人） | 防御+12% | 魏 |
| 曹氏宗族 | 曹仁+曹洪+夏侯惇+夏侯渊（任意2人） | 生命+15% | 魏 |
| 虎痴双雄 | 许褚+典韦（2人） | 攻击+12%，防御+8% | 魏 |

### 2.4 羁绊与武将收集的关系

- 羁绊系统是武将收集的核心驱动力之一
- 激活羁绊需要收集特定组合的武将，与招贤令经济形成闭环
- 阵营羁绊要求阵营平衡收集，避免只抽高级池
- 搭档羁绊增加特定武将的价值（如张飞单独较弱，但与刘关组合极强）

## 3. 武将↔装备联动

> **实现状态**: ✅ 已实现 — `EquipmentSystem` 完整实现，含装备穿戴/卸下/强化/精炼/分解

### 3.1 装备需求

| 条件 | 规则 | 引擎状态 |
|------|------|:-------:|
| 武将等级 | 装备有最低等级要求（白装Lv1/绿装Lv10/蓝装Lv20/紫装Lv30/金装Lv40） | ✅ 已实现 |
| 武将品质 | 高品质装备需武将品质≥FINE | 📝 设计中 |
| 装备槽位 | 每武将4个槽位（武器/防具/饰品/兵书） | ✅ 已实现 |
| 装备绑定 | 装备穿戴后绑定，卸下需消耗 | ✅ 已实现 |

### 3.2 装备加成

```
装备总战力 = Σ(单件装备战力)
单件装备战力 = 基础属性 × (1 + 强化等级 × 0.1) × (1 + 精炼等级 × 0.05) × 套装系数
武将战力 = ... × (1 + 装备总战力 / 1000)
```

| 装备品质 | 基础属性范围 | 强化上限 | 精炼上限 | 套装件数 |
|---------|:----------:|:-------:|:-------:|:-------:|
| 白色(Common) | 10~30 | +5 | — | — |
| 绿色(Fine) | 30~60 | +10 | +3 | 2件套 |
| 蓝色(Rare) | 60~100 | +15 | +5 | 2件套 |
| 紫色(Epic) | 100~180 | +20 | +8 | 4件套 |
| 金色(Legendary) | 180~300 | +25 | +10 | 4件套 |

> **引擎来源**: `EquipmentSystem.ts` — 装备系数 = 1 + totalEquipmentPower / 1000

### 3.3 装备获取与武将成长

| 获取途径 | 装备品质 | 频率 | 引擎状态 |
|---------|:-------:|------|:-------:|
| 关卡掉落 | 白~蓝 | 每次通关 | ✅ 已实现 |
| 装备锻造 | 白→绿→蓝→紫 | 消耗材料 | ✅ 已实现 |
| 商店购买 | 白~紫 | 每日刷新 | ✅ 已实现 |
| 装备分解 | 获得铜钱和材料 | 不定期 | ✅ 已实现 |
| 竞技场奖励 | 紫~金 | 每周结算 | ✅ 已实现 |
| 活动奖励 | 金 | 限时活动 | ✅ 已实现 |

### 3.4 装备推荐系统

> **引擎来源**: `BuildingRecommender.ts` — 建筑推荐系统已实现，装备推荐可复用模式

| 场景 | 推荐逻辑 | 优先级 |
|------|---------|:------:|
| 新获得武将 | 推荐当前库存中最匹配的装备 | 高 |
| 装备升级 | 推荐强化/精炼性价比最高的装备 | 中 |
| 套装收集 | 推荐缺少的套装部件 | 低 |

## 4. 武将↔战斗联动

> **实现状态**: ✅ 已实现 — `CampaignSystem`、`BattleSystem` 已实现完整战斗流程

### 4.1 战力公式

```
武将战力 = (ATK × 2.0 + INT × 2.0 + CMD × 1.5 + POL × 1.0)
           × (1 + 等级 × 0.05)
           × 品质系数
           × 星级系数
           × 装备系数(1 + totalEquipmentPower / 1000)
```

| 系数 | 公式 | 引擎来源 |
|------|------|---------|
| 等级系数 | 1 + 等级 × 0.05 | `hero-config.ts` → `LEVEL_COEFFICIENT_PER_LEVEL` |
| 品质系数 | COMMON=1.0, FINE=1.15, RARE=1.3, EPIC=1.5, LEGENDARY=1.8 | `hero-config.ts` → `QUALITY_MULTIPLIERS` |
| 星级系数 | 1星=1.0, 2星=1.15, 3星=1.35, 4星=1.6, 5星=2.0, 6星=2.5 | `star-up-config.ts` → `STAR_MULTIPLIERS` |
| 装备系数 | 1 + totalEquipmentPower / 1000 | `HeroSystem.calculatePower()` |

### 4.2 战斗编队

| 配置项 | 规则 | 引擎状态 |
|--------|------|:-------:|
| 上阵人数 | 最多5名武将 | ✅ 已实现 |
| 阵营要求 | 无限制（但阵营羁绊需同阵营） | ✅ 已实现 |
| 前后排 | 前排3名（承伤），后排2名（输出/辅助） | ✅ 已实现 |
| 武将状态 | 已派驻建筑的可同时上阵战斗 | ✅ 已实现 |

### 4.3 战斗奖励

| 奖励类型 | 计算方式 | 引擎状态 |
|---------|---------|:-------:|
| 经验值 | 关卡等级 × 10（分配给上阵武将） | ✅ 已实现 |
| 铜钱 | 100 + 关卡等级 × 20 | ✅ 已实现 |
| 装备掉落 | 按关卡配置掉落表 | ✅ 已实现 |
| 碎片掉落 | 精英关专属，按关卡配置 | ✅ 已实现 |
| 突破石掉落 | 精英关专属，1~2个/关 | ✅ 已实现 |
| 首通奖励 | 5~10招贤令 | ✅ 已实现 |

### 4.4 战斗与武将成长闭环

```
┌─────────────────────────────────────────────────────────┐
│                  战斗→武将成长闭环                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  战斗通关                                                │
│    ├── 经验值 → 武将升级 → 属性提升 → 战力提升            │
│    ├── 铜钱 → 商店购买 → 招贤令/突破石/技能书             │
│    ├── 装备掉落 → 武将穿戴 → 装备系数提升 → 战力提升       │
│    ├── 碎片掉落 → 武将升星 → 星级系数提升 → 战力提升       │
│    └── 突破石 → 武将突破 → 等级上限提升 → 可继续升级       │
│                                                         │
│  战力提升                                                 │
│    ├── 通关更高难度关卡 → 更好的奖励                       │
│    ├── 竞技场排名提升 → 更多铜钱/竞技币                    │
│    └── 远征解锁 → 技能书/装备材料                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```
