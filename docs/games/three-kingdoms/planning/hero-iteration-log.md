### 二、招贤令经济循环修复

**问题**：招贤令产出极低（原0.001/秒），获取途径缺失，导致玩家无法正常进行招募。

**解决方案**（PRD新增 HER-7 章节）：

| 修复项 | 具体措施 | 预期效果 |
|--------|---------|---------|
| 被动产出提升 | 0.001→0.003/秒（3倍提升） | 在线日产出约259招贤令 |
| 新手礼包 | 首次登录赠送50招贤令 | 新手立即体验招募 |
| 日常任务 | 每日完成3个任务奖励5招贤令 | 日活跃奖励 |
| 商店购买 | 100铜钱/个，每日限购30个 | 铜钱转化通道 |
| 关卡首通 | 3~5招贤令/关 | 推关动力 |
| 活动奖励 | 10~20招贤令 | 活跃度刺激 |
| 离线收益 | 50%效率产出 | 离线不浪费 |

**预期日产出**：在线玩家每日约264~294招贤令，可支持2~3次高级招贤或大量普通招贤。

### 三、突破石获取途径补充

**问题**：突破石是武将突破的必要材料（需求5→10→20→40），但无任何获取途径定义。

**解决方案**（PRD新增 HER-8 章节）：

| 获取途径 | 产出 | 频率 |
|---------|------|------|
| 关卡掉落 | 1~3个/关 | 每次通关 |
| 关卡扫荡 | 1~2个/次 | 每日扫荡 |
| 商店购买 | 500铜钱/个 | 每日限购20个 |

| 活动奖励 | 10~30个 | 限时活动 |

**设计目标**：单武将全部突破需75个突破石，约2~3周完成。

## v1.2 — R2经济修正与武将扩展路线图

**日期**: 2026-06-10
**触发原因**: R2评测（综合评分6.8/10）发现6个关键问题：招贤令日产出高估37%~67%、铜钱竞争激烈但无产出定义、突破石过于宽松、武将品质头重脚轻、远征频率未定义、文档与引擎隐性不一致
**变更范围**: 仅修改文档，不修改引擎代码

### 一、R2发现的6个问题及修复方案

#### 问题1：招贤令日产出高估37%~67%

**现象**：PRD基于24h在线计算264~294招贤令/日，实际4h在线仅156~186。100抽保底需54~64天而非PRD暗示的33天。

**根因**：经济设计以"24小时在线"为基准，严重脱离真实玩家行为。被动产出占比70%+，玩家行为对产出影响极低。

**修复**（HER-10.2 招贤令经济修正）：
- 被动产出从0.003/秒降至0.002/秒（降低被动占比）
- 日常任务从5→15招贤令（提升主动获取3倍）
- 商店限购从30→50个，单价从100→50铜钱（降低铜钱压力）
- 新手礼包从50→100招贤令（可立即体验1次高级招贤）
- 关卡首通从3~5→5~10招贤令（增强推关动力）
- 活动奖励从10~20→20~30招贤令

**修复后效果**：
- 中度玩家（4h在线）日产出约191招贤令（含活动）
- 重度玩家（8h在线）日产出约246招贤令
- 100抽保底：中度玩家约52天，重度玩家约41天
- 主动获取途径占比提升至50%+

#### 问题2：铜钱竞争激烈但无产出定义

**现象**：每日商店全额购买需13,000铜钱，加上升级/升星/突破/技能/碎片6条线并行消耗，但铜钱日产出未定义。

**修复**（HER-10.3 铜钱经济新增）：
- 定义铜钱日产出：建筑8,000 + 关卡3,000~5,000 + 日常2,000 + 扫荡3,000 + 其他2,000~5,000 ≈ 19,000~22,000
- 定义各系统消耗占比：商店38% + 升级18% + 技能8% + 碎片12% + 升星储备15%
- 设计原则：日产出≈日消耗，商店占比≤40%，优先级引导

#### 问题3：突破石过于宽松

**现象**：日产出47.5个，全突破仅需1.6天，远低于PRD声称的"2~3周"。

**根因**：关卡掉落"每次通关1~3个"过于慷慨，普通关卡也掉落突破石。

**修复**（HER-10.4 突破石经济修正）：
- 关卡掉落改为精英关专属（普通关不再掉落）
- 扫荡次数从5次减至3次，产出减半
- 商店限购从20个减至10个
- 联盟限购从30个减至15个
- 活动产出从10~30减至5~15

**修复后效果**：
- 日产出从47.5降至~17.5个
- 单武将全突破约4~5天（快速反馈）
- 5武将并行全突破约3周（符合原"2~3周"设计意图）

#### 问题4：武将品质头重脚轻

**现象**：传说36% vs 稀有仅7%（1名），吴国仅2名武将无法触发3人羁绊。

**修复**（HER-11 武将扩展路线图新增）：
- 短期（+6名）：补充3名RARE品质（甘宁/太史慈/魏延），4名吴国武将
- 中期（+10名）：司马懿/孙策/庞统等，阵营平衡
- 长期（+20名）：完整三国武将库，目标50名

#### 问题5：远征频率未定义

**现象**：技能书经济无法评估，远征是技能书核心来源但频率未定义。

**修复**（HER-10.5 技能书经济新增）：
- 定义远征频率：每日2次，30分钟/次
- 技能书日产出约8.8本
- 单武将双技能满级约3天，5武将约2周

#### 问题6：文档与引擎隐性不一致

**现象**：PRD写0.003/秒但引擎仍是0.001/秒，所有经济系统仅文档未实现。新开发者可能误以为这些功能已实现。

**修复**：
- PRD每个章节开头添加实现状态标记（✅已实现/📝设计中/🔧部分实现）
- 明确标注"待实现"状态，区分已实现和设计中的功能
- HER-7/8/9标注"📝 设计中，本章节保留作为设计参考，详见HER-10"

## v1.3 — R3招贤令经济数值修复与流程文档补充

**日期**: 2026-06-12
**触发原因**: R2评测发现招贤令经济存在严重数值问题（P0-R2-1）：日产出264~294基于24h在线假设，实际玩家仅45~67；普通招贤日抽264次，14名武将1天集齐，收集玩法崩溃。R2同时指出缺少武将系统核心流程文档（P1-R2-1）。
**变更范围**: 引擎配置修改（普通招募消耗1→5）+ PRD文档更新 + 新建流程文档

### 一、引擎配置修改

#### 1.1 普通招募消耗修正（P0-R2-1 修复）

| 配置项 | 修改前 | 修改后 | 文件 |
|--------|:-----:|:-----:|------|
| `RECRUIT_COSTS.normal.amount` | 1 | **5** | `hero-recruit-config.ts` |

**修改理由**：
- 原设计普通招募消耗1招贤令，日抽264次导致14名武将1天集齐
- 修正为5招贤令后，日抽次数降至~48次，收集周期延长至2~4周
- 高级招募消耗维持100招贤令不变，保持100:5=20:1的消耗梯度

- `INITIAL_LEVEL_CAP = 30` / `FINAL_LEVEL_CAP = 70`：突破系统控制的动态等级上限
- `HeroLevelSystem` 未读取 `HeroStarSystem.getLevelCap()` 的动态上限，导致突破到三阶/四阶后无法继续升级
- PRD文档中 `HERO_MAX_LEVEL = 50` 与突破系统最终上限70的描述存在矛盾

**为什么连续多轮标记为"未解决"？**

3. **属性命名**：引擎使用英文命名（attack/defense），PRD使用中文缩写（ATK/CMD），PRD已有映射表。**根因**：映射关系不够醒目，容易被忽略。

1. PRD变更历史中的旧数值使用 ~~删除线~~ 标注已过时

### 三、战力公式（R7更新）

```
战力 = floor(statsPower × levelCoeff × qualityCoeff × starCoeff × equipmentCoeff × bondCoeff)
```

| 乘区 | 系数 | 来源 | 默认值 |
|------|------|------|--------|
| 基础属性 | statsPower = ATK×2 + DEF×1.5 + INT×2 + SPD×1 | hero-config | — |
| 等级系数 | 1 + level × 0.05 | hero-config | 1.05 (Lv1) |
| 品质系数 | QUALITY_MULTIPLIERS[quality] | hero-config | 1.0~2.0 |
| 星级系数 | getStarMultiplier(star) | star-up-config | 1.0 (1星) |
| 装备系数 | 1 + equipPower / 1000 | EquipmentSystem注入 | 1.0 |
| **羁绊系数** | BondSystem.getBondMultiplier(ids) | BondSystem注入 | **1.0** |
