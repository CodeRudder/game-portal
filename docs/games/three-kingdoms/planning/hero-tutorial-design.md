## 1. 设计背景与目标

### 1.1 R4问题分析

| 问题编号 | 来源 | 严重度 | 问题描述 |
|:--------:|:----:|:------:|---------|
| TUT-R4-01 | R4评测 (4.5/10) | P1 | 新手引导评分持续偏低（R1:3.0→R2:3.5→R3:4.0→R4:4.5），提升缓慢 |
| TUT-R4-02 | R4评测 | P1 | 引导流程仅覆盖招募，升级/编队/战斗/升星/羁绊均无引导 |
| TUT-R4-03 | R4评测 | P2 | 引导不可跳过，老玩家/回流玩家体验差 |
| TUT-R4-04 | R4评测 | P1 | 引导后玩家不知道"接下来该做什么"，缺乏后续目标引导 |

### 1.2 设计目标

| 目标 | 指标 | 当前值 | 目标值 |
|------|------|--------|--------|
| 新手引导评分 | R5评测 | 4.5/10 | ≥ 7.5/10 |
| 引导完成率 | 新玩家数据 | ~30%（估算） | ≥ 85% |
| 首次招募后留存 | 次日留存 | ~40%（估算） | ≥ 65% |
| 引导后武将升级率 | 新玩家数据 | ~15%（估算） | ≥ 70% |
| 引导后首次战斗参战率 | 新玩家数据 | ~40%（估算） | ≥ 90% |

### 1.3 设计原则

1. **渐进式教学**：每次只教一个概念，不叠加信息
2. **即时正反馈**：每步引导完成后立即给予奖励，建立成就感
3. **可跳过可回看**：右上角始终有"跳过"按钮，设置中可重播
4. **非强制不卡死**：引导等待玩家操作，但超时后自动推进
5. **新手加速期**：前7天享受特殊优惠，降低收集门槛

## 2. 引导架构设计

### 2.1 5步引导流程总览

```
游戏开始
  │
  ▼
┌─────────────────────────────────────────────┐

               │ 完成后自动触发
               ▼
┌─────────────────────────────────────────────┐

│ 触发：完成第1步后立即触发                       │

└──────────────┬──────────────────────────────┘
               │ 完成后自动触发
               ▼
┌─────────────────────────────────────────────┐
│ 第3步：编队上阵（~7分钟）                      │
│ 触发：完成第2步后立即触发                       │
│ 核心：编队界面 → 拖拽上阵 → 确认                │
│ 奖励：铜钱×2000 + 招贤令×50                    │
└──────────────┬──────────────────────────────┘
               │ 完成后自动触发
               ▼
┌─────────────────────────────────────────────┐
│ 第4步：首次战斗（~8分钟）                      │
│ 触发：完成第3步后立即触发                       │
│ 核心：进入关卡 → 自动战斗 → 战利品              │

│ 奖励：铜钱×5000 + 突破石×10                   │
└─────────────────────────────────────────────┘
```

### 2.2 与现有引导系统的衔接

```
现有引导流程（14-tutorial.md / TUT-tutorial-prd.md）：
  步骤1: 主城概览
  步骤2: 建造农田
  步骤3: 招募武将 ← 本方案衔接点
  步骤4: 首次出征
  步骤5: 查看资源
  步骤6: 科技研究

本方案替换步骤3，并扩展为5步完整引导链：
  步骤1: 主城概览（不变）
  步骤2: 建造农田 + 招贤馆（扩展）
  步骤3: 首次招募 ← 替换原步骤3
  步骤4: 武将升级 ← 新增
  步骤5: 编队上阵 ← 新增
  步骤6: 首次战斗 ← 替换原步骤4，强化武将战斗体验
  步骤7: 查看资源（不变）
  步骤8: 科技研究（不变）
  延迟: 升星体验 ← 碎片积累后触发
```

### 3.3 首次招募保底设计

为保证引导体验，首次招募使用**固定池**：

```
首次招募池（仅引导第1次生效）：
  必出品质: 精良 (FINE) — 保证获得有价值的武将
  池内武将: 关羽 / 赵云 / 张飞（三选一随机）
  设计目的: 
    1. 让玩家首次获得有辨识度的三国名将
    2. 为后续"桃园结义"羁绊埋下伏笔（关羽/张飞是桃园结义成员）
    3. 精良品质不会太强也不会太弱，保持后续成长空间
```

### 3.4 视觉规范

| 元素 | PC端 | 手机端 |
|------|------|--------|
| 招贤馆高亮 | 金色边框 + 外扩8px + 脉冲动画 | 金色边框 + 外扩12px + 脉冲动画 |
| 招募动画 | 卡牌Y轴180°翻转（400ms） | 全屏卡牌翻转（400ms） |
| 武将信息卡 | 420×380px 居中浮层 | 底部弹出 Bottom Sheet |
| 完成奖励飘字 | 金色 `+500💰` 上浮渐隐 | 同PC端 |
| 遮罩 | 半透明黑色遮罩(0.6)，高亮区域透明 | 同PC端 |

### 3.5 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 招贤馆未建造 | 不应发生（步骤2已建造），若异常则引导先建造 |
| 网络断开 | 引导暂停，恢复后从当前子步骤继续 |
| 玩家已有武将 | 跳过1-2/1-3，直接展示"你已有武将，继续升级吧" |

### 4.3 升级资源保障

```
引导期间资源保障：
  铜钱: 自动补发至 1000（若不足）
  方式: 引导开始时检查铜钱余额，不足则静默补发（不弹提示）
  目的: 确保玩家能完成至少1次升级操作
```

### 4.4 属性变化展示设计

```
升级前 → 升级后 展示动画：
  武力  82 → 85 (+3) ↑    ← 绿色箭头 + 数字滚动
  智力  75 → 77 (+2) ↑
  统率  88 → 91 (+3) ↑
  政治  70 → 72 (+2) ↑
  战力  680 → 710 (+30) ↑  ← 金色大字突出

  动画: 每行依次亮起（间隔200ms），数值从旧值滚动到新值
```

| 异常场景 | 处理方式 |

### 5.3 编队引导辅助

```
编队引导辅助设计：
  1. 武将列表中，引导目标武将带金色边框+手指动画
```

### 6.3 首次战斗保底设计

```
首次战斗关卡设计（黄巾之乱·序章）：
  敌人配置: 3名普通品质敌兵（极弱）
  我方优势: 引导武将（精良品质）碾压
  战斗时长: ~20-30秒（加速结束）
  必定掉落: 
    - 铜钱 ×200~300
    - 武将碎片 ×3~5（随机武将）
    - 经验值 ×50
  设计目的: 确保玩家首次战斗轻松获胜，建立信心
```

### 6.4 战斗教学要点

|---------|---------|------|

| 伤害数字 | 飘字显示 | 每次攻击 |
| 战利品 | 结算界面列表 | 战斗结束时 |

### 6.5 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 战斗失败 | 不应发生（保底设计），若异常则降低敌人属性重试 |
| 跳过战斗 | 直接显示胜利结算，发放保底奖励 |
| 网络断开 | 战斗为本地计算，不受影响 |

## 7. 第5步：升星体验

### 7.1 基本信息

| 属性 | 值 |
|------|-----|
| 引导ID | `tutorial_hero_5` |
| 引导名称 | 「百尺竿头」 |
| 触发条件 | 累计获得 ≥10 个同一武将碎片（延迟触发） |
| 预计游戏时长 | ~20-30分钟（含碎片积累时间） |
| 预计引导时长 | 45-60秒 |
| 跳过选项 | ✅ 支持 |
| 完成奖励 | 铜钱×5000 + 突破石×10 |

### 7.2 触发条件详解

```
触发条件：累计获得 ≥10 个同一武将碎片

碎片来源（前20-30分钟预期积累）：
  - 首次招募保底：获得精良武将（关羽/赵云/张飞）
  - 首次战斗掉落：3~5碎片（随机武将，可能命中招募武将）
  - 引导奖励：10碎片（第4步奖励，随机分配给已有武将）
  - 重复招募转化：若再次抽到相同武将，转化10碎片（精良品质）

触发保障：
  - 第4步引导奖励的10个碎片，保证分配给第1步招募的武将
  - 确保玩家在完成第4步后不久即可触发第5步
```

### 7.3 引导流程

| 子步骤 | 操作类型 | 高亮目标 | 提示文案 | 完成条件 | 超时处理 |
|:------:|:-------:|---------|---------|---------|---------|

| 5-2 | 引导点击 | 武将详情中的"升星"按钮 | 「消耗碎片提升星级，获得属性飞跃！」 | 点击升星按钮 | 8秒后自动点击 |
| 5-3 | 自动展示 | 升星动画 | 星级提升光效（1星→2星），属性飞跃展示 | 动画完毕(~3秒) | — |
| 5-4 | 自动展示 | 属性变化 | 「全属性+15%！星级越高武将越强」 | 点击确认 | 5秒后自动确认 |
| 5-5 | 自动展示 | 羁绊变化（如有） | 「羁绊等级提升！效果增强」 | 2秒后自动继续 | — |
| 5-6 | 自动展示 | 奖励飘字 | 「引导奖励：铜钱×5000 突破石×10」 | 2秒后自动继续 | — |

### 7.4 升星动画设计

```
升星动画（1星→2星）：
  Phase 1 (0-500ms): 武将卡片缩小至80%，背景变暗
  Phase 2 (500-1500ms): 碎片从四周飞入卡片中心，金色粒子汇聚
  Phase 3 (1500-2500ms): 星星图标从1颗变为2颗，爆发光芒
  Phase 4 (2500-3000ms): 卡片恢复原大小，属性数字刷新（绿色↑箭头）

  音效: 碎片汇聚→能量蓄积→爆发→属性提升（4段式音效）
```

### 7.5 新手加速期开启

第5步完成后自动开启"新手加速期"，详见[第9节](#9-新手加速期设计)。

### 7.6 异常处理

|---------|---------|

## 8. 引导系统框架

### 8.1 引导管理器数据结构

```typescript
/** 引导步骤状态 */
interface TutorialStepState {
  stepId: string;           // 引导步骤ID
  status: 'locked' | 'active' | 'completed' | 'skipped';
  startedAt?: number;       // 开始时间戳
  completedAt?: number;     // 完成时间戳
  skippedAt?: number;       // 跳过时间戳
  currentSubStep: number;   // 当前子步骤索引
}

/** 引导全局状态 */
interface TutorialState {
  version: number;          // 引导版本号（用于版本更新后重置）
  steps: TutorialStepState[];
  isTutorialActive: boolean; // 是否有引导正在展示
  skipAll: boolean;         // 是否已跳过全部引导
  newbieBonusEndAt?: number; // 新手加速期结束时间
}

/** 引导配置 */
interface TutorialConfig {
  stepId: string;
  stepName: string;
  triggerCondition: string;  // 触发条件描述
  subSteps: TutorialSubStep[];
  rewards: Reward[];
  skipAllowed: boolean;
  autoProgressTimeout: number; // 超时自动推进（毫秒）
}
```

### 8.2 跳过机制设计

| 跳过方式 | 操作 | 效果 |
|---------|------|------|
| 单步跳过 | 右上角"跳过 >"按钮 | 跳过当前引导步骤，标记为"skipped" |
| 全部跳过 | 长按"跳过 >"按钮3秒 | 跳过所有未完成引导，标记所有为"skipped" |
| 设置中关闭 | 设置→引导→关闭新手引导 | 不再触发任何引导 |
| 设置中重播 | 设置→引导→重播新手引导 | 重置所有引导状态，从头开始 |

**跳过后补偿**：
- 跳过的引导步骤不发放完成奖励
- 但新手加速期仍会开启（完成第4步或跳过全部引导后自动开启）
- 设置中可随时重播已跳过的引导

### 8.3 超时自动推进

| 场景 | 超时时间 | 处理方式 |
|------|:-------:|---------|
| 等待玩家点击 | 8-15秒 | 目标元素闪烁提示 |
| 二次超时 | 再8秒 | 自动执行点击操作 |
| 展示类步骤 | 2-5秒 | 自动继续下一步 |
| 战斗步骤 | 不超时 | 战斗自动进行 |

### 8.4 引导持久化

```
存储位置: localStorage
存储键: 'three_kingdoms_tutorial_state'
存储内容: TutorialState JSON
更新时机: 每个子步骤完成时自动保存
读取时机: 游戏启动时检查，恢复到上次中断位置
```

## 9. 新手加速期设计

### 9.1 加速期概述

| 属性 | 值 |
|------|-----|
| 名称 | 新手加速期 |
| 持续时间 | 7天（从引导完成/跳过时开始计算） |
| 触发条件 | 完成第4步引导 或 跳过全部引导 |
| 目的 | 降低新手收集门槛，7天内获得5+武将 |

### 9.2 加速期福利

| 福利 | 具体效果 | 原价 | 加速价 | 节省 |
|------|---------|------|-------|------|
| **招募消耗减半** | 普通招募消耗 5→2 招贤令 | 5/次 | 2/次 | 60% |
| **首次十连8折** | 首次高级十连消耗 1000→800 招贤令 | 1000 | 800 | 20% |
| **每日签到翻倍** | 签到奖励×2 | — | ×2 | 100% |
| **关卡首通奖励翻倍** | 首通招贤令奖励×2 | 5~10 | 10~20 | 100% |
| **商店限购提升** | 招贤令限购 50→80个/日 | — | — | +60% |

### 9.3 加速期经济验证

| 指标 | 正常期（4h在线） | 加速期（4h在线） | 提升 |
|------|:---------------:|:---------------:|:----:|
| 日招贤令产出 | ~162 | ~240（含翻倍+限购提升） | +48% |
| 普通招募次数 | ~32次 | ~120次（消耗减半） | +275% |
| 高级招募次数 | ~1.6次 | ~2.4次 | +50% |
| 7天预期获得武将数 | ~5~8名 | ~12~18名 | +150% |

**目标达成验证**：
- 加速期7天，日均120次普通招贤 + 2.4次高级招贤
- 普通品质(60%概率)：~72次命中 → 全部集齐
- 精良品质(30%概率)：~36次命中 → 全部集齐
- 稀有品质(8%概率)：~10次命中 → 获得3~5名
- 史诗品质(2%概率)：~2.4次命中 → 获得1~2名
- **7天预期获得：12~18名武将** ✅ 远超5+目标

### 9.4 加速期结束过渡

```
加速期结束前24小时：
  - 顶部横幅提示"新手加速期即将结束"
  - 引导玩家使用剩余加速资源

加速期结束时：
  - 弹出"加速期已结束"提示
  - 展示加速期成果统计（获得武将数、战力提升、关卡进度）
  - 引导查看"下一步目标"（如羁绊激活、精英关卡等）
```
