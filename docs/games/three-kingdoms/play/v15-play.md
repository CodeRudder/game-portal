# v15.0 事件风云 — 玩游戏流程

> 核心循环: 挂机产出→随机事件触发→做出选择→获得奖励/应对危机→连锁事件推进→限时活动冲刺→NPC奇遇解锁
> **修订**: r4 (2026-04-21) — Gap分析r4补充：事件频率日上限、首次触发延迟、堆积超出策略、默认选项索引、连锁环节数对齐

## 1. 事件系统

### 1.1 随机遭遇事件池
- 操作: 挂机期间触发引擎每分钟tick → 按公式 P = 0.03 × (1 + 等级×0.02) × 时间衰减系数 计算触发概率 → 触发后从4子类型池(奇遇/天灾/人祸/商队)随机抽取事件模板 → 急报横幅弹出事件通知(6级优先级排队: 限时>天灾>剧情>随机>NPC>里程碑) → 最多缓存5条急报，3秒自动收起 → 玩家点击横幅打开事件详情弹窗(标题+描述+2~3选项+后果标注🟢/🔴/🟡) → 选择后计算资源变动+声望+好感度变化 → 概率后果按配置随机选择结果
- 频率与上限: 随机遭遇30~60min间隔(首次进入游戏10min后触发，日上限24次) | NPC事件60~120min间隔(首次20min后触发，日上限12次) | 天灾人祸120~200min间隔(首次30min后触发，日上限8次) | 限时机遇固定时段(日上限3次) → 超出日上限后当日不再触发该类型事件
- 验证: 20+事件模板覆盖4子类型(最低分配: 奇遇6+/天灾5+/人祸5+/商队4+)且权重分配合理 | 概率公式计算结果与预期一致(±5%误差) | 急报横幅按优先级排序显示不乱序 | 冷却机制正确(同类型60min/同事件4h/新手保护30min) | 连续2次负面后下一次必定正面或中性 | 选择后资源/声望/好感度数值精确变化 | 概率后果随机分布符合配置权重(1000次采样偏差≤5%) | **日上限验证**: 随机遭遇≤24次/天、NPC事件≤12次/天、天灾人祸≤8次/天、限时机遇≤3次/天 | **首次触发验证**: 随机遭遇进入游戏10min后首次触发、NPC事件20min后、天灾人祸30min后 | 各类型事件日上限精确执行(24/12/8/3次) | 首次触发延迟正确(10/20/30min)
- 预期: 随机事件为挂机体验注入不确定性和惊喜，冷却与保护机制避免负面体验过载，后果标注帮助玩家做出知情选择，日上限和首次延迟确保新手不被事件淹没

### 1.2 剧情事件与天灾人祸
- 操作: 剧情事件: 通关指定关卡后自动触发→一次性不可重复→推进主线故事(展示剧情CG+对话)→完成后解锁新功能或区域 → 天灾人祸: 概率 P = 0.015 × (1 + 关卡难度×0.01) 触发→消耗资源/考验防御→玩家选择应对策略(加固/疏散/硬抗)→结果影响资源产出和民心 → 离线期间天灾人祸按防御值自动减免损失
- 验证: 剧情事件仅在指定关卡通关后触发且不可重复 | 天灾概率与关卡难度正相关 | 天灾首次触发在进入游戏30min后，日上限≤8次 | 离线自动处理结果不超过手动最优选择的损失 | 剧情事件离线时暂停等待玩家上线 | 天灾自动损失上限为当前资源总量15%
- 预期: 剧情事件推进叙事节奏，天灾人祸增加策略深度，离线自动处理保障放置体验不被惩罚

### 1.3 事件选项与后果
- 操作: 事件弹窗展示2~3个选项按钮 → 每个选项标注后果颜色(🟢正面/🔴负面/🟡中性) → 部分选项附带概率后果(如"70%获得资源/30%遭遇伏击") → 玩家选择后: 资源即时增减+声望值变化+NPC好感度变化 → 结果弹窗展示具体变动数值和获得物品 → 确认后关闭事件，变动写入玩家数据
- 默认选项索引: 每个事件数据结构中包含`默认选项索引`字段，用于离线自动处理时确定选择策略 → 正面事件默认选择"保守"选项(索引由配置指定) → 负面事件默认选择"最小损失"选项 → 中性事件默认选择第一个选项 → 默认选项在事件配置时指定，非硬编码
- 验证: 后果标注颜色与实际结果一致 | 概率后果随机分布符合配置权重 | 资源变动精确(扣除不超过持有量) | 声望/好感度变化正确触发对应系统判定 | 默认选项索引正确用于离线自动处理 | 正面事件默认选项确实为保守方案(50%基础奖励)
- 预期: 选项系统赋予玩家决策权，后果标注降低选择焦虑，概率后果增加紧张感和重玩价值，默认选项确保离线处理有明确策略

### 1.4 事件选项数值缩放规则
- 操作: 资源奖励 = 基础值 × (1 + 主城等级 × 0.1) → 资源损失 = 基础值 × max(0.3, 1 - 防御建筑总等级 / (防御建筑总等级 + 50)) → 示例: 主城等级10时奖励倍率2.0x → 防御等级30时损失减免37.5% → 天灾自动损失上限 = 当前资源总量的15%
- 验证: 奖励缩放公式与主城等级正确联动 | 损失减免与防御建筑等级正确联动 | 天灾自动损失不超过资源总量15% | 防御减免率 = 防御建筑总等级 / (防御建筑总等级 + 50)
- 预期: 数值缩放确保事件奖励/惩罚随玩家进度合理增长，防御投资有明确回报

### 1.5 里程碑事件
- 操作: 玩家达成成就时自动触发里程碑事件(如"首次占领城池""武将满级"等) → 里程碑事件优先级为6级(最低) → 提供正反馈强化(资源奖励+成就展示) → 一次性不可重复 → 离线期间暂停等待玩家上线触发
- 验证: 里程碑事件仅在对应成就首次达成时触发 | 离线期间不触发不推进，等玩家在线时触发 | 奖励内容与成就难度匹配
- 预期: 里程碑事件为玩家关键节点提供仪式感反馈，强化成就感和继续前进的动力

### 1.6 事件日志查询
- 操作: 主界面点击"事件日志"入口(或从急报横幅区域展开) → 事件日志面板展示最近100条已处理事件记录 → 每条记录包含: 事件标题/类型标签(颜色编码)/触发时间/玩家选择/结果摘要 → 连锁事件链进度展示(链ID+当前节点+历史选择) → 点击单条记录可展开查看详细后果(资源变动/声望变化/好感度变化) → 连锁事件链可查看链路径可视化(已完成节点+当前节点+分支预览) → 筛选功能: 按类型(随机/剧情/NPC/天灾/限时/里程碑)/按时间(今日/近7天/全部)/按结果(正面/负面/中性)
- 验证: 日志记录上限100条，超出后FIFO淘汰最旧记录 | 连锁事件链进度与链追踪面板数据一致 | 筛选功能正确过滤 | 详细后果数值与实际发生一致
- 预期: 事件日志帮助玩家回顾决策历史，连锁链进度可视化辅助理解因果，筛选功能快速定位特定类型事件
- **对应Plan功能点**: 数据流设计→事件日志子系统

## 2. 链式事件

### 2.1 连锁事件引擎
- 操作: 完成某个事件选项后触发后续事件(链ID关联) → 连锁事件追踪面板展示链路径可视化(当前节点+历史选择+分支预览) → 每个节点2~3个分支选项 → 分支条件判定确定后续节点(如选择"帮助"→善线/选择"拒绝"→恶线) → 链状态持久化存储(断线/离线不丢失进度) → 链完成时发放最终奖励+链状态归档 → 链长度通常2~4环(大部分为3环)，设计上限5个节点
- 验证: 链ID追踪准确，节点跳转无死链 | 分支条件判定与玩家选择一致 | 链状态持久化后重登可继续 | 链完成奖励按配置正确发放 | 节点数通常2~4环，不超过5个 | 大部分链为3环结构(符合PRD EVT-4设计)
- 预期: 连锁事件提供"蝴蝶效应"体验，选择有长期影响，追踪面板帮助玩家理解因果，链状态持久化保障连续性

### 2.2 连锁事件数据结构与分支
- 操作: 事件数据结构包含: 事件ID/链ID/后续事件ID列表/分支条件/链状态字段 → 玩家选择后引擎读取后续事件ID → 匹配分支条件(资源阈值/声望等级/好感度/前置选择) → 确定唯一后续节点并触发 → 无匹配后续时链自然结束 → 链中途可暂停(玩家不立即响应)，下次登录继续
- 验证: 数据结构字段完整无遗漏 | 分支条件匹配逻辑精确 | 无匹配时链正常结束不报错 | 暂停后恢复时上下文完整
- 预期: 结构化数据支撑复杂分支叙事，条件匹配增加事件与游戏状态的关联性，暂停机制尊重玩家节奏

### 2.3 连锁事件触发规则与限制
- 操作: 触发条件: 玩家在事件中选择带后续事件ID的选项 + 后续事件前置条件全部满足 → 触发延迟: 选择后10~30分钟随机触发下一环 → 不触发条件: 选择无后续事件ID的选项则链中断 / 后续条件不满足则链中断无提示 → 玩家离线则链暂停，上线后继续计时
- 验证: 触发延迟在10~30min范围内随机 | 离线暂停后上线恢复计时正确 | 无后续事件ID时链正常终止不报错 | 前置条件不满足时链静默中断
- 预期: 随机延迟增加事件链的自然感，离线暂停保障玩家不丢失链进度

### 2.4 连锁事件超时与并发规则
- 操作: 单环等待超过24小时未响应 → 链自动终止 → 按已完成环节数给予30%阶段奖励作为中断补偿 → 最多同时进行1条事件链(剧情链不受此限制，可与普通链并行) → 同一事件链完成后7天冷却才可再次触发
- 验证: 24h超时后链正确终止 | 中断补偿按已完成环节数×30%计算 | 普通链+剧情链可并行不冲突 | 7天冷却期内同链不重复触发
- 预期: 超时机制防止链永久挂起，中断补偿降低惩罚感，并发限制避免信息过载，冷却保持新鲜感

### 2.5 剧情事件链
- 操作: 剧情事件链为特殊连锁事件，与主线章节绑定 → 例: 第一章「黄巾之乱」剧情链: 1-1黄巾初现→1-2颍川之战→1-3广宗之战→1-4张角决战 → 剧情链特性: 不受链并发限制(可与普通链并行) / 无环间延迟(连续触发) / 必须全部完成才能推进到下一章 / 选择影响后续章节的NPC关系和可招募武将
- 验证: 剧情链与普通链可同时进行互不干扰 | 剧情链环节间无延迟连续触发 | 未完成剧情链时下一章主线不可推进 | 选择结果正确影响后续章节NPC关系
- 预期: 剧情链将叙事与玩法深度结合，选择影响后续体验增加重玩价值，连续触发保持叙事节奏

## 3. 活动系统

### 3.1 限时机遇事件
- 操作: 每日固定时段(10:00/14:00/20:00)触发限时机遇 → 触发后急报横幅最高优先级通知(红色边框+脉动) → 事件详情弹窗显示倒计时(30~60min窗口) → 玩家在窗口内响应获得丰厚奖励(稀有资源/武将碎片/元宝) → 窗口关闭后事件消失不可补领 → 每日3次机会，错过仅损失单次不影响核心进度
- 验证: 触发时段精确(服务器时间±1min) | 倒计时准确且窗口关闭后事件确实消失 | 奖励丰厚程度显著高于普通事件 | 离线期间限时机遇错过不补偿
- 预期: 限时机遇创造紧迫感，鼓励玩家定时回归，丰厚奖励补偿时间投入，错过惩罚温和不挫败

### 3.2 限时活动完整流程
- 操作: 限时活动开始(3~7天周期) → 活动入口出现在主界面(横幅通知) → 进入活动页面展示: 任务列表+里程碑进度+代币余额+活动倒计时 → 完成每日任务/挑战任务/累积任务获代币+积分 → 积分推进里程碑(按顺序线性解锁不可跳过) → 里程碑达成后需手动领取奖励 → 代币兑换商店(七阶体系，详见§3.5) → 排行榜(详见§3.6) → 离线进度: 限时活动30%效率自动累积(每日上限8h) → 活动到期结算(详见§3.8)
- 验证: 活动周期起止时间精确 | 里程碑按顺序解锁不可跳过 | 离线进度按效率系数累积无溢出 | 代币余额实时更新
- 预期: 限时活动提供周期性内容追求驱动日常活跃，完整流程从开始到结算环环相扣

### 3.3 节日活动框架
- 操作: 节日活动7~14天周期 → 进入活动页面展示: 特殊玩法入口+限定剧情+独占奖励预览+代币余额 → 特殊玩法(春节放烟花/端午龙舟/中秋猜灯谜等) → 限定剧情事件(与主线独立，可触发连锁事件链) → 独占奖励(限定武将/皮肤/建筑外观) → 代币兑换商店(七阶体系) → 离线50%效率自动累积(每日上限12h) → 节日活动代币类型为"节庆灯笼"(独立不可跨活动使用) → 活动到期结算
- 验证: 节日活动特殊玩法可正常交互 | 限定剧情事件不可重复 | 独占奖励仅在节日活动期间可获得 | 离线效率50%正确累积
- 预期: 节日活动丰富文化体验，特殊玩法提供新鲜感，独占奖励驱动参与

### 3.4 赛季活动
- 操作: 赛季活动14~28天长周期 → 里程碑系统: 按顺序线性解锁(不可跳过)，达成后需手动领取奖励 → 七阶商店: 青铜铺(活动开始即解锁)→白银肆(累计消耗500代币)→黄金坊(累计消耗2000代币)→翡翠阁(里程碑达成3个)→赤金殿(排行前50%)→龙纹宝库(排行前10%)→天命秘藏(最终里程碑) → 排行榜: 全服/魏蜀吴分组，每5min刷新，活动结束后1h内结算 → 代币经济: 战斗积分+里程碑+任务+排行+离线累积+每日活跃+元宝购买(1代币/元宝) → 离线效率50%，声望等级加成: 效率系数 = 50% × (1 + 声望等级 × 0.03)
- 验证: 里程碑按顺序解锁不可跳过 | 七阶商店解锁条件与PRD一致 | 排行榜分组和刷新频率正确 | 代币获取途径完整(7种) | 离线效率系数含声望等级加成 | 最终里程碑需手动确认领取
- 预期: 赛季活动提供长线追求目标，七阶商店创造分层进度感，声望加成奖励系统间联动

### 3.5 代币兑换商店（七阶体系）独立流程
- 操作: 进入活动页→点击"兑换商店"Tab → 商店页面展示: 七阶Tab栏(青铜铺/白银肆/黄金坊/翡翠阁/赤金殿/龙纹宝库/天命秘藏) + 代币余额(右上角) + 已解锁/未解锁阶层标识 → 未解锁阶层显示灰色+解锁条件文字(如"累计消耗500代币解锁") → 已解锁阶层显示商品网格(商品图标+名称+价格+限购标记) → 点击商品→弹出兑换确认(商品详情+价格+剩余限购次数) → 确认兑换→扣除代币→物品入背包→限购次数-1 → 已售罄商品显示灰色"已购完"标记 → 青铜铺/白银肆每日0点刷新限购次数 → 黄金坊及以上阶层限购次数为活动期间总量不刷新
- 验证: 七阶Tab按阶层顺序排列 | 未解锁阶层不可点击且显示解锁条件 | 代币余额实时更新(兑换后立即刷新) | 限购次数精确(每日刷新/活动期间总量) | 代币不足时兑换按钮灰色不可点击 | 兑换成功后物品正确入背包
- 预期: 七阶商店创造分层目标感，阶层解锁条件明确，限购管理防止囤积

## 9. 异常处理与边界场景

### 9.1 事件系统异常处理
- 操作: **事件触发时服务器异常**: 事件触发失败→重试3次→仍失败则跳过本次触发不影响后续 → **选项选择时网络中断**: 选择请求发送但未收到响应→重登后检查事件状态→未处理则重新展示事件弹窗→已处理则展示结果 → **连锁事件链数据损坏**: 检测到链状态异常→按已完成环节数给予30%补偿→链自动终止→记录错误日志 → **事件池为空**: 所有事件模板处于冷却期→触发引擎跳过本次tick→不触发空事件
- 验证: 服务器异常时事件不丢失不重复 | 网络中断后重登事件状态正确恢复 | 链数据损坏时补偿正确发放 | 事件池为空时不报错不崩溃
- 预期: 异常处理确保事件系统在各种边界情况下稳定运行，不丢失玩家数据

### 9.2 活动系统异常处理
- 操作: **活动商店兑换时代币不足**: 点击兑换→弹窗提示"代币不足"→显示当前余额和所需数量→引导获取途径 → **活动商店限购已满**: 商品显示"已售罄"灰显状态→不可点击 → **排行榜结算时排名相同**: 按积分达成时间先后排序(先达成者排名靠前) → **多个活动同时到期**: 按活动类型优先级结算(赛季>节日>限时>联盟)→逐一生成结算报告 → **里程碑手动领取时背包已满**: 物品转存邮箱待领取→弹窗提示"背包已满，奖励已发送至邮箱"
- 验证: 代币不足时兑换被正确拦截 | 限购已满商品不可点击 | 同分排名按时间排序 | 多活动同时到期按优先级结算不冲突 | 背包已满时奖励转存邮箱
- 预期: 活动系统异常处理覆盖常见边界场景，引导玩家解决问题而非阻断流程

### 9.3 离线回归异常处理
- 操作: **离线超过72h**: 堆积事件超时自动以"保守"方案处理→仅保留最近72h内事件→更早事件按保守方案自动结算 → **离线回归面板展示中断**: 关闭回归面板后可从"邮件"入口重新查看离线汇总 → **一键领取时部分奖励领取失败**: 成功的奖励正常发放→失败的奖励转存邮箱→弹窗提示部分奖励已转存邮箱
- 验证: 超72h离线事件正确按保守方案处理 | 回归面板关闭后可重新查看 | 部分奖励失败时不影响已成功部分
- 预期: 离线回归异常处理确保长时间离线玩家也能正常回归，不丢失关键奖励
