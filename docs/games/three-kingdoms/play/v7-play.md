# v7.0 草木皆兵 — 装备域 Play 文档 (Round 2)

> **引擎域**: engine/equipment/ | **核心功能**: 装备获取/穿戴/锻造/强化/套装/分解

---

## 流程1: 获取装备
- 通关关卡(normal/elite/boss) → 按权重掉落装备 → 自动入背包
- 开启装备箱(equipment_box) → 按权重生成 → 入背包
- 事件奖励(event) → 按权重生成 → 入背包
- **验证**: 掉落权重 CAMPAIGN_DROP_WEIGHTS 正确、背包容量不超限、uid唯一

## 流程2: 穿戴/脱下
- 选择武将 → 选择装备 → equip() → 装备绑定武将slot
- 选择武将 → 选择已穿装备 → unequip() → 装备回背包
- 装备已占位 → 自动替换：旧装备回背包，新装备穿上
- **验证**: HeroEquipSlots 6槽位(weapon/armor/helmet/boots/accessory/mount)正确

## 流程3: 锻造升级
- 3件同品质(白) → 基础炼制 basic → 绿/蓝/紫(按概率)
- 5件同品质 → 高级炼制 advanced → 更高概率出紫/金
- 3件+指定slot → 定向炼制 targeted → 指定部位
- 保底: 10次未紫必紫, 30次未金必金 (ForgePityManager)
- **验证**: 消耗正确、保底计数递增/重置、输出品质概率合规

## 流程4: 强化系统
- 消耗铜钱+强化石 → enhance() → 成功率曲线(15级上限)
- 失败: 安全等级以上降级, 保护符防降级
- autoEnhance: 循环强化到目标等级
- transferEnhance: 源装备强化等级→目标(有损耗)
- **验证**: 成功率曲线 ENHANCE_CONFIG、降级规则、保护符消耗

## 流程5: 套装效果
- 穿戴同setId装备 → 统计件数 → 2件/4件激活套装效果
- 7套套装定义(EQUIPMENT_SETS) → 属性加成聚合
- **验证**: SetBonusTier阈值判定、ActiveSetBonus属性汇总

## 流程6: 分解回收
- 选择装备 → getDecomposePreview() → 预览奖励
- 确认分解 → decompose() → 铜钱+强化石到账, 装备销毁
- 批量分解 → batchDecompose() → 按品质筛选批量处理
- **验证**: 奖励=基础×(1+强化等级×0.1)、图鉴更新
