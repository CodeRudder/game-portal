## 4. Gap 分析（核心输出）

### 4.1 Plan 中有但 Play 缺失的流程

虽然 Play 覆盖了所有 25 个 Plan 功能点的**基本流程**，但以下流程细节存在缺失：

| Gap编号 | 缺失流程 | 涉及功能点 | 严重程度 | 说明 |
|---------|----------|-----------|:--------:|------|
| GAP-P-01 | **资源交易流程** | F1.06(间接) | 🔴 高 | PRD RES-5 定义了资源交易系统（汇率/手续费/每日上限），Plan未列入，Play完全缺失 |
| GAP-P-02 | **资源保护触发流程** | F1.09(间接) | 🟡 中 | PRD RES-6 定义了5种保护机制（天命保护/新手保护/铜钱安全线），Play仅验证了最低粮草保留 |
| GAP-P-03 | **建筑加速/取消流程** | F1.18 | 🟡 中 | PRD BLD-4 定义了铜钱加速/天命加速/元宝秒完成，Play BLD-FLOW-5仅验证队列满/释放，缺少加速和取消 |
| GAP-P-04 | **建筑自动升级开关** | F1.18 | 🟢 低 | PRD BLD-4 提到"可开启开关，系统自动将可升级建筑加入队列"，Play未覆盖 |
| GAP-P-05 | **一键收取流程** | — | 🟡 中 | PRD BLD-5 定义了一键收取+收取动画，Plan和Play均未涉及 |
| GAP-P-06 | **离线建造进度流程** | F1.25(间接) | 🟡 中 | PRD BLD-4 定义了离线建造进度规则（离线效率×1.2），Play仅验证离线资源收益，未验证离线建筑升级 |
| GAP-P-07 | **日历系统对玩法的影响** | F1.05 | 🟢 低 | Plan列为P1，Play验证了显示但未验证季节/天气对产出或玩法的影响 |

### 6.4 是否需要新增 Plan 版本

**结论：不需要新增独立版本，但建议对 v1.0 Plan 进行修订（v1.1）。**

理由：
1. SET PRD 范围过大（17个功能点），但大部分属于 v17~v19 范围，v1.0 只需实现基础设置

4. 新功能从 v21 开始编号的规则仅适用于全新版本，v1.0 的补充应作为修订

**建议修订方式**：
- 更新 `plans/v1.0-基业初立.md`，增加模块 E（设置/体验增强），补充 F1.26~F1.32
- 修订 Tab 数量从 4→7（与PRD对齐）
- 修订资源种类（明确v1.0范围）
- 更新 `play/v1-play.md`，补充第6.1节列出的缺失流程
