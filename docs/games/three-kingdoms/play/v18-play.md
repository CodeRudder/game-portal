# v18.0 新手引导 — 玩游戏流程

> 核心循环: 引导状态机驱动6步核心+6步扩展引导流程→步骤高亮+遮罩聚焦引导玩家操作→8段剧情事件穿插增强沉浸感→教学关卡实战验证→提示系统辅助自由探索→跳过/重玩机制兼顾老玩家与回顾需求

## 0. 首次启动流程（SET → TUT 衔接）

### 0.1 首次启动检测与初始化
- 操作: 全新安装首次启动 → **语言检测**（自动识别系统语言，支持简体中文/繁體中文/English/日本語，无法识别时默认简体中文） → **画质检测**（检测CPU核心+内存：≥8核且≥8GB→高画质，≥4核且≥4GB→中画质，其他→低画质，检测结果标注「★ 推荐」朱砂印章，玩家可手动覆盖） → **权限申请**（弹出数据授权面板：存储/网络/通知/定位，逐项申请） → 检测无引导记录 → 自动触发E1桃园结义剧情 → 进入6步核心引导
- 验证: 语言检测跟随系统语言准确 | 画质推荐与设备性能匹配 | 权限面板逐项弹出不遗漏 | 首次启动BGM延迟3秒播放先展示画面 | 低电量(<20%)自动降低BGM音量50%
- 预期: 新玩家首次启动3步初始化(语言+画质+权限)在30秒内完成，无感知负担

### 0.2 老用户回归流程
- 操作: 非首次启动（检测到引导记录或账号数据） → 跳过初始化流程 → 直接进入自由游戏 → 按回归弹窗优先级队列展示：①离线收益弹窗(离线>5分钟，不可跳过) → ②版本更新公告(可跳过) → ③限时活动提醒(可跳过) → ④回归欢迎面板(离线>1小时，可跳过) → ⑤军师建议更新(不可跳过)
- 验证: 有引导记录的用户不触发任何引导 | 回归弹窗按优先级顺序展示 | 离线收益按效率阶梯正确计算(0~2h:100%, 2~8h:80%, 8~24h:60%, 24~48h:40%, 48~72h:20%, >72h:15%封顶)
- 预期: 老用户回归无引导打扰，快速回到游戏核心体验

## 1. 新手引导系统

### 1.1 引导状态机与6步核心引导
- 操作: 首次启动游戏 → 检测无引导记录 → 状态机进入「核心引导中」 → 步骤1-1主城概览(高亮主城建筑+气泡说明"这是您的主城") → 步骤1-2建造农田(高亮空地→点击→确认建造) → 步骤1-3招募武将(高亮酒馆→进入→点击招募) → 步骤1-4首次出征(高亮战役入口→选择关卡→完成战斗) → 步骤1-5查看资源(高亮资源栏→逐项说明粮草/铜钱/元宝) → 步骤1-6科技研究(高亮书院→选择科技→开始研究) → 步骤6完成发放「初出茅庐」礼包 → 状态机切换至「自由探索过渡」
- 状态机完整转换路径:
  ```
  [未开始] ──首次进入──→ [核心引导中] ──步骤6完成──→ [自由探索过渡] ──过渡完成──→ [自由游戏]
                                │                                                │
                                └──加速跳过──→ [自由探索过渡]                       └──条件触发──→ [Mini-tutorial]
  ```
- 验证: 首次用户自动触发引导 | 老用户(有引导记录)直接进入自由游戏 | 6步顺序严格线性不可跳步 | 每步完成条件明确(如农田建造完成才算步骤通过) | 礼包发放时机准确 | 加速跳过路径正确进入自由探索过渡
- 预期: 新玩家在5分钟内完成核心引导，理解游戏基本循环(建造→招募→战斗→升级)

  | 参数 | 保护期内 | 保护期后 |
  |------|---------|---------|
  | 随机事件类型 | 仅正面(奇遇/商队) | 全部类型 |
  | 建筑升级消耗 | ×0.5 | ×1.0 |
  | 科技研究消耗 | ×0.5 | ×1.0 |
  | 招募消耗 | ×0.5 | ×1.0 |
  | 敌方HP | ×0.7 | ×1.0 |
  | 敌方攻击力 | ×0.7 | ×1.0 |
  | 计时方式 | 仅在线时间 | — |
- 新手保护倒计时UI:
  - **位置**: 屏幕右上角，资源栏下方，固定悬浮显示
  - **样式**: 水墨卷轴造型，内含倒计时数字(分:秒)，底色为淡金色(保护期内)→灰色(保护期结束)
  - **交互**: 点击倒计时展开保护期详情面板（显示各项保护参数：资源消耗×0.5、敌方HP×0.7等）

  - **过渡动画**: 倒计时最后10秒数字变为红色+脉冲动画 → 归零时水墨晕散效果消失
- 验证: 保护期内无负面事件触发 | 资源消耗确实减半 | 战斗难度明显降低(敌方HP和攻击力均为70%) | 30分钟在线计时准确 | 保护期结束后参数恢复正常 | 结束提示正确推送 | 倒计时UI正确显示且不遮挡核心操作区域 | Toast通知3秒消失 | 点击倒计时展开详情面板参数正确
- 预期: 新手保护期让新玩家在安全环境中熟悉游戏，30分钟后自然过渡到正常难度

## 8. 异常处理

### 8.1 引导异常场景处理
| 异常场景 | 处理方式 | 恢复策略 |
|---------|---------|---------|
| 网络断开 | 引导暂停+显示"网络连接中断"提示+当前步骤进度已保存至localStorage | 网络恢复后自动从断点继续，不重复已完成步骤 |
| 强制关闭应用 | 下次启动检测到未完成引导→自动进入断点恢复流程 | 从最后一个已完成步骤的下一步开始 |
| 步骤卡住（玩家>60s未操作） | 显示"快速完成"按钮（不可跳过步骤除外） | 点击快速完成跳过当前子步骤 |
| 引导目标元素未加载 | 等待最多5秒+重试检测3次 | 3次重试失败→跳过当前子步骤+记录错误日志+军师提示"如有疑问可在设置中回顾引导" |
| 剧情播放中崩溃 | 剧情进度实时保存（每段对话完成时保存） | 重启后从最后完成的对话继续播放 |
| 跨设备同步冲突 | 取completed_steps并集+completed_events并集（取最大进度） | 以进度最多的为准，不回退任何已完成步骤 |
| 新手保护期内断线 | 保护期计时仅计算在线时间 | 断线时间不计入30分钟保护期 |
| 引导遮罩渲染异常 | 遮罩组件内置兜底：3秒未渲染完成→降级为简单边框高亮 | 确保引导流程不因遮罩问题阻塞 |

### 8.2 教学关卡异常处理
| 异常场景 | 处理方式 |
|---------|---------|
| 教学关卡战斗失败 | 不可能失败（敌方HP降至30%+我方属性加成）→如因bug失败→自动判定胜利+发放奖励+记录错误日志 |

|------|---------|

### 9.5 引导中断恢复状态机异常路径

- **触发条件**: 引导过程中发生异常（网络断开/强制关闭/元素未加载/保护期计时异常/遮罩渲染异常）
- **操作步骤（按异常类型分述）**:

  **路径1: 网络断开恢复**
  1. 引导进行中 → 检测到网络断开 → 引导状态机进入「引导暂停」状态
  2. 显示"网络连接中断"提示（半透明浮层，不遮挡引导内容）
  3. 当前步骤进度实时保存至localStorage（每完成一个子步骤即保存）
  4. 网络恢复 → 自动从断点继续 → 状态机回到原状态（如「核心引导中」）
  5. 新手保护期计时在断线期间暂停，恢复后继续

  **路径2: 强制关闭恢复**
  1. 引导进行中 → 应用被强制关闭
  2. 下次启动 → FirstLaunchDetector检测到未完成引导记录（completed_steps不完整）
  3. 自动进入断点恢复流程 → 从最后一个已完成步骤的下一步开始
  4. 跳过首次启动初始化（语言/画质/权限已完成过）
  5. 不重播已完成的剧情事件（completed_events记录）

  **路径3: 元素未加载恢复**
  1. 引导步骤指向目标元素 → 元素未在DOM中渲染
  2. 等待5秒 → 检测元素是否出现 → 未出现则重试（最多3次，共15秒）
  3. 3次重试全部失败 → 自动跳过当前子步骤 → 记录错误日志（step_id + element_id + timestamp）
  4. 军师提示"如有疑问可在设置中回顾引导"（3秒后自动消失）
  5. 继续下一步骤 → 不阻塞引导流程

  **路径4: 保护期计时异常恢复**
  1. 检测到本地计时与服务端时间偏差>60秒 → 请求服务端校准
  2. 校准期间保护期计时暂停 → 校准完成后以服务端时间为准继续计时
  3. 若校准失败 → 继续使用本地计时 + 标记为"待校准" + 下次网络请求时重试

  **路径5: 遮罩渲染异常恢复**
  1. 引导遮罩组件启动 → 3秒内未完成渲染（检测遮罩DOM节点是否存在+透明度是否正确）
  2. 自动降级为简单边框高亮（2px金色边框 #FFD700 围绕目标元素）
  3. 引导流程不因遮罩问题阻塞 → 继续执行步骤
  4. 记录降级日志 + 后续步骤尝试恢复完整遮罩
