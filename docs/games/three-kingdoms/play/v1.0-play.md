# v1.0 基业初立 — 玩游戏流程

> Round 2 进化迭代测试流程

## 一、主界面导航 (NAV)

### NAV-1: 主界面功能定位
1. 打开游戏页面 `http://localhost:5173/idle/three-kingdoms`
2. 等待3秒加载完成
3. 验证：资源栏在顶部、Tab在底部/侧边、中央场景区显示

### NAV-2: 顶部资源栏
1. 查看资源栏显示4种资源图标+数值
2. 等待2秒，验证资源数值持续增长
3. 验证产出速率显示（如 +X/秒）

### NAV-3: Tab切换
1. 点击"建筑"Tab → 中央区域切换为建筑面板
2. 点击"武将"Tab → 中央区域切换为武将面板
3. 点击"科技"Tab → 中央区域切换为科技面板
4. 点击"关卡"Tab → 中央区域切换为关卡面板
5. 验证：每次切换无闪烁，状态保持

### NAV-4: 中央场景区
1. 默认显示建筑俯瞰场景
2. Tab切换后中央区域内容变化

### NAV-5: 游戏日历
1. 查看日历区域显示年号/季节/天气
2. 验证季节效果文字描述

## 二、资源系统 (RES)

### RES-1: 4种核心资源
1. 查看资源栏：粮草/铜钱/兵力/天命
2. 验证每种资源有图标+数值+产出速率

### RES-2: 资源产出
1. 记录当前资源值
2. 等待5秒
3. 验证资源增长 = 产出速率 × 5秒（允许±10%误差）

### RES-3: 资源消耗
1. 点击建筑Tab
2. 选择一个建筑
3. 点击"升级"按钮
4. 验证：资源扣除正确，Toast提示成功

### RES-4: 资源存储与上限
1. 使用GameEventSimulator添加大量资源
2. 验证：资源不超过上限
3. 验证：容量进度条显示

### RES-5: 天命资源
1. 验证天命资源独立显示
2. 验证天命获取途径（建筑产出/任务奖励）

## 三、建筑系统 (BLD)

### BLD-1: 8座建筑总览
1. 进入建筑Tab
2. 验证显示8座建筑
3. 验证每座建筑有图标+名称+等级+状态

### BLD-2: 建筑升级
1. 选择"农田"建筑
2. 查看升级费用（粮草+铜钱）
3. 确认资源足够
4. 点击升级
5. 验证：资源扣除、等级+1、产出增加

### BLD-3: 建筑产出
1. 升级一个资源建筑
2. 查看资源栏产出速率变化
3. 验证：产出 = 基础产出 × (1 + 建筑等级加成)

### BLD-4: 建筑联动
1. 查看主城等级
2. 验证：其他建筑等级上限 ≤ 主城等级
3. 尝试升级超过主城等级的建筑 → 应提示"需要先升级主城"

### BLD-5: 建筑队列
1. 同时升级2个建筑
2. 验证：队列管理正常
3. 验证：升级完成通知

## 四、全局规范 (SPEC)

### SPEC-1: 自动保存
1. 升级一个建筑
2. 刷新页面
3. 验证：建筑等级保持不变

### SPEC-2: 离线收益
1. 使用GameEventSimulator.fastForwardHour()
2. 验证：离线收益计算正确
3. 验证：离线收益弹窗显示

### SPEC-3: Toast提示
1. 执行建筑升级
2. 验证：Toast显示"建筑升级成功"
3. 验证：Toast自动消失

### SPEC-4: 面板/弹窗规范
1. 打开建筑升级弹窗
2. 验证：弹窗有半透明遮罩
3. 验证：ESC可关闭弹窗
4. 验证：点击遮罩可关闭弹窗
