- **PRD补充**: 每阶段5个关卡，共30个关卡；各阶段推荐战力范围如下：

### 1.1a 关卡地图UI详细验收 🆕 [P1]

> 关卡地图的UI交互细节验收，确保节点操作、滚动和切换体验流畅

- **节点点击响应**:
  - 操作: 单击关卡节点（PC端鼠标点击 / 手机端触摸）
  - 验证: 点击后 200ms 内弹出关卡详情卡片（PC端右侧侧边栏 / 手机端Bottom Sheet）
  - 预期: 
    - 可挑战节点: 卡片显示关卡名称、敌方首领、推荐战力、「出征」按钮（亮色可点击）
    - 已通关节点: 卡片显示星级评定、上次通关时间、「出征」+「扫荡」（三星时）按钮
    - 未解锁节点: 卡片显示关卡名称（灰色）+ 锁图标 + 解锁条件提示（如"通关前一关"）
    - 点击空白区域 / 按ESC键关闭卡片
  - 异常处理: 快速连续点击（<300ms）仅响应首次，防抖处理避免重复弹窗

- **滚动边界**:
  - 操作: 在关卡地图区域执行滚动操作（PC端鼠标滚轮/拖拽滚动条 / 手机端上下或左右滑动）
  - 验证: 滚动到达章节边界时正确停止，无橡皮筋效果（PC端）或弹性回弹（手机端原生行为）
  - 预期:
    - PC端横向卷轴: 滚动到章节首/末关卡时停止，显示渐变遮罩提示"已到边界"
    - 手机端纵向滚动: 滚动到章节首/末关卡时停止，边界处有微弱弹性效果（≤20px回弹）
    - 滚动惯性: 松手后自然减速停止，不会突然停止或过度滑动
    - 滚动到未解锁区域时，未解锁节点渐变为更淡的灰色，视觉提示"未探索区域"
  - 异常处理: 触控设备滚动与关卡节点点击冲突时，滑动距离 >10px 判定为滚动（不触发节点点击）

- **章节切换动画**:
  - 操作: 点击章节Tab（PC端顶部 / 手机端顶部Tab栏）或左右滑动切换章节（手机端）
  - 验证: 章节切换有过渡动画，新章节关卡数据正确加载
  - 预期:
    - PC端: 当前章节地图向左/右滑出（0.3s ease-out），新章节地图从对侧滑入（0.3s ease-in）
    - 手机端: 整体横向滑动切换，跟随手指实时偏移，松手后自动吸附到最近章节（snap效果）
    - 切换过程中关卡节点依次淡入（每节点间隔50ms），营造"地图展开"感
    - 章节名称在顶部以卷轴展开动画显示（0.5s）
    - 已解锁章节正常显示，未解锁章节显示为灰色剪影 + "通关前一章解锁"
  - 异常处理: 
    - 切换动画中点击节点无效（动画期间屏蔽交互，0.6s后恢复）
    - 快速连续切换章节时取消前一次动画，仅执行最后一次切换

## 2. 战前布阵（模块B）

### 2.1 进入布阵界面
- 操作: 点击「可挑战」关卡节点→弹出战前布阵面板
- 验证: PC端中央弹出卷轴面板（620×520px），手机端Bottom Sheet弹出
- 预期: 显示前排3+后排3共6个阵位，当前编队武将已填入
- **PRD补充**: 前排承受更多伤害，后排相对安全；部分武将技能有位置要求（如"前排攻击+15%"）

### 2.2 一键布阵
- 操作: 点击「一键布阵」按钮
- 验证: 系统自动选择战力最高的武将填满6个位置
- 预期: 空位被最高战力武将补充，已有武将可能被替换为更强者
- **PRD补充**: 一键布阵规则细化：
  - 优先填充空位，不替换已有武将
  - 按战力从高到低依次填入
  - 已在其他编队的武将默认不选（可手动覆盖）
  - 填入后显示总战力变化（如"战力 +320 ↑"）

### 2.3 手动调整阵容
- 操作: 拖拽武将头像到目标阵位，或点击阵位选择武将
- 验证: 武将可在前排/后排间自由调换，同位置替换生效
- 预期: 阵容实时更新，支持取消操作恢复原阵容

### 2.4 查看战力预估
- 操作: 布阵完成后查看面板上方战力对比区
- 验证: 显示我方总战力 vs 敌方总战力的数值和进度条
- 预期: 战力差距直观可见，帮助判断是否需要调整阵容
- **PRD补充**: 战力预估判定规则：
  - 我方战力 ≥ 推荐战力 → 绿色"战力充足 ✓"
  - 我方战力 < 推荐战力 → 红色"战力不足 ✗" + 差距百分比

### 2.5 查看智能推荐
- 操作: 点击「智能推荐」按钮
- 验证: 基于敌方阵容推荐克制武将，高亮推荐位置
- 预期: 推荐武将带有克制标识，一键应用推荐阵容
- **PRD补充**: 智能推荐算法权重：
  - 战力匹配（40%）: 总战力尽可能接近关卡推荐战力
  - 阵营羁绊（25%）: 3同阵营攻击+15%；6同阵营攻击+25%+防御+15%
  - 职业均衡（20%）: 坦克+输出+治疗+控制合理搭配
  - 关卡克制（15%）: 多灼烧敌人→高智/治疗型；多物理→高防/坦克型
  - 输出1~3套推荐方案，每套含：6名武将及站位、总战力、匹配度评分、推荐理由

### 2.6 查看敌方预览
- 操作: 在布阵面板查看右侧/上方敌方阵容信息
- 验证: 显示敌方武将列表、兵种、战力
- 预期: 可据此调整我方阵容进行克制

### 3.3 伤害计算验证
- 操作: 观察伤害数值飘字
- 验证: 伤害值 = 攻击×技能倍率 - 防御×减免，暴击有额外加成
- 预期: 普通伤害白色飘字，暴击红色大字，克制伤害有标识
- **PRD补充**: 完整伤害计算公式：
  ```
  基础伤害 = 攻击方攻击力 × (1 + 攻击加成) - 防守方防御力 × (1 + 防御加成)
  最终伤害 = max(1, 基础伤害) × 暴击系数 × 技能倍率 × 克制系数 × 随机波动(0.9~1.1)
  ```
  系数明细：
  - 暴击系数: 暴击率随机触发 → ×1.5
  - 技能倍率: 使用技能时 → ×1.5~3.0
  - 克制系数: 属性克制 → ×1.5（克制）/ ×0.7（被克制）

### 3.4 技能释放观察
- 操作: 观察怒气条变化及技能释放时机
- 验证: 怒气满时自动释放大招，怒气条从满清空
- 预期: 大招有专属特效动画，伤害明显高于普攻
- **PRD补充**: 技能系统完整规则：
  - 大招冷却: 每个大招有独立CD（通常3~5回合）
  - 怒气系统: 普攻/受击积攒怒气，满怒气可释放大招
  - 被动技能: 自动触发，无消耗

### 3.5 兵种克制验证
- 操作: 观察骑兵vs步兵、步兵vs弓兵、弓兵vs骑兵的伤害
- 验证: 骑兵>步兵>弓兵>骑兵的克制关系生效
- 预期: 克制方伤害提升，被克制方伤害降低，飘字有克制提示
- **⚠️ 矛盾修正**: Plan中描述"骑兵>步兵>弓兵>骑兵"（三角循环），但PRD中的克制关系为"骑兵>步兵>枪兵>骑兵"，弓兵和谋士无特殊克制关系。以PRD为准，修正为：
  - 骑兵 克制 步兵（×1.5），被 枪兵 克制（×0.7）
  - 步兵 克制 枪兵（×1.5），被 骑兵 克制（×0.7）
  - 枪兵 克制 骑兵（×1.5），被 步兵 克制（×0.7）
  - 弓兵: 无特殊克制/被克制
  - 谋士: 无特殊克制/被克制
  - 克制系数: ×1.5（伤害提升50%），被克系数: ×0.7（伤害降低30%）

### 3.7a 手动模式操作流程 🆕 [P1-1]

> 手动模式下玩家完全控制每个武将的行动，以下为完整操作步骤

- **Step 1 — 技能选择面板弹出**:
  - 触发: 轮到玩家武将行动时，画面进入短暂慢动作（0.5s），底部弹出技能选择面板
  - 面板内容: 武将头像 + 普攻按钮 + 可用技能按钮（显示CD状态和怒气消耗）+ 「自动」切换按钮
  - 按钮: 灰色不可用（CD中/怒气不足）、亮色可用（满足释放条件）、金色高亮（推荐技能）
  - 操作: 点击目标技能按钮 → 进入目标选择阶段
  - 超时: 15秒无操作自动使用普攻（防御性AI，优先攻击最近敌人）

- **Step 2 — 目标高亮与选择**:
  - 触发: 选择技能后，可选目标区域高亮显示
  - 高亮规则:
    - 单体技能: 可选目标闪烁红色边框，不可选目标半透明
    - 群体技能（横排/纵排/全体）: 目标区域显示红色半透明覆盖层
    - 治疗技能: 友方目标闪烁绿色边框
  - 操作: 点击高亮目标确认选择
  - 智能提示: 克制目标额外显示⚔️图标，低HP目标显示💀图标

- **Step 3 — 确认释放**:
  - 触发: 点击目标后，技能释放确认
  - 动画: 武将播放技能前摇动画 → 技能特效 → 伤害飘字/治疗飘字 → 技能后摇
  - 数据更新: 目标HP/怒气/状态效果实时更新，CD倒计时开始
  - 日志: 战斗日志记录本次操作详情

- **Step 4 — 取消操作**:
  - 触发: 在Step 1或Step 2阶段点击空白区域 / 按ESC键 / 点击「取消」按钮
  - 行为: 回退到上一步（Step 2→Step 1，Step 1→等待重新选择）
  - 限制: 技能释放动画播放中不可取消（Step 3阶段不可打断）
  - 手机端: 底部工具栏提供「↩取消」按钮，替代ESC键

### 3.8 调整战斗速度
- 操作: 点击倍速按钮
- 验证: 支持默认1倍速和2倍速切换
- 预期: 2倍速下战斗节奏加快，动画加速但不跳过
- **PRD补充**: 完整加速倍率体系：
  | 倍率 | 解锁条件 | 信息显示策略 |
  |:----:|---------|-------------|
  | 1× | 默认 | 显示所有伤害数字 |
  | 2× | 默认 | 隐藏普攻数字，仅显示暴击和技能伤害 |
  | 3× | VIP3+ | 仅显示暴击数字 |
  | ⏩极速 | VIP5+ | 完全隐藏伤害数字，仅显示KO提示 |

### 3.9 大招时停机制（半自动模式）🆕
- **触发条件**: 半自动模式下，同时 ≥ 2个大招就绪
- **操作**: 战斗速度降至0.3×（慢动作），玩家选择释放大招的目标和时机
- **预期**: 
  - 最大持续5秒
  - 结束条件：所有大招释放完毕 / 5秒超时 / 玩家点击跳过
  - 超时处理：未释放大招自动排队释放
- **异常处理**: 网络延迟导致时停不同步时，以服务端时间为准

## 6. 战斗↔武将系统串联流程 🆕

> 覆盖战斗系统与v2.0武将（HER）系统的跨系统交互

### 6.1 武将属性→战斗参数映射
- **触发条件**: 进入战前布阵或战斗初始化时
- **操作步骤**:
  1. BattleEngine 从 HER 系统读取武将基础属性（攻击/防御/速度/暴击率/HP）
  2. 计算最终战斗属性 = 基础属性 + 装备加成 + 科技加成 + 羁绊加成
  3. 按速度属性排序生成行动序列
- **预期结果**: 武将属性正确映射为战斗参数，速度决定行动顺序
- **异常处理**: 武将数据缺失时使用默认值并记录警告日志

#### 6.1a 技能数据映射规则 🆕
- **触发条件**: 战斗初始化时，BattleEngine 读取武将技能数据
- **映射规则**:

  | 武将技能字段 (HER) | 战斗参数 (BattleEngine) | 说明 |
  |---------------------|------------------------|------|
  | `skill.id` | `BattleSkill.id` | 技能唯一标识，用于战斗日志和结算 |
  | `skill.name` | `BattleSkill.name` | 技能名称，用于战斗飘字和日志显示 |
  | `skill.type` | `BattleSkill.type` | 技能类型：主动(active)/被动(passive)/大招(ultimate) |
  | `skill.element` | `BattleSkill.element` | 属性元素：物理/火/冰/雷/毒/治疗 |
  | `skill.multiplier` | `DamageCalculator.skillMultiplier` | 技能倍率，代入伤害公式 `最终伤害 × skillMultiplier` |
  | `skill.targetType` | `BattleSkill.targetType` | 目标类型：单体(single)/横排(row)/纵排(column)/全体(all)/自身(self) |
  | `skill.targetPriority` | `BattleSkill.targetPriority` | 目标优先级：最低HP/最高攻击/随机/最近 |
  | `skill.cooldown` | `BattleSkill.cooldown` | 冷却回合数，大招通常3~5回合 |
  | `skill.rageCost` | `BattleSkill.rageCost` | 怒气消耗，大招通常消耗100点怒气 |
  | `skill.statusEffects[]` | `BattleSkill.statusEffects[]` | 附带状态效果列表（灼烧/冰冻/中毒/眩晕/流血等） |
  | `skill.statusEffects[].type` | `StatusEffect.type` | 状态类型枚举 |
  | `skill.statusEffects[].duration` | `StatusEffect.duration` | 持续回合数 |
  | `skill.statusEffects[].value` | `StatusEffect.value` | 效果数值（如灼烧5%HP的0.05） |
  | `skill.buffs[]` | `BattleSkill.buffs[]` | 增益效果列表（攻击提升/防御提升等） |
  | `skill.debuffs[]` | `BattleSkill.debuffs[]` | 减益效果列表（防御降低/速度降低等） |
  | `skill.animationId` | `BattleRenderer.animationId` | 技能特效动画资源ID，用于渲染层播放 |
  | `skill.positionReq` | `FormationValidator.positionReq` | 位置要求（如"前排攻击+15%"的触发条件） |

- **被动技能映射**: 被动技能在战斗初始化时注册为触发器(trigger)，绑定事件条件：
  - `onBattleStart`: 战斗开始时触发（如开场加盾）
  - `onAttack`: 攻击时触发（如连击概率）
  - `onHit`: 受击时触发（如反伤）
  - `onKill`: 击杀时触发（如击杀回血）
  - `onAllyDeath`: 友军阵亡时触发（如复仇buff）
  - `onHPBelow(0.3)`: HP低于30%时触发（如绝境爆发）
- **异常处理**: 
  - 技能数据缺失时武将仅可使用普攻（倍率×1.0），记录警告日志
  - 未知状态效果类型忽略并记录，不影响战斗流程
  - 技能动画资源缺失时使用默认闪光特效替代

### 6.2 战前布阵↔编队系统联动
- **触发条件**: 打开战前布阵面板
- **操作步骤**:
  1. 调用 GeneralTeamSystem(v2.0) 获取当前编队配置
  2. FormationPanelComponent 渲染当前编队武将到对应阵位
  3. 玩家调整阵容（拖拽/一键布阵/智能推荐）
  4. 调整结果写回编队系统（临时编队，仅本次战斗生效）
- **预期结果**: 编队数据双向同步，布阵面板实时反映编队状态
- **异常处理**: 编队为空时自动触发一键布阵；武将已在其他编队时弹出确认提示

### 6.3 战斗经验→武将成长
- **触发条件**: 战斗结算完成
- **操作步骤**:
  1. 计算参战武将获得的经验值（基于关卡难度和星级）
  2. 经验值写入HER系统，更新武将等级和属性
  3. 如触发升级，更新武将战力值
- **预期结果**: 参战武将获得经验，等级提升后属性和战力同步更新
- **异常处理**: 经验溢出时自动处理升级，多级连升时逐级计算属性成长

### 6.3a 战斗经验值公式 🆕 [P1]

> 战斗结算时参战武将获得经验值的完整计算规则

- **基础公式**:
  ```
  经验值 = 关卡基础经验 × 星级倍率 × 首通倍率
  ```

- **关卡基础经验**（按章节递增）:

  | 章节 | 基础经验/关 | 说明 |
  |------|:----------:|------|
  | 黄巾之乱 | 50 | 新手阶段，快速升级 |
  | 群雄割据 | 120 | 约为前章 ×2.4 |
  | 官渡之战 | 250 | 约为前章 ×2.1 |
  | 赤壁之战 | 500 | 约为前章 ×2.0 |
  | 三国鼎立 | 1,000 | 约为前章 ×2.0 |
  | 一统天下 | 2,000 | 约为前章 ×2.0 |

- **星级倍率**:
  | 星级 | 倍率 |
  |:----:|:----:|
  | ★☆☆ | ×1.0 |
  | ★★☆ | ×1.5 |
  | ★★★ | ×2.0 |

- **首通倍率**: 首次通关 ×3.0（仅一次，与奖励首通倍率一致）

- **经验分配规则**:
  - 参战武将均分总经验值（6名武将各得 1/6）
  - 阵亡武将同样获得经验（不因阵亡减少）
  - 未参战武将不获得经验

- **示例计算**:
  - 赤壁之战 4-1 首次三星通关: 经验值 = 500 × 2.0 × 3.0 = 3,000，每名武将获得 500 经验
  - 三国鼎立 5-3 重复二星通关: 经验值 = 1,000 × 1.5 × 1.0 = 1,500，每名武将获得 250 经验

- **异常处理**:
  - 经验溢出（超过升级所需）时自动处理多级连升，逐级计算属性成长
  - 扫荡经验 = 正常经验 × 0.8（扫荡效率略低于手动战斗）
  - 离线推图经验 = 正常经验 × 0.5（离线效率较低）

### 6.4 武将碎片掉落→武将解锁/升星

  2. 碎片写入HER系统对应武将的碎片库存
  3. 如碎片数量达到解锁/升星阈值，弹出提示（"关羽碎片已集齐，可解锁！"）
- **预期结果**: 武将碎片正确入账，达到阈值时给出解锁提示
- **异常处理**: 碎片溢出（已满星武将）时自动转化为通用货币

## 7. 战斗↔资源系统串联流程 🆕

> 覆盖战斗系统与资源（RES）系统的跨系统交互

### 7.1 战斗消耗→资源扣减
- **触发条件**: 点击「出征」按钮确认出征
- **操作步骤**:
  1. 检查出征所需资源（兵力/粮草）是否充足
  2. 扣减对应资源：兵力按关卡难度计算，粮草按出征距离计算
  3. 资源扣减成功后进入战斗加载
- **预期结果**: 出征成功扣减资源，主城资源数值实时更新
- **异常处理**: 
  - 资源不足时弹出提示"粮草不足，请等待产出或前往商店购买"，按钮置灰不可点击
  - 扣减失败（并发冲突）时自动重试一次，仍失败则取消出征

### 7.2 战斗奖励→资源入账
- **触发条件**: 战斗胜利结算
- **操作步骤**:
  1. RewardDistributor 计算总奖励（基础奖励 × 星级倍率 × 首通倍率）
  2. 奖励飞出动画播放
  3. 资源（粮草/铜钱/经验书/兵符）写入RES系统
  4. 道具（装备碎片/武将碎片）写入对应库存系统
  5. 更新主城资源显示
- **预期结果**: 所有奖励正确入账，主城资源数值实时更新
- **异常处理**: 奖励入账失败时缓存到本地，下次登录时重试入账

### 7.3 首通奖励→资源暴击
- **触发条件**: 首次通关某个关卡
- **操作步骤**:
  1. 检测该关卡是否为首通（flag标记）
  2. 首通倍率 ×3.0 叠加到基础奖励
  3. 首通奖励包含天命、武将碎片、装备等特殊物品
  4. 清除首通flag，后续通关不再享受首通倍率
- **预期结果**: 首通奖励远高于重复通关，且有特殊物品
- **异常处理**: 首通flag异常（重复首通）时以服务端记录为准

### 7.4 重复奖励→日常资源获取
- **触发条件**: 重复通关已通关关卡
- **操作步骤**:
  1. 按掉落表抽取重复奖励（粮草/铜钱/经验书/兵符）
  2. 不再享受首通倍率（×3.0）
  3. 仍享受星级倍率（×1.0~×2.0）
- **预期结果**: 重复通关获得基础资源，用于日常养成
- **异常处理**: 同一关卡连续快速重复通关时，奖励正常发放不丢失

### 7.5 兵力/粮草资源获取与恢复流程 🆕 [P0]

> 兵力和粮草是出征的核心消耗资源，以下为完整的获取、恢复与上限规则

|---------|:------:|------|------|
| 兵营产出 | 基础50/小时 | 持续 | 兵营等级越高产出越快，每级+10/小时 |
| 商店购买 | 500/次 | 不限次数 | 消耗元宝×50/次 |
| 任务奖励 | 200~1,000 | 按任务 | 日常/成就/主线任务 |
| 战斗奖励 | 100~300 | 每次通关 | 胜利后按关卡难度获得 |
| 离线挂机 | 基础20/小时 | 离线累积 | 已三星通关关卡产出 |

#### 7.5.2 粮草获取途径

| 获取途径 | 产出量 | 频率 | 说明 |
|---------|:------:|------|------|
| 农田产出 | 基础30/小时 | 持续 | 农田等级越高产出越快，每级+8/小时 |

#### 7.5.3 上限值与自然恢复

| 资源 | 基础上限 | 上限提升方式 | 自然恢复速率 |
|------|:--------:|-------------|-------------|

#### 7.5.4 完整流程

- **预期结果**: 资源获取和消耗形成闭环，玩家有明确的资源规划意识
- **异常处理**:
  - 产出计算异常时使用基础产出兜底，不中断游戏流程
  - 购买失败（元宝不足）时弹窗提示充值或获取元宝途径
  - 并发扣减冲突时以服务端数据为准，客户端显示可能短暂延迟

## 8. 地图↔战斗系统串联流程 🆕 **[v4.0预览内容，不纳入v3.0验收]**

> ⚠️ **MAP系统预览标注**: 以下流程涉及MAP系统，属于v4.0攻城略地(下)的预览内容，v3.0仅验收CBT模块。MAP相关功能点不纳入v3.0验收范围。
>
> 覆盖世界地图（MAP）系统与战斗（CBT）系统的跨系统交互
> 注：MAP系统完整功能属于v3.0攻城略地(下)范围，此处仅覆盖与v3.0(上)战斗核心循环相关的串联流程

### 8.1 领土征服→触发战斗
- **触发条件**: 在世界地图上选择相邻非己方领土，点击「征服」
- **操作步骤**:
  1. 检查征服条件：领土相邻 + 可用兵力 ≥ 目标驻防兵力 × 1.5
  2. 系统预估胜率 = min(95%, max(5%, (我方战力 / 敌方战力) × 50% + 地形修正))
  3. 显示胜率预估（翠绿/金色/琥珀橙/赤红四档）
  4. 玩家确认出征 → 扣减兵力 → 进入战前布阵 → 进入战斗
  5. 战斗结算后更新领土归属
- **预期结果**: 征服流程完整连通，领土归属实时更新
- **异常处理**: 
  - 兵力不足时"征服"按钮置灰，提示"兵力不足"
  - 非相邻领土点击"征服"提示"领土不相邻，无法出征"
  - 战斗中断（网络断开）时保存战斗状态，重连后继续

### 8.2 地形效果→战斗参数修正
- **触发条件**: 在不同地形领土上发起战斗
- **操作步骤**:
  1. BattleEngine 读取目标领土的地形类型
  2. 应用地形战斗效果修正：
     - 山地: 防守方防御 +20%
     - 城池: 防守方城防加成
     - 森林: 伏兵突袭加成
     - 关隘: 防守方防御 +40%
     - 平原/水域: 无特殊战斗修正
  3. 修正后的参数用于伤害计算和战斗模拟
- **预期结果**: 不同地形对战斗结果产生可感知的影响
- **异常处理**: 地形数据缺失时默认为"平原"（无修正）

### 8.3 攻城战→特殊战斗流程
- **触发条件**: 对城池类领土发起征服
- **操作步骤**:
  1. 检查攻城条件：领土相邻 + 出征兵力 ≥ 城池驻防兵力 × 2.0 + 粮草×500 + 每日次数 ≤ 3
  2. 计算城防值 = 基础城防(1000) × 城市等级 × (1 + 城防科技加成) ⚠️ *以MAP PRD v1.1统一声明为准*
  3. 城防效果：每点城防值降低攻方伤害0.02%（上限-50%）
  4. 进入战斗，战斗中攻方伤害受城防减免
  5. 胜利条件：城防值归零（**唯一条件**） ⚠️ *旧三条件(城防归零+守军全灭+战力≥1.3倍)已简化*
  6. 占领后获得城市产出，初始50%，24小时后100%
  7. 攻占后24小时内不可被反攻（新手保护期）
- **预期结果**: 攻城战难度明显高于普通征服，城防减免效果可感知
- **异常处理**: 
  - 每日攻城次数耗尽时提示"今日攻城次数已用完"
  - 攻城失败：损失30%出征兵力，粮草不返还 ⚠️ *旧规则"返还50%"已废弃，以MAP PRD统一声明为准*
  - 城防值计算异常时使用基础城防值(1000)兜底

### 8.4 征服结果→领土状态更新
- **触发条件**: 领土征服战斗结算完成
- **操作步骤**:
  1. 胜利：领土归属切换为攻方，触发离线领土变化标记（金色脉冲光环+"新"标签）
  2. 新占领领土自动分配50%兵力上限的驻防
  3. 失败：领土不变，攻方损失30%出征兵力
  4. 更新地图渲染，领土颜色和状态实时刷新
- **预期结果**: 领土状态在地图上即时更新，视觉反馈清晰
- **异常处理**: 领土状态更新失败时缓存操作，下次进入地图时同步

### 8.5 地图事件→战斗触发 🆕
- **触发条件**: 在世界地图上遇到山贼/流寇等战斗类事件，选择"强攻"
- **操作步骤**:
  1. 地图事件触发（每小时10%概率，最多3个未处理事件）
  2. 玩家点击事件图标 → 显示事件详情和选择分支（强攻/谈判/忽略）
  3. 选择"强攻" → 进入战前布阵 → 进入战斗
  4. 战斗胜利获得事件奖励（装备/资源）
  5. 战斗失败无惩罚，可重新选择其他分支
- **预期结果**: 地图事件战斗作为支线战斗，提供额外资源和乐趣
- **异常处理**: 
  - 事件超时未处理自动消失，无惩罚
  - 事件战斗中事件不消失，战斗结束后事件标记为已处理
  - 同时最多3个未处理事件，超出时不再触发新事件

### 9.4 VIP系统依赖说明 🆕

- **降级保护**: VIP等级不降级，已解锁功能永久可用
- **异常处理**: VIP状态校验失败时降级为默认速度/功能，不影响核心战斗流程

### 9.5 关卡↔扫荡↔离线统一状态机 🆕

> 三星通关、扫荡、离线挂机三者形成递进解锁关系，统一状态机描述如下

```
关卡状态流转（统一状态机）:

[未解锁] ──通关前置关卡──→ [可挑战]
                               │
                         ┌─────┴─────┐
                         │  进入战斗   │
                         └─────┬─────┘
                               │
                    ┌──────────┼──────────┐
                    ▼          ▼          ▼
              [已通关]    [已通关]    [已通关]
               ★☆☆        ★★☆        ★★★
                    │          │          │
                    │          │    ┌─────┴─────┐
                    │          │    │ 解锁扫荡   │
                    │          │    │ 解锁离线挂机│
                    │          │    └───────────┘
                    ▼          ▼          ▼
              可重新出征  可重新出征   可出征/扫荡/挂机
              追求满星    追求满星    收益持续累积
```

**状态机规则**:
1. **三星通关 → 解锁扫荡**: 关卡获得★★★评价后，解锁「扫荡」按钮，可消耗扫荡令快速获取资源
2. **三星通关 → 解锁离线挂机**: 已三星通关的关卡自动累积挂机收益（每小时产出 = 基础产出 × 加成系数）
3. **扫荡 ≠ 离线推图**: 扫荡是主动消耗扫荡令获取即时奖励；离线推图是系统自动尝试推进未通关关卡
4. **离线挂机收益上限**: 最多累积12小时产出，超时不再增长，需登录领取后重新累积
5. **星级可刷新**: 已通关关卡可重新出征追求更高星级，星级提升后即时更新可用功能

**离线推图前置条件**（与扫荡独立）:
- 当前关卡为已三星通关关卡的下一关
- 我方战力 ≥ 推荐战力 × 1.2
- 离线推图开关已开启（默认开启）
- 离线最多推进3关（防止过度推进错过三星）

### 9.5a 扫荡状态回写规则 🆕 [P1-3]

> 扫荡执行完成后，需将结果同步回写到多个系统，确保全局状态一致性

- **回写步骤（原子事务）**:
  1. **关卡进度回写**: 更新关卡最后扫荡时间戳（`lastSweepAt`），不改变星级和通关状态
  2. **资源系统回写**: 扫荡奖励写入RES系统——粮草/铜钱/经验书/兵符即时入账，触发主城资源UI刷新
  3. **道具库存回写**: 武将碎片/装备碎片写入HER/EQP对应库存，碎片达到阈值时标记可合成flag
  4. **扫荡令扣减**: 扣减扫荡令库存（`sweepTicket -= N`），库存为0时禁用扫荡按钮
  5. **武将经验回写**: 参战编队武将获得扫荡经验（= 关卡经验 × 0.8），经验溢出时触发连升
  6. **每日计数更新**: 挑战/扫荡每日计数器 +N（`dailySweepCount += N`），达到上限时禁用批量扫荡
  7. **离线挂机计时器重置**: 如该关卡同时处于离线挂机状态，扫荡后重置挂机收益累积计时器（从0开始重新累积）

- **回写一致性保证**:
  - 所有回写操作在同一数据库事务中完成，任一步失败则整笔回滚
  - 回滚后扫荡令不扣减，奖励不入账，玩家可重新执行扫荡
  - 客户端本地缓存同步更新，避免下次进入时显示过期数据

- **异常处理**:
  - 网络中断: 扫荡请求缓存到本地队列，重连后自动重试（幂等设计，扫荡令扣减使用UUID防重）
  - 并发冲突: 同一关卡同时被扫荡和手动出征时，以先到达服务端的请求为准，后者返回"关卡状态已变更"提示
  - 批量扫荡部分失败: 按成功次数发放奖励，失败次数的扫荡令退还

## 11. 挑战关卡流程 🆕

> PRD中存在 [CBT-8] 挑战关卡，Plan和原Play文档中均缺失，此处补充

### 11.1 进入挑战关卡
- **触发条件**: 在战役界面底部查看挑战关卡入口
- **操作步骤**:
  1. 点击底部8个烽火台图标网格中的任一挑战关卡
  2. 查看关卡详情（敌方阵容、奖励、消耗兵力）
  3. 确认挑战 → 进入战前布阵 → 进入战斗
- **预期结果**: 挑战关卡独立于主线战役，固定难度不随主线进度变化
- **异常处理**: 
  - 兵力不足时提示"兵力不足，无法挑战"
  - 每日重置后可重新挑战

### 11.2 挑战关卡结算
- **触发条件**: 挑战关卡战斗结束
- **操作步骤**:
  1. 胜利：获得特殊材料、高级碎片等独特奖励
  2. 失败：显示失败原因和推荐提升方向
  3. 每日重置挑战次数
- **预期结果**: 挑战关卡提供主线之外的高级资源获取途径
- **异常处理**: 每日挑战次数耗尽时提示"今日挑战次数已用完，明日重置"

### 11.3 挑战关卡资源串联（扣减→结算入账完整路径）🆕 [P1-2]

> 挑战关卡涉及资源消耗与奖励发放的完整闭环，以下为从扣减到入账的全链路

- **Phase 1 — 资源扣减（出征确认时）**:
  1. **兵力扣减**: 按挑战关卡配置表读取兵力消耗量（`cost.army`），调用RES系统扣减 `army -= cost.army`
  2. **挑战次数扣减**: 每日挑战计数器 +1（`dailyChallengeCount++`），达到上限（默认3次/日）时锁定入口
  3. **体力扣减**: 扣减体力值（`stamina -= cost.stamina`，通常为12~20点），体力不足时弹窗提示恢复途径
  4. **前置校验**: 兵力 ≥ 消耗 AND 每日次数未满 AND 体力 ≥ 消耗 → 全部通过才允许进入布阵
  5. **预锁定机制**: 资源预锁定（不实际扣减但冻结），战斗结束后确认扣减；战斗中断时释放预锁

- **Phase 2 — 战斗执行（战斗过程中）**:
  1. 进入战前布阵 → 进入战斗 → 战斗过程同主线关卡（§3）
  2. 挑战关卡固定难度不随主线进度变化，敌方阵容由关卡配置表决定
  3. 战斗中断处理同§12.1，预锁资源释放，不计挑战次数

- **Phase 3 — 结算入账（战斗胜利时）**:
  1. **确认资源扣减**: 预锁资源正式扣减（兵力/体力），释放预锁记录
  2. **计算挑战奖励**: 
     - 基础奖励 = 挑战关卡奖励表（固定掉落 + 概率掉落）
     - 挑战关卡无星级倍率（仅有通关/未通关两种结果）
     - 首通额外奖励：首次通关该挑战关卡，额外获得天命×50 + 指定武将碎片×5
  3. **特殊材料入账**: 挑战关卡独有材料（如「虎符」「兵书残页」「锻造石」）写入专用材料库存
  4. **高级碎片入账**: 橙色/红色品质武将碎片写入HER系统碎片库存
  5. **通用资源入账**: 粮草/铜钱/经验书写入RES系统，触发主城UI刷新
  6. **武将经验入账**: 参战武将获得挑战关卡经验（= 主线同难度关卡 × 1.5）
  7. **入账动画**: 奖励飞出动画 + 稀有物品金色高亮 + 首通奖励全屏特效

- **Phase 4 — 失败处理（战斗失败时）**:
  1. **释放预锁资源**: 兵力/体力预锁释放，不实际扣减
  2. **挑战次数保留**: 失败不消耗每日挑战次数（仅胜利时计数）
  3. **失败面板**: 显示失败原因 + 推荐提升方向 + 「重新挑战」按钮
  4. **连续失败保护**: 连续失败3次时弹出"建议提升战力后继续"提示，挑战入口不锁定

- **完整路径时序图**:
  ```
  [点击挑战] → 前置校验(兵力/次数/体力) 
    → 通过: 预锁资源 → 布阵 → 战斗
      → 胜利: 确认扣减 → 计算奖励 → 入账(材料/碎片/资源/经验) → 结算面板
      → 失败: 释放预锁 → 失败面板(次数保留)
    → 不通过: 提示不足项 → 引导获取途径
  ```

## 12. 异常与边界场景处理 🆕

> 覆盖全流程中的异常场景和边界条件

### 12.1 战斗中断处理
- **触发条件**: 战斗过程中网络断开/浏览器关闭/应用切后台
- **操作步骤**:
  1. 战斗状态自动保存到服务端（每回合结束时存档）
  2. 重新连接后弹出"检测到未完成的战斗"提示
  3. 玩家选择"继续战斗"从断点恢复 或 "放弃战斗"（不扣资源，不计败绩）
- **预期结果**: 战斗不丢失，玩家可选择继续或放弃
- **异常处理**: 存档损坏时自动判定为平局，返还消耗资源

### 12.2 回合上限耗尽
- **触发条件**: 战斗达到8回合上限，双方均未全灭
- **操作步骤**:
  1. 系统判定为平局
  2. 显示平局面板："回合耗尽，未能分出胜负"
  3. 不发放通关奖励，不扣除首通flag
  4. 推荐提升方向："提升输出能力，争取在8回合内击败敌人"
- **预期结果**: 平局不计为通关也不计为失败，玩家可重新挑战
- **异常处理**: 无

### 12.3 武将阵亡处理
- **触发条件**: 战斗中我方武将HP降为0
- **操作步骤**:
  1. 武将播放阵亡动画后从战场移除
  2. 不再参与后续回合行动
  3. 结算时根据存活人数计算星级评定
  4. 阵亡武将不损失任何永久属性（仅本次战斗生效）
- **预期结果**: 武将阵亡仅影响本次战斗，不影响武将养成数据
- **异常处理**: 无

### 12.4 全军覆没处理
- **触发条件**: 我方所有武将HP降为0
- **操作步骤**:
  1. 触发战斗失败结算
  2. 显示失败面板，分析失败原因：
     - 战力不足：显示差距百分比
     - 阵容被克制：显示克制关系
     - 策略失误：显示关键回合事件
  3. 给出具体推荐（如"升级关羽至30级"、"使用骑兵克制敌方步兵"）
  4. 不扣减武将永久属性，不消耗首通flag
- **预期结果**: 失败体验友好，给出可操作的改进建议
- **异常处理**: 无

### 12.5 资源溢出处理
- **触发条件**: 战斗奖励导致某项资源超过存储上限
- **操作步骤**:
  1. 正常入账到上限值
  2. 溢出部分弹出提示："粮草已达上限，溢出的×500粮草已转化为铜钱"
  3. 溢出资源按一定比例转化为其他资源
- **预期结果**: 资源不浪费，玩家清楚知道溢出去向
- **异常处理**: 无

### 12.6 关卡数据异常处理
- **触发条件**: 关卡配置数据缺失或损坏
- **操作步骤**:
  1. 显示通用错误提示："关卡数据加载失败"
  2. 提供"重试"按钮
  3. 后台记录错误日志，上报异常
  4. 不影响其他关卡的正常游玩
- **预期结果**: 单个关卡异常不影响整体游戏体验
- **异常处理**: 连续3次加载失败后提示"请检查网络连接后重启游戏"

  - 小屏设备（≤320px宽）节点过密时自动缩放至60%
  - 滑动手势与页面滚动冲突时，关卡区域优先响应横向滑动（切章节）

### 13.2 手机端战前布阵
- **触发条件**: 手机端点击关卡「出征」进入布阵
- **操作步骤**:
  1. 布阵面板以Bottom Sheet形式从底部弹出，占屏幕高度70%
  2. 阵位排列为2行×3列网格（前排/后排），尺寸适配触控操作（最小44×44pt）
  3. 触控拖拽替代鼠标拖拽：长按武将头像0.3s触发拖拽，拖拽到目标阵位松手
  4. 武将选择列表在Bottom Sheet下半部分纵向滚动，支持滑动浏览
  5. 点击武将头像弹出快捷菜单：「上阵」「查看详情」「取消」
  6. 「一键布阵」「智能推荐」「出征」按钮固定在Bottom Sheet底部
- **预期结果**: 手机端布阵操作流畅，触控精准，按钮易于点击
- **异常处理**: 
  - 误触时支持撤销操作（摇一摇或点击撤销按钮）
  - 拖拽到无效位置时武将头像弹回原位并震动反馈

### 13.3 手机端战斗场景
- **触发条件**: 手机端进入战斗
- **操作步骤**:
  1. 战斗场景全屏显示，自动隐藏浏览器地址栏（如支持）
  2. 倍速/模式切换/暂停按钮移至屏幕底部工具栏，图标按钮横排
  3. 怒气条和血条适配小屏尺寸（宽度缩至PC端60%，高度不变）
  4. 武将站位整体缩放适配屏幕宽度，保持左右对称布局
  5. 伤害飘字字号缩小至PC端70%，避免遮挡
  6. 暂停菜单以全屏覆盖层形式弹出（非PC端侧边栏）
  7. 武将详情面板以Bottom Sheet形式弹出
- **预期结果**: 手机端战斗体验与PC端一致，信息可读，操作便捷
- **异常处理**: 
  - 横屏/竖屏切换时自动适配布局（横屏使用PC端布局比例）
  - 低端设备帧率<30fps时自动降低特效质量（关闭粒子效果）

### 13.4 手机端结算面板
- **触发条件**: 手机端战斗结束
- **操作步骤**:
  1. 结算面板以Bottom Sheet形式从底部弹出，占屏幕高度80%
  2. 奖励列表纵向排列，每行显示物品图标+名称+数量
  3. 底部固定「确认」按钮（高度48px，易于触控）
  4. 星级评定动画在面板顶部播放
  5. 奖励飞出动画简化为列表逐行高亮（替代PC端飞入动画，避免小屏视觉混乱）
- **预期结果**: 手机端结算信息完整可读，操作便捷
- **异常处理**: 奖励列表过长时支持滚动，底部按钮保持固定可见
