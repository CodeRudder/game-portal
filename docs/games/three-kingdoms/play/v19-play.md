# v19.0 天下一统(上) — 玩游戏流程

> 核心循环: 统一系统驱动全局打磨进程→设置系统完善(基础/音效/画面/账号)提供统一基石→动画规范统一全系统节奏→存档管理保障统一进度→品质感提升为天下一统铺路

---

## 1. 统一系统

### 1.1 基础设置统一
- 操作: 打开设置面板 → 切换语言(简中/繁中/EN/日本語) → 确认重启弹窗→重启验证语言生效 → 设置时区(UTC+8) → 验证日常重置/活动倒计时即时变更 → 通知总开关关闭 → 验证建筑完成/远征归来/活动提醒/好友消息/联盟通知5项全部静默
- 验证: 4种语言切换无遗漏文本 | 时区变更即时影响日常重置时间 | 通知总开关=OFF时屏蔽全部子项
- 预期: 基础设置为统一进程提供语言/时区/通知基础设施，全平台体验一致

### 1.2 通知精细控制
- 操作: 通知总开关开启 → 逐项关闭建筑完成/远征归来/活动提醒/好友消息/联盟通知 → 验证每项关闭后对应通知不推送、其余项仍正常 → 设置免打扰时段(23:00~07:00) → 在免打扰时段内触发建筑完成 → 验证通知被静默 → 时段外触发通知正常推送 → 连续触发5条同类型通知 → 验证5分钟内仅推送1条(频率控制) → 全天累计推送达30条 → 验证当日不再推送(每日上限)
- 验证: 5项独立开关互不影响 | 免打扰时段内全部静默 | 同类型5分钟内仅1条 | 每日上限30条
- 预期: 通知精细控制避免打扰疲劳，免打扰时段保护玩家休息，频率上限防止刷屏

### 1.3 音效系统统一
- 操作: 主音量设80% → BGM设60% → 验证实际输出=48%(分类×主音量) → 音效总开关关闭→验证等同于主音量=0% → 切后台→BGM渐弱1s至静音 → 模拟来电→立即静音→恢复渐入0.5s → 低电量(<20%)→BGM自动降50% → 首次启动→BGM延迟3s播放
- 验证: 实际输出=分类音量×主音量公式精确 | 4通道(BGM/音效/语音/战斗)独立可控 | 特殊场景(后台/来电/低电量/首次启动)自动处理
- 预期: 音效系统为统一进程营造沉浸式音频体验，4通道精细控制满足不同玩家偏好

### 1.4 音效开关独立验证
- 操作: BGM开关关闭→验证背景音乐停止播放但音效/语音仍正常 → 语音开关关闭→验证武将语音/剧情配音静音但字幕保留 → 战斗音效开关关闭→进入战斗→验证战斗场景音效静音但BGM仍播放 → 全部开关开启→验证4通道全部恢复正常
- 验证: 4个开关(音效总开关/BGM开关/语音开关/战斗音效开关)独立控制正确 | 关闭BGM=停止播放非静音 | 关闭语音仅保留字幕
- 预期: 4通道开关独立控制满足不同场景需求，语音关闭保留字幕保障信息获取

### 1.5 画质管理统一
- 操作: 首次启动→自动检测(CPU≥8核+内存≥8GB→高) → 手动切低画质→验证粒子关+阴影关+水墨关+30fps → 切高画质→全部开启+60fps → 展开高级选项→独立关闭水墨晕染→验证即时生效+水墨过渡0.6s → 低端设备验证高级选项隐藏
- 验证: 4档预设(低/中/高/自动)参数正确 | 5项高级选项独立可控 | 切换时水墨晕染过渡无闪烁
- 预期: 画质系统为统一进程提供视觉品质保障，自动检测推荐合理，手动覆盖即时生效

### 1.6 画质自动检测与手机端专属设置
- 操作: 模拟低端设备(CPU 2核+内存2GB)→验证自动推荐低画质+标注「★ 推荐」朱砂印章 → 模拟中端设备(CPU 4核+内存4GB)→验证推荐中画质 → 手动覆盖推荐→验证可切换 → 手机端开启省电模式→验证强制低画质+30fps+关闭省电后恢复原设置 → 开启触控反馈→操作时验证震动 → 开启左手模式→验证UI布局镜像翻转 → 开启屏幕常亮→验证游戏内屏幕不熄灭 → 切换字体大小(小/中/大)→验证文字尺寸变化
- 验证: 自动检测逻辑精确(CPU+内存→推荐档位) | 手动覆盖随时可切 | 手机端5项专属设置全部生效 | 省电模式强制低画质可恢复
- 预期: 自动检测为不同设备提供最优推荐，手机端专属设置保障移动端体验

### 1.7 高级画质选项独立控制
- 操作: 画质设为高 → 逐项关闭粒子特效→验证战斗/建筑产出粒子消失 → 关闭实时阴影→验证建筑角色动态阴影消失 → 关闭水墨晕染→验证面板过渡无水墨效果 → 帧率限制切30fps→验证渲染帧率下降 → 关闭抗锯齿→验证边缘锯齿可见 → 全部恢复开启→验证效果恢复
- 验证: 5项高级选项(粒子/阴影/水墨/帧率/抗锯齿)独立可控 | 每项切换即时生效 | 低端设备隐藏高级选项显示「当前设备不支持高级画质」
- 预期: 高级选项为高端设备玩家提供精细控制，低端设备自动隐藏避免无效操作

## 2. 统一条件

### 2.1 账号绑定条件
- 操作: 游客登录→绑定手机号→短信验证码流程→确认绑定→检查元宝×50到账 → 继续绑定邮箱→验证多方式共存 → 绑定微信→验证第三方授权跳转 → 验证游客账号30天自动清除倒计时显示
- 验证: 手机号/邮箱/第三方(微信/QQ/Apple)三种方式均可绑定 | 首次绑定奖励元宝×50仅发一次 | 游客未绑定30天清除提示可见
- 预期: 账号绑定是统一条件第一步，确保玩家身份持久化，奖励激励完成绑定

### 2.2 账号绑定边界验证
- 操作: 绑定手机号后尝试再次绑定另一手机号→验证提示已绑定 → 解绑手机号→验证7天内不可重新绑定同类型(换绑冷却) → 绑定上限验证: 1个手机号+1个邮箱+1个第三方共存正确 → 尝试绑定第2个邮箱→验证提示已达上限 → 游客账号未绑定→验证30天倒计时显示+到期清除提示
- 验证: 每类型限绑1个(手机号/邮箱/第三方各1) | 换绑冷却7天 | 游客30天清除倒计时可见 | 首次绑定奖励仅发一次(多方式绑定不重复发放)
- 预期: 绑定规则清晰防滥用，换绑冷却保障安全，游客清除提示驱动及时绑定

### 2.3 云存档同步条件
- 操作: 设置同步频率"退出时" → 退出游戏→验证自动上传(进度指示器) → 另一台设备登录→云端下载 → 设置冲突策略"始终询问" → 两台设备分别存档→触发冲突弹窗→选择保留云端版本→验证数据正确 → 开启WiFi仅同步→移动网络下验证同步阻止
- 验证: 自动同步按频率执行(退出时/每小时/仅手动) | 端到端加密存储 | 冲突策略(最新/云端/询问)三选一正确 | WiFi仅同步在移动网络阻止
- 预期: 云存档确保统一进度跨设备不丢失，冲突解决策略保障数据完整性

### 2.4 多设备管理
- 操作: 查看已绑定设备列表→验证最多5台 → 标记1台为主力设备→验证优先同步 → 解绑一台非主力设备→验证24h冷却期内不可绑定新设备 → 冷却期结束→绑定新设备成功 → 3种冲突策略切换: "始终保留最新"→两设备不同修改→保留时间最新的 → "始终保留云端"→保留云端数据 → "始终询问"(默认)→弹出冲突选择弹窗
- 验证: 最大5台设备+主力设备标记+解绑24h冷却 | 3种冲突策略(最新/云端/询问)行为正确 | 默认策略为"始终询问"
- 预期: 多设备管理支持跨设备游玩，主力设备标记优化同步优先级，冲突策略灵活可选

### 2.5 存档槽位条件
- 操作: 查看存档列表→验证3免费+1付费(灰锁)槽位 → 手动保存到槽位1 → 等待15分钟→验证自动存档覆盖 → 读取槽位1→验证全量数据恢复(资源/建筑/武将/科技) → 删除槽位1→确认二次弹窗 → 第4槽位支付200元宝解锁
- 验证: 3免费槽位可用，第4槽位需200元宝 | 自动存档每15分钟覆盖 | 存档2-5MB含全量数据
- 预期: 存档槽位为统一进程提供多节点存档能力，玩家可安全回溯任意进度

### 2.6 退出登录流程
- 操作: 点击退出登录 → 验证自动触发云存档上传(进度指示器显示) → 上传完成→确认退出 → 验证本地缓存保留 → 重新登录→验证快速恢复(无需重新下载全部资源) → 连续退出登录→验证无冷却限制
- 验证: 退出前自动云存档上传 | 本地缓存保留可快速恢复 | 退出无冷却
- 预期: 退出流程自动保存保障数据安全，本地缓存加速下次登录体验

  6. 验证免打扰时段内累积的通知在07:00:00后不批量补发(静默即丢弃)
  7. 修改免打扰时段为00:00~00:00(全天免打扰) → 验证全天静默
  8. 关闭免打扰时段 → 验证恢复正常推送
- 预期结果: 免打扰边界精确到秒 | 静默通知不补发 | 全天免打扰有效 | 关闭后恢复
- 异常处理: 边界时间偏差>1秒→标记为P1时间Bug | 静默通知被补发→标记为P1逻辑Bug

### 7.5 画质自动检测手动覆盖持久化验证（SET-3）[R5新增]
- 触发条件: 首次启动自动检测后手动覆盖画质档位
- 操作步骤:
  1. 首次启动 → 自动检测推荐中画质(★推荐印章) → 手动覆盖为高画质
  2. 退出游戏 → 重新启动 → 验证画质仍为高画质(手动覆盖持久化，不重新检测)
  3. 点击"重新检测"按钮 → 验证重新执行自动检测 → 仍推荐中画质+★推荐印章 → 手动覆盖为低画质
  4. 退出 → 重新启动 → 验证画质为低画质(持久化)
  5. 切换设备(如从PC换到手机) → 云同步读取 → 验证画质为低画质(同步来的) → 但显示★推荐中画质提示(本机检测)
  6. 验证手动设置优先于自动检测，自动检测仅作为推荐参考
- 预期结果: 手动覆盖持久化不重新检测 | 重新启动保留手动设置 | 跨设备同步后仍可显示本机推荐 | 手动设置优先
- 异常处理: 重启后恢复为自动检测值→标记为P0持久化Bug | 跨设备同步覆盖手动设置→标记为P1同步Bug
