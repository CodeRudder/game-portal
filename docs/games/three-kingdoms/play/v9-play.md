# v9.0 离线收益 — 玩游戏流程

## 1. 离线收益系统

### 1.1 离线计算核心
- 操作: 玩家下线 → 系统立即记录快照(资源产出/建筑队列/科技队列/远征队列/商队状态) → 玩家上线 → 离线≤5min静默入账不弹窗 / 离线>5min弹出收益弹窗
- 验证: 快照记录完整覆盖所有产出源；基础收益 = 净产出速率 × 离线秒数；效率系数 = 衰减系数 × (1 + Σ加成)；最终收益 = 基础收益 × 效率系数
- 预期: 离线计算公式 `净产出 × 秒数 × 衰减 × (1+加成)` 正确执行，快照72h有效期
- PRD引用: [SPEC-offline OFR-1 核心公式](../ui-design/prd/SPEC-offline.md#ofr-1-1)

### 1.2 六档衰减系数
- 操作: 分别测试离线 1h / 5h / 15h / 36h / 60h / 80h → 对比各档衰减系数 → 验证封顶72h
- 验证: 0~2h→100% | 2~8h→80% | 8~24h→60% | 24~48h→40% | 48~72h→25% | >72h→15%(封底值，不再继续衰减，但超出72h部分不产出额外收益)
- 预期: 衰减分档平滑递减，72h后收益封顶不新增，温和引导玩家2天内回归
- PRD引用: [SPEC-offline OFR-1 基础衰减系数表](../ui-design/prd/SPEC-offline.md#ofr-1-2)
- 注意: 衰减系数采用分段计算，每个时段独立应用对应系数后累加（见1.7分段计算验证）

### 1.3 加成系数叠加
- 操作: 提升科技加成(离线收益I +10% / II +15% / III +20%，累计+30%) → 激活VIP加成(VIP1 +5% / VIP3 +10% / VIP5 +20%，累计+20%) → 提升声望加成(声望Lv1 +3% / Lv3 +5% / Lv5 +10% / Lv10 +25%，累计+25%) → 验证叠加上限
- 验证: 三项加成加法累加最大 +75%；继续叠加时上限封顶 +100%；最终效率系数 = 衰减 × (1 + min(加成总和, 100%))
- 预期: 加成来源清晰可查，上限+100%防止收益膨胀，付费与免费玩家差距可控
- PRD引用: [SPEC-offline OFR-1 加成系数](../ui-design/prd/SPEC-offline.md#ofr-1-3)

### 1.4 离线收益弹窗与翻倍
- 操作: 离线>5min后上线 → 弹窗展示离线时长/效率系数/各资源收益/来源占比饼图 → 数字滚动动画(800ms) → 选择翻倍方式
- 验证: 弹窗数据与后台计算一致；广告翻倍(3次/天)和元宝翻倍(无限制)二选一不可叠加；领取后资源入账
- 预期: 回归第一眼有获得感，翻倍选项满足免费/付费两类玩家，饼图帮助理解收益来源
- PRD引用: [SPEC-offline OFR-3 翻倍机制](../ui-design/prd/SPEC-offline.md#ofr-3)

### 1.5 广告翻倍细节
- 操作: 离线≥30分钟 → 上线 → 弹窗中点击「观看广告×2」→ 观看30秒激励视频 → 收益×2
- 验证: 离线<30分钟时广告翻倍按钮置灰不可用；广告加载失败时自动降级为1.5倍补偿；每日3次计数跨日重置(0点)
- 预期: 30分钟门槛防止滥用；广告失败有保底补偿，不因广告SDK问题影响玩家体验
- PRD引用: [SPEC-offline OFR-3 广告翻倍](../ui-design/prd/SPEC-offline.md#ofr-3)

### 1.6 元宝翻倍细节
- 操作: 离线收益弹窗 → 点击「消耗XX元宝×2」→ 验证元宝扣除 → 收益×2
- 验证: 元宝消耗 = ceil(离线收益总价值 / 500) × 10，最低10元宝，最高200元宝；元宝不足时按钮置灰+提示充值；每日无使用次数限制
- 预期: 元宝消耗与收益成正比，上限200防止过度消费；与广告翻倍互斥，同一次收益只能选一种翻倍方式
- PRD引用: [SPEC-offline OFR-3 元宝翻倍](../ui-design/prd/SPEC-offline.md#ofr-3)

### 1.7 分段计算验证
- 操作: 离线10h → 验证分段计算: 时段1(0~2h)=基础速率×7200×100% + 时段2(2~8h)=基础速率×21600×80% + 时段3(8~10h)=基础速率×7200×60% → 三段累加得总收益
- 验证: 每个时段独立应用对应衰减系数后累加；min(离线秒数-前段累计, 本段上限)确保不越界；>72h部分不参与计算
- 预期: 分段计算确保各时段收益准确，不会出现跨档计算误差
- PRD引用: [RES-resources-prd 离线收益计算公式](../ui-design/prd/RES-resources-prd.md#res-4)

## 2. 离线经验

### 2.1 经验产出快照
- 操作: 下线前记录当前经验产出速率(主城等级+科技加成+VIP加成) → 离线后上线 → 验证经验收益
- 验证: 离线经验 = 基础经验速率 × 离线秒数 × 衰减系数 × (1+经验加成)；受72h封顶约束
- 预期: 经验离线收益与在线产出比例合理，衰减后仍有可观获得感

### 2.2 经验溢出与等级提升
- 操作: 低等级玩家长时间离线 → 上线领取大量经验 → 验证是否触发升级 → 检查升级奖励邮件
- 预期: 离线经验可推动升级；升级奖励通过邮件系统自动发放；回归综合面板提示等级变化

## 3. 离线资源

### 3.1 各系统离线行为
- 操作: 下线前启动建筑升级+科技研究+远征+商队运输 → 离线超过完成时间 → 上线验证各系统状态
- 验证: 资源产出×1.0 | 建筑升级×1.2(加速完成) | 科技研究×1.0 | 远征×0.85(保守结算) | 商队自动完成交易
- 预期: 各系统按独立效率修正结算，建筑有加成鼓励下线前排队，远征保守防损失
- PRD引用: [SPEC-offline OFR-1 各系统离线效率修正](../ui-design/prd/SPEC-offline.md#ofr-1)

### 3.2 资源溢出处理
- 操作: 仓库接近上限时离线 → 上线领取收益 → 验证有上限资源截断+提示升级仓库
- 验证: 有上限资源(粮草截断至粮仓容量、兵力截断至兵营容量)收益截断至仓库上限，弹窗提示溢出量+引导升级；无上限资源(铜钱∞、天命∞)全额发放
- 预期: 溢出提示清晰，引导玩家扩容而非"白挂"，减少负面体验
- ⚠️ 注意: 铜钱无上限(∞)，不会截断；粮草和兵力有仓库上限才会截断
- PRD引用: [SPEC-offline OFR-2 资源溢出规则](../ui-design/prd/SPEC-offline.md#ofr-2) | [RES-resources-prd 溢出规则](../ui-design/prd/RES-resources-prd.md#res-4)

### 3.3 离线收益预估
- 操作: 打开预估面板 → 拖动滑块选择预估时长(2h/8h/24h/48h/72h) → 查看各资源预估收益+效率系数+是否溢出提示
- 验证: 预估数据与实际计算公式一致；效率系数随时长递减可视化；72h后显示封顶提示；基于当前产出速率和加成系数实时计算
- 预期: 帮助玩家决策"现在下线能拿多少"，降低离线焦虑
- PRD引用: [SPEC-offline OFR-2 离线收益预估面板](../ui-design/prd/SPEC-offline.md#ofr-2)

### 3.4 自动推图离线行为
- 操作: 开启自动推图后下线 → 离线超过推图时间 → 上线验证推图进度
- 验证: 自动推图效率系数 = 全局系数 × 0.80；仅推进已三星通关关卡；离线推图仅获得一星评价；奖励约为在线的60%
- 预期: 离线推图效率低于在线，鼓励玩家在线推新图，离线自动刷已通关关卡
- PRD引用: [SPEC-offline OFR-1 各系统离线效率修正](../ui-design/prd/SPEC-offline.md#ofr-1)

### 3.5 事件系统离线行为
- 操作: 下线前有未处理事件 → 离线期间事件继续堆积 → 上线验证事件处理
- 验证: 事件系统不自动处理，最多堆积5个事件；超过5个时超时事件以保守方案自动处理(取最低奖励)；限时事件正常计时不过期
- 预期: 事件不自动处理防止玩家错过决策，堆积上限防止回归后事件过多造成负担
- PRD引用: [SPEC-offline OFR-2 各系统离线行为](../ui-design/prd/SPEC-offline.md#ofr-2)

### 3.6 NPC系统离线行为
- 操作: 下线前有NPC交互进行中 → 离线后上线 → 验证NPC状态
- 验证: NPC正常刷新周期不受离线影响；好感度不衰减；限时NPC正常计时(可能离线期间消失)；回归面板提示即将消失的限时NPC
- 预期: NPC系统离线期间自然运行，限时NPC提醒帮助玩家不错过机会
- PRD引用: [SPEC-offline OFR-2 各系统离线行为](../ui-design/prd/SPEC-offline.md#ofr-2)

### 3.7 商店离线补货
- 操作: 下线前商店商品已售罄 → 离线超过刷新周期 → 上线验证商店补货
- 验证: ShopSystem离线补货最多累积2次；补货商品与正常刷新一致
- 预期: 离线不导致商店资源完全浪费，但累积上限2次防止囤积
- PRD引用: [Plan v9.0 改进现有子系统 ShopSystem](../plans/v9.0-离线收益.md)

## 4. 回归奖励

### 4.1 回归综合面板
- 操作: 领取离线收益后 → 进入Step2回归综合面板 → 查看建筑/科技/远征/事件/NPC/版本更新六类摘要
- 验证: 建筑升级完成列表含名称/等级/完成时间；科技研究完成列表同格式；远征归来摘要含战利品；自动推图进度展示；待处理事件含限时NPC提醒；离线期间有版本更新时显示更新摘要
- 预期: 面板可跳过但不遗漏信息，帮助玩家快速了解离线期间变化并决策下一步
- PRD引用: [SPEC-offline OFR-3 Step2回归综合面板](../ui-design/prd/SPEC-offline.md#ofr-3)

### 4.2 回归流程时间预算
- 操作: 离线>5min后上线 → 测量完整回归流程耗时
- 验证: Step1弹窗弹入+数字滚动 ~1.1s → 玩家阅读+操作 ~2s → 领取动画+关闭 ~0.9s → Step2面板淡入+操作 ~1.2s → 总计约5.2s
- 预期: 回归流程简洁高效，5秒内可完成核心操作，不阻碍玩家快速进入游戏
- PRD引用: [SPEC-offline OFR-3 回归流程时间预算](../ui-design/prd/SPEC-offline.md#ofr-3)

## 5. 邮件系统

### 5.1 邮件面板与分类
- 操作: 打开邮件面板 → 切换分类Tab(全部/系统📋/战斗⚔️/社交👥/奖励🎁) → 验证各分类邮件正确归类
- 验证: 系统邮件(古铜金#C6A456)含维护通知/版本更新/活动通知；战斗邮件(朱砂红#B8372D)含攻城战报/竞技场结算；社交邮件(松石绿#4A8C6F)含好友赠礼/联盟邮件/好友申请；奖励邮件(明金色#D4A843)含补偿/活动奖励/成就奖励/签到奖励
- 预期: 分类清晰，色彩标识直观，玩家快速定位目标邮件
- PRD引用: [MAL-mail-prd MAL-1 邮件分类定义](../ui-design/prd/MAL-mail-prd.md#mal-1-2)

### 5.2 邮件状态流转
- 操作: 接收新邮件(未读) → 点击查看(已读未领) → 领取附件(已读已领) → 等待过期(已过期)
- 验证: 未读=标题加粗+红点；已读未领=「待领取」标签；已读已领=透明度60%+「已领取」标签；已过期=透明度30%+灰色叠加+「已过期」戳记
- 预期: 四态流转清晰，玩家一眼识别邮件状态
- PRD引用: [MAL-mail-prd MAL-1 邮件状态](../ui-design/prd/MAL-mail-prd.md#mal-1)

### 5.3 邮件附件领取
- 操作: 打开邮件详情 → 点击「领取全部附件」→ 验证物品入账
- 验证: 铜钱/经验/元宝/兵力直接入账不占背包；强化石/装备/武将碎片/招募令入背包；背包满时附件不可领取，邮件保持「已读未领」状态+提示「背包已满，请清理后领取」
- 预期: 货币类直接入账体验流畅，背包满有明确提示不丢失奖励
- PRD引用: [MAL-mail-prd MAL-3 领取规则](../ui-design/prd/MAL-mail-prd.md#mal-3)

### 5.4 邮件批量操作
- 操作: 邮件列表 → 勾选多封邮件 → 一键领取全部附件 / 删除已读已领邮件 / 全部标记已读
- 验证: 一键领取领取所有附件但不删除邮件；删除仅限已读已领状态邮件；未领附件邮件不可删除；全选可跨类型选择
- 预期: 批量操作减少操作负担，防止误删未领取奖励
- PRD引用: [MAL-mail-prd MAL-2 批量操作规则](../ui-design/prd/MAL-mail-prd.md#mal-2)

### 5.5 邮件容量管理
- 操作: 邮箱接近满载 → 验证容量提示和暂存机制
- 验证: 0~70封正常(绿色容量条) → 71~90封警告(黄色+提示清理) → 91~99封拥挤(红色脉动+强烈提示) → 100封满载(红色闪烁+新邮件暂存队列最多20封+清理后补发)
- 预期: 容量管理温和引导玩家清理，满载不丢失邮件仅暂存
- PRD引用: [MAL-mail-prd MAL-2 容量上限](../ui-design/prd/MAL-mail-prd.md#mal-2)

### 5.6 邮件过期与自动清理
- 操作: 等待邮件过期 → 验证过期提醒和自动清理
- 验证: 系统邮件30天/战斗邮件7天/社交邮件14天/奖励邮件30天保留；奖励邮件过期前3天黄色⚠️标签→过期前1天登录Toast提醒→过期当天红色边框闪烁(5秒/次)；每日05:00自动清理过期邮件；奖励邮件过期后铜钱/经验补发50%到新邮件，道具/装备不补发
- 预期: 过期提醒分级递进，重要奖励有50%保底补偿减少损失感
- PRD引用: [MAL-mail-prd MAL-2 过期时间规则](../ui-design/prd/MAL-mail-prd.md#mal-2)

### 5.7 邮件自动触发规则
- 操作: 触发各种游戏事件 → 验证对应邮件自动生成
- 验证: 服务器维护→系统邮件+补偿附件；版本更新→系统邮件；活动开启/结束→系统邮件；攻城战结算→战斗邮件+战利品附件；竞技场每日21:00结算→战斗邮件+排名奖励；好友赠送兵力→社交邮件+兵力附件；成就达成→奖励邮件+成就奖励附件；离线收益→奖励邮件+离线产出附件
- 预期: 各系统事件自动触发邮件，玩家不遗漏任何奖励和通知
- PRD引用: [MAL-mail-prd MAL-2 发送条件](../ui-design/prd/MAL-mail-prd.md#mal-2)

### 5.8 离线收益邮件特殊规则
- 操作: 离线不同时长后上线 → 验证离线收益邮件标题和内容
- 验证: 离线<30分钟不生成邮件；30分钟~2小时标题「离线收益·短暂小憩」100%效率；2~6小时「离线收益·充分休整」100%效率；6~12小时「离线收益·深度养精」80%效率；12~24小时「离线收益·久别重逢」60%效率；>24小时「离线收益·主公归来」50%效率(上限48小时)
- 预期: 邮件标题有温度感，效率系数与离线时长匹配
- ⚠️ **消歧结论(已确认)**: MAL-3中的离线收益效率系数(5档展示值)与SPEC-offline的6档衰减(实际计算值)为两套独立体系。**实际收益统一按SPEC-offline 6档分段衰减计算**，邮件标题中的效率系数为简化展示值(向下取整到最近的MAL-3档位)，仅用于文案展示，不影响实际收益。详见§7.14消歧验证。
- PRD引用: [MAL-mail-prd MAL-3 离线收益邮件特殊规则](../ui-design/prd/MAL-mail-prd.md#mal-3)

## 6. 活动系统离线联动

### 6.1 活动离线积分累积
- 操作: 参与赛季/限时/日常/节日活动 → 下线 → 上线验证活动积分和代币累积
- 验证: 赛季活动50%效率累积积分+代币；限时活动30%效率；日常活动100%效率；节日活动50%效率；离线效率系数=基础效率×(1+声望等级×0.03)；每日上限: 赛季12h/限时8h/日常24h/节日12h
