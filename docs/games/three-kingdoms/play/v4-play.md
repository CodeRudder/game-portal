# v4.0 攻城略地(下) — 玩游戏流程

## 1. 战斗系统

### 1.1 战斗场景
- 操作: 从v3.0关卡列表选择关卡→点击「出征」进入战斗
- 验证: 战斗场景加载完成，我方与敌方单位正确显示在战场上
- 预期: 回合制战斗正常进行，武将自动攻击，伤害数字飘出

### 1.2 速度控制
- 操作: 战斗中点击倍速按钮，切换1x/2x/4x
- 验证: 战斗动画速度随倍速档位实时变化
- 预期: 1x正常节奏，2x动画加速，4x动画加速+简化特效，战斗节奏显著加快

### 1.3 大招系统
- 操作: 半自动模式下，等待武将怒气条充满→触发时停→点击「释放」
- 验证: 大招时停生效，战斗暂停等待玩家操作；释放后大招动画播放，伤害计算正确
- 预期: 时停效果触发期间可从容决策，大招释放后敌方受到大量伤害，暴击数字高亮显示

#### 1.3.1 大招时停详细规则
- 触发条件: 同时≥2个武将大招就绪时触发时停
- 时停效果: 战斗速度降至0.3×（慢动作），非完全暂停
- 最大持续: 5秒
- 结束条件: 所有大招释放完毕 / 5秒超时 / 玩家点击跳过
- 超时处理: 未释放的大招自动排队释放
- 验证: 单个大招就绪时不触发时停，≥2个大招同时就绪才触发

#### 1.3.2 战斗模式自动切换
- 操作: 观察不同战力差距下的战斗模式自动选择
- 验证:
  - 我方战力 ≥ 推荐战力 ×1.43（即推荐战力 ≤ 我方 ×0.7）→ 全自动模式
  - 推荐战力 ≤ 我方 ×1.0 → 半自动模式（含大招时停）
  - 推荐战力 > 我方 ×1.0 → 全手动模式
- 预期: 系统根据战力差距自动选择合适的战斗模式，玩家也可手动切换

### 1.4 武技特效
- 操作: 战斗中观察武将释放技能时的视觉表现
- 验证: 技能释放时产生粒子/光效特效，不同技能有不同特效风格
- 预期: 特效表现丰富且不影响战斗流畅度；4x加速时特效简化，避免性能问题

### 1.5 伤害数字动画
- 操作: 战斗中观察伤害/治疗/暴击数字的表现
- 验证:
  - 普通伤害: 白色数字飘出
  - 暴击伤害: 红色大字+高亮效果
  - 治疗数字: 绿色数字飘出
  - 不同倍速下数字显示策略不同（1x全显示，2x仅暴击/技能，4x仅暴击）
- 预期: 数字动画清晰可读，高倍速时信息精简但不丢失关键反馈

### 1.6 跳过战斗
- 操作: 战斗中点击「跳过」按钮
- 验证: 跳过战斗动画，直接进入结算界面
- 预期: 根据双方战力计算胜负结果，结算界面正确显示奖励与星级评定

#### 1.6.1 跳过战斗详细流程
- 操作: 在不同战斗场景下测试跳过战斗功能
- 验证:
  | 场景 | 跳过条件 | 跳过后处理 | 验证点 |
  |------|---------|-----------|--------|
  | 普通关卡 | 战斗开始后任意时刻 | 根据战力比即时计算胜负，显示结算界面 | 胜负结果与手动战斗一致 |
  | 攻城战 | 战斗开始后任意时刻 | 城防值按战力差计算损耗，判定是否归零 | 城防归零→占领；未归零→失败 |
  | 自动推图 | 自动触发跳过 | 全自动模式无需手动跳过，直接结算 | 自动推图每场均跳过战斗动画 |
  | 碾压模式(战力≥1.43倍) | 自动跳过 | 直接显示胜利结算 | 无需任何操作，瞬间结算 |
  - 跳过后星级评定: 根据剩余兵力比例计算（≥80%→3星，≥50%→2星，>0%→1星）
  - 跳过后奖励: 与手动战斗相同掉落表，奖励数量一致
  - 跳过按钮: 战斗界面右上角，战斗开始3秒后显示（防止误触）
- 预期: 跳过战斗功能在所有场景下正确计算结果，不丢失奖励，不影响星级评定

### 1.7 手机端战斗布局
- 操作: 在手机端(375×667)进入战斗场景
- 验证:
  - 技能按钮尺寸适配触摸操作（≥44×44px）
  - 战斗信息布局合理，不遮挡关键战斗画面
  - 大招释放按钮在时停期间突出显示
- 预期: 手机端战斗体验流畅，触摸操作精准

### 1.8 兵种克制验证
- 操作: 编队中配置不同兵种武将，观察对克制/被克制敌人的伤害差异
- 验证:
  - 骑兵→步兵: 伤害×1.5（克制）
  - 步兵→枪兵: 伤害×1.5（克制）
  - 枪兵→骑兵: 伤害×1.5（克制）
  - 被克制方向: 伤害×0.7
  - 弓兵/谋士: 无特殊克制关系
- 预期: 兵种克制关系正确影响最终伤害，克制方明显占优

## 2. 扫荡系统

### 2.1 扫荡解锁条件
- 操作: 进入关卡列表，查看已三星通关的关卡
- 验证: 三星通关关卡显示「扫荡」按钮，未三星通关关卡扫荡按钮置灰不可用
- 预期: 扫荡功能仅在满足三星通关条件时解锁

### 2.2 扫荡操作
- 操作: 选择已三星关卡→点击「扫荡」→选择扫荡次数(1/3/5/10)→确认
- 验证: 扫荡执行，不进入战斗场景，直接显示收益汇总
- 预期: 资源/经验/武将碎片正确获得，收益数量与手动战斗一致

### 2.3 扫荡令消耗
- 操作: 执行扫荡时检查扫荡令数量
- 验证: 每次扫荡消耗1个扫荡令，扫荡令不足时提示获取途径
- 预期: 扫荡令通过每日任务/商店购买获取，消耗数量与扫荡次数匹配

#### 2.3.1 扫荡令获取途径
| 来源 | 数量 | 说明 |
|------|:----:|------|
| 每日登录 | 3枚 | 每日固定赠送 |
| 任务奖励 | 不定 | 完成特定任务 |
| 商店购买 | 10元宝/枚 | 无购买上限 |
| VIP5+每日免费 | 3次 | 不消耗扫荡令 |
- 验证: 各获取途径均能正确获得扫荡令，库存数量实时更新

### 2.4 扫荡产出规则
- 操作: 对比手动三星通关与扫荡的奖励
- 验证:
  - 扫荡奖励按★★★倍率（×2.0）计算
  - 批量扫荡结果合并显示（如"粮草 ×7500"）
  - 稀有掉落额外高亮+金色光效
  - 使用相同掉落表，扫荡仅跳过战斗过程
- 预期: 扫荡产出与手动三星通关产出一致，批量扫荡汇总清晰

### 2.5 元宝替代扫荡
- 操作: 扫荡令不足时，选择使用元宝替代扫荡令
- 验证: 元宝扣除正确，扫荡正常执行
- 预期: 扫荡令不足时可使用元宝替代，不影响扫荡流程

## 3. 自动推图

### 3.1 开启自动推图
- 操作: 在关卡界面开启「自动推图」开关
- 验证: 系统自动选择当前最远可挑战关卡→自动进入战斗→自动结算
- 预期: 持续推进关卡进度，无需手动操作，资源持续增长

### 3.2 自动推图停止条件
- 操作: 观察自动推图在以下情况的行为
  - 战斗失败: 自动停止并提示战力不足
  - 体力不足: 自动停止并提示补充体力
  - 关卡全部通关: 自动停止并提示已完成
- 验证: 每种停止条件均正确触发，不会卡死或无限循环
- 预期: 失败后不盲目重试，资源耗尽时合理停止

### 3.3 自动推图战斗模式
- 操作: 观察自动推图过程中的战斗模式选择
- 验证:
  - 自动选择全自动战斗模式
  - 我方战力 ≥ 推荐战力 ×1.2 时才自动推图（确保碾压）
  - 战斗结束后自动进入下一场
  - 可随时手动中断
- 预期: 自动推图在碾压条件下高效运行，非碾压时不盲目推图

## 4. 武将升星

### 4.1 碎片收集
- 操作: 通过招募重复武将/关卡掉落/商店兑换获取武将碎片
- 验证: 碎片正确计入对应武将的碎片进度
- 预期: 碎片进度可视化显示当前数量/所需数量

#### 4.1.1 碎片获取途径详细验证
| 途径 | 说明 | 验证点 |
|------|------|--------|
| 招募重复武将 | 抽到已有武将自动转化为碎片 | Uncommon→5片、Rare→10片、Epic→20片、Legendary→40片、Mythic→80片 |
| 关卡掉落 | 特定关卡掉落特定武将碎片 | 每次掉落1~3片，碎片归属正确 |
| 商店兑换 | 每日刷新碎片商品 | 消耗代币正确，碎片到账正确 |
| 活动奖励 | 限时活动碎片 | 活动期间碎片正确发放 |
| 远征掉落 | 远征关卡随机碎片 | 碎片随机但合理 |

### 4.2 碎片进度可视化
- 操作: 查看武将详情面板中的碎片进度条
- 验证:
  - 0~50%: 蓝色进度条
  - 50~80%: 紫色进度条
  - 80~100%: 金色脉冲进度条+提示即将升星
  - 显示"当前碎片数/所需碎片数"（如"35/80"）
- 预期: 碎片进度直观展示，接近升星时有明显视觉提示

### 4.3 升星操作
- 操作: 武将碎片足够+铜钱足够→点击「升星」
- 验证: 消耗碎片和铜钱，升星动画播放（碎片汇聚+光芒爆发），星级+1
- 预期: 武将属性大幅提升，品质上限突破，等级上限解锁

#### 4.3.1 升星消耗与效果详细验证
| 星级 | 消耗碎片 | 效果 | 验证点 |
|:----:|:-------:|------|--------|
| 1→2星 | 20 | 全属性+10% | 属性数值正确提升 |
| 2→3星 | 40 | 全属性+10%，技能等级上限+1 | 技能可继续升级 |
| 3→4星 | 80 | 全属性+10%，解锁新被动 | 新被动技能出现 |
| 4→5星 | 150 | 全属性+10%，技能强化 | 技能效果数值提升 |
| 5→6星 | 300 | 全属性+10%，解锁终极技能强化 | 终极技能强化解锁 |

### 4.4 突破系统
- 操作: 武将等级达到突破节点（Lv.10/20/30/40）时进行突破
- 验证:
  - 突破1（Lv.10）: 消耗碎片×20+铜钱×2000，全属性+5%，解锁被动技能强化
  - 突破2（Lv.20）: 消耗碎片×40+铜钱×5000，全属性+8%，解锁新技能
  - 突破3（Lv.30）: 消耗碎片×80+铜钱×12000，全属性+12%，技能等级上限+1
  - 突破4（Lv.40）: 消耗碎片×150+铜钱×30000，全属性+15%，解锁终极技能强化
- 预期: 突破前等级无法继续提升，突破后解锁新的等级上限和技能强化

### 4.5 升星与突破的关联
- 操作: 观察升星后武将等级上限的变化
- 验证: 星级达到阈值后解锁等级上限，需先突破才能继续升级
- 预期: 升星→突破→继续升级的养成循环正确运转

## 5. 科技系统

### 5.1 科技树浏览
- 操作: 进入科技界面，查看三条科技路线（军事/经济/文化）
- 验证: 节点+连线+状态正确渲染（未解锁/可研究/研究中/已完成/互斥锁定）
- 预期: 科技树全景图清晰展示依赖关系与当前进度

#### 5.1.1 科技树结构验证
- 操作: 检查科技树的整体结构
- 验证:
  - 三条路线: 军事(红/刀剑图标) / 经济(黄/铜钱图标) / 文化(紫/书卷图标)
  - 每条路线7个层级节点，第5~7层为互斥分支（A/B二选一）
  - 4个融合科技节点（跨路线）
  - 总计25个科技节点（21基础+4融合）
  - 画布支持缩放（50%~150%）和拖拽平移
- 预期: 科技树结构完整，节点状态准确，依赖关系连线清晰

#### 5.1.2 节点状态验证
| 状态 | 视觉表现 | 说明 |
|------|---------|------|
| 未解锁 | 灰色 | 前置科技未完成 |
| 可研究 | 亮色 | 满足前置条件，资源足够 |
| 研究中 | 动画效果 | 正在研究，显示进度 |
| 已完成 | 金色 | 研究完成，效果生效 |
| 互斥锁定 | 红色 | 因选择另一分支而永久锁定 |

### 5.2 科技研究
- 操作: 选择可研究节点→检查前置+互斥+资源→确认研究
- 验证: 消耗铜钱+科技点，进入研究队列，倒计时开始
- 预期: 研究完成后效果自动注册，军事/经济/文化加成正确应用

#### 5.2.1 研究启动条件验证
| 条件 | 说明 | 验证点 |
|------|------|--------|
| 科技点足够 | 研究消耗科技点 = 基础费用 × (1 - 书院研究速度加成) | 科技点扣除正确 |
| 前置科技已研究 | 前置节点必须已完成 | 未完成前置时无法启动 |
| 无其他正在研究的科技 | 默认1个研究槽位 | 队列满时提示升级 |

#### 5.2.2 科技点系统
- 操作: 观察科技点的产出与消耗
- 验证:
  - 书院等级决定科技点产出速率（Lv.1: 0.3/s → Lv.20: 8.0/s）
  - 科技点无上限，持续累积（离线也累积）
  - 研究消耗 = 基础费用 × (1 - 书院研究速度加成)
- 预期: 科技点产出稳定，消耗合理，书院升级显著提升产出

#### 5.2.3 研究队列规则
- 操作: 测试研究队列的各项规则
- 验证:
  - 默认1个研究槽位
  - VIP3解锁第2研究槽位
  - 当前研究完成后，队列中下一项自动开始
  - 取消研究: 返还80%已消耗资源，研究进度清零
  - 切换研究: 消耗100铜钱暂停当前研究，切换到另一项（进度保留）
- 预期: 研究队列管理灵活，取消/切换惩罚合理

#### 5.2.4 加速机制
- 操作: 在科技研究过程中使用不同加速方式
- 验证:
  | 加速方式 | 效果 | 消耗 | 限制 |
  |---------|------|------|------|
  | 铜钱加速 | 立即完成50% | 铜钱×剩余时间(秒)×0.5 | 无限制 |
  | 天命加速 | 立即完成80% | 天命×⌈剩余时间/3600⌉ | 每次研究最多1次 |
  | 元宝秒完成 | 立即完成全部 | 元宝×⌈剩余时间/1800⌉ | 每日限3次 |
  | 书院等级加速 | 研究速度+2%~+38%（被动） | 无 | 永久 |
  | 声望等级加速 | 研究速度×(1+声望等级×0.02) | 无 | 永久 |
- 预期: 加速效果正确叠加（加法叠加后乘法应用），消耗计算准确

#### 5.4.1 军事路线效果

### 10.0A 领土产出→科技点入账验证
```
占领领土 → 领土产出计算 → 科技点产出 → 书院累积 → 科技研究消耗
```
- 操作: 占领一块新领土（尤其是长安地标），观察科技点产出变化
- 验证:
  1. 领土产出公式: 总产出 = 基础产出 × 地形加成 × 阵营加成 × 科技加成 × 声望加成 × 地标加成
  2. 占领长安地标 → 科技点产出+30%立即生效，科技点产出速率数值增加
  3. 科技点在书院持续累积（无上限），产出速率 = 基础速率 × (1 + 领土科技点加成 + 声望等级加成)
  4. 科技研究消耗 = 基础费用 × (1 - 书院研究速度加成)，从累积科技点中扣除
  5. ResourcePointSystem正确接入领土科技点产出，占领/失去领土时产出实时更新
- 预期: 领土→科技点→科技研究形成正循环，占领更多领土→更多科技点→更快研究→更强攻城
