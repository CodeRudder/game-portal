### 修复详情

#### P2-4: 任务跳转按钮

**实现方式**：
- 从`questDef.jumpTarget`读取跳转路由
- 点击"前往→"按钮时尝试调用`engine.getRouter().navigate()`
- 路由不可用时显示Toast提示
- 仅在任务未完成时显示跳转按钮

**代码位置**：`QuestPanel.tsx` — 每个任务卡片的奖励预览下方

#### P2-7: Badge红点通知

**实现方式**：
- 使用`useMemo`计算所有活跃任务中`status === 'completed' && !rewardClaimed`的数量
- 当`claimableCount > 0`时在面板顶部显示红色提示条
- 提示条包含精确的可领取数量

**代码位置**：`QuestPanel.tsx` — 面板顶部活跃度条上方

#### P2-8: 引擎初始化默认任务

**实现方式**：
- 新增`initializeDefaults()`公共方法
- 自动接受`quest-main-001`（初入乱世）
- 自动调用`refreshDailyQuests()`刷新日常任务
- 不在`init()`中自动调用，避免影响测试

**代码位置**：`QuestSystem.ts` — 新增方法

#### P2-9: 刷新时自动领取奖励

**修复前**：已完成未领取的任务被标记`expired`后直接删除，奖励丢失
**修复后**：
```typescript
if (instance.status === 'completed' && !instance.rewardClaimed) {
  instance.rewardClaimed = true;
  instance.status = 'expired';
  deps.emit('quest:autoClaimed', { instanceId, questId, reason: 'daily_refresh' });
}
deps.activeQuests.delete(id);
```

#### P2-10: 多样性保证算法

**算法流程**：
1. D01（每日签到）必定出现
2. 从保证类别（battle/training/auto）中各选至少1个
3. 填充剩余名额，每类最多2个
4. 使用Fisher-Yates洗牌保证均匀分布

**代码位置**：`QuestSystem.helpers.ts` — `pickDailyWithDiversity()`
