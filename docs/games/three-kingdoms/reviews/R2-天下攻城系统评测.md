### 修复详情

#### P0-1: 攻城结果弹窗

**新增文件**: `src/components/idle/panels/map/SiegeResultModal.tsx`

**功能特性**：
- 胜利/失败/无法攻城三种状态，不同图标和颜色
- 目标信息：城池名称、原属势力
- 战损统计：出征兵力、消耗粮草、兵力损失（30%）
- 获得奖励：资源、领土经验、道具掉落（按稀有度着色）
- 使用 SharedPanel 统一弹窗容器

**代码链路**：
```
WorldMapTab.handleSiegeConfirm → siegeSystem.executeSiege → SiegeResult
→ setSiegeResultData(result) → SiegeResultModal visible=true
```

#### P0-3: 攻城资源直接扣减

**修复方式**：在 `SiegeSystem.resolveSiege()` 中新增 `deductSiegeResources()` 方法：

```typescript
private deductSiegeResources(cost: SiegeCost): void {
  try {
    const resourceSys = this.deps?.registry?.get<any>('resource');
    if (resourceSys) {
      if (cost.troops > 0) resourceSys.consume?.('troops', cost.troops);
      if (cost.grain > 0) resourceSys.consume?.('grain', cost.grain);
    }
  } catch { /* 资源系统不可用时静默处理 */ }
}
```

- 胜利时：扣减全部兵力消耗 + 粮草消耗
- 失败时：扣减30%兵力损失 + 全部粮草消耗

#### P1-5: autoGarrison事件消费

**修复方式**：在 `GarrisonSystem.init()` 中注册事件监听：

```typescript
deps.eventBus?.on?.('siege:autoGarrison', (data) => {
  this.handleAutoGarrison(data);
});
```

`handleAutoGarrison` 验证领土归属后发出 `garrison:autoGarrisoned` 确认事件。

#### 小地图缩略图

在 WorldMapTab 右侧信息面板新增缩略图区域：
- 6×4 CSS Grid 布局
- 每个领土用12×12像素方块表示
- 按归属着色：player=绿色、enemy=红色、neutral=灰色
- 选中领土高亮边框
- 点击缩略图跳转到对应领土

#### 攻城每日次数显示

在统计卡片下方新增攻城信息面板：
- 显示"今日攻城"剩余次数
- 次数>0时绿色显示，次数=0时红色显示"已用完"
