### 修复代码质量分析

**删除编队确认弹窗（P1-01）**：
```tsx
// FormationPanel.tsx — 删除确认流程
const handleDeleteRequest = (id: string) => {
  const f = formationSystem.getFormation(id);
  setDeleteConfirm({ visible: true, formationId: id, formationName: f?.name ?? '编队' });
};

// Modal组件 — danger类型 + dangerConfirm标记
<Modal type="danger" title="删除编队" dangerConfirm ...>
  <p>确定要删除编队「{deleteConfirm.formationName}」吗？</p>
  <p style={{ color: '#ff6b6b' }}>此操作不可撤销，编队中的武将将被移出。</p>
</Modal>
```
✅ 危险操作使用红色警告样式（`type="danger"` + `dangerConfirm`）
✅ 警告文案具体："此操作不可撤销，编队中的武将将被移出"
✅ 弹窗中显示编队名称，避免误删

**移除武将确认弹窗（P1-02）**：
```tsx
// FormationPanel.tsx — 移除武将确认流程
const handleRemoveHeroRequest = (fid: string, gid: string) => {
  const hero = heroSystem.getGeneral(gid);
  setRemoveConfirm({ visible: true, formationId: fid, heroId: gid, heroName: hero?.name ?? gid });
};

<Modal type="warning" title="移除武将" dangerConfirm ...>
  <p>确定要将「{removeConfirm.heroName}」从编队中移除吗？</p>
</Modal>
```
✅ 使用 `type="warning"` 黄色警告样式（比删除的danger轻一级）
✅ 显示武将名称，操作对象明确

**编队扩展机制（P1-03）**：
```tsx
// FormationPanel.tsx — 扩展配置
const FORMATION_EXPANSION_CONFIG = [
  { slot: 4, cost: { gold: 2000, jade: 100 }, label: '第4编队位' },
  { slot: 5, cost: { gold: 5000, jade: 300 }, label: '第5编队位' },
];

// 引擎层 — 动态编队上限
setMaxFormations(max: number): void {
  this.maxFormations = Math.min(Math.max(max, MAX_FORMATIONS), 5);
}

// 扩展流程：检查资源 → 确认弹窗 → 扣除资源 → 更新localStorage → 同步引擎
```
✅ 渐进式解锁：第4位 → 第5位，不可跳级
✅ 资源消耗合理：2000金+100玉 / 5000金+300玉，与游戏经济匹配
✅ localStorage持久化，刷新不丢失
✅ 引擎层 `setMaxFormations` 动态调整，向后兼容
✅ 资源不足时按钮禁用 + 显示"资源不足"

## 二、R3 苏格拉底式10D评分

### D1 可发现性（Discoverability）— 8.5/10 ↑0.5

> **苏格拉底之问**：新玩家能否快速找到编队功能并理解其用途？

**改善点**：
- ✅ 编队扩展区域使用🔓图标 + "扩展编队位"标题，功能意图明确
- ✅ 创建编队按钮始终可见（顶部右侧）
- ✅ 空状态提示"尚无编队，点击「创建编队」开始组建"
- ⚠️ 编队嵌套在武将Tab内，入口层级仍较深

**评分依据**：扩展区域的视觉引导和空状态提示改善了可发现性，但入口嵌套问题未变。

### D2 可理解性（Understandability）— 9.5/10 ↑0.5

> **苏格拉底之问**：编队的前排/后排、羁绊、扩展机制是否容易理解？

**改善点**：
- ✅ 删除确认弹窗明确说明"此操作不可撤销，编队中的武将将被移出"
- ✅ 扩展确认弹窗展示解锁后编队上限（"解锁后可创建最多 N 个编队"）
- ✅ 扩展区域显示具体资源消耗（💰 2000 金 · 💎 100 玉）
- ✅ 前排/后排标签、羁绊标签、战力数字持续清晰

**评分依据**：确认弹窗的文案设计精准，既说明了操作后果又提供了决策信息。

### D3 可操作性（Operability）— 9.5/10 ↑1.0

> **苏格拉底之问**：创建、编辑、删除编队的操作是否安全且高效？

**改善点**：
- ✅ 删除编队：从"一键直接删除"变为"确认弹窗→确认/取消"，防误操作
- ✅ 移除武将：从"一键直接移除"变为"确认弹窗→确认/取消"，防误操作
- ✅ 编队扩展：资源不足时按钮禁用，避免无效操作
- ✅ 一键编队按钮（🤖）提供快速编队方案
- ✅ 阵容收藏（FormationSaveSlot）支持保存/加载编队配置

**评分依据**：确认弹窗将"危险操作"变为"安全操作"，编队扩展将"固定上限"变为"渐进解锁"，操作体验质变。

### D4 反馈性（Feedback）— 9.0/10 ↑1.0

> **苏格拉底之问**：每次编队操作后，玩家能否清楚知道发生了什么？

**改善点**：
- ✅ 删除确认弹窗本身就是操作反馈（"你正在删除编队X"）
- ✅ 移除武将确认弹窗显示武将名称
- ✅ 扩展确认弹窗展示资源消耗详情
- ✅ 扩展成功后编队上限实时更新（`useEffect` 同步引擎）
- ⚠️ 删除/移除操作成功后无Toast提示（依赖UI状态变化作为反馈）

**评分依据**：确认弹窗提供了"操作前反馈"，但缺少"操作后反馈"（Toast）。扣0.5分。

### D5 完整性（Completeness）— 9.5/10 ↑0.5

> **苏格拉底之问**：编队系统的功能闭环是否完整？

**闭环验证**：
- ✅ 创建（前置条件+资源消耗）→ 编辑（添加/移除武将+确认）→ 激活 → 删除（确认弹窗）
- ✅ 编队扩展（渐进解锁+资源消耗+持久化）
- ✅ 一键编队（自动选将+防御排序）
- ✅ 阵容收藏（保存/加载/删除）
- ✅ 羁绊预览（阵营羁绊+搭档羁绊+潜力提示）

**评分依据**：编队系统的功能闭环在R3已非常完整，从创建到扩展到收藏全覆盖。

### D6 数据合理性（Data Rationality）— 9.5/10 ↑0.5

> **苏格拉底之问**：编队扩展的资源消耗是否合理？战力计算是否准确？

**改善点**：
- ✅ 扩展资源消耗渐进递增：第4位（2000金+100玉）→ 第5位（5000金+300玉），2.5倍递增合理
- ✅ 引擎层 `maxFormations` 上限硬编码为5，防止无限扩展
- ✅ 战力计算包含羁绊加成（每羁绊+5%），R2已验证
- ✅ localStorage持久化解锁数量，数据不丢失

**评分依据**：扩展机制的数值设计合理，渐进递增+硬上限的组合既提供了成长感又防止了数值膨胀。

### D7 前置条件（Prerequisites）— 9.5/10 ↑0.5

> **苏格拉底之问**：创建编队和扩展编队位的前置条件是否清晰？

**改善点**：
- ✅ 创建编队需主城≥3级+铜钱500（R2已修复）
- ✅ 扩展编队位需资源充足（实时检查余额）
- ✅ 资源不足时按钮禁用+显示"资源不足"
- ✅ 扩展确认弹窗展示消耗详情，让玩家在确认前了解代价
- ✅ 渐进式解锁：必须先解锁第4位才能解锁第5位

**评分依据**：前置条件系统设计优秀，从创建到扩展都有清晰的条件检查和反馈。

### D8 错误处理（Error Handling）— 9.5/10 ↑1.0

> **苏格拉底之问**：误操作和异常场景是否得到妥善处理？

**改善点**：
- ✅ 删除编队确认弹窗防止误删（最重要的错误预防）
- ✅ 移除武将确认弹窗防止误操作
- ✅ 扩展编队位确认弹窗防止误消费
- ✅ 资源不足时按钮禁用，从源头阻止无效操作
- ✅ 创建编队防抖（`isCreatingRef`）防止重复创建
- ✅ 删除当前激活编队后自动切换到第一个可用编队

**评分依据**：三层防护（按钮禁用+确认弹窗+引擎层校验）将误操作风险降到最低。

### D9 连贯性（Coherence）— 9.5/10 持平

> **苏格拉底之问**：编队系统与其他系统的联动是否流畅？

**维持点**：
- ✅ 编队扩展与资源系统联动（检查余额+扣除资源）
- ✅ 编队与羁绊系统联动（羁绊预览+战力加成）
- ✅ 编队与派遣系统联动（标注已派遣武将）
- ✅ 编队与阵容收藏联动（保存/加载配置）
- ✅ 引擎层 `setMaxFormations` 动态同步

**评分依据**：编队扩展建立了与资源系统的新联动，进一步强化了系统间连贯性。

### D10 重复可玩性（Replayability）— 9.0/10 ↑1.0

> **苏格拉底之问**：编队系统是否鼓励玩家反复调整和优化？

**改善点**：
- ✅ 编队扩展（3→5个编队位）增加了编队配置的多样性
- ✅ 阵容收藏支持保存多套配置，鼓励尝试不同阵容
- ✅ 羁绊预览（包括潜力提示"还差X名Y阵营武将"）驱动武将收集
- ✅ 一键编队降低了重新配置的成本
- ✅ 确认弹窗让玩家更敢于尝试（误操作有保障）

**评分依据**：编队扩展从3个增至5个是长期可玩性的重要提升，多编队配置增加了策略深度。
