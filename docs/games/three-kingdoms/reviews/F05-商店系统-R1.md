## 二、流程追问记录

### 一、主流程追问

**Q1: 玩家如何进入商店？**
→ 通过底部Tab栏"更多"菜单或直接入口进入商店面板
→ 追问1a: 商店入口在TabBar中是否直接可见？
  - 代码溯源: 商店面板以`SharedPanel`形式弹出，不是独立Tab。需通过"更多"菜单或其他途径触发
  - 结论: ⚠️ 商店入口不在一级Tab栏中，可发现性受限
→ 追问1b: 商店是否需要解锁条件？
  - 代码溯源: `shop-config.ts` → `SHOP_LEVEL_CONFIG`，初级集市需主城Lv1
  - 结论: 初始即可使用 ✅
→ 追问1c: 商店面板打开时触发什么引擎操作？
  - 代码溯源: `ShopPanel.tsx` → `shopSystem.getShopGoods(activeTab)` 获取商品列表
  - 结论: 打开时加载商品数据 ✅

**Q2: 进入商店后看到什么？**
→ 四个商店Tab（杂货铺/竞技商店/远征商店/联盟商店）+ 货币余额栏 + 排序工具栏 + 商品列表
→ 追问2a: 四个Tab的名称是否准确？
  - 代码溯源: `ShopPanel.tsx:62-67` → `SHOP_TABS` 定义为杂货铺(normal)/竞技商店(black_market)/远征商店(limited_time)/联盟商店(vip)
  - ⚠️ Tab标签与ShopType语义不匹配：`black_market`对应"竞技商店"（应为黑市），`limited_time`对应"远征商店"（应为限时特惠），`vip`对应"联盟商店"（应为VIP商店）
  - 结论: Tab标签语义混乱，存在命名不一致 ⚠️
→ 追问2b: 商品列表数据从哪来？
  - 代码溯源: `ShopSystem.ts` → `getShopGoods(shopType)` → 从`shops[shopType].goods`读取
  - 初始数据: `ShopSystem` 构造函数 → `emptyShop()` → `restockShop()` 生成初始商品
  - 结论: 引擎初始化时创建商品 ✅
→ 追问2c: 商品为空时显示什么？
  - 代码溯源: `ShopPanel.tsx` → 空状态判断 `sortedGoods.length === 0` → 显示"🏪 商店系统未加载"或"暂无商品"

### 四、数据链路追问

**D1: 铜钱消耗链路**
```
产生: CurrencySystem初始值100,000（测试中设置）
→ 存储: CurrencySystem内部balance映射
→ 消耗: ShopSystem.executeBuy() → CurrencySystem.spendByPriority(shopType, totalCost)
→ 效果: 铜钱减少 → 商品库存减少 → 限购计数增加
→ 关联: 铜钱减少影响建筑升级、武将招募等系统
```

**D2: 商品库存链路**
```
产生: ShopSystem初始化 → restockShop() → 根据GOODS_DEF_MAP生成商品实例
→ 存储: ShopState.goods[] → GoodsItem { stock, dailyPurchased, lifetimePurchased }
→ 消耗: executeBuy() → stock -= quantity
→ 重置: resetDailyLimits() → dailyPurchased归零
→ 关联: 库存影响购买可用性，限购影响每日策略
```

**D3: 折扣价格链路**
```
产生: 商品定义basePrice + 折扣率(discount/NPC好感度/活跃折扣)
→ 计算: calculateFinalPrice() → min(itemDiscount, npcRate, activeRate) × basePrice
→ 展示: UI显示原价删除线 + 折后价
→ 扣费: executeBuy() 使用finalPrice × quantity
→ 关联: 折扣影响购买决策和资源消耗
```
