## 二、问题清单

### P0 — 阻塞性问题（必须修复）

> 本次评测未发现P0级阻塞问题。

### P1 — 重要问题（强烈建议修复）

| # | 问题 | 详情 | 影响范围 |
|---|------|------|---------|
| P1-1 | **executeBattle使用简化战力估算** | PvPBattleSystem.executeBattle()的战力估算为 `score×10 + heroCount×1000 + 5000`，不调用真实BattleEngine。胜率计算基于战力差简单公式，非真实战斗模拟 | 战斗结果与实际武将属性/技能/阵型无关，PvP体验缺乏深度 |
| P1-2 | **缺少战前布阵界面** | PRD PVP-2定义了战前布阵（双方阵容对比/战力分析/阵型选择/战斗模式选择），ArenaPanel直接挑战无布阵步骤 | 玩家无法调整进攻阵容和阵型 |
| P1-3 | **缺少防守编队编辑UI** | PRD PVP-4定义了防守阵容设置面板（阵型预览/武将列表/AI策略/防守记录），ArenaPanel仅展示防守日志，无编队编辑入口 | 玩家无法通过UI修改防守阵容 |
| P1-4 | **缺少竞技商店UI** | PRD PVP-1-4定义了竞技商店弹窗（分类Tab/商品网格/限购/购买），ArenaShopSystem已实现，但ArenaPanel无商店入口 | 玩家无法使用竞技币兑换商品 |
| P1-5 | **缺少战斗回放面板** | PRD PVP-3-2定义了战斗回放面板（记录列表/回放播放/倍速），PvPBattleSystem有replays数据，但UI无回放入口 | 玩家无法回顾战斗过程 |
| P1-6 | **段位积分范围与PRD不一致** | PRD定义青铜V为0~300分，引擎RANK_LEVELS中青铜V为0~299分（maxScore:299）。PRD白银V为2001~2500，引擎为1500~1799。段位积分映射存在系统性偏差 | 段位判定与PRD设计不符 |

### P2 — 优化建议

| # | 问题 | 详情 | 影响范围 |
|---|------|------|---------|
| P2-1 | **缺少赛季结算弹窗** | PRD PVP-3-1定义了赛季结算弹窗（排名/段位变化/奖励/统计），ArenaSeasonSystem已实现settleSeason，但UI无结算展示 | 赛季结束时玩家无法直观看到成果 |
| P2-2 | **缺少战力分析对比** | PRD PVP-2定义了四维对比条（攻击/防御/速度/谋略），UI未实现 | 玩家无法评估对手强弱 |
| P2-3 | **对手卡片信息不够丰富** | PRD定义对手卡片含头像/排名/战力/胜率/阵营/难度星级，ArenaPanel仅显示名称/战力/排名 | 信息量不足以支撑策略决策 |
| P2-4 | **缺少防守成功额外奖励** | PRD定义连续防守成功5次额外奖励竞技币×20，引擎ArenaSystem.addDefenseLog未实现连续成功计数 | 防守激励机制缺失 |
| P2-5 | **ArenaPanel中段位解析不完整** | parseRank函数使用RANK_META映射，但缺少PLATINUM/MASTER等PRD中不存在的段位，与引擎21级段位定义可能不匹配 | 段位显示可能不准确 |
| P2-6 | **缺少半自动战斗模式** | PRD定义了半自动模式（普攻自动/大招手动），引擎PvPBattleMode有SEMI_AUTO枚举但无实现 | 战斗操作深度不足 |

| 排名范围 | ±5~±20 | rankMinOffset:5/rankMaxOffset:20 | ✅ |
| 候选对手数 | 3人 | candidateCount:3 | ✅ |
| 免费刷新间隔 | 30分钟 | freeIntervalMs:1800000 | ✅ |
| 手动刷新消耗 | 500铜钱 | manualCostCopper:500 | ✅ |
| 每日免费挑战 | 5次 | dailyFreeChallenges:5 | ✅ |
| 购买单价 | 50元宝 | buyCostGold:50 | ✅ |
| 胜利积分 | +30~+60 | winMinScore:30/winMaxScore:60 | ✅ |
| 失败积分 | -15~-30 | loseMinScore:15/loseMaxScore:30 | ✅ |
| 防守加成 | 全属性+5% | defenseBonusRatio:0.05 | ✅ |
| 超时判定 | 防守方胜 | timeoutWinner:'defender' | ✅ |
| 青铜V积分 | 0~300 | 0~299 | ❌ 差1分 |
| 白银V积分 | 2001~2500 | 1500~1799 | ❌ 系统性偏差 |
| 黄金V积分 | 5001~5800 | 3000~3399 | ❌ 系统性偏差 |

| 段位大段 | 青铜/白银/黄金/钻石/王者 | BRONZE/SILVER/GOLD/DIAMOND/KING | ✅ |

| 战斗回放保留 | 7天 | retentionMs:7×24×60×60×1000 | ✅ |

> ⚠️ **段位积分偏差说明**：引擎的段位积分范围与PRD存在系统性差异。PRD使用较大的积分区间（如青铜V为300分跨度），引擎使用更紧凑的区间（如青铜V为300分但maxScore=299，白银从1500开始而非2001）。这导致相同积分在PRD和引擎中可能对应不同段位。
