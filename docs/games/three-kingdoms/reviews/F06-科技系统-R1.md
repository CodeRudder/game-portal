## 二、流程追问记录

### 一、主流程追问

**Q1: 玩家如何进入科技Tab？**
→ 底部Tab栏点击"科技"按钮（📜图标）
→ 追问1a: 科技Tab是否需要解锁条件？
  - 代码溯源: `tech-config.ts` → 书院(Academy)建筑需要主城Lv3解锁，科技系统依赖书院
  - 测试辅助: `createTechSim()` → 先升级主城到Lv3再升级书院
  - 结论: 需要主城Lv3+书院Lv1才能使用科技系统 ✅ 合理的渐进解锁
→ 追问1b: 未解锁时科技Tab是否可见？
  - 代码溯源: TabBar配置中科技Tab `available: true`
  - 结论: Tab始终可见但内容可能为空 ⚠️ 应有解锁提示

**Q2: 进入科技Tab后看到什么？**
→ 三条路线Tab（军事/经济/文化）+ 科技点信息栏 + 科技树画布 + 研究队列面板
→ 追问2a: 三条路线的内容是什么？
  - 代码溯源: `tech-config.ts` → 军事(8节点4层)、经济(8节点4层)、文化(8节点4层) = 共24个科技节点
  - 结论: 内容量适中，三条路线各有特色 ✅
→ 追问2b: 科技点信息栏显示什么？
  - 代码溯源: `TechTab.tsx` → `.tk-tech-points-bar` 显示科技点数量和产出速率
  - 结论: 显示当前科技点和产出速率 ✅
→ 追问2c: 科技树画布的布局方式？
  - 代码溯源: `TechTab.tsx` → PC端显示三条路线列（`tech-path-military/economy/culture`），移动端可能不同
  - 结论: PC端三列布局 ✅

**Q3: 科技节点的状态有哪些？**
→ locked / available / researching / completed
→ 追问3a: 状态角标是否直观？
  - 代码溯源: `FLOW-06-07` → available=🔓, locked=🔒
  - 结论: 角标直观 ✅ 但"🔓"表示"可研究"语义不太明确，建议改为更直观的图标
→ 追问3b: Tier1节点初始状态是什么？
  - 代码溯源: `tech-config.ts` → Tier1节点无前置(`prerequisites: []`)，初始应为available
  - 结论: Tier1初始可研究 ✅
→ 追问3c: Tier2+节点如何变为available？
  - 代码溯源: `TechTreeSystem.ts` → 前置节点completed后，后续节点变为available
  - 结论: 前置条件驱动解锁 ✅

**Q4: 玩家如何研究一个科技？**
→ 点击节点 → 打开详情弹窗 → 点击"开始研究" → 消耗科技点 → 进入研究队列
→ 追问4a: 研究需要消耗什么？
  - 代码溯源: `tech-config.ts` → `costPoints`（科技点）+ `researchTime`（研究时间）
  - Tier1: 50点/120秒, Tier2: 120点/300秒, Tier3: 300点/720秒, Tier4: 800点/1800秒
  - 结论: 双重消耗（科技点+时间）✅
→ 追问4b: 科技点不足时能否研究？
  - 代码溯源: `TechResearchSystem.ts` → `startResearch()` → 检查科技点余额
  - 结论: 不足时无法研究 ✅
→ 追问4c: 研究队列有大小限制吗？
  - 代码溯源: `tech-config.ts` → `BASE_RESEARCH_QUEUE_SIZE = 1`, `ACADEMY_QUEUE_SIZE_MAP` → 书院等级决定队列大小（Lv1=1, Lv5=2, Lv10=3...）
  - 结论: 队列大小随书院等级增长 ✅

**Q5: 研究完成后发生什么？**
→ 节点状态变为completed → 科技效果生效 → 解锁后续节点

### 四、数据链路追问

**D1: 科技点链路**
```
产生: 书院建筑产出（ACADEMY_TECH_POINT_PRODUCTION）+ 金币兑换
→ 存储: TechPointSystem内部state.current
→ 消耗: startResearch() → trySpend(costPoints)
→ 返还: cancelResearch() → refund(refundPoints)
→ 关联: 书院等级→产出速率，科技点余额→可研究科技
```

**D2: 科技节点状态链路**
```
产生: TechTreeSystem.init() → 所有节点初始为locked/available
→ 变更: startResearch() → researching; tick完成 → completed
→ 效果: completed → getAllCompletedEffects() → 影响战斗/资源系统
→ 解锁: completed → 后续节点prerequisites满足 → available
→ 互斥: completed → 同mutexGroup节点 → locked
```

**D3: 研究队列链路**
```
产生: startResearch() → 加入队列
→ 推进: tick() → 减少剩余时间
→ 完成: remainingTime <= 0 → 节点completed → 效果生效
→ 取消: cancelResearch() → 移出队列 → 返还科技点
→ 限制: 队列大小 = ACADEMY_QUEUE_SIZE_MAP[academyLevel]
```
