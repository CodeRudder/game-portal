#### Fix #1：空防守阵容引导提示（P1-1）

**首次进入引导Banner**：
```tsx
// ArenaPanel.tsx
const isDefenseEmpty = useMemo(() => {
  const slots = ps?.defenseFormation?.slots;

useEffect(() => {
  if (visible && isDefenseEmpty) {
    setShowDefenseGuide(true);
  }

// 引导Banner UI
{showDefenseGuide && isDefenseEmpty && (

      <button onClick={() => { setShowDefenseGuide(false); setShowDefenseEdit(true); }}>
        立即设置
      </button>
      <button onClick={() => setShowDefenseGuide(false)}>稍后</button>
    </div>
  </div>
)}
```

**低胜率策略建议**：
```tsx
{defenseStats && defenseStats.totalDefenses >= 5 && defenseStats.winRate < 0.4 && !isDefenseEmpty && (
  <div className="tk-arena-strategy-tip">
    <span>💡</span>

      <b>{STRATEGY_LABELS[defenseStats.suggestedStrategy]}</b>策略
    </span>
    <button onClick={() => setShowDefenseEdit(true)}>调整</button>
  </div>

```

**验证结论**：✅ 修复完整且超出预期。

1. **首次引导**：空阵容时自动弹出引导Banner，含"立即设置"快捷按钮直接跳转编队编辑
2. **低胜率建议**：防守≥5次且胜率<40%时显示策略建议，含"调整"快捷按钮
3. **视觉设计**：Banner使用金色渐变+脉冲动画，策略建议使用黄色高亮+呼吸动画
4. **双按钮设计**："立即设置"（主操作）+"稍后"（次要操作），符合UX最佳实践

**苏格拉底追问**：引导Banner是否会在每次打开面板时重复弹出？
→ 是的，只要`isDefenseEmpty`为true且`visible`为true，就会显示。这是合理的行为——空防守阵容是严重问题，应该持续提醒直到解决。用户可以点"稍后"临时关闭，但下次打开仍会提示。

**苏格拉底追问**：低胜率策略建议的触发条件是否合理？
→ `totalDefenses >= 5`确保有足够样本量，`winRate < 0.4`是合理的低胜率阈值。`!isDefenseEmpty`避免与引导Banner冲突。`suggestedStrategy`来自引擎的智能推荐，设计合理。

#### Fix #2：战斗摘要弹窗（P1-2）

**战斗评级系统**：
```tsx
function getBattleRating(result: any): { grade: string; label: string; color: string } {
  const turns = result?.totalTurns ?? 99;
  const won = result?.attackerWon ?? false;
  const scoreChange = result?.scoreChange ?? 0;

  if (!won) return { grade: 'D', label: '惜败', color: '#ff6464' };
  if (turns <= 3 && scoreChange >= 20) return { grade: 'S', label: '完美碾压', color: '#FFD700' };
  if (turns <= 5 && scoreChange >= 15) return { grade: 'A', label: '出色胜利', color: '#7EC850' };
  if (turns <= 7) return { grade: 'B', label: '稳健获胜', color: '#4FC3F7' };
  return { grade: 'C', label: '艰难取胜', color: '#d4a574' };
}
```

**战斗摘要弹窗UI**：
```tsx
<Modal visible={!!battleResult} type={battleResult?.attackerWon ? 'success' : 'danger'}
  title={battleResult?.attackerWon ? '🎉 挑战胜利' : '💔 挑战失败'}>
  <div className="tk-arena-battle-summary">
    {/* 评级区域：大字评级+描述 */}
    <div className="tk-arena-summary-rating">
      <span className="tk-arena-summary-grade" style={{ color: battleRating.color }}>
        {battleRating.grade}
      </span>
      <span className="tk-arena-summary-grade-label">{battleRating.label}</span>
    </div>
    {/* 核心数据网格：积分变化/回合数/是否超时/当前积分 */}
    <div className="tk-arena-summary-stats">...</div>
    {/* 战斗行动摘要：最近3条行动记录 */}
    <div className="tk-arena-summary-actions">...</div>
  </div>
</Modal>
```

**验证结论**：✅ 修复完整且设计出色。

1. **四级评级**：S（完美碾压）/ A（出色胜利）/ B（稳健获胜）/ C（艰难取胜）/ D（惜败），基于回合数+积分变化
2. **核心数据**：积分变化（绿色正/红色负）、战斗回合、是否超时、当前积分
3. **行动摘要**：展示最近3条战斗行动（攻击者→目标+伤害/治疗），超出3条显示"还有N条记录"
4. **视觉设计**：评级使用32px大字+text-shadow光晕，数据使用2列网格布局

**苏格拉底追问**：评级算法是否合理？
→ S级要求"≤3回合且积分+≥20"，这是很高的标准，符合"完美碾压"的语义。D级仅判断胜负，简单明了。B级和C级的区分（≤7回合）也合理。整体评级体系清晰且激励性强。

**苏格拉底追问**：战斗行动摘要是否增加了信息过载？
→ 只展示最近3条，且使用紧凑的行格式（"R1 我方 → 敌方 伤害500"），不会过载。超出部分折叠为"还有N条记录"，平衡了信息量和可读性。

**对手卡片增强**：

```tsx
  {oppRecord.slice(0, 5).map((r, i) => (

    </span>
  ))}

```

**验证结论**：✅ 修复完整且信息丰富。

1. **基础信息**：战力+积分+排名，三项核心指标
2. **阵容预览**：阵型名称+5个阵位（填充⚔️/空位·）+武将数量
3. **最近战绩**：最近5场胜负记录，绿色"胜"/红色"负"
4. **视觉设计**：卡片从简单行布局改为详细列布局（`tk-arena-opp-card--detailed`）

**苏格拉底追问**：阵容预览是否暴露了过多信息？
→ 只显示阵型名称和阵位填充状态（有/无武将），不显示具体武将名称和属性。这是合理的"情报"级别——知道对手用什么阵型、上了几个武将，但不知道具体阵容。增加了策略深度而不破坏平衡。
