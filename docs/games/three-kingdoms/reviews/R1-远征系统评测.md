## 二、问题清单

### P0 — 阻塞性问题（必须修复）

> 本次评测未发现P0级阻塞问题。

### P1 — 重要问题（强烈建议修复）

| # | 问题 | 详情 | 影响范围 |
|---|------|------|---------|
| P1-1 | **路线地图缺少树状分支可视化** | PRD EXP-1定义了树状分支路线结构（主线+支线），节点间有连线，含蛇形排列。ExpeditionPanel仅用列表展示路线和节点，缺少空间感和分支选择 | 玩家无法直观感知路线分支结构，降低探索乐趣 |
| P1-2 | **缺少编队选择界面** | PRD EXP-1-2定义了武将卡片横向排列、拖拽排序、点击更换的编队界面。ExpeditionPanel只展示已有队伍，无创建队伍/选择武将/选择阵型的UI | 玩家无法通过UI创建远征队伍，只能通过引擎API |
| P1-3 | **缺少远征配置弹窗** | PRD EXP-3定义了出发前配置确认弹窗（路线信息/推荐配置/消耗预览/自动选项），ExpeditionPanel直接派遣无确认步骤 | 玩家无法评估远征风险和收益 |
| P1-4 | **缺少远征结算详细展示** | PRD EXP-2定义了结算弹窗（战斗统计/奖励列表/队伍HP状态），ExpeditionPanel的战斗结果Modal仅显示节点名和"战斗胜利" | 玩家无法了解战斗过程和具体奖励 |
| P1-5 | **阵型效果未在UI展示** | PRD定义了6种阵型（鱼鳞/鹤翼/锋矢/雁行/长蛇/方圆）各有不同加成，引擎FormationType已定义，但UI未展示阵型选择和加成信息 | 玩家无法理解阵型对战斗的影响 |
| P1-6 | **ExpeditionPanel使用内联样式混合CSS类** | 面板部分使用内联样式`s`对象，部分使用CSS类`tk-expedition-*`，风格不统一 | 代码维护性差 |

### P2 — 优化建议

| # | 问题 | 详情 | 影响范围 |
|---|------|------|---------|
| P2-1 | **缺少路线难度星级视觉标识** | PRD定义了难度星级（⭐~⭐⭐⭐⭐⭐），UI中仅用颜色区分难度，无星级图标 | 信息传达不够直观 |
| P2-2 | **缺少兵力恢复倒计时** | 兵力恢复为1点/5分钟，UI中只显示当前/最大兵力，无恢复倒计时 | 玩家不知道何时兵力恢复满 |
| P2-3 | **缺少一键远征分配逻辑UI** | PRD EXP-4定义了按队伍战力从高到低自动分配路线的逻辑，UI未实现 | 玩家需手动逐一分配 |
| P2-4 | **缺少远征日志面板** | PRD定义了远征日志（队伍/路线/时间/遭遇/结果），UI未实现 | 玩家无法回顾远征历史 |
| P2-5 | **缺少扫荡功能UI入口** | 引擎有executeSweep方法，但UI无扫荡按钮和扫荡类型选择 | 玩家无法使用扫荡功能 |
| P2-6 | **ExpeditionTab未在ACC中测试** | 存在ExpeditionTab.tsx但FLOW-14未测试Tab级别的渲染和交互 | Tab集成风险 |
