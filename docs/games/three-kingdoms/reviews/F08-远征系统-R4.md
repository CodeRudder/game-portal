### 修复详情分析

#### Fix #1：武将远征中锁定（P1-1）

**引擎层**：
```tsx
// 获取正在远征中的武将ID集合
const expeditioningHeroIds: Set<string> = useMemo(() => {
  return exp?.getExpeditioningHeroIds?.() ?? new Set<string>();
}, [exp, teams]);
```

**UI层**：
```tsx
// 编队弹窗中——远征中武将显示锁定标记
const isExpeditioning = expeditioningHeroIds.has(h.id);
// 锁定样式：红色边框+半透明+不可点击
<button style={isExpeditioning ? s.heroLockedChip : ...}
  onClick={() => !isExpeditioning && toggleHero(h.id)}
  disabled={isExpeditioning}>
  {h.name} ⚔️{h.power}
  {isExpeditioning && <span style={s.heroExpeditioningTag}>远征中</span>}
</button>
```

**锁定样式定义**：
```tsx
heroExpeditioningTag: {
  fontSize: 9, color: '#ff6464', marginLeft: 2,
  background: 'rgba(255,100,100,0.15)', borderRadius: 'var(--tk-radius-sm)',
},
heroLockedChip: {
  padding: '3px 8px', border: '1px solid rgba(255,100,100,0.2)',
  background: 'rgba(255,100,100,0.08)', color: '#ff6464',
  cursor: 'not-allowed', opacity: 0.5,
},
```

**验证结论**：✅ 修复完整。
1. 引擎层通过`getExpeditioningHeroIds()`获取远征中武将集合
2. UI层正确标记远征中武将（红色"远征中"标签）
3. 远征中武将不可选中（`disabled`+`cursor: not-allowed`+`opacity: 0.5`）
4. 视觉区分清晰：正常武将绿色边框，远征中红色边框+半透明

**苏格拉底追问**：如果引擎不支持`getExpeditioningHeroIds`会怎样？
→ 降级为空集合`new Set<string>()`，所有武将都可选。这是合理的降级行为，不会导致错误，但会失去锁定保护。

#### Fix #2：快速派遣（P1-2）

```tsx
// 获取上次派遣配置
const lastDispatchConfig = useMemo(() => exp?.getLastDispatchConfig?.() ?? null, [exp]);

// 快速重派按钮
{lastDispatchConfig && (
  <button onClick={handleQuickRedeploy}>
    ⚡ 快速重派（上次: {routes.find(r => r.id === lastDispatchConfig.routeId)?.name}）
  </button>
)}

// 快速重派处理
const handleQuickRedeploy = useCallback(() => {
  try {
    const ok = exp?.quickRedeploy?.();
    flash(ok ? '🚀 快速重派成功！' : '❌ 重派失败（无空闲队伍或路线不可用）');
  } catch (e: any) { flash(e?.message ?? '重派失败'); }
}, [exp, flash]);
```

**验证结论**：✅ 修复完整。
1. 记录上次派遣配置（队伍+路线）
2. 有历史配置时显示"⚡ 快速重派"按钮
3. 按钮文案显示上次路线名称，提供上下文
4. 成功/失败均有Toast反馈
5. 失败原因明确（"无空闲队伍或路线不可用"）

**苏格拉底追问**：快速重派是否考虑了上次路线已锁定/已完成的情况？
→ `quickRedeploy()`返回false时显示"重派失败"提示，但未区分具体失败原因。建议细化错误信息（如"上次路线已通关，请选择新路线"）。

#### Fix #3：远征进度条（P1-3）

**全局路线进度**：
```tsx
// 路线完成进度条
const overallPercentage = Math.round((clearedIds.size / routes.length) * 100);
// 显示: "X/Y 通关 (Z%)"
<div style={{ background: 'linear-gradient(90deg, #7EC850, #d4a574)', width: `${overallPercentage}%` }} />
```

**队伍节点进度**：
```tsx
// 每个队伍的节点进度
const teamProgress = t.isExpeditioning ? exp?.getTeamNodeProgress?.(t.id) : null;
// 显示: "📍 路线名 X/Y节点 (Z%)"
<div style={s.nodeProgressBar}>
  <div style={{ ...s.nodeProgressFill, width: `${teamProgress.percentage}%` }} />
</div>
```

**验证结论**：✅ 修复完整。
1. 全局进度条：显示"X/Y通关(Z%)"，渐变色进度条
2. 队伍进度条：显示"路线名 X/Y节点(Z%)"，每支队伍独立进度
3. 进度条使用`transition: width 0.3s ease`平滑过渡

**苏格拉底追问**：进度条是否在所有状态下正确显示？
→ 全局进度条始终显示（基于`clearedIds.size`），队伍进度条仅在`isExpeditioning`时显示。逻辑正确。
