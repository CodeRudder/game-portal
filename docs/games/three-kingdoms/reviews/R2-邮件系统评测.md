## 二、R2修复详情

### R2-1: UI布局一致性修复 ✅

**分类色左边框**:
```typescript
const CATEGORY_COLORS = {
  system: '#C6A456',    // 金色
  battle: '#B8372D',    // 红色
  social: '#4A8C6F',    // 绿色
  reward: '#D4A843',    // 黄色
  trade: '#7C4DFF',     // 紫色
  alliance: '#42A5F5',  // 蓝色
};
```
- 未读邮件：古铜金左边框（突出未读状态）
- 已读邮件：分类色左边框（区分邮件类型）
- 完全对齐PRD §MAL-1定义的分类色

**已读已领灰化效果**:
- `read_claimed` 状态邮件 opacity: 0.6
- 视觉上明确区分已处理和未处理邮件

**邮箱容量条**:
- 底部新增容量进度条
- 显示"邮箱容量 X/100"
- >80%时进度条变红色预警
- 宽度变化带0.3s过渡动画

### R2-2: 引擎初始化欢迎邮件 ✅

**修复位置**: `MailSystem.ts` → `init()` → `_initDefaultData()`

**欢迎邮件内容**:
- 标题：欢迎来到三国霸业！
- 内容：主公，欢迎来到乱世！招募武将、建设城池、征战天下，成就一番霸业！请查收新手礼包。
- 附件：💰 铜钱 ×1000 + 🎫 招贤令 ×5
- 仅首次初始化发送（`this.mails.size > 0` 时跳过）

**设计考量**:
- 1000铜钱足够进行数次建筑升级和武将升级
- 5招贤令可进行5次普通招募或1次高级招募（不够），引导玩家了解招募系统
- 欢迎邮件作为system类型，保留30天

### R2-3: 邮件删除确认机制 ✅

**两步确认流程**:
1. 首次点击删除按钮 → 按钮文字变为"⚠️ 确认删除？"，样式高亮
2. 3秒内再次点击 → 执行删除
3. 3秒超时 → 自动取消确认状态

**安全性**:
- 有附件的邮件仍可删除（引擎层已有附件保护机制）
- 删除后自动关闭详情面板
- Toast提示"邮件已删除"

### R2-E1: 资源名称翻译 ✅

**修复前**: `gold: 100`、`recruitToken: 5`
**修复后**: `💰 铜钱 ×100`、`🎫 招贤令 ×5`

新增 `RESOURCE_NAMES` 映射覆盖8种主要资源类型。
