## P0问题（必须修复）

### P0-1: 武将池品质分布不均 — COMMON/FINE 无武将定义

**问题**: GENERAL_DEFS 中10个武将全部为 RARE(1)/EPIC(4)/LEGENDARY(5)，无 COMMON 和 FINE 品质武将。当 rollQuality 抽到 COMMON/FINE 时，fallbackPick 降级到 RARE，导致：
- 普通招募 85% 的抽卡（COMMON 60% + FINE 25%）实际都降级到 RARE
- 保底计数器几乎不累积（RARE >= RARE 重置 tenPullPity）
- 招募体验"过于慷慨"，失去低品质武将的收集层次感

**修复建议**:
```typescript
// 在 GENERAL_DEFS 中补充 COMMON 和 FINE 品质武将
// 至少各2个，覆盖4个阵营
{ id: 'soldier_a', name: '民兵队长', quality: Q.COMMON, faction: 'shu', ... },
{ id: 'soldier_b', name: '黄巾力士', quality: Q.COMMON, faction: 'qun', ... },
{ id: 'guard_a', name: '校尉', quality: Q.FINE, faction: 'wei', ... },
{ id: 'guard_b', name: '都尉', quality: Q.FINE, faction: 'wu', ... },
```

### P0-2: 十连招募缺少价格激励

**问题**: TEN_PULL_DISCOUNT=1.0，十连招募消耗 = 10 × 单抽价格，无任何折扣。在大多数同类游戏中，十连通常有9折优惠或"十连送一"机制，激励玩家进行十连消费。

**修复建议**:
```typescript
// hero-recruit-config.ts
export const TEN_PULL_DISCOUNT = 0.9; // 十连九折
// 或保持1.0但赠送额外1抽（十连出11个结果）
```

## 亮点

### 🌟 1. 回调解耦设计精妙
HeroRecruitSystem/HeroLevelSystem 通过 `ResourceSpendFn`/`ResourceCheckFn` 回调解耦资源系统，实现了领域系统间的零依赖，DDD聚合根边界清晰。这一设计使得测试可以完全 mock 资源系统，也是测试覆盖率达到100%的关键。

### 🌟 2. 测试体系完善
238条测试用例覆盖引擎层和UI层，包括：
- 确定性RNG测试概率分布
- 保底计数器边界条件
- 序列化/反序列化往返一致性
- UI交互（筛选/排序/弹窗/按钮状态）
- 资源不足/满级/空列表等边界场景

### 🌟 3. 配置与逻辑彻底分离
hero-config.ts 和 hero-recruit-config.ts 严格遵循"零逻辑，只有常量和数据结构"原则，所有数值调整只需修改配置文件，无需触碰逻辑代码。

### 🌟 4. 建筑地图布局体验升级
从 card 列表改为地图式布局后，城池管理的沉浸感大幅提升：
- 绝对定位的8座建筑错落有致
- 升级中建筑蓝色脉冲动画
- 可升级建筑绿色光晕+弹跳指示器
- 右上角悬浮升级队列面板

### 🌟 5. 武将数据设计用心
10个武将覆盖蜀(5)/魏(3)/吴(1)/群(1)四个阵营，每个武将配有2个技能（含技能描述），技能类型涵盖 active/passive/faction，为后续战斗系统奠定了扎实基础。
