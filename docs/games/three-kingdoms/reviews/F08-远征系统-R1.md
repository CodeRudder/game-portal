## 二、流程追问记录

### 一、主流程追问

**Q1: 远征系统的整体结构是什么？**
→ 三大区域（虎牢关/汜水关/洛阳）× 多条路线 × 多种难度
→ 追问1a: 区域解锁条件？
  - 代码溯源: `expedition.types.ts` → `ExpeditionRegion { requiredLevel, requiredRegionId, requireHardClear }`
  - 虎牢关默认解锁，汜水关需前置区域通关
  - 结论: 渐进解锁设计合理 ✅
→ 追问1b: 路线难度分布？
  - 代码溯源: `FLOW-14-02` → EASY/NORMAL/HARD/EPIC/AMBUSH 五种难度

→ 追问1c: 节点类型有哪些？
  - 代码溯源: `expedition.types.ts` → `NodeType` = BATTLE/EVENT/TREASURE/REST/BOSS/SHOP/BANDIT/HAZARD = 8种
  - 结论: 节点类型丰富，远征体验多样 ✅

**Q2: 玩家如何创建远征队伍？**

### 四、数据链路追问

**D1: 远征队伍生命周期**
```
创建: createTeam() → ExpeditionTeam { id, heroIds, formation, troopCount, totalPower }
→ 派遣: dispatchTeam(teamId, routeId) → isExpeditioning=true
→ 推进: advanceNode() → currentNodeId变更
→ 完成: completeRoute() → isExpeditioning=false, 奖励发放
→ 回收: 队伍回到空闲状态
```

**D2: 奖励链路**
```
产生: 路线完成 → BASE_REWARDS[difficulty] + 首通奖励 + 评级加成
→ 计算: ExpeditionRewardSystem.calculateReward()
→ 发放: 资源/经验/装备掉落/远征币
→ 关联: 远征币→远征商店，装备→装备系统，经验→武将升级
```

**D3: 里程碑链路**
```
定义: MilestoneType 枚举
→ 追踪: ExpeditionState.achievedMilestones
→ 奖励: 里程碑达成→额外奖励
→ 关联: 里程碑影响区域解锁
```
