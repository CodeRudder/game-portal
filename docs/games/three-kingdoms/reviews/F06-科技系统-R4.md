### 修复详情分析

#### Fix #1：未解锁提示（P1-1）
```tsx
// TechTab.tsx
const isTechUnlocked = useMemo(() => {
  const castleLevel = engine.building.getLevel('castle');
  const academyLevel = engine.building.getLevel('academy');
  return castleLevel >= 3 && academyLevel >= 1;
}, [engine, snapshotVersion]);

// 未解锁时显示提示区域
{!isTechUnlocked && (
  <div className="tk-tech-locked-hint">
    <div className="tk-tech-locked-hint-icon">🔒</div>
    <div className="tk-tech-locked-hint-text">
      <p>科技系统未解锁</p>
      <p>升级主城至Lv3并建造书院后解锁科技系统</p>
    </div>
  </div>
)}
```
**验证结论**：✅ 修复完整。检测逻辑正确（主城≥Lv3 + 书院≥Lv1），提示文案明确告知解锁条件。CSS动画`tk-tech-locked-bounce`让🔒图标有轻微弹跳，增加视觉吸引力。

**苏格拉底追问**：如果引擎不支持`building.getLevel`会怎样？
→ 代码使用`try { ... } catch { return true; }`容错，默认解锁。这是合理的降级策略。

#### Fix #2：研究完成Toast通知（P1-2）
```tsx
// TechTab.tsx
useEffect(() => {
  const eventBus = engine.eventBus;
  if (!eventBus) return;
  const unsubscribe = eventBus.on('economy:techCompleted', (payload) => {
    if (data?.techName) {
      Toast.success(`${data.techName}科技研究完成！`);
    }
  });
  return unsubscribe;
}, [engine]);
```
**验证结论**：✅ 修复完整。通过事件总线监听科技完成事件，弹出成功Toast。

**苏格拉底追问**：事件名称`economy:techCompleted`是否覆盖所有路线？
→ 事件名使用了`economy:`前缀，这看起来像是经济路线的专属事件。如果军事路线和文化路线的研究完成使用不同事件名（如`military:techCompleted`），则只会有经济路线的Toast通知。建议统一为`tech:researchCompleted`或监听所有三条路线的事件。

⚠️ **潜在问题**：事件名前缀可能导致非经济路线的研究完成不触发Toast。

#### Fix #3：推荐标记（P1-3）
```tsx
// TechTab.tsx
const RECOMMENDED_TECH_IDS = new Set([
  'mil_t1_attack',    // 军事：锐兵术
  'eco_t1_farming',   // 经济：精耕细作
  'cul_t1_education', // 文化：兴学令
]);

// 节点渲染中
{RECOMMENDED_TECH_IDS.has(nodeDef.id) && status !== 'completed' && (
  <span className="tk-tech-recommend-tag">⭐ 推荐</span>
)}
```
**验证结论**：✅ 修复完整。每条路线推荐1个Tier1基础科技，覆盖三路线。推荐标记有金色光晕动画（`tk-tech-recommend-glow`），视觉醒目。

**苏格拉底追问**：推荐标记是否对已完成科技隐藏？
→ 是的，`status !== 'completed'`条件确保已完成科技不再显示推荐标记。
