### 四、数据链路追问

**D1: 竞技币链路**
```
产生: 赛季结算奖励 + 每日段位奖励 + 挑战胜利奖励
→ 存储: ArenaPlayerState.arenaCoins
→ 消耗: ArenaShopSystem.buyItem() → 扣除竞技币
→ 关联: 竞技币→竞技商店→武将碎片等
```

**D2: 积分/段位链路**
```
产生: 挑战胜利+积分，失败-积分
→ 存储: ArenaPlayerState.score, rankId
→ 变更: score达到阈值→段位升级/降级
→ 效果: 段位→每日奖励金额→赛季结算奖励
→ 重置: 每日重置不影响积分/段位，赛季结算可能重置段位
```

**D3: 挑战次数链路**
```
产生: 每日重置→dailyFreeChallenges=5
→ 消耗: consumeChallenge() → dailyChallengesLeft--
→ 补充: buyChallenge() → 花费50元宝+1次，每日最多5次
→ 上限: dailyFreeChallenges + dailyBuyLimit = 5 + 5 = 10次/日
```

## 七、架构亮点

1. **三层经济闭环**：挑战→竞技币→商店→武将碎片，形成完整PvP经济系统
2. **智能策略建议**：`getDefenseStats().suggestedStrategy` 根据胜率自动推荐防守策略
3. **双重刷新机制**：免费刷新（30分钟CD）+ 手动刷新（500铜钱/次），灵活且有限制
4. **防守日志系统**：最多50条日志 + 胜负统计 + 胜率计算，提供防守优化依据
5. **赛季+每日双奖励**：赛季结算大奖励 + 每日段位小奖励，短期+长期动力并存
6. **周重置限购**：竞技商店限购每周重置，控制经济通胀
7. **版本兼容**：反序列化版本不匹配时返回默认状态，防止数据损坏
8. **测试覆盖100%**：30个测试用例覆盖所有核心场景和边界条件
