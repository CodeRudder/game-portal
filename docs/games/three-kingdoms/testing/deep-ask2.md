你是三国霸业项目的QA架构师。 
**任务：建立一套完整的游戏流程评测方法论，包含两层——"深度流程追问"和"10维评测"，两者并存互补。** 
**项目根目录**：`/mnt/user-data/workspace/game-portal` 
### 方法论定位 
这套方法论用于逐个评测三国霸业游戏的每个功能模块，确保： 
1. 游戏流程没有断裂（玩家能真正玩通每个功能） 
2. 游戏数据链路完整（从产生到消耗到效果到关联影响） 
3. 集成测试不遗漏（每个步骤都有测试覆盖） 
### 方法论结构 
## 第一层：深度流程追问（穷举法） 
**目的**：确保每个功能的所有操作步骤、分支路径、异常场景都被枚举。 
**方法**：对每个功能，从玩家第一次接触开始，逐步追问，每一步追问"然后呢？"直到流程结束。每个分支也要追问。至少追问10步以上。 
**追问模板**（以"建筑升级"为例示范）： 
``` === 主流程追问 === 
Q1: 玩家如何进入建筑Tab？ 
→ 点击底部Tab栏的"建筑"按钮 
→ 追问1a: Tab栏是否始终可见？是否可能被遮挡？ 
→ 追问1b: 建筑Tab是否需要解锁？初始状态是否可点击？ 

Q2: 进入建筑Tab后看到什么？
→ 显示建筑列表/建筑地图 
→ 追问2a: 建筑列表数据从哪来？引擎的BuildingSystem？ 
→ 追问2b: 如果BuildingSystem未初始化，显示什么？ 
→ 追问2c: 建筑图标、名称、等级是否正确显示？ 

Q3: 玩家点击某个建筑后发生什么？ 
→ 弹出建筑详情/升级面板 
→ 追问3a: 面板显示哪些信息？（等级、产出、升级消耗、升级效果） 
→ 追问3b: 这些数据从哪来？引擎实时计算还是缓存？ 
→ 追问3c: 数据为空或异常时面板显示什么？ 

Q4: 玩家点击"升级"按钮后发生什么？ 
→ 检查资源是否足够 → 扣除资源 → 开始升级 
→ 追问4a: 资源不足时按钮是否禁用？是否有提示？ 
→ 追问4b: 资源扣除的精确值是什么？公式是什么？ 
→ 追问4c: 扣除后资源栏是否立即刷新？ 
→ 追问4d: 如果扣除失败（并发操作）怎么办？ 

Q5: 升级过程中玩家看到什么？ 
→ 升级进度条/等待时间 
→ 追问5a: 等待时间多长？是固定值还是随等级变化？ 
→ 追问5b: 等待期间玩家能否操作其他功能？ 
→ 追问5c: 等待期间能否加速？加速需要什么条件？ 

Q6: 升级完成后发生什么？
→ 建筑等级+1，产出增加 
→ 追问6a: 产出增加的精确值是多少？公式是什么？ 
→ 追问6b: 产出增加后资源栏是否立即刷新？ 
→ 追问6c: 是否有升级完成的动画/提示？ 

Q7: 升级完成后影响了哪些其他系统？ 
→ 追问7a: 是否触发任务进度更新？（如"升级建筑1次"任务） 
→ 追问7b: 是否触发成就检查？ 
→ 追问7c: 是否触发引导步骤推进？ 
→ 追问7d: 是否解锁新建筑或新功能？ 
→ 追问7e: 主城升级是否影响子建筑上限？ 

Q8: 升级操作是否被正确保存？ 
→ 追问8a: 自动保存还是手动保存？ 
→ 追问8b: 保存的数据结构是什么？ 
→ 追问8c: 重新加载后升级状态是否恢复？ 

Q9: 异常场景 
→ 追问9a: 升级过程中关闭游戏，重新打开后状态如何？ 
→ 追问9b: 升级过程中切换Tab，回来后状态如何？ 
→ 追问9c: 网络断开时升级操作是否丢失？ 
→ 追问9d: 存档数据损坏时建筑数据如何修复？ 

Q10: 重复操作场景 
→ 追问10a: 玩家能否同时升级多个建筑？ 
→ 追问10b: 升级队列是否有上限？ 
→ 追问10c: 快速连续点击升级按钮是否有防抖？ 
→ 追问10d: 升级到满级后按钮如何变化？ 

=== 数据链路追问 === 
Q11: 建筑升级消耗的金币从哪来？ 
→ 初始资源 + 产出 + 任务奖励 + 其他 
→ 追问11a: 初始金币是多少？ 
→ 追问11b: 金币产出速率是多少？受什么影响？ 
→ 追问11c: 金币上限是多少？超出怎么办？ 

Q12: 建筑升级后的产出增加如何计算？ 
→ 基础产出 × 等级系数 × 加成 
→ 追问12a: 等级系数的具体公式是什么？ 
→ 追问12b: 加成来源有哪些？（科技? 联盟? 领土?） 
→ 追问12c: 加成是否叠加？如何叠加？ 

Q13: 产出增加后影响了什么？ 
→ 资源总量增加 → 可用于其他操作 
→ 追问13a: 资源总量增加是否触发资源上限检查？ 
→ 追问13b: 资源总量增加是否影响商店价格？（动态定价？）
→ 追问13c: 资源总量增加是否影响任务条件判定？
``` 

 ## 第二层：10维评测（打分法） 
 **目的**：对每个功能的整体体验进行量化评估。 
 **10个维度**（每个1-10分）： 
 1. 可发现性：入口是否明显 
 2. 可理解性：功能是否易懂 
 3. 可操作性：操作是否顺畅 
 4. 反馈性：操作后是否有反馈 
 5. 完整性：流程是否闭环 
 6. 数据合理性：数值是否平衡 
 7. 前置条件：条件是否合理 
 8. 错误处理：异常是否友好 
 9. 连贯性：与其他功能衔接 
 10. 重复可玩性：是否有持续动力 
 
 **评分标准**： 
 - 9-10分：体验优秀，封版 
 - 7-8分：基本可用，有小问题需修复 
 - 4-6分：有明显问题，需重点修复 
 - 1-3分：严重断裂，功能不可用
