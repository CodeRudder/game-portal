## 二、游戏引擎需求

### 新增子系统
| 子系统 | 职责 | 依赖 |
|--------|------|------|
| SweepSystem | 扫荡逻辑：验证条件→计算奖励→批量结算 | CampaignProgressSystem |

## 三、数据流设计

```
战斗深化
  │
  ├─ 半自动模式
  │   ├─ 战斗正常自动进行
  │   ├─ 武将怒气满 → 大招时停 → 等待玩家确认释放
  │   └─ 玩家点击「释放」→ 播放大招动画 → 继续战斗
  │
  ├─ 战斗加速
  │   ├─ 1x: 正常速度
  │   ├─ 2x: 动画加速2倍（默认）
  │   └─ 4x: 动画加速4倍 + 简化特效
  │
扫荡系统
  │
  ├─ 扫荡条件检查
  │   ├─ 关卡已三星通关 ✓
  │   ├─ 扫荡令数量足够 ✓
  │   └─ 通过 → 进入扫荡
  │
  ├─ 扫荡执行
  │   ├─ 选择关卡 + 扫荡次数(1/3/5/10)
  │   ├─ SweepSystem: 批量计算奖励（不进入战斗）
  │   └─ 显示扫荡结果汇总
  │
武将升星
  │
  ├─ 碎片收集
  │   ├─ 招募重复武将 → 转化为碎片
  │   ├─ 关卡掉落 → 获得碎片
  │   └─ 商店兑换 → 消耗代币获得碎片
  │
  ├─ 升星流程
  │   ├─ 碎片数量足够 + 铜钱足够
  │   ├─ GeneralStarSystem: 消耗碎片 → 星级+1 → 属性大幅提升
  │   ├─ 升星动画（碎片汇聚+光芒爆发）
  │   └─ 突破：星级达到阈值 → 解锁等级上限
  │
科技系统
  │
  ├─ 科技树渲染
  │   ├─ TechTreeViewComponent: 3条路线并行显示
  │   ├─ 节点状态: 未解锁(灰)/可研究(亮)/研究中(动画)/已完成(金)/互斥锁定(红)
  │   └─ 连线: 显示前置依赖关系
  │
  ├─ 研究流程
  │   ├─ 选择可研究节点 → 检查前置+互斥+资源
  │   ├─ 消耗: 铜钱 + 科技点
  │   ├─ 进入研究队列 → 倒计时
  │   ├─ 研究完成 → TechEffectSystem注册效果
  │   └─ 效果应用到: 资源产出(经济) / 战斗属性(军事) / 经验获取(文化)
  │
  └─ 互斥分支
      ├─ 同层两个节点，选择A → B永久锁定
      └─ 显示锁定原因 + 推荐选择提示
```

- [ ] 升星消耗碎片+铜钱，属性正确大幅提升

- [ ] 突破机制生效：星级达到阈值解锁等级上限

- [ ] 互斥分支选择后另一节点永久锁定
- [ ] 科技研究消耗铜钱+科技点，倒计时完成后生效

- [ ] UI层分支覆盖率/通过率 100%
- [ ] 无控制台错误

### 可玩性验收
- [ ] 核心循环连通：扫荡关卡→获得碎片/资源→升星武将→战力提升→挑战更难关卡

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 科技树Canvas渲染复杂 | 节点+连线+状态渲染性能差 | 使用离屏Canvas预渲染静态部分，仅动态更新状态 |
| 互斥分支逻辑复杂 | 选择后状态管理出错 | 使用状态机管理节点状态，单元测试覆盖所有转换 |
| 科技效果全局影响 | 效果叠加计算不一致 | TechEffectSystem统一管理所有加成，各系统通过接口查询 |
| 扫荡奖励计算偏差 | 批量扫荡奖励与手动战斗不一致 | 使用相同掉落表，扫荡仅跳过战斗过程 |
| 武技特效性能 | 多个特效同时渲染卡顿 | 限制同屏特效数量，4x加速时简化特效 |
