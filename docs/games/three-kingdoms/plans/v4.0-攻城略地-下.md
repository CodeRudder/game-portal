## 四、数据流设计

```
战斗深化
  │
  ├─ 半自动模式
  │   ├─ 战斗正常自动进行
  │   ├─ 武将怒气满 → 大招时停 → 等待玩家确认释放
  │   └─ 玩家点击「释放」 → 播放大招动画 → 继续战斗
  │
  ├─ 战斗加速
  │   ├─ 1x: 正常速度
  │   ├─ 2x: 动画加速2倍（默认）
  │   └─ 4x: 动画加速4倍 + 简化特效
  │
  ├─ 兵种克制
  │   ├─ 骑兵→步兵: ×1.5（克制）
  │   ├─ 步兵→枪兵: ×1.5（克制）
  │   ├─ 枪兵→骑兵: ×1.5（克制）
  │   └─ 被克制方向: ×0.7
  │
扫荡系统
  │
  ├─ 扫荡条件检查
  │   ├─ 关卡已三星通关 ✓
  │   ├─ 扫荡令数量足够（或元宝替代） ✓
  │   └─ 通过 → 进入扫荡
  │
  ├─ 扫荡执行
  │   ├─ 选择关卡 + 扫荡次数(1/3/5/10)
  │   ├─ SweepSystem: 批量计算奖励（不进入战斗）
  │   └─ 显示扫荡结果汇总
  │
武将升星
  │
  ├─ 碎片收集
  │   ├─ 招募重复武将 → 转化为碎片
  │   ├─ 关卡掉落 → 获得碎片
  │   └─ 商店兑换 → 消耗代币获得碎片
  │
  ├─ 升星流程
  │   ├─ 碎片数量足够 + 铜钱足够
  │   ├─ GeneralStarSystem: 消耗碎片 → 星级+1 → 属性大幅提升
  │   ├─ 升星动画（碎片汇聚+光芒爆发）
  │   └─ 突破：星级达到阈值 → 解锁等级上限
  │
科技系统
  │
  ├─ 科技树渲染
  │   ├─ TechTreeViewComponent: 3条路线并行显示
  │   ├─ 节点状态: 未解锁(灰)/可研究(亮)/研究中(动画)/已完成(金)/互斥锁定(红)
  │   └─ 连线: 显示前置依赖关系
  │
  ├─ 研究流程
  │   ├─ 选择可研究节点 → 检查前置+互斥+资源
  │   ├─ 消耗: 铜钱 + 科技点
  │   ├─ 进入研究队列 → 倒计时
  │   ├─ 研究完成 → TechEffectSystem注册效果
  │   └─ 效果应用到: 资源产出(经济) / 战斗属性(军事) / 经验获取(文化)
  │
  ├─ 互斥分支
  │   ├─ 同层两个节点，选择A → B永久锁定
  │   └─ 显示锁定原因 + 推荐选择提示
  │
  └─ 融合科技
      ├─ 跨路线解锁: 军经合一/军文并举/经世济民/霸王之道
      └─ 效果与基础科技乘法叠加
  │
地图系统（新增）
  │
  ├─ 地图渲染
  │   ├─ MapSystem: 20×15六边形网格 + 三大区域 + 6种地形
  │   ├─ 特殊地标: 洛阳(+50%全资源) / 长安(+30%科技点) / 建业(+30%铜钱)
  │   └─ 缩放50%~200% + 拖拽平移
  │
  ├─ 领土管理
  │   ├─ TerritorySystem: 产出计算(基础×地形×阵营×科技×声望×地标)
  │   ├─ 产出气泡: 5种场景(己方/非己方/刚占领/产出0/缩放<60%)
  │   ├─ 驻防机制: 上限=等级×100，自动分配50%
  │   └─ 领土等级: Lv1→5→10→15，产出/驻防递增
  │
  ├─ 攻城战
  │   ├─ SiegeSystem: 城防 = 1000×城市等级×(1+科技加成)
  │   ├─ 攻城消耗: 兵力×100 + 粮草×500
  │   ├─ 占领条件: 城防归零即占领
  │   ├─ 失败惩罚: 损失30%兵力
  │   └─ 重复攻占: 铜钱×5000 + 产出×2(24h)
  │
  ├─ 地图事件
  │   ├─ MapEventSystem: 9类事件(4基础+5扩展)
  │   ├─ 触发: 每小时10%，最多3个，已探索区域
  │   └─ 分支: 强攻(高收益)/谈判(中收益)/忽略(可能负面)
  │
  ├─ 筛选与热力图
  │   ├─ 筛选: 阵营∩等级(AND) + 地形∪收益(OR)
  │   └─ 热力图: 5档颜色(深金/浅金/翠绿/浅绿/灰绿)
  │
  └─ 地图统计
      ├─ 5维度: 领土概览/资源产出/战斗统计/探索进度/事件参与
      └─ 数据来源: 领土系统(实时)/战斗系统(战后)/资源系统(每秒)/地图系统(探索时)
```

## 六、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 科技树Canvas渲染复杂 | 节点+连线+状态渲染性能差 | 使用离屏Canvas预渲染静态部分，仅动态更新状态 |
| 互斥分支逻辑复杂 | 选择后状态管理出错 | 使用状态机管理节点状态，单元测试覆盖所有转换 |
| 科技效果全局影响 | 效果叠加计算不一致 | TechEffectSystem统一管理所有加成，各系统通过接口查询 |
| 扫荡奖励计算偏差 | 批量扫荡奖励与手动战斗不一致 | 使用相同掉落表，扫荡仅跳过战斗过程 |
| 武技特效性能 | 多个特效同时渲染卡顿 | 限制同屏特效数量，4x加速时简化特效 |
| 地图六边形网格渲染性能 | 15块领土+产出气泡+热力图叠加渲染卡顿 | 分层渲染（静态地形层+动态数据层），视口裁剪 |
| 攻城战城防计算矛盾 | PRD内部公式不一致导致实现混乱 | 以MAP-4底部公式为权威，统一Plan/Play/实现 |
| 9类地图事件状态管理 | 限时事件+未处理上限+事件链复杂 | MapEventSystem使用事件状态机，独立管理每个事件生命周期 |
| 领土产出多因子叠加 | 6个乘法因子叠加计算精度问题 | 使用定点数运算，TerritorySystem统一计算入口 |
