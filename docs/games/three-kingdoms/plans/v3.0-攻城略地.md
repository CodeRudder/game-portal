# v3.0 攻城略地 — 关卡战斗系统

> **版本目标**：引入关卡系统和战斗机制。玩家可以挑战6个历史阶段的关卡，每关3波战斗，战斗自动进行，胜利获得资源和奖励，解锁下一关卡。核心循环扩展为"武将+资源→发起战斗→获得战利品→解锁新关卡→更多资源"。

---

## 一、版本目标

### 核心循环
```
积累资源+招募武将 → 进入关卡Tab → 选择关卡 → 自动战斗 → 战利品奖励
       ↑                                              │
       └──────────────────────────────────────────────┘
```

### 玩家在v3.0中能做什么
1. 在关卡Tab中看到6个历史阶段的关卡地图
2. 查看每个关卡的敌人信息、推荐战力、奖励预览
3. 点击"开战"发起战斗（需要至少1名已招募武将）
4. 观看自动战斗过程（简化版：战斗日志+结果展示）
5. 战斗胜利获得资源奖励和战利品
6. 击败Boss波次解锁下一关卡
7. 挑战失败时损失部分兵力

### 2.2 战斗系统

#### 战力计算
```
玩家战力 = Σ(已招募武将的 attack + defense + intelligence + command) × 武将等级
```

#### 自动战斗流程
1. 玩家点击"开战" → 检查条件（有武将、关卡已解锁）
2. 显示战斗界面 → 自动播放战斗日志
3. 每回合：玩家武将攻击 → 敌人反击 → 双方HP扣除
4. 一方HP归零 → 战斗结束
5. 胜利：发放奖励 + 标记波次完成
6. 失败：损失部分兵力 + 可重试

#### 简化战斗公式
```
玩家伤害 = Σ(武将attack × (1 + 武将等级 × 0.1)) × 随机(0.8~1.2)
敌人伤害 = 敌人attack × 随机(0.8~1.2) - 玩家defense × 0.3
```

## 三、数据流设计

### 3.1 引擎→UI
```
ThreeKingdomsEngine
  ├── getCurrentStage() → StageDef
  ├── getStageProgress(stageId) → { completedWaves, totalWaves }
  ├── isStageUnlocked(stageId) → boolean
  ├── getBattleState() → BattleState
  ├── getPlayerPower() → number
  └── getBattleRewards(battleId) → Record<string, number>
```

### 3.2 UI→引擎
```
开始战斗：
  handleStartBattle(battleId)
    → engine.startBattle(battleId)
      → 检查：有武将 + 关卡已解锁
      → 开始自动战斗
      → emit('battleStarted')

战斗结束：
  engine.on('battleEnded', (result) => {
    if (result.victory) {
      发放奖励 → 更新关卡进度 → 检查解锁下一关
    }
  })
```

## 六、数值设计

### 6.1 战力对比
```
玩家战力 / 推荐战力 > 1.5 → 三星
玩家战力 / 推荐战力 > 1.0 → 二星
玩家战力 / 推荐战力 > 0.7 → 一星（勉强通关）
玩家战力 / 推荐战力 ≤ 0.7 → 失败
```
