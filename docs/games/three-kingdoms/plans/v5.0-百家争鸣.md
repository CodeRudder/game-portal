## 三、数据流设计

```
科技系统深化
  │
  ├─ 科技点系统
  │   ├─ 书院建筑 → 产出科技点/秒（Lv1: 0.3/s → Lv20: 8.0/s）
  │   ├─ 科技点无上限累积（离线也累积，受离线效率系数影响）
  │   ├─ 研究启动消耗 = 基础费用 × (1 - 书院研究速度加成)
  │   ├─ 铜钱兑换科技点：100:1（需书院Lv5+）
  │   └─ TechPointSystem: 产出速率计算 / 消耗扣减 / 余额查询
  │
  ├─ 互斥分支
  │   ├─ 每条路线 Tier5~7 为互斥分支（A/B 二选一）
  │   ├─ 选中 A → B 永久锁定（mutex-locked）
  │   ├─ 3 条路线 × 1 组互斥 = 3 组不可逆选择（转生除外）
  │   └─ 军事(进攻/防御) / 经济(重农/重商) / 文化(仁政/霸道)
  │
  ├─ 研究队列
  │   ├─ 默认 1 个研究槽位，VIP3 解锁第 2 槽位
  │   ├─ 当前研究完成 → 队列中下一项自动开始
  │   ├─ 取消研究：返还 80% 已消耗资源，进度清零
  │   └─ 切换研究：消耗 100 铜钱暂停当前，进度保留
  │
  ├─ 加速机制
  │   ├─ 6 种加速方式：铜钱加速 / 天命加速 / 元宝秒完成 / 书院等级 / 声望等级 / 联盟加速
  │   ├─ 叠加公式：实际研究时间 = 基础研究时间 / (1 + 书院 + 声望 + 联盟)
  │   ├─ 道具效果：实际研究时间 × (1 - 道具减免比例)
  │   └─ 联盟加速前置条件：需加入联盟（需联盟系统支持）
  │
  ├─ 融合科技（总计25个科技节点：21个基础节点 + 4个融合科技）
  │   ├─ 每条路线7层级节点（前4层 + 第5~7层互斥分支A/B二选一）= 3条路线 × 7 = 21个基础节点
  │   ├─ 4个融合科技：军经合一、军文并举、经世济民、霸王之道
  │   ├─ 玩家在两条路线各完成指定节点 → 解锁融合科技
  │   ├─ 融合科技不占用基础路线研究槽位
  │   ├─ 融合科技可与基础科技同时研究（需第2研究槽位/VIP3）
  │   └─ 研究流程与普通科技相同
  │
  ├─ 科技联动
  │   ├─ TechLinkSystem: 注册所有联动关系
  │   │   ├─ 建筑: 经济科技→农田产出+20%
  │   │   ├─ 武将: 军事科技→骑兵攻击+15%
  │   │   └─ 资源: 文化科技→天命获取+10%
  │   └─ 科技完成 → 自动应用联动效果 → 相关系统数值更新
  │
  ├─ 离线研究
  │   ├─ 玩家离线 → 记录离线时间
  │   ├─ TechOfflineSystem: 研究进度 += 离线时间 × 效率衰减
  │   ├─ 回归 → 显示离线研究进度面板
  │   └─ 已完成的科技自动生效
  │
世界地图
  │
  ├─ 地图渲染
  │   ├─ MapRenderer: 读取地图数据 → 渲染地形底纹
  │   ├─ 三大区域着色区分（魏/蜀/吴+中立）
  │   ├─ 特殊地标标记（城池图标/关卡旗帜/资源点标记）
  │   └─ 己方领土高亮 + 产出气泡
  │
  ├─ 领土管理
  │   ├─ TerritorySystem: 管理所有领土状态
  │   ├─ 己方领土: 产出资源 → 汇入资源系统
  │   ├─ 领土等级: 消耗资源升级 → 产出增加
  │   └─ 产出气泡: 领土上方常驻显示产出值
  │
  └─ 攻城战
      ├─ 选择目标领土（必须相邻己方领土）
      ├─ SiegeSystem: 检查兵力/粮草 → 显示消耗预览
      │   ├─ 兵力消耗: 兵力×100，出征兵力≥城池驻防兵力×2.0
      │   ├─ 粮草消耗: 粮草×500（固定消耗，⚠️旧公式已废弃）
      │   └─ 每日次数上限: 3次
      ├─ 城防计算: 城防值=基础(1000)×城市等级×(1+城防科技加成)（⚠️旧公式已废弃）
      ├─ 确认攻城 → 进入战斗（复用BattleEngine）
      ├─ 胜利条件: 城防值归零（唯一条件，⚠️旧三条件已废弃）
      ├─ 胜利 → 领土归属变更 → 开始产出资源（初始50%→24h后100%）
      ├─ 失败 → 损失30%兵力（⚠️旧规则"返还50%"已废弃） → 推荐算法分析战力差距 → 推荐科技/武将/领土提升方案
      ├─ 驻防机制: 驻防上限=领土等级×100，每点兵力+0.1%防御 ★r3
      ├─ 地图事件: 5种事件类型(流寇/商队/天灾/遗迹/阵营冲突)，每小时10%概率触发 ★r3
      ├─ 地图统计: 5维度(领土/资源/战斗/探索/事件)实时汇总 ★r3
      └─ 转生处理: 核心领土保留100%/普通50%+攻城状态清理+奖励补发 ★r3
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 世界地图渲染性能 | 大地图Canvas渲染卡顿 | 使用离屏Canvas预渲染地形，仅动态更新领土状态 |
| 科技联动效果复杂 | 效果叠加计算出错 | TechLinkSystem统一注册，各系统通过getTechBonus()查询 |
| 攻城战与关卡战斗差异 | 两套战斗逻辑维护成本高 | 复用BattleEngine，攻城战仅增加城防值计算 |
| 领土产出平衡性 | 领土产出过强/过弱 | 初期保守设定，根据测试数据调整 |
| 地图数据量大 | 三大区域+城池+地形=大量配置 | 使用MapGenerator程序化生成基础地形，手动配置关键地标 |
| 互斥分支不可逆导致玩家后悔 | 玩家选错分支后体验下降 | 提供充分的分支预览信息，转生可重选作为安全网 |
| 研究队列与加速叠加复杂 | 队列+加速+离线组合计算出错 | 单元测试覆盖所有组合场景，边界条件专项测试 |
| 联盟加速依赖外部系统 | 联盟系统未就绪时功能不可用 | 独立实现联盟加速逻辑，通过接口解耦，联盟系统就绪后接入 |
| PRD公式废弃追踪 | 旧公式(城防/粮草/占领/失败)在多处引用，容易遗漏 | 在play附录C中维护统一声明修订记录，代码注释标注⚠️ |
| 转生领土数据一致性 | 转生时领土保留/丢失/产出重置逻辑复杂，易产生脏数据 | 转生流程作为原子事务执行，失败全部回滚 |
