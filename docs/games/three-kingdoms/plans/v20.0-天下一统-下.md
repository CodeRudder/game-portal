## 三、数据流设计

```
全系统联调
  │
  ├─ 核心循环端到端
  │   ├─ 挂机产出: 4资源每秒增长 → 资源栏实时更新
  │   ├─ 建筑升级: 消耗资源 → 等级+1 → 产出增加 → 循环加速
  │   ├─ 武将招募: 消耗铜钱 → 武将入队 → 派遣建筑 → 产出加成
  │   ├─ 战斗推图: 编队出战 → 自动战斗 → 获得奖励 → 解锁新关
  │   ├─ 科技研究: 消耗资源+时间 → 全局加成 → 解锁新功能
  │   └─ 资源加速: 建筑+武将+科技 → 产出倍增 → 更快升级
  │
  ├─ 跨系统数据流验证
  │   ├─ 资源→建筑: 产出速率正确+升级消耗正确+加成生效
  │   ├─ 建筑→武将: 招募建筑解锁+派遣加成正确
  │   ├─ 武将→战斗: 编队战力计算正确+属性影响战斗结果
  │   ├─ 战斗→装备: 掉落表正确+品质概率正确
  │   ├─ 装备→武将: 穿戴加成正确+套装效果激活
  │   ├─ 武将→科技: 研究条件满足+效果正确应用
  │   └─ 全系统→声望: 各途径声望获取正确+加成生效
  │
  ├─ 转生循环验证
  │   ├─ 条件检查: 5项条件全部满足
  │   ├─ 数据重置: 保留项不丢失/重置项归零/衰减项按比例
  │   ├─ 倍率生效: 产出/经验/速度全面加速
  │   ├─ 加速重建: 初始资源+低级建筑瞬间+一键重建
  │   └─ 再次循环: 更快推图→更高声望→更高倍率
  │
  └─ 离线全系统验证
      ├─ 离线收益: 基础产出×秒数×效率系数(分段衰减)
      ├─ 离线事件: 正面保守50%/负面防御减免/限时消失
      ├─ 离线活动: 赛季50%/限时30%/日常100%
      ├─ 离线远征: 队伍继续行军+战斗自动结算
      └─ 离线贸易: 商队继续往返+收益累积

数值平衡
  │
  ├─ 资源产出平衡
  │   ├─ 粮草: 主要产出(农田)→主要用于维持兵力
  │   ├─ 铜钱: 通用货币→建筑/招募/强化/商店
  │   ├─ 兵力: 战斗消耗→兵营产出+时间恢复
  │   └─ 天命: 高级货币→加速/高级招募/保护符
  │   └─ 验证: 0~30分钟可升级第一座建筑，1小时可招募第一个武将
  │
  ├─ 武将战力平衡
  │   ├─ 战力公式: (ATK×2+INT×2+CMD×1.5+POL×1)×等级系数×星级系数×装备系数
  │   ├─ 品质差距: Uncommon 60~75 vs Mythic 100~120 (约1.6倍基础差距)
  │   └─ 养成补偿: 低品质武将通过升级/突破/装备可追赶
  │
  ├─ 战斗难度曲线
  │   ├─ 第1~3关: 推荐战力500~1500，教学难度
  │   ├─ 第4~8关: 推荐战力2000~8000，正常难度
  │   ├─ 第9~12关: 推荐战力10000~30000，挑战难度
  │   └─ 第13~15关: 推荐战力40000~80000，极限难度
  │
  ├─ 经济系统平衡
  │   ├─ 铜钱: 最易获取→最大消耗(建筑/招募/强化)
  │   ├─ 天命: 较难获取→关键加速(研究/招募/保护符)
  │   ├─ 招贤榜: 日常获取→普通招募
  │   └─ 求贤令: 稀有获取→高级招募(100次保底)
  │
  └─ 转生倍率平衡
      ├─ 1次Lv.5: 1.88× (明显加速但不过分)
      ├─ 3次Lv.15: 3.19× (中后期合理加速)
      ├─ 5次Lv.20: 4.50× (长线玩家奖励)
      └─ 10次Lv.30: 7.20× (上限控制，20次封顶)

性能优化
  │
  ├─ 渲染优化
  │   ├─ 离屏Canvas预渲染静态背景
  │   ├─ 脏矩形：仅重绘变化区域
  │   ├─ 批量绘制：相同纹理合并DrawCall
  │   └─ 目标：60fps稳定，低端设备30fps
  │
  ├─ 内存优化
  │   ├─ 资源按需加载：仅加载当前场景资源
  │   ├─ 纹理合并：小图标合并为Sprite Atlas
  │   ├─ 对象池：粒子/飘字/子弹复用
  │   └─ 目标：内存占用<200MB
  │
  └─ 加载优化
      ├─ 首屏加载<3s（核心资源优先）
      ├─ 分包加载：按模块分包，按需加载
      ├─ 缓存策略：LocalStorage缓存已加载资源
      └─ 预加载：空闲时预加载下一场景资源
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 跨系统数据流断裂 | 某系统数据未正确传递导致功能异常 | 自动化端到端测试+数据流追踪工具+逐一验证 |
| 数值平衡偏差 | 某资源/战力/难度曲线不合理 | 自动化平衡测试+模拟100小时游戏进程+调整参数 |
| 性能瓶颈 | 特定场景卡顿(战斗/地图/大量粒子) | 性能监控面板+瓶颈定位+针对性优化 |
| 兼容性问题 | 特定设备/浏览器上表现异常 | 多设备测试矩阵+降级方案+Polyfill |
| 全功能验收工作量大 | 204功能点逐一验收耗时 | 自动化测试覆盖80%+人工验收关键路径20% |
| 修复引发新Bug | 平衡性调整可能引入新问题 | 每次调整后回归测试+增量发布 |
