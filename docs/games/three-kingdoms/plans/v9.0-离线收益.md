## 三、数据流设计

```
离线收益系统
  │
  ├─ 快照机制
  │   ├─ 玩家下线 → 立即记录快照
  │   │   ├─ 各资源净产出速率
  │   │   ├─ 经验产出速率(主城+科技+VIP加成)
  │   │   ├─ 建筑队列状态
  │   │   ├─ 科技队列状态
  │   │   ├─ 远征队列状态
  │   │   ├─ 商队运输状态
  │   │   ├─ 领土攻城状态(进行中攻城战+城防值+驻防兵力) 【MAP-4统一声明】
  │   │   ├─ 活动进行中状态(积分/代币/任务进度)
  │   │   └─ 自动推图状态(当前关卡/已三星列表)
  │   ├─ 快照有效期72h → 超过封顶时长不再使用
  │   └─ 快照丢失降级 → 使用最后成功快照或默认速率，弹窗提示异常
  │
  ├─ 离线计算（分段计算引擎）
  │   ├─ 玩家上线 → 计算离线时长
  │   ├─ 离线≤5分钟 → 静默添加收益，不弹窗
  │   ├─ 离线>5分钟 → 弹出离线收益弹窗
  │   │   ├─ 分段计算: 每时段独立应用衰减系数后累加
  │   │   │   ├─ 时段1(0~2h): 速率×min(剩余秒数,7200)×100%
  │   │   │   ├─ 时段2(2~8h): 速率×min(剩余秒数-7200,21600)×80%
  │   │   │   ├─ 时段3(8~24h): 速率×min(剩余秒数-28800,57600)×60%
  │   │   │   ├─ 时段4(24~48h): 速率×min(剩余秒数-86400,86400)×40%
  │   │   │   ├─ 时段5(48~72h): 速率×min(剩余秒数-172800,86400)×25%
  │   │   │   └─ >72h部分不参与计算
  │   │   ├─ 加成叠加: 科技+30% | VIP+20% | 声望+25% (上限+100%)
  │   │   ├─ 各系统修正: 建筑×1.2 | 远征×0.85 | 自动推图×0.80
  │   │   └─ 经验产出: 基础经验速率×离线秒数×衰减×(1+经验加成)
  │   └─ 资源溢出检测: 有上限资源截断+提示升级
  │
  ├─ 回归流程（时间预算 ~5.2s）
  │   ├─ Step1: 离线收益弹窗 (~4s含动画+操作)
  │   │   ├─ 展示离线时长+效率系数+各资源收益
  │   │   ├─ 数字滚动动画(800ms)
  │   │   ├─ 翻倍选项: 广告×2(≥30min,3次/天,失败降级1.5×) 或 元宝×2(无限制,10~200元宝)
  │   │   ├─ 溢出警告: 资源将溢出时标注⚠️
  │   │   ├─ 封顶提示: >72h时显示警告条
  │   │   └─ 一键领取
  │   └─ Step2: 回归综合面板(可跳过, ~1.2s)
  │       ├─ 建筑升级完成列表
  │       ├─ 科技研究完成列表
  │       ├─ 远征队归来摘要
  │       ├─ 自动推图进度展示
  │       ├─ 待处理事件列表(最多5个)
  │       ├─ 限时NPC提醒
  │       ├─ 活动进度摘要(进行中/已结束)
  │       └─ 版本更新摘要
  │
活动离线联动
  │
  ├─ 积分累积
  │   ├─ 赛季活动: 50%效率累积, 每日上限12h
  │   ├─ 限时活动: 30%效率累积, 每日上限8h
  │   ├─ 日常活动: 100%效率累积, 每日上限24h
  │   ├─ 节日活动: 50%效率累积, 每日上限12h
  │   └─ 声望加成: 基础效率×(1+声望等级×0.03)
  │
  ├─ 任务推进
  │   ├─ 累积型任务: 离线自动推进
  │   ├─ 每日自动类(登录/收取): 离线正常计入
  │   ├─ 挑战任务: 离线暂停等待
  │   └─ 商店兑换: 离线不自动消耗代币
  │
  └─ 活动到期结算
      ├─ 回归面板展示已结束活动(灰色「已结束」标签)
      ├─ 未使用代币按10%比例转化为铜钱
      └─ 排行奖励通过邮件发放
  │
邮件系统
  │
  ├─ 邮件生成
  │   ├─ 系统邮件: 版本更新/维护通知 → 自动生成
  │   ├─ 奖励邮件: 活动奖励/排行奖励/补偿/签到 → 自动生成+附件
  │   ├─ 社交邮件: 借将通知/援助通知/切磋结果 → 事件触发
  │   ├─ 战斗邮件: 攻城战报/竞技场结算 → 战斗结束触发
  │   └─ 离线收益邮件: 按时长生成差异化标题+效率系数
  │
  ├─ 邮件管理
  │   ├─ 分类Tab: 全部/系统/战斗/社交/奖励
  │   ├─ 状态流转: 未读 → 已读未领 → 已读已领 → 已过期
  │   ├─ 容量管理: 正常(0~70) → 警告(71~90) → 拥挤(91~99) → 满载(100,暂存20封)
  │   ├─ 过期处理: 分级提醒(3天⚠️/1天Toast/当天红色闪烁) → 自动清理(05:00)
  │   └─ 批量操作: 全选+一键领取+全部已读+删除已读
  │
  └─ 附件领取
      ├─ 单封: 点击邮件 → 展开详情 → 领取附件
      ├─ 批量: 勾选多封 → 一键领取全部附件
      ├─ 背包满处理: 货币类直接入账/道具类不可领取+提示清理
      ├─ 过期补偿: 奖励邮件过期后铜钱/经验补发50%
      └─ 领取后: 物品入背包/资源入账 → 状态变为已读已领
```

## 四、验收标准

### 功能验收
- [ ] 离线≤5分钟静默添加收益，不弹窗
- [ ] 离线>5分钟弹出离线收益弹窗，展示完整收益信息
- [ ] 6档衰减系数正确应用（100%→80%→60%→40%→25%→15%）
- [ ] 分段计算引擎正确：每时段独立应用衰减系数后累加，无跨档误差
- [ ] 科技/VIP/声望加成正确叠加，上限+100%
- [ ] 快照机制在下线时立即记录，72h有效期
- [ ] 快照丢失时降级处理正确，使用最后成功快照或默认速率，弹窗提示
- [ ] 封顶72h正确执行，超过部分不产出收益
- [ ] 各系统离线行为正确执行（建筑×1.2/远征×0.85/自动推图×0.80等）
- [ ] 自动推图离线仅推已三星关卡，一星评价，奖励≈在线60%
- [ ] 事件系统离线不自动处理，最多堆积5个，超时保守方案

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 离线计算数据量大 | 长时间离线后计算耗时 | 使用分段计算，预缓存中间结果 |
| 快照数据丢失 | 浏览器清理localStorage导致快照丢失 | 多重存储：localStorage + IndexedDB + 降级处理 |
| 资源溢出玩家不满 | 离线收益因上限截断导致"白挂了" | 弹窗明确提示溢出+引导升级建筑 |
| 邮件系统性能 | 大量邮件导致列表卡顿 | 分页加载，每页20条，虚拟滚动 |
| 翻倍机制公平性 | 免费玩家vs付费玩家收益差距 | 广告翻倍免费可用(3次/天)，元宝翻倍为可选加速 |
| 活动离线积分膨胀 | 高声望玩家离线活动收益过高 | 声望加成线性增长(×0.03/级)，每日上限限制有效时长 |
| 离线收益邮件效率系数不一致 | MAL-3的4档与SPEC-offline的6档差异 | 以SPEC-offline 6档衰减为准，邮件标题仅作展示用途 |
| 回归面板信息过载 | 离线期间变化太多导致面板过长 | 分区折叠展示，可跳过，关键信息优先展示 |
| 离线声望累积闭环 | 声望等级提升影响离线效率系数，需确保闭环 | 声望累积→等级提升→加成更新→影响后续离线计算，实时生效 |
| 邮件暂存队列溢出 | 暂存20封上限外邮件丢失 | 超过20封时发送系统通知告知玩家有邮件丢失，建议清理邮箱 |
| MAP统一声明数值一致性 | 离线攻城战需引用MAP-4统一声明(城防公式/攻城消耗/失败惩罚)，数值不一致导致结算错误 | 以MAP PRD v1.1统一声明为准，离线攻城结算严格使用: 城防=1000×等级×(1+科技加成)、消耗=兵力×100+粮草×500、失败=损失30%兵力 |
| 离线科技完成时序 | 科技离线完成后加成生效时点影响后续产出计算 | 快照机制确保本次离线收益使用下线时加成，科技完成后加成立即生效影响后续产出(不影响本次) |
| 离线领土视觉标记性能 | 大量领土变化(>10块)导致地图渲染卡顿 | 超过10块变化时折叠显示"共X块变化"，脉冲动画使用CSS transform保证GPU加速 |
| 天命阁升级产出曲线 | 天命阁高等级产出过高导致天命通胀 | 天命消耗用途同步扩展(高级招募/科技加速/转生重置)，产出与消耗保持平衡 |
| 声望→活动三角闭环时序 | 声望升级时刻与活动积分计算冲突 | 以声望升级时刻为界分段计算活动积分，升级前用旧加成，升级后用新加成 |
| ExperienceSystem注册失败 | 经验系统未正确注册离线行为导致经验收益归零 | OfflineRewardSystem检测未注册时使用通用资源快照中的经验字段计算，不归零 |
