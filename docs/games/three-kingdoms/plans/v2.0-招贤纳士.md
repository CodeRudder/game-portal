## 二、游戏引擎需求

### 新增子系统
| 子系统 | 职责 | 依赖 |
|--------|------|------|
| GeneralRecruitSystem | 招募概率计算、保底计数、抽卡动画触发 | RES(货币消耗)、GeneralData |
| GeneralLevelSystem | 武将升级经验计算、属性成长、一键强化逻辑 | RES(铜钱消耗) |
| GeneralTeamSystem | 编队管理、前后排配置、羁绊计算 | HER数据 |
| GeneralSkillSystem | 技能等级管理、技能效果注册 | HER数据 |
| 🆕 GeneralBreakthroughSystem | 突破节点管理、等级上限提升、技能解锁 | GeneralLevelSystem, GeneralData |
| 🆕 GeneralStarSystem | 升星碎片消耗、属性加成、碎片进度可视化 | GeneralData, RES(碎片) |
| 🆕 GeneralFormationSystem | 多编队管理(3编队)、武将互斥、活跃编队切换、智能推荐 | GeneralTeamSystem |
| 🆕 RedDotNotificationSystem | 红点/角标触发与消除、今日待办聚合 | GeneralLevelSystem, GeneralStarSystem, GeneralData |

### 改进现有子系统
| 子系统 | 改进内容 |
|--------|----------|
| GeneralData.ts | 验证武将数据完整性（属性/品质/技能/阵营），补充缺失数据 |
| GeneralPortraitRenderer | 连通品质→边框/背景渲染，支持详情面板大图展示 |
| QCharacterRenderer | 支持列表卡片中的Q版角色缩略图渲染 |
| RES资源系统 | 新增招贤榜/求贤令两种代币的消耗逻辑 |

|------|------|

## 三、数据流设计

```
玩家切换到「武将」Tab
  │
  ├─ 武将列表渲染
  │   ├─ 读取 engine.getGenerals() → 筛选/排序 → 卡片网格
  │   ├─ 每张卡片: 品质边框 + Q版头像 + 名字 + 等级 + 战力
  │   ├─ 🆕 红点检测: RedDotNotificationSystem → 标记可升级/可升星武将
  │   ├─ 🆕 今日待办横幅: 聚合可升级/可升星/免费招募等待办事项
  │   └─ 空列表时显示「招募引导」按钮
  │
  ├─ 招募流程
  │   ├─ 点击「招募」→ 选择普通/高级 → 选择单抽/十连
  │   ├─ 🆕 每日免费招募: 检查免费次数 → 不消耗代币 → 执行招募
  │   ├─ GeneralRecruitSystem: 检查代币 → 扣除 → 计算概率(含保底)
  │   ├─ 🆕 UP武将/卡池: 显示本期UP武将 → 出Legendary时50%为UP
  │   ├─ 播放招募动画 → 揭示武将品质
  │   ├─ 新武将 → 加入列表 | 重复武将 → 转化碎片+Toast提示
  │   └─ 更新保底计数器
  │
  ├─ 武将详情
  │   ├─ 点击武将卡片 → 详情面板展开
  │   ├─ 左侧: 画像 + 品质 + 等级 + 经验条
  │   ├─ 右侧: 雷达图 + 技能列表 + 羁绊状态 + 编队信息
  │   ├─ 🆕 碎片进度可视化: 进度条颜色(蓝→紫→金) + 特效
  │   └─ 操作: 升级 / 一键强化 / 技能升级 / 🆕突破 / 🆕升星
  │
  ├─ 武将升级
  │   ├─ 点击「升级」→ 检查铜钱+经验道具 → 扣除 → 属性成长
  │   ├─ 一键强化: 自动计算最优升级路径 → 预览面板 → 确认执行
  │   ├─ 🆕 突破系统: 等级达上限 → 消耗碎片+铜钱 → 等级上限提升+技能解锁
  │   ├─ 🆕 升星系统: 碎片足够 → 消耗碎片 → 全属性+10%+技能强化
  │   └─ 属性变化 → 雷达图动画更新 → 战力重新计算(✏️含装备系数)
  │
  └─ 编队管理
      ├─ 选择6个武将 → 配置前后排(3+3)
      ├─ GeneralTeamSystem: 计算阵营羁绊加成
      ├─ 🆕 多编队管理: 3支编队 → 武将互斥 → 活跃编队切换
      ├─ 🆕 智能编队推荐: 根据关卡特性推荐1~3套方案
      └─ 保存编队 → 影响战斗出战阵容
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| GeneralData.ts武将数据不完整 | 招募/属性计算失败 | 验证并补全所有武将的四维属性/品质/技能/阵营数据 |
| 招募概率计算复杂 | 保底逻辑出错 | 编写单元测试覆盖所有概率边界条件 |
| 雷达图Canvas渲染性能 | 列表滚动卡顿 | 雷达图仅在详情面板渲染，列表用简化属性条 |
| 编队与后续战斗系统耦合 | v3.0战斗需要编队数据 | 本版本预留编队接口，v3.0直接调用 |
| 武将画像资源缺失 | 部分武将无头像 | 使用GeneralPortraitRenderer程序化生成，确保所有武将有画像 |
