# v2.0 招贤纳士 — 武将系统

> **版本目标**：在v1.0资源+建筑的基础上，引入武将系统。玩家可以招募武将、查看武将属性、升级武将、派遣武将到建筑增加产出。核心循环扩展为"建筑产出→资源积累→招募武将→武将增强建筑"。

---

## 一、版本目标

### 核心循环
```
建筑产出资源 → 积累铜钱/粮草 → 招募武将 → 武将派遣到建筑 → 建筑产出增加
     ↑                                              │
     └──────────────────────────────────────────────┘
```

### 玩家在v2.0中能做什么
1. 在武将Tab中查看12位武将的卡片（头像、名称、阵营、稀有度、属性）
2. 使用铜钱+粮草+兵力招募武将
3. 查看已招募武将的详细属性（攻击/防御/智力/统帅）
4. 消耗资源升级武将等级（属性成长）
5. 派遣武将到建筑（武将统帅值增加建筑产出百分比）
6. 看到武将派遣后建筑产出数值的变化

## 二、功能清单

### 2.1 武将招募

#### 招募条件
| 稀有度 | 粮草 | 铜钱 | 兵力 | 示例武将 |
|:------:|:----:|:----:|:----:|---------|
| uncommon | 200 | 300 | 100 | 许褚 |
| rare | 500 | 800 | 300 | 夏侯惇、甘宁、陆逊 |
| epic | 1,500 | 2,000 | 800 | 关羽、张飞、司马懿、周瑜 |
| legendary | 5,000 | 8,000 | 3,000 | 刘备、曹操、孙权 |
| mythic | 15,000 | 25,000 | 10,000 | 诸葛亮 |

#### 招募流程
1. 打开武将Tab → 看到武将列表
2. 点击未招募武将卡片 → 显示武将详情面板
3. 点击"招募"按钮 → 检查资源 → 扣除费用 → 武将加入
4. 招募成功Toast + 音效 + 粒子特效

#### 免费赠送
- 新手免费赠送许褚（第一个uncommon武将）
- 在首次进入武将Tab时自动触发

### 2.2 武将属性

#### 四维属性
| 属性 | 图标 | 说明 | 成长影响 |
|------|:----:|------|---------|
| attack | ⚔️ | 攻击力 | 战斗伤害 |
| defense | 🛡️ | 防御力 | 战斗减伤 |
| intelligence | 🧠 | 智力 | 科技研究加成 |
| command | 🏳️ | 统帅 | 建筑产出加成 |

#### 等级系统
- 初始等级：1
- 最高等级：50
- 升级费用：`基础费用 × 1.05^当前等级`
- 升级资源：铜钱 + 粮草
- 每级属性成长：基础属性 × 成长率 × 0.01

### 2.3 武将派遣

#### 派遣规则
- 每个建筑最多派遣1名武将
- 每名武将只能派遣到1个建筑
- 派遣效果：建筑产出 × (1 + 武将统帅值/200)
- 可随时更换派遣

#### 派遣效果示例
| 武将 | 统帅值 | 派遣效果 |
|------|:------:|---------|
| 许褚(Lv.1) | 55 | 产出 +27.5% |
| 刘备(Lv.1) | 90 | 产出 +45% |
| 诸葛亮(Lv.10) | ~104 | 产出 +52% |

## 三、数据流设计

### 3.1 引擎→UI数据流
```
ThreeKingdomsEngine
  ├── isGeneralRecruited(id) → boolean
  ├── getGeneralLevel(id) → number
  ├── getGeneralStats(id) → { attack, defense, intelligence, command }
  ├── getGeneralDispatch(id) → string | null  // 派遣到的建筑ID
  ├── getRecruitCost(id) → { grain, gold, troops }
  ├── getUpgradeCost(id) → { grain, gold }
  └── getBuildingDispatchGeneral(buildingId) → string | null
```

### 3.2 UI→引擎操作流
```
招募武将：
  handleRecruitClick(generalId)
    → engine.recruitGeneral(id)
      → 成功：Toast + 音效 + 粒子
      → 失败：Toast "资源不足" / "已招募"

升级武将：
  handleUpgradeClick(generalId)
    → engine.upgradeGeneral(id)
      → 成功：属性增长动画
      → 失败：Toast "资源不足"

派遣武将：
  handleDispatch(generalId, buildingId)
    → engine.dispatchGeneral(generalId, buildingId)
      → 成功：建筑产出更新
      → 失败：Toast "该建筑已有武将"
```

## 六、数值设计

### 6.1 武将升级费用
```
升级费用 = { grain: 100 × 1.05^level, gold: 80 × 1.05^level }
```

### 6.2 武将属性成长
```
每级属性增长 = 基础属性 × 成长率 × 0.01
```
| 稀有度 | 成长率 |
|:------:|:-----:|
| uncommon | 3 |
| rare | 4.5 |
| epic | 6 |
| legendary | 8 |
| mythic | 10 |

### 6.3 派遣加成公式
```
建筑产出倍率 = 1 + 武将统帅值 / 200
```

## 九、已知依赖和风险

### 依赖
- v1.0的建筑系统和资源系统完全正常
- `UnitSystem` 的 `unlock()` 和 `getState()` 方法正常
- `recruitGeneral()` 引擎方法已存在

### 风险
1. **武将派遣是全新功能**：引擎中无dispatch相关逻辑，需要新增
2. **武将升级是全新功能**：引擎中无upgrade相关逻辑，需要新增
