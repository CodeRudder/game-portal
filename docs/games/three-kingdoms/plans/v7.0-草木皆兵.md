## 三、数据流设计

```
NPC巡逻与高级交互
  │
  ├─ NPC巡逻
  │   ├─ NPCPatrolSystem: 读取路径数据 → 驱动NPC移动
  │   ├─ NPCPathFollower.ts: 沿路径点移动，到达终点后折返
  │   └─ 地图渲染: NPC图标实时更新位置
  │
  ├─ NPC交易（F7.24）
  │   ├─ 与NPC对话 → 选择「交易」（Lv.1默认解锁）
  │   ├─ NPCTradingComponent: 显示NPC专属商品列表
  │   ├─ 折扣计算: 基础价格 × (1 - 好感度折扣 - 结盟折扣)
  │   │   ├─ Lv.1~2: 无折扣
  │   │   ├─ Lv.3~4: 10%折扣
  │   │   └─ Lv.5结盟(商人): 10%+5%叠加=15%折扣
  │   └─ 购买 → 资源扣除 → 商品入背包 → EventBus通知QuestTrackerSystem
  │
  ├─ NPC赠送
  │   ├─ 与NPC对话 → 选择「赠送」（需Lv.3）
  │   ├─ GiftSelectComponent: 显示背包物品，偏好物品标记⭐
  │   ├─ 选择物品 → NPCGiftSystem: 计算好感度（偏好×倍率）
  │   └─ 好感度提升 → 升级检查 → 声望奖励 → Toast反馈
  │
  ├─ NPC好感度等级解锁（F7.22）
  │   ├─ Lv.1（0点）: 基础对话 + 交易 + NPC任务链第1-2个
  │   ├─ Lv.2（100点）: 解锁「情报」→ 获得地图线索
  │   ├─ Lv.3（300点）: 解锁「赠送」+ 商店折扣10%
  │   ├─ Lv.4（600点）: 解锁「切磋」+ 专属NPC任务链
  │   └─ Lv.5（1000点）: 解锁羁绊技能 → 可结盟 → 永久被动加成
  │
  ├─ NPC好感度声望奖励（F7.25）
  │   ├─ Lv.1→Lv.2: 声望+200
  │   ├─ Lv.2→Lv.3: 声望+300
  │   ├─ Lv.3→Lv.4: 声望+500
  │   ├─ Lv.4→Lv.5: 声望+800
  │   └─ 首次满级(Lv.5): 额外天命×150（每位NPC仅一次，转生不重复）
  │
  ├─ NPC专属任务链（F7.23）
  │   ├─ 每个NPC有独立任务链（5-6个任务），按好感度等级逐步解锁
  │   ├─ 任务链按顺序解锁，同一NPC同时只能进行1个任务
  │   ├─ 可放弃重接（进度重置），奖励随好感度等级递增
  │   └─ 终章任务（Lv.5解锁）奖励含天命+史诗道具
  │
  ├─ NPC切磋
  │   ├─ 与武将NPC对话 → 选择「切磋」（需Lv.4）
  │   ├─ NPCTrainingSystem: 进入简化战斗（3回合制，无消耗）
  │   ├─ 胜利: 武将获得经验 + 随机道具
  │   └─ 失败: 无惩罚，提示提升方向
  │
  ├─ NPC离线行为
  │   ├─ 离线期间: NPC按类型自动执行行为
  │   │   ├─ 农民: 粮草（基础产出×50%，累积上限8h）
  │   │   ├─ 商人: 铜钱（基础产出×50%，累积上限8h）
  │   │   ├─ 学者: 科技点加速（×0.5倍率，累积上限4h）
  │   │   ├─ 斥候: 地图情报碎片（3条/次）
  │   │   ├─ 士兵: 降低被袭击概率（全程）
  │   │   └─ 全NPC: 好感度自然增长（+0.1/小时，上限+2/次）
  │   ├─ 离线特殊事件（离线≥4h+好感度≥3+随机5%/h）
  │   │   ├─ NPC求助: 回归后弹出对话，需提供资源，奖励稀有道具
  │   │   ├─ NPC送礼（离线≥6h+好感≥4）: NPC赠送稀有道具
  │   │   └─ NPC引荐（离线≥8h+好感≥5）: 解锁新NPC
  │   └─ 回归 → NPC离线行为摘要面板
  │
事件系统深化
  │
  ├─ 连锁事件
  │   ├─ 完成事件A选择选项1 → 触发连锁事件B1
  │   ├─ 完成事件A选择选项2 → 触发连锁事件B2
  │   ├─ 连锁链最多3-4层，环间延迟10-30分钟随机
  │   ├─ 边界规则: 链超时>24h自动终止(30%阶段奖励) / 链并发:仅1条 / 链冷却7天 / 离线暂停
  │   └─ 剧情事件链（特殊连锁）: 与主线章节绑定，无环间延迟，不受并发限制
  │       例: 第一章"黄巾之乱": 1-1黄巾初现→1-2颍川之战→1-3广宗之战→1-4张角决战
  │
  ├─ 历史剧情事件
  │   ├─ 满足条件触发（章节进度/特定时间）
  │   ├─ 全屏沉浸式展示：水墨背景+角色立绘+对话文本
  │   ├─ 玩家选择影响后续剧情走向
  │   └─ 完成后记录到事件日志
  │
任务系统
  │
  ├─ 任务面板
  │   ├─ Tab分类: 主线/支线/日常/周常/成就
  │   ├─ 任务列表: 名称+进度+奖励预览
  │   └─ 点击任务 → 展开详情 → 跳转按钮
  │
  ├─ 日常任务
  │   ├─ 每日05:00刷新: 从20个任务池中随机抽取6个（与PRD QST-3一致）
  │   ├─ 抽取保证: ≥1战斗类 / ≥1养成类 / ≥1自动完成类 / 最多2个同类 / D01登录必出
  │   ├─ QuestTrackerSystem: 实时检测进度（通过EventBus监听全系统事件）
  │   │   ├─ "升级建筑1次" → 监听建筑升级事件
  │   │   ├─ "通关关卡3次" → 监听战斗结算事件
  │   │   ├─ "完成1次NPC交互" → 监听NPC交互事件（D12）
  │   │   ├─ "完成1次事件响应" → 监听事件完成事件（D19）
  │   │   └─ "招募武将1次" → 监听招募事件
  │   ├─ 离线进度衰减: 0-1h:100% / 1-4h:80% / 4-8h:50% / 8-12h:30% / 12h+:20%
  │   ├─ 完成任务 → 领取奖励 → 活跃度+N（10-20点/任务）
  │   └─ 活跃度宝箱: 40活跃(铜钱+强化石) / 60活跃(元宝+招募令) / 80活跃(元宝+紫装箱) / 100活跃(元宝+金碎片)
  │
  ├─ 周常任务（F7.26）
  │   ├─ 每周一05:00刷新: 从12个任务池中随机抽取4个（与PRD QST-3一致）
  │   ├─ 奖励比日常高3-5倍（低/中/高三档）
  │   └─ 进度跨日累计，仅周一重置
  │
  ├─ 成就系统（F7.27）
  │   ├─ 五大类: 战斗(15-20) / 养成(15-20) / 收集(10-15) / 社交(8-12) / 探索(8-12)
  │   ├─ 三级递进揭示: 🥉铜级初始可见 → 🥈银级铜级完成后揭示 → 🥇金级银级完成后揭示
  │   ├─ 成就点数累计解锁商店: 100/300/500/1000/2000五档
  │   └─ 成就达成 → 触发里程碑事件（与EVT系统联动）
  │
  ├─ 任务追踪
  │   ├─ 主界面左上角常驻面板（最多显示3个）
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 任务系统涉及面广 | 任务目标覆盖所有子系统 | QuestTrackerSystem使用EventBus监听，各系统只需抛出事件 |
| NPC巡逻渲染性能 | 大量NPC同时移动导致卡顿 | 限制同屏NPC数量，视口外NPC仅逻辑更新不渲染 |
| 连锁事件数据复杂 | 事件链数据量大，配置易出错 | 使用JSON模板+验证器，单元测试覆盖所有链路 |
| NPC任务链数据量大 | 每个NPC 5-6个任务，23+个NPC任务链数据配置繁重 | 使用任务模板+参数化配置，复用任务类型框架 |
| 装备系统交叉依赖 | v7.0任务跳转/赠送/切磋奖励涉及装备系统(EQP)接口 | 定义清晰的EQP公共接口（强化/穿戴/获取），v7.0仅调用接口不修改EQP逻辑 |
| 离线计算复杂度 | NPC离线行为+事件离线处理+任务离线进度三者叠加计算 | OfflineCalculator分层计算：先NPC产出→再事件处理→最后任务进度（含衰减） |
| NPC PRD数值矛盾 | NPC PRD末尾简化摘要与正文定义矛盾（赠送5次/切磋消耗兵力/结盟3个） | 以正文为准，末尾简化摘要标记为废弃；Play文档中明确标注权威来源 |
| 成就系统数据量大 | 60-100个成就配置+揭示规则+点数商店 | 使用成就模板+条件化配置，铜级初始可见，银/金级条件揭示减少初始数据量 |
| 周常+日常+成就三线并行 | 三种任务系统同时运作可能造成玩家负担 | 日常6个+周常4个数量可控；成就被动追踪不需主动接取；活跃度仅计算日常 |

> **v1.2修订**: 新增3条风险点（NPC PRD数值矛盾、成就系统数据量、三线并行负担）；更新已有风险点描述

## 六、跨系统依赖声明（v1.3新增）

> **目的**: 明确v7.0与MAP/CBT/BLD/RES/TEC五大系统的接口边界和数据流，确保数值一致性。

### 6.1 MAP攻城系统依赖

| 依赖项 | v7.0调用方式 | MAP PRD权威来源 |
|--------|------------|----------------|
| 城防公式 | NPC攻城类任务（如"占领3座商业城池"）触发攻城 | MAP-4城防计算: `基础城防(1000) × 城市等级 × (1 + 城防科技加成)` |
| 攻城消耗 | NPC攻城类任务→攻城时消耗兵力+粮草 | MAP-4攻城消耗: `出征兵力≥驻防×2.0 + 粮草×500` |
| 失败惩罚 | NPC攻城类任务→攻城失败时兵力损失 | MAP-4攻城奖励: `损失30%出征兵力，粮草不返还` |
| 占领条件 | NPC攻城类任务→城防归零即占领 | MAP-4占领条件: `城防值归零（唯一条件）` |
| 士兵羁绊→城防 | 士兵NPC结盟"同袍之义"城防+15% | 叠加在城防公式: `(1 + 城防科技加成 + 0.15)` |

> ⚠️ **统一声明引用**: 以上四项均以MAP PRD v1.1统一声明为准。旧公式"基础城防(500~5000)+城墙等级+驻防×0.5"、"出征距离×50+城防等级×100"、"返还50%兵力"、"城防归零+守军全灭+战力≥1.3倍"均已废弃。

### 6.2 CBT战斗系统依赖
