## 三、数据流设计

```
远征系统
  │
  ├─ 路线选择
  │   ├─ 进入远征营 → 显示路线地图(树状分支结构)
  │   ├─ 节点状态: ✅已通关 / 🔄行军中 / 🔒未解锁
  │   ├─ 选择路线 → 检查解锁条件(前置区域通关)
  │   └─ 路线难度: 简单⭐/普通⭐⭐⭐/困难⭐⭐⭐⭐/奇袭⭐⭐⭐⭐⭐
  │
  ├─ 队伍编成
  │   ├─ 选择最多5名武将(已出战武将灰显互斥)
  │   ├─ 选择阵型(6种): 鱼鳞/鹤翼/锋矢/雁行/长蛇/方圆
  │   ├─ 阵营羁绊: 同阵营≥3名 → 全属性+10%
  │   ├─ 智能编队: 自动填充最优组合
  │   └─ 兵力消耗: 出发每武将20兵力
  │
  ├─ 远征推进
  │   ├─ 出发 → 沿路线行军(显示进度条)
  │   ├─ 遭遇节点 → 自动战斗
  │   │   ├─ 山贼: 低难度普通战斗
  │   │   ├─ 天险: 中等难度地形挑战
  │   │   ├─ Boss: 高难度精英战斗
  │   │   ├─ 宝箱: 直接获取奖励
  │   │   └─ 休息点: 恢复部分兵力
  │   ├─ 战斗结果: 大捷⭐⭐⭐/小胜⭐⭐/惨胜⭐/惜败
  │   └─ 路线完成 → 结算奖励 → 可选重新出发
  │
  ├─ 自动远征
  │   ├─ 设置: 重复次数/失败处理/背包满处理/兵力不足处理
  │   ├─ 自动推进已通关路线(扫荡模式)
  │   ├─ 暂停条件: 兵力耗尽/背包满/连续失败2次/手动
  │   └─ 离线期间: 继续推进(效率×0.85, 72h上限)
  │
  ├─ 扫荡系统
  │   ├─ 解锁: 路线三星通关(大捷×3或累计通关×3)
  │   ├─ 普通扫荡: 扫荡令×1 → 基础奖励×100% (5次/路线/天)
  │   ├─ 高级扫荡: 扫荡令×3 → 基础奖励×150%+保底稀有 (3次/路线/天)
  │   └─ 免费扫荡: 无消耗 → 基础奖励×50% (1次/路线/天)
  │
  └─ 奖励系统
      ├─ 基础奖励: 粮草+铜钱+铁矿+装备碎片+经验(按难度)
      ├─ 掉落: 装备碎片30~80%/武将碎片5~25%/技能书2~20%
      ├─ 首通: 稀有武将碎片×1+元宝×50+声望+20
      └─ 里程碑: 初出茅庐→百战之师→远征名将→天下布武

排行榜
  │
  ├─ 5个维度: 战力榜/财富榜/远征榜/竞技榜/赛季战绩榜
  ├─ 展示: 前100名+自己(底部固定高亮)
  ├─ 刷新: 战力/远征/竞技实时 / 财富/公会每日0:00
  └─ 奖励: 按排名梯度发放(元宝+铜钱+竞技币)
```

- [ ] 远征地图正确展示树状分支路线和5种节点类型
- [ ] 路线按区域逐步解锁，奇袭路线需通关困难路线
- [ ] 队伍编成支持最多5名武将+6种阵型+阵营羁绊
- [ ] 武将互斥规则正确（同一武将不可编入多支队伍）
- [ ] 智能编队正确基于战力和阵营羁绊自动填充
- [ ] 远征战斗全自动，阵型克制正确（克制方全属性+10%）
- [ ] 战斗结果4级评定正确（大捷/小胜/惨胜/惜败）
- [ ] 自动远征按设置正确执行，满足暂停条件时停止
- [ ] 离线远征正确推进（效率×0.85，72h上限）
- [ ] 扫荡系统在三星通关后解锁，3种扫荡类型正确执行
- [ ] 远征奖励正确发放（基础+掉落+首通+里程碑）
- [ ] 排行榜5个维度正确展示和刷新
- [ ] 排行榜每日奖励按排名梯度正确发放

- [ ] UI层分支覆盖率/通过率 100%

### 可玩性验收

- [ ] 远征提供稳定的资源获取渠道（中后期占比30%+）

- [ ] 排行榜提供竞争目标和社交展示

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 远征战斗平衡性 | 路线难度过高/过低 | 按推荐战力倍率(×1.0~×3.0)配置，测试验证 |
| 多队伍并行复杂度 | 武将互斥导致队伍搭配困难 | 智能编队辅助+互斥武将灰显提示 |
| 离线远征数据量大 | 长时间离线多次战斗数据 | 使用摘要计算，仅保存关键结果 |
| 扫荡收益失衡 | 扫荡收益过高导致无需手动 | 每日次数限制(5/3/1次)+扫荡收益≤手动 |
| 排行榜实时性 | 高频更新导致性能问题 | 战力/竞技实时，财富/公会每日刷新 |
