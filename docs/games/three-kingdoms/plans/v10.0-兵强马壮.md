## 三、数据流设计

```
装备生成与获取
  │
  ├─ 关卡掉落
  │   ├─ 普通关卡: 白60%/绿30%/蓝8%/紫2%
  │   ├─ 精英关卡: 白30%/绿40%/蓝22%/紫7%/金1%
  │   └─ Boss关卡: 白10%/绿25%/蓝35%/紫22%/金8%
  │
  ├─ 装备炼制
  │   ├─ 基础炼制: 3件同品质 → 概率高一品质
  │   │   ├─ 3白→绿85%/蓝14%/紫1%
  │   │   ├─ 3绿→蓝80%/紫18%/金2%
  │   │   └─ 3蓝→紫75%/金22%
  │   ├─ 高级炼制: 5件 → 概率更优
  │   ├─ 定向炼制: +定向石 → 指定部位
  │   ├─ 保底: 连续10次未紫→必紫 / 30次未金→必金
  │   └─ 非定向产出部位概率: 武器30%/防具30%/饰品25%/坐骑15%
  │
装备属性计算
  │
  ├─ 主属性 = 基础值 × 品质倍率 × (1 + 强化等级 × 0.15~0.18)
  ├─ 副属性 = 随机抽取 × 品质倍率 × (1 + 强化等级 × 0.05~0.08)
  └─ 特殊词条 = 按品质概率附带(白0% / 蓝5% / 紫20% / 金100%)
  │
装备强化
  │
  ├─ 强化流程
  │   ├─ 选择装备 → 检查强化上限(品质决定)
  │   ├─ 消耗: 铜钱(按等级递增) + 强化石(按等级递增)
  │   ├─ 成功率判定: +1~+2必成 / +3→+4(95%) / +4→+5(90%) / +5起概率递减
  │   ├─ 成功: 等级+1 → 属性提升 → 动画反馈
  │   └─ 失败: +3/+4失败不降级；+5→+6失败降至+4，+6→+7失败降至+5（等级-1） → 失败动画
  │       └─ 使用保护符: 失败不降级(消耗保护符)
  │       └─ 金色+12以上: 失败不降级仅消耗材料
  │
  ├─ 自动强化
  │   ├─ 设置目标等级 → 系统自动连续强化
  │   ├─ 停止条件: 材料不足/铜钱不足/达到目标/手动停止
  │   └─ 可选: 自动使用保护符开关
  │
武将穿戴
  │
  ├─ 穿戴流程
  │   ├─ 从背包拖拽装备到武将槽位 → 部位匹配检查
  │   ├─ 穿戴新装备 → 自动卸下旧装备(返回背包)
  │   └─ 装备属性叠加到武将总战力
  │
  ├─ 一键穿戴推荐
  │   ├─ EquipmentRecommendSystem: 分析武将属性短板
  │   ├─ 排序: 品质优先 > 属性增量优先 > 等级匹配优先 > 套装完整性优先
  │   ├─ 展示更换前后属性对比弹窗
  │   └─ 确认后批量穿戴
  │
  ├─ 红点提示（v3新增）
  │   ├─ 背包存在更好装备 → 武将卡片蓝点
  │   ├─ "今日待办"横幅 → "N件新装备可穿戴" + [去穿戴]
  │   └─ 穿戴后蓝点实时更新
  │
  └─ 套装效果
      ├─ 检测同武将穿戴的同套装件数
      ├─ 套装归属: 金色固定 / 紫色25%概率
      ├─ 2件: 激活2件套效果
      ├─ 4件: 激活4件套效果
      └─ 同一套装效果不叠加
  │
铁匠铺建筑（v3补充）
  │
  ├─ 解锁条件: 主城Lv3
  ├─ 等级限制: ≤主城等级
  ├─ Lv1: 基础强化 + 基础炼制 + 基础分解
  ├─ Lv5: 高级炼制
  ├─ Lv10: 定向炼制
  ├─ Lv15: 自动强化
  └─ 被动产出: 强化材料/小时(Lv1=1, Lv5≈5, Lv10≈15, Lv20≈40)，受主城加成
  │
扫荡与离线装备掉落（v4补充）
  │
  ├─ 扫荡掉落: 三星通关后扫荡，装备掉落概率与手动战斗一致
  │   ├─ 普通关卡扫荡: 白60%/绿30%/蓝8%/紫2%
  │   ├─ 精英关卡扫荡: 白30%/绿40%/蓝22%/紫7%/金1%
  │   └─ Boss关卡扫荡: 白10%/绿25%/蓝35%/紫22%/金8%
  ├─ 离线收益掉落: 按v9.0离线收益规则计算，装备掉落概率与对应关卡一致
  └─ 背包满处理: 扫荡装备部分丢失+提示；离线装备放入临时邮箱(24h)
  │
装备数据持久化（v3补充）
  │
  ├─ 存储格式: 压缩格式(关键属性+种子)，非完整JSON
  ├─ 持久化: localStorage
  └─ 跨会话保持: 保底计数器(紫色10次/金色30次) + 装备实例数据
```

## 四、验收标准

### 功能验收
- [ ] 装备背包正确展示，支持5列网格+分类Tab+排序筛选
- [ ] 四种装备部位（武器/防具/饰品/坐骑）属性正确生成
- [ ] 五级品质（白/绿/蓝/紫/金）颜色和属性倍率正确
- [ ] 基础炼制3件→1件，概率表符合PRD定义；非定向炼制产出部位概率(武器30%/防具30%/饰品25%/坐骑15%)正确
- [ ] 高级/定向/保底炼制各自规则正确执行
- [ ] 保底计数器跨会话保持，触发时必出对应品质
- [ ] 装备属性正确计算（主属性+副属性+特殊词条）
- [ ] 7套套装效果在穿戴2件/4件时正确激活
- [ ] 套装归属：金色固定套装，紫色25%概率附带套装
- [ ] 强化成功率表正确应用（+1~+2必成，+3→+4为95%，+4→+5为90%，+5起递减），失败按PRD成功率表执行（+3/+4失败不降级，+5→+6失败降至+4，+6→+7失败降至+5，以此类推）
- [ ] 金色装备+12以上失败不降级
- [ ] 强化保护符正确消耗，失败时不降级；保护符等级适用范围正确（铜+5~+8/银+8~+12/金全等级）
- [ ] 自动强化按目标等级连续执行，满足停止条件时停止；目标超过品质上限时自动修正

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 炼制概率引发不满 | 连续炼制失败导致挫败感 | 保底机制(10次必紫/30次必金)+概率公示 |
| 强化降级过于严厉 | 玩家不敢强化高等级装备 | +3/+4失败不降级+保护符可用+金色+12以上不降级 |
| 套装效果平衡性 | 某些套装过强导致同质化 | 7套套装各有侧重(攻击/防御/技能/机动)，持续监控数据 |
| 背包管理负担 | 装备过多时管理繁琐 | 批量分解+品质过滤+一键穿戴推荐+自动整理+蓝点提示 |
| 装备数据量 | 大量装备实例占用存储 | 装备数据压缩存储，仅保存关键属性+种子 |
| 铁匠铺等级解锁节奏 | 功能解锁过慢导致体验断层 | 铁匠铺Lv1即开放基础功能，高级功能按等级逐步解锁 |
