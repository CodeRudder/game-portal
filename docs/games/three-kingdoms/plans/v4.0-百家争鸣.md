# v4.0 百家争鸣 — 科技树系统

> **版本目标**：引入3条科技路线（军事/经济/文化），玩家消耗资源研究科技，获得各种加成效果。科技树有前置依赖和互斥分支，增加策略深度。

---

## 一、版本目标

### 核心循环
```
积累资源 → 选择科技路线 → 投入资源研究 → 等待研究时间 → 科技效果生效 → 产出增加
     ↑                                                          │
     └──────────────────────────────────────────────────────────┘
```

### 玩家在v4.0中能做什么
1. 在科技Tab中看到3条科技路线（军事/经济/文化）
2. 查看每个科技节点的效果描述、研究费用、前置条件
3. 投入资源开始研究（有研究时间，可加速）
4. 研究完成后效果自动生效（产出倍率增加等）
5. 选择互斥分支（如阵法精通 vs 连弩术，只能选一个）
6. 看到科技效果对资源产出的实际影响

### 2.1 科技路线

```

         → 屯田制   → 丝绸之路

#### 文化路线（7节点）
```
招贤纳士 → 礼贤下士 → 王道仁政 → 天下归心
         → 招贤令   → 礼贤下士·贰

### 2.2 科技研究机制

#### 研究条件
1. 前置科技已完成
2. 资源足够支付研究费用
3. 互斥分支中未选择另一分支

#### 研究时间
| 层级 | 基础研究时间 | 可加速 |
|:----:|:----------:|:-----:|
| Tier 1 | 30秒 | ✅ |
| Tier 2 | 45-60秒 | ✅ |
| Tier 3 | 80-120秒 | ✅ |
| Tier 4 | 180秒 | ✅ |

> **推理补充**：原constants.ts中研究时间为30-180秒，对于网页放置游戏来说偏长。v4.0先保持原值，但增加"消耗铜钱加速"选项（50%费用加速50%时间）。

#### 科技效果
| 科技 | 效果 | 影响范围 |
|------|------|---------|
| 兵法入门 | 兵力产出 ×1.5 | troops产出 |
| 阵法精通 | 战斗伤害 +20% | 战斗系统 |
| 农耕改良 | 粮草产出 ×1.5 | grain产出 |
| 招贤纳士 | 武将招募费用 -15% | 武将招募 |
| ... | ... | ... |

## 三、数据流设计

### 3.1 引擎→UI
```
ThreeKingdomsEngine
  ├── isTechResearched(id) → boolean
  ├── isTechResearching(id) → boolean
  ├── getTechProgress(id) → { elapsed, total } | null
  ├── getTechCost(id) → Record<string, number>
  ├── getTechRequirements(id) → string[]
  ├── getResearchedTechs() → string[]
  └── getTechMultiplier(target) → number  // 获取特定目标的科技倍率
```

### 3.2 UI→引擎
```
研究科技：
  handleResearchClick(techId)
    → engine.researchTech(id)
      → 检查：前置完成 + 资源足够 + 非互斥
      → 扣除资源 + 开始研究计时
      → emit('techResearched')
```

## 六、数值设计

### 6.1 科技倍率叠加
```
最终产出倍率 = 基础产出 × 阶段倍率 × 科技倍率 × 武将派遣倍率
```

### 6.2 互斥分支选择影响
- 阵法精通(+战斗伤害) vs 连弩术(+攻击速度)：战斗流派选择
- 商贸繁荣(+铜钱) vs 屯田制(+粮草)：经济侧重选择
- 礼贤下士(+武将经验) vs 招贤令(-招募费用)：武将培养选择
