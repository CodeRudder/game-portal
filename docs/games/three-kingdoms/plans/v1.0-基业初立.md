## 二、游戏引擎需求

### 新增子系统
| 子系统 | 职责 | 依赖 |
|--------|------|------|
| SaveSystem | localStorage存取、自动保存、存档版本管理 | 无 |
| ResourceCapSystem | 资源上限计算（基于建筑等级）、溢出检测 | BLD数据 |
| BuildingUnlockSystem | 建筑解锁条件检查（主城等级→其他建筑上限） | BLD数据 |
| OfflineCalculator | 离线时长计算、资源产出补算、衰减系数 | RES产出公式 |

### 改进现有子系统
| 子系统 | 改进内容 |
|--------|----------|
| ThreeKingdomsEngine | 确保update(dt)正确触发资源产出；暴露getResources()/getResourceRate()/getResourceCap() |
| ResourcePointSystem | 连通UI，确保产出数据实时可读 |
| ChineseBuildingRenderer | 连通建筑升级逻辑，等级变化时更新渲染 |
| constants.ts | 验证BUILDINGS数组包含完整升级费用表、产出公式参数 |

### 基础设施
| 组件 | 说明 |
|------|------|
| GameLoop | 统一游戏主循环（requestAnimationFrame），100ms tick驱动资源产出 |
| EventBus | 全局事件总线，建筑升级→资源产出变化→UI刷新 |
| UIComponent Library | 通用面板/弹窗/Toast组件库，后续版本复用 |

## 三、数据流设计

```
游戏启动
  │
  ├─ 加载存档 (localStorage) 或 初始化新档
  │   └─ SaveSystem.load() → 检测离线时长 → OfflineCalculator.calculate()
  │
  ├─ 主循环启动 (requestAnimationFrame)
  │   └─ 每100ms tick:
  │       ├─ ResourceCapSystem: 计算当前资源上限
  │       ├─ 资源产出: resources += productionRate * dt (不超过上限)
  │       ├─ 建筑状态: 检查升级队列完成
  │       └─ 每30s: SaveSystem.autoSave()
  │
  ├─ 玩家操作
  │   ├─ 点击"升级建筑"
  │   │   ├─ BuildingUnlockSystem: 检查解锁条件
  │   │   ├─ 检查资源是否足够 (RES消耗表)
  │   │   ├─ 扣除资源 → 建筑等级+1
  │   │   ├─ 更新产出速率
  │   │   └─ Toast: "建筑升级成功"
  │   │
  │   └─ Tab切换 → 中央场景区切换 (天下/出征/武将/科技/建筑/声望/更多▼)
  │
  └─ UI刷新 (React re-render)
      ├─ 资源栏: 显示最新值 + 产出速率 + 容量进度条
      ├─ 建筑面板: 等级/状态/升级费用/产出
      └─ 中央场景: 建筑俯瞰渲染
```

## 四、验收标准

### 功能验收
- [ ] 5种核心资源（粮草/铜钱/兵力/科技点/天命）每秒自动增长，数值正确
- [ ] 8座建筑可建造/升级，消耗正确数量资源
- [ ] 建筑升级后资源产出速率增加
- [ ] 资源达到上限后停止增长，显示容量警告
- [ ] 主城等级限制其他建筑等级上限
- [ ] 资源不足时升级按钮灰显+红色提示
- [ ] Tab切换正常（天下/出征/武将/科技/建筑/声望/更多▼）
- [ ] 游戏日历显示年号/季节/天气
- [ ] 自动保存：刷新页面后进度保留
- [ ] 离线收益：关闭浏览器再打开，补算离线资源

### 质量验收
- [ ] pnpm run build 成功
- [ ] 游戏逻辑层分支覆盖率 100%
- [ ] UI层分支覆盖率/通过率 100%
- [ ] 无控制台错误

### 可玩性验收
- [ ] 核心循环连通：建筑升级 → 资源产出增加 → 解锁更多建筑 → 继续升级
- [ ] 玩家能感受到进度：资源数值持续增长、建筑等级提升、产出速率加快
- [ ] 30秒内能理解玩法：看到资源增长→点击建筑→升级→产出增加

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 引擎update(dt)未正确计算资源产出 | 核心循环断裂 | 优先验证engine.getResources()返回值，编写单元测试覆盖产出公式 |
| constants.ts升级费用表不完整 | 建筑升级失败 | 检查并补全所有建筑的1~50级费用数据 |
| 建筑解锁条件逻辑未实现 | 玩家可跳过解锁直接升级 | 实现BuildingUnlockSystem，主城等级→建筑上限映射表 |
| 离线收益计算不准确 | 玩家回归体验差 | 简化版先实现：基础产出×离线时长×衰减系数 |
| UI组件库不统一 | 后续版本重复开发 | 本版本建立通用Panel/Modal/Toast组件，文档化接口 |
