# v1.0 基业初立 — 最小可玩版本

> **版本目标**：玩家打开游戏后，能看到资源在增长、能建造和升级建筑、能感受到"挂机→资源增长→升级建筑→更多资源"的核心循环。这是整个游戏的基石。

---

## 一、版本目标

### 一句话描述
玩家拥有一个初始城池，4种资源自动产出，可以建造/升级8种建筑来加速资源积累。

### 核心循环
```
挂机等待 → 资源自动增长 → 点击建造/升级建筑 → 建筑加速产出 → 更多资源
     ↑                                                        │
     └────────────────────────────────────────────────────────┘
```

### 玩家在v1.0中能做什么
1. 看到顶部资源栏实时显示4种资源的数量和每秒产出速率
2. 在建筑面板中查看8种建筑卡片（名称、等级、产出、升级费用）
3. 点击建筑卡片进行建造（首次）或升级
4. 资源不足时看到灰色"不可购买"状态和提示
5. 建筑升级后看到产出数值增长
6. 点击主城区域获得少量额外资源（点击产出）
7. 观察资源飘字动画反馈（+1 粮草 等）

## 二、功能清单

### 2.1 资源系统（核心）

#### 资源定义
| 资源ID | 名称 | 图标 | 初始值 | 用途 |
|--------|------|:----:|:-----:|------|
| grain | 粮草 | 🌾 | 500 | 建造/升级基础资源 |
| gold | 铜钱 | 💰 | 300 | 建造/升级/招募通用货币 |
| troops | 兵力 | ⚔️ | 100 | 征服领土/战斗消耗 |
| destiny | 天命 | 👑 | 0 | 声望转生货币（v9.0使用） |

> **推理补充**：原constants.ts定义了7种资源（grain/gold/iron/wood/troops/destiny/morale），v1.0先只用4种核心资源，iron/wood/morale在后续版本解锁。这简化了初始体验，让玩家不用理解太多资源类型。

#### 资源产出公式
```
每秒产出 = 建筑基础产出 × 建筑等级 × 阶段倍率 × 科技倍率
```
- v1.0中阶段倍率=1.0（固定黄巾之乱阶段）
- v1.0中科技倍率=1.0（无科技研究）
- 建筑基础产出见constants.ts BUILDINGS定义

#### 资源显示
- 顶部资源栏显示：图标 + 名称 + 当前数量 + 每秒产出速率
- 数量格式化：K/M/B/T自动转换
- 资源变化时有飘字动画（+XX 粮草）

### 2.2 建筑系统（核心）

#### v1.0可用建筑（8种）
| ID | 名称 | 图标 | 产出资源 | 基础费用 | 解锁条件 |
|----|------|:----:|---------|---------|---------|
| farm | 屯田 | 🌾 | grain | grain:10 | 初始解锁 |
| market | 商行 | 💰 | gold | grain:50 | 初始解锁 |
| barracks | 军营 | ⚔️ | troops | grain:30,gold:20 | 累计100粮草 |
| smithy | 铁匠铺 | 🔨 | iron | gold:100,wood:30 | 军营Lv.3 |
| academy | 太学 | 📚 | gold | gold:200 | 累计500铜钱 |
| clinic | 药庐 | 💊 | morale | gold:80,grain:60 | 累计300粮草 |
| wall | 城防 | 🏰 | wood | gold:150,iron:50 | 军营Lv.5 |
| tavern | 招贤馆 | 🏯 | gold | gold:500 | 太学Lv.3 |

> **推理补充**：原constants.ts有13种建筑，v1.0先用前8种核心建筑。后5种（烽火台/钱庄/锻兵坊/茶馆/粮仓）作为进阶建筑在v2.0-v4.0逐步解锁。

#### 建筑交互
1. **查看**：建筑卡片显示名称、图标、等级、当前产出/秒、升级费用
2. **建造**（首次购买）：点击卡片 → 检查资源 → 扣除费用 → 建筑等级变为1
3. **升级**：点击已有建筑卡片 → 检查资源 → 扣除费用 → 等级+1
4. **锁定状态**：未解锁建筑显示为灰色锁定卡片，显示解锁条件文字
5. **资源不足**：可购买但资源不够时，费用数字变红，点击弹出Toast"资源不足"

#### 建筑费用公式
```
升级费用 = 基础费用 × 费用倍率^当前等级
```
- 费用倍率见constants.ts各建筑的costMultiplier（1.07~1.18）

## 三、数据流设计

### 3.1 引擎→UI数据流

```
ThreeKingdomsEngine
  ├── getResources() → { grain: 500, gold: 300, troops: 100, destiny: 0 }
  ├── getProductionRates() → { grain: 0.5, gold: 0.3, troops: 0.1 }
  ├── getBuildingLevel(id) → number
  ├── getBuildingCost(id) → Record<string, number>
  ├── isBuildingUnlocked(id) → boolean
  ├── buyBuildingById(id) → boolean
  └── upgradeBuilding(id) → boolean
```

### 3.2 需要新增的引擎公共方法

| 方法 | 返回值 | 用途 |
|------|--------|------|
| `getResources()` | `Record<string, number>` | 获取所有资源当前数量 |
| `getProductionRates()` | `Record<string, number>` | 获取每秒产出速率 |
| `getBuildingCost(id)` | `Record<string, number>` | 获取建筑升级费用 |
| `isBuildingUnlocked(id)` | `boolean` | 检查建筑是否已解锁 |
| `getUnlockCondition(id)` | `string` | 获取解锁条件描述文字 |

> **注意**：部分方法可能已在引擎中存在但未暴露为public。需要检查并补充。

### 3.3 UI→引擎操作流

```
用户点击建筑卡片
  → handleBuildingClick(buildingId)
    → engine.buyBuildingById(id) 或 engine.upgradeBuilding(id)
      → 成功：addToast("建造成功！") + triggerBuildingFlash(id)
      → 失败：addToast("资源不足！")
```

## 九、已知依赖和风险

### 依赖
1. `IdleGameEngine` 基类的 `onUpdate(dt)` 循环正常工作
2. `BuildingSystem` 的 `purchase()` 和 `getTotalProduction()` 方法正常
3. `ThreeKingdomsRenderStateAdapter` 正确转换引擎状态为渲染数据

### 风险
1. **handleBuildingClick调用错误**：当前代码中 `handleBuildingClick` 调用的是 `(engine as any).buyBuilding()` 而非 `buyBuildingById(id)`，需要修复
2. **资源产出速率未暴露**：`getProductionRates()` 可能需要新增
3. **建筑解锁检查**：`checkUnlocks()` 的条件检查逻辑需要验证
4. **UI组件过大**：5000+行的组件文件，修改时需小心
