## 二、数据流设计

### 2.1 核心数据流

```
游戏启动
  │
  ├─ 加载存档（localStorage）或初始化
  │   └─ ThreeKingdomsEngine.loadState()
  │
  ├─ 启动游戏循环（requestAnimationFrame / setInterval）
  │   └─ 每100ms: engine.update(deltaTime)
  │       ├─ 计算资源产出: resources += production * dt
  │       ├─ 更新建筑状态: 检查升级完成等
  │       ├─ 触发事件: 检查随机事件
  │       └─ 返回 GameRenderState
  │
  ├─ UI更新（React re-render）
  │   ├─ 资源栏: 显示最新资源值
  │   ├─ 建筑面板: 显示建筑等级和状态
  │   └─ 其他面板: 根据状态更新
  │
  └─ 玩家操作
      ├─ 建造/升级建筑 → engine.upgradeBuilding(id)
      │   ├─ 检查资源是否足够
      │   ├─ 扣除资源
      │   ├─ 建筑等级+1
      │   └─ 更新产出速率
      └─ 点击资源 → engine.clickResource(type)
          └─ 增加少量资源
```

### 2.2 关键接口

```typescript
// 引擎需要暴露的方法
interface ThreeKingdomsEngine {
  // 资源
  getResources(): Record<string, number>;
  getResourceRate(resourceId: string): number;
  getResourceCap(resourceId: string): number;
  
  // 建筑
  getBuildings(): Record<string, { level: number; }>;
  canUpgrade(buildingId: string): boolean;
  getUpgradeCost(buildingId: string): Record<string, number>;
  upgradeBuilding(buildingId: string): boolean;
  
  // 存档
  saveState(): void;
  loadState(): void;
}
```
