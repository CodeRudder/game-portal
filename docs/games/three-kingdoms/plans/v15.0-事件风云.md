## 三、数据流设计

```
事件触发引擎
  │
  ├─ 触发检测（每分钟tick）
  │   ├─ 随机遭遇: P = 0.03 × (1 + 等级×0.02) × 时间衰减（首次10min后，日上限24次）
  │   ├─ NPC事件: 好感度阈值检测 + 日程匹配（首次20min后，日上限12次）
  │   ├─ 天灾人祸: P = 0.015 × (1 + 关卡难度×0.01)（首次30min后，日上限8次）
  │   ├─ 限时机遇: 固定时段(10:00/14:00/20:00)触发
  │   └─ 冷却检查: 同类型60min / 同事件4h / 新手保护30min
  │
  ├─ 事件分发
  │   ├─ 优先级排队（6级：限时>天灾>剧情>随机>NPC>里程碑）
  │   ├─ 急报横幅展示（最多缓存5条，3秒自动收起）
  │   └─ 玩家点击 → 打开事件详情弹窗
  │
  ├─ 选项处理
  │   ├─ 展示2-3个选项（后果标注：🟢正面/🔴负面/🟡中性）
  │   ├─ 玩家选择 → 计算后果（资源变动+声望+好感度）
  │   ├─ 概率后果：按概率随机选择结果
  │   └─ 连锁事件：检查后续事件ID → 触发下一环节
  │
  ├─ 连锁事件追踪
  │   ├─ 链ID → 当前节点 → 历史选择记录
  │   ├─ 分支条件判定 → 确定后续节点
  │   └─ 链完成 → 最终奖励 + 链状态归档
  │
  ├─ 离线事件处理
  │   ├─ 正面事件 → 自动选择"保守"选项 → 50%奖励
  │   ├─ 负面事件 → 按防御值自动减免损失
  │   ├─ 限时机遇 → 错过即消失
  │   └─ 剧情事件 → 暂停等待玩家上线
  │
  └─ 事件日志
      ├─ 所有已处理事件记录（最近100条）
      ├─ 连锁事件链进度
      └─ 离线事件汇总报告

活动系统
  │
  ├─ 限时活动流程
  │   ├─ 活动开始 → 生成任务列表+里程碑+商店
  │   ├─ 任务完成 → 代币+积分奖励 → 里程碑推进
  │   ├─ 代币兑换商店（七阶体系，按积分解锁阶层）
  │   ├─ 排行榜每5min刷新，活动结束1h内结算
  │   └─ 离线进度：限时30% / 赛季50% / 日常100%
  │
  ├─ 节日活动
  │   ├─ 特殊玩法（春节放烟花/端午龙舟/中秋猜灯谜）
  │   ├─ 限定剧情事件
  │   ├─ 独占奖励（限定武将/皮肤/建筑外观）
  │   └─ 离线50%效率自动累积
  │
  └─ 签到系统
      ├─ 7天循环（铜钱×1000→粮草×1000→日常代币×100→兵力×500→加速道具×1→日常代币×200→求贤令×1）
      ├─ 连续加成：3天+20%日常代币获取/7天+50%日常代币获取
      ├─ 补签：元宝×50/次，每周最多2次
      └─ 累计奖励：14天/21天/28天里程碑（跨循环累计，断签不重置累计天数）
```

## 五、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 事件池内容量不足 | 玩家快速看完全部事件产生重复感 | 20+模板×多选项分支=60+种体验，后续版本持续扩充 |
| 连锁事件分支爆炸 | 数据量和复杂度失控 | 每条链最多5个节点，2-3个分支，总量可控 |
| 限时机遇错过体验差 | 玩家因错过限时事件产生挫败感 | 每日3次机会+错过仅损失一次+不影响核心进度 |
| 活动商店七阶体系复杂 | 玩家不理解阶层解锁规则 | UI明确标注解锁条件+阶层进度条+推荐标记 |
| 离线事件自动处理不满意 | 玩家想自己选择但被自动处理 | 离线事件堆弹窗展示结果+仅正面事件自动保守处理 |
| 事件触发过于频繁打断游戏 | 放置体验被事件打断 | 同类型60min冷却+每日上限+可设置事件频率 |
