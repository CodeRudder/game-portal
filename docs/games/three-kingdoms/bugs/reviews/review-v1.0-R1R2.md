## Phase 3 修复（R1→R2）

### P1-1: 资源栏未固定顶部
- 根因：tk-game-frame使用固定1280×800 + scale缩放，导致内容被挤压
- 修复：改为flex column全屏布局，资源栏order:1固定顶部

### P1-2: Tab栏不在底部
- 根因：同上，且DOM顺序Tab栏在场景区之前
- 修复：Tab栏order:3移到底部，场景区order:2中间自适应

### P1-3: 字体仅10px
- 修复：.tk-res-cap和.tk-res-rate从10px→12px

## 经验教训
- LL-162: sticky定位在overflow:hidden容器中不生效，应使用flex布局方案
- LL-163: 游戏布局应从设计阶段就确定全屏flex column结构，避免后期重构
- LL-164: scale缩放方案在响应式布局中有局限性，应优先使用flex/grid
