## 问题汇总

### P0 — 必须修复（影响功能/布局破坏）

| # | 组件 | 问题 | 修复建议 |
|---|------|------|---------|
| P0-01 | ThreeKingdomsGame.css | `[style*="position: fixed"] > div` 通配选择器会给所有 fixed 定位元素的子元素强加 `max-height: 80vh`，可能破坏非弹窗组件的布局 | 移除该通配选择器，改为精确的类名选择器如 `.tk-scrollable-content` |
| P0-02 | Panel.css | `.tk-panel-content` 硬编码 `max-height: 800px`，在低分辨率屏幕（如 768px 高度）上会溢出视口 | 改为 `max-height: calc(90vh - 60px)` 或依赖 flex 布局自动分配 |
| P0-03 | RecruitModal.css | `.tk-recruit-results-grid` 在 `@media (max-width: 767px)` 中被声明了两次（重复），第二次会覆盖第一次 | 合并为单次声明 |

### P1 — 强烈建议（体验明显劣化）

| # | 组件 | 问题 | 修复建议 |
|---|------|------|---------|
| P1-01 | HeroCompareModal.css | 完全没有 `@media` 响应式规则，在手机端 `max-width: 600px` 占据全屏但无适配 | 添加手机端适配：底部滑入或全屏模式 |
| P1-02 | GuideOverlay.css | 无 `@media` 适配，位置变体在手机端可能遮挡关键内容 | 添加手机端居中定位规则 |
| P1-03 | HeroStarUpModal.css / HeroStarUpPanel.css | 使用 `@media (max-width: 480px)` 断点，与项目其他组件统一使用的 `767px` 不一致 | 统一为 `@media (max-width: 767px)` |
| P1-04 | HeroTab.css | `.tk-hero-grid` 使用 `grid-template-columns: repeat(4, 160px)` 硬编码，在 768-1024px 屏幕上 4×160=640px 加上 gap 和 padding 可能超出容器 | 改为 `repeat(auto-fill, minmax(140px, 160px))` 或在平板区间使用 3 列 |
| P1-05 | HeroDetailModal.css | `width: 800px` 在 768-1024px 区间接近 `max-width: 95vw`，左右分栏可能挤压 | 添加 `@media (max-width: 1024px)` 单栏布局 |
| P1-06 | ResourceBar.css | `.tk-res-cap` 使用 `var(--tk-text-muted)` (#666) 在 `rgba(30,35,48,0.6)` 背景上对比度不足（约 2.8:1，WCAG AA 要求 4.5:1） | 将容量文字颜色提升至 `#888` 或使用 `var(--tk-text-secondary)` (#A0A0A0) |
| P1-07 | Panel.css | 完全没有手机端 `@media` 适配规则 | 添加 `@media (max-width: 767px)` 规则：全宽、调整 padding |

## 统计

| 等级 | 数量 | 说明 |
|------|------|------|
| P0 | 3 | 必须修复：通配选择器、硬编码高度、重复声明 |
| P1 | 7 | 强烈建议：缺失响应式、断点不一致、对比度不足 |
| P2 | 6 | 优化提升：细节打磨 |

**整体评价**：UI 体系设计规范度高，CSS 变量体系完善，绝大多数面板都有 PC/手机双端适配。主要问题集中在：① 少数组件缺失响应式规则（HeroCompareModal、GuideOverlay）；② 断点不统一（部分用 480px 而非 767px）；③ 个别选择器过于宽泛可能产生副作用；④ 少量文字对比度不足。建议按 P0 → P1 → P2 顺序修复。
