# 代码质量知识 (EVO-CODE)

> **来源**: R1~R20 评测迭代中发现的代码级经验。
> **不归档**: 持续有效，新增时追加。

---

### EVO-CODE-001: 提取即删除
- 代码提取/迁移后，必须立即删除原文件中的重复代码
- **来源**: R1 | 原EVO-001

### EVO-CODE-002: 废弃清理
- 重构完成后立即清理：grep验证零引用→删除废弃文件→`pnpm run build`确认
- `find . -name "bak" -type d` 全局扫描废弃目录（不同域可能独立存在bak/）
- @deprecated标记后下轮必删，删除前确认破坏性为零
- **来源**: R1,R3,R4,R6 | 原EVO-005,023,026,053 合并

### EVO-CODE-003: 文件行数预警
- 每个版本迭代后检查是否有文件接近500行边界，提前拆分
- **来源**: R3 | 原EVO-009

### EVO-CODE-004: data-testid强制
- 所有新增UI元素必须添加data-testid，技术审查时检查缺失
- 命名规范: kebab-case，动态列表项使用模板（如 `resource-item-${id}`）
- 资源图标应有aria-label或data-testid辅助测试
- **来源**: R3,R7 | 原EVO-011,014,056 合并

### EVO-CODE-005: 截图辅助测试
- UI测试应结合截图分析，不能仅依赖DOM文字搜索
- **来源**: R3 | 原EVO-013

### EVO-CODE-006: 合并冲突标记CI检测
- `grep -rn "<<<<<<" src/` 检测，发现立即阻断
- **来源**: R19 | 原EVO-039

### EVO-CODE-007: 性能监控先行原则
- 先写监控→再写优化→最后对比验证。不盲目优化
- **来源**: R20 | 原EVO-043

### EVO-CODE-008: 交互规范代码化
- 转化为可执行审查器: InteractionAuditor/VisualConsistencyChecker/AnimationAuditor
- **来源**: R20 | 原EVO-045

### EVO-CODE-009: as any 零容忍
- engine/ 不允许新增 `as any`，修复优先级: 扩展接口>泛型>精确断言>全局扩展
- **来源**: R6 | 原EVO-054

### EVO-CODE-010: 模块命名一致性
- 模块命名必须与实际目录一致，以实际目录为准更新文档
- **来源**: R6 | 原EVO-055

### EVO-CODE-011: 函数签名冲突检测
- 明确权威版本和委托关系，禁止两模块各自实现相同逻辑
- **来源**: R7 | 原EVO-057
