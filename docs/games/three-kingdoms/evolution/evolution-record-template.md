## 填写指南

| 章节 | 填写时机 | 关注点 |
|------|---------|--------|
| 一、评测范围 | Phase 1 准备阶段 | 明确范围，避免扩大 |
| 二、冒烟结果 | Phase 2 结束 | P0必须修完才能继续 |
| 三、深度评测 | Phase 3 结束 | 每个问题必须有截图证据 |
| 四、修复回归 | Phase 4 每个修复后 | 回归次数体现修复质量 |
| 五、架构审查 | Phase 5 结束 | 聚焦架构违规 |
| 六、封版判定 | Phase 6 | 五项全通过才能封版 |
| 七、经验教训 | Phase 6 复盘 | 这是进化的核心产出 |
| 八、度量数据 | Phase 6 | 数据驱动进化决策 |
| 九、下轮计划 | Phase 6 | 为下一轮准备改进方向 |
