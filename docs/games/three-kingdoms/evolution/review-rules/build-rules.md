# 构建与部署检查规则

> **来源**: EVO-039,071,072 等及评测经验。
> **进化**: CI/构建相关发现新模式时新增。

---

## P1 规则

### BLD-01: 编译必须通过
- 每次变更后 `pnpm run build` 必须通过
- TypeScript 编译 0 错误
- **来源**: 基本要求

### BLD-02: 构建产物完整
- pnpm build 成功 + 游戏入口 JS chunk > 0 bytes
- **来源**: EVO-071

### BLD-03: 基本功能可用性
- 封版前验证: 页面加载、引擎初始化、资源栏显示、Tab切换
- **来源**: EVO-072

### BLD-04: 合并冲突标记检测
- CI 或提交前 `grep -rn "<<<<<<" src/` 检测
- 发现冲突标记立即阻断
- 合并代码后必须 `pnpm run build` 确认
- **来源**: EVO-039

### BLD-05: 引擎 API 冒烟测试
- 每轮 UI 测试前先验证: 引擎模块可加载、新子系统正确导出、类可实例化
- 作为 UI 测试的"测试0"最先执行
- **来源**: EVO-041
