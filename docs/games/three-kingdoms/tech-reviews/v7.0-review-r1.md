## P0 阻断问题

### P0-1: Quest域未在门面导出
- **文件**: `engine/index.ts`, `engine/exports-v6.ts`
- **问题**: QuestSystem、QuestTrackerSystem、ActivitySystem 未在引擎门面中导出，外部消费者无法直接 import
- **影响**: 任务系统无法被UI组件正确引用，可能导致运行时错误
- **修复**: 在 `exports-v6.ts` 中补充 quest 域导出

### P0-2: Event域大量系统未在门面导出
- **文件**: `engine/exports-v6.ts`
- **问题**: 以下系统未在门面中导出：
  - `ChainEventSystem` + 类型
  - `ChainEventEngine` + 类型
  - `StoryEventSystem` + 类型
  - `EventTriggerEngine`
  - `OfflineEventHandler`
- **影响**: 连锁事件、历史剧情事件等v7.0核心功能无法被UI层引用
- **修复**: 在 `exports-v6.ts` 中补充所有event域导出

## P2 改进建议

### P2-1: NPC域缺少PatrolPathCalculator门面导出
- **文件**: `engine/exports-v6.ts`
- **问题**: PatrolPathCalculator, GiftPreferenceCalculator 及其类型未在门面中导出
- **影响**: 外部消费者需要直接引用子模块路径

### P2-2: Event域存在重复系统
- **问题**: event目录下存在功能重叠的系统：
  - `ChainEventSystem.ts` vs `ChainEventEngine.ts` vs `ChainEventSystemV15.ts` — 三个连锁事件系统
  - `EventTriggerSystem.ts` vs `EventTriggerEngine.ts` — 两个事件触发系统
  - `OfflineEventSystem.ts` vs `OfflineEventHandler.ts` — 两个离线事件处理器
- **建议**: v8.0统一为单一实现，标记废弃的文件

### P2-3: NPCPatrolSystem.ts 接近500行边界 (496行)
- **文件**: `engine/npc/NPCPatrolSystem.ts`
- **问题**: 496行，接近500行限制
- **建议**: 预防性拆分

### P2-4: NPCTrainingTypes.ts 可合并到 NPCTrainingSystem.ts
- **文件**: `engine/npc/NPCTrainingTypes.ts` (257行)
- **问题**: 独立类型文件体积较小，但NPCTrainingSystem.ts本身已有365行，合并后不超限
- **建议**: 维持现状，保持关注
