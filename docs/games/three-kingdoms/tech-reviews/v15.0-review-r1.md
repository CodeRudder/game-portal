## P1 重要问题（已修复）

### P1-1: EventTriggerEngine 和 ChainEventEngine 未注册到引擎 ✅
- **文件**: `engine/engine-event-deps.ts`, `engine/engine-getters.ts`, `engine/engine-getters-types.ts`
- **问题**: v15.0 新增的 EventTriggerEngine（触发条件+概率+冷却）和 ChainEventEngine（分支追踪+快照）未注册到 EventSystems 和引擎
- **影响**: UI 层无法通过引擎门面访问 v15.0 事件触发引擎和连锁事件引擎
- **修复**:
  - engine-event-deps.ts: EventSystems 接口添加 triggerEngine 和 chainEngine 字段
  - engine-event-deps.ts: createEventSystems() 添加实例化
  - engine-event-deps.ts: initEventSystems() 添加初始化
  - engine-getters.ts: 添加 getter
  - engine-getters-types.ts: 添加类型声明

### P1-2: OfflineEventHandler 未注册到引擎 ✅
- **文件**: `engine/engine-event-deps.ts`
- **问题**: OfflineEventHandler 是 v15.0 离线事件堆积处理器，未注册到 EventSystems
- **影响**: 离线事件堆积处理功能无法通过引擎访问
- **修复**: 注册到 EventSystems 并添加 getter

## P2 改进建议（记录）

### P2-1: event/ 目录文件数量过多（17个.ts文件）
- **问题**: 事件模块包含 EventTriggerSystem + EventTriggerEngine + ChainEventSystem + ChainEventEngine + ChainEventSystemV15 + EventChainSystem 等多个功能重叠文件
- **建议**: 后续版本合并重复系统（ChainEventSystem vs EventChainSystem vs ChainEventEngine），减少冗余

### P2-2: ChainEventSystemV15 与 ChainEventSystem 功能重叠
- **文件**: `engine/event/ChainEventSystemV15.ts` (458行) vs `engine/event/ChainEventSystem.ts` (453行)
- **问题**: 两个文件实现类似功能，增加维护负担
- **建议**: v17.0 统一为一个实现

### P2-3: EventEngine.ts 与 EventTriggerSystem.ts 职责模糊
- **问题**: EventEngine.ts(360行) 和 EventTriggerSystem.ts(487行) 都涉及事件触发逻辑
- **建议**: 明确职责边界，EventEngine 负责核心引擎，EventTriggerSystem 负责系统级管理
