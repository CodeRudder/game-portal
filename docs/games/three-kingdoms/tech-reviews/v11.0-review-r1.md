## 五、`as any` 检测

| 文件 | 行号 | 代码 | 严重性 | 说明 |
|------|------|------|--------|------|
| ExpeditionSystem.ts | 427 | `(data as any).routeNodeStatuses` | ⚠️ P2 | 反序列化兼容旧存档字段 |

```typescript
// 问题代码
const savedNodeStatuses = (data as any).routeNodeStatuses as Record<string, Record<string, string>> | undefined;
```

**分析**: 此处 `as any` 用于读取旧版存档中可能不存在的 `routeNodeStatuses` 字段，属于合理的向后兼容处理。建议添加注释说明兼容的存档版本，或使用 `hasOwnProperty` 检查替代。

```
ExpeditionSystem (456行) — 聚合根

  ├── ExpeditionBattleSystem (338行) — 战斗子系统
  ├── ExpeditionRewardSystem (381行) — 奖励子系统
  ├── AutoExpeditionSystem (470行) — 自动远征子系统
```
