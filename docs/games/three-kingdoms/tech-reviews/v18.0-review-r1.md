## 四、问题详情

### P1: guide 模块未集成到 ThreeKingdomsEngine 主类

**文件**: `engine/ThreeKingdomsEngine.ts`

**现象**: ThreeKingdomsEngine 没有创建/注册 guide 子系统实例。guide 模块的 7 个子系统（TutorialStateMachine、TutorialStepManager 等）虽然实现了 ISubsystem 接口，但并未在引擎构造函数中实例化，也未注册到 SubsystemRegistry。

**影响**:
- UI 层 GuideOverlay 通过 `registry.get('tutorial-state')` 获取 TutorialStateMachine，但引擎未注册该子系统
- 引导状态机无法参与引擎的统一 tick/update 循环
- 引导进度无法通过引擎的 save/load 机制持久化

**修复建议**:
1. 在 ThreeKingdomsEngine 中添加 guide 系统实例化和注册
2. 在 engine-getters.ts 中添加 `getTutorialStateMachine()` 等 getter
3. 在 engine-save.ts 中集成 TutorialSaveData 的保存/加载

**状态**: ⏳ 待修复（非阻塞，GuideOverlay 有 localStorage 回退机制）

### P2: GuideOverlay 门面违规

**文件**: `src/components/idle/panels/hero/GuideOverlay.tsx:12`

```typescript
import { TutorialStateMachine } from '@/games/three-kingdoms/engine/guide';
```

应改为通过门面导入：
```typescript
import { TutorialStateMachine } from '@/games/three-kingdoms/engine';
```

**影响**: 绕过引擎统一导出层，可能导致循环依赖或构建优化问题。

**修复建议**: 修改导入路径。
