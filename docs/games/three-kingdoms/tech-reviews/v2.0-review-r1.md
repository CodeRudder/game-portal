当前 `FormationPanel.tsx` 已完整集成羁绊功能：

### 4.2 RecruitModal 品质动画硬编码（⚠️ 未变）

```typescript
// RecruitModal.tsx:56-62
const QUALITY_REVEAL_ANIM: Record<Quality, string> = {
  COMMON: 'tk-reveal-common',
  FINE: 'tk-reveal-fine',
  RARE: 'tk-reveal-rare',
  EPIC: 'tk-reveal-epic',
  LEGENDARY: 'tk-reveal-legendary',
};
```

**问题分析：**
- CSS class 名称以字符串字面量硬编码在组件内部
- 若品质体系扩展（如新增 `MYTHIC` 品质），需同时修改此映射 + CSS 文件 + hero.types.ts
- 建议将映射移至 hero 域的常量配置中，或通过品质枚举动态生成 class 名

**修复建议：**
```typescript
// hero-config.ts 或 hero.types.ts 中统一定义
export const QUALITY_REVEAL_CSS_CLASS: Record<Quality, string> = { ... };
```

## 五、新增发现

### 5.1 bond 门面类型导出缺失（P1）

**问题**：`engine/bond/index.ts` 仅导出 `BondSystem` 类，未导出任何类型。

**影响**：
- `FormationPanel.tsx` 被迫从 `@/games/three-kingdoms/core/bond` 直接导入类型（门面违规）
- `engine/index.ts` 也未从 `core/bond` 重导出类型
- UI 层若需使用 `ActiveBond`、`BondEffect`、`BondType` 等类型，必须绕过 engine 门面

**修复建议**：
```typescript
// engine/bond/index.ts 补充
export type {
  BondType, BondEffect, ActiveBond, FormationBondPreview,
  BondPotentialTip, StoryEventDef, StoryEventCondition,
  StoryEventReward, HeroFavorability, BondSaveData,
} from '../../core/bond';
export { BOND_NAMES, BOND_DESCRIPTIONS, BOND_SAVE_VERSION } from '../../core/bond';
```

### 5.2 HeroStarUpModal.css 余量不足（P1）

当前 489 行，距 500 行限制仅 11 行。包含星级动画、突破特效、手机端适配等样式。后续版本若新增特效将立即超限。

### 5.3 data-testid 严重不足（P1）

核心交互组件（HeroTab / RecruitModal / FormationPanel / HeroDetailModal）均无 `data-testid`，仅升星相关组件有覆盖。这会影响：
- E2E 测试的选择器稳定性（依赖 CSS class 或文本匹配）
- CI/CD 中的自动化回归测试可靠性
- 组件关键交互点的可测试性
