## P0 阻断问题（已修复）

### P0-1: PrestigeShopSystem 和 RebirthSystem 未注册到 R11Systems ✅
- **文件**: `engine/engine-r11-deps.ts`
- **问题**: PrestigeShopSystem 和 RebirthSystem 有 getter（engine-getters.ts 引用 `this.r11.prestigeShopSystem` / `this.r11.rebirthSystem`），但 R11Systems 接口和 createR11Systems() 中缺失这两个字段
- **影响**: 运行时调用 `getPrestigeShopSystem()` 或 `getRebirthSystem()` 将抛出 undefined 异常
- **修复**:
  - engine-r11-deps.ts: R11Systems 接口添加 prestigeShopSystem 和 rebirthSystem 字段
  - engine-r11-deps.ts: createR11Systems() 添加实例化
  - engine-r11-deps.ts: registerR11Systems() 添加注册
  - engine-r11-deps.ts: resetR11Systems() 添加重置

### P0-2: RebirthSystemV16 测试文件引用不存在的类 ✅
- **文件**: `engine/prestige/__tests__/RebirthSystemV16.test.ts`
- **问题**: 测试文件名为 RebirthSystemV16.test.ts 但实际测试的是 RebirthSystem 中的 v16 功能，不存在独立的 RebirthSystemV16 类
- **影响**: 文件命名可能引起误解
- **修复**: 记录现状，v16 深化功能已集成到 RebirthSystem 和 HeritageSimulation(工具函数模块)中
