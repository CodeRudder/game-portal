## 问题清单

### P1 — 必须修复

#### P1-1: ResourceSystem 反序列化存在 `as any` 类型逃逸
- **文件**: `src/games/three-kingdoms/engine/resource/ResourceSystem.ts:391`
- **代码**: `this.resources[type] = Math.max(0, Number(val) || 0) as any;`
- **原因**: `type` 为 `ResourceType`（string），用索引赋值时 TS 推断类型不匹配
- **修复建议**: 使用类型断言 `as Resources[typeof type]` 或逐字段赋值
- **影响**: 类型安全性降低，可能导致运行时类型错误

#### P1-2: 日历测试文件大量 `as any`（11 处）
- **文件**: `src/games/three-kingdoms/engine/calendar/__tests__/calendar-advanced.test.ts`（8 处）
- **文件**: `src/games/three-kingdoms/engine/calendar/__tests__/CalendarSystem.test.ts`（3 处）
- **代码**: `calendar.init(deps as any)` 等模式
- **原因**: 测试 mock 对象未完整实现 ISystemDeps 接口
- **修复建议**: 创建 `createMockDeps()` 工厂函数（参考 ResourceSystem 测试的做法）
- **影响**: 测试类型安全性降低，接口变更时测试不会报错

### P2 — 建议修复

#### P2-1: v1.0 核心 UI 组件缺失 `data-testid`
- **文件**: 以下 6 个组件无 `data-testid`:
  - `src/components/idle/panels/resource/ResourceBar.tsx`
  - `src/components/idle/panels/building/BuildingPanel.tsx`
  - `src/components/idle/panels/building/BuildingUpgradeModal.tsx`
  - `src/components/idle/panels/building/BuildingIncomeModal.tsx`
  - `src/components/idle/three-kingdoms/CalendarDisplay.tsx`
  - `src/components/idle/IdleResourceBar.tsx`
- **影响**: E2E 测试和集成测试无法通过 testid 定位元素
- **修复建议**: 为每个组件的根元素和关键交互元素添加 `data-testid`

#### P2-2: 废弃 bak 文件未清理
- **文件**: 
  - `src/components/idle/backup/ThreeKingdomsPixiGame.css.bak`（5054 行）
  - `src/components/idle/backup/ThreeKingdomsPixiGame.tsx.bak`（6079 行）
  - `src/components/idle/ThreeKingdomsPixiGame.tsx.v1.bak`（506 行）
- **总大小**: 11639 行废弃代码
- **影响**: 污染代码库，增加仓库体积
- **修复建议**: 删除所有 .bak 文件和 backup 目录

#### P2-3: 未引用的 exports 辅助文件
- **文件**: 
  - `src/games/three-kingdoms/engine/exports-v9.ts`
  - `src/games/three-kingdoms/engine/exports-v12.ts`
- **原因**: 这两个文件未被任何地方 import（index.ts 已直接导出对应模块）
- **影响**: 死代码，维护负担
- **修复建议**: 确认无引用后删除

#### P2-4: OfflineEarningsCalculator 标记为 DEPRECATED 但仍被使用
- **文件**: `src/games/three-kingdoms/engine/resource/OfflineEarningsCalculator.ts`
- **注释**: `⚠️ DEPRECATED: 本模块是 resource 域的简化版离线收益计算器`
- **实际**: 仍被 `ResourceSystem.ts` 正常引用

- **修复建议**: 移除 DEPRECATED 标记（v1.0 的简化版是正确的），或明确注释仅在 v1.0 使用
