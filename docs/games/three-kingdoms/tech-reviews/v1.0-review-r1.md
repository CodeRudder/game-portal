### 引擎层架构评价

**DDD 分层合规性：✅ 良好**
- 引擎层 → 子系统领域层 → 配置层 的依赖方向严格单向
- 各子系统（ResourceSystem / BuildingSystem / CalendarSystem）实现了 `ISubsystem` 接口，通过 `ISystemDeps` 注入依赖
- 子系统之间通过 EventBus 通信，无直接互相引用
- 配置层文件（`*-config.ts` / `*-types.ts`）严格零逻辑

**门面模式：✅ 已实现**
- `engine/index.ts` 作为统一导出入口，re-export 所有引擎 API
- UI 层理论上应通过此门面访问引擎

**拆分策略：✅ 有效**
- `ThreeKingdomsEngine.ts` 已拆分出 `engine-tick.ts`、`engine-save.ts`、`engine-building-ops.ts` 等模块
- 各子系统域内部也进行了合理拆分（如 `resource-calculator.ts`、`OfflineEarningsCalculator.ts`）
