- `EquipmentSystem` 作为聚合根(412行), 将背包管理委托给 `EquipmentBagManager`, 分解委托给 `EquipmentDecomposer`

## 三、代码质量检查

### ✅ 优点
1. **职责单一**: 每个文件职责清晰, 聚合根通过委托降低行数
2. **类型安全**: 核心类型集中在 `core/equipment/`, 引擎层引用不重复定义
3. **事件驱动**: 通过 `emitEvent` 回调解耦, 不直接依赖 EventBus 实现
4. **保底机制**: ForgePityManager 独立管理, 三级保底(蓝/紫/金)
5. **存档支持**: EquipmentSystem.serialize/loadState 完整
6. **配置外置**: 概率权重、成本、阈值全部定义为常量, 可维护

## 四、数据流验证

```
获取装备 → EquipmentGenerator → EquipmentBagManager.add() → emit('equipment:added')
穿戴装备 → EquipmentSystem.equip() → HeroEquipSlots更新 → 属性计算
锻造装备 → EquipmentForgeSystem.forge() → 消耗输入 → 生成输出 → 保底更新
强化装备 → EquipmentEnhanceSystem.enhance() → 成功率判定 → 属性增长/降级
套装效果 → EquipmentSetSystem.getActiveSetBonuses() → 件数统计 → 效果激活
分解装备 → EquipmentDecomposer.decompose() → 奖励计算 → 装备销毁 → emit('equipment:decomposed')
推荐装备 → EquipmentRecommendSystem.recommend() → 评分排序 → 最优组合
```

**数据流完整性**: ✅ 所有路径闭环, 无死代码
