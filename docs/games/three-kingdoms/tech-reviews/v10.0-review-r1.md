## 问题清单

### P1 问题（6个）

#### P1-1: 强化成功率 +1~+3 非必成
- **文件**: `core/equipment/equipment-config.ts:137-139`
- **PRD要求**: "+1~+3必成"
- **实际情况**: ENHANCE_SUCCESS_RATES[1]=0.95(1→2), [2]=0.90(2→3)，非100%
- **修复方案**: 将索引0、1、2的值全部改为1.0
```typescript
export const ENHANCE_SUCCESS_RATES = [
  1.0,   // 0→1: 100% (必成)
  1.0,   // 1→2: 100% (必成)
  1.0,   // 2→3: 100% (必成)
  0.80,  // 3→4: 80%
  // ... 后续不变
];
```

#### P1-2: 特殊词条概率与PRD严重不符
- **文件**: `core/equipment/equipment-config.ts:393-399`
- **PRD要求**: 白0% / 蓝5% / 紫20% / 金100%
- **实际情况**: 白0% / 绿0% / 蓝25% / 紫50% / 金80%
- **修复方案**: 修改 `RARITY_SPECIAL_EFFECT_CHANCE`
```typescript
export const RARITY_SPECIAL_EFFECT_CHANCE: Record<EquipmentRarity, number> = {
  white: 0,
  green: 0,
  blue: 0.05,
  purple: 0.20,
  gold: 1.0,
};
```

#### P1-3: 保底阈值配置不一致（ForgePityManager vs equipment-config）
- **文件A**: `engine/equipment/ForgePityManager.ts:18-20` — `PITY_GOLD_THRESHOLD = 30`
- **文件B**: `core/equipment/equipment-config.ts:121-125` — `targetedGoldPity: 20`
- **PRD要求**: 连续30次未金→第31次必金
- **实际情况**: ForgePityManager使用硬编码30（正确），但core配置文件中写的是20（错误）
- **影响**: 配置文件未作为single source of truth，可能导致维护混乱
- **修复方案**: ForgePityManager应引用core配置，或将core配置改为30
```typescript
// ForgePityManager.ts 改为引用配置
import { FORGE_PITY_THRESHOLDS } from '../../core/equipment';
const PITY_PURPLE_THRESHOLD = FORGE_PITY_THRESHOLDS.advancedPurplePity; // 10
const PITY_GOLD_THRESHOLD = FORGE_PITY_THRESHOLDS.targetedGoldPity;     // 需改为30
```

#### P1-4: 基础炼制3蓝→金概率与PRD不符
- **文件**: `engine/equipment/EquipmentForgeSystem.ts:45`
- **PRD要求**: 3蓝→紫75%/金22%（总计97%，可能有3%其他或PRD笔误）
- **实际情况**: `blue: { purple: 75, gold: 25 }`
- **修复方案**: 确认PRD意图后修改。如果PRD的97%是笔误，则代码75/25是合理的；如果PRD正确，则应改为75/22（剩余3%回退到蓝）
```typescript
// 方案A: 严格按PRD
blue: { purple: 75, gold: 22 },
// 方案B: PRD笔误，保持代码不变（需与策划确认）
```

#### P1-5: 默认背包容量与PRD不符
- **文件**: `core/equipment/equipment-config.ts:329`
- **PRD要求**: "50格默认"
- **实际情况**: `DEFAULT_BAG_CAPACITY = 100`
- **修复方案**: 改为50或与策划确认
```typescript
export const DEFAULT_BAG_CAPACITY = 50;
```

#### P1-6: EquipmentSetSystem重复实例化
- **文件**: `engine/engine-r11-deps.ts:114-115`
- **问题描述**: `createR11Systems()` 中创建了两个 `EquipmentSetSystem` 实例
  - 第114行: `equipmentSetSystem: new EquipmentSetSystem(eq)`
  - 第115行: `equipmentRecommendSystem: new EquipmentRecommendSystem(eq, new EquipmentSetSystem(eq))`
- **影响**: `EquipmentRecommendSystem` 内部的 `setSystem` 与注册表中的 `equipmentSetSystem` 是不同实例，套装数据不同步
- **修复方案**: 复用同一实例
```typescript
const setSystem = new EquipmentSetSystem(eq);
return {
  // ...
  equipmentSetSystem: setSystem,
  equipmentRecommendSystem: new EquipmentRecommendSystem(eq, setSystem),
  // ...
};
```

### P2 问题（5个）

#### P2-1: EquipmentGenerator.ts 与 EquipmentGenHelper.ts 代码重复
- **文件A**: `engine/equipment/EquipmentGenerator.ts` (228行)
- **文件B**: `engine/equipment/EquipmentGenHelper.ts` (150行)
- **问题描述**: 两个文件包含几乎相同的函数（generateUid, resetUidCounter, weightedPickRarity, generateBySlot, generateByTemplate, genMainStat, genSubStats, genSpecialEffect, isSlot, seedPick）
- **影响**: 维护成本翻倍，修改一处容易遗漏另一处
- **修复方案**: 删除 `EquipmentGenerator.ts`，将门面导出 `index.ts` 中的引用改为指向 `EquipmentGenHelper.ts`

#### P2-2: EquipmentDecomposer.ts 含废弃品质 orange/red
- **文件**: `engine/equipment/EquipmentDecomposer.ts:16-21`
- **问题描述**: `DECOMPOSE_COPPER_BASE` 和 `DECOMPOSE_STONE_BASE` 包含 `orange` 和 `red` 品质，但当前品质体系只有 white/green/blue/purple/gold
- **影响**: 无功能影响（不会被访问到），但属于死代码
- **修复方案**: 移除 orange/red 条目，或改为从 core 配置导入 `DECOMPOSE_OUTPUT`

#### P2-3: EquipmentDecomposer.ts 重复定义分解配置
- **文件A**: `engine/equipment/EquipmentDecomposer.ts:14-25` — 本地硬编码
- **文件B**: `core/equipment/equipment-config.ts` — `DECOMPOSE_OUTPUT`、`DECOMPOSE_COPPER_BASE`、`DECOMPOSE_STONE_BASE`
- **问题描述**: 分解产出数据在两处定义，值不完全一致（Decomposer中purple=2000，config中purple=1000）
- **修复方案**: EquipmentDecomposer 应引用 core 配置的 `DECOMPOSE_OUTPUT`

#### P2-4: UI组件 data-testid 覆盖不足
- **文件**: `components/idle/panels/equipment/EquipmentTab.tsx` — 仅1个 `data-testid="equipment-tab"`
- **文件**: `components/idle/panels/equipment/EquipmentPanel.tsx` — 0个 data-testid
- **缺失项**: 装备卡片、锻造面板、强化面板、详情弹窗、筛选按钮、排序按钮、分解按钮等均缺少 data-testid
- **修复方案**: 添加关键元素的 data-testid
  - `data-testid="equipment-grid"` — 装备网格
  - `data-testid="equipment-card-{uid}"` — 装备卡片
  - `data-testid="forge-panel"` — 锻造面板
  - `data-testid="enhance-panel"` — 强化面板
  - `data-testid="equipment-detail-modal"` — 详情弹窗
  - `data-testid="decompose-btn"` — 分解按钮
  - `data-testid="enhance-btn"` — 强化按钮
  - `data-testid="forge-btn"` — 锻造按钮

#### P2-5: EquipmentForgeSystem 未引用 core 配置的炼制权重
- **文件**: `engine/equipment/EquipmentForgeSystem.ts:42-55`
- **问题描述**: `BASIC_FORGE_WEIGHTS`、`ADVANCED_FORGE_WEIGHTS` 在 Engine 层硬编码，而 core 配置中也有 `BASIC_FORGE_CONFIG`、`ADVANCED_FORGE_CONFIG`、`TARGETED_FORGE_CONFIG`
- **影响**: 配置分散，维护时容易遗漏
- **修复方案**: Engine 层应引用 core 配置，或明确 core 配置仅作为参考文档
