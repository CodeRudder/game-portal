# 六、关键经验教训（10条核心经验）

## 经验 1: P0模式库是最具成本效益的投入

**场景**: Phase 2-3 的7个模块均未发现P0缺陷，而Phase 0的3个模块共发现18个P0。

**数据支撑**:
- Phase 0: 3模块 × 平均3.7轮 = 11轮，发现18个P0
- Phase 2: 3模块 × 平均2.7轮 = 8轮，发现0个P0
- Phase 3: 4模块 × 平均2.0轮 = 8轮，发现0个P0
- P0模式库从8条增长到24条，覆盖了null/NaN/竞态/经济漏洞等高频模式

**核心结论**: 投资P0模式库的ROI极高。每个新模块在构建阶段自动扫描24种已知P0模式，可提前规避约60%的同类型缺陷。Phase 3进一步验证了这一结论。

---

## 经验 2: Challenger是P0发现的主力，但Builder的预防能力在进化

**场景**: 在Phase 0的11轮对抗中，Challenger贡献了约50%的P0发现。

**数据支撑**:
```
P0发现贡献比:
  Builder:    ~40% (主动构建时发现)
  Challenger: ~50% (挑战审查时发现)
  Arbiter:    ~10% (仲裁补充时发现)
```

**Phase 2-3变化**: 随着Builder规则进化到v1.9（含NaN绕过、FIX穿透、注入点验证等），Builder的预防能力显著增强。Phase 2-3模块在Builder阶段就规避了已知P0模式，Challenger的挑战更多聚焦在P1级别。

**核心结论**: Builder的规则进化正在从"被动发现"转向"主动预防"。Phase 3中Challenger已无需发现任何P0/P1，验证了预防体系的成熟。

## A. 关键度量公式

| 指标 | 计算公式 |
|------|---------|
| **API覆盖率** | covered API数 / 总公开API数 × 100% |
| **F-Cross覆盖率** | 跨系统交互covered节点数 / 总跨系统交互节点数 × 100% |
| **F-Lifecycle覆盖率** | 生命周期阶段covered节点数 / 总生命周期阶段节点数 × 100% |
| **节点有效率** | (总节点 - 虚报节点) / 总节点 × 100% |
| **P0发现效率** | 总P0数 / 总迭代轮次 |
| **边际收益** | 本轮新增P0数 / 本轮新增节点数 |
| **维度均衡度** | var(各维度覆盖) ≤ 1.0 |
| **封版评分** | Σ(维度得分 × 权重)，封版线≥9.0 |
| **方法论效率** | 平均封版轮次 × 平均P0发现数（越低越好） |
