## 二、迭代计划

### Sprint 1: 地图基础(P0)

**目标**: 地图渲染→领土选择→基础征服 完整链路

| 任务 | 流程 | 内容 | 验收标准 |
|------|------|------|---------|
| 1.1 | MAP-F01 | 进入天下Tab | 场景切换→地图渲染→己方高亮 |
| 1.2 | MAP-F02 | 地图浏览与缩放 | 拖拽移动→缩放50%~200%→视口裁剪 |
| 1.3 | MAP-F03 | 领土选择与详情 | 点击领土→详情面板(名称/等级/地形/产出/驻防) |
| 1.4 | MAP-F04 | 领土征服(基础) | 校验(相邻+兵力)→部署→胜率预估→出征→结算 |
| 1.5 | MAP-F07 | 筛选与热力图 | 5维度筛选→热力图分档算法 |
| 1.6 | MAP-F01-01 | 新手引导 | 首次进入→遮罩→引导征服→新手礼包 |

**测试用例**:
- [ ] 点击天下Tab→地图正确渲染(20×15六边形网格)
- [ ] 拖拽移动→视口不超出地图边界
- [ ] 缩放50%~200%→瓦片自适应→<60%隐藏气泡
- [ ] 点击领土→详情面板显示完整信息
- [ ] 征服非相邻领土→提示"不相邻"
- [ ] 兵力不足→提示"兵力不足"
- [ ] 征服胜利→领土归己方→自动驻防50%
- [ ] 征服失败→损失30%兵力→领土不变
- [ ] 筛选"高产领地"→前5名高亮
- [ ] 热力图5档颜色正确显示

### Sprint 2: 攻城战系统(P0)

**目标**: 攻城→策略选择→内应信闭环→奖励 完整链路

| 任务 | 流程 | 内容 | 验收标准 |
|------|------|------|---------|
| 2.1 | MAP-F06 | 攻城战(基础) | 校验→消耗→城防衰减→占领 |
| 2.2 | MAP-F06-02 | 攻城策略选项 | 4策略(强攻/围困/夜袭/内应)四维差异化 |
| 2.3 | MAP-F06-07 | 内应信消费流程 | 三态卡片→扣取→执行→成功/失败分支 |
| 2.4 | MAP-F08 | 攻城奖励领取 | 首次/重复奖励→道具掉落 |
| 2.5 | MAP-F08-01 | 内应信掉落 | 攻城胜利20%概率掉落 |
| 2.6 | MAP-F06-01 | 攻城中断处理 | 退出→暂停→重连→继续/取消 |
| 2.7 | MAP-F06-05 | 攻城锁定机制 | 攻城开始→锁定→暂停24h |

**测试用例**:
- [ ] 攻城校验: 相邻+兵力×2.0+粮草×500+次数<3
- [ ] 强攻: 时间×0.5，损耗×1.5，奖励×0.9，城防-50%
- [ ] 围困: 时间×2.0，损耗×0.8，奖励×1.0，产出-20% 24h
- [ ] 夜袭: 需夜袭令，夜间+10%，奖励×1.2
- [ ] 内应: 需内应信，+20%成功率，奖励×1.5，城防保留
- [ ] 内应信三态: 可点击/暴露冷却/道具不足
- [ ] 内应失败→暴露冷却24h→三态卡片显示倒计时
- [ ] 冷却管理器: 10s扫描+乐观锁+destroy()
- [ ] 攻城中断→暂停→重连→继续
- [ ] 攻城锁定→其他玩家不可攻

### Sprint 4: 离线系统(P0)

**目标**: 离线事件→回归面板→快速处理→情报值→重试令牌 完整链路

| 任务 | 流程 | 内容 | 验收标准 |
|------|------|------|---------|
| 4.1 | MAP-F12 | 离线领土变化查看 | 回归面板(新占领/失去/收益变化) |
| 4.2 | MAP-F12-01 | 离线事件回归体验 | 过期累计衰减+快速处理+过期惩罚 |
| 4.3 | MAP-F12-02 | 情报值系统 | 获取/上限/兑换/约束逻辑 |
| 4.4 | MAP-F12-02 | 战斗重试令牌 | 兑换→使用→期望值分析 |
| 4.5 | MAP-F12-02 | 双计数器机制 | 在线/离线独立计数 |
| 4.6 | MAP-F01-02~04 | 产出上限管理 | 上限公式+预警+概览面板+一键收取 |

**测试用例**:
- [ ] 离线事件累积: 最多5个(loginEventQueueSize=5)
- [ ] 过期累计衰减: 24h→12h→6h→强制过期(72h上限)
- [ ] 快速处理: 推荐分支+80%奖励
- [ ] 山贼快速处理: 铜钱×100+情报值+1(保底)
- [ ] dailyLimit约束: 每日情报值获取≤5
- [ ] 情报值兑换: 10点→战斗重试令牌×1
- [ ] 重试令牌: 失败后手动使用→不额外征兵→失败正常扣损
- [ ] 产出上限: 80%预警→100%已满→一键收取
- [ ] 一键收取: 已超上限跳过+溢出丢弃+30s冷却

## 三、评测标准

### 3.1 功能完整性

| 等级 | 标准 |
|:----:|------|
| A | 12主流程+100子流程+15跨系统+10触发事件全部实现 |
| B | 核心流程(P0)全部实现，P1完成80%+ |
| C | 核心流程(P0)全部实现，P1完成60%+ |
| D | 核心流程(P0)有缺失 |

### 3.2 数据一致性

| 检查项 | 标准 |
|--------|------|
| 产出公式 | 6因子乘法公式正确叠加 |
| 胜率公式 | 地形修正+科技加成正确 |
| 内应信闭环 | 获取→存储→查询→扣取→效果→消耗 6阶段完整 |
| 情报值约束 | dailyLimit=5，每日获取不超限 |
| 声望衰减 | per-faction判定，3个豁免条件正确 |
| 冷却管理器 | 10s扫描，乐观锁去重，destroy()生命周期 |

### 3.3 性能指标

| 指标 | 标准 |
|------|------|
| 地图渲染 | 60fps(20×15网格) |
| 离线计算 | 8h离线结算<100ms |
| 冷却扫描 | 10s间隔，单次<1ms |
| 内存占用 | 天下Tab<30MB |
