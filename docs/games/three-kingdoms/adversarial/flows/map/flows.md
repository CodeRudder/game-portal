## 产出公式

> 📋 → [PRD: 领土产出](../../../ui-design/prd/MAP-world-prd.md#map-3)

```
领土产出 = 基础产出(领土等级表)
         × 地形加成(平原1.0/山地0.8/水域0.9/城池1.2/森林1.1/关隘0.7/沙漠0.7/雪地0.6/沼泽0.5)
         × 阵营加成(己方1.0/中立0.5/敌方0.0)
         × 科技加成(1 + 屯田/铸币/募兵 × 1.5%/级)
         × 声望加成(1 + 声望等级 × 2%)
         × 领土等级倍率(Lv1×1.0 ~ Lv15×2.0, 完整映射表见MAP-F11)
// 【P1-06修复】注意: "关隘"为TerritoryType(领土类型)而非TerrainType(地形类型)，产出系数0.7由CITY(1.2)+MOUNTAIN(0.8)组合推导，详见F02a-01注释。其余均为TerrainType枚举值。
```

**产出上限**:
```
领土产出上限 = 基础上限(10000) × 领土等级 × (1 + 仓储术等级 × 5%)
```
> 与建筑系统共用"仓储术"科技，统一上限管理(R4)

**产出公式参数参考值**【M3-13修复】:

基础产出表(按领土等级):

|------|-------------------|------|
| Lv1 | 100 | x1.0 |
| Lv2 | 105 | x1.05 |
| Lv3 | 110 | x1.10 |
| Lv4 | 115 | x1.15 |
| Lv5 | 120 | x1.20 |
| Lv6 | 130 | x1.30 |
| Lv7 | 140 | x1.40 |
| Lv8 | 150 | x1.45 |

中间等级插值方式: 线性插值
  倍率(n) = 倍率(floor(n)) + (倍率(ceil(n)) - 倍率(floor(n))) x (n - floor(n))

声望等级上限: 50级(加成上限+100%)

产出上限: 基础产出 x 10 (防止异常堆叠)

## MAP-F01 进入天下Tab

> 📋 → [PRD: 天下Tab入口](../../../ui-design/prd/MAP-world-prd.md#map-1)
> 🎨 → [UI: 天下Tab](../../../ui-design/ui-layout/MAP-world.md#map-1)

**流程概述**: 点击底部"天下"Tab，切换到世界地图场景，显示领土分布。【R1】

### 流程步骤

1. [玩家操作] 点击底部导航栏"天下"Tab
2. [系统响应] 场景路由器切换到世界地图场景
2a. [加载状态] 地图数据加载中，显示加载反馈【M-13修复】:
   - **加载开始**: 显示全屏加载状态（地图轮廓骨架屏 + "天下加载中..."文字）
   - **加载进度**: 骨架屏渐进填充（地形数据分批渲染，每批10%进度条）
   - **加载成功**: 骨架屏淡出(300ms) -> 完整地图显示，进入步骤3
   - **加载失败**: 显示错误面板（"加载失败，请检查网络" + "重试"按钮 + "返回"按钮），
     "重试"重新从步骤2a开始，"返回"退回前一Tab
   - **超时处理**: 15秒无响应自动弹出提示"网络较慢，是否继续等待？" +
     "继续等待"按钮(保持加载) + "取消"按钮(退回前一Tab)
3. [系统加载] 初始化地图渲染 -> 像素色块渲染体系(8x8像素色块, 非网格有机布局, 详见MAP-F02a)
   > **[v2.0权威]** v1.0的"20x15瓦片网格, 六边形"渲染已被v2.0像素渲染替代，渲染层统一由MAP-F02a提供
4. [系统计算] 根据玩家阵营确定初始视角(己方领土区域)

**己方领土高亮样式**【M-51修复】:

> 精确定义己方领土与非己方领土的视觉差异，确保阵营归属一目了然。

- **己方领土高亮**: 领土色块边缘添加2px宽阵营色发光边框(`box-shadow: 0 0 4px ${factionColor}`)
- **脉冲动画**: 边框亮度以2秒周期缓慢脉冲(100%→70%→100%)，CSS animation: `pulse-glow 2s ease-in-out infinite`
- **非己方领土**: 无边框，色块饱和度降低30%(灰度混合，`filter: saturate(0.7)`)

5. [UI反馈] 全屏显示世界地图，己方领土高亮，产出气泡显示，地图图例组件就绪

**"定位到己方领土"快捷按钮**【M-43修复】:
- **位置**: 地图右上角(不遮挡产出气泡区域)
- **图标**: 十字准星图标(24x24px)，白色半透明背景圆形(40x40px)
- **功能**: 点击后地图视口平滑移动+缩放到己方核心领土区域(动画500ms，缓动函数easeInOutCubic)
- **快捷键**: PC端按 Home 键触发，等同于点击按钮效果

**地图图例(Legend)组件**【M-14修复】:

> 首次用户引导组件，帮助理解地图各视觉元素的含义。

- **位置**: 地图右下角，可展开/收起
- **收起状态**: 显示一个"?"图标(24x24px)，半透明背景，不遮挡地图交互
- **展开状态**: 半透明面板(宽200px，高自适应内容)，包含以下图例内容:
  - 地形颜色对照: 平原(绿色) / 山地(棕色) / 水域(蓝色) / 森林(深绿色)
  - 阵营颜色说明: 魏(蓝色) / 蜀(红色) / 吴(绿色) / 中立(灰色)
  - 图标说明: 城市(城池图标) / 资源点(资源图标) / 关隘(关隘图标)
  - 产出气泡状态: 绿色(<80%正常) / 黄色(80%~99%预警) / 红色(100%已满)
- **首次用户**: 首次进入天下Tab时图例自动展开，3秒后自动收起(用户可提前手动收起)
- **交互**: 点击收起状态的"?"图标展开图例；点击图例外区域或图例内关闭按钮收起

**前置条件**: 玩家已选择阵营(魏/蜀/吴)
**后置流程**: -> MAP-F02(地图浏览)、MAP-F03(领土选择)

**Tab切换状态持久化策略**【M-47修复】:
- **保持状态**: 视口位置(x, y)、缩放级别(zoom)、选中领土ID(selectedTerritoryId)
- **存储方式**: 客户端内存(Tab切换不销毁)；Tab销毁时保存到localStorage(`map_viewport_state`)
- **恢复策略**: 再次进入时恢复到离开时的状态(无加载动画，直接显示)
- **过期策略**: 超过24小时的视口状态重置为默认(己方核心领土中心 + 缩放1.0x)
- **恢复判定**: 检查localStorage中时间戳，`Date.now() - savedTimestamp > 86400000` 时清除并使用默认视口

## MAP-F02 地图浏览与缩放

**流程概述**: 拖拽地图浏览，缩放查看细节，定位目标领土。【R1】
> **[v2.0权威]** 渲染层由MAP-F02a(像素地图渲染)统一提供，MAP-F02仅定义浏览/缩放交互逻辑

### 流程步骤

1. [玩家操作] 单指拖拽/鼠标拖拽移动地图
2. [系统响应] 视口偏移，约束在地图边界内

**手势冲突处理**【M-48修复】:

> 手势优先级规则，避免地图操作与系统手势冲突。

- **手势优先级**:
  1. 双指操作优先 → 地图缩放(无论区域)
  2. 单指操作 - 地图区域 → 地图拖拽
  3. 单指操作 - 屏幕边缘20px区域 → 系统手势(iOS返回等)，地图不响应
- **判定逻辑**: 触摸起点落在边缘20px区域内时，事件不传递给地图手势处理器

**边界反馈**【M-48修复】:

> 到达地图边界时提供视觉反馈，明确告知用户已到达边界。

- **移动端**: 到达地图边界时显示弹性回弹动画(回弹距离30px，持续300ms，easeOut缓动)
- **PC端**: 鼠标拖到地图边界时，光标变为`not-allowed`样式，提示用户已到边界

3. [玩家操作] 双指缩放/滚轮缩放
4. [系统响应] 缩放比例0.5x~3.0x【v2.0权威，以MAP-F02a为准】，色块自适应渲染
5. [UI反馈] 缩放<0.6x时隐藏产出气泡避免拥挤【M-17修复】:
   - **气泡消失动画**: 缩放到0.6x以下时，气泡缩小至0%+淡出(200ms，`transform: scale(0)` + `opacity: 0`)
   - **气泡出现动画**: 缩放到0.6x以上时，气泡从0%放大至100%+淡入(200ms，`transform: scale(1)` + `opacity: 1`)
   - **隐藏时替代指示**: 城市图标上方显示小圆点(4x4px, 阵营颜色)指示有产出信息

**后置流程**: → MAP-F03(领土选择)

**小地图交互**【M-49修复】:

> 小地图提供全局视野与快速导航能力，确保玩家不迷失方向。

- **位置**: 地图左下角，默认尺寸150x100px(非缩放状态)
- **显示内容**: 缩略版完整地图(仅阵营色块，无细节)，当前视口矩形框(白色边框，2px)
- **点击跳转**: 点击小地图任意位置，主视口跳转到点击位置(无动画，立即跳转)
- **拖拽同步**: 拖拽小地图中的视口矩形框，主地图同步平移

## MAP-F02a 像素地图渲染【v3.1】

**流程概述**: 使用8×8像素色块渲染三国世界地图，非网格有机布局，城市通过道路连接。

### 流程步骤

1. [系统加载] 读取地图配置(地形数据+城市坐标+道路网络)
2. [地形渲染] 逐色块绘制地形(平原/山地/水域/森林)

**缩放插值策略**【M-50修复】:

> 像素地图必须保持锐利边缘，禁用平滑插值以保持像素风格一致性。

- **适用范围**: 地形色块、城市图标、道路渲染、精灵动画 — 所有像素风格元素统一使用最近邻插值

## MAP-F02b 行军动画【v3.1】

**流程概述**: 军队沿道路行军，显示精灵动画、路线预览、预计到达时间。

**"行军时间"术语定义**:

| 术语 | 英文 | 含义 | 量级 | 用途 |
|------|------|------|------|------|
| **行军战略时间** | March Strategic Time | 游戏内行军消耗的游戏时间 | 分钟~小时级 | 用于游戏策略决策（资源调度、防守窗口） |
| **行军动画时长** | March Animation Duration | 精灵动画在屏幕上的实际播放时间 | 10s~60s(硬约束) | 用于视觉表现和玩家体验 |

**两者映射关系**:

```
行军战略时间 = 路径距离(px) × 游戏时间系数 / BASE_SPEED
  // 游戏时间系数: 1px = 1游戏秒(可调)，即跨1个色块(8px) = 8游戏秒
  // 示例: 路径240px → 战略时间 = 240×1/30 = 8游戏分钟

行军动画时长 = clamp(路径距离(px) / actualSpeed, 10s, 60s)
  // actualSpeed经过时长约束调整后的实际播放速度

时间压缩比 = 行军战略时间 / 行军动画时长
  // 通常为 10x~100x（动画加速以保持玩家体验）
  // 显示: "行军中(预计X分钟)" 使用战略时间
```

> **原则**: 路线预览面板显示"行军战略时间"，精灵动画播放使用"行军动画时长"。两者独立计算，动画结束后战略效果即时生效。

**行军速度精确定义**:

| 参数 | 值 | 坐标系 | 说明 |
|------|---|--------|------|
| **基准行军速度** | **30 px/s** | 屏幕像素坐标（1个8x8色块 = 8px） | 代码常量 `BASE_SPEED = 30` |
| 精灵帧速度 | 30/8 = 3.75 px/精灵帧 | 屏幕像素 | 每个精灵帧(125ms)移动3.75px |
| 色块速度 | 30/8 = 3.75 色块/秒 | 8x8色块坐标 | 每秒跨越3.75个色块 |
| 渲染帧速度 | 30/60 = 0.5 px/渲染帧 | 屏幕像素 | 每个渲染帧(16.67ms)移动0.5px |

**术语澄清**:
- "帧"统一指**精灵帧**(8fps)，不指渲染帧(60fps)
- 原文"2px/帧"为错误值，正确值为 **3.75 px/精灵帧**(即30px/s ÷ 8fps)
- 地形修正作用于基准速度: `actualSpeed = BASE_SPEED × terrainModifier`

> **注**: 此速度定义与MAP-F06-11保持一致。MAP-F06-11为行军速度的权威定义来源。
7. [地形影响] 道路加速(×1.5)、山地减速(×0.5)
7a. [时长约束] 行军动画时长约束算法（引用MAP-F06-11的10s~60s硬约束）:
   - **目标**: 行军精灵移动总时长落在[10s, 60s]区间内
   - **计算公式**:
     ```
     // 1. 计算路径总距离(考虑地形)
     pathDistance = sum(terrainMoveCost(segment) × segmentLength) for each segment in path

     // 2. 计算自然行军时长
     naturalDuration = pathDistance / BASE_SPEED  // BASE_SPEED = 30px/s

     // 3. 时长约束调整速度
     MIN_DURATION = 10s
     MAX_DURATION = 60s
     targetDuration = clamp(naturalDuration, MIN_DURATION, MAX_DURATION)
     actualSpeed = clamp(pathDistance / targetDuration, MIN_SPEED, MAX_SPEED)

     // 速度边界:
     MIN_SPEED = 10px/s   // 不低于此值，确保视觉可见移动
     MAX_SPEED = 120px/s  // 不超过此值，避免精灵闪烁过快
     ```
   - **特殊处理**: actualSpeed达到MAX_SPEED仍超过60s时，行军时间不受上限约束（超远距离行军）
   - **超长行军UI规范**【M3-15修复】: 行军时间超过60s时:

#### MAP-F02b-01 路线计算

- [输入] 起点城市ID + 终点城市ID
- [算法] A*寻路(沿道路网络)
- [输出] 路径节点列表 + 总距离 + 预计时间

**PathNode完整接口定义**【M3-05修复】:
```
interface PathNode {
  x: number          // 色块坐标X (0~319)
  y: number          // 色块坐标Y (0~239)
  px: number         // 像素坐标X (x*8+4，色块中心)
  py: number         // 像素坐标Y (y*8+4，色块中心)
  terrain: TerrainType  // 该节点地形类型
  moveCost: number      // 该节点移动成本
}
// 路径总距离 = Σ(相邻节点moveCost)
// 预计时间(秒) = 路径总距离 / 基础速度(30px/s) / 速度修正系数
```

**A*寻路权重定义**:

```
A*核心公式:
  f(n) = g(n) + h(n)

g值计算（移动成本）:
  g(neighbor) = parent.g + terrainMoveCost(neighbor)

h值（启发函数）:
  h(n) = manhattanDistance(n, goal) × minMoveCost
  // minMoveCost = 1.0 (道路，最小成本)

地形移动成本表 (terrainMoveCost，引用MAP-F02a-01地形类型):

| 地形类型       | moveCost | 对应速度修正 | 说明          |
|---------------|----------|------------|--------------|
| ROAD (道路)    | 1.0      | ×1.5       | 最优路径，加速 |
| PLAIN (平原)   | 1.5      | ×1.0       | 标准地形      |
| DESERT (沙漠)  | 1.5      | ×1.0       | 标准地形      |
| FOREST (森林)  | 1.875    | ×0.8       | 轻微减速      |
| WATER (水域)   | 2.14     | ×0.7       | 显著减速      |
| MOUNTAIN (山地)| 3.0      | ×0.5       | 最大减速      |
| SNOWLAND (雪地)| 2.5      | ×0.6       | 较大减速      |
| SWAMP (沼泽)   | 3.75     | ×0.4       | 极大减速      |
| CITY (城市)    | 1.0      | ×1.0       | 可通行，标准   |

moveCost推导公式:
  moveCost = 基础成本(1.5) / 速度修正系数
  // 道路特例: moveCost = 1.0 (道路网络优先)

不可通行判定:
  - 无 TerrainType 值标记为不可通行时，寻路自动绕行（高moveCost = 自然回避）
  - 未来扩展: 可添加 PASSABLE=false 标记的硬阻断地形
```

> **注**: 地形速度修正系数与MAP-F06-11地形表一致，moveCost为速度修正的倒数关系（成本越高，移动越慢）。

## MAP-F04 领土征服

**流程概述**: 选择目标→部署兵力→确认出征→战斗结算→占领领土。【R1】

### 流程步骤

1. [玩家操作] 点击非己方领土，点击"征服"按钮
2. [系统校验] 检查征服条件:
   - a. 目标领土与己方领土相邻
   - b. 可用兵力 ≥ 目标驻防兵力 × 1.5(非城池) / × 2.0(城池)
     > 注: 城池类型领土统一走MAP-F06攻城流程(F03按钮分支表已定义), 此城池分支为防御性校验(正常不触发)
3. [校验失败] 显示失败原因(不相邻/兵力不足)
4. [校验通过] 显示兵力部署滑块
5. [玩家操作] 拖动滑块选择出征兵力
6. [系统计算] 预估胜率(见子流程 MAP-F04-02)
7. [UI反馈] 胜率颜色编码(翠绿>80%/金色60-80%/琥珀橙40-60%/赤红<40%)
8. [玩家操作] 点击"确认出征"
9. [系统执行] 扣除出征兵力，进入战斗结算

**征服战斗结算规则**【M3-12修复】:
```
征服战斗结算规则:
- 结算方式: **概率判定** — random(0, 100) < 胜率 则为胜利
- 兵力损失:
  * 胜利: 损失 = 出征兵力 × (1 - 胜率) × 0.5，最低损失10兵力
  * 失败: 损失 = 出征兵力 × (1 - 胜率)，最低损失50兵力
- 领土归属: 胜利立即获得领土(产出从0%开始充能)  // 【P1-05修复】征服(非城池)产出从0开始，与F06攻城(城池)产出从50%开始为不同规则
- 失败: 出征兵力回到出发城市(扣除损失后)
```
10. [战斗结算] 概率判定: roll = random(0,100)，如果 roll < 胜率(由MAP-F04-02计算)则判定为胜利，否则为失败
11. [胜利结果] 领土归己方，自动驻防50%兵力上限，产出从0%充能；**征服伤亡**: 胜利方损失出征兵力×10%  // 【P1-05修复】征服(普通领土)产出从0%开始，与F06攻城(城池)产出从50%开始为两种不同占领方式的差异
12. [失败结果] 损失出征兵力×30%，领土不变
13. [UI反馈] 显示征服结果(胜利特效/失败提示)

**前置条件**: → MAP-F03(领土选择)
**后置流程**: → MAP-F05(驻防管理)、RES-F01(资源产出)

#### MAP-F04-02 胜率公式【R2/R6修正】

```
胜率 = min(95%, max(5%, (我方战力/敌方战力)×50% + 地形修正))

地形修正:
  平原: 0%
  山地: -10%(防守方+10%)
  水域: 0%
  城池: -20%(城防加成)
  森林: +5%(伏兵加成)
  关隘: -30%(一夫当关)
  沙漠: +5%(攻方优势，守方补给困难)
  雪地: -5%(守方优势，严寒消耗攻方)
  沼泽: -8%(攻方劣势，地形陷阱+守方伏击)  // 【P0-03/P0-04修复】原值+8%与"攻方劣势"描述矛盾，且与F02a-01地形加成表(沼泽攻击-8%)不一致，已修正为-8%
```

> **【P0-03修复】胜率地形修正与地形加成表(F02a-01)关系说明**: 胜率地形修正为胜率公式的专用参数，其值基于F02a-01地形加成表但经过独立平衡调整(如山地胜率-10%vs地形表攻击-5%，因胜率公式中已有50%基准乘数)。修正后沼泽已与地形表一致(-8%)。其余地形的差异为有意的平衡设计。

**全局兵力池定义**【M3-11修复】:
```
- 作用域: 每个玩家拥有一个全局兵力池(跨所有己方领土共享)

```

5. [系统执行] 从总兵力池分配/调回兵力
6. [UI反馈] 防御值实时更新

**后置流程**: → MAP-F06(攻城防守)

**总兵力池数据模型**:
```
TotalArmyPool {
  // 未分配兵力 = 总兵力池 - sum(所有己方领土驻防兵力)
  unallocated: number

  // 总兵力上限 = sum(所有己方领土等级 × 100)
  // 即: 每块己方领土按其等级贡献驻防上限容量作为兵力池容量上限
  cap: number

  // 每小时自然补充 = sum(所有己方领土驻防产出速率)
  // 驻防产出速率 = 领土产出公式计算值的 10%
  replenishRate: number  // 兵力/小时
}

// 不变量约束:
//   unallocated >= 0
//   unallocated + sum(己方领土.garrison) <= cap
//   增加驻防时: unallocated -= 调入量, 检查 unallocated >= 0
//   减少驻防时: unallocated += 调出量, 检查 unallocated + sum(garrison) <= cap
```

**攻占任务数据结构**:
```
/** 路径节点 — 描述行军路线上的一个色块坐标 */
interface PathNode {
  x: number;           // 色块列坐标 (0~319)
  y: number;           // 色块行坐标 (0~239)
  px: number;          // 像素坐标X (x*8+4，色块中心)【M3-05修复】
  py: number;          // 像素坐标Y (y*8+4，色块中心)【M3-05修复】
  terrain: TerrainType; // 该点地形类型
  moveCost: number;    // 该节点移动成本(查MAP-F02b-01地形表)【M3-05修复】
}

SiegeTask {
  id: string              // 唯一ID
  expeditionForce: ExpeditionForce  // 编队
  strategy: SiegeStrategyType       // 策略
  targetId: string                  // 目标城池
  sourceId: string                  // 出发城市
  status: 'preparing' | 'marching' | 'sieging' | 'settling' | 'returning' | 'completed'
  createdAt: number                 // 创建时间戳
  estimatedArrival: number          // 预计到达时间
  marchPath: PathNode[]             // 行军路径(PathNode色块坐标序列)
  totalDistance: number             // 总路径距离(px) = sum(相邻PathNode间欧氏距离 × 8px/色块)
  estimatedTime: number             // 预计行军时长(s) = sum(各段距离 / 该段地形实际速度)
}
// totalDistance计算: 遍历marchPath相邻节点, dist += sqrt((dx*8)^2 + (dy*8)^2)
// estimatedTime计算: 各段距离除以该段terrain对应实际速度(px/s, 查MAP-F06-11地形表)
```

**山贼战斗公式**(R6修正):
```
山贼战力 = max(100, 玩家等级 × 100) × 难度系数

难度系数:
  玩家等级 1~20:  ×0.3（弱匪）
  玩家等级 21~40: ×0.5（悍匪）
  玩家等级 41~60: ×0.7（精锐山贼）
  玩家等级 61+:   ×0.9（山贼王）

战斗胜率 = min(95%, max(5%,
  (玩家总兵力 / 山贼战力) × 50%
  + 地形修正(森林+10% / 山地-10% / 其他0%)
  + 科技加成(攻城术等级×2%)
))

胜利损耗 = 出征兵力 × min(35%, max(5%, 20% × (山贼战力 / 玩家总兵力)))
  // R7: 增加50%上限截断; R8: 修正为35%(k=1.75推导)

失败损耗 = 出征兵力 × 20%
```

> R8设计公理声明: k=1.75为设计假设，35%=20%×1.75，胜/败损耗比上限1.75x

**遗迹探索三档判定**(R6修正/R7数据修正):
```
三档随机判定:
  roll = random(0, 100)  // [0, 100) 左闭右开浮点数区间
  // 即: 0.0 <= roll < 100.0, 不包含100.0本身

  失败阈值 = max(5%, 30% - 科技加成 - 地形修正)
  部分成功阈值 = max(失败阈值+5%, 50% - 科技加成×0.5 - 地形修正×0.5)

  if roll < 失败阈值 → 失败: 无奖励, 损失10%兵力
  elif roll < 部分成功阈值 → 部分成功: 50%奖励, 内应信概率12.5%
  else → 成功: 完整奖励, 内应信概率25%

科技加成: 攻城术等级×2%
地形修正: 山地+5% / 森林+3% / 其他0%
```

## A/B测试方法论【R10/R11】

> 详见 R10 `MAP-F09-02-R10` 和 R11 `MAP-F09-02-R11`

### 测试方案设计

**测试名称**: 胜利损耗k值优化测试

**测试组设计**:

| 组别 | k值 | 上限 | 胜/败比(极端) | 目的 |
|------|-----|------|-------------|------|
| 对照组A | 1.0 | 20% | 1.0x | 测试公理1(是否应有惩罚) |
| 实验组B | 1.5 | 30% | 1.5x | 保守方案 |
| 实验组C | 1.75 | 35% | 1.75x | 当前方案(默认) |
| 实验组D | 2.0 | 40% | 2.0x | 激进方案 |

**分流策略**: 玩家ID哈希取模，4组均分(各25%)，首次进入天下Tab时分配

### 样本量计算

```
主要指标: 有意义战斗参与率(MBPR)(R11修正)
基线: 30%
最小可检测效应(MDE): 5个百分点(30%→35%)
显著性水平(α): 0.05(双尾) → Bonferroni校正后 0.0083
统计功效(1-β): 0.80
每组样本量: ~1,000名活跃玩家
总样本量: 4,000名
```

**DAU敏感性分析**(R11):

| DAU | 14天样本量/组 | 满足1000/组 | 建议时长 |
|-----|-------------|:-----------:|---------|
| 1,000 | 3,500 | ✅ | 14天 |
| 3,000 | 10,500 | ✅ | 14天 |
| 10,000 | 35,000 | ✅ | 14天 |

> DAU=10,000为初始假设值，需根据实际运营数据调整

### 主要指标(R11修正)

**有意义战斗参与率(MBPR)**:
- 定义: 每日参与"胜率>30%"战斗的独立玩家数 / 当日DAU
- 方向: k=1.75组MBPR > k=1.0组MBPR(惩罚抑制低胜率出征，提升决策质量)
- 判定: p<0.05 且 Cohen's h ≥ 0.2

**次要指标**:
- 低兵力(<3000)出征频率 — 预期k=1.75组更低
- 重试令牌使用率 — 预期k=1.75组更高
- 7日留存率 — 预期k=1.75组 ≥ k=1.0组

### 决策规则

| 场景 | 判定 | 行动 |
|------|------|------|
| k=1.75组MBPR显著高于k=1.0组 | 公理1成立 | 保留惩罚机制 |
| 两组无显著差异 | 公理1存疑 | 进一步测试 |
| k=1.0组MBPR显著高于k=1.75组 | 公理1被推翻 | 移除惩罚(k=1.0) |

### 公理验证失败回退

- 公理1被推翻 → k=1.0(无额外惩罚)，监控战斗参与率
- 公理2被推翻 → 调整胜/败比边界(如1.8x)

## 事件队列替换策略【R11】

> 详见 R11 `MAP-F12-02-R11b`

### 最低价值优先替换

**事件价值定义**:

| 事件类型 | 价值 | 说明 |
|---------|:----:|------|
| 普通山贼 | 1 | 低价值 |
| 遗迹探索 | 2 | 中价值 |
| 阵营冲突 | 2 | 中价值 |
| 精英山贼 | 3 | 高价值 |

**替换规则**:
1. 队列已满且新事件到达时，找出队列中价值最低的事件
2. 新事件价值 > 最低价值 → 替换最低价值事件
3. 新事件价值 ≤ 最低价值 → 替换最旧的最低价值事件(FIFO)
4. 多个事件同为最低价值 → 替换其中最旧的

**替换通知**(默认行为):
- 每次替换时记录被替换事件
- 登录时显示: "您有X个事件因队列已满被替换(含Y个高价值事件)"
- 高价值事件被替换时额外提示
