> **版本**: v1.0 | **日期**: 2026-05-03

---

## 1. 迭代开发机制

### 1.1 核心流程

```
评测(对抗性) → 发现问题 → 改进实现 → 测试验证 → 架构审查 → 评测(下一轮)
```

### 1.2 每轮迭代包含

1. **计划**: 明确本轮目标和验收标准
2. **评测**: 以玩家视角进行对抗性检查，使用E2E真实测试
3. **改进**: 修复发现的问题
4. **测试**: 运行全部相关测试，确保无回归
5. **架构审查**: 检查模块依赖、数据流、坐标一致性
6. **复盘**: 回顾最近几轮改进，总结经验

### 1.3 对抗性评测标准

- 以玩家视角操作，不看代码
- 检查视觉效果是否符合预期
- 检查交互响应是否及时
- 检查数据是否一致(显示值与实际值)
- 检查边界情况(空数据、大量数据、极端值)

### 3.2 对抗性检查清单

- [ ] 地图打开时是否自动居中放大
- [ ] 城市/资源点是否清晰可见
- [ ] 点击响应是否及时(<100ms)
- [ ] 攻城流程是否完整(无断裂)
- [ ] 行军流程是否完整(精灵可见)
- [ ] 离线奖励是否正常弹出
- [ ] 产出数据是否实时更新
- [ ] 缩放/平移是否流畅
- [ ] 数据一致性(显示值=实际值)
