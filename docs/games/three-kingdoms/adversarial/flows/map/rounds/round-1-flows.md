## 一、主流程

### MAP-F01 进入天下Tab {#map-f01}

**玩家视角**: 点击底部"天下"Tab → 加载世界地图 → 看到领土分布

**流程步骤**:
1. [玩家操作] 点击底部导航栏"天下"Tab
2. [系统响应] 场景路由器切换到世界地图场景
3. [系统加载] 初始化地图渲染(20×15瓦片网格, 六边形)
4. [系统计算] 根据玩家阵营确定初始视角(己方领土区域)
5. [UI反馈] 全屏显示世界地图，己方领土高亮，产出气泡显示

**数据变更**:
| 数据 | 变更前 | 变更后 | 来源 |
|------|--------|--------|------|
| currentTab | 上一个Tab | '天下' | 玩家点击 |
| mapViewport | — | 己方领土中心 | 系统计算 |

**前置条件**: 玩家已选择阵营(魏/蜀/吴)
**后置流程**: → MAP-F02(地图浏览)、MAP-F03(领土选择)

---

### MAP-F02 地图浏览与缩放 {#map-f02}

**玩家视角**: 拖拽地图浏览 → 缩放查看细节 → 定位目标领土

**流程步骤**:
1. [玩家操作] 单指拖拽/鼠标拖拽移动地图
2. [系统响应] 视口偏移，约束在地图边界内
3. [玩家操作] 双指缩放/滚轮缩放
4. [系统响应] 缩放比例50%~200%，瓦片自适应渲染
5. [UI反馈] 缩放<60%时隐藏产出气泡避免拥挤

**子流程**:
- MAP-F02-01: 缩放约束检查(min/max)
- MAP-F02-02: 瓦片坐标↔像素坐标转换
- MAP-F02-03: 视口范围裁剪(只渲染可见瓦片)

**后置流程**: → MAP-F03(领土选择)

### MAP-F03 领土选择与详情查看 {#map-f03}

**玩家视角**: 点击领土 → 查看详情面板 → 了解产出/驻防/地形

**流程步骤**:
1. [玩家操作] 点击地图上的领土瓦片
2. [系统响应] 高亮选中领土，弹出详情面板(PC右侧/手机Bottom Sheet)
3. [UI反馈] 显示信息:
   - 领土名称 + 等级标签
   - 阵营归属(己方绿/敌方红/中立灰)
   - 地形类型 + 地形加成
   - 当前产出(粮草/铜钱/兵力)
   - 驻防兵力 + 防御值
   - 操作按钮(征服/驻防/放弃)

**数据变更**:
| 数据 | 变更前 | 变更后 | 来源 |
|------|--------|--------|------|
| selectedTerritoryId | null | 领土ID | 玩家点击 |

**前置条件**: → MAP-F02(地图浏览)
**后置流程**: → MAP-F04(征服)、MAP-F05(驻防管理)、MAP-F06(攻城战)

### MAP-F04 领土征服流程 {#map-f04}

**玩家视角**: 选择目标 → 部署兵力 → 确认出征 → 战斗结算 → 占领领土

**流程步骤**:
1. [玩家操作] 点击非己方领土，点击"征服"按钮
2. [系统校验] 检查征服条件:
   - a. 目标领土与己方领土相邻
   - b. 可用兵力 ≥ 目标驻防兵力 × 1.5(非城池) / × 2.0(城池)
3. [校验失败] 显示失败原因(不相邻/兵力不足)
4. [校验通过] 显示兵力部署滑块
5. [玩家操作] 拖动滑块选择出征兵力
6. [系统计算] 预估胜率 = min(95%, max(5%, (我方战力/敌方战力)×50% + 地形修正))
7. [UI反馈] 胜率颜色编码(翠绿>80%/金色60-80%/琥珀橙40-60%/赤红<40%)
8. [玩家操作] 点击"确认出征"
9. [系统执行] 扣除出征兵力，进入战斗结算
10. [战斗结算] 胜利/失败判定
11. [胜利结果] 领土归己方，自动驻防50%兵力上限，产出从0充能
12. [失败结果] 损失30%出征兵力，领土不变
13. [UI反馈] 显示征服结果(胜利特效/失败提示)

**数据变更**:
| 数据 | 变更前 | 变更后 | 来源 |
|------|--------|--------|------|
| resources.troops | X | X - 出征兵力 | 征服扣费 |
| territory.owner | 敌方/中立 | 己方 | 战斗胜利 |
| territory.garrison | 0 | 兵力上限×50% | 自动驻防 |

**前置条件**: → MAP-F03(领土选择)
**后置流程**: → MAP-F05(驻防管理)、RES-F01(资源产出)

### MAP-F05 驻防管理 {#map-f05}

**玩家视角**: 查看驻防 → 调整兵力分配 → 优化防御

**流程步骤**:
1. [玩家操作] 点击己方领土，进入驻防管理
2. [UI反馈] 显示当前驻防兵力/上限、防御加成

4. [系统校验] 驻防 ≤ 驻防上限(领土等级×100)
5. [系统执行] 从总兵力池分配/调回兵力
6. [UI反馈] 防御值实时更新

**数据变更**:
| 数据 | 变更前 | 变更后 | 来源 |
|------|--------|--------|------|
| territory.garrison | X | 新值 | 玩家调整 |
| resources.troops(可用) | X | X ± 变化量 | 兵力池调整 |
| territory.defense | 旧防御值 | garrison×0.1% | 系统计算 |

**前置条件**: → MAP-F04(征服后)或MAP-F03(选择己方领土)

### MAP-F06 攻城战流程 {#map-f06}

**玩家视角**: 选择城池 → 检查条件 → 确认攻城 → 消耗城防 → 占领城池

**流程步骤**:
1. [玩家操作] 点击城池类领土，点击"攻城"按钮
2. [系统校验] 检查攻城条件:
   - a. 领土与己方相邻
   - b. 出征兵力 ≥ 城池驻防兵力 × 2.0
   - c. 消耗兵力×100 + 粮草×500
   - d. 每日攻城次数 < 3
3. [校验失败] 显示失败原因
4. [校验通过] 显示攻城确认弹窗:
   - 城防值(当前/上限)
   - 预计攻城时间(基础30分钟 + 城防值/100分钟)
   - 消耗明细(兵力/粮草)
   - 每日剩余次数
5. [玩家操作] 点击"确认攻城"
6. [系统执行] 扣除消耗，开始攻城倒计时
7. [系统计算] 城防值每秒递减(基于攻方战力)
8. [城防归零] 占领成功，获得城池产出(初始50%，24h后100%)
9. [UI反馈] 攻城结果弹窗(胜利/失败/无法攻城三态)

**数据变更**:
| 数据 | 变更前 | 变更后 | 来源 |
|------|--------|--------|------|
| resources.troops | X | X - 兵力×100 | 攻城消耗 |
| resources.grain | X | X - 粮草×500 | 攻城消耗 |
| city.defense | 原值 | 0 | 攻城成功 |
| city.owner | 敌方 | 己方 | 攻城成功 |
| siege.dailyCount | N | N+1 | 次数记录 |

**前置条件**: → MAP-F03(领土选择)
**后置流程**: → MAP-F05(驻防管理)、MAP-F08(攻城奖励)

### MAP-F07 筛选与热力图 {#map-f07}

**玩家视角**: 使用筛选器过滤领土 → 切换热力图查看收益分布

**流程步骤**:
1. [玩家操作] 点击筛选按钮(顶部工具栏/手机底部)
2. [UI反馈] 展示筛选面板(5个维度: 阵营/地形/收益/等级/热力图)
3. [玩家操作] 选择筛选条件(如: 蜀国+平原+粮草)
4. [系统计算] 过滤匹配领土
5. [UI反馈] 匹配领土正常显示+古铜金高亮边框，不匹配领土亮度40%+灰度
6. [玩家操作] 切换热力图模式
7. [系统计算] 按收益高低计算颜色等级(5档)
8. [UI反馈] 领土按收益深浅着色

**快捷筛选**:
- MAP-F07-01: "我的领土" → 过滤己方阵营
- MAP-F07-02: "高产领地" → 按总收益排序前5
- MAP-F07-03: "可征服" → 显示与己方相邻的非己方领土
- MAP-F07-04: "热力图" → 切换收益热力图模式

**后置流程**: → MAP-F03(选择筛选后的领土)

### MAP-F09 地图事件处理 {#map-f09}

**玩家视角**: 发现事件图标 → 点击查看 → 做出选择 → 获得结果

**流程步骤**:
1. [系统触发-TE05] 每小时10%概率触发，最多3个未处理事件
2. [UI反馈] 地图上出现脉冲闪烁事件图标 + 顶部事件条提示
3. [玩家操作] 点击事件图标
4. [UI反馈] 弹出事件面板，显示事件描述+2~3个选择分支
5. [玩家操作] 选择分支(强攻/谈判/忽略)
6. [系统执行] 根据选择计算结果
7. [UI反馈] 显示结果(奖励/惩罚/无效果)

**事件类型**:
| 类型 | 概率 | 选择 | 结果 |
|------|------|------|------|
| 商队遇险 | 30% | 援助/忽略 | 援助→金币; 忽略→无 |
| 流民 | 25% | 接纳/遣散 | 接纳→兵力; 遣散→无 |
| 宝箱 | 20% | 开启 | 直接资源 |
| 山贼 | 25% | 战斗/绕行 | 战斗→装备; 绕行→无 |

**后置流程**: → MAP-F10(事件奖励)

## 二、跨系统流程(XI)

### XI-MAP-01 领土产出→资源系统

**涉及系统**: MAP → RES
**衔接点**: MAP-F04(征服) → RES-F01(资源累积)
**流程**: 领土占领后，产出按公式(基础×地形×阵营×科技×声望×地标)计算，每秒流入资源池

### XI-MAP-02 攻城→战斗系统

**涉及系统**: MAP → CBT
**衔接点**: MAP-F06(攻城) → CBT-F01(战斗结算)
**流程**: 攻城触发战斗系统，计算战力对比、城防减伤、胜负判定

### XI-MAP-03 科技加成注入

**涉及系统**: TEC → MAP
**衔接点**: TEC-F02(科技效果) → MAP-F01(产出计算)
**流程**: 科技"屯田"等效果注入领土产出公式

### XI-MAP-04 声望加成注入

**涉及系统**: PRS → MAP
**衔接点**: PRS-F01(声望等级) → MAP-F01(产出计算)
**流程**: 声望等级×2%加成注入领土产出公式

### XI-MAP-05 武将派驻→领土加成

**涉及系统**: HER → MAP
**衔接点**: HER-F02(武将派驻) → MAP-F01(产出计算)
**流程**: 武将政治属性加成领土产出

### XI-MAP-06 领土→任务/成就系统

**涉及系统**: MAP → QST/ACH
**衔接点**: MAP-F04(征服) → QST-F01(任务进度)
**流程**: 占领领土触发任务进度更新、成就解锁

### XI-MAP-07 领土→活动系统

**涉及系统**: MAP → ACT
**衔接点**: MAP-F06(攻城) → ACT-F01(活动进度)
**流程**: 攻城操作计入活动进度

### XI-MAP-08 建筑城防→攻城防御

**涉及系统**: BLD → MAP
**衔接点**: BLD-F06(城墙防御) → MAP-F06(攻城防御计算)
**流程**: 建筑系统城墙等级影响攻城战城防值

## 四、流程断裂点(GAP)

### GAP-MAP-01 地图系统与建筑系统无联动

**位置**: MAP-F04(征服) ↔ BLD(建筑)
**现象**: 占领领土后，建筑系统无任何响应
**影响**: 两个核心系统割裂，缺乏整体感
**建议**: 占领特定领土解锁建筑加成/新建筑

### GAP-MAP-02 领土等级提升缺乏玩家操作

**位置**: MAP-F11(领土等级)
**现象**: 等级提升主要靠"等待"，玩家无有意义的决策
**影响**: 可玩性低，变成纯放置
**建议**: 增加投资决策(选择投资方向: 产出/防御/特殊效果)

### GAP-MAP-03 攻城战缺少战斗过程展示

**位置**: MAP-F06(攻城)
**现象**: 攻城只是倒计时+结算，无战斗过程
**影响**: 缺乏紧张感和参与感
**建议**: 增加战斗动画/进度条/关键节点选择

### GAP-MAP-04 事件系统缺少后续影响

**位置**: MAP-F09(地图事件)
**现象**: 事件选择后无后续影响，每次独立
**影响**: 事件缺乏深度，玩家不关心选择
**建议**: 增加事件链(选择A→触发后续事件B/C)

### GAP-MAP-05 新手引导缺失

**位置**: MAP-F01(进入天下Tab)
**现象**: 新手进入天下Tab后不知道做什么
**影响**: 新手流失风险
**建议**: 增加新手引导(首次进入→引导征服第一块领土)

### GAP-MAP-06 领土放弃机制缺失

**位置**: MAP-F05(驻防管理)
**现象**: PRD未定义放弃领土的流程和后果
**影响**: 玩家无法战略撤退
**建议**: 增加放弃领土(损失驻防兵力，领土变为中立)

### GAP-MAP-07 攻城冷却期体验缺失

**位置**: MAP-F06(攻城后)
**现象**: 24h冷却期内无任何玩法
**影响**: 冷却期变成等待
**建议**: 冷却期内可进行"加固防御"/"恢复城防"等操作

### GAP-MAP-08 多人交互缺失

**位置**: 全局

**影响**: "天下"缺乏社交竞争感

## 五、PRD修订建议(PRD-R)

### PRD-R001 领土数量vs玩法深度

**PRD原文**: 15块领土
**问题分析**: 15块领土对于SLG来说偏少，征服很快就会完成
**玩家视角**: 玩家可能在短时间内占领所有领土，之后无事可做
**修订方案**: 增加到25-30块领土，或增加领土变体(资源点/关隘/都城)
**影响评估**: 增加游戏时长和策略深度
**优先级**: P1

### PRD-R002 地形加成感知度

**PRD原文**: 6种地形类型(平原/山地/水域/城池/森林/关隘)
**问题分析**: 地形加成只是数值修正，玩家感受不明显
**玩家视角**: 选择征服哪块领土时，地形因素可能被忽略
**修订方案**: 增加地形特殊效果(森林伏兵/水域渡河消耗/关隘一夫当关)
**影响评估**: 增加征服决策的策略深度
**优先级**: P1

### PRD-R003 攻城战策略深度

**PRD原文**: 攻城只需兵力达标+消耗
**问题分析**: 攻城决策过于简单(兵力够就打)，缺乏策略选择
**玩家视角**: 玩家没有"怎么打"的选择，只有"打不打"
**修订方案**: 增加攻城策略(强攻/围困/夜袭/内应)，不同策略不同消耗/时间/成功率
**影响评估**: 大幅提升攻城战可玩性
**优先级**: P0

### PRD-R004 事件系统深度

**PRD原文**: 4种事件类型，选择后一次性结束
**问题分析**: 事件缺乏深度和记忆性
**玩家视角**: 每次事件都是独立的，选择无长期影响
**修订方案**: 增加事件链/事件记忆/事件声望(如多次帮助商队→商队好感度→解锁特殊奖励)
**影响评估**: 增加事件系统的长期吸引力
**优先级**: P2

### PRD-R005 终局目标缺失

**PRD原文**: 无明确终局目标
**问题分析**: 占领所有领土后无事可做
**玩家视角**: "然后呢？"
**修订方案**: 增加"一统天下"终极目标(占领全部15块→触发统一事件→特殊奖励+转生加成)
**影响评估**: 提供明确的长期目标
**优先级**: P1
