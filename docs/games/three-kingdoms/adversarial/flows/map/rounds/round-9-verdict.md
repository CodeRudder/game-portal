### R9新增方法论: "设计公理声明"

R9在"设计意图验证"之前增加"设计公理声明"步骤:
1. **声明假设**: 列出推导依赖的公理假设(如"低兵力应有惩罚")
2. **标注性质**: 区分"实证结论"、"设计哲学判断"、"行业惯例"、"主观判断"
3. **列出反论**: 收集Challenger提出的反对意见(如"低兵力本身已是高风险")
4. **标注裁决状态**: "已验证"、"设计假设需测试"、"初始设计值需A/B测试"

**方法论评价**: 这是R9最重要的方法论进步，直接解决了R8"循环论证"的核心缺陷。R8的"设计意图验证"验证的是"R8数据是否符合R8自己的设计意图"——必要但不充分。R9的"设计公理声明"将验证范围扩展到"推导的前提假设是否被显式声明"——使质疑者能精准定位推导根基的主观性。k=1.75的3条公理声明是该方法论的最佳实践——假设声明(3条)+性质标注(设计哲学判断)+反论收集(3条)+裁决状态(需测试)四步完整。

**方法论的局限性**: "公理声明"解决了"假设可见性"问题，但未解决"假设可验证性"问题。R9的A/B测试建议(CH-MAP-R9-02)暴露了这个缺口——声明了假设但未提供完整的验证方案。**R10建议**: 在"公理声明"后增加"验证方案定义"步骤——为每条设计假设定义验证方法(实证/测试/迭代)、成功标准(通过/失败条件)和回退方案(假设被推翻时的调整路径)。

### R9新增方法论的缺陷: "三方交叉校验"缺失

R9暴露了一个关键的方法论缺陷: **模拟数值缺乏"行为描述→推算→模拟表"三方交叉校验**。

R9的休闲玩家模拟存在三方矛盾:
- 行为描述: 每周登录1-2次，每日+2获取
- 推算: 每日净变化约-0.14/天(Challenger推算)
- 模拟表: 每日净变化-3/天(R9声称)

三方不一致，但R9未执行交叉校验——如果在产出前执行了"行为描述→推算→模拟表"的三方校验，-3/天的矛盾会被发现。

**R10建议**: 在模拟产出流程中增加"三方交叉校验"步骤:
```
模拟产出验证流程(建议):
  1. 行为描述: 定义玩家行为(登录频率/事件参与/获取来源)
  2. 数值推算: 从行为描述推算每日/每周净变化(含衰减/豁免计算)
  3. 模拟表: 运行30天/60天模拟
  4. 三方校验: 模拟表结果必须与数值推算一致(误差<10%)
  5. 如不一致: 检查推算假设(衰减计算/豁免判定/per-faction语义)
```

- **可玩性连续五轮正增长(7.4→7.8→8.2→8.4→8.6→8.7)**，得益于每轮的体验改善累积(R9: 冷却10s扫描+destroy+per-faction豁免)。但增速从+0.2降至+0.1，因为休闲玩家模拟数据不可靠影响了平衡性结论的可信度。

### R10方法论改进建议

| 改进项 | 当前问题 | 建议方案 |
|--------|---------|---------|
| **三方交叉校验** | 模拟数值缺乏"行为描述→推算→模拟表"三方一致性验证 | 在模拟产出流程中增加三方校验步骤，误差>10%时触发修正 |
| **验证方案定义** | "公理声明"解决了可见性但未解决可验证性 | 为每条设计假设定义验证方法+成功标准+回退方案 |
| **A/B测试方法论** | 测试方案缺少关键要素(样本量/持续时间/统计方法) | 建立A/B测试方案模板(7要素: 样本量/持续时间/主指标/阈值/分流/效应量/对照) |
| **数值推导文档化** | 模拟表数值的推导过程不透明 | 每个模拟数值附推算公式(衰减计算/豁免判定/per-faction语义) |
| **沉没成本检查** | 期望值分析纳入沉没成本 | 在决策分析流程中增加"沉没成本检查"步骤 |
