## P0 质疑

### CH-MAP-R4-01 内应信产出后无消费闭环 [P0]

**质疑流程**: MAP-F08-01, MAP-F09-01, XI-MAP-13
**追问**: "然后呢？" — 内应信获得之后，玩家下一步做什么？

**问题**: R4 新增了3条内应信获取途径(攻城胜利20%掉落、地图事件掉落、事件分支掉落)，但内应信的消费流程(MAP-F06-02 攻城策略选择——使用内应策略)仅在"后置流程"中被引用，R4并未补充该流程的具体步骤。具体缺失包括:
1. 使用内应信时的前置条件(是否需要在攻城前消耗？是否需要在目标城池有NPC关系？)
2. 内应策略的完整流程(消耗数量、成功率计算、失败后果)
3. 内应信在背包中的堆叠上限、是否可交易、是否可丢弃
4. 内应信是否有保质期或使用等级限制

**影响**: 内应信道具处于"只进不出"状态。玩家获得道具后无法使用，道具形同虚设；如果MAP-F06-02已有定义但未在R4中被引用，则R4的数据追溯链(XI-MAP-13)不完整——声称"攻城策略选择时从背包扣取"却未说明扣取规则。

**建议**: 在R5中补充MAP-F06-02的完整流程定义，至少包含: 使用前置条件、消耗数量、成功率公式、成功/失败结果分支、UI交互步骤。

### CH-MAP-R4-02 离线事件登录时的处理压力 [P0]

**质疑流程**: MAP-F12-01
**追问**: "如果中途退出呢？" — 玩家登录后还没处理完事件就退出了怎么办？

**问题**: MAP-F12-01规定玩家登录后在"回归欢迎面板"处理离线事件，但存在以下流程断裂:
1. **退出未完成**: 如果玩家登录后逐个处理到第3个就退出了(关掉APP)，剩余2个事件的过期时间如何计算？是从登录时开始算还是从原始触发时间算？未定义。
2. **决策疲劳**: 最多5个事件同时弹出，每个事件需要阅读文本+选择分支，玩家刚打开游戏就要做5个决策，体验压力大。
3. **过期惩罚叠加**: 5个事件全部过期，50%概率触发"流寇骚扰"——但每个事件独立判定还是整体判定？如果是独立判定，5个事件都过期时有约3.125%概率叠加5层debuff(产出-10%×5=-50%)，这个数值极度惩罚。
4. **"一键忽略"的代价**: 一键忽略不获得任何奖励，但对于"阵营冲突"事件还叠加"阵营声望-10"，玩家可能不知道忽略的隐性代价。

**影响**: 离线回归体验差——本应是"回归奖励"变成"回归压力"。重度离线玩家(如隔天登录)几乎必然面对事件过期惩罚，反而惩罚了最需要被挽留的流失边缘玩家。

**建议**:
- 过期惩罚改为"最后一个过期事件才触发"(而非每个独立判定)
- 回归面板增加"一键快速处理"(自动选择推荐分支，简化决策)
- 未处理事件的过期时间从登录后重新计算(给玩家缓冲期)

## P1 质疑

### CH-MAP-R4-03 离线事件累积与在线事件上限冲突 [P1]

**质疑流程**: MAP-F12-01, TE-MAP-08
**追问**: "和XX系统有什么关系？" — 离线事件累积机制与在线事件上限(3个)如何共存？

**问题**: MAP-F12-01的GAP-MAP-05-R4已经标记了这个问题，但R4未给出解决方案。具体场景:
1. 玩家在线时已有2个未处理事件(未达上限3个)，此时下线。离线期间按TE-MAP-08规则继续累积事件，此时pendingEvents队列可能达到5个。
2. 玩家重新上线时，在线事件上限(3个)与离线队列(5个)如何合并？是先处理离线队列再恢复在线检测，还是在线上限瞬间被突破？
3. 如果离线队列有5个事件，玩家处理了3个后退出，再次上线时在线上限3个如何计算——已处理的3个是否占位？

**影响**: 事件系统的计数逻辑不一致，可能导致事件丢失(超过上限被丢弃)或事件轰炸(离线+在线事件同时弹出)。

**建议**: 明确定义: 离线队列与在线事件是两套独立计数器。玩家上线时优先处理离线队列，处理完毕后恢复在线事件检测。离线队列不占在线事件上限的配额。

### CH-MAP-R4-04 新增事件"战斗"分支缺乏流程定义 [P1]

**质疑流程**: MAP-F09-01, MAP-F09事件表修订
**追问**: "如果失败呢？" — 事件中的"战斗"分支失败了怎么办？

**问题**: R4新增了"山贼→战斗→内应信(15%)+装备"事件分支，但"战斗"的具体流程未定义:
1. 山贼战斗是使用什么机制？是自动结算(基于兵力)还是进入战斗系统？如果是战斗系统，触发的是哪种战斗模式？
2. 战斗失败的后果是什么？损失兵力？事件直接结束？还是可以重试？
3. "装备"奖励的品质和类型未定义，与装备系统的衔接(XI)缺失。
4. "遗迹发现→探索"分支也有类似问题——"探索"是否有失败概率？失败后果是什么？

**影响**: 玩家在事件中选择"战斗"分支时，不知道自己在承担什么风险。如果战斗失败只损失少量兵力，玩家会无脑选战斗；如果失败损失大量兵力，玩家会永远选安全分支。风险-收益不透明导致策略选择失去意义。

**建议**: 为每个需要"战斗"的事件分支定义: 战力门槛(或自动结算公式)、失败后果(兵力损失比例)、是否可重试。参考MAP-F06攻城战的胜率预估公式，在事件面板中显示战斗预估。

### CH-MAP-R4-05 产出上限"已满"缺乏主动管理手段 [P1]

**质疑流程**: MAP-F01-03
**追问**: "老玩家还做什么？" — 老玩家如何高效管理多个领土的产出上限？

**问题**: MAP-F01-03定义了产出上限机制和"已满"UI反馈，但缺少以下关键环节:
1. **无预警机制**: 玩家只有在产出"已满"后才知道，没有"即将满"的预警(如80%时黄色提示)。等看到"已满"时产出已经停了。
2. **无批量管理**: 老玩家可能拥有5~8块领土，每块领土的产出上限不同，需要逐个检查哪些"已满"——没有总览面板。
3. **无自动消耗建议**: "已满"后的建议是"消耗资源(升级建筑/征兵/投资)"，但没有一键跳转到这些操作的入口。
4. **产出上限数值不可见**: 玩家在领土详情中看不到当前上限值和距离上限的差额，只能看到"已满/未满"二态。

**影响**: 产出上限机制变成了"惩罚机制"而非"管理机制"。玩家的体验是"资源白白浪费了"而非"我需要规划资源使用"。老玩家管理多块领土时尤其痛苦。

**建议**:
- 产出气泡增加百分比显示(如"8500/10000")
- 80%时黄色预警，100%时红色"已满"
- 地图统计面板(MAP-F6)增加"产出概览"——列出所有领土的产出状态(正常/预警/已满)
- "已满"气泡点击后弹出快捷操作菜单(升级建筑/征兵/投资)

### CH-MAP-R4-06 内应信掉落机制的公平性 [P1]

**质疑流程**: MAP-F08-01, MAP-F09-01
**追问**: "这个数值有意义吗？" — 20%/15%/10%/25%/20%的掉落概率组合意味着什么？

**问题**: 内应信有5个独立掉落源，每个有不同概率。存在以下数值问题:
1. **概率叠加不透明**: 玩家无法感知自己"获得内应信的整体概率"是多少。数学上，如果一个活跃玩家每天攻城1次+事件2次，每天获得至少1封内应信的概率约为1-(0.8×0.85×0.9)=38.8%，看似合理但玩家感知不到。
2. **内应信稀缺性不确定**: 如果内应策略(MAP-F06-02)每次消耗1封，每天0.4封的期望获取率是否足够？如果每次消耗多封，则获取率严重不足。由于MAP-F06-02未定义消耗量(R4未补充)，概率数值的合理性无法验证。
3. **堆叠上限未知**: 内应信是否有持有上限？如果没有上限，长期积累后内应策略变成"免费策略"，破坏策略平衡。

**影响**: 掉落概率是拍脑袋定的还是经过数值验证的？在MAP-F06-02消费流程缺失的情况下，这些概率数值缺乏验证锚点。

**建议**: 在MAP-F06-02消费流程确定后，回过头验证内应信的获取-消耗平衡。建议内应信持有上限为10封，避免无限积累。

## P2 质疑

### CH-MAP-R4-07 "阵营冲突"事件与声望系统衔接缺失 [P2]

**质疑流程**: MAP-F09-01, MAP-F09事件表修订
**追问**: "和XX系统有什么关系？" — "阵营声望"是什么系统？

**问题**: "阵营冲突→刺探情报→声望"和"按兵不动"分支中，"阵营声望-10"作为过期惩罚。但"阵营声望"这个数据维度在R4中首次出现，之前未定义:
1. 阵营声望的初始值、最大值、获取途径均未定义
2. 阵营声望对游戏的具体影响未定义(影响NPC态度？影响商店折扣？影响阵营任务？)
3. -10声望意味着什么？如果玩家声望为0，是否会变成负数？
4. 声望系统与MAP-F09"选择分支"表中"声望加成"(声望等级×2%产出)是否同一系统？

**影响**: 玩家无法评估"阵营声望-10"的严重程度，"按兵不动"分支的选择依据缺失。

**建议**: 明确"阵营声望"数据的定义范围(来源/上限/效果)，或在R4中引用声望系统的PRD编号(XI衔接)。

### CH-MAP-R4-08 强攻倍率调整缺乏策略差异化说明 [P2]

**质疑流程**: MAP-F06-04-R4
**追问**: "这个机制有趣吗？" — 强攻从×0.8调到×0.9后，与围困(×1.0)的差异还剩多少？

**问题**: 强攻×0.9 vs 围困×1.0，差距仅10%。考虑到:
- 强攻可能有"速度快"的优势(攻城时间更短？未定义)
- 围困可能有"时间长"的劣势(未定义)
- 如果两者的时间成本接近，10%的奖励差距不足以让玩家选择风险更高的强攻

R4只调整了奖励倍率，但未说明强攻与其他策略的核心差异点是什么。玩家视角下，4种策略(强攻/围困/夜袭/内应)除了倍率不同，还有什么选择理由？

**影响**: 策略选择变成纯数学比较(选倍率最高的)，缺乏策略深度。

**建议**: 为每种策略定义差异化维度: 时间成本(强攻快/围困慢)、兵力损耗(强攻高/围困低)、前置条件(夜袭需夜袭令/内应需内应信)、特殊效果(强攻损坏城防/围困逼降守军)。

### CH-MAP-R4-09 离线事件过期后"流寇骚扰"叠加合理性 [P2]

**质疑流程**: MAP-F12-01
**追问**: "这个数值有意义吗？" — 50%概率×每个过期事件独立判定的设计是否经过验证？

**问题**: MAP-F12-01 Step 7规定"已过期事件50%概率触发流寇骚扰(-10%产出2h)"，且不同事件类型的过期惩罚不同(阵营冲突是"声望-10")。但:
1. 50%概率是每个事件独立判定。如果玩家离线24h+，5个事件全部过期，触发0~5层debuff的分布不均匀(二项分布)。
2. 最坏情况: 5个事件全部触发debuff，产出-50%持续2h，这对回归玩家来说是"登录惩罚"。
3. "流寇骚扰"与"阵营冲突过期(声望-10)"是不同惩罚类型，玩家无法通过同一策略规避。
4. 过期惩罚的2h是从什么时候开始计算？从事件过期时刻还是玩家登录时刻？

**影响**: 离线回归的负面体验设计不合理。P2是因为与CH-MAP-R4-02有重叠(都在MAP-F12-01)，但如果P0的建议不被采纳，此问题会升级。

**建议**: 与CH-MAP-R4-02合并处理。核心原则: 离线回归应该是"奖励"而非"惩罚"。

## 总结

| 级别 | 数量 | 核心主题 |
|------|------|---------|
| P0 | 2 | 内应信消费闭环断裂、离线回归体验设计 |
| P1 | 4 | 事件计数冲突、战斗分支未定义、产出管理缺失、数值验证缺失 |
| P2 | 3 | 阵营声望未定义、策略差异化不足、过期惩罚合理性 |

**R4 整体评价**: R4在"增加内容"方面做得很好(内应信途径、科技定义、离线事件)，但在"闭环验证"方面存在明显不足。内应信只定义了"进"没定义"出"，离线事件只定义了"正常路径"没定义"异常路径"。建议R5重点补充消费侧流程(MAP-F06-02)和异常场景处理(离线中断恢复)。
