### R11核心方法论: "数值推导过程文档化"

R11在期望值分析中建立了"数值推导过程文档化"标准:

```
R11期望值推导文档标准:
  1. 公式定义: 显式提供临界奖励公式(R_critical = ...)
  2. 场景参数: 列出每个场景的已知量(S/P(W)/P(L)/L_win/L_lose)
  3. 推导过程: 从公式到数值的完整推导(使用精确分数而非近似小数)
  4. 验算: 将推导结果代入原始公式验证E=0
  5. 三方校验: 公式推导=数值验算=表格阈值，三方一致
```

**方法论评价**: 这是R11最重要的方法论贡献，直接解决了R10"期望值表数值不可信"的核心缺陷。R10的阈值计算错误(偏差+165)如果在产出前执行了推导过程文档化，会被立即发现——因为推导过程会暴露近似小数(0.67/0.33)的累积误差。R11的4个场景逐场景推导+验算+三方校验是该方法论的最佳实践。

**方法论的局限性**: "数值推导过程文档化"解决了"数值可验证性"问题，但R11的10000兵场景(CH-MAP-R11-01)暴露了一个缺口——推导过程文档化应覆盖所有数值产出(包括"不适用"场景)。**R12建议**: 在推导过程文档化标准中增加"完整性检查"步骤——确认所有数值产出均有推导过程。

### R11新增方法论: "方法论执行检查"

R11建立了"方法论执行检查清单":

```
R11方法论执行检查流程:
  1. 列出前一轮声明的所有方法论
  2. 逐项检查当前轮是否执行
  3. 记录执行结果(完全执行/部分执行/未执行)
  4. 如未执行: 说明原因并计划执行时间
```

**方法论评价**: 这是R11在方法论治理上的贡献。R10建立了多项方法论(三方校验/沉没成本检查/A/B测试模板)，但未验证后续轮次是否持续执行。R11的方法论执行检查确保了方法论不是"一次性声明"而是"持续执行"。5项方法论在R11中全部被执行(3项完全+2项扩展)——这是方法论治理的良好实践。

### R11新增方法论: "编号唯一性检查"

R11在流程编号管理中增加"编号唯一性检查"步骤:

```
R11编号唯一性检查流程(四步法):

  Step 2: 新增流程编号前检查是否已存在
  Step 3: 如存在冲突，使用后缀区分(如R10a, R10b)或分配新子序号

```

**方法论评价**: R10的MAP-F12-01-R10编号重复问题(CH-MAP-R10-03)如果在产出前执行了编号检查，会被立即发现。该检查机制具有通用性——任何需要编号管理的文档(流程/PRD/测试用例)都应执行编号唯一性检查。
