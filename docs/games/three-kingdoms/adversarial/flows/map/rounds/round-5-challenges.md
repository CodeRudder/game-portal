## P0 质疑

### CH-MAP-R5-01 内应暴露冷却与策略卡片UI交互断裂 [P0]

**质疑流程**: MAP-F06-07 Step 2, Step 4, Step 11b
**追问**: "然后呢？" — 内应暴露debuff生效后，玩家在策略卡片上看到什么？

**问题**: MAP-F06-07定义了内应失败后的暴露机制（Step 11b: "目标城池24h内对该策略免疫"），但Step 2的策略卡片状态判定仅检查背包内应信数量，未检查目标城池的暴露状态。具体断裂:

1. **卡片状态判定遗漏**: Step 2定义了两种卡片状态（有内应信=可点击/无内应信=置灰），但缺少第三种状态——"有内应信但目标城池处于暴露冷却中"。此时卡片应显示什么？置灰+冷却倒计时？还是仍可点击但成功率归零？
2. **校验顺序不明确**: Step 4的系统校验只检查背包数量（`insiderLetter ≥ 1`），未检查`city.insiderExposed`状态。如果校验通过但城池处于暴露期，是在Step 5计算时将成功率设为0%，还是在Step 4就拦截？
3. **玩家决策误导**: 如果卡片仍可点击（显示"持有xN"），玩家选择内应策略后才发现"成功率0%"，这是严重的UX欺骗——玩家已经形成了"内应策略可行"的心智模型，却被系统在最后一步否定。
4. **暴露冷却的UI展示缺失**: GAP-MAP-06-R5仅标记"需UI系统确认"，但未定义具体展示方案。冷却倒计时显示在哪里？策略卡片上？城池详情面板上？攻城确认弹窗上？

**影响**: 这是R5声称"闭合内应信生命周期"的核心漏洞。生命周期不仅仅是数据链路闭合（获取→消耗），还需要交互链路闭合（用户感知→操作→反馈）。当前状态是数据闭合、交互断裂。

**建议**: 在MAP-F06-07 Step 2中增加第三种卡片状态判定:
- 若`city.insiderExposed = true`: 卡片置灰，显示"内应已暴露(剩余XX:XX)"，点击后弹出提示"该城池内应已被识破，请等待冷却结束"
- Step 4系统校验增加`city.insiderExposed = false`条件
- 在GAP-MAP-06-R5中明确: 冷却倒计时显示在策略卡片右上角角标 + 攻城确认弹窗的策略说明中

### CH-MAP-R5-02 快速处理"推荐绕行/按兵不动"等于变相忽略 [P0]

**质疑流程**: MAP-F12-01-R5 Step 7, 快速处理推荐逻辑表
**追问**: "这个机制有趣吗？" — 快速处理山贼和阵营冲突事件，玩家得到什么？

**问题**: MAP-F12-01-R5的快速处理推荐逻辑存在严重的"奖励空洞":

1. **山贼事件快速处理 = 零收益**: 推荐分支为"绕行"，快速处理奖励为"无奖励"。玩家点击"一键快速处理"后，山贼事件被消耗，玩家什么都没得到。这与"一键忽略"的结果完全相同——都是零收益。
2. **阵营冲突事件快速处理 = 零收益**: 推荐分支为"按兵不动"，快速处理奖励为"无奖励"。同样，与忽略无异。
3. **6种事件中2种快速处理无收益**: 在6种事件类型中，山贼(25%概率)和阵营冲突(概率未明确)的快速处理完全没有奖励。这意味着"一键快速处理"并非真正的"奖励机制"——对约30-40%的事件来说，它只是"不惩罚的忽略"。
4. **与R5核心目标矛盾**: R5声称"离线回归应该是奖励而非惩罚"，但快速处理山贼/阵营冲突的收益为零，玩家的体验是"我点了快速处理，但什么都没得到"。这不是奖励，这是"不惩罚"——而"不惩罚"不等于"奖励"。

**影响**: 快速处理机制的价值被大幅稀释。玩家会逐渐发现: 山贼和阵营冲突用快速处理等于白用，不如直接忽略(至少忽略还有确认弹窗的心理缓冲)。这导致快速处理功能的使用率下降，最终沦为"流民和商队专用按钮"。

**建议**:
- 方案A: 快速处理"绕行"时给予少量保底奖励（如铜钱x100），体现"虽然没战斗但搜集了一些情报/物资"
- 方案B: 快速处理"按兵不动"时给予微量声望(+1)，体现"保持克制获得了阵营认可"
- 方案C: 快速处理按钮对无收益事件显示特殊提示——"山贼: 快速处理将绕行，无奖励。建议登录后手动处理获取装备奖励"

## P1 质疑

### CH-MAP-R5-03 山贼战斗胜率公式数学错误——兵力不影响胜率 [P1]

**质疑流程**: MAP-F09-02 山贼战斗公式
**追问**: "这个数值有意义吗？" — 山贼战力公式让玩家兵力变得无关紧要？

**问题**: MAP-F09-02的山贼战斗胜率公式存在数学错误:

```
山贼战力 = 玩家总兵力 × 山贼难度系数
战斗胜率 = (玩家总兵力 / (玩家总兵力 × 山贼难度系数)) × 50% + 修正
         = (1 / 山贼难度系数) × 50% + 修正
```

**玩家总兵力在公式中被完全约掉了。** 这意味着:
- 带100兵去打山贼，和带100000兵去打山贼，胜率完全相同
- 胜率仅取决于"山贼难度系数"(由玩家等级决定)和地形/科技修正
- 玩家无法通过"带更多兵"来提升胜率，只能被动接受等级决定的固定胜率

**具体数值验证**（假设无地形/科技修正）:

| 玩家等级 | 难度系数 | 胜率 | 说明 |
|---------|---------|------|------|
| Lv10 | ×0.3 | 50%/0.3 = 166.7% → min(95%, 95%) = 95% | 几乎必胜 |
| Lv30 | ×0.5 | 50%/0.5 = 100% → 95% | 几乎必胜 |
| Lv50 | ×0.7 | 50%/0.7 = 71.4% | 较高胜率 |
| Lv70 | ×0.9 | 50%/0.9 = 55.6% | 一半一半 |

等一下——重新计算。公式是`(玩家总兵力 / (玩家总兵力 × 山贼难度系数)) × 50%`，即`(1/难度系数) × 50%`。对于×0.3难度，结果是166.7%，被min(95%)截断为95%。这意味着低等级玩家打山贼几乎必胜，高等级玩家反而胜率下降。

这与设计意图完全相反——低等级玩家应该面对"弱匪"(容易)，高等级玩家面对"山贼王"(困难)——但胜率差异仅来自难度系数，而非玩家的实际兵力投入。**玩家的兵力决策变得毫无意义。**

**影响**: 山贼战斗变成了"等级决定论"而非"策略决策"。玩家无法通过增加兵力来提升胜率，战斗预估面板中的"预计损失"也因此变得不可靠——因为胜率与兵力无关，"带多少兵"的决策失去了策略深度。

**建议**: 修正公式，将山贼战力改为独立数值（而非基于玩家兵力的倍率）:
```
山贼战力 = 基础战力 × 难度系数（基于玩家等级）
基础战力 = 玩家等级 × 100（或其他固定锚点）
战斗胜率 = (玩家总兵力 / 山贼战力) × 50% + 修正
```
这样玩家增加兵力确实能提升胜率，"预计损失"才有意义。

### CH-MAP-R5-04 过期时间"登录重算"可被反复登录无限重置 [P1]

**质疑流程**: MAP-F12-01-R5 Step 4, MAP-F12-02 Step 5
**追问**: "如果失败呢？" — 如果玩家利用重算机制规避过期惩罚呢？

**问题**: MAP-F12-01-R5规定过期时间从登录时重新计算24h（Step 4: `event.expiresAt = loginTime + 24h`），MAP-F12-02 Step 5也确认"下次登录时继续从上次中断处处理，过期时间重新计算"。这创造了以下利用空间:

1. **无限重置漏洞**: 玩家可以在过期前登录一次（不处理任何事件），所有事件的过期时间重置为`loginTime + 24h`。然后退出，24h内再登录一次，再次重置。理论上玩家可以永远不过期——只要每24h登录一次但不处理事件。
2. **快速处理变无限缓冲**: 玩家登录 → 看到5个事件 → 不处理 → 退出 → 下次登录 → 事件仍在，过期时间重置 → 循环。快速处理的"80%奖励"变成了"永远100%的保底"——因为事件永远不会过期。
3. **与双计数器的冲突**: MAP-F12-02规定"离线队列清空后恢复在线事件检测"。如果玩家永远不清空离线队列（通过反复登录重置过期时间），在线事件检测将永远被暂停——玩家失去了在线事件的触发机会。

**影响**: 过期惩罚机制形同虚设。R5声称"仅最后一个过期事件触发惩罚"，但如果事件可以被无限重置，连"最后一个"都不会触发。这不是"奖励化"，这是"惩罚移除"——而完全移惩罚会降低事件的紧迫感和决策权重。

**建议**:
- 方案A: 过期时间仅重算一次。首次登录时重算24h，之后不再重算——即使玩家再次登录，过期时间仍从首次登录时刻计算
- 方案B: 过期时间累计衰减。每次登录重算但过期时间缩短（如首次24h → 重算后12h → 再重算后6h），防止无限重置
- 方案C: 离线队列设置总存活时间上限（如72h），超过后无论是否登录都强制过期

### CH-MAP-R5-05 遗迹探索"部分成功"概率区间设计失当 [P1]

**质疑流程**: MAP-F09-02 遗迹探索公式
**追问**: "这个数值有意义吗？" — 三档结果的概率分布是否合理？

**问题**: MAP-F09-02定义了遗迹探索的三档结果:

```
探索成功率 = 70%(基础) + 科技加成(攻城术×1%) + 地形修正
成功(70%+): 完整奖励
部分成功(30%~70%): 50%奖励
失败(<30%): 无奖励+损失10%兵力
```

问题在于基础成功率就是70%，这意味着:

1. **"部分成功"几乎不会触发**: 基础成功率70%，即随机数落在0~70%区间为成功，70%~100%区间需要进一步判定。但"部分成功"的区间是30%~70%——这是对随机数的判定还是对成功率的判定？如果是对成功率的判定，则基础70%的玩家直接落入"成功"档，"部分成功"和"失败"永远不会触发。
2. **三档判定逻辑混乱**: 如果是`random(0,100) < 探索成功率`判定成功，则只有成功/失败两档，没有"部分成功"的空间。如果是三档判定（如`random(0,100) < 30%`为失败，`30%~70%`为部分成功，`>70%`为成功），则基础成功率70%毫无意义——最终结果完全由random决定。
3. **科技加成的实际效果不明**: 如果基础70%已经落入"成功"档，科技加成(+1%/级)有什么用？它只是把成功率从70%提到80%，但结果判定仍然是"成功"。科技加成只有在三档判定模式下才有意义——但三档判定模式下基础成功率是多余的参数。

**影响**: 遗迹探索的结果判定逻辑不清晰，开发者无法正确实现，玩家无法理解"提升科技有什么用"。三档结果设计的初衷是增加探索的层次感，但当前公式使其形同虚设。

**建议**: 明确判定逻辑为三档随机判定:
```
random(0,100):
  < 失败阈值 → 失败(无奖励+兵力损失)
  失败阈值 ~ 部分成功阈值 → 部分成功(50%奖励)
  > 部分成功阈值 → 成功(完整奖励)

失败阈值 = max(5%, 30% - 科技加成 - 地形修正)
部分成功阈值 = max(30%, 70% - 科技加成 - 地形修正)
```
这样科技加成同时降低失败率和提升完整成功率，玩家能明确感知"升级科技→探索变好"。

### CH-MAP-R5-06 产出概览面板缺乏批量操作能力 [P1]

**质疑流程**: MAP-F01-04 Step 5, Step 6
**追问**: "老玩家还做什么？" — 老玩家看到5个领土预警后，需要逐个操作？

**问题**: MAP-F01-04解决了R4的预警缺失问题（CH-MAP-R4-05），但批量管理仍然不足:

1. **概览面板只展示不操作**: 产出概览面板（Step 5）列出了所有领土的产出状态，但操作入口仍需逐个点击气泡（Step 6）。老玩家有5-8块领土，看到"3个预警、2个已满"后，需要逐一点击气泡→选择操作→跳转→返回→点击下一个气泡，操作链路过长。
2. **无优先级排序**: 概览面板按什么顺序排列？如果按领土名称排序，玩家需要自己判断"哪块最紧急"。应该按产出比例从高到低排序，让"已满"的排在最前面。
3. **快捷操作菜单缺少"一键收取"**: Step 6的快捷操作菜单包括"升级建筑/征兵/投资/科技研究"，但没有"一键收取产出"。如果产出接近上限是因为玩家没有及时收取，最直接的解决方案是"收取"而非"升级上限"。
4. **预警阈值不可配置**: 80%预警阈值是固定的，但不同玩家对"什么时候该管理产出"有不同的偏好。硬核玩家可能希望60%就预警，休闲玩家可能95%才关心。

**影响**: 产出管理从"没有工具"升级为"有工具但不好用"。概览面板的价值在于"总览"，但如果总览后仍需逐个操作，面板就变成了"焦虑面板"——看得见问题但解决起来很麻烦。

**建议**:
- 概览面板默认按产出比例降序排列（已满排最前）
- 增加"一键收取全部"按钮（收取所有领土的待收取产出）
- 预警阈值在设置中可调（60%/70%/80%/90%）

## P2 质疑

### CH-MAP-R5-07 内应信"永久有效"与"稀有道具"定位矛盾 [P2]

**质疑流程**: MAP-F06-07 内应信道具定义, XI-MAP-14 平衡验证
**追问**: "这个数值有意义吗？" — 永久有效的稀有道具，长期玩家的存量会如何变化？

**问题**: MAP-F06-07定义内应信"保质期: 无，永久有效"，堆叠上限10封。XI-MAP-14计算日均获取0.4封，获取-消耗比1:7.5，结论是"偏稀缺"。但长期来看:

1. **存量饱和**: 活跃玩家每天获取0.4封，如果内应策略不是每天都用（因为不是每次攻城都值得用内应），存量会逐渐积累。约25天后达到上限10封。此后如果玩家不常用内应策略，存量长期维持在10封——内应策略变成"随时可用的免费策略"。
2. **稀缺感消失**: 当玩家背包长期有10封内应信时，内应策略不再是"稀有策略"，而是"默认策略"（因为奖励倍率×1.5最高）。此时内应信的"稀缺性"仅对新玩家成立，对老玩家不成立。
3. **与堆叠上限的意图不匹配**: 堆叠上限10封的意图是"防止无限积累"，但10封对于日均消耗0.4-1封的玩家来说已经非常充裕。上限应该更低（如5封），或者引入保质期（如30天）来维持稀缺感。

**影响**: 内应信的长期经济平衡存在隐患。当前设计在前25天表现良好（稀缺感强），但25天后逐渐失去稀缺性。这是一个"长线问题"，不会在短期测试中暴露，但会影响游戏的长期策略深度。

**建议**:
- 方案A: 降低堆叠上限至5封，维持稀缺感
- 方案B: 引入软保质期——内应信30天后品质降级（成功率+20% → +10%），鼓励及时使用
- 方案C: 不改设计，但在R6中验证长期存量数据，作为后续平衡调整的依据

### CH-MAP-R5-08 声望来源与消耗不对称——只有事件加减，缺乏系统性消耗 [P2]

**质疑流程**: XI-MAP-15 阵营声望数据框架
**追问**: "然后呢？" — 声望达到100后玩家做什么？声望有什么持续消耗？

**问题**: XI-MAP-15定义了声望的来源和效果，但来源-消耗不对称:

1. **来源多、消耗少**: 声望来源有5种（阵营冲突+5、过期-5、忽略-5、攻城+3、日常+2），但消耗仅有"过期/忽略"两种被动消耗。玩家如果积极处理事件+日常攻城，声望会稳步上升直至100封顶。
2. **声望100后无事可做**: 声望到100后，所有来源都变成"无效产出"——+5/+3/+2全部浪费。此时阵营冲突事件的"刺探情报"分支失去了声望价值，玩家只会选择"按兵不动"（零风险）。
3. **缺乏声望消费场景**: 声望效果是被动折扣（商店-20%/NPC好感+10%），没有主动消费声望的机制。玩家无法"花费声望"换取特殊物品或特权，声望变成了"积累后就无用的数字"。

**影响**: 声望系统在中后期失去驱动力。当玩家声望达到80+（崇敬级别），所有声望相关事件分支的选择依据消失——选什么都一样，因为声望已经满了。这与"阵营声望影响策略选择"的设计意图矛盾。

**建议**:
- 引入声望消费场景: 如"声望x20兑换阵营专属道具"、"声望x10刷新阵营商店"
- 或引入声望衰减: 每周自然衰减5点，维持玩家持续关注声望管理
- 或在R6中将声望与其他系统（如阵营战、联盟系统）深度绑定，创造持续消耗

## 总结

| 级别 | 数量 | 核心主题 |
|------|------|---------|
| P0 | 2 | 内应暴露UI交互断裂、快速处理对部分事件无收益 |
| P1 | 4 | 战斗公式数学错误、过期时间可无限重置、探索判定逻辑混乱、产出管理仍需逐个操作 |
| P2 | 2 | 内应信长期稀缺性隐患、声望系统缺乏持续消耗 |

### R5整体评价

R5在"补全流程"方面取得了显著进展——内应信生命周期从数据链路角度已闭合，离线体验从"纯惩罚"改善为"有条件奖励"，战斗分支有了完整的公式和结果定义。但R5在"闭环验证"上仍有不足:

1. **交互闭环不完整**: 内应信的数据链路闭合了（获取→存储→消耗），但交互链路断裂（暴露冷却在UI上不可见、不可操作）。
2. **数值公式有硬伤**: 山贼战斗公式让兵力变得无关、遗迹探索的三档判定与基础成功率冲突——这些都是可实现性问题，不修正则无法开发。
3. **机制设计有漏洞**: 过期时间可被无限重置、快速处理对部分事件无收益——这些是"设计意图"与"实际效果"的偏差。

**建议R6重点**: 数值公式修正（山贼战斗/遗迹探索）+ 过期时间防滥用机制 + 快速处理保底奖励设计。
