# MAP 天下地图 — 迭代式开发改进计划

> **版本**: v1.0 | **日期**: 2026-05-03

---

## 1. 开发理念

### 1.1 核心原则

以**玩家视角**驱动开发，通过**对抗性测试**发现真实问题，用**迭代循环**持续改进。

```
评测(玩家视角) → 发现问题 → 改进实现 → 测试验证 → 架构审查 → 评测(下一轮)
```

### 1.2 对抗性测试定义

对抗性测试 = 以挑剔玩家的视角，实际操作游戏的每个流程，验证：
- 功能是否**真实可用**（不是"代码存在"而是"玩家能用"）
- 体验是否**流畅自然**（无卡顿、无错位、无断裂）
- 信息是否**所见即所得**（地图上的标记与实际数据一致）
- 流程是否**完整闭环**（从操作到反馈到结果，无中间断裂）

### 1.3 E2E真实测试要求

每轮评测必须使用Playwright进行E2E测试：
- 启动游戏 → 导航到天下Tab → 实际操作
- 截图记录每个关键步骤
- 对比预期行为与实际行为
- 记录所有不符合预期的问题

## 2. 迭代循环结构

### 2.1 单轮迭代流程

```
┌─────────────────────────────────────────────────┐
│ 第N轮迭代                                        │
│                                                   │
│  1. 评测 (30min)                                  │
│     ├─ 玩家视角E2E测试                            │
│     ├─ 对抗性检查每个功能点                        │
│     └─ 生成评测报告(问题清单+截图)                 │
│                                                   │
│  2. 改进 (60min)                                  │
│     ├─ 按优先级修复问题                            │
│     ├─ 补充缺失功能                               │
│     └─ 优化体验细节                               │
│                                                   │
│  3. 测试 (20min)                                  │
│     ├─ 运行单元测试                               │
│     ├─ 运行集成测试                               │
│     └─ 回归测试(确保旧功能未破坏)                  │
│                                                   │
│  4. 架构审查 (15min)                              │
│     ├─ 代码质量检查                               │
│     ├─ 模块依赖检查                               │
│     └─ 性能影响评估                               │
│                                                   │
│  5. 文档 (10min)                                  │
│     ├─ 生成本轮过程文档                            │
│     ├─ 更新任务清单                               │
│     └─ 回顾最近3轮改进趋势                        │
│                                                   │
└─────────────────────────────────────────────────┘
```

### 2.2 评测检查清单

每轮评测必须覆盖以下检查点：

#### A. 像素地图渲染
- [ ] 地图打开时自动居中，无大面积黑色空白
- [ ] 城市标记位置与实际地图数据一致
- [ ] 资源点显示对应图标(🌾💰⚔️🌟)
- [ ] 道路网络清晰可见
- [ ] 阵营颜色正确(己方绿/敌方红/中立灰)
- [ ] 缩放/平移流畅无卡顿

#### B. 城市交互
- [ ] 点击城市高亮选中
- [ ] 点击空白区域取消选中
- [ ] 信息面板显示城市详情
- [ ] 资源点可点击查看信息
- [ ] 选中城市不移动视窗

#### C. 攻城流程
- [ ] 选中敌方城市显示攻城按钮
- [ ] 点击攻城弹出确认弹窗
- [ ] 确认弹窗显示消耗/条件/兵力选择
- [ ] 攻城执行后显示结果弹窗
- [ ] 攻城成功后地图颜色更新
- [ ] 攻城动画正常播放
- [ ] 攻城后产出系统更新

#### D. 行军流程
- [ ] 行军路线在地图上显示(黄色虚线)
- [ ] 行军精灵沿路线移动
- [ ] 行军到达后切换为攻城动作
- [ ] 行军精灵全程可见(不消失)
- [ ] 多支军队同时行军互不干扰

#### E. 离线/产出
- [ ] 离线>10秒后弹出奖励弹窗
- [ ] 奖励弹窗显示正确资源
- [ ] 领取后资源更新
- [ ] 产出面板显示各领土产出
- [ ] 存储满时显示警告

#### F. 鸟瞰图
- [ ] 小地图在右下角悬浮
- [ ] 点击小地图跳转视窗
- [ ] 拖拽小地图移动视窗
- [ ] 视口矩形正确显示

#### G. 性能
- [ ] 地图渲染60fps无卡顿
- [ ] 缩放/平移响应及时
- [ ] 无内存泄漏

## 3. 问题分类与优先级

| 优先级 | 类型 | 描述 | 处理方式 |
|:------:|------|------|----------|
| P0 | 功能缺失 | 核心流程不可用 | 立即修复 |
| P1 | 流程断裂 | 流程中间断开 | 当轮修复 |
| P2 | 体验问题 | 不流畅/不直观 | 下轮优化 |
| P3 | 细节打磨 | 视觉/交互细节 | 后续迭代 |

## 4. 迭代版本规划

### Round 1: 基础功能完整性
**目标**: 所有核心功能可用，流程无断裂

重点:
- 资源点在地图上显示图标并可点击
- 攻城流程完整闭环
- 行军流程完整闭环
- 离线奖励正常触发

### Round 2: 数据一致性
**目标**: 所见即所得，地图标记与实际数据一致

重点:
- 城市位置与LANDMARK_POSITIONS一致
- 阵营颜色与实际ownership一致
- 产出数据与ProductionSystem一致
- A*路径与实际道路一致

### Round 3: 交互体验优化
**目标**: 操作流畅，反馈及时

重点:
- 缩放/平移流畅
- 点击响应准确
- 动画平滑无卡顿
- 快捷键支持

### Round 4: 性能与稳定性
**目标**: 60fps无卡顿，无崩溃

重点:
- 脏标记渲染优化
- 视口裁剪
- 内存管理
- 边界情况处理

### Round 5: 完整E2E验证
**目标**: 全链路无报错，玩家体验优秀

重点:
- 完整E2E测试覆盖
- 存档/读档一致性
- 多设备适配
- 最终文档更新

## 6. 停止条件

迭代循环在以下条件满足时停止：
1. 所有P0/P1问题清零
2. 评测检查清单通过率 ≥ 95%
3. 单元测试通过率 ≥ 99%
4. E2E测试覆盖所有核心流程
5. 连续2轮无新增P0/P1问题
