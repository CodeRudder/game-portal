## B. 道路连接与行军系统 ✅ 已完成

> **核心设计原则**: 城市可达性以地图数据(world-map.txt)中的道路连接为准。
> 道路网络从地图ASCII数据中解析，A*寻路在地图网格上沿道路格子计算，
> 不依赖手写的ADJACENCY_MAP。相邻关系由道路路径自动推导。

### B1: 道路网格构建 ✅

1. ✅ 解析world-map.txt获得100×60的地图网格(cell[y][x])
2. ✅ 标记可行走格子: 道路(░/-/|/+/·/:/=)和城市位置(A-Z)为可行走
3. ✅ 构建WalkabilityGrid: boolean[60][100]，道路=true，地形=false
4. ✅ 城市位置(LANDMARK_POSITIONS)在网格中标记为可行走节点
5. ✅ 提取的城市坐标与LANDMARK_POSITIONS配置一致
6. ✅ 道路网格在游戏初始化时构建一次，缓存复用

### B2: A*寻路(基于地图网格) ✅

1. ✅ 实现A*寻路算法，在WalkabilityGrid上计算两城市间路径
2. ✅ 路径输出为GridPosition[]序列(沿道路格子的逐格坐标)
3. ✅ 启发函数使用曼哈顿距离(4方向移动)或欧几里得距离(8方向)
4. ✅ 不可达城市返回空路径(无道路连接时提示"不可达")
5. ✅ 路径经过关隘(pass-*)时正确衔接(关隘也是可行走节点)
6. ✅ 路径经过资源点(res-*)时正确衔接
7. ✅ A*性能: 100×60网格上单次寻路<5ms

### B3: 城市相邻关系推导 ✅

1. ✅ 从道路网格自动推导城市相邻关系(替代手写ADJACENCY_MAP)
2. ✅ 推导方法: 对每对城市执行A*，若路径存在且站点数≤阈值则视为相邻
3. ✅ 或: 从城市位置沿道路做BFS，遇到的第一个其他城市即为相邻城市
4. ✅ 推导结果缓存为adjacency graph，供攻城/行军判断使用
5. ✅ 推导结果与原ADJACENCY_MAP交叉验证，发现不一致时记录日志
6. ✅ 地图数据变更时(编辑器修改道路)自动重新推导

### B4: 行军路线计算与显示 ✅

1. ✅ 选中己方城市→点击目标城市→调用A*计算网格级路径
2. ✅ 路线沿道路格子逐格经过(非城市间跳跃)
3. ✅ 路线高亮显示在Canvas地图上(黄色虚线连接路径点)
4. ✅ 显示预计到达时间(基于路径长度和地形移动速度)
5. ✅ 显示途径站点列表(洛阳→虎牢关→许昌)，从路径中提取经过的城市
6. ✅ 不可达城市不显示路线，提示"无道路连接"

### B5: 行军精灵渲染 ✅

1. ✅ 发起行军后，Canvas上出现行军精灵(像素小人队列)
2. ✅ 精灵沿A*计算的网格路径逐格移动(非城市间跳跃)
3. ✅ 精灵数量反映兵力(>1000:5个, 500~1000:3个, <500:1个)
4. ✅ 精灵颜色反映阵营(魏蓝/蜀红/吴绿)
5. ✅ 精灵有行走动画(腿交替，8fps)
6. ✅ 道路上移动速度×1.5，山地×0.5
7. ✅ 行军到达后精灵不消失，切换为攻城动作动画
8. ✅ 攻城完成后精灵显示撤退/回城路径动画，直到返回出发城市
9. ✅ 军队从派遣到回城全程在地图上可见，任何阶段不失踪
10. ✅ 行军取消后精灵沿原路返回出发城市
11. ✅ 多支军队同时行军互不干扰，精灵不重叠

### B6: 攻城效果渲染 ✅

1. ✅ 攻城开始→目标城市区域出现颜色渐变(原阵营色→新阵营色)
2. ✅ 阶段1(0~40%): 战斗粒子效果(橙色火花)
3. ✅ 阶段2(40~70%): 旗帜升起动画(旗杆+新阵营旗帜)
4. ✅ 阶段3(70~100%): 结果文字("攻占成功"绿色/"攻城失败"红色)
5. ✅ 动画总时长3秒，结束后自动清理
6. ✅ 多城市同时攻城时动画独立播放
7. ✅ 动画播放不影响其他地图交互

### B7: 道路数据一致性验证 ✅

1. ✅ world-map.txt中的道路(░符号)构成的网络覆盖所有城市
2. ✅ 每个城市至少有一条道路连接到另一个城市(无孤立城市)
3. ✅ ASCIIMapParser提取的道路格子与渲染器显示的道路一致
4. ✅ A*推导的相邻关系与原ADJACENCY_MAP对比，差异需人工确认
5. ✅ 新增城市/道路时，只需修改map数据文件，无需改代码
