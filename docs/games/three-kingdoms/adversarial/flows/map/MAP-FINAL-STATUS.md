## 3. 关键架构决策

### 3.1 ADJACENCY_MAP自动推导

**决策**: 从地图数据自动推导城市相邻关系，替代手写配置

**实现方式**:
1. 解析world-map.txt获取100×60地图网格
2. 构建WalkabilityGrid标记可行走格子
3. 从城市位置沿道路做BFS，遇到的第一个其他城市即为相邻城市
4. 推导结果缓存为adjacency graph

**优势**:
- 新增城市/道路时只需修改map数据文件，无需改代码
- 相邻关系与地图数据完全一致
- 支持动态地图编辑

### 3.2 坐标对齐

**决策**: 城市坐标与LANDMARK_POSITIONS配置一致

**实现方式**:
1. ASCIIMapParser提取城市坐标
2. 与LANDMARK_POSITIONS配置交叉验证
3. 不一致时记录日志并人工确认

**优势**:
- 确保地图渲染与游戏逻辑坐标一致
- 避免点击偏移、行军路径错误等问题

### 3.3 A*寻路算法

**决策**: 基于WalkabilityGrid的100×60网格实现A*寻路

**实现方式**:
1. 启发函数使用曼哈顿距离(4方向移动)
2. 路径输出为GridPosition[]序列(沿道路格子的逐格坐标)
3. 不可达城市返回空路径

**性能指标**:
- 100×60网格上单次寻路<5ms
- 支持实时行军路线计算

### 3.4 行军精灵系统

**决策**: 精灵沿A*路径逐格移动，支持行走动画

**实现方式**:
1. 精灵数量反映兵力(>1000:5个, 500~1000:3个, <500:1个)
2. 精灵颜色反映阵营(魏蓝/蜀红/吴绿)
3. 行走动画8fps，道路上移动速度×1.5，山地×0.5

**优势**:
- 视觉效果直观
- 支持多支军队同时行军
- 全程可见(派遣→行军→攻城→撤退)
