## 一、医馆简化质疑

### CH-F066 [P1] 医馆被动加成是否足以让医馆"有感"

**质疑对象**: BLD-F13-01 医馆被动加成子流程

**问题描述**: R3的核心目标之一是让医馆从"僵尸实体"变为"有感实体"。R4将医馆简化为被动加成(等级×1.5%)，但被动加成本质上是"看不见的数字增长"——玩家升级医馆后，产出数字变大了一点，但没有主动交互、没有视觉反馈、没有操作节奏。

**游戏步骤对比**:

```
玩家A(R4医馆体验):
1. 打开游戏
2. 看到医馆Lv10, 产出详情显示"医馆加成: +15%"
3. 升级医馆到Lv11
4. 产出详情变为"医馆加成: +16.5%"
5. 没有任何其他变化
→ 体验: 无感。+1.5%的差异淹没在8个乘法因子中，玩家感知不到。

玩家B(R4医馆体验——治疗操作):
1. 打开游戏
2. 点击医馆，看到治疗按钮
3. 点击治疗，恢复200兵力，获得10分钟Buff
4. 等待10分钟冷却
5. 再次治疗
→ 体验: 每次登录有2~3次治疗操作，但每次恢复200兵 vs 日产27,648兵(0.7%)，操作价值极低。
```

**核心矛盾**: R3说医馆是"僵尸实体"，R4的修复方案是"让它变成一个被动百分比"——这不是让医馆有感，而是让医馆变成一个"看不见的数字"。

**质疑**: 医馆的被动加成(1.5%/级)是否足以让玩家主动升级医馆？与主城加成(2%/级)相比，医馆更弱，且没有任何主动交互——玩家为什么要优先升级医馆而不是主城？

**建议**: 医馆被动加成应有可见的视觉反馈(如建筑周围的光环等级随加成变化)，或者治疗操作的价值应大幅提升(恢复量从0.7%日产提升到5%~10%日产)。

### CH-F067 [P2] 医馆加成与主城加成的比值设计

**质疑对象**: BLD-F13-01 数值表

**问题描述**: 医馆加成1.5%/级 vs 主城加成2%/级，医馆始终弱于主城。数值验证表(5.2节)显示医馆/主城比值在0.75~0.94之间。

**质疑**: 这个比值是否故意设计？如果是，理由是什么？医馆需要额外投入资源升级，但回报始终低于主城——玩家的理性选择是永远优先升级主城，医馆沦为"主城升满后的次选"。

**数值验证**:
| 医馆等级 | 医馆加成 | 主城加成 | 差距 |
|:-------:|:-------:|:------:|:----:|
| Lv5 | +7.5% | +8% | -0.5% |
| Lv10 | +15% | +18% | -3% |
| Lv20 | +30% | +38% | -8% |

差距随等级扩大，医馆在高等级时价值更低。

## 二、存储容量质疑

### CH-F068 [P1] Lv1建筑30分钟缓冲是否足够

**质疑对象**: BLD-F10-03 库存容量公式

**问题描述**: R4新公式确保Lv1建筑缓冲30分钟，Lv25缓冲2小时。但Lv1是新玩家的起点——新玩家的前30分钟体验至关重要。

**游戏步骤推演**:
```
新玩家前30分钟:
1. 进入游戏，建造农田(Lv1)
2. 农田开始产出粮草(0.8/秒)
3. 完成新手引导(约10分钟)
4. 农田库存: 0.8×600 = 480/1440(33%)
5. 继续游戏，建造市集(Lv1)
6. 20分钟后，农田库存: 0.8×1200 = 960/1440(67%)
7. 30分钟后，农田库存: 1440/1440 = 100% → 溢出降速
8. 新玩家可能还不知道"收取"操作
→ 体验: 新玩家在完成新手引导前就遇到溢出降速，产出减半。
```

**质疑**: Lv1的30分钟缓冲是否考虑了新玩家的学习曲线？新玩家需要时间理解"建筑产出→库存→收取"的循环，30分钟可能不够。

**建议**: Lv1~5的缓冲时间应更长(如45分钟~1小时)，或新手引导阶段禁用溢出降速。

### CH-F069 [P1] 溢出降速50%的边界体验

**质疑对象**: BLD-F10-03 溢出降速机制

**问题描述**: 溢出时降速50%而非停止产出，这是R4的重要改进。但降速50%的实际体验如何？

**游戏步骤推演**:
```
玩家离线2小时(Lv10农田):
1. 农田产出8.0/秒，缓冲时间64分钟
2. 离线64分钟后，库存满(30,600)
3. 剩余56分钟，产出降速50%(4.0/秒)
4. 降速期间产出: 4.0×3360 = 13,440
5. 总产出: 30,600 + 13,440 = 44,040
6. 正常产出(无溢出): 8.0×7200 = 57,600
7. 实际损失: (57,600-44,040)/57,600 = 23.5%

玩家离线8小时(Lv10农田):
1. 离线64分钟后库存满
2. 剩余416分钟降速产出: 4.0×24960 = 99,840
3. 总产出: 30,600 + 99,840 = 130,440
4. 正常产出: 8.0×28800 = 230,400
5. 实际损失: 43.4%
```

**质疑**: 降速50%在长时间离线(8小时睡眠)场景下，损失仍然高达43%。虽然比R3的"停止产出"(100%损失)好很多，但对于"睡觉"这个高频场景，43%的损失是否会让玩家感到"不付费就亏"？

**数值验证表**:
| 离线时间 | Lv5(缓冲45min) | Lv10(缓冲64min) | Lv20(缓冲101min) |
|:-------:|:--------------:|:---------------:|:----------------:|
| 1小时 | 损失8.3% | 损失0% | 损失0% |
| 2小时 | 损失33.3% | 损失23.5% | 损失0% |
| 4小时 | 损失58.3% | 损失47.0% | 损失29.7% |
| 8小时 | 损失75.0% | 损失43.4% | 损失46.5% |

**问题**: Lv20在8小时离线时损失46.5%，与Lv10的43.4%几乎相同——高等级建筑的更长缓冲时间在长时间离线场景下没有显著优势。

### CH-F070 [P2] 一键收取与溢出降速的交互

**质疑对象**: BLD-F10 + BLD-F10-03 交互

**问题描述**: 一键收取清空所有建筑库存。如果玩家在溢出降速状态时收取，会发生什么？

**游戏步骤推演**:
```
1. 农田处于溢出降速状态(产出4.0/秒)
2. 玩家点击一键收取
3. 系统清空农田库存
4. 库存降至上限以下 → 产出立即恢复100%(8.0/秒)
5. 但玩家收取后，库存从0开始累积
6. 如果玩家收取后立即离线，64分钟后再次溢出
```

**质疑**: 收取操作从"降速状态"恢复到"正常状态"，但这个恢复是瞬时的——玩家可能不知道自己从降速中恢复了。UI上是否显示"产出已恢复"的提示？

## 三、VIP机制质疑

### CH-F071 [P1] VIP加成是否足以成为付费动力

**质疑对象**: BLD-F10-01 VIP收取加成

**问题描述**: R4取消了VIP自动收取，改为VIP3++10%/VIP6++20%收取加成。但这个加成是否足以让玩家付费？

**游戏步骤推演**:
```
非VIP玩家日常:
1. 打开游戏
2. 点击一键收取 → 获得5000资源
3. 关闭游戏
4. 2小时后重复
5. 每日收取6次 → 30,000资源

VIP6玩家日常:
1. 打开游戏
2. 点击一键收取 → 获得6000资源(+20%)
3. 关闭游戏
4. 2小时后重复
5. 每日收取6次 → 36,000资源
6. 每日多获得6000资源(20%)
```

**质疑**: 20%的差距在游戏中后期是否显著？当玩家每日产出达到百万级别时，20%意味着多20万资源——但这20万是否能转化为实际优势(如更快升级、更强兵力)？

**对比其他游戏**:
- 典型SLG的VIP优势: 自动建造、额外队列、加速折扣——这些是"功能解锁"
- R4的VIP优势: +20%收取——这只是"数值加成"

**问题**: 纯数值加成的VIP设计是否足够有吸引力？玩家付费后，体验没有任何变化(仍然是手动收取)，只是数字变大了。

### CH-F072 [P0] VIP玩家取消自动收取后的体验落差

**质疑对象**: BLD-F10 VIP收取流程

**问题描述**: R3的VIP3+有自动收取(80%阈值)，R4取消了自动收取改为手动+加成。这意味着从R3到R4，VIP玩家的核心体验被"降级"了——他们失去了自动收取的便利，只获得了+10%/+20%的数值补偿。

**游戏步骤对比**:
```
R3 VIP6玩家:
1. 打开游戏 → 资源已自动收取(80%阈值)
2. 不需要任何操作
3. 离开游戏
→ 体验: 零操作，永不丢失产出

R4 VIP6玩家:
1. 打开游戏
2. 必须点击一键收取
3. 获得+20%加成
4. 离开游戏
5. 2小时后必须再次登录收取
→ 体验: 每2小时需要手动操作，否则溢出降速损失43%
```

**核心问题**: R4将VIP从"付费跳过操作"改为"付费获得加成但必须操作"——这是设计理念的正确方向，但对于已经习惯自动收取的VIP玩家，这是体验降级。

**质疑**: 如何处理R3→R4的过渡？现有VIP玩家是否会因为失去自动收取而退款/流失？是否需要补偿机制(如过渡期保留自动收取，或赠送特化重置符)？

**建议**: 考虑VIP6+保留"半自动收取"(如库存达到80%时自动收取基础部分，VIP加成部分需手动领取)，作为过渡方案。

## 四、特化系统质疑

### CH-F073 [P1] 8座建筑×2选择的决策质量

**质疑对象**: BLD-F16-01 各建筑特化方向

**问题描述**: 8座建筑各有2个特化方向，但这些选择是否都是"有意义的决策"？

**逐建筑分析**:

| 建筑 | 特化A | 特化B | 决策质量 |
|:----:|-------|-------|:-------:|
| 主城 | 仁政:产出+15%,升级费+20% | 霸业:队列+1,产出+5% | **高** — 短期vs长期权衡 |
| 农田 | 丰产:粮草+30%,铜钱-50% | 商农:粮草+10%,铜钱+20% | **高** — 单资源vs双资源 |
| 市集 | 通商:铜钱+30%,粮草-50% | 农商:铜钱+10%,粮草+20% | **高** — 与农田对称 |
| 兵营 | 精兵:兵力+25%,训练费+30% | 屯田:兵力+10%,粮草消耗-30% | **中** — 精兵几乎必选(兵力是核心) |
| 铁匠铺 | 锻造:材料+30%,品质+1级 | 修补:材料+10%,耐久+50% | **低** — 品质+1级远强于耐久+50% |
| 书院 | 博学:科技点+25%,研究速度+20% | 专精:科技点+10%,特定科技+50% | **中** — 取决于"特定科技"定义 |
| 医馆 | 妙手:治疗量+50%,冷却-30% | 军医:被动加成+50%,治疗量-30% | **低** — 军医几乎必选(被动加成是全局) |
| 城墙 | 坚壁:城防+50%,防御+10% | 利器:城防+10%,反击伤害+30% | **中** — 防御vs反击 |

**核心问题**:
1. **铁匠铺**: "装备品质+1级"的效果远超"耐久+50%"——品质影响装备属性，耐久只是消耗品。玩家几乎必然选择"锻造"。
2. **医馆**: "被动加成+50%"意味着医馆从1.5%/级变为2.25%/级，这是全局效果；而"治疗量+50%"只是单次操作的数值提升。理性玩家必然选择"军医"。
3. **兵营**: "兵力+25%"是核心资源产出，"粮草消耗-30%"只是节省消耗——兵力产出的价值远高于消耗节省。

**质疑**: 8个建筑中，至少3个(铁匠铺/医馆/兵营)存在"假选择"——一个方向明显优于另一个。这是否违背了"每个选择都有取舍"的设计原则？

### CH-F074 [P0] 特化重置机制的设计缺陷

**质疑对象**: BLD-F16-03 特化重置子流程

**问题描述**: 特化重置需要"特化重置符"道具，获取途径为: 活动奖励(限时)、元宝商店(高价值)、成就奖励(一次性)。

**游戏步骤推演**:
```
玩家A的特化后悔:
1. 农田Lv10，选择"丰产"(粮草+30%,铜钱-50%)
2. 游戏进入中期，铜钱严重短缺
3. 想重置为"商农"(粮草+10%,铜钱+20%)
4. 检查背包: 没有特化重置符
5. 查看获取途径:
   - 活动奖励: 当前没有活动
   - 元宝商店: 需要500元宝(假设)
   - 成就奖励: 已经用完了
6. 无法重置，被迫继续使用"丰产"
→ 体验: 被困在错误选择中，无法纠正。
```

**核心问题**:
1. **永久惩罚**: 特化选择"永久生效"，但重置道具获取困难——玩家在Lv10做的选择可能在Lv20时发现是错误的，但无法纠正。
2. **信息不对称**: Lv10时玩家可能不了解游戏后期的资源需求，做出错误选择。重置道具的稀缺性放大了这个错误的惩罚。
3. **付费壁垒**: 元宝商店是获取重置符的稳定途径，这意味着"纠正错误选择"需要付费——这是P2W的变体。

**质疑**: 特化重置符的稀有度是多少？如果是"每月1个"(活动)或"500元宝"(约50元人民币)，那么玩家在8座建筑上的特化选择就变成了"一次性不可逆决策"——这对于一个手游来说是否过于惩罚？

**建议**:
1. 特化重置符应有稳定的免费获取途径(如每周任务奖励)
2. 首次重置应免费(给玩家"试错"机会)
3. 或者: 特化选择改为"可随时切换，但有24小时冷却"

### CH-F075 [P1] 特化与协同系统的交互缺失

**质疑对象**: BLD-F16 + BLD-F14 交互

**问题描述**: 特化系统和协同系统是R3/R4新增的两个核心机制，但R4流程中没有描述它们的交互。

**游戏步骤推演**:
```
玩家的困惑:
1. 农田选择"丰产"特化(粮草+30%)
2. 市集选择"通商"特化(铜钱+30%)
3. 同时激活"丰收市集"协同(农田Lv10+市集Lv10 → 铜钱+15%)
4. 问题: 丰产特化取消了农田的铜钱产出(-50%)，但协同要求农田Lv10+市集Lv10
5. 协同的+15%铜钱加成是基于哪个基数？
   - 如果基于农田的铜钱产出: 丰产特化后农田铜钱为0，协同无效？
   - 如果基于市集的铜钱产出: 协同正常生效
```

**质疑**: 特化效果(如"铜钱产出-50%")是否会与协同效果(如"铜钱+15%")产生冲突？R4流程中没有明确说明特化在产出公式中的位置与协同的关系。

**产出公式确认**:
```
R4公式: 最终产出 = 基础产出 × 主城加成 × 科技加成 × 武将加成 × 医馆加成 × 协同加成 × 特化加成
```

**问题**: 如果农田选择"丰产"(铜钱-50%)，那么:
- 农田的铜钱基础产出 = 基础 × 特化(-50%) = 基础 × 0.5
- 但"丰产"描述是"铜钱产出-50%"，不是"取消铜钱产出"
- 协同的+15%是在特化之后叠加还是之前？

这个交互在R4流程中没有明确说明。

### CH-F076 [P2] 特化选择时机(Lv10)的信息充分性

**质疑对象**: BLD-F16-01 触发条件

**问题描述**: 特化在建筑首次达到Lv10时触发，玩家需要立即做出选择。但Lv10时玩家可能还没有足够的信息来做出正确决策。

**游戏步骤推演**:
```
新玩家的特化决策:
1. 游戏第3天，农田升到Lv10
2. 弹出特化选择面板
3. 面板显示:
   - 丰产: 粮草产出+30%, 铜钱产出-50%
   - 商农: 粮草产出+10%, 铜钱产出+20%
4. 玩家此时还不知道:
   - 后期铜钱的需求量有多大
   - 其他建筑的特化选择
   - 协同系统的影响
5. 玩家凭直觉选择"丰产"(粮草更多)
6. 10天后发现铜钱严重短缺
→ 体验: 在信息不充分时被迫做出永久决策。
```

**质疑**: 是否应该将特化选择推迟到更晚的等级(如Lv15)，让玩家有更多时间了解游戏？或者在选择面板中显示"推荐"和"后期价值评估"？

## 五、数值验证质疑

### CH-F077 [P1] 产出公式的因子膨胀

**质疑对象**: R4产出公式

**问题描述**: R4的产出公式有8个乘法因子:
```
最终产出 = 基础产出 × 主城加成 × 科技加成 × 武将加成 × 医馆加成 × 协同加成 × 特化加成 × 治疗Buff
```

**数值验证**:
| 因子 | 典型值(Lv15玩家) |
|:----:|:---------------:|
| 基础产出 | 16.0 粮草/秒 |
| 主城加成 | ×1.28 (+28%) |
| 科技加成 | ×1.10 (+10%) |
| 武将加成 | ×1.10 (+10%) |
| 医馆加成 | ×1.225 (+22.5%) |
| 协同加成 | ×1.15 (+15%) |
| 特化加成 | ×1.30 (+30%) |
| 治疗Buff | ×1.10 (+10%) |
| **最终产出** | **16.0 × 1.28 × 1.10 × 1.10 × 1.225 × 1.15 × 1.30 × 1.10 = 56.2 粮草/秒** |

**总加成**: 56.2 / 16.0 = **3.51倍** (即+251%)

**质疑**: 8个乘法因子叠加后，基础产出被放大3.5倍。这意味着:
1. **新手vs老手差距巨大**: 新手(只有基础产出) vs 老手(3.5倍产出)的差距是251%——这是否会导致新手追赶困难？
2. **数值膨胀**: 如果未来新增更多因子(如武将技能、装备加成)，产出会进一步膨胀。
3. **平衡困难**: 8个因子中任何一个的调整都会影响最终产出——调一个因子需要验证所有组合。

**建议**: 考虑将部分乘法因子改为加法叠加，限制总加成的膨胀。例如:
```
最终产出 = 基础产出 × (1 + 主城% + 科技% + 武将%) × 医馆加成 × 协同加成 × 特化加成 × 治疗Buff
```

### CH-F078 [P2] 升级ROI在高等级段的合理性

**质疑对象**: 5.5节 升级ROI验证

**问题描述**: 数值验证表显示:
- 农田Lv15→20: ROI 0.0037/s/千，回本时间74小时
- 市集Lv5→10: ROI 0.061/s/千，回本时间11.4小时

**质疑**: Lv15→20需要74小时(3天)回本——在这3天内，玩家的产出停滞(资源用于升级)。对于一个手游来说，3天的"投资期"是否过长？

**对比**: 典型手游的升级回本时间:
- 休闲游戏: 几分钟~1小时
- 中度游戏: 1~4小时
- 重度SLG: 4~12小时

R4的74小时回本时间远超典型SLG，可能导致玩家在高等级段"无所事事"。

2. 后续重置通过稳定的免费途径获取(如每周任务)

**影响范围**: BLD-F16-03

### 7.2 R4新暴露的方法论问题

**问题1: "修复旧问题"vs"引入新问题"的平衡**
- R4成功修复了3个P0(士气/VIP/存储)，但引入了新的P0(特化重置)和多个P1
- 根因: 新增系统(特化)的设计深度不足，没有充分推演游戏步骤
- 改进: 新增系统必须经过"游戏步骤推演"——从Lv1到终局的完整体验路径

**问题2: "永久决策"的惩罚设计**
- 特化选择是永久的，重置道具稀缺——这是"惩罚犯错"的设计
- 根因: 设计时假设玩家会做出"正确"选择，但玩家在Lv10时信息不充分
- 改进: 永久决策必须有"低成本纠正"机制，或推迟决策时机

**问题3: 乘法因子膨胀的风险**
- R4的产出公式有8个乘法因子，总加成可达3.5倍
- 根因: 每个新系统都独立设计加成，没有考虑因子叠加后的总效果
- 改进: 产出公式必须有"因子上限"或"递减收益"机制

### 7.3 R4 Challenger视角

**R4-Challenger做得好的地方**:
1. 聚焦游戏步骤: 每个质疑都从"玩家实际操作"角度出发，而非纯数值分析
2. P0质疑精准: CH-F074(特化重置)和CH-F072(VIP体验落差)命中核心设计缺陷
3. 新增PRD修订建议: 3个新PRD修订(R018/R019/R020)有明确的问题来源和修订方案

**R4-Challenger可改进的地方**:
1. 对特化系统的分析可以更深入——如8座建筑的16个特化方向的平衡性逐一分析
2. 数值验证可以更系统——如8个乘法因子的所有组合(2^8=256种)的产出范围
