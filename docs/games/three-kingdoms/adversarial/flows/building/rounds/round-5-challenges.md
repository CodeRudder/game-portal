# 建筑系统 Round 5 — Challenger 质疑报告

> Challenger: FlowChallenger v3.0 | 时间: 2026-05-02
> 基于: R5-Builder(round-5-flows.md) + R4-Verdict(round-4-verdict.md)
> 用户反馈: "不需要补充细节，先补充游戏步骤" — 本轮聚焦游戏步骤中的体验问题

---

## 总览

R5-Builder提交了PRD 17/17全覆盖、GAP归零的建筑系统流程。表面上看收敛指标很漂亮，但Challenger的核心质疑是: **这些流程步骤在实际游玩中是否构成"好玩的游戏"？**

本轮质疑不纠缠数值细节，而是逐一流程推演玩家操作步骤，找出"看起来完整但实际玩起来有问题"的环节。

**质疑统计**: P0 x 2, P1 x 6, P2 x 4, 共12项

## 一、医馆价值质疑

### CH-F079 [P0] 医馆被动加成"有感"了吗？——2%/级仍然看不见

**R5方案**: 加成从1.5%/级提升至2%/级，新增浮动"+20%产出"指示器，每日治愈报告。

**游戏步骤推演**:

```
步骤1: 玩家登录游戏
步骤2: 看到医馆建筑上浮动"+20%产出"绿色文字
步骤3: 玩家内心OS: "+20%是什么意思？我该关心吗？"
步骤4: 玩家点击医馆，看到"升级后粮草产出 8.0→8.16 粮草/秒"
步骤5: 玩家内心OS: "多了0.16？这也太少了吧，我还是先升农田吧"
步骤6: 玩家收到治愈报告: "全局产出加成+20%，额外产出粮草+3,840"
步骤7: 玩家内心OS: "+3,840是多少？我一天产多少来着？"
```

**核心问题**: 浮动指示器和治愈报告是"信息展示"，不是"价值感知"。

- **"+20%产出"是相对值** — 玩家不知道20%对应多少绝对收益。看到"+20%"和看到"+0.16粮草/秒"的感受完全不同。
- **治愈报告每日1次** — 玩家看到"额外产出3,840"时，没有对比基准(不升级医馆会少多少？)。
- **8个乘法因子中的1个** — 即使有高亮，玩家在产出详情中看到7行加成，医馆只是其中一行绿色文字，淹没在信息洪流中。

**游戏步骤缺陷**: R5增加了"展示信息"的步骤，但没有增加"让玩家感受到损失"的步骤。玩家感知价值的方式是"失去比得到更强烈"。

**建议方向**:
- 升级医馆前后，所有建筑的产出数字做一次"对比动画"(旧值→新值跳动)
- 医馆等级为0时，产出详情中显示"医馆未建造: 全局产出-20%"(用红色负值让玩家感知缺失)
- 治愈报告改为"若医馆降1级，你今天会少产出XXX"(损失框架)

### CH-F080 [P1] 治疗5%日产——操作步骤是否值得执行？

**R5方案**: 治疗量从固定值改为兵营日产5%，冷却10分钟。

**游戏步骤推演(战斗后)**:

```
步骤1: 玩家打完一场战役，损失5,000兵力，产生5,000伤兵
步骤2: 兵营Lv10日产27,648，治疗量=27,648×5%=1,382
步骤3: 玩家需要治疗5,000÷1,382≈4次才能恢复全部伤兵
步骤4: 每次冷却10分钟，4次=40分钟
步骤5: 每次消耗粮草，4次=粮草消耗×4
步骤6: 玩家内心OS: "我要花40分钟点4次才能恢复？不如等兵营自然产出"
步骤7: 兵营自然产出恢复5,000兵需要: 5,000÷(27,648÷86,400)≈17,300秒≈4.8小时
步骤8: 玩家内心OS: "4.8小时vs40分钟，还是治疗吧……但要点4次好烦"
```

**核心问题**: 5%日产的治疗量让"有效治疗"需要多次操作，但每次操作的单次体验不满足。

- **单次治疗1,382兵** — 对比5,000伤兵，单次只恢复27.6%，玩家感受是"才治了这么点？"
- **需要4次才能恢复** — 操作重复性高，每次间隔10分钟，打断游戏节奏
- **Buff只有10分钟** — 如果花40分钟治疗4次，前几次的Buff早已过期

**建议方向**:
- 单次治疗量提升至10%~15%日产(减少操作次数至2~3次)
- 或: 治疗量随医馆等级递增(5%+0.5%×医馆等级)，让升级医馆直接提升治疗体验
- 或: 一次治疗恢复全部伤兵，但冷却时间更长(30分钟)

### CH-F081 [P1] "军医"vs"妙手"——真的是不同玩法风格的选择吗？

**R5方案**:
- 妙手: 治疗量+100%，冷却5分钟，Buff效果+15%
- 军医: 被动+3%/级，治疗量-50%，治愈报告额外奖励

**游戏步骤推演(选军医的玩家)**:

```
步骤1: Lv10选择军医特化
步骤2: 被动加成从2%/级变为3%/级，Lv20从+40%变为+60%
步骤3: 玩家登录，看到"+60%产出"浮动指示器
步骤4: 玩家内心OS: "不错，比选妙手多了20%"
步骤5: 发生战斗，产生伤兵，点击治疗
步骤6: 治疗量减半: 1,382×50%=691兵
步骤7: 玩家内心OS: "才恢复691兵？这也太少了吧"
步骤8: 看到治愈报告有额外奖励
步骤9: 玩家内心OS: "被动加成高+治疗拉胯+报告有奖励，算了就这样吧"
```

**游戏步骤推演(选妙手的玩家)**:

```
步骤1: Lv10选择妙手特化
步骤2: 被动加成不变(2%/级)
步骤3: 治疗量翻倍: 1,382×200%=2,764兵，冷却5分钟
步骤4: 发生战斗，产生5,000伤兵
步骤5: 治疗1次恢复2,764兵，等5分钟
步骤6: 治疗2次恢复全部伤兵，总耗时10分钟
步骤7: 玩家内心OS: "两次就治好了，爽！"
步骤8: 获得+15%攻防/产出Buff
步骤9: 玩家内心OS: "治疗又快又强还有Buff，妙手太值了"
```

**核心问题**: 两个方向的"操作体验"差异太大。

- **妙手**的操作体验是"快速治愈+强力Buff"——每次治疗都有强烈正反馈
- **军医**的操作体验是"被动加成高一点+治疗拉胯"——日常无感，治疗时负面感受
- **"适合离线玩家"vs"适合在线玩家"**这个设计意图是好的，但军医的"治愈报告额外奖励"没有具体描述是什么奖励——这是一个模糊的卖点

**关键缺陷**: 军医方向的-50%治疗量是一个"惩罚性设计"。玩家选军医不是因为"我想要更高被动加成"，而是"我不得不接受治疗变弱"。这会让选择军医的玩家每次治疗时都感到后悔。

**建议方向**:
- 军医不应降低治疗量，而是降低治疗冷却(如15分钟→20分钟)——减少操作频率而非操作效果
- 军医的"治愈报告额外奖励"需要明确: 每日额外获得多少资源？这个数字需要让玩家觉得"值得放弃治疗翻倍"
- 或: 军医的治疗变为"群体治疗"——每次治疗恢复所有建筑驻军的伤兵(而非单个建筑)

## 二、特化平衡质疑

### CH-F082 [P1] 铁匠铺"锻造30%概率"——随机性引入决策不确定性

**R5方案**: 锻造方向的"品质+1级"从100%触发改为30%概率触发。

**游戏步骤推演**:

```
步骤1: 玩家Lv10选择铁匠铺特化，看到"锻造: 材料产出+20%，装备品质+1级(30%概率)"
步骤2: 玩家内心OS: "30%概率？那我有70%概率得不到品质+1？"
步骤3: 玩家打铁10次，期望3次品质+1，实际可能1次都没有
步骤4: 玩家内心OS: "这30%概率也太低了吧，我选修补多好，耐久+100%是确定的"
步骤5: 另一个玩家选了修补，装备耐久翻倍+维修-50%，每次战斗都很稳定
步骤6: 选锻造的玩家看到选修补的朋友装备持久，内心后悔
```

**核心问题**: 30%概率的"品质+1"是一个期望值设计，但玩家感受的是"单次体验"。

- **期望值**: 10次打铁平均3次品质+1，看起来合理
- **单次体验**: 每次打铁有70%概率"白选了锻造特化"，负面感受强烈
- **对比修补**: 耐久+100%和维修-50%是100%生效的确定性收益

**设计原则**: 特化选择是永久决策，不应该引入高方差的随机效果。玩家选择特化是为了"稳定的玩法风格"，不是为了"赌运气"。

**建议方向**:
- 锻造的品质+1改为"每5次打铁必定1次品质+1"(保底机制)
- 或: 品质+1改为100%触发，但效果从+1级降为+0.5级(半级，数值上等效但更稳定)
- 或: 品质+1改为"品质上限+1级"(100%触发，但只提升最高品质，不保证每次出)

### CH-F083 [P1] 兵营"屯田"——铜钱+10%是跨建筑加成还是兵营本身加成？

**R5方案**: 屯田方向 = 兵力产出+5%，粮草消耗-50%，铜钱产出+10%。

**游戏步骤推演**:

```
步骤1: 玩家Lv10选择兵营"屯田"特化
步骤2: 玩家内心OS: "铜钱产出+10%，不错，我缺铜钱"
步骤3: 玩家查看兵营产出详情: 兵力产出+5%，粮草消耗-50%
步骤4: 玩家内心OS: "等等，铜钱+10%在哪里？兵营不是产兵力的吗？"
步骤5: 玩家查看市集产出: 铜钱产出没有变化
步骤6: 玩家内心OS: "所以铜钱+10%到底是加在哪里的？"
```

**核心问题**: 兵营的功能是"产出兵力"，给兵营特化加"铜钱产出+10%"违反了建筑功能直觉。

- **兵营产兵力，市集产铜钱** — 这是玩家的基本认知模型
- **兵营特化加铜钱** — 打破认知模型，玩家会困惑"铜钱是从哪里来的？"
- **跨建筑加成的归属** — 如果铜钱+10%是全局加成，那它和医馆被动加成有什么区别？如果是兵营自身的铜钱产出，那兵营什么时候产铜钱了？

**建议方向**:
- 屯田方向改为: 兵力产出+5%，粮草消耗-50%，**训练速度+20%** — 保持兵营功能一致性
- 或: 如果一定要加铜钱，改为"屯田: 每次训练士兵额外获得铜钱(训练费的10%返还)" — 有明确的因果关系
- 或: 改为"屯田: 兵力产出+5%，粮草消耗-50%，**兵力上限+20%**" — 强化兵营的核心功能

### CH-F084 [P2] TE-22特化推荐——推荐标识会让玩家停止思考吗？

**R5方案**: Lv10选择特化时，系统根据玩家状态显示"推荐"金色角标。

**游戏步骤推演**:

```
步骤1: 玩家Lv10，弹出特化选择面板
步骤2: 看到左侧卡片有金色"推荐"角标
步骤3: 玩家内心OS: "系统推荐这个，那就选这个吧"
步骤4: 玩家直接点击推荐方向，跳过阅读两个方向的详细描述
步骤5: 二次确认弹出，玩家直接点"确认"
步骤6: 特化选择完成，玩家从未真正理解两个方向的差异
```

**核心问题**: 推荐标识可能让特化选择变成"无脑跟推荐"，削弱了决策的参与感。

- **推荐标识的权威性** — 玩家倾向于信任系统推荐，尤其是新手
- **跳过思考** — 有了推荐，玩家不再比较两个方向的优劣
- **后悔时的归因** — 如果推荐方向后来不好用，玩家会怪"系统推荐错了"而非反思自己的选择

**这不是P0**因为推荐标识确实帮助了不知道怎么选的新手。但需要设计推荐标识的"弱化"机制:

**建议方向**:
- 推荐标识只在玩家"犹豫超过10秒"后才显示(先让玩家自己看)
- 或: 推荐标识显示为"适合你的玩法: [方向A]"(解释推荐原因，而非直接标"推荐")
- 或: 两个方向都显示"适合人群"描述，让玩家自行匹配

## 三、跨系统流程质疑

### CH-F085 [P1] BLD-F17跨系统流程——5条连接是否覆盖了"玩家视角的完整路径"？

**R5方案**: BLD-F17-01~05覆盖了BLD↔RES/TEC/HER/CPN/EVT五条连接。

**游戏步骤推演(一个完整的"建筑→战斗"路径)**:

```
步骤1: 农田产出粮草 → 资源池
步骤2: 资源池的粮草用于兵营训练兵力
步骤3: 兵力用于战役
步骤4: 战役产生伤兵
步骤5: 医馆治疗伤兵 ← BLD-F13-02
步骤6: 战役获得资源奖励 → 回到步骤1
```

**这个路径在BLD-F17中是否完整？**

- BLD-F17-01(BLD→RES): 覆盖了步骤1 ✓
- BLD-F17-02(BLD→TEC): 覆盖了科技加成 ✓
- BLD-F17-03(BLD→HER): 覆盖了武将派驻 ✓
- BLD-F17-04(BLD→CPN): 覆盖了城墙防御→战役 ✓
- BLD-F17-05(BLD→EVT): 覆盖了建筑事件 ✓

**缺失的路径**:

```
步骤2→步骤3: 资源→兵力→战役 这条路径中，"兵力"这个中间资源的流向不清晰
  - 兵力产自兵营，消耗于战役
  - 兵力是"建筑产出"还是"独立资源"？
  - BLD-F17-01只写了"资源池"，没有区分粮草/铜钱/兵力三种资源的不同流向
```

**核心问题**: BLD-F17把"资源"当做一个统一概念，但实际上建筑系统涉及三种资源(粮草、铜钱、兵力)，它们的产出-消耗循环不同:

| 资源 | 产出建筑 | 主要消耗 | 特殊性 |
|------|---------|---------|--------|
| 粮草 | 农田 | 兵力训练、建筑升级、治疗 | 产出量最大，消耗最多 |
| 铜钱 | 市集 | 建筑升级、科技研究 | 产出量中等，消耗稳定 |
| 兵力 | 兵营 | 战役(消耗/损失) | 产出后直接用于战斗，不在"资源池"中久留 |

**建议**: BLD-F17-01需要区分三种资源的独立流向图，而非笼统的"产出→资源池→消耗"。

### CH-F086 [P0] 建筑事件30%登录触发——频率和时机设计有严重缺陷

**R5方案**: 每次登录30%概率触发1个建筑事件，事件针对等级最高的建筑。

**游戏步骤推演(高频玩家)**:

```
步骤1: 玩家早上8点登录 → 30%概率触发事件 → 假设触发了
步骤2: 玩家处理事件(选择A或B)
步骤3: 玩家中午12点再次登录 → 30%概率触发事件 → 假设又触发了
步骤4: 玩家晚上8点再次登录 → 30%概率触发事件 → 假设又触发了
步骤5: 一天登录3次，期望触发0.9次事件
步骤6: 如果每次都触发，玩家内心OS: "又要处理事件？我就想收个资源而已"
```

**游戏步骤推演(低频玩家)**:

```
步骤1: 玩家每天只登录1次 → 30%概率触发事件
步骤2: 连续3天没触发任何事件
步骤3: 玩家内心OS: "事件系统是假的吧？我怎么从来没遇到过？"
步骤4: 第4天终于触发了，但玩家已经忘了事件系统是什么
```

**核心问题**:

1. **30%每次登录 vs 每日1次** — 没有每日上限，高频玩家可能一天遇到多次事件，低频玩家可能连续几天遇不到。这是"运气决定体验"。
2. **事件针对"等级最高的建筑"** — 玩家最高等级建筑始终是同一个(如农田)，所以事件总是围绕农田发生。7类建筑×3种事件=21种事件，但玩家只会看到农田的3种。
3. **登录时触发** — 玩家登录的目的是"收取资源/升级建筑"，突然弹出事件面板打断了核心操作流程。
4. **事件面板是"全屏遮罩"吗？** — 如果是，玩家必须先处理事件才能做其他事，这是强制中断。

**建议方向**:
- **频率**: 改为"每日首次登录100%触发1次"，而非"每次登录30%"。确定性>随机性。
- **目标建筑**: 改为"随机选择一座Lv5+建筑"，而非"等级最高的建筑"。增加多样性。
- **时机**: 事件不弹出全屏遮罩，而是显示在建筑卡片上的"事件气泡"，玩家点击才查看详情。非强制中断。
- **冷却**: 同一建筑24小时内最多触发1次事件。

### CH-F087 [P2] 事件池平衡——农田事件和书院事件的"赌注"不对等

**R5方案**: 每类建筑3种事件，各有选择A和选择B。

**游戏步骤对比**:

```
农田事件"丰收之年":
  选择A: 投入500粮草 → 获得2000粮草(净赚1500)
  选择B: 不投入 → 获得500粮草(净赚500)
  → 选择A的回报是选择B的3倍

农田事件"虫灾来袭":
  选择A: 投入1000铜钱 → 正常产出
  选择B: 不处理 → 粮草产出-50%持续2小时
  → 选择A是"花铜钱保平安"，选择B是"承受惩罚"
```

**核心问题**: 事件的"赌注"差异巨大。

- **丰收之年**: 500粮草的投入对任何阶段的玩家都是微不足道的，所以"选择A"永远是正确答案
- **虫灾来袭**: 1000铜钱也很少，"选择A"永远是正确答案
- **干旱预警**: 3000粮草有一定分量，但+20%产出4小时的回报远超投入

**结论**: 所有事件的选择A都是"正确答案"，选择B都是"惩罚"。这不是"选择"，是"付费免罚"。

**建议方向**:
- 每个事件的两个选择应该是"两种不同方向的收益"，而非"付费vs惩罚"
- 例: "丰收之年" — 选择A: 立即获得2000粮草; 选择B: 粮草产出+10%持续4小时(长期收益更高但需要等待)
- 例: "虫灾来袭" — 选择A: 消灭虫灾，正常产出; 选择B: 研究虫灾，获得"虫害防治"永久科技(+2%粮草产出)

## 四、产出公式质疑

### CH-F088 [P1] 7个乘法因子——新手到老手的数值差距有多大？

**R5公式**:
```
最终产出 = 基础产出 × 主城加成 × 科技加成 × 武将加成 × 医馆加成 × (协同+特化) × 治疗Buff
```

**游戏步骤推演(新手vs老手)**:

```
新手(Lv5，无特化/协同/武将/治疗Buff):
  基础产出: 3.0 粮草/秒
  × 主城加成: 1.10 (主城Lv5)
  × 科技加成: 1.00 (无科技)
  × 武将加成: 1.00 (无武将)
  × 医馆加成: 1.10 (医馆Lv5)
  × (协同+特化): 1.00 (无)
  × 治疗Buff: 1.00 (无)
  = 3.63 粮草/秒

老手(Lv20，全加成):
  基础产出: 28.0 粮草/秒
  × 主城加成: 1.40 (主城Lv20)
  × 科技加成: 1.15 (多科技)
  × 武将加成: 1.05 (Lv10武将)
  × 医馆加成: 1.40 (医馆Lv20)
  × (协同+特化): 1.60 (协同25%+特化30%+治疗10%=65%，但协同+特化上限80%，治疗Buff独立)
  × 治疗Buff: 1.10
  = 28.0 × 1.40 × 1.15 × 1.05 × 1.40 × 1.55 × 1.10 = 132.6 粮草/秒

差距: 132.6 ÷ 3.63 = 36.5倍
```

**核心问题**: 新手产出3.63，老手产出132.6，差距36.5倍。这个差距是否合理？

- **等级差距**: Lv5 vs Lv20，基础产出差距28/3=9.3倍
- **加成差距**: 加成乘数1.21 vs 5.29，差距4.4倍
- **总差距**: 9.3 × 4.4 = 40.9倍(与36.5倍基本一致)

**在SLG中36倍差距是否正常？** — 对比参考:
- 《率土之滨》新手vs老手资源差距: 约20~50倍
- 《三国志战略版》新手vs老手资源差距: 约30~60倍

**结论**: 36倍差距在SLG中属于正常范围，但问题在于**差距来源的可理解性**:

- 新手看到老手的产出是自己的36倍，会问"为什么？"
- 答案是7个乘法因子的叠加，但新手无法直观理解"每个因子贡献了多少"
- **产出详情面板**需要增加"加成贡献排名"——哪个因子对最终产出影响最大

**建议**: 在产出详情面板中，每个因子后面显示"等效产出贡献"(如"主城加成: +40% → 贡献+11.2粮草/秒")。

### CH-F089 [P2] 协同+特化同层加法叠加，+80%上限——上限是否可达？

**R5方案**: 协同加成和特化加成在同一层加法叠加，上限+80%。

**当前最大叠加计算**:

```
最高协同: +25%(丰收市集，需要农田+市集同时高等级)
最高特化: +30%(丰产/通商)
治疗Buff: +10%(军心振奋，独立层不在上限内)

协同+特化最大值: 25% + 30% = 55%
距离80%上限还有: 25%
```

**核心问题**: 当前最大叠加55%，距离上限80%还有25%的空间。

- **上限是否太宽松？** — 55%<80%，意味着当前没有任何组合能触及上限
- **未来新增协同/特化** — 如果新增1个+15%的协同，就能达到70%，接近上限
- **上限的设计意图** — 是防止"未来膨胀"还是限制"当前最优解"？

**结论**: +80%上限是合理的"安全阀"设计。当前55%的最大值说明系统还有扩展空间，不会因为新增协同/特化而立即触及上限。

**建议**: 无需修改。但应在UI中显示"当前叠加: 55%/80%"，让玩家知道还有优化空间。

## 五、可玩性综合评估

### CH-F090 [P1] R5建筑系统的"核心游戏循环"是什么？

**一个好玩的建筑系统应该有清晰的核心循环**:

```
建造 → 产出 → 收取 → 升级 → 更多产出 → ...
```

**R5建筑系统的核心循环**:

```
建造(唯一操作)
  → 产出(自动，被动)
  → 库存累积(自动)
  → 收取(每30~120分钟1次)
  → 升级(消耗资源+等待时间)
  → 特化(Lv10一次性选择)
  → 协同(被动触发)
  → 医馆加成(被动)
  → 治疗(偶尔操作)
  → 事件(登录时30%概率)
  → 回到产出
```

**问题**: 这个循环中有多少步骤是"玩家主动做的"？

| 步骤 | 玩家操作 | 频率 | 主动/被动 |
|------|---------|------|:--------:|
| 建造 | 点击建造 | 一次性 | 主动 |
| 产出 | 无 | 持续 | 被动 |
| 库存 | 无 | 持续 | 被动 |
| 收取 | 点击收取 | 每30~120min | 主动 |
| 升级 | 点击升级+等待 | 每几小时 | 主动 |
| 特化 | 选择方向 | 一次性 | 主动 |
| 协同 | 无 | 被动触发 | 被动 |
| 医馆 | 无 | 持续 | 被动 |
| 治疗 | 点击治疗+选择Buff | 每10min | 主动 |
| 事件 | 选择A/B | 登录时30% | 半主动 |

**主动操作只有**: 收取、升级、治疗、事件选择。其中:
- **收取**是"例行公事"，没有决策
- **升级**是"按顺序点"，没有决策(有最优升级路径)
- **治疗**是"选择Buff"，有决策但决策简单(产出vs战斗)
- **事件**是"选择A/B"，有决策但当前设计中A永远是最优

**结论**: R5建筑系统的主动操作太少，且主动操作的决策深度不够。玩家在建筑系统中大部分时间是"等待"和"点击收取"。

**建议方向**:
- **增加建筑管理操作**: 如"调配建筑优先级"(哪个建筑优先使用资源)、"建筑巡查"(随机获得小Buff)
- **增加收取决策**: 如"收取时机"(库存50%收取vs100%收取有不同收益)
- **增加升级决策**: 如"建筑协同升级"(同时升级农田+市集有额外奖励)

### CH-F091 [P2] R5建筑系统缺少"成就感"和"里程碑"

**游戏步骤推演(玩家玩了1周)**:

```
Day 1: 建造农田、市集、兵营 → 有新鲜感
Day 2: 升级建筑到Lv5 → 感受到成长
Day 3: 解锁协同 → 有一些惊喜
Day 4: 升级建筑到Lv7 → 重复操作
Day 5: 升级建筑到Lv9 → 升级时间变长，等待无聊
Day 6: 到达Lv10，选择特化 → 有决策感
Day 7: 继续升级Lv11 → 又回到等待
```

**核心问题**: 建筑系统的"里程碑"太少。玩家在以下节点感受到成就:
- Lv1: 建造(新鲜感)
- Lv5: 解锁协同(惊喜)
- Lv10: 选择特化(决策)
- Lv15: ? (无特殊事件)
- Lv20: 满级(结束)

**Lv5到Lv10之间，Lv10到Lv20之间，没有任何里程碑**。玩家在这两段区间内只有"升级→等待→升级→等待"。

**建议方向**:
- Lv5/10/15/20各增加一个"建筑外观变化"(视觉里程碑)
- Lv12解锁"建筑皮肤"或"建筑特效"
- Lv15解锁"建筑专属技能"(如农田Lv15: "丰收祭"临时产出翻倍)
- 每5级增加一个"建筑故事片段"(三国剧情)

## 七、方法论复盘

### 7.1 R5 Challenger 视角: "完整"不等于"好玩"

R5-Builder的PRD执行率从14/17提升至17/17，GAP从2降至0，流程数从65增至78。这些指标都指向"完整性"的提升。

但Challenger的核心质疑是: **完整性是手段，可玩性是目的**。

- R5增加了浮动指示器、治愈报告、推荐标识、事件系统——这些都是"信息展示"和"随机触发"
- R5没有增加"玩家需要做出有意义决策的环节"
- R5没有增加"玩家的操作能产生可见差异的环节"

**建筑系统的核心问题不是"流程不完整"，而是"操作不有趣"**。

### 7.2 "被动系统"vs"主动系统"的设计张力

R5的建筑系统本质上是一个"被动系统":
- 产出是自动的
- 加成是被动的
- 协同是自动触发的
- 医馆加成是持续生效的

玩家的主动操作只有: 收取(无决策)、升级(无决策)、治疗(简单决策)、事件(假选择)。

**一个好的SLG建筑系统应该在"被动产出"和"主动管理"之间找到平衡**:
- 被动产出提供"成长感"(数字变大)
- 主动管理提供"掌控感"(我的选择有影响)

R5目前偏重被动产出，缺乏主动管理。

### 7.3 R4→R5 改进效果评估

| R4 Arbiter布置的R5任务 | R5执行情况 | 效果评估 |
|----------------------|-----------|:-------:|
| 特化重置机制(PRD-R018) | 首次免费+每周任务 | **有效** — 解决了永久决策无法纠正的问题 |
| 新手溢出保护(PRD-R020) | 缓冲45min+新手引导 | **有效** — 新手体验改善明显 |
| 特化假选择修正(CH-F073) | 3座建筑重平衡 | **部分有效** — 铁匠铺引入随机性(新问题)，兵营引入跨建筑加成(新困惑) |
| 特化-协同交互规则(CH-F075) | 同层加法叠加+上限80% | **有效** — 规则清晰，上限合理 |
| 医馆视觉反馈(CH-F066) | 浮动指示器+治愈报告 | **部分有效** — 有展示但感知仍不足(CH-F079) |
| VIP功能特权(CH-F071/072) | 未处理，延后R6 | **未执行** |

### 7.4 本轮Challenger的创新点

1. **游戏步骤推演替代数值分析**: 本轮不再计算"5%日产是多少兵"，而是推演"玩家看到5%日产时的内心OS"——从数值合理性转向体验合理性
2. **"假选择"的深层挖掘**: 不只指出"选择A永远最优"，而是分析"为什么选择B让人后悔"(如军医-50%治疗量是惩罚性设计)
3. **"被动系统"vs"主动系统"的框架**: 明确指出建筑系统的核心问题是"主动操作不足"，而非"流程不完整"
