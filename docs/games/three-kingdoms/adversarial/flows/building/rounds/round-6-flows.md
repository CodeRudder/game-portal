## 一、医馆损失框架设计

> **设计目标**: 让玩家从"看不见医馆价值"变为"清楚知道没有医馆会损失多少"——通过绝对值展示、升级对比、未建造负值、损失框架报告四个维度

**R5问题(CH-F079)**: 浮动"+20%产出"是相对值，玩家不知道20%对应多少绝对收益；治愈报告"额外产出3,840"缺乏对比基准。R5方案是"信息展示"而非"价值感知"。

**R6方案**: 采用"损失框架"——让玩家看到"如果没有医馆/如果医馆降级，你会损失什么"。心理学依据: 损失厌恶(人对损失的感知是收益的2倍)。

6. [持续更新] 每个产出周期更新绝对值显示

### BLD-F19-01(新增) 绝对值展示子流程

**PRD来源**: PRD-R021 "浮动指示器改为绝对值"

**玩家视角**: 看到医馆上显示"今日已为您节省 12,345 粮草" → 知道医馆具体值多少

**流程步骤**:
1. [系统计算] 每个产出周期(1秒)，计算医馆加成带来的额外产出:
   ```
   clinicSavingPerSecond = 其他建筑总产出 × clinicProductionBonus
   其中 clinicProductionBonus = clinic.level × 2%
   ```
2. [累计统计] 累计医馆今日节省的资源:
   ```
   clinicDailySaving[grain] += clinicSavingPerSecond × dt
   clinicDailySaving[copper] += clinicCopperSavingPerSecond × dt
   clinicDailySaving[troops] += clinicTroopSavingPerSecond × dt
   ```
3. [UI显示] 医馆建筑卡片上方浮动显示:
   - 第一行: "医馆今日为您节省了 12,345 粮草" (绿色，绝对值)
   - 第二行: "+3.2 粮草/秒" (小字，当前速率)
   - 数字每秒滚动更新(从右向左滚动动画)
4. [重置机制] 每日0点重置累计统计:
   - `clinicDailySaving = {grain: 0, copper: 0, troops: 0}`
   - UI显示从0开始重新累计

**数据变更**:
| 数据 | R5 | R6 | 说明 |
|------|----|----|------|
| clinicFloatingIndicator | "+X% 产出" | "今日已节省 X 粮草" | 相对值改为绝对值 |
| clinicDailySaving | 不存在 | {grain, copper, troops} | 每日累计节省量 |
| clinicSavingPerSecond | 不存在 | 数值 | 每秒节省速率 |

**前置条件**: 医馆 level > 0
**后置流程**: 无(持续显示)

### BLD-F19-02(新增) 升级对比动画子流程

**PRD来源**: PRD-R021 "升级前后对比动画"

**玩家视角**: 点击升级医馆 → 看到"升级前 vs 升级后"的产出对比 → 直观感受升级价值

**流程步骤**:
1. [触发条件-TE14] 医馆升级完成时触发
2. [数据采集] 记录升级前后的产出数据:
   - `beforeBonus = (clinic.level - 1) × 2%`
   - `afterBonus = clinic.level × 2%`
   - `beforeOutput = 基础产出 × (1 + beforeBonus)`
   - `afterOutput = 基础产出 × (1 + afterBonus)`
3. [UI展示] 弹出对比动画面板(持续3秒):
   - 左侧: "升级前" + 绿色数字(旧产出)
   - 中间: 绿色箭头动画
   - 右侧: "升级后" + 绿色数字(新产出)
   - 底部: "医馆Lv{N} 为所有建筑额外产出 +{X} 粮草/秒"
   - 数字从旧值滚动到新值(0.5秒动画)
4. [对比延伸] 面板底部显示年化收益:
   - "升级医馆每年(游戏内)可额外产出 {X} 粮草"
   - 计算: `年化收益 = (afterOutput - beforeOutput) × 86400 × 365`
5. [自动关闭] 3秒后自动关闭，或点击任意位置关闭

**数据变更**:
| 数据 | R5 | R6 | 说明 |
|------|----|----|------|
| upgradeCompareData | 不存在 | {before, after, diff} | 升级对比数据 |

**前置条件**: 医馆升级完成
**后置流程**: → BLD-F13-01(加成重算)

### BLD-F19-03(新增) 未建造损失提示子流程

**PRD来源**: PRD-R021 "未建造时显示红色负值"

**玩家视角**: 还没建医馆 → 看到所有建筑产出详情里有一行红色"医馆未建造: -40%产出" → 强烈感知医馆的价值

**流程步骤**:
1. [系统检查] 检查医馆是否已建造:
   - `clinicBuilt = buildings.clinic.level > 0`
2. [条件触发] 医馆未建造时(`clinicBuilt == false`)，在所有建筑的产出详情中插入红色行:
   ```
   基础产出: 8.0 粮草/秒
   × 主城加成: +20% (主城Lv10)
   × 科技加成: +10% (农耕改良)
   × 武将加成: +5% (关羽驻守)
   × 医馆加成: [未建造] -40% ← 红色闪烁
   × 协同+特化: +45%
   × 治疗Buff: +10%
   = 最终产出: 13.7 粮草/秒 (损失了 9.1 粮草/秒!)
   ```
3. [损失计算] 红色行的数值:
   - `-40%` 对应医馆Lv20的满级加成(假设玩家已知医馆最高加成)
   - 实际显示根据玩家当前主城等级推算: `假设医馆=主城等级，则损失 = 主城等级 × 2%`
   - 损失量: `损失产出 = 当前产出 × (1 / (1 - 损失百分比) - 1)`
4. [UI样式] 红色行特殊样式:
   - 红色字体，闪烁3次后保持红色
   - 行尾显示红色向下箭头图标
   - 最终产出旁显示红色叹号 + "损失了 X 粮草/秒"
5. [建造引导] 红色行可点击:
   - 点击后跳转到医馆建造界面
   - 建造界面顶部显示: "建造医馆可为所有建筑增加 2%/级 产出加成"

**数据变更**:
| 数据 | R5 | R6 | 说明 |
|------|----|----|------|
| clinicNotBuiltLoss | 不存在 | 数值 | 未建造医馆的损失量 |

**前置条件**: 医馆未建造(level == 0)
**后置流程**: → 医馆建造界面

### BLD-F19-04(新增) 损失框架治愈报告子流程

**PRD来源**: PRD-R021 "治愈报告改为损失框架"

**玩家视角**: 每日登录看到"若医馆降1级，你今天会少产出 3,840 粮草" → 对降级产生强烈抵触

**流程步骤**:
1. [触发条件-TE17] 每日首次登录时触发(与R5治愈报告同一触发器)
2. [数据计算] 计算损失框架数据:
   - `currentLevel = clinic.level`
   - `downgradeLoss = 当前产出 - (当前产出 × (1 + (currentLevel-1) × 2%) / (1 + currentLevel × 2%))`
   - `dailyLoss = downgradeLoss × 86400`
   - `noClinicLoss = 当前产出 × (1 - 1 / (1 + currentLevel × 2%))`
   - `dailyNoClinicLoss = noClinicLoss × 86400`
3. [UI展示] 弹出损失框架报告面板:
   ```
   医馆Lv10 每日价值报告
   ━━━━━━━━━━━━━━━━━
   今日医馆为您节省:
     粮草: 12,345
     铜钱: 8,567
     兵力: 4,321

   若医馆降为Lv9，明日您将损失:
     粮草: -3,840/天 ← 红色
     铜钱: -2,640/天 ← 红色
     兵力: -1,320/天 ← 红色

   若无医馆，您每日损失:
     粮草: -18,500/天 ← 红色加粗
     铜钱: -12,800/天 ← 红色加粗
     兵力: -6,400/天 ← 红色加粗
   ━━━━━━━━━━━━━━━━━
   升级医馆可进一步减少损失!
   ```
4. [按钮操作] 面板底部:
   - "升级医馆"按钮(跳转升级界面)
   - "知道了"按钮(关闭面板)
5. [频率控制] 每日最多显示1次，玩家可关闭(设置中可重新开启)

**数据变更**:
| 数据 | R5 | R6 | 说明 |
|------|----|----|------|
| dailyHealingReport | {healedTroops, bonusOutput} | {saved, downgradeLoss, noClinicLoss} | 损失框架数据 |

**前置条件**: 医馆 level > 0
**后置流程**: 无(面板关闭后继续游戏)

## 二、事件系统修正

> **设计目标**: 将事件系统从"30%随机触发+假选择"改为"100%每日触发+随机建筑+双向收益选择"

**R5问题(CH-F086)**: 30%每次登录触发是"运气决定体验"；无每日上限；针对最高等级建筑(事件多样性被压缩)；登录时强制弹出(打断核心操作)。

**R6方案**: 每日首次登录100%触发1次；随机选择Lv5+建筑；非强制弹窗(事件气泡)；同一建筑24小时冷却；两个选择都是"不同方向的收益"。

### BLD-F20(新增) 事件重设计主流程

**PRD来源**: PRD-R026 "建筑事件频率重设计" + PRD-R027 "事件选择重平衡"

**玩家视角**: 每日登录 → 看到某座建筑冒出事件气泡 → 点击查看 → 面临两个都有吸引力的选择 → 获得收益

**流程步骤**:
1. [触发] 每日首次登录时100%触发(BLD-F20-01)
2. [选择建筑] 从Lv5+且无冷却的建筑中随机选择(BLD-F20-03)
3. [选择事件] 从该建筑的事件池中随机选择事件
4. [展示事件] 建筑卡片上冒出事件气泡(非强制弹窗)(BLD-F20-01)
5. [玩家选择] 玩家点击气泡查看详情，做出选择(BLD-F20-02)
6. [事件结算] 执行选择结果(BLD-F18-02)
7. [冷却设置] 该建筑进入24小时冷却(BLD-F20-03)

**后置流程**: → BLD-F20-01/02/03(三个子流程)

### BLD-F20-01(新增) 每日事件触发子流程

**PRD来源**: PRD-R026 "每日首次登录100%触发1次"

**玩家视角**: 登录游戏 → 看到某座建筑上方冒出金色气泡 → 气泡上有"!"标记 → 不影响正常操作，想看时再点

**流程步骤**:
1. [触发条件-TE23] 每日首次登录时检查:
   - 条件1: `todayEventTriggered == false` (今日尚未触发)
   - 条件2: 存在至少1座Lv5+且无冷却的建筑
   - 两个条件都满足时，100%触发
2. [建筑选择] 从符合条件的建筑中随机选择:
   ```
   eligibleBuildings = buildings.filter(b => b.level >= 5 && b.eventCooldown <= now)
   targetBuilding = eligibleBuildings[random(0, eligibleBuildings.length - 1)]
   ```
3. [事件选择] 从该建筑类型的事件池中随机选择:
   - 农田: 丰收之年/虫灾来袭/干旱预警
   - 市集: 商队来访/盗贼出没/物价波动
   - 兵营: 新兵入伍/逃兵事件/军事演习
   - 铁匠铺: 名匠造访/矿脉发现/铁匠请假
   - 书院: 学者来访/古籍发现/火灾隐患
   - 医馆: 疫病爆发/名医到访/药材丰收
   - 城墙: 巡逻发现/敌军偷袭/修缮良机
4. [UI展示-事件气泡] 在目标建筑卡片上方显示:
   - 金色气泡动画(轻微上下浮动)
   - 气泡内显示"!"图标
   - 气泡不遮挡建筑操作按钮
   - 气泡持续存在直到玩家点击查看或当日结束
5. [非强制性] 事件气泡不会:
   - 弹出全屏遮罩
   - 打断玩家当前操作
   - 强制玩家立即处理
   - 玩家可以忽略气泡继续游戏
6. [超时处理] 若玩家当日未点击查看:
   - 次日0点气泡消失
   - 不触发冷却(未查看=未参与)
   - 不计入"今日已触发"

**数据变更**:
| 数据 | R5 | R6 | 说明 |
|------|----|----|------|
| eventTriggerProb | 30%每次登录 | 100%首次登录 | 确定性>随机性 |
| eventTargetBuilding | 最高等级建筑 | 随机Lv5+建筑 | 增加多样性 |
| eventDisplay | 全屏弹窗 | 建筑气泡 | 非强制性 |
| todayEventTriggered | 不存在 | true/false | 每日触发标记 |

**前置条件**: 至少一座建筑Lv5+

### BLD-F20-02(新增) 事件选择重平衡子流程

**PRD来源**: PRD-R027 "事件选择重平衡"

**玩家视角**: 点击气泡 → 看到两个都有吸引力的选择 → 根据当前需求选择方向 → 获得对应收益

**设计原则**: 每个事件的两个选择必须是"两种不同方向的收益"，而非"付费vs惩罚"。

**流程步骤**:
1. [玩家操作] 点击建筑上的事件气泡
2. [UI展示] 弹出事件详情面板(半屏，不遮挡全部界面):
   - 顶部: 事件插图 + 事件名称
   - 中部: 事件描述(简短故事，3行以内)
   - 左侧: 选择A卡片(图标+名称+效果描述)
   - 右侧: 选择B卡片(图标+名称+效果描述)
   - 底部: "选择后今日内无法再次触发该建筑事件"
3. [选择A/B设计] 两个选择必须满足:
   - **都是收益**(无惩罚选项)
   - **方向不同**(即时vs持续，资源vs加成，特定vs全面)
   - **强度相当**(不能有明显更优选择)
4. [玩家操作] 选择A或B
5. [系统执行] 立即执行选择效果
6. [UI反馈] 显示获得的收益(绿色飘字)

## 三、主动决策机制

> **设计目标**: 增加建筑系统的"有意义的主动操作"——不是增加操作次数，而是增加操作的决策深度

## 四、建筑进化系统

> **设计目标**: 为满级建筑提供长期目标——当建筑达到最高等级后，可通过"进化"重置等级但获得永久加成和外观变化

**R6方案**: 建筑达到满级(Lv25/30)后解锁进化系统。进化重置等级为Lv1，但获得永久星级加成(1星→2星→3星)。每颗星提供递增的永久加成，满星建筑拥有独特外观。

### BLD-F22-03(新增) 进化效果子流程

**PRD来源**: PRD-R028 "进化永久加成"

**玩家视角**: 进化后建筑获得永久加成 → 1星+10%产出 → 2星+25%产出 → 3星+50%产出+独特外观

**流程步骤**:
1. [永久加成] 进化后立即生效的永久加成:
   - 1星: 全局产出+10%(永久，不随等级变化)
   - 2星: 全局产出+25%(永久，替代1星的10%)
   - 3星: 全局产出+50%(永久，替代2星的25%)
2. [加成注入] 进化加成注入产出公式:
   ```
   最终产出 = 基础产出(Lv表)
            × 主城加成
            × 科技加成
            × 武将加成
            × 医馆加成
            × (协同加成 + 特化加成)
            × 治疗Buff
            × 进化加成(1 + prestigeBonus%)   ← R6新增
            × 焦点加成(1 + focusBonus%)       ← R6新增
   ```
3. [外观变化] 每颗星解锁建筑外观变化:
   - 1星: 建筑边框变为银色
   - 2星: 建筑边框变为金色
   - 3星: 建筑边框变为金色 + 特殊粒子效果 + 建筑图标变化
4. [UI显示] 建筑卡片上的星级标识:
   - 建筑名称旁显示星星图标(1~3颗)
   - 建筑详情面板显示"进化加成: +{X}%"
   - 产出详情中显示"进化加成: +{X}% ({N}星)"
5. [TE-29触发] 进化完成时触发:
   - 播放进化完成动画
   - 弹出恭喜面板: "恭喜! [建筑名]已进化至{N}星"
   - 显示获得的永久加成
   - 显示解锁的新等级上限
   - "继续升级" / "查看外观" 按钮

**进化加成数值表**:
| 星级 | 永久加成 | 等级上限 | 外观 | 进化消耗 |
|:----:|:-------:|:-------:|------|---------|
| 0星 | 0% | Lv25 | 默认 | — |
| 1星 | +10% | Lv30 | 银色边框 | 粮草5万+铜钱3万+元宝100 |
| 2星 | +25% | Lv35 | 金色边框 | 粮草15万+铜钱10万+元宝300 |
| 3星 | +50% | Lv40 | 金边+粒子+新图标 | 粮草50万+铜钱30万+元宝1000 |

**设计说明**:
- 进化后等级重置为Lv1，但星级加成是永久的(即使Lv1也有+10%/+25%/+50%加成)
- 进化加成与现有加成(主城/科技/武将/医馆/协同/特化/治疗)是乘法关系
- 3星进化后再次升到Lv40的产出 = 基础Lv40 × (1+50%) = 相当于Lv60的产出
- 进化系统为满级玩家提供了明确的长期目标

**数据变更**:
| 数据 | R5 | R6 | 说明 |
|------|----|----|------|
| prestigeBonus | 不存在 | 10%/25%/50% | 进化永久加成 |
| building外观 | 默认 | 银/金/金+粒子 | 进化外观 |

**前置条件**: 进化完成
**后置流程**: 无(永久生效)

## 六、总结

### R6 核心成果

1. **医馆损失框架(BLD-F19)**: 从"信息展示"转向"价值感知"——绝对值展示("今日节省12,345粮草")、升级前后对比动画、未建造时红色"-40%产出"、损失框架治愈报告("若降级你将损失X")。四个维度让玩家从"看不见医馆"变为"离不开医馆"。
2. **事件系统重设计(BLD-F20)**: 从"30%随机+假选择"改为"100%每日+双向收益"——每日首次登录100%触发1次、随机选择Lv5+建筑、非强制气泡通知、24h建筑冷却、两个选择都是"不同方向的收益"。
3. **主动决策机制(BLD-F21)**: 新增3个有意义的主动操作——建筑焦点(标记优先建筑获得+5%加成)、每日挑战(限时升级获得奖励)、建筑巡查(定期访问获得随机惊喜)。
4. **建筑进化系统(BLD-F22)**: 为满级建筑提供长期目标——进化重置等级但获得永久星级加成(1星+10%/2星+25%/3星+50%)，每星解锁新等级上限和独特外观。
5. **P0全部修复**: CH-F079(医馆感知)和CH-F086(事件触发)两个P0问题全部修复。
6. **特化重平衡**: 军医(冷却延长不降治疗量)、锻造(保底机制)、屯田(训练速度替代铜钱)三个假选择修正。
7. **治疗优化**: 治疗量从5%提升至10%日产，减少操作次数。
8. **PRD 25/25全覆盖**: 所有PRD修订(R001~R028)全部执行。
