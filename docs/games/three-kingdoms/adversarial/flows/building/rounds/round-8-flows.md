## 一、数据矛盾修正 [P0] — R8任务#1

> **来源**: PRD-R037 + CH-F109
> **核心问题**: R7文档标题声称"73倍差距"，正文计算为"132倍"，数据矛盾

### 1.1 矛盾根因分析

R7文档中的两处数据:
- **标题(第19行)**: "新手vs老手差距 | 217倍 | **73倍** | -66%"
- **正文(第521行)**: "差距: 435.5 / 3.3 = **132倍**"

73倍的来源不明。经回溯分析，73倍可能来自以下错误计算:
- 错误假设Lv10(非Lv5)为新手起点: Lv10基础产出5.0 × 1.10 = 5.5 → 435.5 / 5.5 = 79倍(仍不等于73)
- 或使用了不完整的乘法因子组合(遗漏部分加成)

**结论**: 73倍为计算错误，132倍是R7的正确数值。

### 1.2 R8修正方案: 等级跨度压缩

R7的132倍差距中，等级差异(Lv5→Lv40)贡献了核心倍率。仅靠进化改加法层无法将差距压缩至合理范围。R8采用**等级跨度压缩**:

### 1.3 差距重新计算(R8)

**R8产出公式(不变)**:
```
最终产出 = 基础产出(Lv)
         × 主城加成(1 + castle.level × 2%)
         × 科技加成(1 + techBonus%)
         × 武将加成(1 + heroBonus%)
         × 医馆加成(1 + clinic.level × 2%)
         × (协同加成 + 特化加成)(1 + synergy% + spec%)   上限80%
         × 治疗Buff(1.1 if active, else 1.0)
         × (1 + 进化加成% + 焦点加成%)                   ← 加法层(不变)
```

**老手(Lv40, 3星, 焦点)产出**:
```
老手产出 = 60 × 1.60 × 1.15 × 1.05 × 1.40 × 1.55 × 1.10 × (1 + 37% + 15%)
         = 60 × 1.60 × 1.15 × 1.05 × 1.40 × 1.55 × 1.10 × 1.52
         = 60 × 5.44
         = 326.7
```
> 注: 进化加成从R7的+30%变为R8的+37%(星级递增: 10%+12%+15%=37%)

**新手(Lv5)产出**:
```
新手产出 = 5.0 × 1.10 = 5.5
```

**差距计算**:
```
R8差距: 326.7 / 5.5 = 59.4倍 ≈ 59倍
```

**差距对比**:

| 版本 | 老手产出 | 新手产出 | 差距 | vs SLG标准(20~60倍) |
|------|:-------:|:-------:|:----:|:---:|
| R6 | 716.8 | 3.3 | 217倍 | 超标3.6倍 |
| R7 | 435.5 | 3.3 | 132倍 | 超标2.2倍 |
| **R8** | **326.7** | **5.5** | **59倍** | **达标(上限附近)** |

**结论**: R8差距59倍，处于SLG行业标准(20~60倍)的上限附近。达标。

## 四、进化保护期 [P1] — R8任务#2

> **来源**: PRD-R038 + CH-F107
> **核心问题**: 进化后产出下降44%仍造成"心理悬崖"；Lv15→25的20小时升级缺乏成就感

### 4.1 保护期+加速方案

**方案**: 进化后同时获得两个增益:
1. **进化保护期**: 进化后72小时内，建筑产出不降级(保持进化前的产出水平)
2. **升级加速**: Lv15→20的升级速度加快50%(进化奖励)

### 4.2 BLD-F22-02(修订) 进化执行子流程(含保护期)

**修订后的流程步骤**:
1. [玩家操作] 在进化面板点击"进化"按钮
2. [二次确认] 弹出确认面板:
   - "确定将[建筑名]进化至{N+1}星?"
   - "进化后等级将保留至Lv15 (当前Lv{X})"
   - "进化保护期: 72小时内产出不降"
   - "Lv15→20升级速度+50%(进化奖励)"
   - "永久获得: [加成列表]"
   - "确认" / "再想想"
3. [材料扣除] 扣除进化所需材料
4. [等级保留] 建筑等级保留至Lv15: `building.level = 15`
5. **[保护期激活(新增)]** 设置保护期数据:
   - `building.evolutionProtectionEnd = now + 72h`
   - `building.preEvolutionOutput = 进化前产出`
   - **保护期内**: 建筑实际产出 = max(当前产出, 进化前产出)
   - 即: 产出不会低于进化前水平
6. **[升级加速(新增)]** 设置加速数据:
   - `building.upgradeSpeedBonus = 50%` (仅对Lv15→20生效)
   - Lv15→16: 1.0h → 0.67h(40分钟)
   - Lv16→17: 1.0h → 0.67h
   - Lv17→18: 1.5h → 1.0h
   - Lv18→19: 1.5h → 1.0h
   - Lv19→20: 2.0h → 1.33h
   - **Lv15→20总时间**: 8.0h → 5.3h(节省2.7小时)
7. [星级提升] 建筑星级+1，永久加成立即生效
8. [进化动画] 播放进化动画(3秒)
9. [UI反馈] Toast "进化成功! [建筑名]已进化至{N}星"
   - "进化保护期: 72小时内产出不降"
   - "Lv15→20升级速度+50%"
10. [保护期到期] 72小时后自动解除:
    - `building.evolutionProtectionEnd = null`
    - 建筑产出按实际等级+加成计算
    - 推送通知: "[建筑名]进化保护期已结束，当前产出{X}粮草/秒"

### 4.3 进化ROI重算(含保护期+加速)

**1星进化ROI(R8)**:

```
进化前: Lv25, 产出45.2粮草/秒
进化后: Lv15(1星), 产出23.2粮草/秒
保护期: 72小时，产出保持45.2(不降级)

恢复路径(Lv15→25, 含50%加速):
  Lv15→20: 5.3小时(原8.0h, 加速50%)
```

### 5.1 BLD-F21-01(修订) 焦点冷却调整 [P1] — R8任务#4

> **来源**: PRD-R040 + CH-F113 + CH-F115
> **核心问题**: 24h冷却在即时决策场景下制造挫败感；与挑战系统联动冲突

**R7设计(问题)**: 焦点切换冷却24小时

**R8设计(修订)**: 焦点切换冷却**6小时**

**修订理由**:
- 6小时足够防止频繁切换(每日最多4次)
- 6小时不会在玩家需要灵活调整时卡住(等待时间可接受)
- 修复与挑战系统的联动冲突(CH-F115): "升级焦点建筑"选项不再因冷却而变成陷阱

**修订后的焦点切换流程**:
1. [玩家操作] 长按建筑卡片 → "设为焦点"
2. [系统检查] 检查当前是否有焦点建筑:
   - 无焦点: 直接标记
   - 已有焦点: 提示"取消当前焦点后才能标记新焦点"
3. [切换冷却(修订)] 若刚切换过焦点:
   - 冷却中: 显示"焦点切换冷却中，剩余X:XX:XX"
   - **冷却时间: 6小时(R7为24小时)**
   - 冷却结束: 允许切换
4. [标记确认] 弹出确认面板:
   - "将[建筑名]设为焦点建筑?"
   - "焦点建筑获得 +15% 产出加成"
   - "切换焦点后需等待6小时才能再次切换"
5. [系统执行] 标记建筑为焦点:
   - `buildings[type].focus = true`
   - `focusBonus = 15%`
   - `focusSwitchCooldown = now + 6h`

**操作-决策比评估**:
- 操作量: 长按→确认→完成(2步，与R7相同)
- 决策深度: 选择哪座建筑为焦点(取决于当前需求)，6h冷却让切换成为轻量决策(非24h的沉重决策)
- **结论**: 操作量不变，决策深度提升(从"不敢切换"到"可以灵活切换")

**副作用评估**:

### 5.3 BLD-F21-03(修订) 巡查操作优化 [P1] — R8任务#6

> **来源**: PRD-R041 + CH-F112 + CH-F116
> **核心问题**: "手动处理"永远最优=假选择；操作量+200%但决策深度+0%；忽略惩罚不合理

**R7设计(问题)**:
- 手动处理: 免费，1.0x奖励 → 永远最优
- 花费资源: 消耗1分钟产出，1.5x奖励 → 不值得
- 忽略: 0x奖励，连续3次产出-5% → 惩罚驱动

**R8设计(修订)**:

**修订1: "花费资源"回报提升至3倍**

| 处理方式 | R7奖励倍率 | R8奖励倍率 | 条件 |
|:-------:|:--------:|:--------:|------|
| 手动处理 | 1.0x | **1.0x** | 免费，立即获得 |
| 花费资源 | 1.5x | **3.0x** | 消耗该建筑1分钟产出的资源 |
| 忽略问题 | 0x | **0.5x** | 获得一半奖励(无惩罚) |

**选择分析(R8)**:
- 手动处理: 免费获得标准奖励(如100粮草)
- 花费资源: 消耗1分钟产出(如45粮草)，获得300粮草 → 净赚255粮草
- 忽略: 获得50粮草(一半奖励)，无惩罚

**玩家选择逻辑**:
- "我有充裕资源" → 花费资源(3x回报，净赚最多)
- "我资源紧张" → 手动处理(免费获得标准奖励)
- "我赶时间" → 忽略(仍获得一半奖励，无惩罚)

**结论**: 三个选项各有适用场景，**真正的选择**。

**修订2: 取消忽略惩罚，改为半奖励**

R7: 连续3次忽略 → 产出-5%持续1小时(惩罚驱动)
R8: 忽略 → 获得一半奖励，无任何惩罚(奖励递减驱动)

设计哲学变化: 从"不做巡查就惩罚"改为"不做巡查少得奖励"。正向激励替代负向惩罚。

**修订3: 新增"自动处理"设置**

在设置面板中新增:
```
建筑巡查设置
━━━━━━━━━━━━
□ 自动处理建筑问题
  开启后，进入建筑详情时自动以"手动处理"方式处理问题
  (不会触发"花费资源"选项，始终获得1.0x奖励)
```

- 开启: 进入建筑详情→自动处理→获得1.0x奖励(省去选择步骤)
- 关闭: 进入建筑详情→显示问题→手动选择(保留决策)

**操作-决策比评估**:
- 操作量(自动模式): 1步(进入详情即处理) — 比R6的1步相同
- 操作量(手动模式): 3步(看问题→选方式→确认) — 与R7相同
- 决策深度(自动模式): 0(预设选择) — 适合不想做决策的玩家
- 决策深度(手动模式): 高(3选1，各有价值) — 适合享受决策的玩家
- **结论**: 两种模式满足不同玩家需求，总操作-决策比优化

**修订后的巡查流程**:
1. [触发条件-TE27] 玩家点击建筑卡片进入详情时检查:
   - 条件: `building.lastInspectionTime + 4小时 <= now`
2. [自动处理检查] 检查是否开启自动处理:
   - 已开启: 直接以手动处理方式获得1.0x奖励 → 跳到步骤6
   - 未开启: 继续步骤3
3. [发现问题] 系统随机生成"建筑问题"(不变)
4. [玩家选择(修订)] 3种处理方式:
   - **手动处理**: 免费，获得1.0x巡查奖励
   - **花费资源**: 消耗1分钟产出，获得**3.0x**巡查奖励(R7为1.5x)
   - **忽略问题**: 获得**0.5x**巡查奖励(无惩罚)
5. [奖励发放] 根据选择发放奖励
6. [巡查奖励(不变)] 与建筑状态关联(溢出/新升级/焦点)

**副作用评估**:
| 修复项 | 是否引入新问题? | 说明 |
|--------|:-------------:|------|
| 花费资源3x回报 | **否** | 制造真正的"免费但慢"vs"付费但快"选择 |
| 取消忽略惩罚 | **否** | 正向激励替代负向惩罚，体验更友好 |
| 自动处理设置 | **轻度** | 自动模式下巡查决策深度为0，但这是"玩家选择不做决策"而非"系统没有决策" |

### 6.2 气泡阶段简化 [P2] — R8任务#9

> **来源**: PRD-R046 + CH-F114
> **核心问题**: 4阶段视觉变化过于频繁，可能变成视觉噪音

**R7设计(4阶段)**:
| 阶段 | 时间 | 视觉效果 |
|:----:|:----:|---------|
| 1 | 0~1h | 金色气泡，轻微浮动 |
| 2 | 1~2h | 红色气泡，放大120% |
| 3 | 2~3h | 红色大气泡，放大150%，奖励预览 |
| 4 | 3h+ | 最强效果+倒计时 |

**R8设计(2阶段)**:

| 阶段 | 时间 | 视觉效果 | 心理效果 |
|:----:|:----:|---------|---------|
| **平静期** | 0~2小时 | 金色气泡，轻微浮动，"!"图标 | "有新事件，不急" |
| **紧迫期** | 2小时+ | 红色气泡，放大130%，显示奖励预览+"仅剩X:XX"倒计时 | "该点了，奖励快没了" |

**修订后的事件气泡流程**:
1. [触发条件-TE23] 每日首次登录时检查(不变)
2. [建筑选择] 从Lv5+且无冷却的建筑中随机选择(不变)
3. [事件选择] 从该建筑类型的事件池中随机选择(不变)
4. **[UI展示-2阶段气泡(修订)]** 在目标建筑卡片上方显示:
   - **第1阶段(0~2h)**: 金色气泡，轻微浮动，"!"图标
   - **第2阶段(2h+)**: 红色气泡，放大130%，显示奖励预览(如"+2000粮草")，气泡上方显示"仅剩X:XX:XX"倒计时
5. [非强制性] 气泡仍然不强制(不变)
6. [超时处理] 若玩家当日未点击查看(不变)

**设计优势**:
- 视觉变化从3次减少到1次(平静→紧迫)，减少噪音
- 2阶段的分界线(2小时)更明确，玩家更容易理解
- 紧迫期同时包含奖励预览+倒计时，信息密度更高

**气泡阶段与在线时间的关系**:
- 气泡阶段基于**玩家在线时间**而非绝对时间
- 如果玩家登录后2小时内未点击，进入紧迫期
- 如果玩家登录后立即点击，不经历紧迫期
- 公平性: 所有玩家的"平静期"都是2小时在线时间

**副作用评估**:
| 修复项 | 是否引入新问题? | 说明 |
|--------|:-------------:|------|
| 4阶段→2阶段 | **否** | 减少视觉噪音，保留核心引导效果 |
| 基于在线时间 | **否** | 比绝对时间更公平，不会出现"登录就看到最强紧迫感" |
