## 产出公式

> 🎨 → [UI: 建筑详情面板](../../../ui-design/ui-layout/BLD-buildings.md#bld-2)
> 📋 → [PRD: 资源产出](../../../ui-design/prd/BLD-buildings-prd.md#bld-3)

```
最终产出 = 基础产出(Lv表)
         × 主城加成(1 + castle.level × 2%)
         × 科技加成(1 + techBonus%)
         × 武将加成(1 + heroBonus%)
         × 医馆加成(1 + clinic.level × 2%)
         × (协同+特化)(1 + synergy% + spec%)     上限80%
         × 治疗Buff(1.1 if active, else 1.0)
         × (1 + 进化加成% + 焦点加成%)           加法层
```

**新手vs老手差距**: 59倍 (SLG行业标准20~60倍)

## BLD-F01 资源产出

> 📋 → [PRD: 资源产出](../../../ui-design/prd/BLD-buildings-prd.md#bld-3)

**流程概述**: 建筑每秒产出资源，通过9因子乘法公式计算最终产出。

### 流程步骤

1. [定时触发] 每帧(tick)触发产出计算
2. [基础产出] 从建筑等级表读取基础产出值
3. [主城加成] `× (1 + 主城等级 × 2%)` → 全资源加成
4. [科技加成] `× (1 + 对应科技加成%)` → 科技路线加成
5. [武将加成] `× (1 + 派驻武将属性%)` → 武将属性加成
6. [医馆加成] `× (1 + 医馆等级 × 2%)` → 医馆被动加成
7. [协同+特化] `× (1 + synergy% + spec%)` → 上限80%
8. [治疗Buff] `× 1.1` (激活时) / `× 1.0` (未激活)
9. [进化+焦点] `× (1 + prestigeBonus% + focusBonus%)` → 加法层
10. [资源累加] 最终产出累加到建筑库存
11. [UI飘字] 建筑上方显示产出飘字动画

### 子流程

#### BLD-F01-01 主城加成计算

- [条件] 读取主城等级
- [计算] `castle.level × 2%` → 乘法因子
- [注入] 注入产出公式第2层

#### BLD-F01-02 产出上限检查

- [条件] 资源 ≥ 上限
- [处理] 停止累加，触发溢出降速(见 BLD-F15)

### 触发事件

| 编号 | 类型 | 描述 |
|------|------|------|
| TE-01 | 定时 | 资源产出tick(每帧) |
| TE-12 | 条件 | 资源上限检查(资源变化时) |
| TE-13 | 条件 | 建筑库存溢出(达上限时) |
| TE-15 | 条件 | 库存降速检查(达上限时) |
| TE-19 | 定时 | 跨系统产出注入(每个产出周期) |
| TE-20 | 条件 | 科技点注入(书院产出时) |
| TE-30 | 定时 | 焦点加成计算(每个产出周期) |

## BLD-F02 建筑升级

> 📋 → [PRD: 建筑升级](../../../ui-design/prd/BLD-buildings-prd.md#bld-2)
> 🎨 → [UI: 建筑详情面板](../../../ui-design/ui-layout/BLD-buildings.md#bld-2)

**流程概述**: 消耗资源+等待时间，通过升级队列提升建筑等级。

### 流程步骤

1. [玩家操作] 点击建筑→详情面板→点击"升级"按钮
2. [升级校验] 7项校验(见子流程 BLD-F02-01)
3. [扣费] 扣除粮草+铜钱(+兵力，部分建筑)
4. [入队] 加入升级队列(见子流程 BLD-F02-02)
5. [升级进行] 倒计时推进(tick消费)
6. [升级完成] 等级+1→效果立即生效
7. [解锁链检查] 检查是否触发新建筑解锁(见 BLD-F05)
8. [UI反馈] Toast "升级成功! [建筑名] Lv{N}"

### 子流程

#### BLD-F02-01 升级校验

校验项(按顺序):
1. 建筑未锁定
2. 建筑未在升级中
3. 等级未达上限(≤建筑等级上限)
4. 等级 ≤ 主城等级(主城限制)
5. 队列有空闲槽位
6. 资源充足(粮草+铜钱+兵力)
7. 前置条件满足(Lv4→5需任一建筑Lv4，Lv9→10需任一Lv9)

#### BLD-F02-02 升级队列

- [入队] 加入队列指定槽位
- [tick消费] 每帧减少剩余时间
- [完成] 时间归零→等级+1→效果生效
- [连锁] 队列中下一项自动开始
- [槽位规则] 主城Lv1~5: 1槽 / Lv6~10: 2槽 / Lv11~20: 3槽 / Lv21~30: 4槽

### 触发事件

| 编号 | 类型 | 描述 |
|------|------|------|
| TE-02 | 定时 | 升级完成检查(tick) |
| TE-03 | 条件 | 解锁条件检查(主城升级时) |

## BLD-F07 离线收益

> 📋 → [PRD: 离线建造进度](../../../ui-design/prd/BLD-buildings-prd.md#bld-4)

**流程概述**: 玩家离线期间建筑继续产出，登录后结算离线收益。

### 流程步骤

1. [登录触发] 玩家登录时检测离线时长
2. [离线产出计算] `离线时长 × 产出速率(含全部加成) × 离线效率`
3. [离线效率] 资源效率 × 1.2(建造额外20%效率)
4. [离线面板] 弹出离线收益面板:
   - 离线时长
   - 各建筑产出明细
   - 总收益汇总
5. [玩家操作] 点击"领取"按钮
6. [资源入库] 资源累加到资源系统
7. [升级推进] 离线期间完成的升级正常结算，效果立即生效
8. [队列推进] 队列中等待的升级自动开始

### 子流程

#### BLD-F07-01 离线产出计算

- [输入] 离线时长(秒)
- [计算] `离线产出 = 产出速率 × 离线时长 × 离线效率`
- [限制] 不超过资源上限

#### BLD-F07-02 离线面板

- [展示] 离线时长 + 各建筑产出明细 + 总收益
- [操作] "领取"按钮→资源入库

## BLD-F12 自动升级

> 📋 → [PRD: 自动升级](../../../ui-design/prd/BLD-buildings-prd.md#bld-4)

**流程概述**: 开关控制系统自动将可升级建筑加入队列，按优先级排序。

### 流程步骤

1. [开关控制] 玩家开启/关闭自动升级开关
2. [定时检查] 每个tick检查是否有可升级建筑
3. [优先级排序] 按算法排序(见子流程)
4. [资源检查] 检查资源是否充足(扣除保留阈值)
5. [自动入队] 将优先级最高的可升级建筑加入队列

#### BLD-F12-02 资源保护

2. [资源流转] 建筑产出→下一环节消费→最终回流

### 子流程

#### BLD-F28-01 粮草→兵力→战斗链

2. 市舶司商队派遣消耗粮草
3. 商队返回产出铜钱+繁荣度

#### BLD-F28-04 科技点→科技→全建筑加成链

**链路**: 书院(科技点) → 科技(研究) → 全建筑(加成回流)

1. 书院每秒产出科技点
2. 科技研究消耗科技点
3. 科技完成→全建筑加成
4. 加成→所有建筑产出提升

#### BLD-F28-05 粮草→招募→英雄链

**链路**: 市集/农田(铜钱+粮草) → 酒馆(招募) → 英雄(战力)

1. 农田每秒产出粮草
2. 招募消耗铜钱(市集产出)
3. 招募获得武将
4. 武将派驻建筑→产出加成
5. 武将出征→战斗胜利→战利品

## BLD-F30 城墙陷阱系统

> 📋 → [PRD: 城墙互动设计](../../../ui-design/prd/BLD-buildings-prd.md#bld-2)

**流程概述**: 城墙防御陷阱系统，消耗矿石部署陷阱，攻城时自动触发。

#### BLD-F30-01 陷阱部署

**流程步骤**:

1. [玩家操作] 点击城墙→陷阱面板
2. [陷阱类型] 三种陷阱:
   - **箭塔**: 造成持续伤害
   - **拒马**: 减速敌军
   - **陷坑**: 一次性高伤害
3. [材料检查] 检查矿石是否充足
4. [部署执行] 消耗矿石→陷阱库存+1
5. [库存上限] 陷阱库存上限=城墙等级×5

**数据依赖**:
- [输入] 矿石(矿场产出) + 城墙等级
- [输出] 陷阱库存数量

#### BLD-F30-02 陷阱触发

**流程步骤**:

1. [触发条件] 攻城战开始→城墙有陷阱库存
2. [自动触发] 战斗开始时→陷阱按顺序自动触发
3. [效果计算] 根据陷阱类型计算伤害/减速效果
4. [库存消耗] 触发后陷阱库存-1
5. [战斗影响] 陷阱效果注入战斗计算

**数据依赖**:
- [输入] 陷阱库存 + 敌军属性
- [输出] 战斗伤害/减速效果

#### BLD-F30-03 陷阱补充

**流程步骤**:

1. [触发条件] 攻城战结束后→陷阱库存<上限
2. [补充方式] 手动补充(消耗矿石) / 自动补充(VIP5+)
3. [补充执行] 消耗矿石→陷阱库存补满至上限

**数据依赖**:
- [输入] 矿石 + 陷阱库存上限
- [输出] 陷阱库存补满
