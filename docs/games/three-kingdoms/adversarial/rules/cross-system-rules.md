## 链路一：资源→建筑→资源闭环

### RULE-CROSS-001: 建筑→资源产出→建筑升级 资源循环
- **来源**: building-tech.md BUILD-TECH-007, BUILD-TECH-013 + resource-flow-tree.md RES-GRAIN-001
- **交互链路**: 消耗资源建造农田 → 农田产出粮草 → 粮草用于升级农田 → 升级后产出增加
- **规则描述**: 测试必须验证完整的资源-建筑闭环：农田Lv1产出50/h → 消耗木材400/石料250升级至Lv2 → 产出提升至80/h → 持续产出用于后续升级。每次升级消耗递增（约×1.5倍），产出递增
- **P0模式**: 升级后产出未增加、资源扣除后建筑状态未更新、循环中断
- **验证方法**: 自动化执行建造→等待→升级→验证产出链路，记录每步资源变化
- **自动化程度**: 可自动
- **涉及节点**: BUILD-TECH-001, BUILD-TECH-007, BUILD-TECH-013, RES-GRAIN-001

### RULE-CROSS-002: 仓库容量→资源产出上限→仓库升级→恢复增长
- **来源**: building-tech.md BUILD-TECH-015, BUILD-TECH-034 + resource-flow-tree.md RES-GRAIN-003, RES-GRAIN-007
- **交互链路**: 资源产出达到仓库上限 → 产出停止 → 升级仓库提升上限 → 产出恢复增长
- **规则描述**: 测试必须验证资源产出受仓库容量硬约束：粮仓Lv1上限2000，产出达到上限后触发`resource:overflow`事件并停止增长；升级粮仓至Lv5上限5000后，之前溢出的空间恢复，资源重新开始增长
- **P0模式**: 产出超上限、仓库升级后产出未恢复、overflow事件缺失
- **验证方法**: 填满仓库→验证产出停止→升级仓库→验证产出恢复
- **自动化程度**: 可自动
- **涉及节点**: BUILD-TECH-015, BUILD-TECH-034, RES-GRAIN-003, RES-GRAIN-007

### RULE-CROSS-003: 建筑降级→资源退还→容量截断 级联效应
- **来源**: building-tech.md BUILD-TECH-009, BUILD-TECH-032 + resource-flow-tree.md RES-GRAIN-008
- **交互链路**: 建筑降级 → 退还升级消耗50%资源 → 同时粮仓可能降级导致容量截断
- **规则描述**: 测试必须验证建筑降级的级联效应：农田从Lv3降至Lv2时产出回落至Lv2对应值，退还升级消耗的50%；若同时粮仓降级导致容量上限低于当前粮草量，enforceCaps()必须截断至新上限
- **P0模式**: 降级后产出未回落、退还比例错误、容量截断未执行
- **验证方法**: 构造粮仓+农田同时降级场景，验证产出回退+资源退还+容量截断
- **自动化程度**: 可自动
- **涉及节点**: BUILD-TECH-009, BUILD-TECH-032, RES-GRAIN-008

## 链路二：武将→编队→战斗链路

### RULE-CROSS-004: 武将招募→编队→战斗→经验→升级 全链路
- **来源**: hero-training.md HERO-TRAIN-001 + onboarding.md ONBOARD-024~030 + hero-training.md HERO-TRAIN-009~010
- **交互链路**: 招募武将 → 编入队伍 → 进入战斗 → 战斗获得经验 → 经验满升级 → 属性增长
- **规则描述**: 测试必须验证从招募到升级的完整链路：普通招募获得武将→编入前排→战斗胜利→经验按参战武将平分→经验达到阈值升级→属性按成长率增长。战斗评级基于剩余兵力比例
- **P0模式**: 编队后战斗无经验、经验分配不均、升级属性未增长
- **验证方法**: 端到端自动化：招募→编队→战斗→验证经验和属性
- **自动化程度**: 可自动
- **涉及节点**: HERO-TRAIN-001, ONBOARD-024~030, HERO-TRAIN-009~010

### RULE-CROSS-005: 武将升星→等级上限提升→突破→觉醒技能 联动
- **来源**: hero-training.md HERO-TRAIN-015, HERO-TRAIN-017~018, HERO-TRAIN-025
- **交互链路**: 升星 → 等级上限提升 → 继续升级 → 5→6星突破 → 突破后解锁觉醒技能
- **规则描述**: 测试必须验证升星-突破-觉醒的联动：3星→4星等级上限40→50，溢出经验继承；5→6星需突破材料（天命石×10），突破成功率80%失败返50%；突破+达到指定等级后解锁觉醒技能（消耗觉醒石×5+同名碎片×30，不可重置）
- **P0模式**: 升星后上限未提升、突破材料不消耗、觉醒技能条件判断错误
- **验证方法**: 自动化执行升星→突破→觉醒全链路
- **自动化程度**: 可自动
- **涉及节点**: HERO-TRAIN-015, HERO-TRAIN-017, HERO-TRAIN-018, HERO-TRAIN-025

### RULE-CROSS-006: 装备获取→穿戴→强化→属性加成→战力计算 链路
- **来源**: hero-training.md HERO-TRAIN-026~029 + battle-campaign.md BATTLE-CAMP-005~006
- **交互链路**: 副本掉落装备 → 穿戴到武将 → 强化提升属性 → 属性加成反映在战力 → 战力评估影响推荐
- **规则描述**: 测试必须验证装备全链路：副本掉落装备品质与属性范围匹配→穿戴后武将属性实时增加→强化消耗强化石+铜钱提升属性→强化等级≤武将等级×2→属性加成正确反映在总战力→战力评估与推荐战力对比
- **P0模式**: 穿戴后属性未增加、强化超限、战力计算不含装备加成
- **验证方法**: 获取装备→穿戴→强化→验证属性和战力变化
- **自动化程度**: 可自动
- **涉及节点**: HERO-TRAIN-026~029, BATTLE-CAMP-005~006

### RULE-CROSS-007: 羁绊→编队→战斗属性加成 联动验证
- **来源**: hero-training.md HERO-TRAIN-034 + battle-campaign.md BATTLE-CAMP-007
- **交互链路**: 编入羁绊组合武将 → 羁绊自动激活 → 属性加成叠加到面板 → 战斗中羁绊效果生效
- **规则描述**: 测试必须验证羁绊在编队和战斗中的完整生效：编入刘备+关羽+张飞→"桃园结义"自动激活→属性加成叠加到武将面板→编队界面羁绊高亮显示→战斗中羁绊加成参与伤害计算→下阵后羁绊取消
- **P0模式**: 羁绊激活但战斗未生效、羁绊加成叠加错误、下阵未取消
- **验证方法**: 编入/下阵羁绊组合，验证面板和战斗伤害
- **自动化程度**: 可自动
- **涉及节点**: HERO-TRAIN-034, BATTLE-CAMP-007

## 链路三：联盟→Boss→奖励→商店链路

### RULE-CROSS-008: 联盟捐赠→联盟资金→Boss挑战→奖励分配→商店兑换 全链路
- **来源**: alliance-save.md ALLIANCE-007, ALLIANCE-011~014
- **交互链路**: 成员铜钱捐赠→联盟资金增加→盟主消耗资金发起Boss→成员挑战Boss→击杀按伤害分配奖励→贡献点在商店兑换物品
- **规则描述**: 测试必须验证联盟经济闭环：铜钱捐赠10000→贡献点+100+联盟资金+500→Boss消耗联盟资金2000→击杀后奖励按伤害比例通过邮件发放→贡献点200在商店兑换碎片×5。联盟等级决定商店可兑换物品和Boss难度上限
- **P0模式**: 捐赠转换率错误、Boss消耗资金不正确、奖励未按比例、商店兑换不扣贡献点
- **验证方法**: 模拟完整联盟经济流程，验证每步数值变化
- **自动化程度**: 可自动
- **涉及节点**: ALLIANCE-007, ALLIANCE-011, ALLIANCE-012, ALLIANCE-013, ALLIANCE-014

### RULE-CROSS-009: 联盟战争→攻防积分→个人贡献→奖励邮件 链路
- **来源**: alliance-save.md ALLIANCE-016~018
- **交互链路**: 联盟匹配→成员进攻/防守→积分累计→战争结算→个人贡献排名→奖励邮件发放
- **规则描述**: 测试必须验证联盟战争全链路：匹配按总战力±15%→每人每日3攻1防→胜方联盟资金×5000+全体元宝×200→个人贡献前3名额外元宝→败方安慰奖联盟资金×1000→所有奖励邮件30天有效期
- **P0模式**: 匹配范围超限、攻防次数不限制、奖励未按邮件发放
- **验证方法**: 多联盟协同测试战争全流程
- **自动化程度**: 需手动（多联盟协同）
- **涉及节点**: ALLIANCE-016, ALLIANCE-017, ALLIANCE-018

### RULE-CROSS-010: 联盟退出/解散→存档清理→商店数据清除 联动
- **来源**: alliance-save.md ALLIANCE-019~020, SAVE-018

- **规则描述**: 测试必须验证退出联盟的级联清理：贡献点清零但已兑换物品不回收→本地存档allianceId清除→24小时内不可加入新联盟→解散时所有成员收到邮件→联盟资金/等级/商店数据全部清除→名称24小时后释放
- **P0模式**: 退出后贡献未清零、存档残留联盟数据、冷却期可加入
- **验证方法**: 退出后检查存档数据和商店状态
- **自动化程度**: 可自动
- **涉及节点**: ALLIANCE-019, ALLIANCE-020, SAVE-018

## 链路四：存档→所有子系统序列化

### RULE-CROSS-011: 全系统状态序列化→反序列化往返一致性
- **来源**: alliance-save.md SAVE-004~006, SAVE-007 + daily-loop.md DAILY-033
- **交互链路**: 6大子系统同时变更 → 触发保存 → 序列化所有状态 → 加载存档 → 反序列化 → 验证各系统状态一致
- **规则描述**: 测试必须验证存档包含并正确恢复6大子系统：①武将（ID/等级/星级/技能/装备/编队，空编队位序列化为null）②战斗（关卡进度/章节解锁/每日挑战次数）③建筑（等级/升级队列剩余时间）④科技（研究进度/科技点）⑤联盟（ID/角色/贡献）⑥任务（日常进度/活跃度）。升级中的建筑/科技保存剩余时间，加载时已到期自动完成
- **P0模式**: 子系统数据丢失、空位序列化省略而非null、到期未自动完成
- **验证方法**: 执行全系统操作后保存→清空内存→加载→逐系统验证
- **自动化程度**: 可自动
- **涉及节点**: SAVE-004, SAVE-005, SAVE-006, SAVE-007, DAILY-033

### RULE-CROSS-012: 断线重连→战斗快照保存→校验→恢复 链路
- **来源**: alliance-save.md SAVE-019 + battle-campaign.md BATTLE-CAMP-021
- **交互链路**: 战斗中断线 → 本地保存战斗快照 → 网络恢复 → 服务器校验快照 → 合法则恢复/不合法则判负
- **规则描述**: 测试必须验证战斗中断线的完整保护链：断线时本地立即保存战斗快照（状态+剩余时间+伤害数据）→重连后服务器校验快照合法性（战斗ID/时间戳/伤害）→校验通过从断点继续→校验失败（检测篡改）判负不发奖励→超时按失败处理不扣体力
- **P0模式**: 快照未保存、校验跳过、篡改检测失效
- **验证方法**: 战斗中模拟断网→恢复→验证快照和继续战斗
- **自动化程度**: 需手动（网络操作）
- **涉及节点**: SAVE-019, BATTLE-CAMP-021

### RULE-CROSS-013: 跨设备同步→联盟状态校验→数据覆盖 链路
- **来源**: alliance-save.md SAVE-013~014, SAVE-018
- **交互链路**: A设备保存上传云端 → B设备登录 → 检测云端更新 → 选择覆盖 → 联盟状态与服务器校准
- **规则描述**: 测试必须验证跨设备同步与联盟状态校验：A设备上传存档含allianceId→B设备检测cloudSyncTime>localSyncTime→弹出选择→覆盖后联盟ID与服务器查询一致→若服务器显示已不在联盟（被踢/解散）则本地清除联盟数据并提示
- **P0模式**: 跨设备联盟数据不一致、被踢后仍显示在联盟
- **验证方法**: 模拟双设备同步+服务器端踢人操作
- **自动化程度**: 需手动（双设备+服务器操作）
- **涉及节点**: SAVE-013, SAVE-014, SAVE-018

## 链路五：科技→全局加成→所有产出

### RULE-CROSS-014: 科技研究→资源产出加成→建筑建造加速 联动
- **来源**: building-tech.md BUILD-TECH-025~026, BUILD-TECH-035~037
- **交互链路**: 研究农耕术 → 所有农田产出+10% → 研究高级农耕 → 叠加至+25% → 研究建筑术 → 建造时间-20%
- **规则描述**: 测试必须验证科技对资源产出的全局加成效果：农耕术Lv1完成后所有等级农田统一+10%（Lv1: 50→55/h, Lv3: 120→132/h）→高级农耕叠加至+25%→建筑术完成后新建筑建造时间缩短20%（60min→48min）→科技重置后所有加成立即移除产出恢复基础值
- **P0模式**: 加成仅对部分等级生效、叠加计算错误、重置后加成残留
- **验证方法**: 研究科技前后对比多等级农田产出，重置后验证回退
- **自动化程度**: 可自动
- **涉及节点**: BUILD-TECH-025, BUILD-TECH-026, BUILD-TECH-035, BUILD-TECH-037

### RULE-CROSS-015: 科技→解锁新建筑→新建筑产出新资源 链路
- **来源**: building-tech.md BUILD-TECH-038 + resource-flow-tree.md RES-TECH-001
- **交互链路**: 研究城防学Lv1 → 建造列表新增箭塔 → 建造箭塔 → 箭塔提供防御加成
- **规则描述**: 测试必须验证科技解锁新建筑的完整链路：研究特定科技前建造列表无该建筑→研究完成后列表新增→新建筑可正常建造消耗正确资源→建造完成后功能完整→科技重置后已建造建筑不消失但不可再新建同类型
- **P0模式**: 科技完成后建筑未解锁、新建筑功能异常、重置后已建建筑消失
- **验证方法**: 研究科技前后检查建造列表，建造后验证功能
- **自动化程度**: 可自动
- **涉及节点**: BUILD-TECH-038

### RULE-CROSS-016: 科技→天命获取加成→天命消耗加速→科技研究加速 正反馈循环
- **来源**: resource-flow-tree.md RES-MANDATE-011, RES-TECH-011 + daily-loop.md DAILY-022
- **交互链路**: 科技"名将传承"激活 → 天命获取+15% → 更多天命用于加速研究 → 科技"兴学令" → 科技点获取+10% → 正反馈循环
- **规则描述**: 测试必须验证科技加成的正反馈循环：天命获取+15%后攻城/事件获得的天命按1.15倍率增加→科技点获取+10%后书院产出乘以1.1倍率→两者叠加加速后续科技研究→研究队列占用检查（队列满时拒绝新研究）
- **P0模式**: 加成倍率未叠加、队列限制失效
- **验证方法**: 激活科技前后对比天命和科技点获取量
- **自动化程度**: 可自动
- **涉及节点**: RES-MANDATE-011, RES-TECH-011, DAILY-022

### RULE-CROSS-017: 科技分支互斥→建筑产出方向性差异→战力分化
- **来源**: building-tech.md BUILD-TECH-027~028 + daily-loop.md DAILY-023
- **交互链路**: 选择军事分支 → 兵营产出+15% → 经济分支锁定 → 使用重置卷轴切换 → 军事效果消失 → 经济效果激活 → 市集产出+20%
- **规则描述**: 测试必须验证科技分支互斥对游戏方向的影响：选择军事线后兵营产出+15%但经济线锁定→科技重置卷轴切换分支→军事加成立即消失→经济线激活市集+20%→退还已消耗铜币70%→日常循环中科技互斥组mutexGroup正确生效
- **P0模式**: 互斥分支同时生效、切换后旧加成残留、退还比例错误
- **验证方法**: 选择分支→验证加成→切换→验证旧加成消失+新加成生效
