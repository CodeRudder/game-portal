# Arbiter Agent Rules — 三国霸业

> 版本: v1.1 | 初始化: 2026-05-01
> 每轮复盘后更新此文件

## 评分规则

### 5维度评分（各0-10分）
| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 完备性 | 25% | 所有API和场景是否覆盖 |
| 准确性 | 25% | covered标注是否真实，P0声称是否可靠 |
| 优先级 | 15% | P0/P1/P2分配是否合理 |
| 可测试性 | 15% | 每个节点能否转化为测试用例 |
| 挑战应对 | 20% | Builder是否充分回应了Challenger的质疑 |

### 封版线: 9.0分

### R1特殊考量
- 首轮评分应考虑系统性问题的影响（一个系统性问题可能影响多个节点）
- 虚报率>5%应扣准确性分
- 算法正确性问题应提高优先级

## 封版条件（全部满足）

1. 评分 >= 9.0
2. API覆盖率 >= 90%
3. F-Cross覆盖率 >= 75%
4. F-Lifecycle覆盖率 >= 70%
5. P0节点覆盖 = 100%
6. 虚报数 = 0
7. 最终轮新P0 = 0
8. 所有子系统覆盖 = 是

## 收敛判断规则

1. 连续2轮评分差 < 0.5 → 收敛信号
2. 最终轮新P0 = 0 → 强收敛信号
3. 节点增量 < 30 → 饱和信号
4. Challenger 5分钟内无新P0 → 封版信号

## 三Agent复盘规则

每轮必须对三个Agent进行复盘：
1. Builder表现评分 + 改进建议
2. Challenger表现评分 + 改进建议
3. Arbiter独立发现（约10%补充）
4. 规则进化建议（具体到规则文件）

## 可玩性评估规则

对游戏模块额外评估：
| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 趣味性 | 25% | 玩法是否有趣 |
| 进度平衡 | 25% | 成长曲线是否合理 |
| 经济平衡 | 20% | 资源获取/消耗是否平衡 |
| 玩家体验 | 15% | 整体体验是否流畅 |
| 系统一致性 | 15% | 各系统是否协调 |
