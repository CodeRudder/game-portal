# Challenger Agent Rules — 三国霸业

> 版本: v1.0 | 初始化: 2026-05-01
> 每轮复盘后更新此文件

## 通用规则

1. 从5个维度系统性质疑（F-Normal/F-Boundary/F-Error/F-Cross/F-Lifecycle）
2. 每个质疑必须引用源码（文件+行号）
3. 每个P0声称必须包含复现场景
4. 虚报率必须测量并报告
5. 封版轮仅关注P0级遗漏

## 高频P0检查清单

### null/undefined防护
- 检查每个公开API的参数是否有null guard
- 检查可选参数解构是否有fallback
- 检查deserialize(null)是否安全

### NaN传播
- 检查数值计算链是否有NaN检查
- 检查Math.max/Math.min是否处理NaN输入
- 检查除法是否有除零保护

### 负值漏洞
- 检查伤害/消耗/扣除是否有<=0检查
- 检查负值是否会变成"治疗"或"奖励"

### 浅拷贝副作用
- 检查数组展开后是否修改嵌套属性
- 检查Object.assign后是否修改深层对象

### 竞态条件
- 检查资源预锁后异常是否回滚
- 检查循环体内异常是否导致永久卡死

### 经济漏洞
- 检查购买/兑换是否有日限购/总限购
- 检查是否有无限刷资源路径

## 三国霸业特定挑战策略

1. 武将系统：重点检查双路径一致性（HeroSystem vs HeroLevelSystem）
2. 战斗系统：重点检查NaN/负值传播链
3. 攻城系统：重点检查子系统数据完整性
4. 经济系统：重点检查无限购买/负值漏洞

## 可玩性评估维度

1. 进度曲线是否平滑（不会太快也不会太慢）
2. 奖励反馈是否及时（玩家行为是否有明确反馈）
3. 策略深度是否足够（是否有多种有效策略）
4. 经济平衡是否合理（通胀/通缩风险）
5. 新手体验是否友好（前10分钟体验）
