# 武将培养模块 - 测试流程树

> 模块：武将培养（Hero Training）
> 节点数：36
> 覆盖子系统：武将获取 / 武将升级 / 武将升星 / 技能学习 / 装备系统 / 编队设置
> 生成时间：2025-07-09
> 状态说明：covered=已覆盖 | missing=未覆盖 | partial=部分覆盖

## 一、武将获取（Hero Acquisition）

### HERO-TRAIN-001: 普通招募
- 前置: 无
- 操作: 消耗普通招募令×1，执行普通招募
- 预期: 随机获得1~3星武将，概率符合配置表（3星70%、2星25%、1星5%），背包中新增对应武将卡牌
- 优先级: P0
- 关联系统: 武将获取 / 招募池 / 背包
- 状态: covered

### HERO-TRAIN-002: 高级招募
- 前置: 无
- 操作: 消耗元宝×280或高级招募令×1，执行高级招募
- 预期: 随机获得3~5星武将，保底机制生效（10连必出4星+），概率符合配置表
- 优先级: P0
- 关联系统: 武将获取 / 招募池 / 保底系统
- 状态: covered

### HERO-TRAIN-003: UP池招募
- 前置: 无
- 操作: 在限定UP池中消耗元宝招募，目标获取UP武将（如：赵云）
- 预期: UP武将出现概率为普通同星级武将的3倍；80抽大保底必出UP武将；小保底50%概率歪常驻
- 优先级: P0
- 关联系统: 武将获取 / UP池 / 保底系统
- 状态: covered

### HERO-TRAIN-004: 碎片合成武将
- 前置: HERO-TRAIN-001
- 操作: 收集指定武将碎片达到合成阈值（如：30碎片），执行合成操作
- 预期: 碎片扣除30个，背包新增对应武将卡牌（初始星级根据碎片品质决定），若已有同名武将则转化为碎片
- 优先级: P0
- 关联系统: 武将获取 / 碎片系统 / 背包
- 状态: covered

### HERO-TRAIN-005: 活动限定获取
- 前置: 无
- 操作: 参与限时活动（如：七日登录、跨服争霸），达成条件后领取活动武将
- 预期: 活动武将正确发放到背包；活动结束后未领取的奖励通过邮件补发；活动武将不可通过常规招募获得
- 优先级: P1
- 关联系统: 武将获取 / 活动系统 / 邮件系统
- 状态: partial

### HERO-TRAIN-006: 重复武将处理
- 前置: HERO-TRAIN-001
- 操作: 招募到已拥有的同名同星级武将
- 预期: 系统自动将重复武将转化为对应数量的武将碎片（转化比例符合配置表），弹窗提示"武将已拥有，已转化为碎片×N"
- 优先级: P0
- 关联系统: 武将获取 / 碎片系统
- 状态: covered

### HERO-TRAIN-007: 招募资源不足
- 前置: 无
- 操作: 在招募令/元宝数量不足时点击招募按钮
- 预期: 按钮置灰或弹出资源不足提示，引导至充值/获取途径页面；不执行招募逻辑
- 优先级: P1
- 关联系统: 武将获取 / 支付引导
- 状态: covered

## 二、武将升级（Hero Level Up）

### HERO-TRAIN-008: 经验道具升级
- 前置: HERO-TRAIN-001
- 操作: 选中已拥有武将，消耗经验书（初级/中级/高级）提升经验值
- 预期: 经验值正确累加，经验书扣除，武将当前经验条进度更新；不超出当前等级上限
- 优先级: P0
- 关联系统: 武将升级 / 消耗品
- 状态: covered

### HERO-TRAIN-009: 战斗经验获取
- 前置: HERO-TRAIN-001
- 操作: 将武将编入队伍并完成战斗（关卡/竞技场）
- 预期: 战后结算时武将获得对应经验值，经验来源标注"战斗奖励"；已满级武将不再获取经验
- 优先级: P1
- 关联系统: 武将升级 / 战斗系统
- 状态: covered

### HERO-TRAIN-010: 等级提升与属性增长
- 前置: HERO-TRAIN-008
- 操作: 经验值达到升级阈值，触发升级
- 预期: 等级+1，基础属性（攻击/防御/生命/速度）按成长率正确增长；升级特效播放；若突破新阶段（10/20/30级）触发额外属性加成
- 优先级: P0
- 关联系统: 武将升级 / 属性计算
- 状态: covered

### HERO-TRAIN-011: 铜钱升级消耗
- 前置: HERO-TRAIN-008
- 操作: 武将升级时检查铜钱消耗
- 预期: 每次升级消耗铜钱=等级×100，铜钱不足时提示"铜钱不足"并引导至铜钱获取途径；铜钱正确扣除
- 优先级: P0
- 关联系统: 武将升级 / 经济系统
- 状态: covered

### HERO-TRAIN-012: 等级上限限制
- 前置: HERO-TRAIN-010
- 操作: 武将达到当前星级对应的等级上限（如3星上限40级）时继续使用经验书
- 预期: 提示"已达当前星级等级上限，请先升星"；经验书不可使用；引导至升星界面
- 优先级: P0
- 关联系统: 武将升级 / 武将升星
- 状态: covered

### HERO-TRAIN-013: 一键升级
- 前置: HERO-TRAIN-008
- 操作: 点击"一键升级"按钮，系统自动计算所需经验书和铜钱，将武将升至当前可达到的最高等级
- 预期: 自动选择最优经验书组合（优先消耗低级经验书），铜钱和经验书正确扣除，武将等级正确提升至上限或资源耗尽
- 优先级: P1
- 关联系统: 武将升级 / 消耗品 / 经济系统
- 状态: missing

## 三、武将升星（Hero Star Up）

### HERO-TRAIN-014: 碎片收集检查
- 前置: HERO-TRAIN-004
- 操作: 查看武将升星所需碎片数量，对比当前持有碎片
- 预期: 正确显示当前碎片数/所需碎片数；碎片不足时"升星"按钮置灰并显示差额；碎片充足时按钮高亮可点击
- 优先级: P0
- 关联系统: 武将升星 / 碎片系统
- 状态: covered

### HERO-TRAIN-015: 升星操作执行
- 前置: HERO-TRAIN-014
- 操作: 碎片数量满足时执行升星操作（如：3星→4星）
- 预期: 碎片正确扣除（如3→4星需60碎片），武将星级+1，升星动画播放；属性面板实时更新显示新数值
- 优先级: P0
- 关联系统: 武将升星 / 属性计算
- 状态: covered

### HERO-TRAIN-016: 升星属性大幅增长
- 前置: HERO-TRAIN-015
- 操作: 对比升星前后属性面板
- 预期: 基础属性按星级成长系数大幅增长（升星增长率远高于升级）；生命值增长约30%，攻击/防御增长约25%，速度增长约10%
- 优先级: P0
- 关联系统: 武将升星 / 属性计算
- 状态: covered

### HERO-TRAIN-017: 突破机制触发
- 前置: HERO-TRAIN-015
- 操作: 武将达到特定星级（5星→6星）时触发突破
- 预期: 除碎片外还需额外消耗突破材料（如：天命石×10）；突破后解锁新被动技能或强化已有技能；突破有成功率设定（6星80%），失败返还50%材料
- 优先级: P0
- 关联系统: 武将升星 / 突破系统 / 技能系统
- 状态: covered

### HERO-TRAIN-018: 升星等级上限提升
- 前置: HERO-TRAIN-015
- 操作: 升星后检查武将等级上限变化
- 预期: 等级上限随星级提升（3星40级→4星50级→5星60级→6星80级）；已溢出的经验不丢失，自动计入新等级进度
- 优先级: P0
- 关联系统: 武将升星 / 武将升级
- 状态: covered

### HERO-TRAIN-019: 满星武将处理
- 前置: HERO-TRAIN-015
- 操作: 对已达最高星级（如6星/7星）的武将再次获取碎片
- 预期: 溢出碎片可兑换为通用货币（如：将魂）；升星界面显示"已达最高星级"；不再显示升星入口
- 优先级: P1
- 关联系统: 武将升星 / 碎片系统
- 状态: missing

## 四、技能学习（Skill Learning）

### HERO-TRAIN-020: 技能解锁
- 前置: HERO-TRAIN-010
- 操作: 武将达到指定等级/星级时检查技能槽解锁
- 预期: 按配置表正确解锁技能槽（如：1级解锁普攻，10级解锁主动技，30级解锁被动技，50级解锁觉醒技）；新技能显示"未学习"状态并引导学习
- 优先级: P0
- 关联系统: 技能学习 / 武将升级
- 状态: covered

### HERO-TRAIN-021: 技能学习操作
- 前置: HERO-TRAIN-020
- 操作: 消耗技能书和铜钱学习已解锁的技能
- 预期: 技能书和铜钱正确扣除；技能从"未学习"变为"已学习Lv.1"；武将战斗中可使用该技能
- 优先级: P0
- 关联系统: 技能学习 / 消耗品
- 状态: covered

### HERO-TRAIN-022: 技能升级
- 前置: HERO-TRAIN-021
- 操作: 对已学习技能执行升级操作（Lv.1→Lv.2）
- 预期: 消耗对应品质技能书×2+铜钱×5000；技能伤害/效果数值按配置表提升；技能等级上限为Lv.10
- 优先级: P0
- 关联系统: 技能学习 / 消耗品
- 状态: covered

### HERO-TRAIN-023: 技能书消耗与不足
- 前置: HERO-TRAIN-021
- 操作: 技能书数量不足时尝试学习/升级技能
- 预期: 弹出提示"技能书不足"，显示当前持有/需求数量；提供快捷跳转至技能书获取途径（商店/副本扫荡）
- 优先级: P1
- 关联系统: 技能学习 / 消耗品 / 商店
- 状态: covered

### HERO-TRAIN-024: 技能重置
- 前置: HERO-TRAIN-022
- 操作: 使用技能重置道具将已升级技能回退至Lv.1
- 预期: 返还100%已消耗的技能书和铜钱（以邮件形式发送）；技能等级回到Lv.1；重置道具扣除
- 优先级: P1
- 关联系统: 技能学习 / 邮件系统
- 状态: missing

### HERO-TRAIN-025: 觉醒技能解锁
- 前置: HERO-TRAIN-017, HERO-TRAIN-020
- 操作: 武将突破至指定星级且达到指定等级后解锁觉醒技能
- 预期: 觉醒技能为武将专属终极技能；需消耗觉醒石×5+同名武将碎片×30；觉醒技能不可重置
- 优先级: P0
- 关联系统: 技能学习 / 武将升星 / 突破系统
- 状态: partial

## 五、装备系统（Equipment System）

### HERO-TRAIN-026: 装备获取
- 前置: 无
- 操作: 通过副本掉落/商店购买/活动奖励获取装备
- 预期: 装备正确进入背包；装备品质（白/绿/蓝/紫/橙/红）与属性范围匹配；同品质装备属性在范围内随机
- 优先级: P0
- 关联系统: 装备系统 / 背包 / 副本
- 状态: covered

### HERO-TRAIN-027: 装备穿戴
- 前置: HERO-TRAIN-001, HERO-TRAIN-026
- 操作: 从背包选择装备穿戴到武将对应部位（武器/防具/饰品/兵书）
- 预期: 装备从背包移至武将装备栏；武将属性面板实时增加装备提供的属性加成；同部位已有装备时自动替换，旧装备回到背包
- 优先级: P0
- 关联系统: 装备系统 / 背包 / 属性计算
- 状态: covered

### HERO-TRAIN-028: 装备强化
- 前置: HERO-TRAIN-027
- 操作: 消耗强化石和铜钱对已穿戴装备执行强化（+1→+2）
- 预期: 强化石和铜钱正确扣除；装备强化等级+1；装备属性按强化系数增长；强化等级上限受武将等级约束（强化等级≤武将等级×2）
- 优先级: P0
- 关联系统: 装备系统 / 强化 / 经济系统
- 状态: covered

### HERO-TRAIN-029: 装备强化失败
- 前置: HERO-TRAIN-028
- 操作: 装备强化等级≥+8时执行强化，触发失败
- 预期: 强化等级≥+8时存在失败概率（+8:90%, +9:80%, +10:60%）；失败时强化等级-1（最低降至+7）；使用保护符可避免降级；材料不返还
- 优先级: P0
- 关联系统: 装备系统 / 强化 / 概率系统
- 状态: covered

### HERO-TRAIN-030: 装备替换与卸下
- 前置: HERO-TRAIN-027
- 操作: 将武将身上的装备卸下或替换为另一件装备
- 预期: 卸下装备回到背包，武将属性扣除该装备加成；替换装备时旧装备回背包，新装备属性立即生效；战斗中不可更换装备
- 优先级: P1
- 关联系统: 装备系统 / 背包 / 属性计算
- 状态: covered

### HERO-TRAIN-031: 装备套装效果
- 前置: HERO-TRAIN-027
- 操作: 武将穿戴同一套装的2件/4件装备
- 预期: 2件套激活套装基础效果（如：攻击+10%）；4件套激活套装进阶效果（如：暴击率+15%+攻击+10%）；混搭不同套装分别计算各自件数
- 优先级: P1
- 关联系统: 装备系统 / 套装系统 / 属性计算
- 状态: missing

## 六、编队设置（Formation Setup）

### HERO-TRAIN-032: 武将上阵
- 前置: HERO-TRAIN-001
- 操作: 从武将列表选择武将上阵到编队空位（最多6人）
- 预期: 武将头像显示在编队对应位置；上阵武将不可重复（同名不同星视为重复）；已达上限时提示"阵容已满"
- 优先级: P0
- 关联系统: 编队设置 / 武将管理
- 状态: covered

### HERO-TRAIN-033: 位置调整
- 前置: HERO-TRAIN-032
- 操作: 拖拽武将头像调整在阵容中的站位（前排/后排）
- 预期: 位置交换成功；前排武将承受更多伤害，后排武将获得伤害减免；位置影响技能释放顺序
- 优先级: P0
- 关联系统: 编队设置 / 战斗系统
- 状态: covered

### HERO-TRAIN-034: 羁绊激活
- 前置: HERO-TRAIN-032
- 操作: 上阵满足羁绊条件的武将组合（如：刘备+关羽+张飞="桃园结义"）
- 预期: 羁绊条件满足时自动激活，显示羁绊名称和效果；羁绊属性加成正确叠加到武将面板；同一武将可同时激活多个羁绊
- 优先级: P0
- 关联系统: 编队设置 / 羁绊系统 / 属性计算
- 状态: covered

### HERO-TRAIN-035: 编队保存与切换
- 前置: HERO-TRAIN-032
- 操作: 保存当前编队配置，并切换至另一套预设编队
- 预期: 最多保存3套编队方案；切换编队时所有武将站位、装备、羁绊实时更新；保存操作持久化到服务器
- 优先级: P1
- 关联系统: 编队设置 / 存档系统
- 状态: covered

### HERO-TRAIN-036: 武将下阵与锁定
- 前置: HERO-TRAIN-032
- 操作: 将已上阵武将下阵，或锁定武将防止误操作
- 预期: 下阵后武将回到候选列表，羁绊效果实时更新（若不再满足条件则取消）；锁定武将不可被下阵/分解/出售；正在出战中的武将不可下阵
- 优先级: P1
- 关联系统: 编队设置 / 武将管理
- 状态: missing
