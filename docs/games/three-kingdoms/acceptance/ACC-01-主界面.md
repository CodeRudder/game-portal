# ACC-01 主界面 — 用户验收标准

> 模块版本：v1.0 | 文档日期：2025-07-10
> 对应组件：ThreeKingdomsGame / SceneRouter / TabBar / ResourceBar / FeaturePanelOverlay / WelcomeModal / OfflineRewardModal / CalendarDisplay
> 对应引擎：ThreeKingdomsEngine / ResourceSystem / CalendarSystem / HeroBadgeSystem

---

## 模块概述

主界面是玩家进入三国霸业后看到的第一个完整画面，承载以下玩家体验目标：

1. **即时沉浸**：首次进入弹出欢迎弹窗，展示游戏四大核心玩法（建筑/武将/科技/关卡），点击「开始游戏」即可进入
2. **信息一目了然**：顶部资源栏实时展示6种资源（粮草/铜钱/兵力/天命/科技点/招贤令）的数量、产出速率和容量状态
3. **导航清晰**：底部7个Tab（天下/出征/武将/科技/建筑/声望/更多）覆盖全部一级功能入口，切换无延迟
4. **离线有收获**：重新打开游戏时自动弹出离线收益弹窗，展示离线时长和各资源收益
5. **全局氛围**：日历系统展示年号/季节/天气，营造三国时代沉浸感

## 关键风险点（重点关注）

以下是基于代码分析识别的**核心流程断裂风险**，验收时需额外关注：

### 风险1：UI与引擎数据不同步
- **表现**：升级建筑后资源栏数值不变，或武将升级后属性不刷新
- **根因**：引擎500ms tick + UI 1s刷新间隔，`snapshotVersion` 状态更新依赖 `useEngineEvents` 事件监听
- **验收重点**：ACC-01-20、ACC-01-28、ACC-01-29
- **验证方法**：执行操作后等待2秒，确认UI数值已更新

### 风险2：离线收益计算偏差
- **表现**：离线收益与实际产出速率不匹配，或领取后资源未增加
- **根因**：`OfflineEarningsCalculator` 计算逻辑与 `ResourceSystem.tick` 产出逻辑不一致
- **验收重点**：ACC-01-23、ACC-01-24
- **验证方法**：记录离线前资源值和产出速率，计算预期收益，与弹窗显示值对比

### 风险3：快速操作导致状态不一致
- **表现**：快速切换Tab时出现白屏或组件残留
- **根因**：React状态更新批处理与SceneRouter条件渲染的竞态
- **验收重点**：ACC-01-34、ACC-01-35
- **验证方法**：快速连续操作10次以上，确认无异常

### 风险4：资源溢出后产出丢失
- **表现**：粮草/兵力达到上限后，产出资源被静默丢弃，玩家无感知
- **根因**：`ResourceSystem.tick` 中溢出截断逻辑未在UI层明确提示
- **验收重点**：ACC-01-22、ACC-01-33
- **验证方法**：将粮草填满后观察是否有警告提示
