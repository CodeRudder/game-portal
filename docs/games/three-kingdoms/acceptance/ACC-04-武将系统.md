# ACC-04 武将系统 — 用户验收标准

## 模块概述

武将系统是三国霸业的核心养成模块，玩家通过武将Tab（🦸）进入。系统涵盖武将列表浏览、武将详情查看、武将升级、升星、突破、技能升级等完整养成链路。玩家可对已拥有的武将进行筛选/排序、查看四维属性（武力/统率/智力/政治）雷达图、执行升级/升星/突破操作以提升战力。

**玩家体验目标**：玩家能快速浏览所有武将、直观了解每个武将的属性和技能、顺畅地完成升级→升星→突破→技能升级的完整养成流程，且每次操作后属性数值必须立即、准确地反映在界面上。

**涉及核心组件**：HeroTab、HeroListPanel、HeroCard、HeroDetailModal、HeroUpgradePanel、HeroStarUpModal、HeroBreakthroughPanel、SkillPanel、RadarChart、HeroStatsPanel、atoms/（AttributeBar、QualityBadge、ResourceCost、StarDisplay）

**涉及引擎系统**：HeroSystem、HeroLevelSystem、HeroStarSystem、SkillUpgradeSystem、HeroBadgeSystem

**⚠️ 历史Bug警示**：曾发现"武将升级后属性不变化"的核心流程断裂Bug，根因是 `statsAtLevel` 私有函数未导出。本验收标准将升级/升星/突破/技能升级后的属性即时更新作为最高优先级（P0）验收项。

## 验收重点说明

### 🔴 P0 核心流程断裂风险项（必须100%通过）

以下6项为历史Bug反思后的**最高优先级验收项**，任何一项不通过即为阻断性Bug：

| 编号 | 风险项 | 根因关联 | 验证要点 |
|------|--------|----------|----------|
| ACC-04-20 | 升级后属性面板立即更新 | `statsAtLevel` 未导出导致属性不刷新 | 升级后四维数值必须立即变化 |
| ACC-04-21 | 升级后雷达图立即更新 | 同上，雷达图依赖stats计算 | 升级后雷达图面积必须扩大 |
| ACC-04-22 | 升级后战力数值立即更新 | 战力依赖属性计算 | 升级后战力必须立即增长 |
| ACC-04-23 | 升星后属性面板立即更新 | 升星改变baseStats倍率 | 升星后四维数值必须立即变化 |
| ACC-04-25 | 突破后等级上限立即更新 | 突破改变levelCap | 突破后等级上限必须立即提高 |
| ACC-04-26 | 技能升级后效果数值立即更新 | 技能效果值依赖等级 | 技能升级后效果值必须立即变化 |

**验证方法**：每个P0项执行前后截图对比，确认数值确实发生变化。

### 数据流关键路径

```
玩家点击"升级" 
  → engine.enhanceHero(generalId, targetLevel) 
  → HeroLevelSystem.levelUp() 
  → general.level 更新 
  → onEnhanceComplete() 回调触发 
  → HeroTab 重新获取 getGenerals() 
  → HeroDetailModal 中 stats = statsAtLevel(general.baseStats, general.level) 重新计算 
  → 四维属性条 + 雷达图 + 战力 全部刷新
```

**断裂点**：如果 `statsAtLevel` 函数不可用或 `general` 对象引用未更新，则属性面板不会刷新。
