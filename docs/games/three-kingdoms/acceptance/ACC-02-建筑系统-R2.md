### ✅ BUG-2 / ACC-02-34 [P0] 升级按钮 disabled 条件不完整 — **已修复**

| 属性 | R1 问题 | R2 修复验证 |
|------|---------|------------|
| **问题描述** | 确认按钮仅检查 `canAfford`（资源充足），未检查 `info.canUpgrade`（引擎综合判断），导致建筑升级中/队列满/满级时按钮仍可点击 | ✅ 已完整修复 |
| **修复方式** | — | `isConfirmDisabled = !info.canUpgrade \|\| !canAfford \|\| upgrading \|\| isMaxLevel`（第118行），四重保护 |
| **按钮文案** | — | 根据状态优先级显示：`isMaxLevel ? '已满级' : upgrading ? '升级中...' : !info.canUpgrade ? '无法升级' : canAfford ? '▲ 升级' : '资源不足'`（第248行） |
| **防重复点击** | — | 新增 `upgrading` 状态 + `setUpgrading(true)` 保护（第120-125行），防止连续快速点击 |
| **结论** | — | ✅ **PASS** — 按钮 disabled 条件完整，覆盖所有不可升级场景 |

### 🟡 R2-NEW-01 [P1] BuildingIncomeModal 仍直接访问 engine.building 内部属性

| 属性 | 详情 |
|------|------|
| **描述** | `BuildingIncomeModal.tsx:109` 使用 `engine.building.getProduction(type)` 访问引擎内部子系统。`BuildingPanel.tsx:99` 也使用 `engine.building?.getProduction?.(type)`。两处均通过可选链保护不会崩溃，但耦合度较高 |

### 🟡 R2-NEW-02 [P2] 取消升级缺少确认弹窗

| 属性 | 详情 |
|------|------|

## 五、改进建议

### 1. [P1] 引擎公共 API 封装

```typescript
// ThreeKingdomsEngine.ts — 增加公共方法
getBuildingProduction(type: BuildingType): number {
  return this.building?.getProduction?.(type) ?? 0;
}
```

替代 UI 层的 `engine.building?.getProduction?.(type)` 直接访问。

### 2. [P2] 取消升级确认弹窗

在 `handleCancelUpgrade` 中增加确认步骤：

```typescript
const handleCancelUpgrade = useCallback((type: BuildingType) => {
  if (!confirm('确认取消升级？将返还80%已消耗资源。')) return;
  try {
    engine.cancelUpgrade(type);
  } catch (e: any) { ... }
}, [...]);
```
