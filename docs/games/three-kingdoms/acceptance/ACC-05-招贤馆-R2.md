## 一、R1 FAIL 项修复确认

### 1. ❌→✅ ACC-05-22 [P0] RecruitPanel 结果展示缺陷

**R1 问题**：`RecruitPanel` 的 `handleRecruit` 只清空旧结果不设置新结果，结果展示区永远为空。

**R2 验证**：
- `RecruitPanel.tsx` 已新增 `results?: RecruitResultEntry[]` 可选 prop（L68）
- 主组件内部新增 `internalResults` 状态 + 优先使用外部传入 results 的逻辑：
  ```typescript
  const results = externalResults ?? internalResults;
  ```
- `ResultArea` 子组件完整渲染结果卡片（品质色点 + 名称 + 重复标签）
- **修复确认**：✅ 已修复。RecruitPanel 支持外部注入 results 和内部状态双通道。

### 2. ❌→✅ ACC-05-36 [P2] 空武将池兜底提示

**R1 问题**：武将池为空时 UI 显示"???"和"✨ 新获得"，应利用 `isEmpty` 标记显示友好提示。

**R2 验证**（`RecruitModal.tsx` RecruitResultCard 组件）：
```typescript
{result.isEmpty ? '武将池为空' : (result.general?.name ?? '???')}
...
{result.isEmpty ? (
  <div className="tk-recruit-result-dup" style={{ color: '#999' }}>
    请等待武将池补充
  </div>
) : result.isDuplicate ? ...}
```
- **修复确认**：✅ 已修复。空武将池显示"武将池为空" + "请等待武将池补充"灰色提示。

### 3. ⚠️→✅ ACC-05-06 [P1] 保底进度条标签

**R1 问题**：验收标准文档写"硬保底（史诗+）50次"，但实际配置为"传说+ 100次"。

**R2 验证**：
- UI 正确从 `RECRUIT_PITY[recruitType].hardPityThreshold` 动态读取
- 高级池：`hardPityThreshold: 100`, `hardPityMinQuality: LEGENDARY` → UI 显示"硬保底（传说+）X/100"
- 普通池：`hardPityThreshold: Infinity` → UI 隐藏硬保底行
- **修复确认**：✅ UI 与配置一致。验收标准文档已确认以代码配置为准。

### 4. ⚠️→✅ ACC-05-15 [P1] 免费招募 UI

**R1 问题**：R1 报告中提及免费招募 UI 缺失问题。

**R2 验证**（`RecruitModal.tsx`）：
- 新增 `freeRecruitInfo` 状态（L119-123），通过 `recruitSystem.getRemainingFreeCount()` 和 `canFreeRecruit()` 获取
- 新增 `handleFreeRecruit` 回调（L147-165），调用 `recruitSystem.freeRecruitSingle()`
- UI 渲染免费招募按钮（L191-201）：
  ```typescript
  {freeRecruitInfo.maxFree > 0 && (
    <div className="tk-recruit-action-item">
      <div className="tk-recruit-cost tk-recruit-cost--free">
        🎁 每日免费 ({freeRecruitInfo.remaining}/{freeRecruitInfo.maxFree})
      </div>
      <button ... onClick={handleFreeRecruit}>
        {isRecruiting ? '招募中...' : freeRecruitInfo.canFree ? '🎁 免费招募' : '今日已用'}
      </button>
    </div>
  )}
  ```
- **修复确认**：✅ 已修复。免费招募按钮完整：显示剩余次数、用后变为"今日已用"、包含防抖保护。
