## 一、P0 问题修复验证

### P0-1：离线效率系数冲突 ✅ 已修复

| 维度 | Round 2 状态 | Round 3 状态 | 判定 |
|------|:----------:|:----------:|:----:|
| 资源系统定义 | 24h封顶/5级衰减（旧） | 72h封顶/4级衰减（统一后） | ✅ |
| 建筑系统定义 | 72h无封顶/4级衰减（旧） | 72h封顶/4级衰减（统一后） | ✅ |
| 跨模块规范 | 不存在 | `22-offline-reward-spec.md` 全局统一 | ✅ |
| 一致性声明 | 无 | 资源系统 §六 含完整一致性对照表 | ✅ |

**详细分析**：

资源系统 v4.0 §六 明确声明："以下离线效率系数为全游戏唯一标准，所有模块引用同一套规则"。公式为：0~24h/100% → 24~48h/80% → 48~72h/50% → >72h/封顶50%。同时附有与建筑系统的字段级一致性对照表，逐一比对衰减阶梯、封顶时间、科技加成、VIP加成，全部打勾。

新建的 `22-offline-reward-spec.md` 进一步将离线效率系数提升为**全局规范文档**，定义了更精细的6级衰减曲线（0-2h/100% → 2-8h/80% → 8-24h/60% → 24-48h/40% → 48-72h/25% → >72h/15%），覆盖7个子系统的离线行为。

**⚠️ 残留问题**：资源系统 v4.0 和离线规范 v1.0 采用了**不同的衰减曲线**：
- 资源系统：4级（24h/100% → 48h/80% → 72h/50% → 封顶50%）
- 离线规范：6级（2h/100% → 8h/80% → 24h/60% → 48h/40% → 72h/25% → >72h/15%）

这是**新的跨文档冲突**。离线规范作为全局规范文档，定义了更精细的6级曲线，但资源系统文档未同步更新引用。建议以离线规范为准，资源系统下次迭代时同步更新。

**修复判定**：**实质已修复**（P0 级冲突已消除，创建了统一规范文档），但存在新的精度差异（P1 级），不构成 P0。

| 跨模块规范 | 不存在 | `23-currency-spec.md` 统一定义 | ✅ |

**详细分析**：

资源系统 v4.0 §二 用完整章节定义天命资源，覆盖了获取（9种途径，含频率和单次获取量）、消耗（6种用途，含具体消耗量）、规则（6条，含消耗保护和转生保留）、UI视觉区分（4重区分机制）和经济平衡（4阶段预估存量表）。这是非常完整的设计。

新建的 `23-currency-spec.md` 进一步将天命纳入8种货币的统一体系，明确了天命的层级定位（高级非付费货币，位于元宝之下、招贤榜之上），定义了与求贤令的等价兑换关系（100天命 = 100求贤令），以及防通胀机制（转生保留30% + 高价消耗项）。

**小瑕疵**：资源系统中天命"离线收益（微量）0.01/秒"，而货币规范中天命每日获取约15（活跃度10 + 天命阁5），两者对不上——0.01/秒 = 864/天，远超15/天。建议统一数值。

**修复判定**：**完全修复**。天命从"黑箱"变为全游戏定义最完整的资源之一。

**问题**：
