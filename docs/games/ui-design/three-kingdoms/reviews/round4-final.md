### 给开发团队的建议

**1. 开发分阶段推进（建议 5 Phase）**

| Phase | 模块 | 核心目标 |
|:-----:|------|---------|
| Phase 1 | 主界面 + 资源 + 建筑 + 武将 | 核心循环跑通 |
| Phase 2 | 战斗 + 科技 + 声望转生 | 核心玩法闭环 |
| Phase 3 | NPC + 事件 + 任务 + 活动 | 内容系统填充 |
| Phase 4 | 商店 + 装备 + 远征 + 社交 | 扩展系统完善 |
| Phase 5 | 新手引导 + 设置 + 邮件 + 响应式打磨 | 辅助系统 + 品质打磨 |

**2. 优先建立三大基础设施**

- **组件库**：以 §20 交互规范为基础，建立统一的按钮/弹窗/Toast/进度条/Tab 组件库
- **离线引擎**：以 §22 离线收益规范为核心，先实现统一的离线效率计算服务，所有模块调用同一 API
- **货币服务**：以 §23 货币体系规范为核心，统一管理 13 种货币的获取/消耗/转化逻辑

**3. 关键风险提示**

- ⚠️ **手机端信息密度**：多个模块在 375px 宽度下信息密度偏高，建议首期开发后立即做可用性测试
- ⚠️ **13 种货币认知负担**：建议开发期同步设计货币引导系统（新手期逐步解锁，而非一次性展示）
- ⚠️ **性能预算**：地图产出气泡、战斗动画、离线回归动画等并发场景需提前定义帧率目标

**4. 设计文档使用方式**

- 设计文档即为开发规格书，可直接作为前端开发的参考标准
- 文档中的 ASCII 线框图作为布局参考，实际像素由 UI 设计师在 Figma 中细化
- 数值参数（衰减曲线/公式/阈值）直接用于配表，无需二次推算
