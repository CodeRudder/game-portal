# FL-BLD-10 兵营编队

> **版本**: v2 | **最后更新**: 2026-05-05
> **归属功能**: 建筑系统 — 辅助建筑交互 (Phase E)
> **触发条件**: 玩家在兵营详情面板切换到[编队]Tab
> **结束条件**: 编队保存/训练开始，回到编队面板
> **前置状态**: 兵营已建造且等级 >= 1，详情面板已打开
> **后置状态**: 编队配置已保存/训练任务已创建
> **来源文档**: [PRD BLD-8 兵营编队系统](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-8) / [UI BLD-2 子Tab配置](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2)

```
FL-BLD-10 兵营编队

```

## 关键要求

- 最多3支编队，每支可配置不同武将+兵力
- 3种兵种（步/骑/弓）有克制关系
- 训练消耗粮草加速兵力产出，兵营等级影响效率
- 编队可直接用于地图征服/攻城

### R2 新增约束

- **兵力分配模型**: 兵力从兵营可用兵力池分配，编队锁定已分配兵力（其他编队不可使用）。每支编队兵力上限受兵营等级影响。取消编队时锁定兵力归还兵营池
- **训练加速定义**: 加速幅度 = 产出速度x2, 持续时间 = 10分钟, 消耗 = 粮草(兵营等级x100)
- **酒馆等级联动说明**: "酒馆等级影响训练速度(+1%/级)"待PRD BLD-4确认。若PRD无此规则则从本流程删除
- **手机端编队操作**: 手机端不使用拖拽，改为点击选择（点选武将→选择目标编队位置），避免与列表滚动冲突
- **双击防护**: 保存编队/训练加速按钮点击后立即禁用

## 验收标准

- 编队配置正确保存
- 武将不可重复分配到多支编队
- 训练消耗和产出正确

## 步骤详情

### FL-BLD-10-01 进入编队Tab

> **来源**: [UI BLD-2 子Tab配置](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2)

**用户感知**: 在兵营详情面板点击[编队]Tab → 内容区切换为编队管理面板
**系统行为**: 加载当前编队数据、可用武将、可用兵力 → 渲染编队内容
**关键要求**: 底部按钮从[升级/一键满级]切换为[保存编队]
**验收标准**: Tab切换流畅

**📱 空间要点**: D区420x696px

---

### FL-BLD-10-02 查看编队列表

> **来源**: [PRD BLD-8 编队管理](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-8)

**用户感知**: 看到最多3支编队卡片 → 每支显示：编队名、武将列表、兵力分配、兵种搭配、总战力
**系统行为**: 渲染编队列表，显示各编队详细配置
**关键要求**: 空编队显示"+"号引导创建
**验收标准**: 编队信息完整

---

### FL-BLD-10-03 编辑编队

> **来源**: [PRD BLD-8 编队管理](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-8)

**用户感知**: 点击编队卡片 → 进入编辑模式 → PC端: 拖拽武将分配到编队 / 手机端: 点击武将→选择目标编队位置 → 可调整兵力数量和兵种比例
**系统行为**: 校验武将是否已被其他编队占用 → 校验兵力池是否有足够兵力 → 实时计算编队战力 → 显示兵种克制信息
**关键要求**: 武将不可重复分配。兵力从兵营池分配，编队锁定已分配兵力
**验收标准**: 编辑操作流畅，战力实时更新

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 武将已被其他编队占用 | Toast"该武将已在编队X中"，高亮冲突编队 | 10-03 |
| 兵力不足 | 限制最大分配数量，Toast"可用兵力不足" | 10-03 |

---

### FL-BLD-10-04 保存编队

> **来源**: [PRD BLD-8 编队管理](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-8)

**用户感知**: 点击[保存编队] → 按钮加载态 → Toast"编队保存成功"
**系统行为**: 验证编队配置（至少1个武将+兵力>0） → 持久化编队数据
**关键要求**: 编队配置立即生效
**验收标准**: 编队保存成功，数据持久化

**⏱ 时间要点**: 保存请求 <= 500ms

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 编队无武将 | Toast"请至少配置1名武将" | 10-03 |
| 编队无兵力 | Toast"请分配兵力" | 10-03 |

---

### FL-BLD-10-05 训练加速

> **来源**: [PRD BLD-8 训练模式](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-8)

**用户感知**: 在概览Tab看到兵力产出速率 → 可消耗粮草加速训练 → 点击[加速训练] → 扣除粮草(兵营等级x100) → 兵力产出速度x2持续10分钟
**系统行为**: 校验粮草是否足够 → 扣除粮草 → 兵力产出速度翻倍，持续10分钟 → 倒计时显示
**关键要求**: 训练速度受兵营等级影响。加速消耗 = 兵营等级 x 100粮草，效果 = 产出速度x2持续10分钟
**验收标准**: 加速消耗正确，产出正确提升

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 粮草不足 | Toast"粮草不足" | 10-01 |
