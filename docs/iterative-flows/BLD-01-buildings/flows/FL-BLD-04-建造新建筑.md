# FL-BLD-04 建造新建筑

> **版本**: v6 | **最后更新**: 2026-05-05
> **归属功能**: 建筑系统 — 建筑操作 (Phase B)
> **触发条件**: 玩家点击空地上的[建造]按钮
> **结束条件**: 建筑建造完成，新建筑出现在网格中
> **前置状态**: 玩家已打开建造选择弹窗（E区），存在至少1座可建造建筑
> **后置状态**: 新建筑Lv.1出现在网格原空地位置，建造资源已扣除
> **来源文档**: [PRD BLD-1 建筑类型](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1) / [UI BLD-3 建造选择弹窗](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-3)

## 关键要求

- 建造选择弹窗仅显示当前玩家尚未建造的建筑
- 每种建筑全局唯一（玩家只能有1座农田、1座市集等）
- 建造条件：主城等级满足 + 资源足够 + **队列有空槽**（建造任务共享升级队列槽位）
- 建造完成后建筑为Lv.1
- **建造Lv.1费用**: 等同于PRD升级费用表中该建筑首次升级费用（Lv.0→Lv.1），参见PRD BLD-2各建筑升级费用表

### R2 新增约束

- **建造任务与升级队列**: 建造新建筑的任务**共享升级队列槽位**。队列满时不可建造新建筑，需等待当前任务完成或取消
- **资源不足条目可交互**: 橙色状态（资源不足）的建筑条目可点击，展开详情显示缺少数量+获取建议
- **双击防护**: [建造]按钮点击后立即禁用（加载态），建造完成或失败后恢复
- **网络失败回滚**: 服务端事务机制，建造失败时已扣资源自动返还

### R3 新增约束

- **建造选择过滤**: 弹窗内支持按状态筛选——[全部]/[可建造]/[未解锁]，默认显示全部。筛选Tab栏高度36px，标签式切换（活跃态金色下划线）
- **资源预检流程**: 点击空地打开弹窗时，前端立即执行一次资源预检（对比当前资源与各建筑建造费用），标记每条目状态（绿/灰/橙）。预检结果在弹窗出现时即展示，不等服务端响应
- **建造完成通知**: 建造完成后，除Toast外，网格中新建筑卡片播放出生动画（从空地虚线框→建筑卡片渐变出现，500ms：虚线框淡出200ms+建筑图标从scale 0→1弹跳300ms）。同时在对应空地位置产生一道光芒扩散效果（白色圆形扩散，300ms）
- **建造期间空地状态**: 建造任务进行中，对应空地格子显示建造进度条（绿色，底部8px）+锤子图标动画（上下敲击，1s循环）

### R5 新增约束

- **空地三种状态**: 可建造空地（虚线+号）vs 锁定空地（灰色链条+解锁条件）vs 装饰空地（11座全部建造后，剩余空地变为景观装饰态，无交互）
- **搜索与排序**: 弹窗内支持关键词搜索（建筑名+功能描述），默认按解锁条件排序（已解锁优先→同解锁内按建造费用升序）
- **快速补齐资源**: 资源不足条目展开后显示[快速补齐]按钮，点击弹出资源购买确认（消耗元宝购买缺少资源，自动计算差额）
- **建造取消**: 建造中可取消（返回建造选择弹窗或点击空地弹出确认），返还80%资源（与升级取消一致）
- **服务端校验乐观锁**: 建造请求携带`If-Match: {player_version}`头，服务端校验版本号防止并发冲突
- **首次建造引导**: 玩家首次建造建筑时，出生动画后弹出教学引导气泡，指向新建筑并说明其核心功能
- **网络失败三级处理**: L1瞬时故障(自动重试3次) / L2服务端错误(显示错误码+重试按钮) / L3版本冲突(强制刷新数据)

## 验收标准

- 弹窗正确列出所有未建造建筑，按解锁条件排序（已解锁优先→同解锁内按建造费用升序）
- 已解锁+资源够的建筑显示绿色[建造]按钮
- 未解锁建筑显示灰色锁定状态+解锁条件
- 资源不足的建筑显示橙色标记+缺少资源
- 搜索框可按建筑名/功能描述过滤
- 筛选Tab（全部/可建造/未解锁）正确过滤
- 快速补齐资源按钮正确计算元宝消耗
- 空地三种状态（可建造/锁定/装饰）正确显示
- 建造取消返还80%资源
- 首次建造显示教学引导气泡
- 全部建造完成后空地变为装饰态
- 网络失败三级处理正确执行

### FL-BLD-04-01 空地交互与弹窗打开

> **来源**: [UI BLD-3 建造选择弹窗](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-3) / [PRD BLD-1 建筑类型](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1)

#### 空地三种状态及转换规则

**状态A — 可建造空地**:
- 外观: 金色虚线框 `2px dashed rgba(200,169,110,0.3)` + 居中加号 24px + "点击建造" 提示文字 12px
- 背景: `rgba(200,169,110,0.03)`
- 悬停(PC): 虚线变实线 `2px solid rgba(200,169,110,0.5)` + 背景加深 `rgba(200,169,110,0.08)` + 微上浮 2px `translateY(-2px)` + `transition: all 150ms ease`
- 点击(PC/手机): 打开建造选择弹窗
- 触摸目标: 整个180x120px格子（PC）/ 355x72px条目（手机列表中的空地条目），>=44px热区
- 进入条件: 当前主城等级下存在至少1座未建造的已解锁建筑
- 退出转换: 建造执行后 → 空地变为建造中（虚线框变实线+进度条）→ 建造完成后变为建筑卡片

**状态B — 锁定空地**:
- 外观: 灰色虚线框 `2px dashed rgba(102,102,102,0.3)` + 居中锁链图标 24px + "主城Lv.X解锁" 灰色文字 12px
- 背景: `rgba(102,102,102,0.05)`
- 悬停(PC): cursor:not-allowed，Tooltip显示"升级主城至Lv.X可解锁: [建筑A]、[建筑B]"
- 点击: Toast"升级主城至Lv.X解锁更多建筑"，2秒自动消失
- 进入条件: 当前主城等级低于某些建筑解锁等级，且还有建筑未建造
- 退出转换: 主城升级跨越解锁阈值时 → 锁定空地立即变为可建造空地（动画: 锁链碎裂200ms + 虚线变金色150ms）

**状态C — 装饰空地**（全部11座建筑已建造后）:
- 外观: 淡金色细线框 `1px solid rgba(200,169,110,0.15)` + 小型景观图标（花草/石灯笼/小池等，随机分配，不重复）+ 微弱光泽脉冲动画（4s循环，opacity 0.3→0.6→0.3）
- 背景: `rgba(200,169,110,0.03)` 微透明
- 交互: cursor:default，无点击事件，不响应任何操作，无Tooltip
- 特效: 每隔60秒随机一个装饰空地播放环境动画（蝴蝶飞过/落叶/萤火虫，持续3s）
- 景观图标池: 花丛、石灯笼、小池塘、竹林、假山、梅树、荷花盆、石阶、松树盆景、棋盘石凳（10种不重复分配）
- 进入条件: 11座建筑全部建造完成（包括刚完成最后一座的那一刻）
- 退出转换: 不可逆（建造后建筑不可拆除，PRD无拆除机制）
- 转换触发: 当最后一座建筑建造完成时，所有剩余空地同时播放装饰化动画（虚线框淡出200ms + 装饰图标fade-in 300ms）→ 详见04-07

**空地状态判定算法**（客户端本地，0ms）:
```
if (已建造建筑数 >= 11) → 状态C（装饰）
else if (未建造建筑中存在已解锁的) → 状态A（可建造）
else → 状态B（锁定）
```

**用户感知（see->act->feedback）**:
- **See**: 空地格子显示对应状态外观。可建造空地显示虚线框+加号。锁定空地显示锁链+解锁条件。装饰空地显示景观+微光脉冲
- **Act**: 点击可建造空地（触摸目标>=44px）。锁定/装饰空地点击无效。双击防护立即生效（按钮/空地点击后300ms内忽略第二次点击）
- **Feedback**: 可建造空地点击 → 空地scale缩放（95% 100ms）→ 遮罩层fade-in（150ms）→ 弹窗scale弹出（PC: 从80%->100%, 200ms ease-out）/ Bottom Sheet上滑（手机: 从底部滑出200ms ease-out，overshoot 5%后回弹）

**系统行为**:
1. 判断空地类型（0ms本地缓存）:
   - 已建造建筑在格子上的 → 不触发（已由FL-BLD-03处理）
   - 建造中状态 → 显示建造进度面板（不打开选择弹窗），转至04-04建造中交互
   - 可建造空地（尚有未建造建筑） → 继续打开弹窗
   - 锁定空地（有建筑但全部未解锁） → Toast提示
   - 装饰空地（全部已建造） → 无响应
2. 前端资源预检（04-03a子步骤）: 立即对比当前资源与各建筑建造费用，计算每条目状态（<50ms）
3. 渲染建造选择列表: 按排序规则排列，标记状态（绿/灰/橙）
4. 弹窗出现动画（PC <=200ms / 手机 <=200ms）
5. 并行请求服务端获取最新资源数据 `GET /api/player/resources`（<=500ms确认预检结果，到达后静默刷新标记）

**关键要求**: 弹窗出现动画<=200ms。资源预检结果在弹窗出现时即展示。支持筛选Tab+搜索。空地状态判断基于本地缓存数据，不阻塞UI

**时间要点**:
- 空地状态判断: 0ms（本地缓存数据）
- 前端资源预检: <50ms（弹窗出现前完成）
- 弹窗出现动画: <=200ms
- 服务端资源确认: <=500ms（弹窗打开后并行请求，结果到达后刷新标记）
- 锁定空地解锁动画: 350ms（锁链碎裂200ms + 虚线变金色150ms）
- 装饰化转换动画: 500ms（虚线框淡出200ms + 装饰图标fade-in 300ms）

**空间要点**:
- PC弹窗: 居中500x480px，遮罩rgba(0,0,0,0.6)，z-index:50
  - 标题栏: 40px高，"选择建造建筑"+搜索框(180px宽，带放大镜图标)+[×]关闭按钮(24x24px, 热区32x32px)
  - 筛选Tab栏: 36px高，[全部]/[可建造]/[未解锁]三个标签，活跃态金色下划线2px
  - 列表区: 可滚动，每条目80px高，最大高度=(480-40-36-48)=356px（可显示4.45个条目）
  - 底部信息: 48px高，显示当前资源概况（粮草/铜钱/矿石/木材 四行紧凑显示，每行=图标+数值+增减趋势箭头）
- 手机Bottom Sheet: 底部上滑65%高度=330px
  - 拖拽手柄: 40x4px居中，圆角2px，颜色rgba(200,169,110,0.3)
  - 标题: "选择建造建筑" 16px粗体 + 右上角[×]按钮28x28px
  - 搜索框: 全宽，高度32px，左侧放大镜图标，圆角8px
  - 筛选Tab: 紧凑横向标签24px高，活跃态金色下划线1px
  - 列表条目: 紧凑布局60px高，图标32x32px左侧
  - 关闭手势: 下拉>=80px触发关闭（velocity-based: 快速下拉>1px/ms即使<80px也触发），弹性回弹效果
- 手机Bottom Sheet行为:
  - 上滑: 可拖拽增加高度（最大90%），松手后吸附最近档位（65%/90%）
  - 下滑: 下拉>=80px或快速下滑触发关闭
  - 遮罩: 点击遮罩关闭

### FL-BLD-04-02 浏览与筛选可建造列表

> **来源**: [UI BLD-3 建造条目](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-3) / [PRD BLD-1 建筑依赖关系](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1)

**用户感知（see->act->feedback）**:
- **See**: 看到可建造建筑列表，每条显示图标+名称+核心功能描述、资源消耗（逐项标色）、解锁条件、操作按钮。顶部筛选Tab+搜索框。底部显示当前资源概况
- **Act**: 点击筛选Tab切换列表范围 / 输入搜索关键词过滤 / 点击资源不足条目展开详情+获取建议 / 点击[快速补齐]按钮 / 滚动浏览列表
- **Feedback**: 筛选切换时列表fade过渡（150ms） / 搜索输入时列表实时过滤（200ms防抖，防抖期间搜索框右侧显示旋转加载指示器12px） / 展开条目时详情区域下滑展开（200ms slide-down） / 快速补齐弹出确认对话框 / 滚动到列表底部时显示渐隐遮罩（底部20px，`linear-gradient(transparent, rgba(30,35,48,0.9))`）

**系统行为**:
1. 逐条渲染建造条目：图标(40x40)+名称+描述+资源消耗+解锁条件+操作按钮
2. 状态标记（三级）:
   - **可建造**（绿色`#27ae60`[建造]按钮）：解锁条件满足 + 资源充足 + 队列有空槽
   - **未解锁**（灰色`#666`锁定）：主城等级不满足解锁条件，显示"主城需Lv.X"
   - **资源不足**（橙色`#e67e22`）：解锁条件满足但资源不足或队列满，缺少项标红显示差额
3. 排序规则（默认，用户不可更改）:
   - 第一优先级：已解锁在前，未解锁在后
   - 第二优先级（已解锁内）：按建造费用升序（鼓励先建便宜建筑）
   - 第三优先级（未解锁内）：按所需主城等级升序（即将解锁的排前面）
4. 筛选Tab切换过滤列表：[全部]/[可建造]/[未解锁]
5. 搜索过滤：匹配建筑名或功能描述，不区分大小写，200ms防抖。搜索结果高亮匹配文字（金色`rgba(200,169,110,0.3)`背景）
6. 资源不足条目可点击展开（点击整个条目区域，不只是按钮）:
   - 显示每种缺少资源的确切差额（如"粮草 500/1000，缺少500"）
   - 显示获取建议（如"升级农田增加粮草产出"、"完成日常任务获取粮草奖励"）
   - **[快速补齐]**按钮：显示补齐所需元宝数（换算公式：1元宝=100粮草=80铜钱=60矿石=60木材），点击弹出确认对话框
7. 队列已满的条目也显示为橙色，标注"升级队列已满"而非资源不足，无[快速补齐]按钮

### FL-BLD-04-03 校验建造条件

> **来源**: [PRD BLD-1 建筑依赖关系](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1) / [PRD BLD-4 建筑联动](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-4)

#### FL-BLD-04-03a 前端资源预检（弹窗打开时）

**触发时机**: 空地点击时，弹窗打开前（与弹窗动画并行执行）

**系统行为**:
1. 读取本地缓存的当前资源快照 `{grain, copper, ore, wood}`
2. 读取建筑建造配置表（客户端缓存），获取各建筑Lv.0->Lv.1费用:
   - 农田: 粮草100 + 铜钱50（查PRD农田1->2数据，建造Lv.1费用等同于首次升级费用）
   - 市集: 粮草80 + 铜钱100
   - 矿场: 粮草100 + 铜钱50
   - 伐木场: 粮草100 + 铜钱50
   - 兵营: 粮草120 + 铜钱80
   - 工坊: 粮草200 + 铜钱300 + 矿石100
   - 书院: 粮草150 + 铜钱200
   - 医馆: 粮草100 + 铜钱150
   - 城墙: 粮草300 + 铜钱200 + 矿石100 + 木材100
   - 酒馆: 粮草300 + 铜钱200
3. 逐条计算:
   - 解锁条件满足? (主城等级 >= 要求等级)
   - 资源充足? (每种资源 >= 费用)
   - 队列有空槽? (已用槽位数 < 当前可用槽位数，槽位数由主城等级决定: Lv1-5=1, Lv6-10=2, Lv11-20=3, Lv21-30=4)
4. 标记每条目状态: 绿色(全通过)/灰色(解锁不满足)/橙色(解锁通过但资源或队列不满足)

**资源不足时的差额计算**:
```
每条资源差额 = max(0, 建造费用 - 当前持有)
总差额元宝 = ceil(粮草差额/100) + ceil(铜钱差额/80) + ceil(矿石差额/60) + ceil(木材差额/60)
```

**关键**: 预检结果为初步判定，弹窗打开后并行请求服务端确认（04-03b）。预检与服务端结果不一致时以服务端为准。预检完成时间 <50ms，不阻塞弹窗出现

#### FL-BLD-04-03b 服务端完整校验（建造按钮点击时）

**用户感知（see->act->feedback）**:
- **See**: 建造条目上的操作按钮显示当前状态（绿色可建造/灰色锁定/橙色资源不足）。按钮点击后立即变为加载态（旋转齿轮图标16px）
- **Act**: 点击绿色[建造]按钮（触发服务端校验）。双击防护：按钮立即禁用（loading态），300ms内忽略重复点击
- **Feedback**: 校验通过 → 按钮变为绿色勾标（200ms）→ 进入建造执行(04-04)；校验失败 → 按钮恢复原状态 + Toast显示失败原因（3秒） + 条目状态标记更新

**系统行为**:
1. 前端快速二次预检（<20ms）: 按钮点击时再次确认本地校验通过（防止弹窗打开期间资源变化）
   - 如果二次预检失败: 直接更新条目状态，不发服务端请求
2. 发送 `POST /api/buildings/build` 请求，携带:
   - `building_type`: 建筑类型ID
   - `grid_position`: 目标空地网格坐标 `{row, col}`
   - 请求头 `If-Match: {player_version}` （乐观锁）
3. 服务端校验6项条件（按优先级顺序，短路返回第一个失败项）:
   1. **版本一致**: player_version与服务端当前版本匹配（乐观锁，防止并发冲突）
   2. **全局唯一**: 该建筑类型尚未建造（防止并发重复建造）
   3. **主城等级**: 满足解锁条件（查PRD BLD-1建筑依赖表）
   4. **建造资源**: 足够（建造Lv.1费用=PRD中Lv.0->Lv.1费用）
   5. **队列槽位**: 升级队列有空槽（建造任务共享升级队列槽位，参照FL-BLD-06）
   6. **目标位置**: 目标空地网格坐标未被占用
4. 全部通过 → 执行建造原子操作(04-04)；任一失败 → 返回错误码+消息

**服务端校验响应格式**:
```json
// 成功
{ "success": true, "data": { "building_id": "xxx", "task_id": "yyy", "version": "new_version", "build_time": 15, "start_time": 1714896000 } }

// 失败
{ "success": false, "error": { "code": "BUILD_DUPLICATE", "message": "该建筑已建造" } }
```

**错误码表**:

| 错误码 | 含义 | 客户端处理 | 重试策略 |
|--------|------|-----------|---------|
| BUILD_VERSION_CONFLICT | 版本冲突 | 强制刷新玩家数据+重新计算条目状态 | L3: 强制刷新 |
| BUILD_DUPLICATE | 建筑已建造 | Toast+刷新列表移除该条目 | 不重试 |
| BUILD_LOCKED | 主城等级不足 | Toast"主城需Lv.X"+按钮变灰 | 不重试 |
| BUILD_RESOURCE | 资源不足 | Toast"XX不足，还需要YYYY"+按钮变橙 | 不重试 |
| BUILD_QUEUE_FULL | 队列已满 | Toast"升级队列已满"+建议加速或等待 | 不重试 |
| BUILD_POSITION_TAKEN | 目标位置已被占用 | Toast"位置已被占用"+建议选其他空地 | 不重试 |

**网络失败三级处理（R5）**:

| 级别 | 触发条件 | 处理方式 | 用户感知 |
|------|---------|---------|---------|
| L1 瞬时故障 | 网络超时(>3s) / 502/503 / 连接重置 | 自动重试3次，间隔1s/2s/3s（指数退避）。重试期间按钮保持loading态 | Toast"网络不稳定，正在重试..."(首次重试时显示) |
| L2 服务端错误 | 4xx业务错误 / 5xx服务错误(非502/503) | 显示具体错误码+错误消息+[重试]按钮。不自动重试 | 错误对话框（200x120px），含错误码+消息+[关闭]+[重试] |
| L3 版本冲突 | 409 Conflict | 强制刷新玩家数据`GET /api/player/state`，重新计算所有条目状态，Toast"数据已更新" | 全屏loading 500ms → 弹窗内列表刷新 |

**关键要求**: 校验优先级 版本一致->全局唯一->主城等级->资源->队列槽位->目标位置。建造费用等于PRD中该建筑首次升级费用（Lv.0->Lv.1）。网络失败三级处理正确执行

**时间要点**:
- 前端二次预检: <20ms
- 服务端校验: <100ms（正常）/ <3s（触发L1重试前）
- Toast显示时长: 3秒
- 按钮状态更新动画: 200ms
- L1自动重试间隔: 1s / 2s / 3s
- L3强制刷新: <=500ms

**空间要点**:
- Toast: 顶部居中，最大宽度320px，高40px，圆角8px
- 按钮loading态: 原按钮宽度不变，文字替换为旋转齿轮图标16px
- L2错误对话框: 居中200x120px，标题"建造失败"+错误码(灰色12px)+消息(14px)+[关闭][重试]双按钮
- 按钮状态更新: 颜色渐变200ms（绿->橙->灰三态渐变）

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 主城等级不足 | 按钮灰色锁定+红色条件提示"主城需Lv.X" | 04-02 |
| 资源不足 | 橙色按钮+缺少资源标红，Toast"XX不足，还需要YYYY" | 04-02 |
| 队列已满 | Toast"升级队列已满，请等待当前升级完成"，建议加速 | 04-02 |
| 建筑已建造（并发） | Toast"该建筑已建造"，刷新列表移除该条目，fade-out 200ms | 04-02 |
| 服务端校验与前端不一致 | 以服务端结果为准，前端刷新数据+列表标记+重新排序 | 04-02 |
| 版本冲突(409) | L3处理: 强制刷新玩家数据，重新计算所有条目状态，Toast"数据已更新" | 04-02 |
| 目标位置已被占用 | Toast"该位置已被占用"，刷新网格显示，建议选择其他空地 | 04-02 |
| 前端预检通过但服务端失败 | 以服务端为准。按钮更新为失败状态，Toast显示服务端错误信息 | 04-02 |
| 网络超时(>3s) | L1: 自动重试3次。3次均失败 → Toast"网络超时，请检查网络后重试"+[重试]按钮 | 04-02 |
| 请求发出后应用切换到后台 | 请求继续执行，结果到达后更新本地状态。用户返回时重新拉取最新数据 | 04-02 |
| 服务端返回5xx但非502/503 | L2: 显示错误码+消息+[重试]按钮，不自动重试 | 04-02 |

**触发方式**: 建造状态面板中点击[取消]按钮

**用户感知（see→act→feedback）**:
- **See**: 建造状态面板显示[取消]按钮（红色`#e74c3c`）
- **Act**: 点击[取消] → 弹出确认对话框
- **Feedback**: 确认取消 → Toast"XX建造已取消，返还XX%资源" → 空地恢复为可建造空地（动画: 进度条消失100ms + 锤子消失 + 虚线框恢复150ms）→ 队列任务移除 → 建造选择列表恢复该条目

**确认取消对话框（280x200px）**:
```
┌──── 取消建造 ─────────────────── [×] ┐
│                                         │
│  确定取消医馆建造吗？                    │
│                                         │
│  已消耗资源:                             │
│  🌾 100 / 100  💰 150 / 150            │
│                                         │
│  返还资源 (80%):                        │
│  🌾 80   💰 120                        │
│                                         │
│  ┌──────────┐  ┌──────────┐           │
│  │   继续建造 │  │  确认取消 │           │
│  └──────────┘  └──────────┘           │
└─────────────────────────────────────────┘
```

**系统行为**:
1. 弹出确认对话框，显示已消耗资源+返还资源明细（80%）
2. 用户确认取消 → 发送 `POST /api/buildings/cancel-build` 请求:
   - `building_id`: 建筑实例ID
   - `task_id`: 建造任务ID
   - 请求头 `If-Match: {player_version}` （乐观锁）
3. 服务端原子操作:
   - 校验建造任务存在且状态为active
   - 删除建筑实例
   - 移除队列任务
   - 返还80%资源（每种资源分别计算 `floor(消耗 × 0.8)`）
   - 更新player_version
4. 客户端:
   - 空地恢复为可建造状态（动画序列）
   - 队列悬浮窗移除该任务项
   - 资源栏数字增加动画（200ms）
   - 建造选择列表恢复该建筑条目（下次打开弹窗可见）

**关键要求**: 取消建造返还80%资源（与升级取消一致，参照FL-BLD-06-04）。资源返还逐项取整(`floor`)。建造取消后建筑实例被删除，可重新建造

**时间要点**:
- 取消请求: <=500ms
- 空地恢复动画: 250ms（进度条消失100ms + 虚线框恢复150ms）
- 资源返还动画: 200ms
- Toast显示: 3秒

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 网络请求失败 | 服务端事务回滚，建造继续，Toast"取消失败，请重试" | 04-04 |
| 请求超时（>3秒） | 建造继续，按钮恢复，Toast"网络超时"，后台轮询确认取消状态 | 04-04 |
| 队列在请求期间变满 | 不影响取消操作（取消释放槽位） | 04-04 |
| 建造在取消请求期间刚好完成 | 服务端返回"建造已完成"（BUILD_ALREADY_DONE），客户端执行04-05完成流程 | 04-05 |
| 版本冲突(409) | 强制刷新数据，确认建造状态后重新展示 | 04-04 |
| 建造时间极短（5s），取消时进度>80% | 允许取消（与PRD一致），但Toast额外提示"建造即将完成" | 04-04 |

### FL-BLD-04-06 关闭弹窗

> **来源**: [UI BLD-3 关闭方式](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-3)

**用户感知（see->act->feedback）**:
- **See**: 弹窗/Bottom Sheet处于打开状态，可见遮罩层和弹窗内容

- 状态恢复: 0ms（同步）

**空间要点**:
- PC关闭区域: [×]按钮24x24px右上角（热区32x32px）/ 遮罩层（弹窗外区域）/ ESC键盘快捷键

### FL-BLD-04-07 全部建造完成（装饰态转换）

> **触发条件**: 最后一座建筑建造完成（已建造建筑数 = 11）
> **来源**: [PRD BLD-1 11座建筑总览](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1-1)

**用户感知（see→act→feedback）**:
- **See**: 最后一座建筑出生动画完成 → 全屏幕金色光芒闪烁（200ms，从建筑位置辐射全屏）→ 所有剩余空地同时播放装饰化转换动画 → Toast"所有建筑已建造完成！"（金色边框特殊Toast，3秒）
- **Act**: 无需操作（自动触发）。装饰空地不可交互
- **Feedback**: 所有空地同时变为装饰态（虚线框淡出200ms + 景观图标fade-in 300ms），之后每隔60秒（+0-10s随机抖动）随机一个装饰空地播放环境动画

**装饰化转换动画序列**（所有空地同时执行，总时长500ms）:
1. 空地虚线框闪烁金色（100ms）
2. 虚线框淡出（200ms）
3. 景观图标从中心fade-in+scale(0.5→1)（300ms，与步骤2有100ms重叠）
4. 淡金色细线框fade-in（150ms）
5. 微弱光泽脉冲开始（4s循环，持续）

**系统行为**:
1. 最后一座建筑建造完成时:
   - 设置全局标记 `all_buildings_completed = true`
   - 触发装饰化转换（客户端本地执行，不需要服务端确认）
2. 计算剩余空地数量（36格 - 11建筑 = 25个装饰空地）
3. 为每个装饰空地随机分配景观图标（从图标池不重复分配，池中有10种图标，25个空地循环使用，相邻空地不重复）
4. 后续环境动画调度:
   - 启动定时器，每60秒（+0-10s随机抖动）随机选择一个装饰空地
   - 播放随机环境动画（见下表）
   - 同一时间只有一个装饰空地播放环境动画

**景观图标池**（10种，不重复分配，相邻空地不同图标）:
花丛、石灯笼、小池塘、竹林、假山、梅树、荷花盆、石阶、松树盆景、棋盘石凳

**环境动画池**:

| 动画 | 描述 | 持续时间 |
|------|------|:-------:|
| 蝴蝶飞过 | 一只蝴蝶从左侧飞入，经过装饰空地后从右侧飞出，贝塞尔曲线路径 | 3s |
| 落叶飘落 | 3-5片叶子从上方飘落，左右摇摆，到达底部消失 | 3s |
| 萤火虫 | 5-8个发光粒子在装饰空地周围缓慢飘动，明暗交替 | 3s |
| 花瓣飘散 | 3-5片花瓣从中心向外飘散 | 2.5s |
| 微光闪烁 | 装饰图标上方出现一道微弱金色光芒，闪烁2次 | 2s |

**装饰空地交互规则**: cursor:default，无点击事件，无Tooltip，无悬停效果，不响应任何触摸/点击操作

**手机端差异**:
- 装饰化动画简化（仅fade-in，无scale动画）
- 环境动画频率降为120秒（性能优化）
- 建筑列表中装饰空地显示为紧凑装饰条目（355x40px，图标20x20px + 微光脉冲）

**关键要求**: 装饰化转换在最后一座建筑建造完成后立即触发。景观图标不重复分配（相邻空地不同图标）。环境动画定时触发，同一时间只有一个。不可逆转换

**时间要点**:
- 装饰化转换动画: 500ms（与最后一座建筑出生动画串行执行）
- 环境动画间隔: 60秒 + 0-10秒随机抖动（PC）/ 120秒（手机）
- 环境动画时长: 2-3秒
- 全部完成Toast: 3秒（金色边框特殊样式）

**空间要点**:
- 装饰空地: 180x120px格子（PC）/ 355x40px紧凑条目（手机列表）
- 景观图标: 32x32px居中（PC）/ 20x20px（手机）
- 淡金色细线框: `1px solid rgba(200,169,110,0.15)`
- 微弱光泽脉冲: opacity 0.3→0.6→0.3, 4s循环
- 全部完成Toast: 顶部居中，金色边框`1px solid #c8a96e`，背景`rgba(200,169,110,0.15)`

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 装饰图标资源加载失败 | 显示通用装饰图标（白色小方块16x16px），后台静默重试 | -- |
| 装饰化动画期间用户操作 | 动画不可中断，操作排队到动画完成后执行 | 02-03 |
| 手机端装饰化动画性能问题 | 自动降级: 仅fade-in无scale，环境动画频率降为120秒 | -- |
| 建筑被拆除后重新建造 | 不处理（PRD无拆除机制，防御性设计） | -- |
