## 关键要求

- 8条商路（洛阳/长安/许昌/建业等），按市舶司等级逐步开通
- 商队数量随等级增长（Lv1-5: 2支, Lv6-10: 3支, Lv11+: 更多）
- 10种商品价格6小时刷新，存在地区价差
- 贸易事件（8种）商队运输中随机触发，选项不预显结果
- 繁荣度4等级（萧条/平稳/繁荣/鼎盛），影响产出倍率+NPC商人解锁
- 护卫武将武力>=80可自动处理事件

### R2 新增约束

- **贸易事件超时**: 事件等待超时30分钟。超时后自动选择默认选项（"绕道"/"撤退"等保守策略）。护卫武力>=80时事件触发后立即自动处理（不等超时）
- **贸易事件触发规则**: 每趟商路随机1-3个事件。触发后战斗结果计算: `护卫武力总值 vs 敌人规模阈值`，武力>=阈值=胜利
- **NPC商人刷新规则**: 同时最多出现1个NPC商人。刷新间隔1-4小时（繁荣度越高间隔越短）。繁荣度等级→解锁NPC类型: 萧条(普通商人) / 平稳(行商+普通) / 繁荣(行商+西域商人) / 鼎盛(所有类型)
- **派遣商队手机端适配**: 3步流程使用分步引导(wizard形式)+步骤指示器(1/3, 2/3, 3/3)，手机端切换为全屏模式
- **双击防护**: 所有操作按钮在请求/动画期间禁用
- **网络失败回滚**: 服务端事务机制，购买/派遣失败时已扣资源自动返还

## 验收标准

- 商路列表正确显示开通状态和等级要求
- 商品价格实时显示，6小时刷新
- 商队派遣流程完整（选商路→选商品→选护卫→确认）
- 贸易事件选项不预显结果数值
- 繁荣度等级正确影响NPC商人出现

### FL-BLD-09-05 派遣商队

> **来源**: [UI 商路派遣流程](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2-1)

**用户感知**: 点击[派遣商队] → 展开派遣流程面板（手机端: 全屏分步引导wizard） → 3步(含步骤指示器1/3,2/3,3/3): (1)选择商路 (2)选择商品（+-数量，显示载重和利润预测） (3)选择护卫武将 → 确认派遣
**系统行为**: 校验商队是否空闲 → 计算商品买卖价差 → 计算护卫降低的遇盗风险 → 创建商队派遣
**关键要求**: 利润预测=卖出价-买入价（不含事件影响）；护卫降低遇盗风险百分比。PC端在Tab内展开，手机端切换为全屏模式
**验收标准**: 派遣流程3步完整，利润计算正确

**⏱ 时间要点**: 派遣请求 <= 500ms
**📱 空间要点**: 在商路Tab内展开，[←返回]按钮回到商路列表

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 无空闲商队 | Toast"所有商队均在运输中" | 09-04 |
| 载重为0（未选商品） | [确认派遣]按钮禁用 | 09-05 |
| 铜钱不足购买商品 | 对应商品数量限制，Toast提示 | 09-05 |

### FL-BLD-09-07 贸易事件处理

> **来源**: [UI 贸易事件处理](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2-1) / [PRD 贸易事件设计原则](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-6)

**用户感知**: 弹出贸易事件面板 → 显示事件类型（山贼/暴雨/关税等）+当前货物+护卫信息 → 显示处理选项（2-3个），选项只描述行为，不显示具体结果数值 → 选择处理方式 或 让护卫自动处理
**系统行为**: 根据武将属性+敌人规模+随机因素计算结果 → 记录选择但不预显结果
**关键要求**:
  - **选项不预显结果数值**——只描述行为（如"正面迎战"），不显示"损失10%货物"
  - **事件超时30分钟**: 超时后自动选择默认保守选项（绕道/撤退）
  - **护卫武力>=80立即自动处理**: 不等超时，事件触发后自动选择最优选项
  - **每趟商路随机1-3个事件**
**验收标准**: 选项不包含数值预览

**⏱ 时间要点**: 事件需玩家手动选择（除非护卫可自动处理）

### FL-BLD-09-08 事件结果展示

> **来源**: [UI 事件结果](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2-1) / [PRD 结果判定规则](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-6)

**用户感知**: 显示事件结果面板 → 战斗胜利：获得战利品+繁荣度+武将经验 → 战斗失败：货物损失+时间延迟+武将受伤 → 缴费通过：铜钱扣除 → 绕道成功：无损失 / 绕道失败：迷路延迟
**系统行为**: 根据选项类型+武将属性计算胜负 → 结算奖惩 → 更新繁荣度 → 武将受伤状态（如有）
**关键要求**: 结果在执行后才揭晓。战斗结果计算公式: `护卫武力总值 >= 敌人规模阈值 → 胜利`
**验收标准**: 结果计算逻辑正确，奖惩合理

### FL-BLD-09-09 商队到达结算

> **来源**: [PRD 市舶司核心功能](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-6)

**用户感知**: 商队到达 → Toast"商队已到达XX，利润💰YY" → 商队状态变为空闲 → 利润入账
**系统行为**: 结算商品交易（卖出价格-买入价格=利润） → 增加繁荣度 → 更新商品价格 → 商队回归空闲
**关键要求**: 利润 = 实际卖出价 x 数量 - 实际买入价 x 数量
**验收标准**: 利润计算正确

**⏱ 时间要点**: 到达时自动结算

### FL-BLD-09-10 NPC商人交互

> **来源**: [PRD 市舶司 NPC商人](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-6)

**用户感知**: 点击[NPC商人] → 显示当前出现的NPC商人列表 → 每种商人有不同商品和折扣 → 点击商人查看商品 → 购买
**系统行为**: 根据繁荣度等级决定出现的NPC类型 → 渲染限时商品 → 处理购买
**关键要求**: NPC商人限时出现（显示倒计时），繁荣度越高解锁越高级商人。同时最多1个NPC商人。刷新间隔1-4小时。繁荣度→NPC类型映射:
  - 萧条(0-24): 普通商人
  - 平稳(25-49): 普通商人 + 行商
  - 繁荣(50-74): 行商 + 西域商人
  - 鼎盛(75-100): 所有类型
**验收标准**: NPC商人正确出现，商品可购买

**⏱ 时间要点**: NPC商人限时出现（通常1-4小时）

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 当前无NPC商人 | Toast"暂无商人出现" | 09-01 |
| 购买资源不足 | Toast提示缺少资源 | 09-10 |
