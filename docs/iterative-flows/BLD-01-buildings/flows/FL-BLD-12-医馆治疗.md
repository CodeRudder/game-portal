# FL-BLD-12 医馆治疗

> **版本**: v2 | **最后更新**: 2026-05-05

> **结束条件**: 查看完毕或执行治疗操作
> **前置状态**: 医馆已建造且等级 >= 1
> **后置状态**: 治疗已执行（伤兵恢复+产出Buff），或仅查看

## 关键要求

- 医馆为简单建筑（无Tab），概览+互动合并显示
- 伤兵由战斗系统产生，自动恢复，恢复速率受医馆等级+科技+武将影响
- 主动治疗：消耗粮草（10%日产）立即恢复10%伤兵，冷却30分钟
- 治疗后获得Buff：+10%产出持续10分钟
- 恢复的伤兵回到兵营可用兵力池

### R2 新增约束

- **"10%日产"精确定义**: 日产 = 所有农田当前产出速率（含加成，不含临时Buff） × 86400秒。使用治疗操作开始时的快照值计算
- **Buff叠加规则**: Buff不叠加。重复治疗刷新10分钟计时（不延长总时长，不叠加效果）。即始终维持"+10%产出，剩余时间<=10分钟"
- **双击防护**: [立即治疗]按钮点击后立即禁用，服务端确认完成后恢复

## 验收标准

- 伤兵数量和恢复速度准确显示
- 主动治疗正确执行（粮草扣除+伤兵恢复）
- 治疗Buff正确生效和倒计时
- 冷却时间正确运作

## 步骤详情

### FL-BLD-12-01 查看伤兵状态

> **来源**: [UI 医馆详情面板](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2-1) / [PRD BLD-2 医馆互动设计](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-2)

**用户感知**: 在医馆面板看到 → 当前伤兵数/最大容量（进度条） + 恢复速度（/分钟，含加成明细） + 预计全部恢复时间
**系统行为**: 从战斗系统获取伤兵数据 → 计算恢复速率（医馆等级+科技"急救术"+武将政治） → 实时更新
**关键要求**: 无伤兵时显示"当前无伤兵"，主动治疗区域隐藏
**验收标准**: 伤兵数据和恢复速率准确

**📱 空间要点**: 详情面板"伤兵状态"区块

### FL-BLD-12-02 查看恢复进度

> **来源**: [PRD BLD-2 医馆互动设计](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-2)

**用户感知**: 伤兵数量实时减少 → 进度条推进 → 恢复速度可展开查看加成来源
**系统行为**: 持续恢复伤兵（每分钟） → 恢复的兵力回流到兵营 → 更新显示
**关键要求**: 恢复速率动态显示加成来源
**验收标准**: 伤兵数量实时准确减少

**⏱ 时间要点**: 持续恢复，预计全部恢复时间显示

---

### FL-BLD-12-03 主动治疗

> **来源**: [PRD BLD-2 主动治疗](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-2) / [UI 医馆主动治疗](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2-1)

**用户感知**: 看到[立即治疗]按钮+治疗说明（恢复10%伤兵+消耗粮草(10%日产)+冷却30分钟） → 点击 → 按钮立即禁用 → 扣除粮草 → 伤兵数立即减少 → Toast"治疗成功，恢复XX名伤兵"
**系统行为**: 校验粮草和冷却 → 扣除粮草（10%日产 = 所有农田产出速率含加成不含Buff × 86400秒 × 10%，使用快照值） → 恢复10%伤兵 → 刷新/触发治疗Buff → 开始30分钟冷却
**关键要求**: 消耗=10%日产量粮草（快照值）；恢复10%当前伤兵数；Buff不叠加仅刷新计时
**验收标准**: 粮草扣除正确，伤兵恢复正确，冷却开始计时

**⏱ 时间要点**: 冷却30分钟

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 冷却中 | Toast"治疗冷却中，剩余XX:XX" + 按钮禁用显示倒计时 | 12-01 |
| 粮草不足 | Toast"粮草不足，需要XX" | 12-01 |
| 无伤兵 | [立即治疗]按钮隐藏 | 12-01 |

### FL-BLD-12-04 治疗Buff生效

> **来源**: [PRD BLD-2 治疗Buff](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-2) / [UI Buff状态](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2-1)

**用户感知**: 治疗后面板显示"Buff: +10%产出，剩余X:XX" → 所有建筑产出临时提升 → 倒计时归零后消失
**系统行为**: 触发全局产出+10% Buff → 持续10分钟 → 倒计时结束移除Buff。若已有Buff在生效中，重新治疗仅刷新10分钟计时（不叠加，不延长总时长超过10分钟）
**关键要求**: Buff影响所有建筑产出。Buff不叠加
**验收标准**: 产出临时提升，倒计时准确

**⏱ 时间要点**: Buff持续10分钟
