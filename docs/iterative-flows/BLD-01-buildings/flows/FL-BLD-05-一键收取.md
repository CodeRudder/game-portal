# FL-BLD-05 一键收取

> **版本**: v3 | **最后更新**: 2026-05-05
> **归属功能**: 建筑系统 — 资源管理 (Phase C)
> **触发条件**: 玩家点击右下角一键收取悬浮按钮
> **结束条件**: 收取动画完成，资源入账
> **前置状态**: 至少1座建筑有累积产出
> **后置状态**: 所有建筑累积产出已清零，资源已加到玩家账户
> **来源文档**: [PRD BLD-5 一键收取](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-5)

## 关键要求

- 右下角悬浮按钮，始终可见（非弹窗/面板内）
- 收取动画：资源数字从各建筑飞向顶部资源栏
- VIP3+解锁离线自动收取（无需手动点击）
- 无产出可收时按钮置灰

### R2 新增约束

- **双击防护**: 点击收取按钮后立即禁用（加载态），动画期间不可重新触发，服务端确认完成后恢复
- **动画并发策略**: 11组飞行动画可并发播放，总时长 <= 1.5秒。低端设备降级为瞬间入账（无飞行动画）
- **气泡数字定义**: 气泡显示各资源实际收取量（🌾XX 💰YY ⛏️ZZ 🪓WW），Tooltip中展示折算比率（1矿石=2铜钱, 1粮草=1铜钱, 1木材=1.5铜钱）
- **资源存储上限**: 各资源存储上限 = 该资源当前产出速率 x 86400秒（24小时产量）。溢出行为: 正常收取但超出上限部分自动损耗，Toast"库存已满，溢出资源已损耗"
- **资源溢出处理**: 服务端收取时检查上限，上限内全额入账，超出部分损耗并提示

### R3 新增约束

- **收取动画精确时序**: 点击收取后动画时序：(1)按钮禁用0ms → (2)汇总各建筑产出<50ms → (3)飞行动画开始（各建筑卡片发出资源图标，沿贝塞尔曲线路径飞向顶部资源栏，前200ms加速+后800ms减速，总计<=1.5秒）→ (4)资源栏数字跳动更新（在飞行结束时+100ms触发）→ (5)Toast出现 → (6)按钮恢复
- **资源溢出详细处理**: 收取时若任一资源超出存储上限（产出速率x86400秒），服务端返回`{accepted: amount, wasted: amount}`，客户端在Toast中显示"XX资源溢出，损耗YYYY"（该资源行显示橙色损耗数字）
- **VIP3+自动收取触发**: VIP3+玩家每次上线时，服务端自动结算离线期间的资源产出，客户端显示"离线收益"弹窗（显示离线时长+各资源收益+自动收取状态）。自动收取不播放飞行动画，直接数字跳动+Toast
- **按钮气泡更新频率**: 气泡数字每秒更新一次（本地计算累积产出），服务端每30秒校准一次。若累积产出为0则隐藏气泡
- **零产出建筑排除**: 没有产出功能的建筑（如主城）不参与收取流程，飞行动画中不出现其位置

## 验收标准

- 收取动画流畅，资源数字飞向对应资源图标
- 收取后各建筑累积产出归零
- 资源栏数值即时更新

## 步骤详情

### FL-BLD-05-01 查看收取按钮状态

> **来源**: [PRD BLD-5 一键收取](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-5) / [PRD BLD-3 资源产出](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-3)

**用户感知（see→act→feedback）**:
- **See**: 右下角悬浮按钮显示当前可收取状态 → 有产出时按钮高亮（金色脉冲边框）+数字气泡（显示各资源实际数量🌾XX 💰YY ⛏️ZZ 🪓WW）→ 无产出时按钮置灰（灰色，opacity:0.5）
- **Act**: 视线关注按钮状态（PC端悬停显示Tooltip含折算比率+铜钱等价总额）
- **Feedback**: 气泡数字每秒更新（资源累积实时递增），Tooltip悬停后200ms出现

**系统行为**:
1. 每秒本地计算所有产出建筑累积产出（粮草+铜钱+矿石+木材分别累加）
2. 更新按钮状态：有产出→高亮可点击（金色脉冲边框#c8a96e，1.5s周期）/ 无产出→置灰不可点击
3. 更新气泡数字：各资源类型显示实际累积数量
4. PC端悬停显示Tooltip：折算比率（1矿石=2铜钱, 1粮草=1铜钱, 1木材=1.5铜钱）+ 铜钱等价总额
5. 每30秒服务端校准一次累积产出

**关键要求**: 气泡数字显示各资源实际收取量。Tooltip展示折算比率和铜钱等价总额。零产出建筑（如主城）不计入

**⏱ 时间要点**:
- 气泡数字更新: 每秒1次（本地计算）
- 服务端校准: 每30秒1次
- Tooltip延迟: 200ms（悬停触发）
- 按钮脉冲动画周期: 1.5s

**📱 空间要点**:
- PC: 右下角悬浮按钮56x56px，距底部12px距右侧12px，z-index:40，不遮挡建筑网格（位于网格区域外右下角）
- 气泡: 按钮左上方弹出，宽度自适应（最大160px），高度24px，背景rgba(39,174,96,0.9)，圆角12px
- 手机: 右下角悬浮按钮48x48px，距底部16px距右侧16px，气泡更紧凑
- Tooltip: 最大宽度200px，高度自适应，显示折算比率+铜钱等价总额

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 本地累积与服务端不同步 | 以服务端数据为准，气泡数字可能跳变 | 05-01 |
| 所有建筑产出为0 | 按钮置灰不可点击，气泡隐藏 | 05-01 |
| 部分资源接近存储上限 | 气泡中对应资源数字显示橙色警告色 | 05-01 |

### FL-BLD-05-02 点击一键收取

> **来源**: [PRD BLD-5 一键收取](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-5)

**用户感知（see→act→feedback）**:
- **See**: 右下角悬浮按钮高亮（金色脉冲），气泡显示各资源累积量
- **Act**: 点击一键收取按钮（触摸目标>=44px），双击防护立即生效
- **Feedback**: 按钮立即禁用（opacity:0.6+旋转齿轮图标）→ 按下scale缩放（90% 100ms）→ 服务端响应后触发飞行动画

**系统行为**:
1. 按钮立即禁用（双击防护），记录当前各建筑位置坐标（用于飞行动画起点）
2. 汇总所有产出建筑累积产出，按资源类型累加
3. 发送`POST /api/buildings/collect`请求（原子操作）
4. 服务端：检查存储上限 → 上限内全额入账 → 超出部分损耗 → 返回结果（各资源accepted+wasted）
5. 客户端收到响应后触发飞行动画(05-03)

**关键要求**: 原子操作——要么全部收取成功，要么全部不收。资源溢出时正常入账上限内部分，超出损耗并提示

**⏱ 时间要点**:
- 按钮禁用: 立即（0ms）
- 按下动画: 100ms
- 服务端响应: <=500ms
- 动画期间按钮保持禁用直到动画完成

**📱 空间要点**:
- 按钮禁用态: opacity:0.6，内显旋转齿轮图标16x16px
- 按下缩放: scale 90%

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 无产出可收 | 按钮置灰不可点击（05-01已处理） | 05-01 |
| 网络请求失败 | Toast"收取失败，请重试"（3秒），按钮恢复可用 | 05-01 |
| 请求超时（>3秒） | 按钮恢复，Toast"网络超时，请重试"，后台轮询确认 | 05-01 |
| 服务端返回产出为0（并发已收） | Toast"资源已被收取"（2秒），按钮置灰，气泡隐藏 | 05-01 |

### FL-BLD-05-03 收取动画播放

> **来源**: [PRD BLD-5 收取动画](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-5) / [PRD BLD-3 资源产出](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-3)

**用户感知（see→act→feedback）**:
- **See**: 各产出建筑卡片位置飞出资源图标（🌾/💰/⛏️/🪓），带数字标签
- **Act**: 无需操作（动画自动播放），动画期间可进行其他操作（不阻塞UI）
- **Feedback**: 资源图标沿贝塞尔曲线路径飞向顶部资源栏对应位置 → 飞行过程中图标从20px放大到28px再缩小到16px → 到达目标时产生小型爆发效果（白色粒子散开，200ms）

**系统行为**:
1. 计算每个产出建筑卡片中心坐标（动画起点）和顶部资源栏对应图标坐标（动画终点）
2. 为每种资源创建飞行动画实例（最多4条路径：粮草/铜钱/矿石/木材）
3. 每条路径上可能有多个建筑（如农田产粮草+矿场也产出），图标按建筑位置错开出发（每个间隔80ms）
4. 飞行路径：贝塞尔曲线（起点→中间控制点偏上→终点），总时长1000ms（前200ms加速+后800ms减速）
5. 低端设备降级：跳过飞行动画，直接显示数字跳动更新

**关键要求**: 动画总时长<=1.5秒。11组飞行动画可并发播放（总计不超过44个飞行图标：11建筑x4资源，但通常远少于此）。低端设备降级为瞬间入账

**⏱ 时间要点**:
- 首个图标出发: 响应到达后50ms
- 每组图标间隔: 80ms
- 单个图标飞行时长: 1000ms（贝塞尔曲线，ease-out）
- 全部图标飞行完成: <=1500ms
- 到达爆发效果: 200ms
- 低端设备降级: 0ms（瞬间入账）

**📱 空间要点**:
- 动画起点: 各产出建筑卡片中心坐标（如农田位于第2行第1列，中心=(96, 228)px）
- 动画终点: 顶部资源栏对应资源图标位置（粮草图标x=100, 铜钱x=200, 矿石x=300, 木材x=400）
- 飞行图标: 20px→28px→16px缩放，opacity 1→0.8
- 中间控制点: 起点y-100px + x偏移（曲线路径不重叠）
- 到达爆发效果: 6个白色粒子从终点扩散，半径8px→24px，200ms

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 低端设备（帧率<20fps） | 检测帧率，低于20fps时跳过飞行动画，直接数字跳动更新 | 05-04 |
| 建筑卡片不在可视区域 | 飞行起点从网格区域外对应位置出发（屏幕外坐标） | 05-03 |
| 资源栏位置动态变化 | 每次动画开始前重新获取资源栏图标坐标 | 05-03 |

### FL-BLD-05-04 资源入账确认

> **来源**: [PRD BLD-5 一键收取](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-5) / [PRD BLD-3 资源产出](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-3)

**用户感知（see→act→feedback）**:
- **See**: 飞行动画最后一个图标到达终点 → 顶部资源栏数字跳动更新（scale 1.0→1.15→1.0, 200ms递增动画）→ 各建筑卡片产出累积值归零 → Toast出现
- **Act**: 无需操作（自动完成）
- **Feedback**: 资源栏每种资源数字递增动画（200ms）→ Toast"收取成功！获得🌾XX 💰YY ⛏️ZZ 🪓WW"→ 若有溢出则Toast额外显示"XX库存已满，溢出YYYY已损耗"（橙色文字）

**系统行为**:
1. 飞行动画完成后触发资源入账逻辑
2. 资源栏数字递增动画：旧值→新值，200ms动画
3. 清零各产出建筑累积产出
4. 按钮恢复可用（若仍有产出则重新高亮，若产出为0则置灰）
5. 气泡数字更新（归零后重新开始计算）
6. 显示Toast（含各资源明细+溢出警告）

**关键要求**: Toast显示本次收取的各资源明细。溢出时显示橙色损耗提示。资源数值准确，累积产出归零

**⏱ 时间要点**:
- 资源栏数字跳动: 200ms
- Toast显示: 3秒（溢出提示延长至5秒）
- 按钮恢复: 动画完成后100ms
- 气泡重置: 0ms（立即归零重新计算）

**📱 空间要点**:
- 资源栏数字跳动: scale 1.0→1.15→1.0，font-weight临时加粗，200ms
- 成功Toast: 顶部居中，背景rgba(39,174,96,0.15)，绿色调，高度自适应（含资源明细行）
- 溢出Toast: 同上，但溢出资源行使用橙色文字（#e67e22），显示"溢出YYYY已损耗"
- 各建筑卡片累积值: 归零动画（数字从当前值递减到0，200ms）

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 入账数据与服务端不一致 | 以服务端返回数据为准，资源栏数字可能跳变 | 05-04 |
| 全部资源溢出 | Toast"所有资源库存已满，本次收取全部损耗"，按钮保持可点击（产出持续累积） | 05-01 |
| VIP3+离线自动收取 | 不播放飞行动画，显示"离线收益"弹窗（离线时长+各资源收益），点击[确认]关闭弹窗 | 05-01 |
