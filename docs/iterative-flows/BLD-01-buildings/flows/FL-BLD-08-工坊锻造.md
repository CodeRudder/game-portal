## 关键要求

- 锻造消耗矿石+木材+铜钱，工坊等级提供效率加成（减少实际消耗）
- 装备品质上限受工坊等级限制：白/绿(Lv1-5) → 蓝(Lv6-10) → 紫(Lv11-15) → 橙(Lv16-20)
- 3种锻造模式：快速(少材料/白绿) / 精炼(多材料/蓝紫) / 神炼(Lv10解锁/紫橙)
- 装备分解回收矿石（回收率30%-60%，受工坊等级影响）
- Lv10+解锁批量锻造

### R2 新增约束

- **锻造效率公式**: 效率折扣统一适用于快速/精炼/神炼三种模式。同时减少材料和铜钱消耗
  ```
  actual_cost = base_cost × (1 - workshop_efficiency% - tech_bonus%)
  ```
  示例: 效率+21.6%, 材料 base=x10 → actual = 10 × (1-0.216) = 7.84, 向上取整为x8
- **装备属性生成规则（最小定义）**:
  - 品质 → 主属性倍率: 白x1.0 / 绿x1.2 / 蓝x1.5 / 紫x2.0 / 橙x3.0
  - 品质 → 副属性条数: 白0 / 绿1 / 蓝2 / 紫3 / 橙4
  - 副属性类型从装备部位对应属性池中随机抽取（如武器池: 暴击/穿透/攻速）
  - 强化等级上限: +15
- **双击防护**: 锻造/强化/分解按钮点击后立即禁用，操作完成或失败后恢复
- **网络失败回滚**: 服务端事务机制，锻造/强化失败时已扣资源自动返还

### R7 深度约束

- **装备部位主属性映射**: 武器→攻击力 / 防具→防御力 / 饰品→生命值 / 坐骑→速度
- **锻造模式材料基准** (不含效率折扣):
  - 快速: 矿石x10 + 木材x5 + 铜钱x200
  - 精炼: 矿石x30 + 木材x15 + 铜钱x800
  - 神炼: 矿石x100 + 木材x50 + 铜钱x3000
- **品质概率表** (按锻造模式):
  - 快速: 白60% / 绿30% / 蓝10% / 紫0% / 橙0%
  - 精炼: 白10% / 绿20% / 蓝50% / 紫20% / 橙0%
  - 神炼: 白0% / 绿0% / 蓝20% / 紫50% / 橙30%
- **工坊等级品质上限锁**: 品质概率表中超出上限的品质概率重分配给该段最高品质
  - Lv1-5上限绿: 重分配后快速=白70%/绿30%, 精炼=白20%/绿80%
  - Lv6-10上限蓝: 快速=白60%/绿30%/蓝10%, 精炼=白10%/绿20%/蓝70%
  - Lv11-15上限紫: 精炼=白10%/绿20%/蓝50%/紫20%, 神炼=蓝30%/紫50%/橙20%(橙30→紫)
  - Lv16-20上限橙: 全品质概率正常
- **强化消耗公式**:
  ```
  enhance_cost = base_ore × (1 + 0.3 × current_level) × (1 - workshop_discount% - tech_discount%)
  ```
  base_ore = 品质基础值(白10/绿20/蓝40/紫80/橙160)
  workshop_discount: PRD工坊等级表 强化折扣 -3%~-25%
- **分解回收公式**:
  ```
  recycle_rate = base_rate% + workshop_bonus%
  base_rate = 30% (固定)
  workshop_bonus = 工坊等级 × 1.5% (Lv1=1.5%, Lv20=30%, 上限60%)
  ```
  回收物品 = 原始消耗材料 × recycle_rate (向下取整)
- **批量锻造**: x5/x10可选，每次独立随机，汇总结果面板按品质分类统计
- **材料流来源**: 矿场持续产出矿石 / 伐木场持续产出木材 → 工坊消费端

## 验收标准

- 锻造消耗正确（含效率加成，向上取整）
- 锻造产出的装备品质符合当前工坊等级上限（概率表+品质锁正确）
- 装备属性词条随机生成（主属性固定类型+副属性随机池）
- 强化消耗正确（含工坊折扣）
- 分解回收率正确（30%~60%受工坊等级影响）
- 批量锻造结果逐一展示+品质汇总
- 材料库存实时显示（矿场/伐木场产出→工坊消费链路可见）
- 装备卡片展示完整属性+可跳转英雄系统穿戴

**L3+ 系统行为详述**:

| 行为 | 触发时机 | 计算逻辑 |
|------|---------|---------|
| 计算效率折扣 | 面板初始化/部位切换 | `efficiency = workshop_base% + tech_冶炼工艺% + hero_武力%` |
| 计算实际消耗 | 面板初始化/部位切换 | `actual = ceil(base × (1 - efficiency))` 对三种资源独立计算 |

**L3+ 品质抽取算法详述**:

```
输入: forging_mode, workshop_level
输出: quality (白/绿/蓝/紫/橙)

步骤:
1. 确定 workshop_level 对应的品质上限 Q_max
   Lv1-5 → Q_max = 绿
   Lv6-10 → Q_max = 蓝
   Lv11-15 → Q_max = 紫
   Lv16-20 → Q_max = 橙

2. 从 forging_mode 的品质概率表查出 P = {白:pw, 绿:pg, 蓝:pb, 紫:pp, 橙:po}
   (注意: 概率表已经过品质锁重分配，超出Q_max的概率已分给Q_max)

3. 随机数 R = random(0, 100)
   if R < pw → 白
   else if R < pw+pg → 绿
   else if R < pw+pg+pb → 蓝
   else if R < pw+pg+pb+pp → 紫
   else → 橙

4. 验证: quality <= Q_max (防御性检查，确保不会超限)
```

**L3+ 属性生成算法详述**:

```
输入: quality, slot (部位)
输出: { main_stat, sub_stats[], special_effect? }

步骤1: 主属性
  main_type = 部位主属性映射[slot]  // 武器→攻击力 等
  main_value = slot_base_value[slot] × quality_multiplier[quality]
  quality_multiplier: { 白:1.0, 绿:1.2, 蓝:1.5, 紫:2.0, 橙:3.0 }
  slot_base_value: { 武器:50, 防具:40, 饰品:200, 坐骑:10 }
  main_value = round(main_value)  // 四舍五入取整

步骤2: 副属性
  sub_count = quality_sub_count[quality]  // { 白:0, 绿:1, 蓝:2, 紫:3, 橙:4 }
  sub_pool = slot_sub_pool[slot]  // 武器池: [暴击/穿透/攻速/吸血/命中]
  从 sub_pool 中无放回随机抽取 sub_count 个属性
  每个副属性值 = base × random(0.8, 1.2) × quality_multiplier × 0.3
  // 副属性约为主属性的30%，有±20%随机波动

步骤3: 特殊词条 (仅紫色和橙色)
  if quality >= 紫:
    从 slot_special_pool[slot] 中随机抽取1个特殊词条
    武器特殊池: [连击/破甲/嗜血/致命一击]
    防具特殊池: [铁壁/格挡精通/荆棘/不屈]
    饰品特殊池: [生命汲取/护盾强化/再生/不死]
    坐骑特殊池: [先发制人/追击/疾行/鼓舞]
    橙色额外: 特殊词条效果+50%
```

```
上限: 60%

示例:
  Lv1:  30% + 1.5% = 31.5%
  Lv5:  30% + 7.5% = 37.5%
  Lv7:  30% + 10.5% = 40.5%
  Lv10: 30% + 15% = 45%
  Lv15: 30% + 22.5% = 52.5%
  Lv20: 30% + 30% = 60% (上限)

回收物品 = 向下取整(锻造该装备时的原始材料消耗 × recycle_rate)
  回收矿石 = floor(original_ore_cost × recycle_rate)
  回收木材 = floor(original_wood_cost × recycle_rate)
  回收铜钱 = floor(original_gold_cost × recycle_rate)

强化过的装备额外回收:
  强化返还 = floor(累计消耗矿石 × recycle_rate × 0.5)
  // 强化材料回收率为锻造回收率的50%
```
