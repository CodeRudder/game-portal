# FL-BLD-02 建筑网格浏览

> **版本**: v3 | **最后更新**: 2026-05-05
> **归属功能**: 建筑系统 — 浏览与导航 (Phase A)
> **触发条件**: 玩家点击建筑Tab进入建筑系统
> **结束条件**: 玩家点击某建筑卡片或空地，离开网格浏览状态
> **前置状态**: 玩家已进入游戏主界面
> **后置状态**: 建筑详情面板(D区)或建造选择弹窗(E区)已打开
> **来源文档**: [PRD BLD-1](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1) / [UI BLD-1](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-1)

## 关键要求

- 网格规格: **6列 × 6行 = 36格**（PC端），卡片180x120px，间距12px
- 11座建筑固定位置（见FL-BLD-01网格布局表），锁定建筑显示灰色蒙版+解锁条件
- 每张卡片显示：建筑图标（随等级变化外观）、名称、等级、核心效果、升级进度条（升级中）
- 建筑状态4种（各有独立样式），定义如下：
  - **正常**: 已建造，但资源不足或前置条件不完全满足
  - **可升级**: 已建造 + 资源充足 + 前置满足 + 队列有空槽（绿色边框）
  - **升级中**: 正在升级中（蓝色脉冲边框+旋转齿轮）
  - **锁定**: 未建造且主城等级不足（灰色蒙版+解锁条件）

### R2 新增约束

- **锁定建筑点击行为**: 点击灰色蒙版锁定建筑 → 弹出解锁条件弹窗/Toast"主城Lv.X解锁"
- **空地分类**: 区分"可建造空地"（未建造全部11座前显示"+"号）和"装饰空地"（11座全建成后变为纯装饰，不可点击）
- **双击防护**: 所有卡片点击后立即进入禁用态，面板打开完成后恢复

### R3 新增约束

- **锁定建筑交互细分**: 锁定建筑区分"主城等级不足"与"前置条件未满足"两种锁定原因，分别显示不同提示文案和图标（主城等级不足→显示主城图标+等级数；前置条件不足→显示具体条件描述）
- **空地分类细化**: "可建造空地"内部再细分"有可建造建筑"（+号高亮，可点击）和"全部锁定/资源不足"（+号暗淡，点击后弹窗内全部条目灰显）
- **状态过渡动画**: 建筑卡片状态变化时播放过渡动画（正常→升级中：边框颜色渐变300ms；升级中→可升级：边框闪烁200ms后变绿；锁定→已建造：蒙版淡出400ms+卡片弹跳）
- **手机端列表滚动**: 手机端纵向列表使用虚拟滚动（仅渲染可视区域+上下各2条buffer），滚动惯性系数0.92，下拉刷新触发建筑数据重载
- **网格首次加载骨架屏**: 骨架屏模拟6x6网格布局（36个灰色方块），卡片骨架显示脉冲动画（1.2s循环），数据到达后逐卡片fade-in（每个卡片间隔50ms）

## 验收标准

- 玩家能一眼区分已建造/锁定/升级中/可升级的建筑
- 悬停Tooltip显示简要产出信息
- 点击卡片立即打开详情面板（PC: D区右侧滑入 / 手机: 全屏面板底部上滑）

## 步骤详情

### FL-BLD-02-01 进入建筑系统

> **来源**: [PRD BLD-1 建筑类型](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1-1) / [UI BLD-1 建筑网格场景](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-1)

**用户感知（see→act→feedback）**:
- **See**: 底部Tab栏显示"建筑"图标（当前未选中态，灰色），主界面场景区显示其他子系统内容
- **Act**: 点击"建筑"Tab图标（触摸目标>=44px，点击区域包含图标+文字）
- **Feedback**: Tab图标高亮为金色 → 场景区fade-out(100ms) → 骨架屏fade-in(100ms) → 数据到达后网格逐卡片fade-in（每卡片间隔50ms，总计<=1秒）

**系统行为**:
1. 发送`GET /api/buildings`请求（携带玩家ID）
2. 并行请求升级队列状态`GET /api/buildings/queue`
3. 收到响应后计算每座建筑状态（正常/可升级/升级中/锁定）
4. 渲染6x6网格骨架屏 → 逐卡片替换为真实数据
5. 右上角渲染升级队列悬浮窗（PC）/ 顶部固定条（手机）

**关键要求**: 骨架屏模拟6x6网格布局（36个灰色方块，脉冲动画1.2s循环），数据到达后逐卡片fade-in

**⏱ 时间要点**:
- Tab切换动画: 100ms fade-out + 100ms fade-in
- 骨架屏显示: <=1秒（超时走异常分支）
- 逐卡片fade-in: 每卡片间隔50ms，36格总计约1.8秒
- 请求超时阈值: 3秒

**📱 空间要点**:
- Tab栏: 底部固定56px高，图标32x32px，文字12px
- 骨架屏: 6x6灰色方块，每块180x120px+12px间距，rgba(60,65,78,0.4)填充
- PC场景区: 1280x696px

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 网络超时（>3秒） | 骨架屏保留 + 顶部Toast"数据加载失败，显示缓存数据"（3秒），使用本地缓存渲染 | 02-02 |
| 服务器维护 | 显示全屏维护公告覆盖层（rgba(0,0,0,0.9)），含维护预计结束时间，[关闭]按钮返回主界面 | — (终止) |
| 首次进入无缓存+超时 | 骨架屏保留 + Toast"网络异常，请检查网络后重试" + [重试]按钮 | 02-01 |
| 账号数据异常 | Toast"建筑数据异常，请重新登录" + 自动返回登录页 | — (终止) |

### FL-BLD-02-02 加载建筑网格

> **来源**: [UI BLD-1 建筑网格场景](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-1)

**用户感知（see→act→feedback）**:
- **See**: 看到6x6网格（PC）或纵向列表（手机），每格显示建筑卡片（图标+名称+等级+核心效果+状态样式）
- **Act**: 被动浏览，无需操作（系统自动完成渲染）
- **Feedback**: 卡片从骨架屏状态fade-in为真实内容（每卡片间隔50ms），右上角升级队列悬浮窗同时出现

**系统行为**:
1. 根据建筑数据渲染每张卡片：计算状态（正常/可升级/升级中/锁定）→ 匹配样式（金/绿/蓝/灰边框）→ 填充图标+名称+等级+核心效果
2. 空地格子渲染"+"号或纯装饰样式（11座全建成后）
3. 右上角渲染升级队列悬浮窗（PC: 280x80px / 手机: 顶部固定条32px）
4. 订阅建筑数据变化事件（WebSocket推送），实时更新卡片状态

**关键要求**: 11座建筑固定位置（参见FL-BLD-01 v3.0网格布局表），空地显示"+"号；11座全建成后空地自动变为纯装饰。锁定建筑区分"主城等级不足"与"前置条件不足"

**⏱ 时间要点**:
- 逐卡片渲染: 每卡片50ms间隔，总计约600ms（11建筑+空地）
- 状态推送延迟: <=200ms（WebSocket）

**📱 空间要点**:
- PC: 6列x6行网格（36格），卡片180x120px，间距12px，内边距24px，总网格宽=6x180+5x12+2x24=1200px，高=6x120+5x12+2x24=828px（超出场景区696px时垂直居中裁切）
- 手机: 纵向列表，条目355x72px，升级队列顶部固定32px，列表区域=(507-32)px=475px，可显示约6.6个条目
- 升级队列悬浮窗: PC右上角280x80px，z-index:30，距顶部12px距右侧12px

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 建筑数据字段缺失 | 该建筑卡片显示"数据异常"红色标记，不影响其他卡片 | 02-03 |
| 队列数据异常 | 队列悬浮窗隐藏，3秒后自动重试获取 | 02-03 |

### FL-BLD-02-03 浏览建筑卡片

> **来源**: [UI BLD-1-1 建筑卡片](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-1-1) / [PRD BLD-1-2 建筑外观](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1-2)

**用户感知（see→act→feedback）**:
- **See**: 浏览各建筑卡片，一眼识别4种状态——正常（金框# c8a96e）、可升级（绿框#27ae60+绿色[升级]角标）、升级中（蓝框#3498db脉冲+旋转齿轮）、锁定（灰框#666蒙版+锁定图标+解锁条件文字）
- **Act**: 视线扫视网格（PC）/ 上下滚动列表（手机），手机端左滑露出快捷菜单
- **Feedback**: 升级中卡片进度条每秒推进，产出数值实时递增；状态变化时边框颜色渐变（300ms）

**系统行为**:
1. 每秒更新所有建筑产出累加值（本地计算），每30秒服务端校准
2. 升级中卡片：进度条基于start_time+duration计算百分比，每秒刷新
3. 可升级检测：资源变化或队列空出时，卡片边框从金->绿渐变（300ms）
4. 锁定建筑显示细分锁定原因：主城等级不足（显示🏛️Lv.X图标）vs 前置条件不足（显示条件描述文字）
5. 手机端虚拟滚动：仅渲染可视区域+上下各2条buffer

**关键要求**: 4种状态样式严格区分。锁定建筑细分显示锁定原因。卡片外观随等级变化（Lv1-5茅屋/Lv6-12瓦房/Lv13-20楼阁/Lv21-30宫殿）

**⏱ 时间要点**:
- 产出值刷新: 每秒1次（本地），每30秒服务端校准1次
- 进度条刷新: 每秒1次
- 状态过渡动画: 300ms（边框颜色渐变）
- 手机端滚动帧率: >=30fps（低端设备跳过非可视卡片动画）

**📱 空间要点**:
- PC卡片: 180x120px，图标区60x60px居中上方，名称+等级文字14px居中下方，核心效果文字12px单行，进度条高8px底部
- 手机列表条目: 355x72px，图标40x40px左侧12px，名称+等级+产出居中区域，操作按钮[→]右侧
- 手机端左滑菜单: 宽度120px（3个40px按钮），背景rgba(30,35,48,0.95)

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 手机端快速滚动卡顿 | 虚拟滚动仅渲染可视区域+buffer，跳过非可视区域动画更新 | 02-03 |
| 升级进度与服务端不同步 | 以服务端start_time+duration为准重新计算，进度条平滑跳变 | 02-03 |
| 建筑等级数据异常（Lv>30） | 显示Lv.30+红色警告标记，Toast"数据异常" | 02-03 |

### FL-BLD-02-04 查看建筑Tooltip

> **来源**: [UI BLD-1-1 交互](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-1-1) / [PRD BLD-3 资源产出](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-3)

**用户感知（see→act→feedback）**:
- **See**: PC端鼠标悬停在建筑卡片上（悬停>=200ms后触发）
- **Act**: 无需点击，鼠标移入卡片区域即可
- **Feedback**: Tooltip从卡片上方/下方弹出（自动选择不遮挡方向），显示产出速率+加成摘要，鼠标移出卡片后Tooltip立即消失（<=100ms）

**系统行为**:
1. 鼠标进入卡片区域 → 启动200ms延迟计时器
2. 200ms后计算Tooltip显示位置（优先卡片上方，若空间不足则下方/左/右）
3. 渲染Tooltip内容：建筑名称+等级、主要产出及速率、当前加成百分比、升级进度（升级中时）
4. 鼠标移出 → 销毁Tooltip

**关键要求**: Tooltip不遮挡其他卡片内容，自动选择最佳显示方向。手机端无Tooltip功能

**⏱ 时间要点**:
- 出现延迟: >=200ms悬停后触发（防止快速扫过时闪烁）
- Tooltip淡入动画: 100ms
- 移出消失: <=100ms

**📱 空间要点**:
- PC端Tooltip: 最大宽度240px，高度自适应内容（通常80-120px），背景rgba(30,35,48,0.95)，圆角8px，阴影0 4px 12px rgba(0,0,0,0.3)
- Tooltip与卡片间距: 8px
- 锁定建筑Tooltip: 额外显示解锁条件（主城需达Lv.X 或 前置条件XXX）

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 手机端无悬停 | 不显示Tooltip，点击直接进入详情/弹窗流程 | 02-05 |
| Tooltip数据加载中 | Tooltip显示"加载中..."骨架行（灰色条），数据到达后替换 | 02-04 |
| 卡片位于网格边缘 | Tooltip自动翻转到卡片内侧方向显示 | 02-04 |

### FL-BLD-02-05 点击建筑卡片 → 详情面板

> **来源**: [UI BLD-2 建筑详情面板](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-2) / [UI BLD-1-1 交互](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-1-1)

**用户感知（see→act→feedback）**:
- **See**: 建筑卡片处于正常/可升级/升级中状态，点击区域明确（整张卡片）
- **Act**: 点击建筑卡片（触摸/鼠标），卡片进入禁用态（opacity:0.7）防止双击
- **Feedback**: 卡片scale缩放动画（95% 100ms）→ 面板从右侧滑入（PC 420px, 150ms ease-out）/ 底部上滑全屏（手机, 200ms ease-out）→ 面板内容fade-in（100ms）

**系统行为**:
1. 点击后立即禁用卡片（双击防护），发送`GET /api/buildings/{id}/detail`请求
2. 创建详情面板实例，根据建筑类型决定面板结构（简单/复杂）
3. 加载产出数据、加成明细、升级预览、队列状态
4. 面板打开动画完成后恢复卡片可点击状态

**关键要求**: 面板滑入动画150ms（PC）/ 200ms（手机）。简单建筑无Tab（概览+升级合并），复杂建筑显示Tab栏。锁定建筑点击不打开面板

**⏱ 时间要点**:
- 卡片禁用: 立即（0ms），面板打开后恢复
- 面板滑入: 150ms（PC）/ 200ms（手机）
- 详情数据加载: <=500ms（面板打开期间并行加载）
- 面板内骨架屏: 数据到达前显示

**📱 空间要点**:
- PC: D区右侧420px宽x696px高滑入面板，面板内区块：头部80px+Tab栏40px（复杂建筑）+内容区280-350px+按钮区56px
- 手机: 全屏面板底部上滑，宽度100%，高度100%，顶部显示[←建筑名]返回栏48px，底部固定操作按钮56px
- 锁定建筑: 不打开面板，显示Toast（3秒）或解锁条件弹窗

**分支**:

| 条件 | 转至流程 |
|------|:--------:|
| 简单建筑（农田/市集/矿场/伐木场） | → FL-BLD-03 建筑升级（概览面板内） |
| 复杂建筑（兵营/酒馆/工坊/市舶司） | → 各对应子流程 |
| 辅助建筑（书院/医馆/城墙） | → 各对应子流程 |
| **锁定建筑（主城等级不足）** | Toast"主城Lv.X解锁"（3秒），含主城图标+等级数字 |
| **锁定建筑（前置条件不足）** | 弹出解锁条件弹窗（320x200px），显示条件描述+当前进度 |

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 建筑数据异常 | 面板显示"加载失败"，[重试]按钮 | 02-05 |
| 升级中建筑被点击 | 正常打开面板，面板内显示升级进度条+加速按钮，隐藏[升级]按钮 | 02-05 |
| 面板打开期间网络断开 | 使用缓存数据渲染面板，顶部显示"离线模式"黄色标记 | 02-05 |

### FL-BLD-02-06 点击空地 → 建造选择

> **来源**: [UI BLD-1-2 空地块](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-1-2) / [UI BLD-3 建造选择弹窗](../../games/three-kingdoms/ui-design/ui-layout/BLD-buildings.md#bld-3) / [PRD BLD-1 建筑类型](../../games/three-kingdoms/ui-design/prd/BLD-buildings-prd.md#bld-1)

**用户感知（see→act→feedback）**:
- **See**: 空地格子显示虚线框+加号（未建造全部11座前）或纯装饰格（11座全建成后）
- **Act**: 点击有加号的空地格子（触摸目标>=44px）
- **Feedback**: 空地格子scale缩放（95% 100ms）→ 建造选择弹窗从中心scale弹出（PC 200ms）/ Bottom Sheet从底部上滑（手机 200ms ease-out）

**系统行为**:
1. 判断空地类型：可建造空地（未建造全部11座前）vs 装饰空地（11座全建成后，不可点击）
2. 可建造空地点击后：计算玩家尚未建造的建筑列表
3. 逐条匹配解锁条件（主城等级）和资源状态 → 标记状态（可建造绿/未解锁灰/资源不足橙）
4. 按解锁条件排序（已解锁在前，未解锁按主城等级排序）
