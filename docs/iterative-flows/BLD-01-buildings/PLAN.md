## 背景

建筑系统是三国游戏的核心子系统之一，包含11座建筑（主城、农田、市集、矿场、伐木场、兵营、工坊、书院、医馆、城墙、酒馆），涵盖资源产出、军事训练、装备锻造、科技研究、英雄招募、贸易管理等多个功能。

### 关键业务约束

1. 建筑等级 ≤ 主城等级（硬约束）
2. 建筑依赖链：主城等级决定其他建筑解锁
3. 队列槽位随主城等级递增（1→2→3→4）
4. 资源链循环：产出→消费→回流

## 整体目标

通过20轮迭代，将所有建筑系统流程细化到L3+深度，确保任何工程师可以不经询问直接实施。

### 迭代阶段划分

| 轮次 | 阶段 | 重点 |
|:----:|------|------|
| R1-R3 | 初始提取+基础细化 | 提取所有主流程骨架，完成L1主流程+核心子流程 |
| R4-R6 | 核心流程深化 | 升级/建造/收取流程细化到L3+，异常覆盖 |
| R7-R10 | 复杂建筑流程 | 酒馆/工坊/市舶司流程细化，子系统连接 |
| R11-R14 | 辅助建筑流程 | 兵营/书院/医馆/城墙流程细化 |
| R15-R17 | 全局一致性 | 跨流程一致性检查，引用修复，术语统一 |
| R18-R20 | 最终核验 | 全面核验，P0/P1清零，文档定稿 |
