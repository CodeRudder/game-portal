## 概述

攻城战是针对城池类领土的高难度战斗模式。完整生命周期包含 **18 步**（映射至 10 个主阶段 P1~P10）: 选择目标 → 点击攻占 → 确认弹框 → 编队 → 创建任务 → 显示任务信息 → 行军精灵 → 点击查看路线 → 等待到达 → 攻城动画 → 等待战斗 → 结果动画 → 结算 → 结果事件 → 编队状态更新 → 自动回城 → 行军返回。

**前置流程**: FL-MAP-06 (领土选择与详情)
**后置流程**: FL-MAP-11 (攻城奖励领取), FL-MAP-08 (驻防管理)
**关联流程**: FL-MAP-04 (行军动画), FL-MAP-05 (攻城动画), FL-MAP-16 (编队系统), FL-MAP-17 (伤亡系统)

**关键约束**（具体参数详见PRD）:
- 攻城条件: 领土相邻 + 兵力门槛 + 粮草充足 + 每日次数限制
- 占领条件: 城防值归零（唯一条件）
- 编队上限: 最多 3 个编队同时出征
- 行军时长: 最短 10s，最长不超过 60s
- 攻城战斗时长: 最短 10s，最长不超过 60s
- 回城行军时长: 最短 10s，最长不超过 60s

## 关键要求与验收标准

### 功能要求
- FR-09-01: 玩家可对敌方/中立城池发起攻城战，需满足领土相邻、兵力门槛、粮草充足、每日次数限制、攻城冷却等条件校验
- FR-09-02: 编队系统支持选择将领、分配兵力、战力预览，最多3个编队同时出征（详见 FL-MAP-16）
- FR-09-03: 4种攻城策略（强攻/围困/夜袭/内应）各有差异化属性和前置条件
- FR-09-04: 城防衰减由本地战斗引擎计算（回合制公式），客户端UI实时渲染。**[P0-07 已解决]** 本地计算公式: 每回合城防减少 = 攻方总战力 x 策略修正 / (城防系数 x 回合数上限)。客户端UI按 `计算值/回合帧数` 平滑递减
- FR-09-05: 应用切后台/崩溃恢复后显示恢复遮罩，恢复战斗进度，超时按撤退处理。**[P0-03 已解决]** 应用切后台期间客户端视觉状态有明确定义: 半透明遮罩 + '恢复战斗进度... (30s)' 倒计时文字 + 旋转加载图标 + 攻城动画冻结当前帧。**[P0-V04 已解决]** **[R4-ISS-008/ISS-020 修正]** 恢复超时 = 30秒（移动端检测），超时后按攻城阶段撤退处理（50%资源退还，消耗每日次数，5分钟冷却）。**[R4-ISS-020 Backlog]** 暂停功能不实现（复杂度高），记录为 Backlog 待后续迭代处理。
- FR-09-06: 编队初始战力与战后战力为不同阶段计算公式，不矛盾
- FR-09-07: 攻城冷却时间独立计算，每日次数在本地时间 00:00 重置
- FR-09-08: 行军/攻城/回城三个阶段均有时间约束: 行军 10s~60s，攻城战斗 10s~60s，回城行军 10s~60s
- FR-09-09: 点击行军精灵或攻占任务编队时，显示行军线路（黄色虚线高亮）和迷你进度面板
- FR-09-10: 攻城结果事件（保留24h），可在事件列表中点击查看结果详情
- FR-09-11: 攻城结束后编队自动回城，回城精灵沿去程路径逆序移动（10s~60s）
- FR-09-12: 攻城动画为循环播放直到战斗结束信号，单次动画循环 5~15s，总战斗时长 10~60s。**[P0-14 已解决]** 单次循环时长与总战斗时长为不同概念
- FR-09-ARCH: 本流程基于纯客户端本地架构。战斗引擎为客户端本地回合制计算，城防/战力/伤亡所有计算均在客户端完成。无服务端依赖。

### 非功能要求
- NFR-09-01: 攻城条件校验响应 <200ms
- NFR-09-02: 城防衰减由本地战斗引擎统一计算（回合制），UI渲染与计算在同一线程
- NFR-09-03: 应用切后台恢复遮罩在检测后立即显示，攻城动画暂停冻结当前帧
- NFR-09-04: 回城行军路径使用 A* 寻路算法计算最短路径

### 验收标准
- AC-09-01: 城防衰减公式统一为: 每回合城防减少 = 攻方总战力 x 策略修正 / (城防系数 x 回合数上限)，策略修正（`decay_strategy_modifier`，城防衰减专用）: 强攻1.2/围困0.8/夜袭1.5/内应1.3。客户端UI进度条按 `计算值/回合帧数` 平滑递减（帧级视觉插值）

> **[R4-ISS-003/R4-ISS-015] 策略修正术语区分定义**:
> - `decay_strategy_modifier`（城防衰减专用，本表 AC-09-01）: 强攻1.2 / 围困0.8 / 夜袭1.5 / 内应1.3 — 用于城防衰减公式分母计算
> - `power_strategy_modifier`（编队战力修正，PLAN.md C.2）: 强攻1.5 / 围困0.8 / 夜袭1.2 / 内应2.0 — 用于编队战力计算
> - 两组参数为独立参数，用途不同，不可混用。编队战力修正详见 PLAN.md C.2 和 FL-MAP-16 S2。
- AC-09-02: **[R4-ISS-008 修正]** 应用异常退出后恢复时显示半透明遮罩 + '战斗仍在进行！请在{N}秒内点击[继续战斗]，否则将按撤退处理（损失50%资源）' 倒计时 + 旋转加载图标 + "继续战斗"按钮（可主动点击加速恢复），攻城动画冻结当前帧。恢复超时 = 30秒（移动端检测）。检测系统事件（`pagehide`事件，如来电）时自动暂停倒计时。恢复后移除遮罩从当前城防值继续。超时（30秒）显示'恢复超时，按撤退处理（已退还50%资源）' Toast，按攻城阶段撤退规则结算
- AC-09-03: 编队初始战力 = FL-MAP-09 P3 公式；战后战力 = 初始战力 x (1 - 伤亡比例) x (1 - 受伤减益)，两者为不同阶段不矛盾
- AC-09-04: 攻城冷却时间 = 每次攻城结束后4小时，同一目标的攻城冷却独立计算
- AC-09-05: 每日攻城次数在本地时间 00:00 重置
- AC-09-06: '最近己方城市' = 以地图直线距离（欧几里得距离）最近的己方城市。若无己方城市（极端情况），编队就地解散，兵力损失50%
- AC-09-07: 行军精灵可点击: 点击精灵弹出迷你面板（剩余时间/进度/预计到达），同时地图高亮黄色虚线行军路线
- AC-09-08: 攻占结果事件可在事件列表中查看，点击展示完整结果详情（伤亡/奖励/道具），事件保留24h
- AC-09-09: 攻城动画循环播放（单次循环5~15s），直到本地战斗引擎计算结束才停止。总战斗时长10~60s
- AC-09-10: 回城行军时长落在 [10s, 60s] 范围内，精灵沿去程路径逆序移动
- AC-09-11: 编队状态更新后血条按伤亡比例变色（绿/黄/红），将领按受伤等级标记（轻伤/中伤/重伤），映射见 FL-MAP-17

## 主流程: 10 阶段（映射 18 步生命周期）

### Stage P1: 选择攻城目标（生命周期步骤 1~2）

**用户感知**: 看到 敌方或中立城池 → 操作 点击 → 反馈 高亮选中领土，弹出详情面板 → 操作 点击"攻占" → 反馈 进入确认弹窗

**系统行为**:
1. 玩家在世界地图点击敌方或中立城池
2. 系统高亮选中领土，弹出领土详情面板（显示名称/等级/防御值/归属/产出）
3. 玩家点击"攻占"按钮 → 导航至 Stage P2

**分支**:
| 条件 | 处理方式 | 转至流程 |
|------|---------|---------|
| 点击己方城池 | 进入领土详情，不触发攻城 | FL-MAP-06 |
| 点击资源点 | 进入领土征服，不触发攻城 | FL-MAP-07 |

### Stage P2: 攻城确认弹窗（生命周期步骤 3）

> 📖 参考: [PRD MAP-4 攻城战](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)

**用户感知**: 看到 攻城条件校验结果 → 操作 确认继续 → 反馈 进入编队选择

**系统行为**:
1. 执行攻城条件校验（领土相邻/兵力门槛/粮草充足/每日次数/攻城冷却），逐项显示绿勾或红叉
2. 校验失败 → 显示红色错误提示，列出未通过条件，不可进入下一步
   - **[P1-M-02] 当多条校验不满足时，推荐优先解决顺序：(1)兵力不足→提示'补充兵力'并链接到兵力池管理；(2)粮草不足→提示'获取粮草'并链接到产出收取；(3)每日次数耗尽→提示'今日攻城次数已用完，明日再来'**
3. 并发上限达峰（活跃 SiegeTask >= 3）→ 弹出提示面板，列出任务状态
4. 校验通过 → 显示攻城确认弹窗（目标信息 + 条件检查 + 预估消耗 + 每日次数 + 冷却倒计时）
5. 玩家确认继续 → 导航至 Stage P3

**粮草消耗**: 基于行军时间和兵力规模计算

**粮草消耗公式** [P2-R2-U03]: `消耗 = 基础消耗 x 行军时间(秒) x (1 + 编队兵力/1000)`
- 基础消耗 = 5 粮草/秒（初始设计值）
- 此公式为唯一权威版本，map-interfaces.md 公式索引应引用此版本
- **[P2-R2-U03] 已废弃**: Round 1 旧版公式 `粮草消耗 = 基础消耗(50) x 兵力系数(兵力/100) x 距离系数(格数x0.3) x 策略修正`（乘法链版）和 V-033 索引入口 `粮草消耗 = 基础消耗(100) + 出征兵力 x 0.1 + 行军距离(格数) x 5 + 策略修正`（加法版）均已废弃，不再使用。以本 Stage P2 内联公式为唯一权威来源。

示例：编队800兵力，行军时间30秒
粮草消耗 = 5 x 30 x (1 + 800/1000) = 5 x 30 x 1.8 = 270

**攻城冷却 [P1-28]** **[R4-ISS-013 修正]**: 攻城冷却时间 = 每次攻城结束后4小时。同一目标的攻城冷却独立计算。**[R4-ISS-013]** 明确定义:
- "同一目标" = 同一城池ID（cityId），非城池类型
- 攻下后归属变更冷却仍保留（4小时内不可再次对该城发起攻城，但不影响驻防管理）
- AI攻占导致归属变更不重置玩家冷却
- 冷却数据存储结构: `{ targetCityId: string, cooldownUntil: number }[]`（每个城池独立记录冷却截止时间戳）

**每日攻城次数 [P1-29]**: 每日上限3次。重置时间 = 本地时间 00:00。次数用完后"攻占"按钮置灰，显示Tooltip"今日攻城次数已用完(剩余X小时重置)"。

**攻城次数用完按钮状态 [P2-14]**: "攻占"按钮置灰 + Tooltip"今日攻城次数已用完(剩余X小时X分重置)"。

**移动端适配 [P0-15]**: PC端和移动端攻城步骤顺序统一为: 条件检查 → 编队选择 → 策略选择 → 确认发动。移动端采用多步骤向导模式（4步向导），每步全屏展示。策略选择时的战力预览基于编队已确定的兵力进行计算（编队先于策略，确保预览准确）。

**[V-022] 攻城前预检—仅剩1城警告**: 当玩家仅剩1座城池时，攻城确认弹窗中显示警告'您仅剩1座城池，若攻城失败且失去该城池将导致编队解散并损失50%兵力'

### Stage P3: 选择/创建编队（生命周期步骤 4）

> 📖 参考: [PRD MAP-4 攻城战](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)
> 子流程: FL-MAP-09-07 (编队选择与创建), FL-MAP-16 (编队系统)

**用户感知**: 看到 编队选择面板 → 操作 选择将领、分配兵力 → 反馈 战力预览

**系统行为**:
1. 在弹窗内嵌编队选择面板 (ExpeditionForcePanel)
2. 玩家选择出征将领（受伤/已出征将领不可选）并拖动滑块分配兵力
3. 编队约束校验（至少1名将领、满足最低兵力、将领不重复）
4. 显示编队战力预览 → 玩家确认编队 → 导航至 Stage P4

**编队战力计算**: 攻方总战力=Σ(各编队兵力 x 武将进攻战力系数)。综合将领属性和兵力。**[P1-M-01] 编队总战力简化表述=Σ(各编队兵力×武将进攻战力系数)，完整公式见FL-MAP-16 S2 [P0-02-R2]**

**编队总战力计算公式**:
```
编队总战力 = Σ(每个武将分配的兵力 × 该武将的进攻武将战力系数)
```
其中：进攻武将战力系数 = (武力×3 + 统率×2 + 智力) / 300

**[R4-ISS-001] 权威公式声明**: 此处为简化预览版公式，仅供攻城确认弹窗的快速战力预览。实际战斗计算使用 FL-MAP-16 S2 [P0-02-R2] 完整加法公式: `兵力×1.0 + 攻击×10 + 防御×8 + 智力×5 + 科技加成`。两公式计算结果差约12%，以 FL-MAP-16 S2 为唯一权威来源。

示例：编队含武将A（兵力500，系数1.2）和武将B（兵力300，系数0.8）
编队总战力 = 500×1.2 + 300×0.8 = 600 + 240 = 840

> **P0-V03 完整公式链定义**: 见 [PLAN.md 附录 C: 战力公式链](../PLAN.md#附录-c-战力公式链完整定义)
> - C.1 基础兵力战力: `base_force_power = troops x 1.0 + general.武力 x 10`
> - C.5 战力预览: `force_preview = troops x 1.0 + general.武力 x 10`（初始战力，不含策略和地形修正）

**[P0-V10 已解决] 将领全部不可用空状态**: 当所有将领不可用时（全部受伤或已出征），编队选择面板显示"暂无可用将领"空状态面板:

> **注（编队战力公式阶段说明）**: FL-MAP-09 P3 定义编队初始战力计算公式。FL-MAP-17 步骤4 定义战后战力重算公式（应用伤亡/受伤减益）。两者为不同阶段，不矛盾。初始战力 = FL-MAP-09 P3 公式；战后战力 = 初始战力 x (1 - 伤亡比例) x (1 - 受伤减益)。

### Stage P8: 攻城战斗过程（生命周期步骤 12）

> 📖 参考: [PRD MAP-4 攻城战](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)
> 子流程: FL-MAP-09-11 (攻占战斗回合制)

**用户感知**: 看到 城防血条实时下降 + 攻方战力条 + 回合计数器 + 军队精灵动画 + 粒子效果 + 城池动画 → 反馈 战斗进展

**系统行为**:
1. 读取城防初始值（具体计算公式详见PRD）

**城防初始值计算**:
```
城防初始值 = 基础城防值 × 城防系数
基础城防值 = 目标城市等级 × 100 + 守军驻防兵力 × 0.5
城防系数 = 1.0 + 城墙等级 × 0.05
```
其中：非城池领土的城墙等级 = 0，城防系数 = 1.0
2. 进入回合制战斗（回合间隔1s，最长60s/20回合）
3. 城防衰减（**本地战斗引擎计算，回合制公式**）：每回合城防减少 = 攻方总战力(base) x 策略修正 / (城防系数 x 回合数上限)。**[R4-ISS-001]** 攻方总战力使用 FL-MAP-16 S2 完整公式计算（非 Stage P3 简化预览版），此处"攻方总战力"指 FL-MAP-16 S2 的 base 值（兵力x1.0 + 攻击x10 + 防御x8 + 智力x5 + 科技加成），不含 `power_strategy_modifier`，策略效果通过本公式中的 `decay_strategy_modifier` 独立叠加。策略修正（`decay_strategy_modifier`，城防衰减专用）: 强攻1.2/围困0.8/夜袭1.5/内应1.3。城防系数 = 1.0 + 城墙等级x0.05。**[P0-07 已解决]** UI渲染与计算在同一线程：进度条每帧按 `计算值/回合帧数` 平滑递减

> **P0-V03 完整公式链定义**: 见 [PLAN.md 附录 C: 战力公式链](../PLAN.md#附录-c-战力公式链完整定义)
> - C.1 基础兵力战力: `base_force_power = troops x 1.0 + general.武力 x 10`
> - C.2 编队战力含修正: `force_combat_power = base x strategy_modifier x terrain_modifier`
>   - 策略修正: 强攻 x1.5, 围困 x0.8, 夜袭 x1.2(夜间), 内应 x2.0(需道具)
>   - 地形修正: 平原 x1.00, 山地 x0.90, 森林 x0.95, 沙漠 x0.92, 关隘 x0.85, 雪地 x0.90
> - C.4 城防衰减: `defense_decay = attacker_power x strategy_modifier / (wall_level x 5 + 100) x tick_interval`

> **[P1-M-09] 城防衰减公式版本说明** [R2-Step2]: PLAN.md附录C.4的城防衰减公式为旧版参考，以本流程 AC-09-01 定义为准（即: 每回合城防减少 = 攻方总战力 x 策略修正 / (城防系数 x 回合数上限)）。两版公式在策略修正系数和分母结构上存在差异，实现时应以 AC-09-01 为权威来源。
