## 关键要求与验收标准

### 功能要求
- FR-16-01: 编队创建需选择将领（排除受伤/繁忙）和分配兵力（从全局兵力池扣除）
- FR-16-02: 最多3个编队同时存在，每支编队含1名将领+兵力
- FR-16-03: 编队状态包含：ready/marching/fighting/returning
- FR-16-04: 编队状态由SiegeTask驱动变更，编队不主动变更状态
- FR-16-05: 仅ready/returning状态可解散，marching/fighting不可解散

### 非功能要求
- NFR-16-01: 进入编队创建 < 100ms
- NFR-16-02: 选择将领列表渲染 < 200ms
- NFR-16-03: 系统校验 < 50ms
- NFR-16-04: 编队创建执行 < 300ms
- NFR-16-05: 反馈更新 < 100ms

### 验收标准
- AC-16-01: 可用将领列表正确过滤出战中/恢复中将领
- AC-16-02: 兵力滑块硬限制在可用范围，不超过全局兵力池
- AC-16-03: 编队满3个时创建按钮不可用，Toast提示"编队已满（最多3个）"
- AC-16-04: 客户端本地校验数据完整性，异常时提示重试
- AC-16-05: 编队状态与SiegeTask正确映射

## 步骤详情

### S1 进入编队创建

> 📖 参考: [PRD MAP-7 编队创建](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-7)

**用户感知**: 看到 攻城确认弹窗中编队区域 → 操作 点击"创建编队" → 反馈 编队创建表单展开

**系统行为**:
1. 查询当前玩家已有编队数量
2. 查询可用将领列表（排除受伤和繁忙状态）
3. 查询可分配兵力（全局兵力池）
4. 渲染编队创建表单

**触发场景**:

| 场景 | 入口 | 初始状态 |
|------|------|---------|
| 新建编队 | 攻城确认弹窗 → [+ 编队] | 空表单，将领/兵力均未选 |
| 编辑编队 | 已有编队卡片 → [编辑] | 预填当前将领和兵力 |

### S2 选择将领

> 📖 参考: [PRD MAP-7 将领选择](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-7)

**用户感知**: 看到 可用将领列表 → 操作 选择将领 → 反馈 将领信息展示+胜率更新

**系统行为**:
1. 渲染将领列表，过滤不可选将领（出战中/恢复中），过滤规则详见PRD
2. 选中将领后显示战力、统率加成、技能信息
3. 选将后实时更新胜率预估

**[P2-22] 将领列表空状态**:
当所有将领均不可选时（全部出战中或恢复中），显示空状态面板 **[P0-V10 已解决]**:
- 面板标题"暂无可用将领"
- **受伤将领列表**: 每位显示 {将领名} + {受伤等级: 轻伤/中伤/重伤} + {恢复倒计时}（如: "关羽 中伤 恢复中 12:30"）
- **出征将领列表**: 每位显示 {将领名} + {返回倒计时}（如: "张飞 出征中 返回 02:15"）
- **预计等待时间**: min(所有将领恢复/返回时间)，显示"最早可用: {将领名} {剩余时间}"
- **[返回] 按钮**: 关闭攻城确认面板
- **新手教程保护**: 教程模式下确保教程将领始终可用，数据异常时强制重置为可用
- **[V-048]** 当玩家尚未获得任何将领时（零将领空状态），显示"尚未获得将领，完成主线任务可获得首位将领"

**[P0-02-R2] 编队总战力计算公式**:

编队总战力由兵力战力、将领属性和科技加成三部分组成:

```
编队总战力 =兵力战力 + 将领属性加成 + 科技加成总和

兵力战力 = troops x TROOP_POWER_COEFF
将领属性加成 = general.attack x ATK_COEFF + general.defense x DEF_COEFF + general.intel x INT_COEFF
科技加成总和 = Σ(已解锁科技.level x 科技.power_per_level)
```

**参数定义**:

| 参数 | 值 | 说明 |
|------|-----|------|
| TROOP_POWER_COEFF | 1.0 | 每1名士兵的基础战力系数 |
| ATK_COEFF | 10.0 | 将领攻击力权重 |
| DEF_COEFF | 8.0 | 将领防御力权重 |
| INT_COEFF | 5.0 | 将领智力权重 |
| 科技.power_per_level | 各科技不同 | 每级科技提供的战力加成 |

**示例计算**:
- 编队: 3000兵 + 张飞(攻击95, 防御70, 智力30) + 锋刃科技Lv2(每级+50)
- 总战力 = 3000x1.0 + (95x10 + 70x8 + 30x5) + 2x50
- 总战力 = 3000 + 950 + 560 + 150 + 100 = 4760

> **注意**: 以上参数为初始设计值，最终数值以 PRD v2.0 确认的为准。TROOP_POWER_COEFF 和各权重系数可通过配置表调整。

> **P0-V03 完整公式链定义**: 见 [PLAN.md 附录 C: 战力公式链](../PLAN.md#附录-c-战力公式链完整定义)（含 C.1 基础兵力战力、C.2 编队战力含策略/地形修正、C.4 城防衰减、C.5 战力预览、C.6 战后战力重算）

### S5 确认编队

> 📖 参考: [PRD MAP-7 确认编队](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-7)

**用户感知**: 看到 确认按钮 → 操作 点击 → 反馈 短暂确认反馈

**系统行为**:
1. 按钮点击后置灰（防重复提交）
2. 客户端二次校验所有约束
3. 在本地存储中创建编队记录

**异常分支**:
- 本地数据校验失败 → Toast "编队创建失败，请重试"，刷新数据
- 本地存储写入失败 → Toast "数据写入异常，请重试"

### S6 系统执行编队创建

> 📖 参考: [PRD MAP-7 编队执行](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-7)

**用户感知**: 等待短暂时间后看到编队创建成功

**系统行为**:
1. 从全局兵力池中扣除出征兵力
2. 创建编队记录，状态设为 `ready`
3. 维护不变量（兵力池>=0、编队数量上限），具体约束详见PRD
4. 若紧接着确认攻城，编队状态切换为 `marching`

- **执行耗时**: ⏱ <300ms

## 编队 CRUD 操作

> 📖 参考: [PRD MAP-7 编队管理](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-7)

### 创建（Create）

- 触发: 攻城确认弹窗 → [+ 编队]
- 前提: 编队数 < 3
- 流程: S1 → S2 → S3 → S4 → S5 → S6 → S7

### 查看（Read）

- 触发: 点击已有编队卡片
- 展示: 将领信息、兵力、状态、目标领土

### 更新（Update）

- 触发: 编队卡片 → [编辑]
- 限制: 仅 `ready` 状态可编辑
- 可改项: 将领、兵力
- 流程: 预填数据 → 修改 → S4 校验 → S6 执行 → S7 反馈

### 删除（Delete）

- 触发: 编队卡片 → [解散]
- 限制: 仅 `ready` / `returning` 状态可解散
- 执行: 兵力归还全局兵力池，编队记录删除
- `marching` / `fighting` 状态不可解散（需等待战斗结束或行军到达）

### [P1-39] 编队状态与SiegeTask映射

编队状态由SiegeTask驱动变更，编队不主动变更状态。SiegeTask状态变化时自动触发编队状态更新。映射关系如下：

| 编队状态 (ForceStatus) | 攻城任务 (SiegeTask) | 说明 |
|:----------------------:|:-------------------:|------|
| ready | preparing | 编队就绪，攻城准备中 |
| marching | marching | 编队行军中 |
| fighting | sieging | 编队攻城战斗中 |
| (无映射) | settling | 攻城结算中（编队不处于任何状态）。**settling 阶段编队显示**: 编队列表中该编队显示"结算中..."标签（灰色）；编队详情不可操作（按钮禁用）；预计持续时间 < 5s（结算计算耗时） |
| returning | returning | 编队返回中 |
| **[XC-002] 编队销毁** | completed | 攻城完成后编队对象销毁（兵力归还unallocated，将领释放为available）。如需再次攻城，创建新编队 |
| **[XC-002] 编队销毁** **[P1-R2-02]** **[P1-M-03]** | cancelled | 编队销毁（兵力归还unallocated，将领释放为available） |

## 异常流程汇总

| 异常编号 | 触发条件 | 用户感知 | 系统行为 | 恢复方式 |
|---------|---------|---------|---------|---------|
| E16-01 | 已有 3 个编队时尝试创建 | Toast "编队已满（最多 3 个）" | 不允许创建 | 解散现有编队 |
| E16-02 | 将领被其他编队占用 | 将领列表中灰色 + "出战中"标签 | 过滤不可选将领 | 更换将领或解散占用编队 |
| E16-03 | 将领受伤中 | 将领列表中灰色 + "恢复中"标签 | 过滤不可选将领 | 等待恢复或更换将领 |
| E16-04 | 兵力池不足 | 滑块硬限制在可用范围 | Toast "可用兵力不足" | 等待补充或减少其他编队兵力 |
| E16-05 | 确认时本地数据写入失败 | Toast "编队创建失败，请重试" | 本地数据写入异常 | 刷新数据后重新创建 |
| E16-06 | 确认时本地存储写入异常 | Toast "数据写入异常，请重试" | 编队未创建 | 点击重试 |
| E16-07 | 尝试编辑/解散 marching/fighting 状态编队 | 操作按钮置灰 | 不允许操作 | 等待行军/战斗结束 |
| E16-08 | 编队创建后兵力池为负数（数据边界异常） | Toast "兵力分配冲突" | 自动回滚编队创建 | 刷新后重新操作 |
| E16-09 | 将领在编队创建后突然受伤（客户端判定） | 编队状态变为"将领受伤" | 编队无法出击，需重新选将 | 编辑编队更换将领 |
