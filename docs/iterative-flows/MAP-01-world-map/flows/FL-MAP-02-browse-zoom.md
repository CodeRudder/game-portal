# FL-MAP-02 地图浏览与缩放

> **版本**: v3.2 | **最后更新**: 2026-05-05
> **归属功能**: 天下地图系统 — 地图平移、缩放与导航交互
> **触发条件**: 世界地图加载完成后（FL-MAP-01结束），玩家在地图区域执行拖拽/缩放操作
> **结束条件**: 玩家离开天下Tab或进入其他子流程（领土选择、筛选等）
> **前置状态**: 世界地图已渲染完成，视口处于己方领土区域
> **后置状态**: 视口位于玩家浏览位置，缩放级别为玩家设定值
> **来源文档**: [PLAN.md](../PLAN.md)
> **PRD 参考**: [MAP-world-prd-v2](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md)

## 关键要求与验收标准

### 功能要求
- FR-02-01: 玩家在地图区域执行拖拽操作时，地图应实时平移，响应延迟 < 16ms（60fps）
- FR-02-02: 拖拽到地图边界时施加边界约束（移动端弹性回弹，PC端光标变为not-allowed）
- FR-02-03: 双指捏合（移动端）或鼠标滚轮（PC端）执行缩放操作，以触点为中心缩放
- FR-02-04: 缩放级别受最小/最大值约束，超出范围强制约束
- FR-02-05: 缩放联动UI元素（产出气泡、城市名称、道路宽度）自适应显隐
- FR-02-06: PC端右下角小地图支持点击跳转、拖拽同步、双击展开/收起
- FR-02-07: 色块逻辑尺寸固定8x8px，渲染尺寸 = 8 x zoom（统一公式，所有缩放区间使用同一基础尺寸8px），地图总尺寸320x240色块 x 8px = 2560x1920px不变。表中各缩放区间的数值仅为渲染尺寸参考示例
- FR-02-08: 移动端无小地图，通过底部定位按钮和替代导航方案补偿：(1)底部工具栏"定位"按钮定位到己方核心领土；(2)搜索框输入城市名直接跳转；(3)筛选结果列表点击跳转到目标领土。三种替代方案确保移动端导航能力不弱于PC端

### 非功能要求
- NFR-02-01: 拖拽响应延迟 < 16ms（60fps）
- NFR-02-02: 缩放响应延迟 < 32ms（30fps）
- NFR-02-03: 连续3帧低于30fps时自动降级渲染精度
- NFR-02-04: 视口裁剪仅渲染可见色块，不可见区域跳过绘制调用
- NFR-02-05: 缩放使用最近邻插值（imageSmoothingEnabled = false）保持像素锐利

### 验收标准
- AC-02-01: 单指拖拽时地图实时跟随手指/鼠标移动，无可见延迟
- AC-02-02: 拖拽到地图边界后松手，移动端弹性回弹，PC端光标显示not-allowed
- AC-02-03: 缩放操作受最小/最大值约束，超出范围自动校正
- AC-02-04: 色块渲染尺寸随zoom缩放：渲染尺寸 = 8 x zoom
- AC-02-05: 小地图点击任意位置，主视口立即跳转至该位置（无动画）
- AC-02-06: 移动端底部定位按钮功能为"定位到己方核心领土"
- AC-02-07: 移动端无小地图时，可通过底部工具栏"地图"按钮、搜索框、筛选结果三种方式导航

## 步骤详情

### FL-MAP-02-01 地图拖拽平移

> 📖 参考: [PRD MAP-1 缩放范围与地图规格](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

> **来源**: 旧流程 MAP-F02 步骤1-2（M-48修复）

**用户感知**: 看到 地图内容 → 操作 单指拖拽（移动端）或鼠标按住拖拽（PC端） → 反馈 地图随手指/鼠标实时平移

**系统行为**:
1. 监听触摸/鼠标拖拽事件
2. 根据拖拽偏移量更新视口坐标
3. 调用视口裁剪（→ FL-MAP-02-SP02子流程）仅渲染可见区域色块
4. 使用 `requestAnimationFrame` + 脏矩形标记，仅重绘视口变化区域

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 触摸起点在屏幕边缘（阈值详见PRD） | 事件不传递给地图手势处理器，响应系统手势（如iOS返回） | — |
| 手势冲突（双指+单指） | 双指优先处理为缩放，忽略单指 | FL-MAP-02-03 |

---

### FL-MAP-02-02 边界约束检查

> 📖 参考: [PRD MAP-1 地图边界规格](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

> **来源**: 旧流程 MAP-F02 步骤2（M-48修复）

**用户感知**: 看到 拖拽到地图边界时 → 操作 继续拖拽 → 反馈 移动端弹性回弹，PC端光标变为`not-allowed`

**系统行为**:
1. 每帧计算视口是否超出地图边界，边界规格详见PRD
2. 超出边界时施加约束力：
   - 移动端：允许少量越界，松手后弹性回弹（动画参数详见PRD）
   - PC端：拖拽时立即约束，光标变为`not-allowed`样式

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 视口完全超出边界 | 强制回弹至最近的有效边界位置 | FL-MAP-02-01 |

---

### FL-MAP-02-03 缩放操作（双指/滚轮）

> 📖 参考: [PRD MAP-1 缩放范围](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

> **来源**: 旧流程 MAP-F02 步骤3-4

**用户感知**: 看到 地图内容 → 操作 双指捏合（移动端）或鼠标滚轮（PC端） → 反馈 地图以触点/鼠标位置为中心缩放，色块大小自适应

**系统行为**:
1. 接收缩放增量（Δzoom）
2. 调用缩放约束检查（→ FL-MAP-02-SP01子流程）
3. 计算以缩放中心点为锚的新视口坐标
4. 更新渲染参数，色块尺寸按zoom缩放，具体公式详见PRD
5. 使用最近邻插值保持像素锐利（`imageSmoothingEnabled = false`）

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 双指手势与系统手势冲突 | 双指操作优先 → 地图缩放 | — |

---

### FL-MAP-02-04 缩放联动UI更新

> 📖 参考: [PRD MAP-1 缩放级别与UI显隐规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

> **来源**: 旧流程 MAP-F02 步骤5（M-17/M-50修复）

**用户感知**: 看到 缩放时UI元素自适应：小缩放气泡消失仅显示小圆点（小圆点颜色继承产出气泡三色状态：绿色=正常、黄色=预警、红色=已满），中缩放名称隐藏，大缩放细节丰富

**系统行为**:
1. 根据缩放级别调整UI元素显隐（产出气泡、城市名称、道路宽度、色块尺寸），各级别具体阈值和显隐规则详见PRD
2. 显隐切换时应用淡入淡出过渡动画

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 连续3帧低于30fps | 自动降低渲染精度（跳过纹理细节列表：草地纹理、波浪动画、树冠细节、山体阴影，仅渲染基础色块色值。降级时不做提示以免打扰用户；帧率恢复时自动还原全精度渲染，用户可感知到地图细节微妙恢复） | 帧率恢复后自动还原 |

---

### FL-MAP-02-05 小地图交互

> 📖 参考: [PRD MAP-1 小地图规格](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

> **来源**: 旧流程 MAP-F02（M-49修复）

**用户感知**: 看到 右下角小地图缩略图（阵营色块 + 视口方框） → 操作 点击/拖拽/双击 → 反馈 主视口同步跳转/平移，或小地图展开/收起

**系统行为**:
1. 渲染小地图缩略图和视口矩形框，具体尺寸规格详见PRD
2. 点击小地图任意位置 → 主视口立即跳转至点击位置（无动画）
3. 拖拽小地图视口矩形 → 主地图同步平移（延迟 < 16ms）
4. 双击小地图 → 展开/收起切换（带过渡动画，参数详见PRD）

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 移动端 | 无小地图组件，通过底部定位按钮替代 | FL-MAP-01-05 |

> **移动端定位按钮说明（P1-03）**: FL-MAP-02 底部定位按钮与 FL-MAP-18 筛选工具栏定位按钮为同一组件，功能为"定位到己方核心领土"（核心领土中心定义：阵营初始出生城市坐标，详见 FL-MAP-01 P0-01）。点击定位按钮后视口平滑移动至核心领土中心（500ms动画）。此外，移动端提供两种补充导航方式：(1) 搜索框输入城市名 → 匹配结果列表 → 点击跳转到目标城市；(2) 筛选结果（FL-MAP-10）→ 点击列表项跳转到目标领土

### 约束检查流程

```
接收目标缩放值 targetZoom
  ├── targetZoom < 最小缩放 → 强制设为最小缩放
  ├── targetZoom > 最大缩放 → 强制设为最大缩放
  └── 在有效范围内 → 通过
返回有效缩放值
```

## 子流程: FL-MAP-02-SP02 视口裁剪

> 📖 参考: [PRD MAP-1 地图规格与色块参数](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

> **触发条件**: 每帧渲染tick（TE-MAP-11，60fps）或视口变化（TE-MAP-12）
> **结束条件**: 完成当前帧渲染
> **编号说明（P0-V09修复）**: 子流程使用 SP02 后缀避免与主步骤编号（FL-MAP-02-01~05）冲突

### 裁剪流程

```
获取当前视口矩形 (vx, vy, vw, vh)
  ├── 根据色块尺寸和zoom计算可见色块范围（startCol/endCol/startRow/endRow）
  ├── 仅遍历可见范围内的色块
  ├── 视口外色块跳过（不执行绘制调用）
  └── 使用 requestAnimationFrame + 脏矩形标记优化
```

> **色块尺寸计算说明（ISS-018）**: 色块逻辑尺寸固定 8x8px，渲染尺寸 = 8 x zoom。缩放级别表中的数值为渲染尺寸示例（如 zoom=2 时渲染尺寸为 16x16px）。地图总尺寸 320x240色块 x 8px = 2560x1920px 保持不变，不受缩放影响。

### 性能SLA

| 指标 | 目标值 | 降级策略 |
|------|--------|---------|
| 拖拽延迟 | < 16ms（60fps） | — |
| 缩放延迟 | < 32ms（30fps） | 允许双帧完成 |
| 帧率监控 | 开发模式FPS计数器 | 红<30 / 黄30~55 / 绿≥55 |
| 性能降级 | 连续3帧<30fps | 跳过纹理细节（草地纹理/波浪动画/树冠细节/山体阴影），仅渲染基础色块色值；降级不提示，恢复时自动还原 |
