## 关键要求与验收标准

### 功能要求
- FR-06-01: 玩家点击领土区域后系统判定选中目标（城市图标热区或领土色块区域），触发视觉高亮反馈并弹出详情面板
- FR-06-02: 系统根据点击位置向外扩展搜索最近的领土中心点（领土中心点 = TerritoryData.center，预定义的领土中心坐标，由地图配置数据提供），搜索半径内多领土优先级为距离最近优先
- FR-06-03: PC端面板从右侧滑入，地图自动左移保持领土可见；移动端Bottom Sheet从底部滑入，支持下拉关闭
- FR-06-04: 视口偏移后检查选中领土中心是否在可见区域内（扣除面板宽度480px），若被遮挡则额外平移使领土居中于剩余可见区域
- FR-06-05: 根据 `owner`（阵营归属）和 `type`（领土类型）条件渲染操作按钮，支持驻防/放弃/升级/征服/攻占等分支
- FR-06-06: **[XC-001] 防御战力系数** = 1.0（基础值）+ 驻防将领.统率 x 0.01（将领加成）。无将领时防御战力系数 = 1.0。注：进攻战力系数定义在FL-MAP-07，公式不同

### 非功能要求
- NFR-06-01: 选中判定耗时 <50ms，面板弹出动画耗时 300ms，关闭面板动画耗时 200ms
- NFR-06-02: 本地数据查询响应时间 <50ms
- NFR-06-03: 面板弹出后领土中心必须在可见区域内，不得被面板遮挡

### 验收标准
- AC-06-01: 点击任意领土区域，系统在 50ms 内完成选中判定并开始高亮动画
- AC-06-02: 面板弹出后领土中心点在扣除480px面板宽度后的剩余可见区域内居中显示
- AC-06-03: 数据读取失败时面板显示骨架屏+重试按钮，Toast提示"数据异常"
- AC-06-04: **[XC-001]** 防御战力系数计算正确：有将领时 = 1.0 + 将领统率 x 0.01，无将领时 = 1.0
- AC-06-05: 各归属类型按钮渲染正确：己方显示[驻防][放弃][升级]，非己方非城池显示[征服]，非己方城池显示[攻占]

## 步骤详情

### S1 点击领土

> 📖 参考: [PRD MAP-3 领土系统](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 各领土区域 → 操作 点击感兴趣的领土 → 反馈 触发选中判定

**系统行为**:
1. 接收点击事件坐标，传入选中判定逻辑
2. 点击热区包括城市图标热区和领土色块区域（具体像素规格详见PRD）
3. 点击空白区域（无领土） → 无选中，若已有选中态则取消选中并关闭面板

### S2 系统判定选中目标

> 📖 参考: [PRD MAP-3 领土系统](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 无感知（后台判定）

**系统行为**:
1. 点击位置命中城市图标热区 → 直接选中该城市所属领土
2. 点击位置未命中 → 向外扩展搜索最近的领土中心点
   - **领土中心点定义** (ISS-028): 领土中心点 = TerritoryData.center（预定义的领土中心坐标，由地图配置数据提供）
   - 搜索半径和算法详见PRD
   - **搜索半径内多领土优先级** (P2-08): 距离最近优先；同距离按领土等级降序（高等级优先）；同等级按领土ID升序（稳定排序）
3. 搜索范围内无领土 → 无选中（空白区域点击）
4. 判定耗时: ⏱ <50ms

### S3 视觉高亮反馈

> 📖 参考: [PRD MAP-3 领土系统](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 被选中领土出现明显的高亮边框 → 反馈 与未选中领土区分

**系统行为**:
1. 选中领土后，该领土所有色块添加选中态视觉效果
2. 选中态 = 虚线 + 流动动画；归属态 = 实线 + 无动画（具体边框样式、颜色、动画参数详见PRD）
3. 动画耗时: ⏱ 300ms 淡入

### S4 面板弹出

> 📖 参考: [PRD MAP-3 领土系统](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 地图旁边弹出领土详情面板 → 反馈 展示该领土的完整信息

**系统行为**:
1. 根据视口宽度选择面板形态
2. PC端（**[XC-004]** 使用FL-MAP-18的短边判定规则: Math.min(innerWidth, innerHeight)>767为PC端）: 地图右侧固定面板，从右侧滑入；地图自动左移保持领土可见
   - **视口偏移后遮挡处理** (ISS-029): 视口偏移后检查选中领土中心是否在可见区域内（扣除面板宽度480px），若被遮挡则额外平移使领土居中于剩余可见区域
3. 移动端（**[XC-004]** Math.min(innerWidth, innerHeight)<=767为移动端）: Bottom Sheet，从底部滑入，支持下拉关闭
4. 入场动画耗时: ⏱ 300ms（具体面板尺寸、位置、动画参数详见PRD）

### S5 展示详情信息

> 📖 参考: [PRD MAP-3 领土系统](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 领土的完整信息（名称/等级/阵营/地形/产出/驻防） → 操作 可点击产出区域展开详情

**系统行为**:
1. 前端通过 `territoryId` 从本地数据模型/Store中获取聚合数据
2. **加载状态指示** (P2-09): 数据加载中显示骨架屏配合脉冲效果；超 2 秒未返回追加显示文字"加载中..."
3. 渲染信息区域: 领土名称+等级、阵营归属、地形类型+加成、当前产出、驻防兵力+防御值、操作按钮组
4. **[XC-001] 防御战力系数定义** (ISS-030): 防御战力系数 = 1.0（基础值）+ 驻防将领.统率 x 0.01（将领加成）。无将领时防御战力系数 = 1.0。防御值 = 基础防御 x 防御战力系数。注：进攻战力系数定义在FL-MAP-07，公式不同
5. 数据模型 `TerritoryDetailViewModel` 字段定义详见PRD

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 数据读取失败 | 面板显示骨架屏+重试按钮，Toast提示"数据异常" | S5 |
| 数据返回为空 | 面板显示"该领土数据暂不可用" | -- |

### S6 操作按钮渲染

> 📖 参考: [PRD MAP-3 领土系统](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 与当前领土归属匹配的操作按钮 → 操作 点击对应按钮

**系统行为**:
1. 根据 `owner`（阵营归属）和 `type`（领土类型）条件渲染按钮
2. 按钮分支: 己方→[驻防][放弃][升级]; 非己方非城池→[征服]; 非己方城池→[攻占]; 中立→按类型走征服/攻占（具体按钮样式和条件详见PRD）
3. "征服"(FL-MAP-07) 适用于非城池快速占领；"攻占"(FL-MAP-09) 适用于城池完整攻城流程
4. **按钮可操作性预检** (P1-43): 渲染时执行基本条件预检，不满足条件的按钮置灰并显示 Tooltip 说明原因
   - [征服] 预检: 目标与己方是否相邻 → 不相邻时按钮置灰，Tooltip"需先占领相邻领土"
   - [征服] 预检: 可用兵力是否充足 → 不足时按钮置灰，Tooltip"兵力不足，需要至少 {min} 兵力"
   - [征服] 预检: 目标是否为水域 → 水域时按钮隐藏（水域不可征服）
   - [攻占] 预检: 可用兵力是否充足 → 不足时按钮置灰，Tooltip"兵力不足"
   - [放弃] 预检: 是否为核心领土（阵营初始城市）→ 核心领土时按钮置灰，Tooltip"核心领土不可放弃"
   - [升级] 预检: 领土等级<15 且 资源满足升级条件 → 不满足时按钮置灰，Tooltip"升级条件不满足"（参考FL-MAP-14）
   - 升级按钮可操作性预检表补充:

| 升级 | 领土等级 < 15 且满足升级条件（详见FL-MAP-14） | 可点击，跳转至升级确认 | 置灰，Tooltip "升级条件未满足" |
   - 注: 预检为前端本地校验，所有校验在客户端完成

### S7 玩家操作分支

**用户感知**: 看到 操作按钮 → 操作 点击不同按钮 → 反馈 跳转至对应流程

**系统行为**: 跳转至对应流程

| 操作 | 目标流程 | 说明 |
|------|---------|------|
| 点击 [驻防] | FL-MAP-08 | 进入驻防管理，调整兵力分配 |
| 点击 [放弃] | FL-MAP-08-01 | 二次确认后放弃领土 |
| 点击 [升级] | FL-MAP-14 | 进入领土升级流程 |
| 点击产出区域 | FL-MAP-06-01 | 展开产出详情子面板 |
| 点击 [征服] | FL-MAP-07 | 进入非城池领土征服流程 |
| 点击 [攻占] | FL-MAP-09 | 进入城池攻城战流程 |

### S8 关闭面板

**用户感知**: 看到 面板消失 → 反馈 地图恢复到未选中状态

**系统行为**:
1. 取消领土选中高亮（虚线边框消失）
2. 面板滑出动画（⏱ 200ms）
3. 地图视口恢复原位
4. 触发方式: 点击[x]关闭按钮 / 点击[—]折叠按钮(PC) / ESC键 / 再次点击已选中领土 / 点击地图空白区域 / 移动端下拉关闭

## 子流程

### FL-MAP-06-01 产出详情展开

> 📖 参考: [PRD MAP-3 产出系统](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**流程结构**:

```
FL-MAP-06-01 产出详情展开
├── S1 ⏱<50ms   📱点击产出区域
├── S2 ⏱<200ms  📱弹出产出详情子面板（PC: 子面板 / 手机: Bottom Sheet）
├── S3 ⏱<100ms  📱展示产出明细
└── S4 ⏱200ms  📱关闭子面板
```

**步骤详情**:

#### S1 点击产出区域

**用户感知**: 看到 领土详情面板中的产出信息区域（显示当前产出概要） → 操作 点击产出区域 → 反馈 触发产出详情展开

**系统行为**:
1. 在领土详情面板中检测到产出区域点击事件
2. 从本地数据模型中读取该领土的完整产出数据
3. 触发产出详情子面板弹出

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 产出数据为空 | 子面板显示"暂无产出数据" | — |

---

#### S2 弹出产出详情子面板

**用户感知**: 看到 产出详情子面板弹出动画 → 反馈 PC端从右侧弹出子面板，移动端Bottom Sheet从底部滑入

**系统行为**:
1. 根据视口宽度选择面板形态
2. PC端（**[XC-004]** Math.min(innerWidth, innerHeight)>767为PC端）: 在领土详情面板右侧弹出子面板（具体尺寸详见PRD）
3. 移动端（**[XC-004]** Math.min(innerWidth, innerHeight)<=767为移动端）: Bottom Sheet从底部滑入，支持下拉关闭
4. 入场动画耗时: ⏱ 200ms

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 面板空间不足（PC端） | 子面板覆盖在领土详情面板上方，添加半透明遮罩 | — |

---

#### S3 展示产出明细

**用户感知**: 看到 完整产出明细数据：基础产出、科技加成、建筑加成、总计、占领时间、下次产出提升途径

**系统行为**:
1. 渲染基础产出数值（按资源类型分行显示）
2. 渲染科技加成（逐项列出已解锁科技的产出加成）
3. 渲染建筑加成（逐项列出已建造建筑的产出加成）
4. 计算并显示总产出（基础 + 科技加成 + 建筑加成）
5. 显示占领时间（格式："已占领 X 天 X 小时"）
6. 显示下次产出提升途径提示（如"升级领地等级可提升产出上限"）
7. 各字段具体计算方式和数值格式详见PRD

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 产出计算异常（数据缺失） | 缺失项显示"--"，正常项照常显示 | — |

---

#### S4 关闭子面板

**用户感知**: 看到 产出详情子面板关闭 → 反馈 返回领土详情主面板

**系统行为**:
1. 触发关闭动画（⏱ 200ms滑出）
2. 销毁子面板组件，返回领土详情主面板
3. 触发方式: 点击[x]关闭按钮 / ESC键 / 点击面板外部区域 / 移动端下拉关闭

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 快速重复点击开关 | 防抖处理，忽略200ms内的重复点击 | — |

## 异常流程汇总

| 异常编号 | 触发条件 | 用户感知 | 系统行为 | 恢复方式 |
|---------|---------|---------|---------|---------|
| E06-01 | 点击位置无领土（空白区域） | 无响应 | 不触发选中，若已有选中态则取消选中 | 重新点击有效领土 |
| E06-02 | 数据读取失败（获取领土详情） | 面板显示骨架屏 + "数据异常，请重试" | 保留选中高亮，显示错误提示 | 点击重试按钮 |
| E06-03 | 领土数据为空/不存在 | 面板显示"该领土数据暂不可用" | 记录异常日志 | 返回地图浏览，稍后重试 |
| E06-04 | 选中领土后数据过期（已被AI敌方占领） | 面板信息自动刷新为新归属方信息 | 推送更新数据，重新渲染面板和按钮 | 系统自动处理 |

> **[V-011] AI敌方定义**: "已被AI敌方占领"指离线期间AI行动触发领土变更。AI行动规则：仅在离线期间触发，优先攻击无驻防或低驻防的玩家领土，使用FL-MAP-07概率速战模型，战斗力基于AI阵营等级。完整AI系统定义见 [PLAN.md 附录 A: AI 敌方系统定义](../PLAN.md#附录-a-ai-敌方系统定义)
| E06-05 | PC端面板打开时选中领土不可见 | 地图自动平移保持领土可见 | 地图自动偏移 | 系统自动处理 |
| E06-06 | 移动端Bottom Sheet下拉误触 | 下拉未达阈值时面板弹回 | 达到关闭阈值才关闭 | 松手恢复 |
