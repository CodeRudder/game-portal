# FL-MAP-18 手机端适配

> **版本**: v2.3 | **日期**: 2026-05-05
> **流程编号**: FL-MAP-18
> **旧流程对应**: 无（新增跨切面流程）
> **PRD 参考**: [MAP-world-prd-v2](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md)

## 关键要求与验收标准

### 功能要求
- FR-18-01: 移动端判定使用短边：Math.min(window.innerWidth, window.innerHeight) <= 767。767px为CSS像素宽度（非物理像素），iPad Mini竖屏(768px)归入PC端布局。移动端UA辅助判定仅用于日志统计，不影响布局决策
- FR-18-02: 手机端布局为全屏Canvas + Bottom Sheet替代右侧面板，无小地图
- FR-18-03: 触摸操作映射：单指拖拽/点击、双指缩放、长按(500ms)触发上下文菜单
- FR-18-04: 攻城流程使用Step Wizard 4步分步引导
- FR-18-05: 模态Bottom Sheet禁用下拉关闭手势，关闭按钮需二次确认

### 非功能要求
- NFR-18-01: 设备检测耗时 < 50ms
- NFR-18-02: 手机端布局渲染耗时 < 300ms
- NFR-18-03: 底部按钮栏渲染耗时 < 100ms
- NFR-18-04: 屏幕旋转布局切换300ms防抖

### 验收标准
- AC-18-01: 手机端布局正确隐藏小地图，显示底部按钮栏
- AC-18-02: Bottom Sheet拖拽交互流畅，支持半屏(40%)/全屏(90%)两种高度
- AC-18-03: Step Wizard各步骤校验通过后"下一步"才可点击
- AC-18-04: 模态Bottom Sheet关闭时弹出二次确认"确认放弃当前操作？数据将不会保存"
- AC-18-05: 横屏时仍为手机端布局（短边判定）
- AC-18-06: 长按领土弹出上下文菜单，菜单项根据领土类型、所有权、相邻关系、资源条件动态显示/置灰

## 步骤详情

### S1 检测设备类型

- **用户感知**: 页面根据屏幕尺寸自动切换布局模式。
- **系统行为**:
  - 监听 `window.resize` 和 `matchMedia` 事件。
  - **[P1-40] 移动端判定修正**：使用短边判定 `Math.min(window.innerWidth, window.innerHeight) <= 767`（CSS像素宽度，非物理像素）。横屏时仍为手机端布局（短边判定确保iPad Mini竖屏768px归入PC端布局）。移动端UA(User-Agent)辅助判定仅用于日志统计和功能开关，不影响布局模式决策
  - 短边 <=767px → 切换到手机端布局。
  - 短边 >767px → 切换到 PC 端布局。
  - 切换时保留当前交互状态（选中领土、弹窗状态等）。
- **布局模式判定**:

| 条件 | 布局模式 | 基准尺寸 |
|------|---------|---------|
| 短边 >767px | PC 端 | 1280x696px |
| 短边 <=767px | 手机端 | 375x507px |

### S2 渲染手机端布局

- **用户感知**: 地图全屏显示，无右侧信息面板，无小地图组件。
- **系统行为**: 渲染手机端专属布局。

```
┌──────────────────────────────────┐
│ 搜索框（搜索城市...）[筛选]      │  <- 32px，精简版工具栏
├──────────────────────────────────┤
│                     │
│ 像素地图            │  <- 全屏 Canvas
│ (375x507px)         │     无小地图
│                     │
├─────────────────────┤
│ 底部抽屉            │  <- 可拖拽 Bottom Sheet
│ - 统计数据          │     替代右侧面板
│ - 领土详情          │
│ - 攻城按钮          │
├─────────────────────┤
│ [筛选▼] [热力图] [定位] │ <- 32px 底部按钮栏
└──────────────────────────────────┘
```

**与 PC 端的关键差异**:

| 组件 | PC 端 | 手机端 |
|------|-------|-------|
| 信息面板 | 右侧固定 480x696px | Bottom Sheet（可拖拽展开） |
| 小地图 | 右下角 180x140px | 无（不渲染） |
| 工具栏 | 顶部 36px 全功能 | 顶部 32px 精简版（含搜索框，宽度自适应，placeholder '搜索城市...'） |
| 底部按钮栏 | 无 | 32px: [筛选] [热力图] [定位] |
| 攻城弹窗 | 单页弹窗 480x400px | Step Wizard 4 步分步引导 |

### S3 触摸操作映射

- **用户感知**: 通过触摸手势操作地图。
- **系统行为**: 将触摸事件映射为地图操作。

| 触摸操作 | PC 端对应 | 地图功能 | 说明 |
|---------|---------|---------|------|
| 单指拖拽 | 鼠标拖拽 | 平移视窗 | 移动地图 |
| 单指点击 | 鼠标点击 | 选择领土 | 选中领土，打开Bottom Sheet |
| 双指张开 | 滚轮上滚 | 放大 | 放大地图 |
| 双指合拢 | 滚轮下滚 | 缩小 | 缩小地图 |
| 长按(500ms) | 右键 | 上下文菜单 | 领土快捷操作（详见下方上下文菜单定义） |

**长按交互细节**:
- 长按开始200ms后显示微弱视觉指示（目标领土轻微高亮）
- 触发500ms后执行触感反馈（vibration 50ms）并弹出上下文菜单
- 菜单以领土中心为锚点弹出，超出屏幕边界时自动调整位置
- 菜单外点击或按下返回键关闭菜单，无操作执行

**[P0-V08] 长按上下文菜单项定义**:

上下文菜单根据领土类型和所有权动态显示不同操作项。每个菜单项附带前置条件检查，不满足条件的菜单项显示为置灰状态并附带Tooltip说明原因。

#### 菜单项完整列表

| 菜单项 | 图标 | 目标领土类型 | 操作目标 | 前置条件 | 置灰原因 |
|--------|------|-------------|---------|---------|---------|
| **驻防管理** | 盾牌 | 己方领土 | FL-MAP-08 | 无特殊条件 | — |
| **领土升级** | 上箭头 | 己方领土（等级未满） | FL-MAP-14 | 升级条件满足（占领时长/资源充足） | "升级条件未满足" |
| **放弃** | 旗帜白 | 己方领土 | FL-MAP-08-01 | 非核心领土 | 核心领土不可放弃 |
| **征服** | 剑交叉 | 敌方/中立非城池领土，与己方相邻 | FL-MAP-07 | 可用兵力 >= 驻防 x 1.5 | "兵力不足" 或 "不相邻" |
| **攻占** | 城堡火焰 | 敌方城池，与己方相邻 | FL-MAP-09 | 可用兵力 >= 城防兵力 x 1.5，有可用将领，每日攻城次数未用完 | "兵力不足" / "无可用将领" / "今日次数已用完" |
| **侦查** | 眼睛 | 敌方/中立相邻领土 | — | 无特殊条件（侦查为免费操作，返回目标领土详细信息：驻防兵力、城防值、产出等） | — |
| **标记** | 旗帜 | 非己方非相邻领土 | — | 无特殊条件 | — |
| **查看详情** | 放大镜 | 任意领土 | FL-MAP-06 | 无特殊条件 | — |

#### 菜单显示规则（按领土所有权和类型）

**场景A: 己方领土**
```
┌─────────────────┐
│ 🛡 驻防管理      │  → FL-MAP-08
│ ⬆ 领土升级      │  → FL-MAP-14（条件不满足时置灰）
│ 🏳 放弃          │  → FL-MAP-08-01（核心领土时置灰，附确认弹窗）
└─────────────────┘
```

**场景B: 敌方/中立 非城池领土，与己方相邻**
```
┌─────────────────┐
│ ⚔ 征服          │  → FL-MAP-07（兵力不足时置灰）
│ 👁 侦查          │  → 显示目标详情信息
└─────────────────┘
```

**场景C: 敌方城池，与己方相邻**
```
┌─────────────────┐
│ 🔥 攻占          │  → FL-MAP-09（条件不满足时置灰，显示具体原因）
│ 👁 侦查          │  → 显示目标详情信息
└─────────────────┘
```

**场景D: 非己方领土，与己方不相邻**
```
┌─────────────────┐
│ 🚩 标记          │  → 在地图上添加/移除玩家标记（本地存储，最多20个标记）
│ 🔍 查看详情      │  → FL-MAP-06（只读模式，无操作按钮）
└─────────────────┘
```

**特殊场景**:
- 水域领土: 仅显示 [查看详情]（水域不可征服，胜率修正=0%，见FL-MAP-07）
- 空白/无效区域: 不触发上下文菜单（长按无响应）

**菜单项条件预检逻辑**:
```
onLongPress(territory):
  if territory is null or void:
    return  // 不触发菜单

  menuItems = []

  if territory.owner === player:
    menuItems.add(驻防管理)     // 始终可用
    menuItems.add(领土升级)     // 检查: 升级条件满足? 否则置灰
    menuItems.add(放弃)         // 检查: 非核心领土? 核心领土时置灰

  else if territory is adjacent to playerTerritory:
    if territory.type !== CITY:
      menuItems.add(征服)       // 检查: 兵力 >= 驻防x1.5? 否则置灰
    else:
      menuItems.add(攻占)       // 检查: 兵力+将领+次数? 否则置灰
    menuItems.add(侦查)         // 始终可用

  else:
    // 非相邻领土
    menuItems.add(标记)         // 始终可用
    menuItems.add(查看详情)     // 始终可用

  renderContextMenu(menuItems, anchor=territory.center)
```

> 📖 参考: 菜单项操作分别对应 FL-MAP-06(领土详情)、FL-MAP-07(征服)、FL-MAP-08(驻防)、FL-MAP-08-01(放弃)、FL-MAP-09(攻城)、FL-MAP-14(升级)

- **手势冲突处理**:

| 冲突场景 | 优先级 | 处理方式 |
|---------|--------|---------|
| 单指拖拽 vs 点击 | 距离判定 | 移动 >5px 为拖拽，<=5px 为点击 |
| 双指缩放 vs 单指拖拽 | 双指优先 | 检测到第二根手指时切换为缩放模式 |
| 长按 vs 点击 | 时间判定 | 按下 >500ms 为长按，<500ms 抬起为点击 |

- **缩放约束**: 0.5x ~ 3.0x（与 PC 端一致）。

### S4 Bottom Sheet 交互

- **用户感知**: 点击领土后从底部滑出信息面板，可拖拽展开/收起。
- **系统行为**: 用 Bottom Sheet 组件替代 PC 端的右侧固定面板。

**Sheet 状态**:

| 状态 | 高度 | 说明 |
|------|------|------|
| 隐藏 | 0 | 默认状态（无选中领土时） |
| 半屏 | 40% 视口高度 | 显示领土摘要信息 |
| 全屏 | 90% 视口高度 | 显示领土完整详情 |

**Sheet 内容映射**:

| PC 端面板 | 手机端 Sheet 内容 | 触发 |
|----------|-----------------|------|
| MAP-2 领土详情面板 | 领土名称 + 等级 + 产出 + 驻防 + 相邻 | 点击己方领土 |
| MAP-3 攻城确认弹窗 | Step Wizard（见 S5） | 点击敌方领土 |
| MAP-4 行军预览面板 | 行军路线 + 军队信息 + 确认按钮 | 行军预览时 |

**拖拽交互**:
- 下拉超过 30% Sheet 高度 → 收起到半屏。
- 下拉超过 60% Sheet 高度 → 完全收起。
- 上推超过 30% Sheet 高度 → 展开到全屏。
- 惯性滚动支持。
- **释放吸附行为**: 释放时自动吸附到最近稳定点（隐藏/半屏/全屏），基于释放位置判断。若释放位置在当前状态30%范围内则保持当前状态，否则吸附到最近的稳定点。

**[ISS-050 P1] 模态Bottom Sheet防误关**:
模态Bottom Sheet（攻城确认/编队创建/伤亡报告）：禁用下拉关闭手势。关闭按钮(x)点击后弹出二次确认"确认放弃当前操作？数据将不会保存"。

### S5 攻城流程适配（Step Wizard）

- **用户感知**: 攻城操作分为 4 个步骤，每步独立展示。
- **系统行为**: 将 PC 端的单页攻城弹窗拆分为 Step Wizard。

**Step Wizard 步骤（P0-15：步骤顺序与PC端统一）**:

| 步骤 | 标题 | 内容 | 操作 |
|:----:|------|------|------|
| 1/4 | 目标情报 | 城市图标 + 守军兵力 + 城防值 + 阵营 | [下一步] |
| 2/4 | 编队配置 | 选择将领 + 分配兵力（FL-MAP-16） | [下一步] |
| 3/4 | 攻城策略 | 强攻 / 围困 / 夜袭 / 内应四选一 + 胜率预估（基于已选编队计算） | [下一步] |
| 4/4 | 确认出征 | 消耗汇总 + 奖励预览 + 最终确认 | [确认攻城] / [取消] |

> **步骤顺序统一说明（P0-15）**: PC端和移动端攻城步骤顺序统一为"编队→策略"，确保战力预览在策略选择前基于实际编队计算，保证PC/移动端结果一致。移动端Step Wizard的步骤2(编队)和步骤3(策略)对应PC端弹窗中的编队选择区和策略选择区，功能完全一致，仅交互形式不同（分步 vs 同页）。

**导航规则**:
- 底部显示步骤指示器: 当前步骤高亮。
- 可点击已完成的步骤回退修改。
- [上一步] 按钮返回前一步。
- 每步校验通过后 [下一步] 才可点击。
- 最后一步点击 [确认攻城] → 触发攻城流程（FL-MAP-09）。

### S6 底部按钮栏

- **用户感知**: 屏幕底部固定 32px 高度的快捷操作栏。
- **系统行为**: 渲染底部按钮栏。

```
┌──────────────────────────────────┐
│  [筛选▼]     [热力图]    [定位]   │  <- 32px
└──────────────────────────────────┘
```

| 按钮 | 功能 | 展开方式 |
|------|------|---------|
| [筛选▼] | 阵营/地形/收益筛选下拉 | Bottom Sheet 向上展开筛选面板 |
| [热力图] | 热力图开关切换 | Toggle 切换，无展开面板 |
| [定位] | 定位到己方核心领土（核心领土中心定义见FL-MAP-01） | 居中到核心领土中心（500ms动画），无领土选择列表 |

**移动端导航替代方案（P1-03）**: 移动端无小地图，通过以下三种替代方案补偿导航能力：
1. **底部"定位"按钮**: 一键定位到核心领土中心
2. **搜索框（顶部工具栏）**: 顶部32px精简工具栏包含搜索框（宽度自适应，placeholder '搜索城市...'），输入城市名 → 匹配结果列表 → 点击跳转到目标城市
3. **筛选结果跳转**: 通过筛选面板（FL-MAP-10）选择条件 → 结果列表中点击领土 → 视口跳转到目标

**底部栏层级**:
- 底部栏 z-index 高于地图 Canvas。
- Bottom Sheet 展开时底部栏隐藏。
- 攻城 Step Wizard 进行时底部栏隐藏。
