## 头部元数据

| 字段 | 值 |
|------|---|

| **前置状态** | 已选中非己方、非城池领土（FL-MAP-06），目标领土详情面板已打开 |
| **后置状态** | 胜利: 领土归己方，自动驻防，进入驻防管理（FL-MAP-08）；失败: 兵力损失，可重整（FL-MAP-07-01） |

> **适用范围**: 本流程仅适用于非城池领土的征服（资源点、关隘、普通领土）。城池类领土的攻占请走 FL-MAP-09（攻城战）。

> **设计说明（V-034）**: 非城池领土无城防系统，使用概率速战模型（本流程FL-MAP-07）；城池有城防系统（FL-MAP-09），使用回合制攻城模型。两套系统有意并存，分别针对不同领土类型。

## 关键要求与验收标准

### 功能要求
- FR-07-01: 玩家可对非己方、非城池领土发起征服，需满足相邻和兵力门槛校验
- FR-07-02: 系统实时计算胜率并显示颜色编码的胜率条，胜率范围 5%~95%
- FR-07-03: 胜利后领土自动归属己方，自动驻防，产出从 0% 开始充能
- FR-07-04: 失败后扣除兵力，提供"重整旗鼓"恢复路径
- FR-07-05: 水域领土不可被征服（胜率修正=0%），但行军路线可经过水域色块（moveCost=2.14）

### 非功能要求
- NFR-07-01: 相邻校验耗时 <50ms（本地校验）
- NFR-07-02: 胜率计算耗时 <50ms（前端本地计算）
- NFR-07-03: 兵力部署界面动画耗时 300ms
- NFR-07-04: 战斗结算耗时 <1s
- NFR-07-05: 结果反馈动画耗时 500ms

### 验收标准
- AC-07-01: 不相邻领土征服时，显示错误提示并在地图上高亮己方相邻领土（绿色边框脉冲），提示文案为"需先占领与目标相邻的领土，已为您高亮显示相邻区域"
- AC-07-02: 胜率公式包含完整的进攻武将战力系数和防御系数定义
- AC-07-03: 失败兵力损失 = 出征兵力 x (1 - 胜率)，最低损失为出征兵力的 10%
- AC-07-04: 胜利后自动驻防兵力从出征兵力中扣除，胜利弹窗显示"出征兵力 X → 驻防扣除 Y → 归还 Z"
- AC-07-05: 征服结果弹窗在 500ms 内完成渲染，胜利/失败各有对应特效和按钮

## 步骤详情

### S1 点击征服按钮

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 橙色"征服"按钮 → 操作 点击 → 反馈 触发征服流程

**系统行为**:
1. 记录当前选中领土 ID 和玩家 ID，进入校验阶段
2. 前置条件确认: 目标领土为非己方、非城池类型

### S2 系统校验征服条件

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 短暂等待 → 反馈 校验通过进入兵力部署 / 校验失败显示红色错误提示

**系统行为**:
1. 校验领土相邻（目标与己方有道路连接）
2. 校验兵力充足（可用兵力 >= 目标驻防兵力 x 1.5，具体门槛详见PRD）
   > **[P0-V06] 阈值对齐说明**: 此 1.5x 兵力阈值与 FL-MAP-10-06 "可征服"快捷筛选阈值已统一，确保快捷筛选结果中的领土在实际出征时均能通过兵力门槛校验。
3. 校验耗时: ⏱ <50ms（本地校验）

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 不相邻 | Toast "需先占领与目标相邻的领土，已为您高亮显示相邻区域" + 在地图上高亮与目标相邻的己方领土（绿色边框脉冲），流程中断 | -- |
| 兵力不足 | Toast "兵力不足，需要至少 {min} 兵力"，流程中断 | -- |
| 同时不相邻且兵力不足 | 优先显示不相邻错误（按校验顺序） | -- |

### S3 显示兵力部署界面

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 兵力部署区域弹出 → 反馈 显示目标信息和兵力滑块

**系统行为**:
1. 渲染兵力部署滑块，初始位置 = 最低出征兵力（V-056修复：滑块最小值 = max(1, 目标驻防兵力 x 1.5)）
2. 显示目标领土信息（名称、驻防兵力、地形）
3. 界面元素: 目标信息 + 兵力滑块 + 当前兵力数值 + 胜率预估条 + 确认/取消按钮（具体规格详见PRD）
4. 动画耗时: ⏱ 300ms
5. **兵力不足检测（V-019修复）**: 若可用兵力 < 最低出征要求（目标驻防 x 1.5），显示"当前兵力不足以出征该领土（需{min}，现有{available}）"，[确认出征]按钮置灰

### S4 玩家调整出征兵力

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 拖动滑块 → 反馈 实时兵力数值和胜率变化

**系统行为**:
1. 滑块拖动时实时更新出征兵力数值
2. 每次滑块值变化触发胜率重算（见 S5）
3. 滑块范围约束: 最小值 = 目标驻防兵力 x 1.5；最大值 = 玩家当前可用兵力（具体交互细节详见PRD）

### S5 胜率预估

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 胜率数值和颜色编码的胜率条实时变化 → 反馈 了解出征风险

**系统行为**:
1. 根据胜率公式实时计算（详见子流程 FL-MAP-07-02）
2. 胜率颜色编码按区间显示不同颜色和建议文案（具体颜色和区间详见PRD）
3. 计算耗时: ⏱ <50ms（前端本地计算）

### S6 确认出征

**用户感知**: 看到 "确认出征"按钮 → 操作 点击 → 反馈 按钮置灰等待结算

**系统行为**:
1. 按钮点击后置灰（防重复点击）
2. 本地状态机直接执行征服逻辑，更新领土归属

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 本地数据写入失败 | Toast "出征指令发送失败"，按钮恢复 | S6 |

### S7 战斗结算

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 短暂等待 → 反馈 战斗结果

**系统行为**:
1. 概率判定: roll = random(0, 100)，roll < 胜率则为胜利，否则失败
2. 兵力损失计算（统一公式，S7/S9/S10 一致）(P1-25):
   - 胜利损失 = 出征兵力 x 5%（固定低损失，象征性消耗）
   - 失败损失 = max(出征兵力 x 10%, 出征兵力 x (1 - 胜率))，即最低 10% 损失
   - **最低损失规则说明（XC-005修复）**: 征服失败最低损失为出征兵力10%，攻城战(FL-MAP-17)最低损失为1名士兵。不同战斗系统有不同的损失下限规则
   - **伤亡计算说明（XC-008修复）**: 征服伤亡使用与FL-MAP-17 S2相同的公式和参数，虽内联执行而非显式调用FL-MAP-17
   - **[P1-R2-04] 实施备注**: 建议将伤亡计算提取为共享工具函数 `calculateCasualties(result, troops)`，FL-MAP-07 和 FL-MAP-17 均调用同一函数，确保参数变更同步。此为实施建议而非流程变更。
   - **[P2-R3-10] 伤亡下限规则**: 编队仍有兵力时，最少损失1名士兵（与 FL-MAP-17 一致）。实施建议使用共享函数 `calculateCasualties(result, troops, mode)` 其中 mode='conquest'|'siege' 区分征服简化公式和攻城完整公式。

### S8 胜利结果处理

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 征服胜利特效和结果弹窗 → 反馈 领土归属变更

**系统行为**:
1. 领土归属立即变更为己方
2. **自动驻防兵力来源与扣除** (P1-09): 自动驻防兵力从出征兵力中扣除。驻防数 = min(50% 驻防上限, 出征剩余兵力)。剩余兵力归还全局兵力池 unallocated。**执行顺序（V-031修复）**: 出征总量从unallocated扣除 → 战斗损失扣除(5%) → 自动驻防扣除(min(50%驻防上限, 剩余兵力)) → 归还剩余到unallocated
3. 产出从 0% 开始充能（非城池规则，与攻城战城池从 50% 开始不同）
4. 从全局兵力池扣除出征兵力（减去自动驻防部分）
5. 更新地图上该领土的视觉显示（阵营色变更 + 脉冲光环 + "新"标签）
6. **[P0-M-02] [R2-Step2] 胜利信息展示序列**（避免多信息叠加压垮玩家）:
   - **第1层（弹窗主体，立即显示）**: 胜利弹窗首先展示核心结算结果
     - 标题: "{领土名称} 已归入我方"
     - 兵力明细行: "出征兵力 {X} → 驻防扣除 {Y} → 归还 {Z}"
     - 自动驻防信息行（弹窗体内，非独立Toast）: "已自动驻防 {Y} 兵力于该领土"
   - **第2层（弹窗内高亮区，随第1层一起显示）**: 若 unallocated = 0，在弹窗体内以高亮区块显示: "⚠ 当前无剩余可用兵力"
   - **第3层（延迟Toast，弹窗显示1.5s后）**: 若其他上下文仍需独立Toast通知（如跨系统联动），延迟1.5s后显示，避免与弹窗同时弹出造成信息叠加
   - 弹窗包含可选 [管理驻防] 按钮链接至 FL-MAP-08，玩家可关闭弹窗继续地图操作 → FL-MAP-08 S1 [P1-R2-VERIFY-02]（从胜利弹窗进入，滑块预置为自动驻防数量）
7. **胜利→驻防管理过渡** (P1-44): 自动驻防在后台完成，无需玩家确认。驻防信息已内嵌于胜利弹窗第1层中展示（见上方第6项），不再单独弹出Toast
8. **unallocated=0 警告（V-040修复）**: 已整合至胜利弹窗第2层高亮区（见上方第6项），不再使用独立Toast

### S9 失败结果处理

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 征服失败提示和损失信息 → 操作 可选"重整旗鼓"或"确认"

**系统行为**:
1. **失败损失计算** (P1-25): 损失 = 出征兵力 x (1 - 胜率)，最低损失为出征兵力的 10%。此公式为统一公式，S7/S9/S10 均使用同一计算。剩余兵力归还全局兵力池 unallocated
   - 例：出征 1000 兵力，胜率 60% → 损失 = 1000 x (1 - 0.6) = 400，归还 = 600
   - 例：出征 1000 兵力，胜率 95% → 损失 = 1000 x (1 - 0.95) = 50 < 100(10%最低) → 损失 = 100，归还 = 900
2. 领土归属不变
3. 后续可选: "重整旗鼓" → FL-MAP-07-01；"确认" → 关闭弹窗回地图

### FL-MAP-07-02 胜率预估

> 📖 参考: [PRD MAP-3 征服规则](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**触发**: 兵力滑块值变化时实时计算

**胜率公式** (P0-11, P0-V03, V-027修复): `胜率 = clamp(5%, 95%, (我方战力/敌方战力 x 0.5) + 地形修正)`，其中 clamp 为上下限截断函数，括号内优先级为：先除法 → 再乘以0.5 → 再加地形修正 → 最后截断至[5%, 95%]

> **完整公式链定义**: 见 [PLAN.md 附录 C: 战力公式链](../PLAN.md#附录-c-战力公式链完整定义)（含基础兵力战力 C.1、编队战力 C.2、胜率公式 C.3）

- 我方战力 = 出征兵力 x 进攻武将战力系数
  - **进攻武将战力系数** (P0-11, XC-001修复): 默认值 = 1.0（无武将时）。注意：防御战力系数定义在FL-MAP-06，公式不同
  - 有武将时: 进攻武将战力系数 = (武将.武力 x 3 + 武将.统率 x 2 + 武将.智力) / 300
  - 归一化范围: 约 1.0 ~ 3.0（SSR满级武将约2.5~3.0，SR约1.5~2.0，R约1.0~1.5）
  - **[P2-M-16] 公式差异说明** [R3]: 征服(FL-MAP-07)使用简化单将公式 `(武力x3+统率x2+智力)/300` 计算武将战力系数，攻城(FL-MAP-09)使用完整编队公式 `兵力x1.0 + 攻击x10 + 防御x8 + 智力x5 + 科技` (见FL-MAP-16 S2)。这是设计意图：征服为快速单将领战斗，攻城为复杂编队战斗。两系统独立，不需要统一。
- 敌方战力 = 目标驻防兵力 x 防御系数
  - **防御系数** (P0-11): 默认值 = 1.0（基础值，无城墙无地形加成时）
  - 防御系数 = 1.0 + 城墙等级 x 0.05 + 地形加成
  - 城墙等级范围: 0~10（非城池领土城墙等级=0，即无城墙加成）
  - 地形加成: 各地形有不同的加成值（具体数值详见PRD）
- 地形修正（V-028修复，内联定义）:

| 地形 | 胜率修正 | 说明 |
|------|---------|------|
| 平原 | +0% | 无修正 |
| 山地 | +10%（防守方） | 防守方优势 |
| 森林 | +5%（防守方） | 防守方小幅优势 |
| 水域 | N/A | 不可征服 |
| 沙漠 | -5%（攻方不利） | 攻方不利 |

  注：完整地形修正值表见 [PLAN.md 附录 C.2](../PLAN.md#附录-c-战力公式链完整定义)。上表为攻方视角，完整表含关隘(-15%)和雪地(-10%)。
- **水域领土特殊规则** (P1-26): 水域领土不可被征服（FL-MAP-07 胜率修正=0%，征服按钮在 FL-MAP-06 中隐藏），但行军路线可经过水域色块（FL-MAP-04 moveCost=2.14，仅影响行军速度，不影响征服）。两条规则不矛盾：水域色块是行军地形（可通行），水域领土是征服目标（不可征服）。

**实时计算规则**: 前端本地计算，滑块变化触发重算（具体防抖规则详见PRD）。

**[P2-R3-11] 公式系数差异说明**:
征服胜率公式（FL-MAP-07）使用简化版系数 `(武力x3+统率x2+智力)/300`，仅适用于非城池领土的单将征服。
攻城胜率公式（FL-MAP-09/16）使用完整编队战力 `兵力x1.0 + 攻击x10 + 防御x8 + 智力x5 + 科技`。
两套系数有意并存：征服为快速简化计算，攻城为完整战力对比。征服最终胜率 = min(95%, max(5%, 简化战力/敌方简化战力 x 50% + 地形修正))。
