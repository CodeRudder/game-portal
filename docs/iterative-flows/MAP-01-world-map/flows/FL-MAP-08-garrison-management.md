## 关键要求与验收标准

### 功能要求
- FR-08-01: 玩家可在己方领土详情面板中增减驻防兵力，滑块实时约束在驻防上限和兵力池余量范围内
- FR-08-02: 全局兵力池 TotalArmyPool 维护 unallocated / cap / replenishRate 三要素，所有兵力分配基于此模型
- FR-08-03: 领土放弃流程包含二次确认弹窗、损失预览、冷却期触发
- FR-08-04: 本地原子操作保证数据一致性（单客户端无并发）
- FR-08-05: 放弃领土冷却期到期时游戏内通知提醒玩家（使用浏览器本地Notification API，无需服务端支持）

### 非功能要求
- NFR-08-01: 兵力调动执行耗时 <300ms
- NFR-08-02: 反馈更新耗时 <100ms
- NFR-08-03: 驻防上限/兵力池校验实时响应，无感知延迟
- NFR-08-04: 本地数据写入失败时最大重试 3 次

### 验收标准
- AC-08-01: TotalArmyPool.cap = 基础兵力(500) + 己方领土总数 x 200 + Σ(所有己方兵营建筑等级) x 50
  - **己方领土总数**: 当前玩家名下所有领土（含资源点和城池）的数量总和，实时统计
  - **Σ(所有己方兵营建筑等级)**: 玩家在所有己方领土上建造的兵营建筑等级逐一相加。若某领土未建兵营则不计入；若某领土兵营为 Lv3，则贡献 3。取所有己方领土上兵营等级的算术和，非最高等级
- AC-08-02: 客户端本地校验数据完整性，若数据异常自动修正（最大3次重试，100ms/200ms/400ms 间隔），超过3次 Toast "操作繁忙，请稍后重试"
- AC-08-03: 放弃领土冷却期到期推送通知: '{领土名} 冷却已结束，可重新占领'（游戏内通知，使用浏览器本地Notification API，无需服务端支持 + 游戏内邮件）
- AC-08-04: 增兵/减兵操作后全局兵力池 unallocated 实时更新，UI 反馈延迟 <100ms
- AC-08-05: 滑块拖至 0 时显示警告但不阻止操作

## 步骤详情

### S1 进入驻防管理

> 📖 参考: [PRD MAP-3 驻防机制](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 驻防管理界面 → 反馈 显示驻防信息和兵力池状态

**系统行为**:
1. 记录当前领土 ID，查询全局兵力池状态和当前领土驻防数据
2. 触发场景: 主动驻防(FL-MAP-06) / 征服后驻防(FL-MAP-07胜利) / 攻城后驻防(FL-MAP-09胜利)
3. **征服/攻城胜利→驻防过渡方式** (P1-44): 自动驻防在后台完成（FL-MAP-07-S8/FL-MAP-09-P9），无需玩家确认。过渡链路为：战斗胜利 → 自动计算驻防量 → 后台扣减兵力 → Toast通知"已自动驻防 {Y} 兵力" → 胜利弹窗含可选 [管理驻防] 按钮 → 玩家可点击进入 FL-MAP-08 手动调整，也可关闭弹窗继续地图操作
4. **[P1-R2-VERIFY-02] 从胜利弹窗进入**: 当用户从 FL-MAP-07/FL-MAP-09 胜利弹窗点击 [管理驻防] 按钮进入时，兵力滑块预置为自动驻防数量，用户可在此基础上下调整。自动驻防本身无需额外确认。

**用户感知**: 看到 完整的驻防数据和兵力池状态 → 反馈 了解当前防御能力

**系统行为**:

2. **全局兵力池可视化** (P2-13): 面板底部显示兵力分配概览条: [已驻防 (各领土合计)] + [未分配 (unallocated)] / [上限 (cap)]。鼠标悬停/长按已驻防区域弹出分配明细列表（各领土名称+驻防数），帮助玩家了解兵力全局分布
3. 驻防上限和防御值计算规则详见PRD

### S3 玩家调整兵力

> 📖 参考: [PRD MAP-3 驻防机制](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 拖动滑块 → 反馈 实时兵力数值和防御值变化

**系统行为**:
1. 滑块拖动时实时更新驻防兵力显示和防御值变化
2. 实时显示对全局兵力池的影响（"将消耗/回收 {delta} 兵力"）
3. 增兵显示消耗；减兵显示回收（具体交互细节详见PRD）
4. 实时约束: 最大值 = min(驻防上限, 当前驻防+未分配兵力)；最小值 = 0

### S4 系统校验

> 📖 参考: [PRD MAP-3 驻防机制](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 滑块到达极限位置自动停止 → 反馈 无法继续拖动

**系统行为**:
1. 实时校验: 驻防上限、兵力池余量、兵力池非负、兵力池上限（具体约束规则详见PRD）
2. **驻防上限计算**: 驻防上限 = 查 FL-MAP-14 等级表对应值（无额外加成）。例如：Lv1=100, Lv2=200, ..., Lv15=1500。
3. 增兵冲突处理: 多块领土同时增兵时，按操作时间排序，本地状态原子更新保证一致性
4. 本地状态原子更新（P1-27 修复）:
   - **操作粒度**: 全局兵力池（TotalArmyPool）为操作对象
   - **实现方式**: 直接更新本地存储中的兵力数据，无需乐观锁（单客户端无并发冲突）
   - **数据完整性**: 每次更新前后校验不变量约束（unallocated >= 0 且 <= cap，驻防总量与兵力池一致）
   - **异常恢复**: 若本地数据写入失败，自动重试（最大3次，间隔 100ms/200ms/400ms）。超过3次仍失败则 Toast "操作繁忙，请稍后重试"
   - **读取一致性**: 每次打开驻防面板时从本地存储读取最新数据，滑块拖动期间不刷新（避免抖动），仅在确认提交时校验

### S5 确认分配

**用户感知**: 看到 "确认分配"按钮 → 操作 点击 → 反馈 短暂确认反馈

**系统行为**:
1. 按钮点击后置灰（防重复提交）
2. 更新本地存储中的驻防数据，参数: `{ territoryId, targetTroops }`
3. 本地二次校验驻防上限和兵力池余量

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 本地校验失败（数据不一致） | Toast "兵力分配冲突，请重试"，刷新数据 | S2 |
| 本地存储写入异常 | Toast "数据保存异常，请重试" | S5 |

### S6 系统执行兵力调动

> 📖 参考: [PRD MAP-3 驻防机制](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 短暂等待 → 反馈 操作成功

**系统行为**:
1. 增兵: 从全局兵力池 `unallocated` 扣除增量
2. 减兵: 向全局兵力池 `unallocated` 增加回收量
3. 更新领土驻防兵力值，重新计算防御值
4. 维护不变量约束（具体约束详见PRD）
5. 执行耗时: ⏱ <300ms

### S7 反馈更新

**用户感知**: 看到 驻防信息面板数据刷新 → 反馈 防御值变化

**系统行为**:
1. 更新驻防兵力条（当前/上限）、防御值显示、全局兵力池显示、地图驻防标记
2. Toast "驻防调整成功"
3. 更新耗时: ⏱ <100ms

### FL-MAP-08-01 领土放弃

> 📖 参考: [PRD MAP-3 驻防机制](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**流程结构**:

```
FL-MAP-08-01 领土放弃
├── S1 ⏱<50ms   📱点击"放弃"按钮
├── S2 ⏱300ms   📱弹出确认弹窗（含损失预览）
│   ├── 点击"取消" → 关闭弹窗，流程结束
│   └── 点击"确认放弃" → S3
├── S3 ⏱<300ms  📱系统执行放弃（回收部分兵力，领土变中立）
├── S4 ⏱<100ms  📱触发冷却期
└── S5 ⏱200ms   📱关闭面板，更新地图视觉
```

**步骤详情**:

- **S1 点击放弃按钮**: 在己方领土详情面板中点击灰色"放弃"按钮，触发二次确认流程。
- **S2 确认弹窗**: 显示警告文案 + 损失预览（仅回收部分兵力） + 冷却期提示 + [确认放弃]/[取消]按钮（具体弹窗内容详见PRD）。
- **S3 系统执行放弃**: 回收部分驻防兵力到全局兵力池，损失剩余兵力；领土归属变更为中立，清除驻防将领和兵力，产出停止；更新地图视觉（具体回收比例和视觉变化详见PRD）。
  - **回收比例**: 驻防兵力的 70% 归还全局兵力池（unallocated），30% 损失（V-057修复）
  - **[P2-M-04] 领土视觉过渡**: 领土颜色渐变为灰色（中立色，过渡300ms），驻防图标淡出（200ms），产出进度条消失。
  - **冷却期**: 放弃后该领土进入 4 小时冷却期，冷却期间不可重新征服/攻占

**[P2-R3-18] 放弃领土地图视觉过渡**: 放弃确认后，该领土在地图上的阵营色渐变为中立灰色（500ms过渡动画），旗帜降下动画（300ms）。冷却期 = 4小时（与攻城冷却一致），冷却期间该领土不可被重新占领。
- **S4 触发冷却期**: 该领土进入冷却期（**4小时（14400秒）**），冷却期内仅放弃者本人不可重新占领，AI阵营不受限制可占领该领土（V-020修复，完整AI系统定义见 [PLAN.md 附录 A](../PLAN.md#附录-a-ai-敌方系统定义)）。冷却期到期时推送通知（P1-10 修复）:
  - **通知触发**: 冷却期倒计时归零时，本地定时器检测冷却到期，触发本地通知
  - **通知内容**: '{领土名} 冷却已结束，可重新占领'
  - **领土状态检查**: 推送时同步检查领土当前归属状态:
    - 若领土仍为中立（未被他人占领）: 通知内容为 '{领土名} 冷却已结束，可重新占领'
    - 若领土已被AI阵营占领: 通知内容为 '{领土名} 冷却已结束，但已被{阵营名}占领'
  - **通知渠道（V-012修复）**: 游戏内通知（使用浏览器本地Notification API，无需服务端支持） + 游戏内邮件（双渠道确保送达）
- **S5 关闭面板**: 关闭领土详情面板，地图视口恢复，取消选中态。

## 全局兵力池模型

> 📖 参考: [PRD MAP-3 驻防机制](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

驻防管理的核心数据模型，所有兵力分配均基于此模型。模型定义 `TotalArmyPool` 包含: 未分配兵力(unallocated)、总兵力上限(cap)、每小时自然补充(replenishRate)。

**replenishRate 定义（V-030修复）**: replenishRate = 基础值(50/小时) + 兵营总等级 x 10/小时（详见PRD兵营系统）

**兵力上限公式**: TotalArmyPool.cap = 基础兵力(500) + 己方领土总数 x 200 + Σ(所有己方兵营建筑等级) x 50

> **变量精确定义（P0-12 修复）**:
> - **己方领土总数**: 当前玩家名下所有领土（含资源点和城池）的数量总和，实时统计。新占领即增1，放弃或被夺即减1
> - **Σ(所有己方兵营建筑等级)**: 玩家在所有己方领土上建造的兵营建筑等级逐一相加的算术和。例如：领土A兵营Lv3 + 领土B兵营Lv1 + 领土C无兵营 = 4。若某领土未建兵营则贡献0。此处取所有己方领土上兵营等级的逐个累加值，而非取最高等级

**兵力池分配规则**:

| 操作 | 兵力池变化 | 说明 |
|------|-----------|------|
| 增加驻防 | unallocated -= 增量 | 从未分配池扣减 |
| 减少驻防 | unallocated += 减量 | 回收到未分配池 |
| 征服出征 | unallocated -= 出征兵力（临时扣减） | 战斗结束归还或损失 |
| 征服胜利 | 归还剩余兵力，驻防自动分配 | 自动驻防 |
| 征服失败 | 归还剩余兵力（扣除损失） | 兵力永久损失 |
| 放弃领土 | unallocated += 驻防 x 70%（V-057修复：部分比例=70%） | 仅回收70%，剩余30%永久损失 |
| 新获领土 | cap += 按等级增加 | 兵力上限增加 |

（具体数值参数和不变量约束详见PRD）

**兵力池并发控制总体方案** (P1-42):

兵力池是全局共享资源，所有修改兵力池的操作（驻防增减/征服出征/征服胜利/征服失败/放弃领土/产出补充）均需遵循统一策略：

1. **统一操作对象**: 所有兵力池操作均针对全局兵力池（TotalArmyPool）执行原子更新（非单领土级操作，而是全局兵力池级操作）
2. **操作方**: FL-MAP-08（驻防增减）、FL-MAP-07（征服出征/胜利/失败）、FL-MAP-09（攻城出征/胜利/失败）、FL-MAP-08-01（放弃领土）、FL-MAP-01（产出补充/收取）
3. **本地原子性**: 每次兵力变动均为原子操作——扣减 unallocated + 更新 territory garrison 在同一本地操作内完成（单客户端无并发）
4. **统一异常处理**: 所有流程写入失败时均采用同一策略：刷新本地数据 → 自动重试（最多3次，指数退避）→ 失败 Toast 提示
5. **出征临时锁定**: 征服/攻城出征时从 unallocated 预扣兵力，战斗结束后根据结果结算（归还/损失/驻防分配），避免战斗期间兵力被二次分配
