## 关键要求与验收标准

### 功能要求
- FR-14-01: 等级逐级自动提升，无需手动操作。关键节点（Lv5/Lv10/Lv15）需要手动确认+资源投资
- FR-14-02: Lv1~4每6小时自动升1级（占领24h内），Lv5~9每12小时自动升1级（需Lv5手动确认后），Lv10~14每24小时自动升1级（需Lv10手动确认后）
- FR-14-03: 升级操作为原子事务（扣除投资资源+等级提升+产出倍率更新+驻防上限更新）
- FR-14-04: 产出倍率以表格精确值为准

### 非功能要求
- NFR-14-01: 系统校验耗时 < 0.5s
- NFR-14-02: 扣除资源+等级提升执行耗时 < 0.3s
- NFR-14-03: 升级标签更新动画耗时 < 0.2s

### 验收标准
- AC-14-01: 占领时间达标后升级按钮从灰色变为可点击
- AC-14-02: 点击升级后弹出确认弹窗显示消耗明细和预期收益
- AC-14-03: 资源不足时升级按钮置灰并提示"资源不足"
- AC-14-04: 领土Lv15满级时不显示"升级"按钮
- AC-14-05: 升级事务失败时资源回滚

## 流程结构

```
占领时间达标
  ⏱ 24h ──── Lv5解锁(自动或手动)
  ⏱ 72h+投资 ── Lv10解锁
  ⏱ 7天+重金 ── Lv15解锁
  ⏱ 用户 ──── 点击"升级领土"按钮
  ⏱ 0.5s ──── 系统校验(时间+资源)
  ⏱ 0.3s ──── 扣除投资资源
  ⏱ 0.2s ──── 等级标签更新+产出倍率提升
  📱 手机端: 领土详情面板内"升级"按钮 → 确认弹窗
```

## 步骤详情

### 步骤1: 系统触发 — 占领时间达标

> 📖 参考: [PRD v2 MAP-3 领土系统](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 满足升级条件时，领土详情面板中升级按钮高亮/闪烁提示。若玩家不在该领土详情面板，地图上该领土显示升级图标，并弹出全局Toast"XX领土可升级至Lv{N}"

**系统行为**:
1. 触发事件 TE-MAP-02（领土等级提升，占领时间达标）
2. 检查领土持续占领时间是否满足升级要求
3. 等级解锁条件（占领时间、投资需求）、产出倍率和驻防上限详见PRD
4. **[V-024]** 玩家不在该领土详情面板时：弹出全局Toast"XX领土可升级至Lv{N}"，并在地图上该领土位置显示升级图标（箭头向上+闪光动画），引导玩家点击查看

**[P0-04] 等级提升粒度完整规则**:
等级逐级自动提升，无需手动操作。关键节点（Lv5/Lv10/Lv15）需要手动确认+资源投资。

**15级逐级提升规则表**:

| 等级 | 升级方式 | 所需条件 | 产出倍率 | 驻防上限 |
|:----:|:-------:|---------|:-------:|:-------:|
| Lv1 | 自动（初始） | 占领即得 | x1.00 | 100 |
| Lv2 | 自动 | 占领 6 小时 | x1.05 | 200 |
| Lv3 | 自动 | 占领 12 小时 | x1.10 | 300 |
| Lv4 | 自动 | 占领 18 小时 | x1.15 | 400 |
| Lv5 | **手动确认** | 占领 24 小时 + 铜钱 x2000 | x1.20 | 500 |
| Lv6 | 自动 | Lv5 后 12 小时 | x1.25 | 550 |
| Lv7 | 自动 | Lv5 后 24 小时 | x1.30 | 600 |
| Lv8 | 自动 | Lv5 后 36 小时 | x1.35 | 650 |
| Lv9 | 自动 | Lv5 后 48 小时 | x1.40 | 700 |
| Lv10 | **手动确认** | Lv5 后 72 小时 + 铜钱 x10000 + 元宝 x50 | x1.50 | 1000 |
| Lv11 | 自动 | Lv10 后 24 小时 | x1.55 | 1050 |
| Lv12 | 自动 | Lv10 后 48 小时 | x1.60 | 1100 |
| Lv13 | 自动 | Lv10 后 72 小时 | x1.70 | 1200 |
| Lv14 | 自动 | Lv10 后 96 小时 | x1.80 | 1300 |
| Lv15 | **手动确认** | 占领 7 天 + 铜钱 x50000 + 元宝 x200 | x2.00 | 1500 |

> **说明**: 自动升级无需操作，条件满足后系统自动提升。"手动确认"等级需玩家在领土详情面板点击"升级领土"按钮并确认投资。产出倍率为精确值（非公式近似）。

> **[V-046] 占领时间计算说明**: 占领时间使用挂钟时间(wall-clock time)，存储为绝对时间戳在IndexedDB中，包含离线时间。即玩家关闭浏览器/切出Tab期间，占领时间仍持续累计。

> **[P2-R2-U05] 自动升级时间条件说明**: Lv1~4的时间条件以占领时刻为起点计算。
> Lv5以后的手动确认升级，后续自动升级阶段（Lv6~9、Lv11~14）的时间条件以该确认时刻为起点。
> 即: Lv1~4起点=占领时刻，Lv5~9起点=Lv5确认时刻，Lv10起点=Lv10确认时刻，Lv11~15起点=Lv10确认时刻。

**[P2-M-14] 定时检查频率**: 每分钟检查一次是否有领土满足自动升级条件（Lv1~4）。
同时，每次打开天下Tab时基于时间戳差值计算所有自动升级结果。

### 步骤2: 系统检查 — 条件验证

> 📖 参考: [PRD v2 MAP-3 升级条件](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 满足升级条件时，领土详情面板中升级按钮高亮/闪烁提示

**系统行为**:
1. 验证占领时间是否满足当前等级要求
2. 验证投资资源是否充足
3. 验证领土当前归属（仍为己方）

### 步骤3: UI反馈 — 升级按钮可用

> 📖 参考: [PRD v2 MAP-3 升级预览](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 升级按钮从灰色变为可点击 → 操作 可点击 → 反馈 显示升级预览

**系统行为**:
1. 领土详情面板中"升级"按钮从灰色变为可点击状态
2. 显示升级预览（当前等级→目标等级、产出倍率变化、驻防上限变化、投资消耗明细）
3. **[P2-20]** 等待期间按钮显示倒计时"还需 HH:MM:SS"，倒计时归零后按钮变为可点击

### 步骤4: 玩家操作 — 点击"升级领土"

> 📖 参考: [PRD v2 MAP-3 升级操作](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 升级按钮 → 操作 点击 → 反馈 弹出确认弹窗

**系统行为**:
1. 点击领土详情面板中的"升级"按钮
2. 弹出确认弹窗（消耗明细 + 预期收益）

### 步骤5: 系统执行 — 扣除资源并提升

> 📖 参考: [PRD v2 MAP-3 等级提升执行](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 等待短暂执行时间（按钮置灰显示loading状态，<0.3s完成后自动恢复）

**系统行为**:
1. 扣除投资资源（同一本地存储原子操作）
2. 领土等级提升
3. 产出倍率更新（具体倍率映射详见PRD）
4. 驻防上限更新

**[P1-36] 产出倍率公式与表格不一致修正**:
产出倍率以"15级逐级提升规则表"中的精确值为准。旧公式 `倍率(n) = 1.0 + (n-1) x 0.05 (n<=10)` 为近似值，Lv10处公式值1.45与表格值1.50不一致，实际实现应使用表格中的精确数据。该公式已废弃，仅保留表格。

### 步骤6: UI反馈 — 升级完成

> 📖 参考: [PRD v2 MAP-3 升级完成反馈](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

**用户感知**: 看到 等级标签跳动更新 → 操作 无需操作 → 反馈 产出数值和驻防上限更新

**系统行为**:
1. 等级标签更新（带数字跳动动画）
2. 产出数值更新（倍率提升）
3. 驻防上限更新
4. Toast 提示"领土已升级至 Lv.X"

**[P2-M-02] 自动升级通知（Lv1~4自动升级时）**: 地图上对应领土的升级图标闪烁（3次脉冲动画），不弹Toast。
玩家下次进入该领土详情面板时，面板顶部显示"已升级至 Lv{N}"轻提示条。

**[P2-R3-19] 自动升级通知与触发机制**:
- **触发时机**: 进入天下Tab时统一检查所有己方领土的升级条件（不使用定时器轮询）
- **升级条件**: `当前时间 - occupiedAt >= 升级所需时间`（occupiedAt 为占领时间或上次升级时间）
- **用户通知**: 升级发生时在地图对应领土上方短暂显示 "+1" 升级动画（1.5s 后消失），并推送轻量通知 "XX领土已升级至Lv.X"
- **定时检查频率**: 无定时器。每次进入天下Tab、从其他Tab返回、攻城胜利后各检查一次

**触发事件**:

| 编号 | 类型 | 描述 |
|------|------|------|
| TE-MAP-02 | 条件 | 领土等级提升(占领时间达标) |
| TE-MAP-06 | 条件 | 领土占领冷却结束(24h) |

> 📖 参考: [PRD v2 MAP-3 投资消耗](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

## 异常流程汇总

| 异常场景 | 处理方式 |
|---------|---------|
| 资源不足(铜钱/元宝) | 升级按钮置灰，提示"资源不足" |
| 占领时间未达标 | 升级按钮置灰，显示倒计时"还需 XX:XX:XX" |
| 领土已被敌方占领 | 取消升级，提示"领土已丢失" |
| 升级事务失败 | 资源回滚，提示"升级失败，请重试" |
| 领土Lv15(满级) | 不显示"升级"按钮或显示"已满级" |
| 📱 手机端确认弹窗 | Bottom Sheet 弹出消耗确认，底部固定"确认升级"按钮 |
