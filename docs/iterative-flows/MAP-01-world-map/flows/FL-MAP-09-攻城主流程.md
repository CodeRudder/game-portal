# FL-MAP-09 攻城战 主流程

> **版本**: v1.1.R4 | **最后更新**: 2026-05-05
> **流程类型**: L1 主流程（全局路线图）
> **归属功能**: 天下地图 — 攻城战系统
> **触发条件**: 玩家在世界地图点击敌方/中立城池，点击"攻占"按钮
> **结束条件**: 攻城结算完成、编队回城到达、兵力归还、编队销毁
> **前置状态**: 玩家已打开天下地图 (FL-MAP-01)，处于地图浏览状态 (FL-MAP-02)
> **后置状态**: 攻城结果已记录、奖励已暂存/领取、编队已销毁、将领已释放

## 阶段流转

### Phase 1: 攻占准备

> **映射 Stage**: [P1 选择攻城目标](FL-MAP-09-siege-warfare.md#stage-p1-选择攻城目标生命周期步骤-12) → [P2 攻城确认弹窗](FL-MAP-09-siege-warfare.md#stage-p2-攻城确认弹窗生命周期步骤-3) → [P3 选择/创建编队](FL-MAP-09-siege-warfare.md#stage-p3-选择创建编队生命周期步骤-4) → [P4 选择攻城策略](FL-MAP-09-siege-warfare.md#stage-p4-选择攻城策略并确认) → [P5 创建攻城任务](FL-MAP-09-siege-warfare.md#stage-p5-创建攻城任务生命周期步骤-56)

**用户线性链**: 选择城池/资源点 → 点击攻占 → 攻占弹框 → 选择/创建编队 → 创建攻占任务 → 显示攻占任务编队/时间/状态

| 属性 | 说明 |
|------|------|
| **进入条件** | 玩家已打开天下地图 (FL-MAP-01)，处于地图浏览状态 (FL-MAP-02) |
| **阶段目标** | 选定攻城目标、组建出征编队、选择攻城策略、创建攻占任务 |
| **退出条件** | 攻占任务已创建（SiegeTask 状态: preparing → marching），行军精灵已生成，任务面板已更新 |

**关键子步骤**:

| 步骤 | 对应 Stage | 用户感知 | 时间 |
|------|-----------|---------|------|
| 1. 选择城池/资源点 | P1 | 点击城池，高亮选中，弹出领土详情面板 | 即时 |
| 2. 点击攻占 | P1 | 点击"攻占"按钮，进入确认弹窗 | 即时 |
| 3. 攻占弹框 | P2 | 看到条件校验结果（绿勾/红叉），确认继续 | 即时 |
| 4. 选择/创建编队 | P3 | 选择将领、分配兵力、查看战力预览 | <1min |
| 4b. 选择攻城策略 | P4 | 选择策略卡片（强攻/围困/夜袭/内应），查看效果预览 | <30s |
| 5. 创建攻占任务 | P5 | 任务面板更新，城池标记"攻城中" | <1s |
| 6. 显示任务信息 | P5 | 右上角任务面板显示编队摘要/预计时间/状态标签 | 持续 |

**核心校验（Stage P2）**:
- 领土相邻（有道路连接）
- 兵力满足门槛
- 粮草充足（消耗 = 基础消耗(5/秒) x 行军时间(秒) x (1 + 编队兵力/1000)）
- 每日攻城次数未达上限（3次/天，本地时间00:00重置）
- 攻城冷却已过（4小时，同一目标独立计算）
- 活跃攻城任务未达并发上限（最多3个）

**异常跳转**:

| 异常场景 | 处理方式 | 跳转目标 |
|---------|---------|---------|
| 点击己方城池 | 进入领土详情，不触发攻城 | [FL-MAP-06 领土详情](FL-MAP-06-territory-detail.md) |
| 点击资源点 | 进入领土征服流程 | [FL-MAP-07 领土征服](FL-MAP-07-territory-conquest.md) |
| 条件校验不满足（兵力/粮草/次数/冷却） | 弹窗红色提示，列出未通过项，不可进入下一步 | 留在 P2 |
| 多条校验不满足 | 按优先级推荐: (1)兵力不足→补充兵力 (2)粮草不足→获取粮草 (3)次数耗尽→明日再战 | 留在 P2 |
| 并发上限达峰（活跃任务 >= 3） | 弹出提示面板，列出当前任务状态 | 留在 P2 |
| 仅剩1座城池 | 显示警告"若攻城失败且失去该城池将导致编队解散并损失50%兵力" | 留在 P2，继续确认 |
| 所有将领不可用（受伤/出征） | 编队选择面板显示空状态: 受伤将领恢复倒计时 + 出征将领返回倒计时 + 预计等待时间 | 留在 P3 |
| 夜袭令/内应信未持有 | 对应策略卡片置灰，显示道具获取途径提示 | 留在 P4 |
| 内应信已暴露 | 内应卡片置灰，显示冷却倒计时 + "等待内应恢复中" | 留在 P4 |
| 玩家取消/返回 | 关闭弹窗，回到地图空闲状态 | 地图浏览 (FL-MAP-02) |
| 原子事务失败（扣资源失败） | 全部回滚，提示重试 | 留在 P4 |

### Phase 1 → Phase 2: 攻占准备 → 行军推进

**转换触发**: Stage P5 完成 → SiegeTask 状态从 preparing 变为 marching → 行军精灵在出发城市生成

**转换动作**:
1. 任务面板新增任务卡片（编队摘要 + 状态标签 + 预计到达时间）
2. Toast 通知"任务已创建，行军出发！"
3. 行军精灵起始位置短暂高亮（500ms脉冲动画）
4. 目标城池标记"攻城中"

**[R4-ISS-006] 过渡动画序列**:
1. P5创建后，攻城确认弹窗自动关闭（延迟300ms + 300ms滑出动画）
2. Toast 通知"任务已创建，行军出发！"从顶部滑入（200ms）
3. 地图视口自动平移至行军精灵起始位置（500ms缓动动画）
4. 行军精灵在起始位置脉冲高亮500ms引导注意力（放大120% → 恢复100%，重复2次）
5. 行军精灵开始沿路线移动，进入 Phase 2

### Phase 2: 行军推进

> **映射 Stage**: [P6 行军出发](FL-MAP-09-siege-warfare.md#stage-p6-行军出发生命周期步骤-710)
> **关联流程**: [FL-MAP-04 行军动画](FL-MAP-04-march-animation.md)

**用户线性链**: 地图显示行军精灵 → 点击行军精灵或攻占任务编队 → 显示行军线路 → 等待移动到目标附近

| 属性 | 说明 |
|------|------|
| **进入条件** | 攻占任务已创建（Phase 1 完成），行军精灵已生成 |
| **阶段目标** | 编队行军至目标城池，行军精灵沿道路路径移动 |
| **退出条件** | 行军精灵到达目标城池附近，触发到达判定 |

**关键子步骤**:

| 步骤 | 对应 Stage | 用户感知 | 时间 |
|------|-----------|---------|------|
| 7. 地图显示行军精灵 | P6 | 行军精灵队列沿道路路径移动 | 10s~60s |
| 8. 点击行军精灵/任务编队 | P6 | 弹出迷你面板（剩余时间/进度/预计到达） | 即时 |
| 9. 显示行军线路 | P6 | 地图高亮黄色虚线行军路线 | 持续 |
| 10. 等待到达目标附近 | P6 | 精灵逐步靠近目标城池 | 10s~60s |

**⏱ 时间约束**: 行军动画时长严格落在 [10s, 60s]，由时长约束算法调节实际速度

**行军精灵规范**:
- 精灵数量按兵力分档
- 速度有地形修正（平原/山地/森林/沙漠/关隘/雪地各有系数）
- A*寻路计算最短路径
- 并行行军有渲染优先级规则（详细规则见 FL-MAP-09-09 [P2-M-13]）**[R4-ISS-014]**

**[R4-ISS-007] 行军期间全局进度指示**:
- 右上角任务面板常驻显示（不依赖用户点击精灵），包含：进度条+剩余时间+状态标签"行军中"
- 行军到达前5s显示倒计时 Toast "即将到达目标，{N}s..."
- 行军精灵在视口外时，屏幕边缘指示器常驻显示（黄色箭头+距离标注）
- 行军精灵点击热区：点击后弹出迷你面板显示剩余行军时间/当前进度/预计到达

**异常跳转**:

| 异常场景 | 处理方式 | 跳转目标 |
|---------|---------|---------|
| 玩家取消行军（仅 marching 状态） | 资源和道具 100% 退还，不计入每日次数，不触发冷却，行军精灵淡出移除 | **攻城取消 → 地图空闲** |
| 应用切后台 | 行军继续（本地状态持久化），返回时跳到精灵最新位置 | 留在 Phase 2 |
| 客户端崩溃 | 从任务面板恢复，位置根据本地时间戳计算 | 留在 Phase 2 |
| 页面不可见恢复 | 页面重新可见后从本地存储恢复行军状态 | 留在 Phase 2 |
| 恢复超时（**[R4-ISS-008]** 30s移动/10sPC） | **[R4-ISS-010]** 按攻城阶段撤退处理（50%资源退还，消耗每日次数，5分钟冷却）。marching阶段恢复超时规则与主动取消不同（主动取消=100%退还） | Phase 4（中断结算） |

### Phase 3: 攻城战斗

> **映射 Stage**: [P7 到达→攻城动画切换](FL-MAP-09-siege-warfare.md#stage-p7-到达攻城动画切换生命周期步骤-11) → [P8 攻城战斗过程](FL-MAP-09-siege-warfare.md#stage-p8-攻城战斗过程生命周期步骤-12)
> **关联流程**: [FL-MAP-05 攻城动画](FL-MAP-05-siege-animation.md), [FL-MAP-17 伤亡系统](FL-MAP-17-casualty-system.md)

**用户线性链**: 切换为攻占动画（精灵+粒子+城池） → 等待攻占过程（按回合掉血）

| 属性 | 说明 |
|------|------|
| **进入条件** | 行军精灵到达目标城池附近（Phase 2 完成），集结动画触发 |
| **阶段目标** | 通过回合制攻城战斗决定胜败 |
| **退出条件** | 战斗结束 — 城防归零(胜利) / 20回合超时(失败) / 攻方撤退(中断) / 恢复超时(中断) |

**关键子步骤**:

| 步骤 | 对应 Stage | 用户感知 | 时间 |
|------|-----------|---------|------|
| 11a. 过渡动画 | P7 | 行军精灵淡出 → 攻城场景淡入 → 集结动画（军队汇集+城墙旗帜+战鼓） | 即时过渡 + 3s集结 |
| 11b. 策略攻城动画 | P7 | 按策略播放差异化攻城动画（军队精灵+粒子+城池），循环播放 | 5~15s/循环 |
| 12a. 回合制战斗 | P8 | 城防血条实时下降（绿→黄→红闪烁），攻方战力条，回合计数器 | 10s~60s（最长20回合） |
| 12b. 城防衰减计算 | P8 | 本地战斗引擎回合制计算，UI帧级平滑插值 | 每回合1s |

**⏱ 时间约束**: 总战斗时长落在 [10s, 60s]，攻城动画循环播放直到战斗引擎计算结束

**攻城动画循环**:
- 强攻: 5s/循环
- 围困: 15s/循环
- 夜袭: 8s/循环
- 内应: 10s/循环

**城防衰减公式**: 每回合城防减少 = 攻方总战力 x 策略修正 / (城防系数 x 回合数上限)
- **[R4-ISS-001]** 攻方总战力使用 FL-MAP-16 S2 完整加法公式计算（非简化预览版）
- 策略修正（`decay_strategy_modifier`）: 强攻1.2 / 围困0.8 / 夜袭1.5 / 内应1.3
- 城防系数 = 1.0 + 城墙等级 x 0.05

**交互权限（攻城进行中）**:
- 允许: 缩放地图、拖拽查看、打开任务面板、查看实时信息、**撤退**
- 不允许: 发起新攻城、移动到其他城市、修改策略

**[R4-ISS-017] 战斗信息面板完整布局**（体验优化项，不阻塞主流程）:
- 底部进度条高度 >= 8px（替代原4px），显示城防精确百分比数字
- 战斗信息面板布局: 左侧攻方战力条 + 中间回合计数器 + 右侧城防血条（含百分比文字）
- 动态事件提示: 浮动文字显示关键战斗事件（如"暴击！城防额外下降10%"、"城墙破裂！防御降低"），每次事件显示1.5s后淡出
- 战斗事件触发概率: 暴击5%/回合，城墙破裂10%/回合（城防<50%时），均为纯视觉反馈不影响实际计算

3. 启动结算后执行序列（严格按序）:

   - 生成攻占结果事件
   - 编队回城出发

### Phase 4: 结算回城

> **映射 Stage**: [P9 攻城结果结算](FL-MAP-09-siege-warfare.md#stage-p9-攻城结果结算生命周期步骤-1315) → [P10 编队回城](FL-MAP-09-siege-warfare.md#stage-p10-编队回城生命周期步骤-1618)
> **关联流程**: [FL-MAP-11 攻城奖励领取](FL-MAP-11-siege-rewards.md), [FL-MAP-17 伤亡系统](FL-MAP-17-casualty-system.md)

**用户线性链**: 攻占结果动画 → 攻占结果结算/增加/扣减 → 生成攻占结果事件 → 编队状态更新(血条/轻伤/重伤) → 自动回城 → 军队精灵按线路移动回出发城市

| 属性 | 说明 |
|------|------|
| **进入条件** | 攻城战斗结束（Phase 3 完成），战斗结果已确定 |
| **阶段目标** | 结果结算（伤亡+奖励+事件）、编队状态更新、编队回城、兵力归还、编队销毁 |
| **退出条件** | 编队回到出发城市（或最近己方城市），剩余兵力归还全局兵力池，受伤将领开始恢复，编队对象销毁 |

**关键子步骤**:

| 步骤 | 对应 Stage | 用户感知 | 时间 |
|------|-----------|---------|------|
| 13. 结果动画 | P9 | 胜利(旗帜更换+欢呼3s) / 失败(撤退+损失数字2s) | 2~3s |
| 14a. 伤亡计算 | P9 | 弹窗显示"正在计算伤亡..." → 伤亡报告（5档展示/3档计算） | <200ms |
| 14b. 奖励暂存 | P9 | 弹窗显示"正在计算奖励..." → 奖励明细（策略倍率修正） | <200ms |
| 14c. 结果弹窗 | P9 | SiegeResultModal: 胜败状态+伤亡详情+奖励明细+道具掉落 | 用户点击确认 |
| 15. 生成结果事件 | P9 | 事件列表新增条目（保留24h，可点击查看详情） | 持续24h |
| 16. 编队状态更新 | P10 | 血条按伤亡变色(绿/黄/红)，将领受伤标记(轻伤/中伤/重伤) | 即时 |
| 17. 自动回城 | P10 | 编队自动开始回城行军，回城精灵出发 | 即时出发 |
| 18. 回城行军 | P10 | 回城精灵沿去程路径逆序移动（半透明显示） | 10s~60s |

**⏱ 时间约束**: 结果动画2~3s + 回城行军10s~60s

**结算后执行序列（严格按序）**:

```
Stage P9 触发
  |
  +-- 1. 播放结果动画（胜利3s/失败2s）            ← FL-MAP-05
  |
  +-- 2. 伤亡计算                                  ← FL-MAP-17 S1~S4
  |     +-- 5档→3档映射
  |     +-- 计算士兵损失比例
  |     +-- 判定武将受伤
  |     +-- 应用伤亡（原子操作）
  |     **[R4-ISS-018]** 计算时间<200ms时跳过"正在计算伤亡..."中间加载态，
  |     使用数字滚动动画（伤亡数字从0递增到最终值，持续300ms）代替文字加载提示
  |
  +-- 3. 奖励计算并暂存为 pendingReward（24h有效）  ← FL-MAP-11
  |     **[R4-ISS-018]** 计算时间<200ms时跳过"正在计算奖励..."中间加载态，
  |     使用数字滚动动画（奖励数字从0递增到最终值，持续300ms）
  |
  +-- 4. 显示攻占结果弹窗 (SiegeResultModal)       ← FL-MAP-11
  |
  +-- 5. 玩家点击"领取奖励"                        ← FL-MAP-11（原子事务写入）
  |
  +-- 5b. 生成攻占结果事件（保留24h）              ← 生命周期步骤15
  |
  +-- 6. 编队回城出发                              ← FL-MAP-09 Stage P10
        +-- 更新编队状态
        +-- 自动生成回城行军精灵
        +-- 回城精灵逆序移动（10s~60s）
        +-- 到达: 兵力归还 → 将领释放 → 编队销毁
```

**回城目的地**: 最近己方城市（欧几里得距离最近）。若目标城池刚被攻下即为最近己方城市，则返回出发城市。若无己方城市（极端），编队就地解散，兵力损失50%。

**[R4-ISS-019] 回城后视口行为**:
1. 弹窗关闭后地图停留在当前视口位置（不自动跳转）
2. 回城精灵半透明度 = 60%（opacity 0.6，可见但区分于正常精灵）
3. 右上角任务面板持续显示回城进度条和预计到达时间（不仅依赖Toast）
4. 回城到达时使用持久性通知卡片（非Toast），显示"编队已返回{城市名}"，需用户手动关闭或5s后自动消失

**编队生命周期终点**（统一声明）:
1. 剩余兵力归还全局兵力池（unallocated）
2. 武将释放（恢复可用状态，受伤武将进入恢复期）
3. 编队对象销毁（从编队列表移除）

**异常跳转**:

| 异常场景 | 处理方式 | 跳转目标 |
|---------|---------|---------|
| 奖励领取事务失败 | 显示重试按钮，pendingReward 仍保留 | 留在弹窗 |
| 24h未领取奖励 | 自动发放到账户，邮件通知 | 完成 |
| 背包溢出（道具） | 溢出道具存入邮箱系统（7天有效），Toast提示 | 完成 |
| 回城行军期间应用切后台 | 行军继续（本地状态持久化），返回时跳到最新位置 | 留在 Phase 4 |
| 回城行军期间客户端崩溃 | 从任务面板恢复，位置根据本地时间戳计算 | 留在 Phase 4 |
| 无己方城市可返回（极端） | 编队就地解散，兵力损失50% | 完成（异常终止） |
| 回城到达 → 完成 | Toast"编队已返回{城市名}"，任务面板移除卡片 | **攻城结束 → 地图空闲** |
