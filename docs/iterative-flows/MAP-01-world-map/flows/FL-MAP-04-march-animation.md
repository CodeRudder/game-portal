# FL-MAP-04 行军动画

> **版本**: v3.3 [R3] | **最后更新**: 2026-05-05
> **归属功能**: 天下地图行军系统 — 路线计算、精灵动画、路线预览
> **触发条件**: FL-MAP-09-P6（行军出发阶段）内部调用，传入 SiegeTask 对象（含起点城市/目标城市/编队/策略）
> **结束条件**: 行军精灵到达目标城市，通过 MarchToSiegeTransition 过渡数据结构传递至 FL-MAP-09-P7（攻占动画切换）
> **前置状态**: 攻占任务已创建（FL-MAP-09-P5 完成），玩家已确认行军，路线已计算
> **后置状态**: 行军精灵消失，攻城动画启动（FL-MAP-09-P7），控制权移交攻城流程
> **来源文档**: [PLAN.md](../PLAN.md)

## 关键要求与验收标准

### 功能要求
- FR-04-01: 接收起点城市ID与终点城市ID，使用A*算法沿道路网络计算最短路径，返回路径节点列表、总距离与预计行军时间
- FR-04-02: 行军战略时间（游戏内消耗）与行军动画时长（视觉播放时间）独立计算，动画时长硬约束在 [10s, 60s] 范围内
- FR-04-03: 在起点城市生成行军精灵队列，精灵沿路径节点移动并根据方向切换朝向帧，地形实时影响移动速度
- FR-04-04: 在像素地图Canvas上绘制黄色虚线高亮路线，在E区弹出路线预览面板显示路线信息、军队信息与路径地形分布
- FR-04-05: 行军中途可被敌方拦截，触发拦截战斗；玩家可主动撤退使精灵原路返回
- FR-04-06: 行军到达目标城市后销毁精灵，构造 MarchToSiegeTransition 过渡数据传递至攻城动画流程

### 非功能要求
- NFR-04-01: A*寻路计算耗时 <100ms
- NFR-04-02: 行军精灵动画帧率稳定在 30fps 以上
- NFR-04-03: 超长行军（>60s）期间允许玩家继续操作其他功能（关闭面板、浏览地图、管理领土）

### 验收标准
- AC-04-01: 选择任意两个可达城市后，系统在 100ms 内返回有效路径（路径节点列表非空且连续）
- AC-04-02: 行军动画时长严格落在 [10s, 60s] 范围内，超长行军允许超过但需弹通知
- AC-04-03: 精灵根据移动方向正确切换至少4个朝向的帧动画，无闪烁或跳帧
- AC-04-04: 拦截事件触发后精灵在0.5s内停止并显示红色警报闪光，拦截战斗面板正常弹出
- AC-04-05: 路线预览面板正确显示行军战略时间和动画预计时间两项独立数值

## 步骤详情

### FL-MAP-04-01 路线计算

> 📖 参考: [PRD MAP-3 行军路线与寻路](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

> **来源**: flows.md [MAP-F02b-01]

**用户感知**: 选择目标城市后 → 看到黄色虚线高亮路线和预计时间 → 可查看路径地形分布

**系统行为**:
1. 接收输入：起点城市ID + 终点城市ID
2. 执行 A* 寻路算法，沿道路网络计算最短路径，A*参数（f/g/h值公式）详见PRD
3. 对路径每个节点计算地形移动成本，各地形类型的moveCost和速度修正详见PRD
4. 汇总路径总距离和预计行军时间
5. 返回路径节点列表 + 总距离 + 预计时间

**游戏时间系数定义** (ISS-004, P0-06):
- **游戏时间系数 = 1.0**（默认值，即 1px 路径距离 = 1 秒战略时间当量）
- **行军战略时间（游戏分钟）** = `路径距离(px) / BASE_SPEED(30) x 60`
  - 例：路径距离 900px → 战略时间 = 900/30 x 60 = 1800 游戏分钟（即 30 游戏小时）
- **行军动画时长（秒）** = `路径距离(px) / actualSpeed(px/s)`
  - actualSpeed 受时长约束算法调节，保证动画时长落在 [10s, 60s] 范围内
- **行军战略时间（游戏内消耗）与行军动画时长（视觉播放时间）独立计算**
- 动画时长硬约束在 [10s, 60s] 范围内，超长行军允许超过但需弹通知

**地形移动成本表 (TERRAIN_MOVE_COST)**:

| 地形类型 | moveCost | 说明 |
|---------|:--------:|------|
| 平原 | 1.0 | 基准速度 |
| 森林 | 1.5 | 中等减速 |
| 山地 | 2.0 | 较大减速 |
| 沙漠 | 1.8 | 中大减速 |
| 水域 | 2.14 | 最大减速 |
| snow (雪地) | 1.6 | 移动缓慢，视野受限 |

**A*寻路坐标单位说明** (ISS-023):
- h值使用色块坐标（整数）计算曼哈顿距离，g值也为色块单位
- PathNode 中 px=x*8, py=y*8 为像素坐标，仅用于渲染
- 色块坐标用于寻路逻辑计算，像素坐标用于精灵渲染定位

**[P2-R3-13] A* 寻路参数定义**:

```
f(n) = g(n) + h(n)
g(n) = g(parent) + moveCost[current_terrain]
h(n) = Manhattan距离(current, goal) = |x1-x2| + |y1-y2|
```

- f/g/h 权重均为 1.0（标准A*）
- 启发式函数: 曼哈顿距离（色块坐标）
- moveCost 使用 TERRAIN_MOVE_COST 表（见本文件地形移动成本表）
- 对角线移动: 允许相邻tile的对角线移动，提升视觉平滑移动效果
- 最大搜索节点: 76800（320x240全地图）
- 无路径时: 返回空路径，提示"无法到达目标领土"

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 起点与终点相同 | 拒绝计算，提示"已在该城市" | — |
| 无可达路径（地形阻断） | 提示"无法到达目标城市。需要先征服途中的领土才能到达目标城市" | — |
| 路径跨越敌方城市 | A* 自动绕行，若无法绕行则提示"路径被阻断" | — |

### FL-MAP-04-02 精灵动画

> 📖 参考: [PRD MAP-3 行军精灵与动画](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-3)

> **来源**: flows.md [MAP-F02b-02]

**用户感知**: 看到 像素小人队列沿道路移动 → 小人根据方向切换帧动画 → 到达目标城市后精灵消失

**系统行为**:
1. 在起点城市生成行军精灵队列（数量由兵力决定，具体分档规则详见PRD）
2. 精灵沿 FL-MAP-04-01 计算的路径节点移动
3. 每帧根据移动方向切换精灵朝向帧
4. 地形影响实时速度（各地形速度修正详见PRD）
5. 时长约束算法确保动画时长落在有效范围内
6. 到达目标城市后销毁精灵，触发攻城过渡

**精灵规格**:
- 精灵尺寸、行走循环帧数、动画帧率、方向数、间距等参数详见PRD
- 精灵调色板（各阵营主色、肤色、轮廓色、高光色）详见PRD

> **[INT-R2-07] 交叉引用**: 行军创建反馈面板与 FL-MAP-09 Stage P5 攻城创建反馈使用同一UI组件（任务面板卡片）。

**[P1-M-06] 行军精灵点击热区**: 精灵实际尺寸外扩至最小44x44px触摸热区（参考FL-MAP-03-02城市点击热区定义）。缩放比例<0.5时热区不随缩放缩小，保持最小44x44px。热区为透明矩形，不显示边框

**行军速度与时长约束**:
- 基准行军速度、精灵帧速度、色块速度、渲染帧速度等参数详见PRD
- 时长约束算法（自然时长计算、速度调整、速度边界值）详见PRD

**超长行军处理**（actualSpeed 达到最大速度仍超过最长时长）:
- 允许玩家在行军期间操作其他功能（关闭面板、浏览地图、管理领土）
- 行军精灵持续显示，实时更新位置
- 到达目标时弹出通知，需手动确认后继续

**行军进度查看入口** (P1-04):
- **行军精灵可点击**: 点击行军精灵弹出迷你面板，显示"剩余行军时间"/"当前进度百分比"/"预计到达时间"
- **任务面板入口**: 行军期间屏幕右上角显示行军任务图标（含剩余时间倒计时），点击展开任务列表面板
- **视口外精灵边缘指示**: 行军精灵在视口外时，屏幕边缘显示黄色箭头指示器（箭头朝精灵方向），边缘指示器距离显示为游戏内度量: "约X秒行军距离"（基于当前行军速度和像素距离估算），点击箭头自动将视口平移至精灵位置

**行军拦截状态机定义** (ISS-024, P0-V01):

> **AI 敌方系统定义**: 见 [PLAN.md 附录 A: AI 敌方系统定义](../PLAN.md#附录-a-ai-敌方系统定义) — 行军拦截触发概率 5%/次行军（PLAN.md A.6），拦截兵力 = AI 领土驻防兵力 x 20%

行军拦截由以下状态和转换构成：

| 状态/事件 | 描述 |
|----------|------|
| **拦截触发（V-010修复）** | 行军过程中，客户端本地引擎以每小时10%概率判定是否触发拦截事件（本地随机事件，非多人游戏机制）。拦截时生成AI敌方编队（兵力=当前行军兵力 x 0.6~0.9随机），玩家选择迎战或撤退 |
| **拦截视觉反馈** | 精灵停止 + 红色警报闪光(0.5s) + 屏幕震动 |
| **拦截后处理** | 弹出拦截战斗面板（简版FL-MAP-07战斗），胜利后继续行军，失败则回城 |

**拦截战斗交互定义**:

拦截战斗为简版征服流程（非完整攻城战），交互如下：

1. **拦截触发时**: 地图蒙层 + 红色警告横幅从顶部滑入（300ms），显示"遭遇拦截！"
2. **战斗面板**: 底部弹出面板（40%屏幕高度），显示：
   - 敌方信息：AI 编队兵力 + 领土来源
   - 我方信息：当前编队兵力 + 将领
   - 战力对比条
3. **操作选项**:
   - [迎战]（红色主按钮）：触发简化战斗计算（1回合，基于战力对比计算胜负）
   - [撤退]（灰色次按钮）：编队沿原路返回出发城市
4. **战斗时长**: 10s（简化版，非攻城的 10~60s）
5. **结果**: 胜利→继续行军；失败→编队损失 20%~40% 兵力，自动撤退

**[P0-01-R2] 敌方行军实体数据来源**:
行军拦截为纯客户端本地判定。敌方行军编队由本地 AI 引擎按以下规则生成:
- AI 引擎每隔 30~60 秒检查一次己方领土周围的可行动军目标
- AI 行军编队的兵力 = 目标城池驻防兵力 x 0.6~0.8（随机系数）
- AI 行军路线使用与玩家相同的 A* 寻路算法
- 拦截判定: 玩家行军路径与 AI 行军路径在移动过程中产生交叉（色块坐标距离 <= 2）时触发
- 拦截战斗结果由客户端本地战斗引擎即时计算（不使用攻城动画，即时结算）

状态机转换流程：
1. MARCHING → 行军引擎判定拦截条件满足 → INTERCEPTED（精灵停止 + 红色警报闪光 0.5s + 屏幕震动）
2. INTERCEPTED → 弹出拦截战斗面板（简版 FL-MAP-07 战斗）
3. 拦截战斗胜利 → 恢复 MARCHING，继续行军
4. 拦截战斗失败 → RETREATING，精灵原路返回起点（撤退规则详见FL-MAP-09 P6行军取消规则：资源和道具100%退还，不计入每日次数）

**异常分支**:

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 行军中途被敌方拦截 | 精灵停止，状态机转入 INTERCEPTED → BATTLE | — |
| 玩家点击"撤退"按钮 | 精灵原路返回，状态机转入 RETREATING → IDLE | — |
| 行军到达目标 | 精灵销毁，构造 MarchToSiegeTransition 传递至 FL-MAP-09 对应攻城阶段 | FL-MAP-05 |
