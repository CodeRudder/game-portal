## 关键要求与验收标准

### 功能要求
- FR-13-01: 统计面板展示6个维度：领土概览/势力分布/资源产出/战斗统计/探索进度/事件参与
- FR-13-02: 支持维度Tab切换，点击领土项可跳转到FL-MAP-06（领土详情）
- FR-13-03: 我方领土列表按产出排序，总产出汇总实时更新
- FR-13-04: 面板支持折叠为标题栏和关闭操作

### 非功能要求
- NFR-13-01: 数据查询耗时 < 300ms
- NFR-13-02: 手机端Bottom Sheet 50%高度，内容可滚动
- NFR-13-03: 数据读取异常时使用上次成功加载的数据

### 验收标准
- AC-13-01: 点击统计图标后面板在0.3s内渲染完成
- AC-13-02: 各维度Tab切换后数据正确展示
- AC-13-03: 数据加载失败时显示"统计加载失败"+"重试"按钮
- AC-13-04: 无领土数据时显示空状态提示

## 流程结构

```
玩家点击统计入口
  ⏱ 0s ─── 面板加载(数据查询)
  ⏱ 0.3s ── 渲染统计面板
  ⏱ 用户 ── 查看详情/切换维度
  ⏱ 用户 ── 关闭面板
  📱 手机端: 右上角📊图标 → Bottom Sheet 50%高度
```

## 步骤详情

### 步骤1: 玩家操作 — 打开统计面板 ⏱<200ms

> 📖 参考: [PRD v2 MAP-6 地图统计](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-6)

**用户感知**: 看到 统计图标 → 操作 点击 → 反馈 统计面板弹出

**系统行为**:
1. PC端：点击地图右上角统计图标
2. 手机端：点击右上角统计图标 → Bottom Sheet 50%高度弹出

### 步骤2: 系统响应 — 数据查询

> 📖 参考: [PRD v2 MAP-6 数据来源](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-6)

**用户感知**: 短暂加载后数据呈现

**系统行为**:
1. 从多个子系统查询统计数据（领土/战斗/资源/探索/事件），各数据来源和更新频率详见PRD
2. 汇总查询结果
3. 面板底部显示数据时效性指示: "数据更新于 X 分钟前"（时间戳随数据刷新更新）

### 步骤3: UI反馈 — 统计面板展示

> 📖 参考: [PRD v2 MAP-6 统计维度与布局](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-6)

**用户感知**: 看到 统计面板（领土概览/势力分布/资源产出等） → 操作 浏览数据 → 反馈 数据实时显示

**系统行为**:
1. 渲染统计面板，面板布局和尺寸详见PRD
2. 展示统计维度（领土概览/势力分布/资源产出/战斗统计/探索进度/事件参与），各维度的数据项和说明详见PRD
3. 我方领土列表按产出排序，显示名称、等级、主要产出
4. 总产出汇总实时更新

### 步骤4: 玩家操作 — 查看详情 ⏱<100ms

> 📖 参考: [PRD v2 MAP-6 交互操作](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-6)

**用户感知**: 看到 各维度Tab → 操作 点击切换/点击领土项 → 反馈 详情展示或跳转（维度切换耗时<100ms）

**系统行为**:
1. 切换维度：点击不同统计维度Tab切换显示
2. 领土详情：点击某领土项跳转到 FL-MAP-06（领土选择与详情）
3. 面板操作：[—] 折叠为标题栏 / [x] 关闭面板

## 异常流程汇总

| 异常场景 | 处理方式 |
|---------|---------|
| 数据加载失败 | 显示"统计加载失败" + "重试"按钮 |
| 无领土数据 | 显示空状态 "暂无领土" + 引导文案 |
| 数据读取异常 | 使用上次成功加载的数据，标注"数据加载异常" |
| 面板折叠/展开 | [—] 折叠为标题栏，再次点击展开恢复 |
| 📱 手机端空间不足 | Bottom Sheet 50%高度，内容可滚动 |
