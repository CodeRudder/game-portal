## 关键要求与验收标准

### 功能要求
- FR-12-01: 地图事件系统触发概率为每小时10%，最多3个未处理事件同时存在。**[V-043]** 触发检查频率: 每游戏分钟检查一次，每次概率约0.167%（等效每小时10%）
- FR-12-02: 事件类型为6种：商队遇险/流民/宝箱/山贼/遗迹发现/阵营冲突（以R4最终版为准）
- FR-12-03: 每个事件提供2~3个选择分支，各分支有不同消耗和预期奖励
- FR-12-04: 山贼战斗不使用编队系统，战斗后兵力自动归还
- FR-12-05: 选择"战斗"前显示胜率预估

### 非功能要求
- NFR-12-01: 事件生成判定在后台完成，不影响前端性能
- NFR-12-02: 事件图标脉冲动画持续显示直到玩家处理或事件过期

### 验收标准
- AC-12-01: 地图上正确显示脉冲闪烁事件图标，点击后弹出事件交互面板
- AC-12-02: 选择分支后0.5s内显示结果（奖励/惩罚/无效果）
- AC-12-03: 山贼战斗前显示预估胜率和可能损失兵力
- AC-12-04: 内应信掉落概率按PRD设定正确触发

## 子流程 FL-MAP-12-02: 事件战斗分支机制

> **旧流程**: MAP-F09-02

### 流程步骤

```
FL-MAP-12-02 事件战斗分支机制
├── S1 ⏱<50ms   📱 计算战斗兵力（自动从全局兵力池借用）
├── S2 ⏱<50ms   📱 显示胜率预览（胜率公式同FL-MAP-07-02）
├── S3 ⏱用户     📱 玩家点击"战斗"确认
├── S4 ⏱<1s     📱 本地战斗引擎计算结果（概率判定）
│   ├── roll < 胜率 → 胜利 → S5a
│   └── roll >= 胜率 → 失败 → S5b
├── S5a ⏱<200ms 📱 胜利: 兵力归还全局兵力池 + 战利品入账
└── S5b ⏱<200ms 📱 失败: 损失兵力从全局兵力池扣除 + 无奖励
```

**步骤详情**:
- **S1 计算兵力**: **[V-035]** 出征兵力=min(玩家最近己方领土驻防兵力×80%, 全局未分配兵力×50%)，从全局兵力池临时借用，不扣除
  > **[P2-R2-U06] "最近己方领土"定义**: 距离事件触发位置（色块坐标）欧几里得距离最近的己方领土
- **S2 胜率预览**: 使用与FL-MAP-07-02相同的胜率公式计算，胜率颜色沿用胜率区间色（>80%翠绿 / 60%~80%金色 / 40%~60%琥珀橙 / <40%赤红）
- **S3 确认战斗**: 玩家点击"战斗"按钮确认（可查看预估胜率和预计损失）。**[V-044]** 战斗确认时显示出征兵力数量（"将派遣 XXX 兵力参战"）
- **S4 战斗结算**: 本地战斗引擎概率判定（roll < 胜率则为胜利），不使用编队系统
- **S5a 胜利处理**: 兵力自动归还全局兵力池 + 获得战利品（装备/道具按品质掉落）
- **S5b 失败处理**: 损失兵力 = 出征兵力 x 失败损失比例（20%~40%），从全局兵力池中扣除，无奖励

### 山贼战斗

> 📖 参考: [PRD v2 MAP-5 山贼战斗](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-5)

山贼战力计算、战斗胜率公式、胜败损耗计算等具体参数和公式详见PRD。

**[P1-M-10] [R2-Step2] 山贼战斗防御系数定义**:
山贼战斗防御系数 = 1.0 + 地形加成（无城墙项）。山贼无城墙、无城防系统，城墙等级项为0。具体说明:
- 防御系数 = 1.0 + 城墙等级 x 0.05 + 地形加成
- 山贼场景下: 城墙等级 = 0（山贼不占据城池，无城墙加成）
- 因此: 山贼防御系数 = 1.0 + 0 + 地形加成 = 1.0 + 地形加成
- 地形加成沿用FL-MAP-07-02中定义的地形修正表（平原+0%、山地+10%、森林+5%、沙漠-5%等）
- 此定义与FL-MAP-07-02防御系数公式一致，仅城墙等级项为0

**[P1-34] 山贼战斗兵力来源**:
地图事件战斗不使用编队系统（无编队创建流程）。山贼战斗兵力自动计算:
- **[V-035]** 出征兵力=min(玩家最近己方领土驻防兵力×80%, 全局未分配兵力×50%)
- 出征兵力从全局兵力池中临时借用，不扣除
- 战斗胜利: 兵力自动归还全局兵力池 + 获得战利品
- 战斗失败: 损失兵力 = 出征兵力 x 失败损失比例（20%~40%），从全局兵力池中扣除

**[P2-R3-16] 山贼战斗兵力来源**: 山贼兵力 = 触发事件领土等级 x 50 + 随机(0, 100)。不使用编队系统，即时结算（无动画播放，2s结算动画）。结算结果直接显示在事件面板中。

### 遗迹探索判定

> 📖 参考: [PRD v2 MAP-5 遗迹探索](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-5)

遗迹探索三档判定（失败/部分成功/成功）的具体阈值计算、科技加成和地形修正详见PRD。

**[P1-35] Roll区间边界规范**:
所有概率判定统一使用左闭右开区间 `[0, 阈值)` 规范:
- `roll < 阈值` → 命中该档位
- `roll >= 阈值` → 进入下一档位判定
- roll值范围: [0, 1)，均匀分布随机数
- 示例: 成功概率30% → `if (roll < 0.30) 成功`

## 异常流程汇总

| 异常场景 | 处理方式 | 触发步骤 |
|---------|---------|
| 事件未处理自动消失 | 事件存在时间到期后自动消失，未领取奖励丢失 | FL-MAP-12-01 |
| 同时超过3个事件 | 不生成新事件，等待玩家处理释放槽位 | FL-MAP-12-01 |
| 事件战斗失败 | 正常损失兵力，无奖励 | FL-MAP-12-05 |
| 山贼战斗被应用切后台中断 | 本地状态持久化，恢复后继续战斗或判定失败 | FL-MAP-12-05 |
| 📱 手机端事件弹窗 | 脉冲图标点击 → Bottom Sheet 事件弹窗，选项纵向排列 | FL-MAP-12-04 |

**[P1-46] 离线累积事件与在线3个上限的关系**:
离线累积事件使用独立的离线事件计数器，不受在线"最多3个未处理事件"上限约束。规则如下:
- 在线事件计数器上限: 3个（TE-MAP-05正常触发）
- 离线事件计数器上限: 按离线时长累积，每小时10%概率触发，累积上限 = 离线小时数（最多72h），具体规则详见PRD
- 玩家回归后: 离线事件展示在回归欢迎面板（FL-MAP-15），处理完毕后释放离线计数器
- 在线事件上限不因离线事件而减少：回归后在线事件计数器从0开始

> **[INT-R2-05] 离线定义统一**: 本流程中的"离线事件计数器"与 FL-MAP-15 使用同一阈值（300秒/5分钟）。
> 用户关闭天下Tab超过300秒后产生的事件统一标记为"离线事件"。

**离线事件视觉标识**:
离线期间累积的事件在事件列表中显示特殊标识：
- 事件卡片左侧显示蓝色竖线标记（vs 在线事件无标记）
- 事件时间戳标注"离线期间"前缀
- 事件列表顶部显示分隔线"── 离线期间 ──"
