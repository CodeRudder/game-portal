# FL-MAP-01 进入天下Tab

> **版本**: v3.4 [R3] | **最后更新**: 2026-05-05
> **归属功能**: 天下地图系统入口 — 场景路由切换与地图初始化
> **触发条件**: 玩家点击底部导航栏"天下"Tab
> **结束条件**: 世界地图完整显示，己方领土高亮，产出气泡就绪，图例组件就绪
> **前置状态**: 玩家已选择阵营（魏/蜀/吴），处于其他功能Tab页面
> **后置状态**: 世界地图场景激活，地图渲染完成，玩家可浏览/缩放/交互
> **来源文档**: [PLAN.md](../PLAN.md)
> **PRD 参考**: [MAP-world-prd-v2](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md)

## 关键要求与验收标准

### 功能要求
- FR-01-01: 玩家点击底部导航栏"天下"Tab后，系统应在300ms内完成场景路由切换
- FR-01-02: 地图数据加载期间应显示骨架屏与进度条（含百分比数字，如"加载中 67%"），正常加载1~3s，超时阈值15s。骨架屏进度条应显示预计剩余时间提示
- FR-01-03: 地图渲染初始化应在500ms内完成（视口裁剪优化）
- FR-01-04: 初始视角应自动定位到己方核心领土区域，并执行500ms平滑动画
- FR-01-05: 首次进入用户触发4步新手引导：初征→征服→攻城→布防
- FR-01-06: 产出上限机制持续运行，气泡按资源比例显示绿色/黄色/红色状态。产出上限公式详见 PRD MAP-1
- FR-01-07: 新手引导中，若目标领土不在当前视口内，应自动平移视口至目标位置（500ms动画）
- FR-01-08: 新手引导征服失败时提供重试机制，最多自动重试3次，3次失败后提供跳过选项
- FR-01-09: 一键收取产出时，超上限部分100%丢弃并Toast提示"资源已达上限，溢出部分未收取"。一键收取后进入30s冷却期，冷却期间按钮置灰并显示倒计时
- FR-01-10: Canvas不可用降级时，显示"返回上一Tab"按钮及浏览器建议文案"建议使用Chrome/Firefox浏览器"

### 非功能要求
- NFR-01-01: 场景切换耗时 ≤ 300ms
- NFR-01-02: 地图渲染耗时 ≤ 500ms
- NFR-01-03: UI组件挂载 < 200ms
- NFR-01-04: 视口状态持久化有效期24h（localStorage）
- NFR-01-05: 新手引导中断恢复通过localStorage实现，步骤不丢失

### 验收标准
- AC-01-01: 点击"天下"Tab后300ms内C区全屏替换为天下场景
- AC-01-02: 数据解析失败时显示错误面板，包含"重试"和"返回"按钮
- AC-01-03: 地图渲染完成后己方领土显示阵营色发光脉冲边框
- AC-01-04: 新手引导步骤1中，目标领土不在视口内时视口自动平移至目标（500ms动画完成）
- AC-01-05: 新手引导步骤2征服失败时显示鼓励文案，最多重试3次，3次失败后可跳过
- AC-01-06: 新手引导步骤4完成条件为滑块移动后点击"确认"按钮提交
- AC-01-07: 一键收取时超上限资源丢弃，Toast提示"资源已达上限，溢出部分未收取"
- AC-01-08: Canvas降级时显示"返回上一Tab"按钮和"建议使用Chrome/Firefox浏览器"文案
- AC-01-09: 新手引导步骤3仅要求进入攻城确认弹窗（点击"攻占"按钮），不要求完成完整攻城流程
- AC-01-10: 产出上限公式验证：产出上限 = 基础上限(1000) + 领土等级 x 200 + 科技加成

## 步骤详情

### FL-MAP-01-01 点击"天下"Tab → 场景路由切换

> **来源**: PRD [MAP-1](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1) / UI [MAP-1-1](../../../games/three-kingdoms/ui-design/ui-layout/MAP-world-v2.md#map-1)

**用户感知**: 看到 底部导航栏"天下"图标 → 操作 点击 → 反馈 当前场景淡出，天下场景淡入

**系统行为**:
1. 检查是否有持久化的视口状态（localStorage，24h有效）
2. 若有有效状态 → 直接恢复视口（无加载动画），转至 FL-MAP-01-05
3. 若无有效状态 → 触发地图数据加载（FL-MAP-01-02）

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 玩家未选择阵营 | 阻止进入，弹出"请先选择阵营"提示 | — |
| 视口状态过期（>24h） | 清除缓存，使用默认视口（己方核心领土中心 + 默认缩放） | FL-MAP-01-02 |

---

### FL-MAP-01-02 地图数据加载（含骨架屏反馈）

> **来源**: 旧流程 MAP-F01 步骤2a（M-13修复）

**用户感知**: 看到 全屏地图轮廓骨架屏 + "天下加载中..."文字 + 进度条（含百分比数字，如"加载中 67%"）+ 预计剩余时间（如"预计还需2秒"） → 操作 等待 → 反馈 骨架屏渐进填充（每批10%进度条），完成后骨架屏淡出显示完整地图

**系统行为**:
1. 从IndexedDB读取地图配置数据（地形数据+城市坐标+道路网络+阵营归属）
2. 地形数据分批返回，每批渲染后更新进度条
3. 全部数据加载完成 → 骨架屏淡出 → 进入渲染初始化

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 数据解析失败 | 显示错误面板："加载失败，请重试" + "重试"按钮 + "返回"按钮 | FL-MAP-01-02（重试）或退回前一Tab |
| 本地加载超时15s | **[P1-R2-VERIFY-01]** 弹出提示"数据加载较慢（已加载 XX%），继续等待？" + "继续等待"按钮 + "取消"按钮 | 保持等待或退回前一Tab |

### FL-MAP-01-04 初始视角定位 → 己方领土区域

> **来源**: 旧流程 MAP-F01 步骤4

**用户感知**: 看到 地图从空白/默认位置平滑移动到己方核心领土区域（500ms动画）

**系统行为**:
1. 根据玩家阵营确定核心领土中心坐标：
   - **核心领土中心（P0-01/P1-18）**: 阵营初始出生城市坐标（魏→邺城(80,40)，蜀→成都(40,170)，吴→建业(220,160)）。若阵营初始城市已被敌方占领，降级为所有己方领土的质心（centroid）。坐标使用色块坐标系（320x240）
   - 阵营初始城市定义详见 PRD MAP-2 城市坐标表
2. 计算视口中心使核心领土居中
3. 执行平滑移动+缩放动画

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 己方无领土（极端异常） | 定位到阵营初始出生城市 | FL-MAP-01-05 |

6. 挂载产出概览面板入口按钮
7. 检测是否触发新手引导（→ FL-MAP-01-SP01子流程）

## 子流程: FL-MAP-01-SP01 新手引导

> **触发条件**: `player.tutorialCompleted === false` 且账号等级 ≤ 5
> **结束条件**: 4步引导全部完成，或玩家跳过引导
> **编号说明（P0-V09修复）**: 子流程使用 SP01 后缀避免与主步骤编号（FL-MAP-01-01~05）冲突

### 引导步骤流转

```
步骤1(初征引导) → 步骤2(征服引导) → 步骤3(攻城引导) → 步骤4(布防引导) → 完成
    ↓(跳过)           ↓(跳过)           ↓(跳过)           ↓(跳过)
  确认弹窗 → 跳过    确认弹窗 → 跳过    确认弹窗 → 跳过    确认弹窗 → 跳过
```

### 引导4步详情

| 步骤 | 高亮目标 | 引导文案 | 完成条件 |
|------|---------|---------|---------|
| 1 | 最近的中立领土（视口外时自动平移至目标） | "这是你的天下！点击这块领土查看详情" | 玩家点击目标领土 |
| 2 | 详情面板"征服"按钮 | "点击征服占领这块领土" | 征服成功 |
| 3 | 敌方城池 | "这是一座敌方城池，需要攻城才能占领" | 进入攻城确认弹窗（点击"攻占"按钮） |
| 4 | 己方领土兵力滑块 | "点击你的领土驻防兵力" → "调整兵力保护领土" | 滑块移动后点击"确认"按钮提交 |

### 关键行为

- **"最近的可征服领土"查找算法**: 遍历非己方领土 → 筛选与己方相邻的 → 按距离核心领土中心距离升序排序 → 取第一个
- **"核心领土中心"定义（P0-01/P1-18）**: 阵营初始出生城市坐标（魏→邺城(80,40)，蜀→成都(40,170)，吴→建业(220,160)）。若阵营初始城市已被敌方占领，降级为所有己方领土的质心（centroid = avg(所有己方领土x坐标), avg(所有己方领土y坐标)）。若玩家无任何领土，使用阵营初始城市坐标
- **步骤1视口自动平移（P0-01）**: 若引导目标领土不在当前视口内，自动平移视口至目标领土（500ms平滑动画），确保目标在视口中央。平移完成后再高亮目标领土
- **步骤1无可征服领土处理（P0-01）**: 若不存在相邻可征服领土，显示中文提示"暂无可征服领土，请稍后再试"，并提供"跳过教程"按钮。同时设置30s自动跳过计时器，超时后自动跳过当前步骤进入下一步
- **步骤2征服失败处理（P0-02）**: 征服失败时的完整处理链路：
  1. 显示鼓励文案"别灰心，再试一次！"（2s）
  2. 自动重试，最多3次
  3. 3次均失败后弹出选择对话框："继续尝试" / "跳过此步骤"
  4. 选择"跳过"时进入步骤3（攻城引导）
  5. 选择"继续"时再给3次机会，再次用尽后强制跳过
  6. 失败原因分类提示：兵力不足→"先增强兵力再试试"；策略不当→"尝试换个策略"
- **步骤2新手难度调整（ISS-002）**: 新手引导中的征服战斗始终使用降低难度（敌方兵力强度 x 0.5），确保新手体验
- **步骤3攻城引导深度（P1-02）**: 新手引导步骤3仅要求进入攻城确认弹窗（点击"攻占"按钮），不要求完成完整攻城流程（FL-MAP-09 的10个阶段）。引导进入确认弹窗后显示简短说明"完整攻城包含编队、策略等步骤，稍后可自行体验"，然后自动完成本步骤。这样既让新手了解攻城入口，又避免复杂度劝退
- **[P1-M-02] 步骤3敌方城池查找算法**: 筛选非己方城池类领土 → 按距离核心领土中心距离升序排序 → 取第一个（教程难度降低：城防值×0.5）。若无敌方城池（极端情况），跳过步骤3自动进入步骤4
- **[P2-R2-U02] 步骤3操作细节**: 引导气泡文案为"点击这座敌方城池查看详情"，引导箭头指向目标城池。用户点击城池弹出详情面板后，引导箭头移至面板中的"攻占"按钮。
- **步骤3敌方城池查找算法（P1-R2-01）**:
  1. 遍历所有非己方阵营的城池型领土
  2. 筛选与己方领土相邻的敌方城池（adjacentIds 包含己方领土ID）
  3. 按色块坐标欧几里得距离升序排序
  4. 取第一个作为引导目标，且降低教程难度（城防值x0.5）
  5. 降级: 若无相邻敌方城池，跳过步骤3，引导直接进入步骤4（驻防引导），并在引导进度中标记步骤3为"当前无目标，已跳过"
- **[P1-M-08] 步骤3弹窗关闭时机**:
  1. 引导进入确认弹窗后，弹窗内顶部显示浮动提示条'这是攻城确认弹窗，完整攻城包含编队、策略等步骤，稍后可自行体验'，3s后自动消失
  2. 提示消失后自动完成本步骤并关闭弹窗
  3. 弹窗中禁用[确认]和[攻占]按钮（置灰），仅保留[关闭]按钮可用，防止新手误操作进入完整攻城流程
- **步骤4完成条件（ISS-015）**: 完成条件 = 滑块移动后点击"确认"按钮提交，仅拖拽滑块不视为完成
- **跳过确认**: 弹出"跳过引导可能影响游戏体验，确认跳过？"对话框
- **中断恢复**: 检测localStorage `tutorial_step`，从上次中断步骤继续
- **完成标记**: 本地存储写入（IndexedDB）`player.tutorialCompleted: true`，清除localStorage引导状态
- **新手奖励**: 详见 PRD 奖励配置

> 📖 参考: [PRD MAP-1 新手引导奖励](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

### 异常分支

| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 无相邻可征服领土 | 提示"暂无可征服领土，请稍后再试"，显示"跳过教程"按钮，30s后自动跳过 | 30s后自动跳过或手动跳过 |

## 子流程: FL-MAP-01-SP02 产出上限机制

> **触发条件**: 每秒产出tick（TE-MAP-01），持续运行
> **结束条件**: Tab销毁时停止
> **编号说明（P0-V09修复）**: 子流程使用 SP02 后缀避免与主步骤编号（FL-MAP-01-01~05）冲突

### 产出上限检查流程

```
每秒tick → 计算当前资源量 → 与上限比较
  ├── 资源 < 上限 → 正常累加产出
  ├── 资源 ≥ 上限 → 停止累加，气泡显示"已满"
  └── 玩家消耗资源 → 产出恢复累加
```

> **产出上限公式（P1-16）**: `产出上限 = 基础上限(1000) + 领土等级 x 200 + 科技加成`。每块领土独立计算上限。基础上限1000为初始值，领土等级每提升1级增加200上限，科技加成由科技树解锁提供（具体加成值详见PRD科技系统）。总产出上限 = 各领土产出上限之和。详见 [PRD MAP-1](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

> 📖 参考: [PRD MAP-1 产出上限公式](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-1)

### 产出气泡状态

| 状态 | 显示内容 | 点击行为 |
|------|---------|---------|
| 正常（<80%） | 产出数字 | 无特殊 |
| 预警（80%~99%） | 数字/上限（XX%） | 弹出提示"即将满" |
| 已满（100%） | "已满" + 锁图标 | 弹出快捷操作菜单（首项"收取产出"） |

### 产出概览面板

- **入口**: 地图右上角产出概览按钮（定位按钮下方）
- **内容**: 列出所有己方领土产出状态，按产出比例降序（已满排最前）
- **一键收取**: 确认弹窗 → 资源上限溢出处理 → 30s冷却（冷却期间按钮置灰 + 倒计时显示"冷却中，X秒后可再次收取" + Tooltip提示冷却状态）
- **一键收取边界处理（P1-17）**: 已超上限领土跳过收取 + 提示"已达上限"；正常领土产出收取全额，若收取后总量超过上限则超上限部分100%丢弃
- **一键收取溢出处理**: 超上限部分100%丢弃。收取时先扣除当前持有量中低于上限的部分，超出部分不收取并Toast提示"资源已达上限，溢出部分未收取"。丢弃的资源不可恢复

- **[P2-M-07] 一键收取溢出Toast文案统一**: 一键收取时溢出Toast文案统一为: "资源已达上限，溢出部分未收取"
