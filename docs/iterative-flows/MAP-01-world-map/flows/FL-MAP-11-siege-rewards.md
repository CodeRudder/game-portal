# FL-MAP-11 攻城奖励领取

> **版本**: v3.4 | **最后更新**: 2026-05-05
> **归属功能**: 天下地图系统 — 攻城战后奖励结算与领取
> **触发条件**: FL-MAP-09 攻城战阶段9（攻占结果确定）
> **结束条件**: 奖励领取完成或奖励自动发放
> **前置状态**: 攻城战结算完成，pendingReward 已生成
> **后置状态**: 奖励写入玩家账户，攻城结果弹窗关闭
> **来源文档**: [PLAN.md](../PLAN.md)
> **PRD**: [PRD v2 MAP-4 攻城奖励](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)

## 关键要求与验收标准

### 功能要求
- FR-11-01: 攻城结果弹窗展示胜利/失败状态、伤亡详情、奖励预览
- FR-11-02: 奖励明细按策略倍率修正（强攻/围困/夜袭/内应各有倍率）
- FR-11-03: 奖励领取为原子事务（资源+道具+声望+称号）
- FR-11-04: 背包溢出时道具自动存入邮箱系统（7天有效期）
- FR-11-05: 内应信按概率掉落（攻城胜利后）

### 非功能要求
- NFR-11-01: 结果动画播放 胜利3s/失败2s
- NFR-11-02: 奖励写入事务耗时 < 0.5s
- NFR-11-03: 领取时本地数据写入超时5s后提示用户

### 验收标准
- AC-11-01: 攻城胜利后弹出结果弹窗，显示伤亡详情和奖励明细
- AC-11-02: 点击"领取奖励"后资源到账，Toast确认"奖励已领取"
- AC-11-03: 背包满时溢出道具存入邮箱，显示Toast提示
- AC-11-04: 奖励24h未领取自动发放到账户并邮件通知
- AC-11-05: 事务写入失败时全部回滚，提示重试

## 步骤详情

### FL-MAP-11-01 弹出攻占结果弹窗

> 📖 参考: [PRD v2 MAP-4 攻城奖励](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)

**用户感知**: 看到 攻城结果弹窗弹出 → 操作 等待 → 反馈 胜利/失败状态 + 伤亡详情 + 奖励预览

**系统行为**:
1. 弹出 SiegeResultModal，显示胜利/失败状态标识
2. 展示伤亡详情（士兵损失 + 将领受伤状态）
3. 展示奖励明细预览
4. **[V-021] [P1-R2-03]** 弹窗底部显示'请点击领取奖励，24h内未领取将自动发放至背包'及倒计时（格式: "24h后自动发放 HH:MM:SS"）

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 攻城失败 | 仅显示损失详情，无"领取奖励"按钮 | -- |

### FL-MAP-11-02 结果动画播放

> 📖 参考: [PRD v2 MAP-4](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)

**用户感知**: 看到 胜利旗帜升起/军队撤退动画 → 反馈 攻城结果直观展示

**系统行为**:
1. 胜利动画(3s): 旗帜更换 + 军队入城 + 城市名弹出
2. 失败动画(2s): 军队后退 + 损失数字弹出

### FL-MAP-11-03 奖励明细显示

> 📖 参考: [PRD v2 MAP-4 攻城奖励](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)

**用户感知**: 看到 奖励类型和数量列表 → 操作 确认奖励内容

**系统行为**:
1. 按条件显示奖励内容（首次攻占/重复攻占/攻城失败各有不同，详见PRD）
2. 奖励受策略倍率修正（强攻/围困/夜袭/内应各有倍率，详见下方 [P2-M-11] 倍率表）
3. 道具掉落按领土等级随机生成
4. 奖励暂存为 pendingReward 对象（不直接写入玩家账户）

**[P1-33] PendingReward数据结构**:
```
PendingReward {
  id: string,                         // 奖励唯一ID
  siegeTaskId: string,                // 关联的攻城任务ID
  items: [                             // 奖励项目列表
    {
      type: 'resources'|'items'|'reputation'|'title',
      itemId: string,                  // 资源/道具/称号ID
      amount: number,
      multiplier: number               // 策略倍率修正后的最终数量
    }
  ],
  strategyType: string,                // 攻城策略类型（强攻/围困/夜袭/内应）
  strategyMultiplier: number,          // 策略倍率
  expiresAt: timestamp,                // 过期时间（生成后24h）
  status: 'pending'|'claimed'|'auto_claimed'
}
```

**[P2-M-11] 策略奖励倍率表（对应 PendingReward.strategyMultiplier）** [R3]:

| 策略 | 奖励倍率 | 说明 |
|------|:-------:|------|
| 强攻 (assault) | x1.0 | 基础倍率 |
| 围困 (surround) | x1.3 | 持久战奖励 |
| 夜袭 (night_raid) | x1.5 | 高风险高回报 |
| 内应 (insider) | x2.0 | 需要道具支持 |

> 此为简化综合倍率，用于 PendingReward.strategyMultiplier。详细资源/道具分项倍率见 FL-MAP-09 Stage P9 [P2-R3-07]。

### FL-MAP-11-04 点击"领取奖励"

> 📖 参考: [PRD v2 MAP-4](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)

**用户感知**: 看到 "领取奖励"按钮 → 操作 点击 → 反馈 Toast确认

**系统行为**:
1. 检查 pendingReward 存在且未过期（24h有效期）
2. 奖励写入事务：资源 → 道具(背包) → 声望 → 称号（同一原子事务）
3. 事务失败全部回滚

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| pendingReward 不存在 | 不显示按钮 | -- |
| pendingReward 过期 | 自动发放（见超时处理） | FL-MAP-11-05 |

**[P1-13] 领取时本地数据写入异常客户端状态**:
领取时本地数据写入异常的处理链路:
1. 点击"领取奖励"后按钮立即进入加载状态（显示spinner + 文字变为"领取中..."）
2. 本地数据写入成功 → 正常流程继续
3. 5s超时 → 按钮恢复可点击状态 + Toast"数据写入异常，请重试确认"
4. 重试后确认本地数据写入状态

**[V-023] 关闭弹窗与领取中状态**: 关闭弹窗不影响后台领取，领取结果通过游戏内通知显示，或在下次打开该奖励条目时显示领取状态

**[P1-M-04] 关闭弹窗后奖励领取结果通知规格**: 玩家关闭弹窗后，奖励领取结果通过游戏内通知卡片显示：位置=屏幕顶部中央偏下，样式=资源到账卡片（显示资源类型和数量），停留=5s或玩家手动关闭，动画=从顶部滑入

### FL-MAP-11-05 领取确认

> 📖 参考: [PRD v2 MAP-4](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4)

**用户感知**: 看到 Toast "奖励已领取" → 反馈 资源/道具到账

**系统行为**:
1. 奖励发放到玩家账户
2. 背包溢出处理：溢出道具存入临时邮箱（7天有效期）

**[P1-12] 背包溢出体验**:
背包溢出时的完整用户感知链路:
1. 点击"领取奖励" → 原子事务执行
2. 资源类奖励（铜钱/粮草/声望）→ 正常写入账户，无溢出问题
3. 道具类奖励（内应信/装备等）→ 检查背包剩余格子
4. 背包未满 → 道具直接入背包
5. 背包已满 → 道具存入临时邮箱系统（7天有效期），显示Toast"背包已满，{道具名}已存入邮箱(7天内领取)"
6. 邮箱入口: 主界面右上角信件图标，点击后可查看并领取溢出道具

## 子流程 FL-MAP-11-06: 内应信掉落

> 📖 参考: [PRD v2 MAP-4](../../../games/three-kingdoms/ui-design/prd/MAP-world-prd-v2.md#map-4) | 旧流程 MAP-F08-01

### 流程步骤

1. **[系统计算]** 攻城胜利后按概率触发内应信掉落（掉落概率详见PRD）
2. **[UI反馈]** 攻城结果弹窗增加"额外战利品"区域，显示内应信
3. **[数据变更]** 内应信存入背包

### 未领取超时

- pendingReward 超过24小时未领取 → 自动发放到账户（不经过背包检查）
- 自动发放邮件通知
