## P0 修复验证

### P0-01 架构假设声明 — PASS

**核验项**: 客户端 vs 服务端架构假设是否明确声明。

**核验结果**:
1. INDEX.md 顶部新增"架构假设声明"段落，7个维度完整声明：
   - 数据存储: localStorage / IndexedDB，无服务端依赖
   - 战斗引擎: 客户端本地回合制引擎（FL-MAP-09 Stage P8）
   - 敌方实体: 本地 AI 引擎驱动，非服务端推送
   - 行军拦截: 客户端本地判定（FL-MAP-04-02 状态机）
   - 数据同步: 无在线同步，云存档为可选功能
   - 断线概念: 不存在网络断线，"断线"=App切后台/页面hidden
   - PRD来源: [PRD v2.0 MAP-1 系统概述]
2. FL-MAP-04 补充 [P0-01-R2] 敌方行军实体数据来源：AI引擎触发频率(30~60s)、兵力生成规则、寻路算法、拦截判定
3. FL-MAP-09 添加 FR-09-ARCH 纯客户端架构声明

**结论**: 架构声明完整无矛盾，编码者可明确技术栈选型。

### P0-02 编队总战力公式内联 — PASS (有P1关联)

**核验项**: 编队总战力公式是否可直接编码。

**核验结果**:
1. FL-MAP-16 S2 新增 [P0-02-R2] 完整公式：
   - `编队总战力 = 兵力战力 + 将领属性加成 + 科技加成总和`
   - 参数表: TROOP_POWER_COEFF=1.0, ATK_COEFF=10.0, DEF_COEFF=8.0, INT_COEFF=5.0
   - 示例计算已内联
2. types/map-interfaces.md 公式索引同步收录
3. FL-MAP-09 Stage P3/P8 有交叉引用指向 FL-MAP-16

**关联P1问题**: FL-MAP-09 P3 同时引用 PLAN.md 附录C 简化版 `troops x 1.0 + general.武力 x 10`，与完整版系数体系不完全一致。详见 P1-M-01。

**结论**: 公式本身可直接编码，但需统一文档引用（P1-M-01）。

### P0-03 TypeScript 接口定义 — PASS

**核验项**: 核心数据结构是否有TypeScript接口定义。

**核验结果**:
1. types/map-interfaces.md 定义30个TypeScript接口/类型

   - 根存储接口(1): MapGameState 聚合所有子系统数据

**缺失项**（不阻塞编码框架）:

**结论**: 接口定义覆盖核心数据结构的94%，可直接建立TypeScript项目骨架。

## P1 修复验证

### P1-R2-01 敌方城池查找算法 — PASS

**核验项**: FL-MAP-01 SP01 步骤3 查找算法是否完整内联。

**核验结果**: FL-MAP-01 SP01 步骤3 有完整查找算法：
1. 遍历所有非己方阵营的城池型领土
2. 筛选与己方领土相邻的敌方城池（adjacentIds 包含己方领土ID）
3. 按色块坐标欧几里得距离升序排序
4. 取第一个作为引导目标，且降低教程难度（城防值x0.5）
5. 降级: 若无相邻敌方城池，跳过步骤3，引导直接进入步骤4

**结论**: 算法步骤清晰可编码，含降级处理。

## 状态流转完整性评估

### SiegeTask 状态机

```
preparing → marching → sieging → settling → returning → completed
                |          |
           cancelled    settling(撤退/超时)
                            |
                         returning → completed
```

**验证结果**: 所有状态可达，无死锁状态。settling作为结算中间态设计合理。

**待补充**:
- cancelled终态的UI行为（是否自动关闭面板、Toast提示）— INT-R2-08
- cancelled状态下编队ForceStatus映射 — INT-R2-04

### ForceStatus 编队状态

```
ready → marching → fighting → returning → (销毁)
```

**验证结果**: settling阶段编队无映射（合理），completed阶段编队销毁（XC-002已确认）。

**待补充**:
- cancelled状态编队ForceStatus无对应行 — P1-M-03
- returning到达后销毁的原子操作序列
