## 四、优先级调整说明

### UP P0 维持 P0（2项）

1. **P0-R2-01 (攻城撤退按钮位置未定义)**: R1已提出(P1-INT-15)但R2仍未修复。用户在攻城过程中想撤退但无按钮位置定义，完全阻塞攻城撤退用户流程。维持P0。
2. **P0-R2-02 (胜利多信息叠加)**: 胜利弹窗+2个Toast同时弹出，新手面对三个同时出现的信息不知所措，严重影响首次征服体验。维持P0。

### UP P1 升降调整

1. **P1-R2-02 (行军精灵点击热区)**: UP标P1合理。移动端触摸精度低，精灵尺寸小，热区未定义可能导致无法点击行军精灵。维持P1。
2. **P1-R2-03 (攻城条件多条不满足修正引导)**: 合并DP查找算法缺失问题(P1-New-03)，两问题本质相同——新手在步骤3找不到可攻城的目标。升级为P1-M-02。
3. **P1-R2-05 (回城到达通知)**: 降为P2。Toast通知已定义，缺少离线通知和动画是体验优化。
4. **P1-R2-06 (领取中关闭弹窗通知)**: 合并到P1-M-04，维持P1。通知样式/位置/时长未定义，玩家无法确认奖励到账。
5. **P1-R2-07 (新手引导步骤3弹窗关闭)**: 维持P1。弹窗关闭时机不明确可能导致新手误操作进入编队流程。
6. **P1-R2-01 (奖励领取文案)**: 维持P1。文案可能误导用户不主动领取，导致不必要的24h等待。

### DP P1 维持 P1（5项）

1. **P1-New-01 (编队战力公式双版本)**: 两公式结果不同（张飞攻击95: C.1版=950，R2版=1660），编码者无法确定用哪套。维持P1。
2. **P1-New-02 (城防衰减公式版本冲突)**: 两个版本除数项数学结果差异巨大（Lv10城墙: 版本A=30，版本B=150）。维持P1。
3. **P1-New-03 (新手引导步骤3查找算法)**: 合并到P1-M-02。维持P1。
4. **P1-New-04 (cancelled编队清理)**: 合并到P1-M-03。状态机不完整，编队可能卡在marching状态。维持P1。
5. **P1-New-05 (山贼防御系数)**: 影响FL-MAP-12事件战斗计算正确性。维持P1。

### UP P1 降为 P2（1项）

1. **P1-R2-05 (回城到达通知)**: Toast通知和任务面板已有定义，缺少的是离线通知和到账动画。属于体验细化而非功能缺失。
