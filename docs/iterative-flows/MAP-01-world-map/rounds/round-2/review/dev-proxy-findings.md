## 状态流转完整性评估

### SiegeTask 状态机

```
preparing -> marching -> sieging -> settling -> returning -> completed
                 |          |
            cancelled    settling(撤退/超时)
                             |
                          returning -> completed
```

**验证结果**: 所有状态可达，无死锁状态。settling 作为结算中间态设计合理。

**潜在问题**:
- `cancelled` 是终态，但从 cancelled 恢复的 UI 路径未描述（是否自动关闭面板？是否显示 Toast"行军已取消"？）
- `preparing` 阶段的 marchStartTime 为可选字段，若从 preparing 直接触发取消，时间戳处理未说明

### ForceStatus 编队状态

```
ready -> marching -> fighting -> returning -> (销毁)
```

**验证结果**: settling 阶段编队无映射（合理，编队不参与结算）。completed 阶段编队销毁（XC-002 已确认）。

**潜在问题**:
- cancelled 攻城任务中的编队状态变更未定义（P1-New-04）
- returning 状态编队到达后销毁的原子操作序列未列出
