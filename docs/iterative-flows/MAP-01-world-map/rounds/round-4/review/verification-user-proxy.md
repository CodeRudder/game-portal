## 维度核验详情

### 1. 用户感知连贯性

**Phase 1 -> Phase 2 过渡**: ISS-006 修复后，过渡动画序列完整（5步），从弹窗关闭到行军精灵出发的视觉连贯性已解决。用户注意力引导通过脉冲高亮实现。

**Phase 2 -> Phase 3 过渡**: ISS-016 修复后，攻城开始通知（全屏弹窗+震动+强制视口跳转+音效）确保用户不会错过。ISS-007 修复后行军期间有全局进度指示。

**Phase 3 -> Phase 4 过渡**: ISS-018 修复后消除了闪烁加载态干扰。ISS-012 修复后失败分支差异清晰。

**结论**: 4个阶段过渡均有完整的视觉描述，用户在任何时刻都知道发生了什么。**达标**。

### 2. 操作引导完整性

每个步骤用户知道下一步做什么:
- Phase 1: 选城池 -> 点攻占 -> 校验 -> 编队 -> 策略 -> 确认，每步有明确的用户感知描述
- Phase 2: 观看行军精灵（可点击查看进度/取消行军），到达前5s有倒计时Toast
- Phase 3: 观看战斗（可撤退、可浏览地图），战斗信息面板完整(ISS-017)
- Phase 4: 查看结果弹窗 -> 领取奖励 -> 观看回城

可逆性:
- Phase 1: 任何步骤可取消/返回（P3有上一步按钮、P4有上一步按钮）
- Phase 2: 可取消行军(100%退还)
- Phase 3: 可撤退(5s后，50%退还+代价预览tooltip)

**遗留**: P2->P3 返回路径中策略保留规则未定义（ISS-026, P2级别）。

**结论**: 操作引导基本完整，1项P2级别遗留。**基本达标**。

### 3. 异常场景覆盖

每个阶段至少有一个异常退出路径:

| 阶段 | 正常退出 | 异常退出 |
|------|---------|---------|
| Phase 1 | 任务创建成功 | 条件校验失败(ERR-S01~S05)、取消/返回 |
| Phase 2 | 行军到达目标 | 取消行军(ERR-S06,100%退还)、崩溃/切后台恢复(ERR-S07~S08)、恢复超时(ERR-S09,50%退还) |
| Phase 3 | 战斗结束(胜/败) | 主动撤退(ERR-S12,50%退还)、崩溃/切后台恢复(ERR-S13)、恢复超时(ERR-S14,50%退还)、战斗异常(ERR-S15,100%退还) |
| Phase 4 | 回城到达+编队销毁 | 奖励领取失败(ERR-S16,重试)、背包溢出(ERR-S17,存邮箱)、崩溃恢复(ERR-S18)、无己方城市(ERR-S19,解散) |

ERR-S01~S20 共20个异常场景覆盖全部4个阶段，每个异常有明确处理策略、退款规则、冷却规则。

**结论**: **达标**。

### 4. 移动端适配

- **恢复超时**: ISS-008 修复后移动端30秒/PC端10秒，来电场景通过pagehide事件暂停倒计时。
- **切后台**: Phase 2 行军继续(本地持久化)、Phase 3 战斗暂停(恢复遮罩+30s倒计时)。
- **移动端向导**: Stage P2 定义了移动端4步向导模式（每步全屏展示）。
- **震动反馈**: 攻城开始(ISS-016)和攻城结束均有震动反馈(100ms)。
- **暂停功能**: ISS-020 确认不实现，记录Backlog。30秒恢复窗口对大多数场景足够。

**结论**: **达标**。

### 5. 新手友好性

- **城池区分**: 城池所有权视觉区分（颜色/旗帜）引用 FL-MAP-03 渲染规则，L1/L2 未详细描述。ISS-027(P2) 已记录。
- **策略推荐**: 仅引用PRD，无流程文档内推荐逻辑。ISS-027(P2) 已记录。
- **编队推荐**: 无推荐编队功能描述。ISS-027(P2) 已记录。
- **首次引导**: 无逐步引导描述。ISS-032(P3) 已记录。
- **教程保护**: Stage P3 定义了教程将领始终可用的保护机制。

**结论**: 首次攻城用户能在"看到->操作->反馈"三段式描述引导下走完主流程，但策略选择和编队分配环节缺乏推荐逻辑，新手可能困惑。已记录为P2/P3级别问题。**部分达标**。
