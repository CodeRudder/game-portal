## 评审结论

攻城主流程在**系统行为**层面描述较为完善（状态机、公式、异常处理），但在**用户感知**层面存在明显缺口：

1. **视觉感知不完整**: 多处关键UI元素的样式、位置、布局未描述（弹窗布局、覆盖层参数矛盾、移动端适配细节）。
2. **过渡衔接断裂**: 4个阶段之间的过渡几乎都是"自动触发"，用户注意力如何从上一阶段自然过渡到下一阶段缺乏设计（特别是 Phase 1→2 和 Phase 3→4）。
3. **被动等待过长**: Phase 2 行军(10~60s)和 Phase 3 战斗(10~60s)用户几乎没有主动操作，缺乏参与感和进度反馈。
4. **移动端体验风险高**: 切后台恢复10秒超时机制对移动端用户过于严苛，可能导致无辜惩罚。
5. **新手引导缺失**: 整个流程没有针对新手的引导机制，策略选择、编队分配等环节新手可能困惑。
