### ISS-001 [公式矛盾]: 编队总战力公式在 FL-MAP-09 与 FL-MAP-16 之间结构矛盾

- **来源**: Dev + Integrator
- **流程步骤**: FL-MAP-09 Stage P3 / FL-MAP-16 S2
- **问题分类**: 矛盾
- **问题描述**: FL-MAP-09 P3 使用乘法公式 `兵力 x (武力x3+统率x2+智力)/300`，FL-MAP-16 S2 使用加法公式 `兵力x1.0 + 攻击x10 + 防御x8 + 智力x5 + 科技加成`。同参数计算结果差12%（张飞3000兵: 4150 vs 4660）。城防衰减公式引用的"攻方总战力"未指定使用哪个版本。
- **影响范围**: FL-MAP-09 Stage P3/P8, FL-MAP-16 S2, 城防衰减公式链
- **建议修正**: 明确FL-MAP-16 S2加法公式为唯一权威来源，FL-MAP-09 P3标注为"简化预览版，仅供UI展示，实际计算使用FL-MAP-16公式"。在城防衰减公式中引用FL-MAP-16公式版本号。
- **优先级**: P0

### ISS-002 [公式矛盾]: 城防初始值公式存在两套完全不同的版本

- **来源**: Dev + Integrator
- **流程步骤**: FL-MAP-09 Stage P8 / 关键公式索引 [V-029]

- **影响范围**: FL-MAP-09 Stage P8, 城防衰减公式链, 攻城平衡性
- **建议修正**: 选定一个权威公式（建议Stage P8内联版本，更完整），在[V-029]处标注`[已废弃，以Stage P8内联公式为准]`。在L1主流程Phase 3中同步更新。
- **优先级**: P0

### ISS-004 [公式矛盾]: 城防衰减公式分母结构在两处完全不同

- **来源**: Dev + Integrator
- **流程步骤**: FL-MAP-09 AC-09-01 / PLAN.md C.4
- **问题分类**: 矛盾
- **问题描述**: AC-09-01: 分母 = `(1.0 + 城墙等级x0.05) x 回合数上限`（城墙3=23）; PLAN.md C.4: 分母 = `城墙等级x5 + 100`（城墙3=115）。差异约5倍。AC-09-01版本使得20回合内几乎必然攻下，C.4版本城墙等级高时很难攻下。这影响游戏平衡的根本设计。
- **影响范围**: FL-MAP-09 Stage P8, 攻城平衡性核心
- **建议修正**: 以AC-09-01为权威来源（文档多次声明），在PLAN.md C.4处标注`[已废弃]`。验证AC-09-01分母的游戏平衡性：城墙等级10时分母=30，20回合总伤害=战力x20/30=0.67x战力，确认这是否为设计意图。
- **优先级**: P0

### ISS-008 [体验问题]: 移动端切后台恢复10秒超时=强制撤退，惩罚过重

- **来源**: User + Integrator
- **流程步骤**: FL-MAP-09 Stage P8 / Phase 2异常
