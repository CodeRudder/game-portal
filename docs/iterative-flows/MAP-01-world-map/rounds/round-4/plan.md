# Round 4 计划

> **日期**: 2026-05-05
> **目标**: 基于用户提供的攻城主流程线性链，创建/优化 L1 攻城主流程文档，补全流程细节

## 背景

R3 已完成（P0=0, P1=0, 10维度100%达标, 可实施性98%+）。现有 FL-MAP-09 攻城战文档（v6.3）已包含详细18步生命周期和10阶段展开，但缺少独立 L1 主流程文件（全貌视图）。

## 用户提供的攻城主流程线性链

```
选择城池/资源点 → 点击攻占 → 攻占弹框 → 选择/创建编队 → 创建攻占任务 →
显示攻占任务编队/时间/状态 → 地图显示行军精灵 → 点击行军精灵或攻占任务编队 →
显示行军线路 → 等待移动到目标附近(10s~60s) → 切换为攻占动画(精灵+粒子+城池) →
等待攻占过程(按回合掉血,10s~60s) → 攻占结果动画 → 攻占结果结算/增加/扣减 →
生成攻占结果事件(可点击查看详情) → 编队状态更新(血条/轻伤/重伤) →
自动回城 → 军队精灵按线路移动回出发城市(10s~60s)
```

## R4 具体目标

### 1. 创建 L1 攻城主流程文件
- 文件: `flows/FL-MAP-09-攻城主流程.md`
- 按用户线性链拆分为4个阶段: 攻占准备 → 行军推进 → 攻城战斗 → 结算回城
- 每个阶段明确: 进入条件、阶段目标、对应子流程引用、退出条件、异常跳转
- 提供全貌视图，不展开子流程细节

### 2. 核验现有子流程覆盖完整性
- 对照用户线性链18步，检查 FL-MAP-09 的 P1~P10 阶段是否完整覆盖
- 检查关联流程 (FL-MAP-04/05/06/07/11/16/17) 的引用一致性

### 3. 三代理评审
- 重点评审 L1 主流程的阶段划分、流转条件、异常跳转
- 评审现有子流程与新 L1 主流程的映射一致性
