## 详细发现

### D-01: FL-MAP-09 城防衰减存在两套矛盾公式

**流程**: FL-MAP-09 Stage P8 / FL-MAP-09-13

Stage P8定义了两套城防衰减公式:

**回合制公式**:
```
每回合城防削减 = 攻方总战力 x 策略衰减修正 x (1 - 城防系数) / 回合数上限
策略衰减修正: 强攻 x1.0 / 围困 x0.6 / 夜袭 x1.3 / 内应 x1.5
攻方战力 = SUM(将领(武x3+统x2+智x1) x 等级系数) + 兵力 x 1.0
```

**每秒公式**:
```
城防衰减/秒 = (攻方战力 x 策略修正 x 科技加成) / 城防系数
策略修正: 强攻 1.5 / 围困 0.8 / 夜袭 1.2 / 内应 1.3
攻方战力 = 将领属性总和 + 兵力 x 0.5
城防系数 = 100 (基础除数常数)
```

**问题**: (1) 策略修正系数不一致（回合制: 强攻1.0/内应1.5, 每秒: 强攻1.5/内应1.3）;(2) 攻方战力计算不一致（回合制含编队战力公式, 每秒简化为将领属性+兵力x0.5）; (3) 城防系数定义完全不同（回合制=城墙等级x0.05, 每秒=100常数）。

**影响**: 无法确定实际实现应使用哪套公式。如果两套用于不同场景，需要明确转换关系。

### D-02: FL-MAP-07 征服失败兵力损失公式自相矛盾

**流程**: FL-MAP-07 S7 / S9

S7战斗结算定义:
```
失败: 损失 = 出征兵力 x (1 - 胜率)，最低损失 50 兵力
```

S9失败结果处理定义:
```
损失出征兵力 x 30%，剩余兵力归还全局兵力池
```

当胜率为70%时，S7公式计算损失=出征兵力x30%，恰好与S9一致。但胜率为其他值时两个公式会产生不同结果。例如胜率50%时，S7损失50%，S9损失30%。

**影响**: 实现时不知道以哪个为准，会导致战斗结果不一致。

### D-04: FL-MAP-09与FL-MAP-17伤亡档位定义不一致

**流程**: FL-MAP-09-14 / FL-MAP-17 S1

FL-MAP-09-14定义了5档伤亡:
| 结果 | 判定条件 | 士兵损失 |
|------|---------|---------|
| 大胜 | T < 20s | 10%~20% |
| 胜利 | 20s <= T < 30s | 20%~30% |
| 险胜 | 30s <= T <= 60s | 30%~40% |
| 失败 | 60s超时, 城防<30% | 40%~70% |
| 惨败 | 60s超时, 城防>=30% | 80%+ |

FL-MAP-17-S1定义了3档:
| 结果 | 损失比例 |
|------|---------|
| 胜利 | 5%~15% |
| 失败 | 20%~40% |
| 惨败 | 50%~80% |

两者在"胜利"档位的损失范围完全不重叠（09: 10%~40%, 17: 5%~15%），且09的5档无法直接映射到17的3档。

**影响**: 实现伤亡系统时无法确定使用哪套档位定义，需要明确是否FL-MAP-17仅适用于征服模式，FL-MAP-09-14仅适用于攻城模式。
