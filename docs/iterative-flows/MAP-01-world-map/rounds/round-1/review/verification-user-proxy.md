# User Proxy 核验报告 (Round 1 Verification)

> **核验人**: User Proxy (Simulated End-User Walkthrough)
> **日期**: 2026-05-05
> **核验范围**: FL-MAP-01 ~ FL-MAP-18 (18 个流程文档)
> **核验方法**: 模拟最终用户视角，逐步走读每个流程，识别困惑点、卡住点和缺失引导

---

## 核验结论

- 阻塞问题(P0): **0**
- 体验问题(P1): **8**
- 细化不足(P2): **12**
- 优化建议(P3): **10**

**总体评价**: 18个流程文档整体结构完整、逻辑连贯。每个步骤均包含"用户感知/操作/反馈"三要素，异常分支覆盖全面。P0级别的阻塞问题为0。存在少量P1体验问题集中在：(1)部分流程间的衔接缺乏用户视角的过渡说明；(2)移动端部分触摸交互的反馈描述不够具体；(3)少数长时间等待场景缺少用户安抚信息。

### P3 - 优化建议 (10项)

| # | 流程步骤 | 用户角色 | 问题 | 缺失描述 | 建议 |
|---|---------|---------|------|---------|------|
| P3-01 | FL-MAP-01-05 图例组件 | 新进入用户 | 图例首次进入自动展开3s后收起，3秒可能不足以让用户看清所有图例内容 | 图例展示时间过短，用户可能来不及阅读 | 建议首次展开5s，或添加"我知道了"按钮手动关闭 |
| P3-02 | FL-MAP-02-03 缩放操作 | 新用户 | 缩放联动UI"产出气泡消失仅显示小圆点" — 但小圆点在不同缩放级别的具体表现形式（大小、颜色、间距）未详细描述 | 小圆点在不同缩放级别的视觉规格不够具体 | 建议在PRD引用外补充至少3个缩放级别的UI元素对照描述 |
| P3-03 | FL-MAP-05-02 攻城过程 | 攻城玩家 | 攻城进度条"预计剩余时间"是基于回合制计算，但随战斗进行预估时间可能大幅波动，导致用户体验不佳 | 预计剩余时间波动大可能引起用户困惑 | 建议使用"回合 X/20"替代"预计剩余时间"，更准确且不引起误解 |
| P3-04 | FL-MAP-07 S3 兵力部署 | 征服玩家 | 兵力滑块"初始位置=最低出征兵力" — 但未说明是否显示推荐兵力值 | 缺乏推荐兵力标记，用户需要自己摸索最优兵力 | 建议在滑块上标记"推荐兵力"区间（如高亮80%胜率对应的兵力范围） |
| P3-05 | FL-MAP-08-01 领土放弃 | 放弃领土玩家 | 放弃领土"仅回收部分兵力" — 但未在确认弹窗中显示具体回收比例/数量 | 损失预览中缺乏具体回收数值 | 确认弹窗应显示"当前驻防 X 兵力，回收 Y (Z%)，损失 W" |
| P3-06 | FL-MAP-09 Stage P8 攻城战斗 | 攻城玩家 | 攻城动画期间"不允许: 发起新攻城、移动到其他城市、修改策略" — 但未说明这些操作被阻止时用户会看到什么反馈 | 缺乏操作被阻止时的反馈描述 | 被阻止操作时应显示Toast"攻城进行中，无法执行此操作" |
| P3-07 | FL-MAP-10 热力图颜色 | 使用热力图用户 | 热力图5档颜色中，Top 20%为红色（#FF4444）而80~100%为绿色（#44AA44），红色通常代表"差"而此处代表"收益高"，可能引起认知冲突 | 颜色语义与传统认知（红=差/绿=好）相反，热力图中红=高收益 | 建议在热力图开启时显示一次性说明气泡"颜色越红代表收益越高"，或改用冷暖色梯度 |
| P3-08 | FL-MAP-13 地图统计 | 统计查看用户 | 统计面板"数据更新于 X 分钟前" — 但未说明数据刷新的触发条件 | 缺乏手动刷新数据的操作入口 | 建议在面板右上角增加[刷新]按钮 |
| P3-09 | FL-MAP-15 S5 快速处理 | 回归玩家 | "快速处理 = 80%奖励" — 用户可能不理解为什么只能拿80%，缺乏合理性解释 | 缺乏快速处理奖励折损的游戏内合理性解释 | 建议增加说明文案"快速处理由AI代为决策，效率略低但节省时间" |
| P3-10 | FL-MAP-18 S6 底部按钮栏 | 移动端用户 | Bottom Sheet展开时底部栏隐藏 — 用户在Sheet全屏状态下可能忘记如何返回地图 | 缺乏Bottom Sheet全屏时的返回地图操作说明 | 建议Bottom Sheet全屏状态下在顶部增加"返回地图"按钮或下拉关闭提示 |

## 逐流程核验摘要

### FL-MAP-01 进入天下Tab
- **视觉感知**: 每步有明确"看到"描述。骨架屏+进度条+预计剩余时间完整。
- **操作引导**: 点击Tab→等待加载→视角定位→UI就绪→新手引导，链路清晰。
- **反馈完整性**: 骨架屏渐进填充、阵营色脉冲边框、产出气泡三色状态、图例自动展开。
- **错误恢复**: Canvas降级、数据解析失败、加载超时均有恢复路径。
- **移动端**: 无特殊移动端问题（通用适配由FL-MAP-18覆盖）。
- **发现**: P2-01（英文鼓励文案）、P2-02（英文无可征服提示）。

### FL-MAP-02 地图浏览与缩放
- **视觉感知**: 拖拽/缩放/小地图交互均有"看到→操作→反馈"描述。
- **操作引导**: 拖拽、双指捏合、滚轮、小地图点击/拖拽/双击，操作方式明确。
- **反馈完整性**: 帧率降级自动处理（不提示）、边界回弹、光标变化。
- **错误恢复**: 视口超出边界强制回弹、手势冲突处理。
- **移动端**: 无小地图，3种替代导航方案详细说明。
- **发现**: P2-03（展开态小地图规格缺失）。

### FL-MAP-03 像素地图渲染
- **视觉感知**: 后台渲染流程，用户间接感知（骨架屏/无缝重绘）。
- **操作引导**: 无直接用户操作（纯系统渲染流程）。
- **反馈完整性**: TERRAIN_PALETTE完整内联、交界渐变算法内联。
- **错误恢复**: 未知地形fallback、城市坐标越界跳过、视口坐标NaN重置。
- **移动端**: 城市点击热区扩展描述完整。
- **发现**: 无问题。

### FL-MAP-04 行军动画
- **视觉感知**: 黄色虚线高亮、精灵队列移动、方向切换。
- **操作引导**: 确认行军/取消行军路径清晰。
- **反馈完整性**: 行军状态机完整、拦截视觉反馈（红色闪光+屏幕震动）。
- **错误恢复**: 拦截→战斗→胜利/回城、主动撤退、超长行军处理。
- **移动端**: 边缘箭头指示器描述完整。
- **发现**: P1-03（超长行军缺乏常驻指示器）、P2-04（战略时间概念不直观）。

### FL-MAP-05 攻城动画
- **视觉感知**: 三阶段动画（集结/攻城/结果）均有完整视觉描述。
- **操作引导**: 动画不可跳过（结果动画2s），交互权限明确列出。
- **反馈完整性**: 进度条实时递减、城防血条变色。
- **错误恢复**: 音频静默降级、战斗逻辑异常强制结束、应用切后台恢复。
- **移动端**: 无差异（全屏覆盖层通用）。
- **发现**: 无问题。

### FL-MAP-06 领土选择与详情
- **视觉感知**: 高亮边框、面板弹出动画、信息展示完整。
- **操作引导**: 点击领土→面板弹出→查看详情→操作按钮分支。
- **反馈完整性**: 选中态视觉（虚线+流动动画）、面板弹出300ms动画。
- **错误恢复**: 空白点击取消选中、数据读取失败显示骨架屏+重试、移动端下拉误触弹回。
- **移动端**: Bottom Sheet+下拉关闭描述完整。
- **发现**: P2-05（搜索半径未指定）、P2-06（移动端长按时间未指定）。

### FL-MAP-07 领土征服
- **视觉感知**: 胜率条颜色编码、兵力部署界面、结果特效。
- **操作引导**: 点击征服→校验→滑块调整→确认→结算，每步有操作指引。
- **反馈完整性**: 胜率实时变化、滑块拖动实时反馈、按钮预检置灰+Tooltip。
- **错误恢复**: 不相邻高亮提示、兵力不足提示、本地写入失败重试、征服失败恢复子流程。
- **移动端**: 无特殊差异描述（通用适配）。
- **发现**: P3-04（推荐兵力标记缺失）。

### FL-MAP-08 驻防管理
- **视觉感知**: 驻防信息面板、兵力池可视化条、滑块实时反馈。
- **操作引导**: 滑块拖动→确认分配，操作简单。
- **反馈完整性**: 兵力变化实时显示"将消耗/回收{delta}兵力"、Toast确认。
- **错误恢复**: 兵力池不足滑块限制、写入失败3次重试、核心领土撤防二次确认。
- **移动端**: 无特殊差异描述。

## 流程连续性分析

### 主要用户旅程连续性

**旅程1: 首次进入 → 征服第一块领土**
```
FL-MAP-01(进入Tab) → FL-MAP-01-01(新手引导步骤1:点击领土) → FL-MAP-06(领土详情)
→ FL-MAP-01-01(新手引导步骤2:征服) → FL-MAP-07(领土征服) → FL-MAP-08(驻防管理)
```
- **连续性评价**: 良好。新手引导4步覆盖核心操作，每步有完成条件。
- **断裂风险**: 新手引导步骤2征服失败最多重试3次+跳过选项，恢复路径完整。

**旅程2: 攻城战完整生命周期**
```
FL-MAP-06(领土详情→点击攻占) → FL-MAP-09(P1~P4:选择+编队+策略)
→ FL-MAP-04(行军动画) → FL-MAP-05(攻城动画) → FL-MAP-09(P8~P10:战斗+结算+回城)
→ FL-MAP-17(伤亡计算) → FL-MAP-11(奖励领取) → FL-MAP-08(驻防管理)
```
- **连续性评价**: 优秀。FL-MAP-09 P9结算后执行序列统一声明（P1-45）明确了6步严格顺序。
- **断裂风险**: P5→P6过渡（P1-04）缺乏用户确认反馈。

**旅程3: 日常操作循环**
```
FL-MAP-01(进入Tab) → FL-MAP-02(浏览地图) → FL-MAP-10(筛选/热力图)
→ FL-MAP-06(查看领土详情) → FL-MAP-08(调整驻防) → FL-MAP-01-02(收取产出)
```
- **连续性评价**: 良好。产出上限机制独立运行，一键收取有30s冷却。
- **断裂风险**: 无。

**旅程4: 离线回归**
```
FL-MAP-15(离线变化检测) → FL-MAP-12(处理事件) → FL-MAP-02(正常浏览)
```
- **连续性评价**: 良好。回归面板三种处理方式覆盖不同用户偏好。
- **断裂风险**: 10s超时自动关闭（P1-07）可能在用户阅读中关闭面板。

## 可实施性评估

**结论: YES — 工程师可以在不提出澄清问题的情况下实现所有流程。**

**理由**:

1. **每个步骤有明确的三要素**: "用户感知→操作→反馈"格式统一，工程师可以据此确定UI状态机。
2. **异常分支表格化**: 每个流程都有异常分支汇总表，包含触发条件、处理方式、恢复步骤。
3. **关键参数内联或PRD引用**: 重要数值（如TERRAIN_PALETTE 20色完整定义、胜率公式完整内联、产出上限公式完整内联、15级逐级提升规则表完整内联）都已内联在文档中。部分次要参数（如动画具体帧数、精灵Sheet像素尺寸）通过PRD引用提供。
4. **跨流程衔接明确**: P9结算后执行序列统一声明（FL-MAP-09）确保了攻城战→伤亡→奖励→回城的严格顺序。
5. **5档/3档映射表内联**: FL-MAP-09和FL-MAP-17之间的5档→3档映射关系有完整的双向引用和映射表。
6. **移动端适配独立文档**: FL-MAP-18提供了完整的PC/移动端对照表和触摸交互映射。

**工程师实施时可能需要额外查阅PRD的项**:
- 攻城策略的详细倍率参数（FL-MAP-09 Stage P4引用PRD）
- 城市精灵Sheet具体帧数和像素尺寸（FL-MAP-03引用PRD）
- 行军精灵速度参数和时长约束算法细节（FL-MAP-04引用PRD）
- 兵力补充率和各种不变量约束的具体数值（FL-MAP-08引用PRD）

这些均为合理的"细节参数由PRD提供"设计，不影响流程的可实施性。
