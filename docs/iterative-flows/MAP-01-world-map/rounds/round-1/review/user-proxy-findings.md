## 跨流程问题

### CC-01: 流程间过渡不明确
多个流程通过"跳转至FL-MAP-XX"描述转接，但缺少过渡动画和过渡状态描述：
- FL-MAP-07征服胜利 -> FL-MAP-08驻防管理：同一面板切换还是新面板？
- FL-MAP-09 P9结算 -> FL-MAP-11奖励领取：弹窗如何过渡？
- FL-MAP-04行军到达 -> FL-MAP-05攻城动画：过渡动画0.5s（FL-MAP-09-12有描述，但FL-MAP-04未提及）

### CC-02: 网络异常体验不统一
18个流程中网络异常处理方式不一致：
- 有的用Toast提示（FL-MAP-07/08/16）
- 有的用错误面板（FL-MAP-01）
- 有的静默降级（FL-MAP-03）
- 有的说"服务端完成后客户端同步"但无客户端等待状态（FL-MAP-11）

建议统一网络异常的用户体验规范。

### CC-03: 移动端端到端体验缺乏统一描述
FL-MAP-18描述了移动端适配总体规则，但各具体流程对移动端的描述散落在各文件中。移动端用户的完整体验链路缺乏端到端文档。应创建移动端完整流程路径文档。

### CC-04: 断线恢复策略不一致
多个流程涉及断线恢复（FL-MAP-01新手引导用localStorage、FL-MAP-04/05/09攻城用服务端状态、FL-MAP-18 Step Wizard用sessionStorage），恢复策略不一致且优先级未定义。用户重新进入可能同时面临多个恢复流程。

### CC-05: 征服/攻城伤亡规则不一致
征服（FL-MAP-07）失败损失"出征兵力x30%"，攻城（FL-MAP-09）失败损失"20%~40%"（5档判定）。两套计算体系让新玩家无法预测风险。应在操作按钮旁显示预估风险信息。

### CC-06: 撤销/返回操作缺失
几乎不可逆的操作（放弃领土、解散编队、取消行军）都依赖二次确认弹窗，但缺少"最近操作"历史或"撤销"功能。高频操作场景下误操作恢复成本高。

### CC-07: 多任务并行通知优先级
攻城战允许多达3个并行任务（FL-MAP-09 P2），但未描述多任务同时到达时用户看到什么。通知叠加？按顺序弹出？最新任务覆盖？

### CC-08: 滑块精度不一致
同一游戏中兵力滑块在不同流程中使用不同精度：
- FL-MAP-07征服滑块：10兵力单位
- FL-MAP-08驻防滑块：未明确
- FL-MAP-16编队滑块：50兵力单位
