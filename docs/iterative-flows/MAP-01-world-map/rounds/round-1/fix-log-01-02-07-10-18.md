## 1. P0-V09: 子流程ID冲突修复

### 问题描述

在 FL-MAP-01 中，子流程ID与主步骤ID冲突：
- `FL-MAP-01-01` 同时指代"点击天下Tab场景路由切换"（主步骤）和"新手引导"（子流程）
- `FL-MAP-01-02` 同时指代"地图数据加载"（主步骤）和"产出上限机制"（子流程）

- `FL-MAP-02-01` 同时指代"地图拖拽平移"（主步骤）和"缩放约束"（子流程）

## 2. P0-V06: "可征服"快捷筛选阈值对齐

### 问题描述

FL-MAP-10-06 "可征服"快捷筛选使用阈值 `出征兵力 > 驻防兵力 x 0.5`，但 FL-MAP-07 S2 实际征服校验要求 `可用兵力 >= 目标驻防兵力 x 1.5`。两处阈值不一致（0.5x vs 1.5x），导致快捷筛选结果中的领土在实际出征时可能无法通过兵力门槛，误导用户。

### 修复方案

将 FL-MAP-10-06 快捷筛选阈值对齐至 1.5x，与 FL-MAP-07 S2 保持一致。同时在 FL-MAP-07 S2 添加交叉引用说明。

### 修改文件与位置

- FR-10-04：`兵力充足(出征兵力>驻防兵力x0.5)` -> `可用兵力>=目标驻防兵力x1.5`，附加P0-V06修复说明
- AC-10-03：更新验收标准描述，明确使用1.5x阈值
- FL-MAP-10-06 步骤详情：
  - 按钮名称改回"可征服"（此前被改为"潜在可征服"）
  - 条件内联定义：`出征兵力 > 驻防兵力 x 0.5` -> `可用兵力 >= 目标驻防兵力 x 1.5`
  - 增加"阈值对齐说明"段落，解释修复原因
  - 增加"可用兵力定义"说明
- 异常流程汇总：更新对应提示文本
- 前置/后续流程表：更新引用描述

**FL-MAP-07-territory-conquest.md** (v3.1 -> v3.2):
