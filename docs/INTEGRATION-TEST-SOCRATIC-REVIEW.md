### 1. FLOW-01 天下Tab集成测试（48用例 / 970行）

**数据真实性评分：7/10**  
**流程完整性评分：8/10**

#### 苏格拉底提问

**Q1: 测试中的初始数据从哪里来？**  
测试使用 `createSim()` 创建基础引擎实例，然后通过 `createMapSim()` 添加粮草100000/金币100000/兵力50000。这些数字远超引擎初始值（500/300/50），但通过 `setCap` 绕过了上限限制。

**Q2: 真实玩家能否复现测试中的操作序列？**  
部分能。`createSimWithBaseTerritory()` 直接调用 `territorySys.captureTerritory()` 占领领土，跳过了实际攻城战斗流程。真实玩家需要先有足够的兵力和粮草，再通过攻城战斗占领。

**Q3: 测试是否验证了操作的前置条件？**  
✅ **做得很好**。攻城条件校验覆盖了：
- 相邻己方领土才可攻击（FLOW-01-15）
- 兵力不足时拒绝（FLOW-01-16）
- 粮草不足时拒绝（FLOW-01-17）
- 满级领土升级失败（FLOW-01-14）
- 非己方领土升级失败（FLOW-01-13）

**Q4: 测试是否验证了操作后的完整状态变化？**  
✅ 攻城胜利后验证了领土归属变更（FLOW-01-18），失败后验证归属不变（FLOW-01-19）。每日攻城次数限制（FLOW-01-37）和冷却期（FLOW-01-38）也有覆盖。

**Q5: 边界情况覆盖如何？**  
✅ 覆盖了攻城失败重试、每日3次限制、24h冷却期、Tab切换状态保持、领土筛选等。

#### 发现的问题

| 编号 | 严重度 | 描述 |
|------|--------|------|
| F01-P2-01 | P2 | `makeTestTerritories()` 使用硬编码的 mock 数据（`baseProduction: { grain: 5, gold: 5 }`），与引擎真实领土配置可能不一致 |
| F01-P2-02 | P2 | `createMapSim()` 设置粮草100000远超初始上限2000，通过 `setCap` 绕过，真实玩家不可能在初始阶段拥有这么多资源 |
| F01-P3-01 | P3 | `createSimWithBaseTerritory()` 直接调用 `captureTerritory()` 跳过战斗，未测试"攻占→升级"的完整链路 |

#### 改进建议
1. 将 `makeTestTerritories()` 的数据源改为从引擎配置中读取
2. 添加"从零开始攻占第一块领土"的完整流程测试
3. 考虑使用 `initBeginnerState()` 或 `initMidGameState()` 作为基础状态
