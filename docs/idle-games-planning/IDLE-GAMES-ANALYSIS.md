## 一、放置游戏（Idle/Incremental Games）概述

### 1.1 什么是放置游戏

放置游戏（Idle Games / Incremental Games）是一种以**资源自动积累**和**指数级成长**为核心机制的游戏类型。玩家通过点击、升级、自动化等操作，在在线和离线状态下持续获得进展。

### 1.2 放置游戏的核心吸引力

| 吸引力维度 | 描述 |
|-----------|------|
| **低压力** | 不需要持续操作，碎片化时间友好 |
| **持续进步** | 离线也能获得收益，回归有惊喜感 |
| **成长反馈** | 数字膨胀带来强烈的成就感 |
| **策略深度** | 资源分配、升级路线、Prestige 重置等决策 |
| **叙事发现** | 渐进式解锁新内容和故事 |

### 1.3 放置游戏七大核心机制

1. **线性/点击型（Linear/Clicker）** — Cookie Clicker、AdVenture Capitalist
2. **街机型（Arcade Idle）** — Idle Vlogger、Hypermarket
3. **合成型（Merge Idle）** — Merge Mansion、Merge Dragons
4. **模拟型（Idle Simulation）** — Kittens Game、Universal Paperclips
5. **经营型（Idle Management）** — Idle Miner Tycoon、Game Dev Tycoon
6. **叙事驱动型（Story-driven）** — A Dark Room、Candy Box
7. **RPG 型（Idle RPG）** — Clicker Heroes、AFK Arena、Melvor Idle

### 5.2 离线收益算法

```typescript
function calculateOfflineEarnings(
  baseRate: number,
  multiplier: number,
  offlineSeconds: number,
  offlineEfficiency: number,  // 0~1, 通常0.5~0.8
  maxOfflineHours: number
): number {
  const cappedSeconds = Math.min(offlineSeconds, maxOfflineHours * 3600);
  return baseRate * multiplier * cappedSeconds * offlineEfficiency;
}
```

### 5.3 数值平衡框架

```
资源产出: rate = baseRate × (1 + sum(upgradeBonuses)) × prestigeMultiplier
升级费用: cost = baseCost × (growthRate ^ level)
Prestige:  currency = floor(sqrt(totalEarnings / prestigeDivisor))

推荐数值范围:
  前期(0~1h):  1~10/s    中期(1~24h):  100~10K/s
  后期(1~7d):  10K~1M/s  大后期(7d+):  1e6~1e15/s
```

### 6.2 Canvas 自适应渲染

```typescript
class AdaptiveRenderer {
  resize() {
    const dpr = window.devicePixelRatio || 1;
    const rect = this.canvas.getBoundingClientRect();
    const scale = Math.min(
      rect.width / this.designWidth,
      rect.height / this.designHeight
    );
    this.canvas.width = rect.width * dpr * this.qualityScale;
    this.canvas.height = rect.height * dpr * this.qualityScale;
    this.ctx.scale(scale * dpr * this.qualityScale, scale * dpr * this.qualityScale);
  }
}
```

## 八、总结与建议

### 7.1 开发优先级建议

1. **P0 — 核心引擎**: 通用放置游戏引擎（IdleGameEngine）+ 渲染器 + 自适应系统
2. **P1 — 首批游戏**: 选择题材差异大、技术代表性强的 5 款先开发
3. **P2 — UI框架**: 自适应 UI 框架和画质切换系统
4. **P3 — 批量开发**: 基于引擎和 UI 框架批量开发剩余游戏
