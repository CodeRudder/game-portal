## 六、技术架构（现有）

### 6.1 引擎基类 GameEngine

所有游戏引擎继承 `GameEngine` 抽象基类，实现以下方法：

```
GameEngine (Abstract Base)
├── 生命周期: init() → start() → pause() → resume() → reset() → destroy()
├── 游戏循环: update(deltaTime) → onRender(ctx, w, h)
├── 输入处理: handleKeyDown(key) / handleKeyUp(key)
├── 状态管理: getState() → Record<string, unknown>
├── 事件系统: on/off/emit (statusChange / scoreChange / levelChange)
└── 子类实现: onInit / onStart / update / onRender / handleKeyDown / handleKeyUp / getState
```

### 6.2 集成注册点（每款新游戏需修改 4 处）

1. **`src/types/index.ts`** — 添加 `GameType` 枚举值
2. **`src/games/<game-id>/`** — 创建引擎目录 + Engine 类
3. **`src/components/GameContainer.tsx`** — import + createEngine switch case
4. **`src/components/GameCard.tsx`** — gameInfo 对象添加卡片信息
5. **`src/pages/GamePage.tsx`** — VALID_TYPES 路由映射
6. **`src/pages/HomePage.tsx`** — GAMES 数组添加
7. **`src/services/StorageService.ts`** — GAME_META 添加元信息

## 七、开发流程规范

### 单款游戏开发 SOP

```
1. 类型注册 — GameType 枚举 + GAME_META + gameInfo + VALID_TYPES + GAMES
2. 引擎开发 — 创建 <game-id>/ 目录，实现 Engine extends GameEngine
3. 容器集成 — GameContainer import + createEngine + 特殊交互
4. 测试编写 — Vitest 测试套件（≥ 20 用例）
5. 构建验证 — npm run build 零错误 + npm test 全通过
6. Git 提交 — feat: <game-name> 引擎 + 集成 + 测试
```
